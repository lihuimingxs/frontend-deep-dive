编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 「短重述 + 链回」 —— Bun build 内部 / Bun install 等都在 Bun 主题；本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 每个工具必标版本号 —— 构建工具领域版本演化快（Vite 半年一个 major），每个特性必标版本 + 主流工具对照（Webpack X / Rollup Y / Vite Z）
  • 作者博客 + 官方文档一手优先 —— Vite (Evan You) / Rollup (Rich Harris) / esbuild (Evan W) / SWC / Webpack (Tobias Koppers) 每个工具都是某人的"个人项目"演化出来的；优先作者解释，二手做佐证
  • 关注「实测 vs 营销」差距 —— 构建工具营销战激烈（"快 100x"等），实测往往打折；性能比较时同时引用作者声称 + 第三方 benchmark + 真实项目体验
