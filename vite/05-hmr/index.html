P5 · HMR 与 React Refresh

"改一行 React 代码不丢 state" 是 modern dev 体验核心。本章讲清 HMR 怎么真正工作 + 为什么有时丢 state。

章节数 1 主要来源 Vite HMR API · React Refresh docs · Vue HMR · Svelte HMR 状态 ✅ 完成

本阶段覆盖

  • HMR 协议层：文件变化 → 编译新版本 → WebSocket push → 浏览器 client → fetch 新模块 → accept callback
  • import.meta.hot API：模块声明怎么 accept HMR
  • HMR boundary：自我边界 / 父边界 / 走到顶 full reload
  • React Refresh：自动给每个 React 组件加 HMR boundary，保留 useState/useReducer 状态
  • Vue HMR：SFC 每个 block 独立 HMR + Vue runtime 处理 reactive state
  • Svelte HMR：编译时 instrument + 保留组件实例 state
  • 典型失败场景：修改 hook 数 / export 名变 / 顶层全局状态 / CSS-in-JS 样式名 / 第三方库副作用
