Vite/Rollup 插件 API + Unplugin

Plugin 是 bundler 的扩展机制。Rollup plugin API 是事实标准（Vite/Rolldown/Bun build 都基于或兼容）。Rollup hooks: resolveId / load / transform / buildStart / generateBundle / writeBundle 等；Vite 在 Rollup hooks 之上加: config / configResolved / configureServer / handleHotUpdate / transformIndexHtml；Plugin 顺序: pre / default / post 三档。Unplugin（Anthony Fu）让一份 plugin 代码跑在 Vite/Rollup/Webpack/esbuild/Rspack 多个 bundler 上。典型 plugin: vite-plugin-react/vue/pwa/pages / unplugin-auto-import / unplugin-icons。

阅读时长 ~30 分钟 前置 P2（Vite 架构）+ P3（ESM） 主要来源 Rollup plugin API · Vite plugin docs · Unplugin · Anthony Fu blog

01 速览

  1. Rollup plugin API 是事实标准—— Vite / Rolldown / Bun build 都基于或兼容 Rollup API。学一次跨工具用。
  2. 核心 hooks：resolveId（把 import 字符串解析到文件路径）/ load（读取文件内容）/ transform（改文件内容，编译）/ buildStart / buildEnd（生命周期）/ generateBundle / writeBundle（产物生成）。
  3. Vite 在 Rollup hooks 之上加：config / configResolved（访问 Vite 配置）/ configureServer（在 dev server 上加 middleware）/ handleHotUpdate（自定义 HMR）/ transformIndexHtml（处理 index.html）。这些是 Vite 特有功能。
  4. Plugin 顺序：pre / default / post 三档—— Vite 内置 plugin 有固定顺序；用户 plugin 默认 default。某些 plugin（如 React Refresh）必须 pre。
  5. Unplugin（Anthony Fu）—— 跨 Vite / Rollup / Webpack / esbuild / Rspack 的 plugin 抽象。让 plugin 作者维护一份代码。Vue 生态广泛使用（Anthony Fu 是 Vue 团队 + Vite 核心）。
  6. 典型 plugin 例子：vite-plugin-react / vite-plugin-vue（框架）/ vite-plugin-pwa（PWA）/ vite-plugin-pages（文件路由）/ unplugin-auto-import（自动 import）/ unplugin-icons（icon component）。
  7. 写 plugin 的常见 mistake：transform 顺序错（应该 pre 但用了 default）/ 没处理 query string（?direct / ?worker）/ sourcemap 丢失（transform 没返回 map）/ HMR 没正确处理。

02 底层逻辑

Rollup plugin 基本结构

// 最简 plugin
function myPlugin(options) {
  return {
    name: 'my-plugin',   // 必需，用于错误信息

    // 各种 hooks（按需实现）
    resolveId(source, importer) { ... },
    load(id) { ... },
    transform(code, id) { ... },
    buildStart() { ... },
    generateBundle(options, bundle) { ... }
  };
}

// 使用
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [myPlugin({ option: 'value' })]
});

Rollup hooks 详解

resolveId hook

把 import 字符串解析到文件路径。最常用于"虚拟模块"和"自定义路径解析"：

{
  name: 'virtual-module',
  resolveId(source) {
    if (source === 'virtual:my-module') {
      return source;   // 返回字符串 = 这就是 final id
    }
    return null;       // 返回 null = 让其他 plugin 处理
  }
}

典型场景：

  • 虚拟模块（如 vite-plugin-pages 把 virtual:routes 解析）
  • 自定义 alias
  • "package 没有但生成"的场景

load hook

读 id 对应的文件内容。配合 resolveId 实现虚拟模块：

{
  name: 'virtual-module',
  resolveId(source) {
    if (source === 'virtual:routes') return source;
    return null;
  },
  load(id) {
    if (id === 'virtual:routes') {
      // 返回虚拟模块内容
      return `export const routes = [
        { path: '/', component: () => import('./Home') },
        { path: '/about', component: () => import('./About') }
      ];`;
    }
    return null;   // 让 Rollup 自己读
  }
}

transform hook

改文件内容（编译 / 注入代码 / 等）。最常用 hook：

{
  name: 'inject-version',
  transform(code, id) {
    if (id.endsWith('main.tsx')) {
      return {
        code: `const __VERSION__ = '${pkgVersion}';\n${code}`,
        map: null   // 简单注入不影响 sourcemap，可以 null
      };
    }
    return null;
  }
}

关键：

  • 返回 null 或 undefined = 不修改
  • 返回 { code, map } = 修改后的代码 + sourcemap
  • map 应该正确生成（详见 P6.2）

buildStart / buildEnd hook

构建生命周期：

{
  name: 'log-build',
  buildStart() {
    this.info('Build started');   // 用 plugin context 的 info / warn / error
  },
  buildEnd(error) {
    if (error) this.error(error);
    else this.info('Build finished');
  }
}

generateBundle hook

产物生成阶段，可以修改 bundle 输出：

{
  name: 'add-banner',
  generateBundle(options, bundle) {
    for (const fileName in bundle) {
      const chunk = bundle[fileName];
      if (chunk.type === 'chunk') {
        chunk.code = `/* My Banner v1.0 */\n${chunk.code}`;
      }
    }
  }
}

writeBundle hook

产物写入磁盘后调用。用于 build 后的副作用（如 copy 资源）：

{
  name: 'copy-extra',
  writeBundle(options, bundle) {
    fs.copyFileSync('./LICENSE', `${options.dir}/LICENSE`);
  }
}

Vite 特有 hooks

Vite 在 Rollup hooks 基础上加了几个 Vite 特有 hook：

config / configResolved

{
  name: 'configure-vite',
  config(config, env) {
    // 修改 Vite 配置（合并）
    return {
      define: { __DEV__: env.mode === 'development' }
    };
  },
  configResolved(resolvedConfig) {
    // 配置已 resolve，存下来用
    this.viteConfig = resolvedConfig;
  }
}

configureServer

在 Vite dev server 上加 middleware：

{
  name: 'add-api-mock',
  configureServer(server) {
    server.middlewares.use('/api', (req, res, next) => {
      if (req.url === '/users') {
        res.setHeader('Content-Type', 'application/json');
        res.end(JSON.stringify([{ id: 1, name: 'alice' }]));
      } else {
        next();
      }
    });
  }
}

handleHotUpdate（自定义 HMR）

{
  name: 'custom-hmr',
  handleHotUpdate({ file, server, modules }) {
    if (file.endsWith('.svg')) {
      // SVG 改了，重新加载使用它的所有模块
      server.ws.send({ type: 'full-reload' });
      return [];
    }
    return modules;   // 默认行为
  }
}

transformIndexHtml

处理 index.html（注入 script / meta / 等）：

{
  name: 'inject-meta',
  transformIndexHtml(html) {
    return html.replace(
      '</head>',
      `<meta name="version" content="${VERSION}" />\n  </head>`
    );
  }
}

或返回 tag descriptor（更结构化）：

transformIndexHtml(html) {
  return [
    {
      tag: 'meta',
      attrs: { name: 'version', content: VERSION },
      injectTo: 'head'
    },
    {
      tag: 'script',
      attrs: { src: '/analytics.js' },
      injectTo: 'body-prepend'
    }
  ];
}

Plugin 顺序：pre / default / post

plugin 数组中的顺序决定 hook 执行顺序，但有 enforce 选项强制：

{
  name: 'must-run-first',
  enforce: 'pre',          // 'pre' | 'post'，省略 = default
  transform(code, id) { ... }
}

顺序：

  1. 所有 enforce: 'pre' 的 plugin（按数组顺序）
  2. Vite 内置 plugin（resolve / esbuild transform / etc.）
  3. 所有 default plugin
  4. 所有 enforce: 'post' 的 plugin

典型用法：

  • React Refresh transform 必须 'pre'（在 esbuild transform 之前）
  • Compression / minification 必须 'post'（最后处理）

plugin context

hook 函数里 this 是 plugin context，提供工具：

{
  name: 'my-plugin',
  async transform(code, id) {
    // 报错（这个文件失败）
    this.error('Something wrong', { line: 5, column: 10 });

    // 警告
    this.warn('Deprecated usage');

    // 信息
    this.info('Processing ' + id);

    // 解析依赖
    const resolved = await this.resolve('./other-file', id);
    if (resolved) console.log(resolved.id);

    // 加载文件
    const moduleInfo = await this.load({ id: resolved.id });

    // emitFile（产物中加文件）
    const refId = this.emitFile({
      type: 'asset',
      name: 'my-asset.txt',
      source: 'hello'
    });
    return `const ASSET = ${this.getFileName(refId)};`;
  }
}

Unplugin：跨 bundler 抽象

问题：plugin 作者要维护多个版本（Vite plugin / Rollup plugin / Webpack plugin / esbuild plugin）—— 重复代码。

Unplugin（Anthony Fu, 2022-）解决：定义一份 plugin，自动适配多 bundler：

// my-plugin.ts
import { createUnplugin } from 'unplugin';

export const unplugin = createUnplugin((options) => ({
  name: 'my-plugin',

  resolveId(source) {
    if (source === 'virtual:my') return source;
  },

  load(id) {
    if (id === 'virtual:my') return 'export const x = 1;';
  },

  transform(code, id) {
    if (id.endsWith('.tsx')) {
      return code.replace('FOO', 'BAR');
    }
  }
}));

// 导出多 bundler 适配
export const vitePlugin = unplugin.vite;
export const rollupPlugin = unplugin.rollup;
export const webpackPlugin = unplugin.webpack;
export const esbuildPlugin = unplugin.esbuild;
export const rspackPlugin = unplugin.rspack;

用户在自家 bundler 用：

// vite.config.js
import { vitePlugin as myPlugin } from 'my-plugin';
export default { plugins: [myPlugin()] };

// webpack.config.js
const { webpackPlugin: myPlugin } = require('my-plugin');
module.exports = { plugins: [myPlugin()] };

Unplugin 生态（2026）：

  • unplugin-auto-import（自动 import React/Vue 等）
  • unplugin-icons（icon as component）
  • unplugin-vue-components（Vue 自动注册组件）
  • unplugin-vue-router
  • 很多其他

03 应用场景

场景 A · 写自定义 SVG icon plugin

需求：import Icon from './my-icon.svg?component'，得到 React component。

// vite-plugin-svg-component.ts
import type { Plugin } from 'vite';
import fs from 'node:fs';

export default function svgComponent(): Plugin {
  return {
    name: 'svg-component',
    enforce: 'pre',

    async transform(code, id) {
      if (!id.match(/\.svg\?component$/)) return;

      // 读 SVG 文件（去掉 ?component query）
      const filePath = id.replace(/\?component$/, '');
      const svg = await fs.promises.readFile(filePath, 'utf8');

      // 生成 React component
      return `
        import React from 'react';
        export default function Icon(props) {
          return ${jsxFromSvg(svg)};   // 简化：把 svg 转成 JSX
        }
      `;
    }
  };
}

场景 B · 自动注入版本号到所有 file

import type { Plugin } from 'vite';
import { execSync } from 'node:child_process';

export default function injectVersion(): Plugin {
  const version = execSync('git rev-parse --short HEAD').toString().trim();

  return {
    name: 'inject-version',
    config() {
      return {
        define: {
          __VERSION__: JSON.stringify(version)
        }
      };
    }
  };
}

// 用户代码
console.log(__VERSION__);   // git commit hash

场景 C · 生成虚拟路由模块

// vite-plugin-routes.ts
import { glob } from 'glob';
import type { Plugin } from 'vite';

export default function routes(): Plugin {
  return {
    name: 'routes',

    resolveId(id) {
      if (id === 'virtual:routes') return id;
    },

    async load(id) {
      if (id !== 'virtual:routes') return;

      const files = await glob('src/pages/**/*.tsx');
      const routes = files.map(file => {
        const path = file.replace('src/pages', '').replace('.tsx', '');
        return `{ path: '${path}', component: () => import('${file}') }`;
      });

      return `export const routes = [${routes.join(',\n')}];`;
    }
  };
}

// 用户代码
import { routes } from 'virtual:routes';
// routes = [{ path: '/Home', component: () => import('...') }, ...]

场景 D · dev 模拟 API（mock）

import type { Plugin } from 'vite';

export default function apiMock(): Plugin {
  return {
    name: 'api-mock',

    configureServer(server) {
      server.middlewares.use('/api', (req, res, next) => {
        const url = req.url;

        if (url === '/users') {
          res.setHeader('Content-Type', 'application/json');
          res.end(JSON.stringify([{ id: 1, name: 'alice' }]));
        } else if (url?.startsWith('/posts/')) {
          const id = url.split('/').pop();
          res.setHeader('Content-Type', 'application/json');
          res.end(JSON.stringify({ id, title: 'Post ' + id }));
        } else {
          next();
        }
      });
    }
  };
}

场景 E · 用 Unplugin 写跨工具 plugin

// unplugin-strip-debug/src/index.ts
import { createUnplugin } from 'unplugin';

export interface Options {
  enabled?: boolean;
}

export const unplugin = createUnplugin<Options>((options = {}) => {
  const enabled = options.enabled ?? true;

  return {
    name: 'unplugin-strip-debug',

    transformInclude(id) {
      return id.endsWith('.ts') || id.endsWith('.tsx');
    },

    transform(code) {
      if (!enabled) return code;
      // 删除所有 debug() 调用
      return code.replace(/debug\([^)]*\);?/g, '');
    }
  };
});

export const vite = unplugin.vite;
export const rollup = unplugin.rollup;
export const webpack = unplugin.webpack;
export const esbuild = unplugin.esbuild;

场景 F · 修改 index.html（注入 analytics）

import type { Plugin } from 'vite';

export default function analytics(id: string): Plugin {
  return {
    name: 'analytics',

    transformIndexHtml(html, ctx) {
      // 仅 production 注入
      if (ctx.server) return html;

      return [
        {
          tag: 'script',
          attrs: { async: true, src: `https://www.googletagmanager.com/gtag/js?id=${id}` },
          injectTo: 'head'
        },
        {
          tag: 'script',
          children: `window.dataLayer = window.dataLayer || [];
            function gtag(){dataLayer.push(arguments);}
            gtag('js', new Date());
            gtag('config', '${id}');`,
          injectTo: 'head'
        }
      ];
    }
  };
}

04 典型代码：完整 plugin

带类型定义的 Vite plugin

// vite-plugin-banner.ts
import type { Plugin, ResolvedConfig } from 'vite';

export interface BannerOptions {
  text: string;
  files?: string | RegExp;
}

export default function banner(options: BannerOptions): Plugin {
  let config: ResolvedConfig;
  const { text, files = /\.js$/ } = options;

  return {
    name: 'banner',

    configResolved(resolvedConfig) {
      config = resolvedConfig;
    },

    generateBundle(_, bundle) {
      for (const fileName in bundle) {
        const chunk = bundle[fileName];
        if (chunk.type !== 'chunk') continue;

        const matches =
          typeof files === 'string'
            ? fileName.includes(files)
            : files.test(fileName);

        if (matches) {
          chunk.code = `/* ${text} */\n${chunk.code}`;
        }
      }
    }
  };
}

使用：

// vite.config.js
import banner from './vite-plugin-banner';

export default {
  plugins: [
    banner({ text: 'My App v1.0 (c) 2026' })
  ]
};

plugin 处理 query string（asset import）

// 处理 import logo from './logo.svg?inline'
{
  name: 'svg-loader',

  async load(id) {
    if (!id.match(/\.svg\?inline$/)) return;

    const filePath = id.replace(/\?inline$/, '');
    const svg = await fs.promises.readFile(filePath, 'utf8');
    const dataUrl = `data:image/svg+xml,${encodeURIComponent(svg)}`;
    return `export default ${JSON.stringify(dataUrl)};`;
  }
}

05 常见陷阱

陷阱 1：transform 没返回 sourcemap

详见 P6.2。修改 code 必须返回 map，否则下游 plugin 的 sourcemap 链断。

陷阱 2：忘记处理 query string

Vite 用 query 区分不同导入方式（?direct / ?worker / ?raw / ?url）。如果 plugin 不处理 query，可能错误处理或忽略：

// ❌ 没考虑 query
transform(code, id) {
  if (id.endsWith('.ts')) {   // 'foo.ts?worker' 不匹配！
    ...
  }
}

// ✅ 用 path-only 部分检查
transform(code, id) {
  const [path, query] = id.split('?');
  if (path.endsWith('.ts')) {
    ...
  }
}

陷阱 3：plugin 顺序错

典型：自定义 transform 在 esbuild transform 之后跑（应该之前）。

对策：明确设 enforce: 'pre'。

陷阱 4：dev 和 build 行为不一致

有些 plugin hook 只在 dev 调用（configureServer）；有些只 build（generateBundle）。测试两种模式：

$ vite                    # dev
$ vite build && vite preview   # build + preview

陷阱 5：在 transform 里 fetch 远程资源

// ❌ 慢
async transform(code, id) {
  const remote = await fetch('https://api.example.com/data');   // 每次 transform 都 fetch！
  return code.replace('REMOTE_DATA', JSON.stringify(remote));
}

// ✅ buildStart 只一次
async buildStart() {
  this.remoteData = await fetch('https://api.example.com/data').then(r => r.json());
},
transform(code, id) {
  return code.replace('REMOTE_DATA', JSON.stringify(this.remoteData));
}

陷阱 6：忘记 invalidate 模块（HMR 后）

Vite plugin 修改了某个文件相关的虚拟模块，要让 HMR 知道：

{
  configureServer(server) {
    fs.watch('config.json', () => {
      const mod = server.moduleGraph.getModuleById('virtual:config');
      if (mod) server.moduleGraph.invalidateModule(mod);
      server.ws.send({ type: 'full-reload' });
    });
  }
}

陷阱 7：plugin 没正确处理 SSR

Vite plugin 在 SSR 模式下行为可能不同。config hook 接收 ssr context；load 等 hook 也有 ssr 参数：

{
  load(id, options) {
    const isSSR = options?.ssr;
    if (isSSR) {
      // 服务端不同处理
    }
  }
}

陷阱 8：没 publish Unplugin 多版本

用 Unplugin 写 plugin，但 npm publish 时只导出 vite → 用户用 Webpack 时找不到。

对策：package.json exports 字段：

{
  "exports": {
    "./vite": "./dist/vite.js",
    "./rollup": "./dist/rollup.js",
    "./webpack": "./dist/webpack.js",
    "./esbuild": "./dist/esbuild.js",
    "./rspack": "./dist/rspack.js"
  }
}
