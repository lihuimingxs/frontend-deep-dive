P2 · Vite 双模架构

Vite 最颠覆的设计：dev mode 用一套实现，build mode 用另一套。dev 求"启动快 + 单文件转换快"，build 求"代码体积小 + 优化彻底"——两个目标不同，所以工具不同。这一阶段两章把它讲透。

章节数 2 主要来源 Vite 官方文档 · Evan You 演讲 · Rolldown roadmap 状态 ✅ 全部完成

本阶段覆盖

  • 核心思想：浏览器原生支持 ESM，dev mode 不打包 → 让浏览器请求每个 module，server on-demand 转换返回
  • 依赖预构建：node_modules 用 esbuild 预 bundle 成单 ESM 文件（避免几百个请求）
  • 缓存机制：.vite/ 目录 + hash invalidation
  • on-demand transform：源代码按浏览器请求逐个转换（TS / Vue SFC / JSX）
  • import 重写：bare specifier → /node_modules/.vite/deps/ 路径
  • 冷启动 < 1s + HMR < 50ms：因为 dev 不全量打包，1000 文件项目和 100 文件几乎一样快
  • Build mode 用 Rollup：因为生产需要 chunk 分包 / tree shaking / 代码压缩 / sourcemap，esbuild 做得不够好
  • Chunk 分包策略：每个 dynamic import 一个 chunk + vendor 分割 + manualChunks
  • Rolldown 接班：Rust Rollup 兼容 bundler，2026-Q2 stable，Vite 7+ 默认
