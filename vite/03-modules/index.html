P3 · ESM vs CJS 模块系统

模块系统是 bundler 的"输入 + 输出"基础。Web 标准 ESM 与 Node CJS 共存让 bundler 必须处理 interop，这是 2026 仍存在的"原罪"。

章节数 2 主要来源 ECMA-262 §16 · Node Modules docs · 各 bundler interop 文档 状态 ✅ 全部完成

本阶段覆盖

  • ESM 静态分析：import 必须顶层 + 字符串 literal → tree shaking 基础
  • ESM live binding：与 CJS 复制值不同；外部能看到后续变化
  • import.meta：每个 module 独立元数据；import.meta.url / .env
  • import attributes（2025 stage 4）：with { type: 'json' }
  • dynamic import + top-level await：运行时 import + 顶层 await
  • import maps：浏览器原生 bare specifier 解析
  • CJS 与 ESM 的 4 个不兼容点：同步/异步、复制值/live binding、动态/静态、namespace/named
  • 双 package hazard：一个包同时发 CJS 和 ESM → 状态不一致风险
  • package.json exports 字段：精细声明每个 entry 的 CJS/ESM/types
  • bundler 处理 CJS：Vite (Rollup commonjs plugin) / Webpack 原生 / esbuild 自动 interop
  • 现实场景：lodash vs lodash-es / node-fetch / Vite SSR / CJS-only 库 lock 项目
