P6 · 编译器后端

bundler 内部用什么编译器把 TS / JSX / 现代 JS → 浏览器能跑的 JS。这一阶段两章覆盖 esbuild / SWC / Babel + sourcemap 工程化。

章节数 2 主要来源 esbuild docs · swc.rs · babeljs.io · Source maps spec v3 状态 ✅ 全部完成

本阶段覆盖

  • Babel（2014-）：JS 实现 + 插件 ecosystem 最大 + 慢 100-1000ms/file
  • esbuild（2020-）：Go 实现 + 并行 + AST in memory + 极快 < 10ms/file + 不支持完整 plugin AST 操作
  • SWC（2020-）：Rust 实现 + 接近 Babel 兼容性 + 接近 esbuild 速度 + Wasm plugin
  • 三者做的工作：transform 现代 JS 到老 JS / JSX / TS / polyfill 注入
  • 性能 benchmark：100 文件 Babel ~5-10s, SWC ~0.5-1s, esbuild ~0.1-0.5s
  • Source Map v3：JSON + mappings (VLQ encoded) + sources + names
  • VLQ 编码：变长 base64 编码每行偏移；让 sourcemap 紧凑
  • composite source maps：source → TS → JS → bundled → minified 多层
  • 生产部署：inline / external / 私有 CDN + Sentry
