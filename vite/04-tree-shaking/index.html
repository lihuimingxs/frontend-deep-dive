P4 · Tree Shaking 真相

行业最被误解的特性之一。"我开了 tree shaking 怎么 bundle 还这么大"是常见困惑——答案是 tree shaking 不是开关，依赖整个生态共同配合。

章节数 1 主要来源 Rollup 文档 · Webpack tree shaking docs · esbuild 文档 · Rich Harris talks 状态 ✅ 完成

本阶段覆盖

  • Tree shaking 定义：编译时分析模块图 → 标记未使用 export → 不打包
  • 前提：模块用 ESM（CJS 不能 tree shake）
  • side effects 概念：模块加载时执行的代码（console.log / 注册 / polyfill）有副作用 → 默认保守保留
  • package.json sideEffects 字段：false（整包纯）/ 数组（指定有副作用文件）
  • 静态分析极限：动态 import 不能分析；调用未知函数保守；re-export 扩展图；destructuring import 是必要不充分
  • Rollup vs Webpack vs esbuild 实现差异：Rollup 最严最干净 / Webpack 依赖 sideEffects + Babel 标注，常假阳性 / esbuild 中等
  • 典型失效场景：默认导入整个 lodash / 第三方包不标 sideEffects / CSS 副作用 / 模块顶层 console.log / Babel 转译破坏 ESM 静态结构
  • 关键工具：/* @__PURE__ */ 注释 / rollup-plugin-visualizer / Vite build --mode development
