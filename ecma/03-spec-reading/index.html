P3 · 怎么读规范

ECMA-262 第 16 版（ES2025）正文超过 800 页，但规范不是"文档"，是"形式化伪代码"。一旦掌握它的"语法"（算法步骤怎么读、? / ! 简写什么意思、Let _x_ be 是局部变量声明），800 页规范就变成一份可以逐行 trace 执行的精确程序。本阶段一章训练这套元能力，是 P4-P7 规范精读的入门基础。

章节数 1 / 1 ✅ 对应 ECMA-262 §5 Notational Conventions + Annex B 状态 完成

本阶段覆盖

  • 规范结构 —— 29 个 clause + Annex、Normative vs Informative、Editorial vs Normative
  • Annex B 的特殊地位 —— Web 兼容性遗留（HTML 注释、.substr、正则 octal 转义）；浏览器必须实现，Node CLI 不必
  • 抽象操作命名约定 —— CamelCase 用户可见 vs 无前缀规范内部
  • 算法步骤记法 —— Let / If / Assert / ? abrupt 简写 / ! 断言不抛错
  • Stage 流程文档 —— tc39/proposals 仓库、proposal repo 模板
