Track B · 规范精读

6 章训练"读 800 页 ECMA-262 规范"的元能力。不复述语言特性（那是 JS 主题的工作），只讲规范的记法、抽象操作、内部槽、上下文模型这些"读规范的语法糖"。完成本轨后，你能直接读 V8 / SpiderMonkey 的 GitHub 讨论（贡献者用 §X.Y 编号交流），写正确的 polyfill（处理 abrupt completion），追踪 TC39 提案的 spec text 修改。

章节数 0 / 6（编写中） 来源 ECMA-262 第 15 版（ES2024）、TC39 process document 前置 Track A 完成（推荐）或扎实的 JS 工程经验

本 Track 解决什么问题

  • 规范的"语法"是什么 —— B1 教读"? Op()" / "! Op()" / "Assert:" / "Let _x_ be ..." 这些算法记法
  • 用户看不见的内部类型 —— B2 拆 Reference Record / Completion Record / Property Descriptor 这三个"规范专属类型"
  • 每个 JS 操作背后调用的抽象操作 —— B3 把 ToPrimitive、ToObject、SameValueZero、IsCallable 等核心抽象操作讲透
  • 对象的本质是 11 个内部方法 —— B4 拆 essential internal methods + Ordinary vs Exotic 区分（Proxy 的本质就在这）
  • 规范怎么描述"执行机器" —— B5 覆盖 §9 全章：Environment Record 五种、Job、Host hooks 各自的用途
  • 一个特性怎么从 Stage 0 走到 §X —— B6 选 Top-Level Await（成功）+ Records & Tuples（卡住）做 case study

阅读建议

  • 建议连读 B1 → B2 → B3：规范阅读"线性入门三件套"
  • 双子章 B4 + B5：内部槽 → 执行上下文是规范的"机器视角"，建议一起看
  • 独立跳读 B6：是 case study，不依赖前面任何 B 章
  • 不依赖 Track A：本轨完整自洽，但 Track A 的 A3（执行模型）和 B5（§9 上下文）从两个视角讲同一套机器，建议先 A3 再 B5
