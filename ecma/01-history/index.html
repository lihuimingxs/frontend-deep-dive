Track A · 历史脉络与设计

5 章按 4 个问题展开 ECMAScript 的 30 年演化：为什么存在（A1）→ 它是什么（A2）→ 机器怎么跑（A3）→ 单线程怎么并发（A4），最后一章讲现代生态与目标版本策略（A5）。这 4 问骨架是后续 JS / TS / React 主题"补充章"的统一模板。

章节数 5 / 5（A1 + A2 + A3 已完成） 来源 ECMA-262 §4 历史段、TC39 meeting notes、Brendan Eich blog、HTML 规范 前置 任意 JavaScript 经验

本 Track 解决什么问题

  • 为什么 JS 这么古怪 —— A1 + A2 共同解释：10 天写出的脚本、ES4 大爆炸式失败、原型而非类、动态弱类型的设计渊源
  • 规范层和引擎层是如何分工的 —— A3 拆开 Realm / Agent / Job / ExecutionContext，讲清楚哪些是 ECMA-262 规定的、哪些是 V8/HTML 规范实现的
  • 异步模型是怎么演化到今天的 —— A4 串起回调地狱 → Promise → async/await → `using` / SharedArrayBuffer 的演进路径
  • 团队该用什么 ES 版本作为 baseline —— A5 给出"客户端保守 / 客户端激进 / 服务端"三种打包目标版本策略

章节清单

序号 章节 核心收获
A1 标准发展史 ——「为什么存在？」 ES1-ES2025 共 16 版完整时间线、ES4 失败教训、TC39 流程改革、Stage 0-4、"标准滞后实现"是常态
A2 设计原则与语言特性 ——「它是什么？」 三大灵感来源（Self/Scheme/Java）、原型对象模型、动态弱类型、错误处理设计、迭代协议、模块系统、Symbol
A3 执行模型 ——「机器怎么跑？」 Realm / Agent / Job 三层容器、Execution Context、Environment Record 五种、strict mode、生成器暂停、Host hooks
A4 异步演进 ——「单线程怎么并发？」🚧 Event Loop 责任划分（HTML vs ECMA）、Promise 状态机、组合器、async/await、async iterator、AbortController、`using`
A5 现代生态 ——「该用什么版本？」🚧 TC39 Stage 流程详解、ES2016-ES2025 全特性表、打包器目标版本策略、Babel preset-env + core-js、Stage 3 提案快照

阅读建议

  • 建议连读 A1 → A2 → A3 → A4：4 问是一条叙事弧，分开读会泄气
  • 可独立跳读 A5：作为新人入职 onboarding，看完 A5 就能定团队 baseline
  • 产品工程师：A1 + A2 + A5 已足够日常工作；不必读 A3/A4 的规范层细节
  • 底层 / 编译器方向：A1-A4 全读 + 配合 Track B 规范精读
