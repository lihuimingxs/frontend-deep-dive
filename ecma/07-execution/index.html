P7 · §9 执行模型与并发

ECMAScript 这台抽象机器的"VM 架构图"：Realm / Agent / Job 三层容器、Execution Context 与 Environment Record 五种、Event Loop 与 30 年异步演进（callback → Promise → async/await → using）、Source Text Module Records 4 阶段加载 + Top-Level Await 死锁检测、§24 Memory Model + Atomics 多线程心智。本阶段 5 章是叙事 + 规范精读双重视角——P7.1/P7.3 偏用户视角的故事弧，P7.2/P7.4 偏规范层的算法精确度，P7.5 多线程压轴。

章节数 5 / 5 ✅ 对应 ECMA-262 §9 + §16.2 + §24 + §27 状态 全部完成

本阶段覆盖

  • 三层容器 —— Agent（线程级）/ Agent Cluster（共享 SAB 边界）/ Realm（全局环境，独立 Intrinsics）/ Job（异步执行单元）
  • 执行上下文 —— ExecutionContext 内部槽、LexicalEnvironment / VariableEnvironment / PrivateEnvironment、running execution context
  • Environment Record —— 5 种（Declarative / Function / Object / Module / Global）+ ResolveBinding 算法 + 闭包机制
  • strict mode 与生成器 —— [[Strict]] 字段、Generator 的 ExecutionContext 暂停机制（async/await 的底层）
  • Event Loop —— ECMA-262 Job 抽象 vs HTML 规范 task source、microtask vs macrotask、HostEnqueuePromiseJob
  • 异步演进 —— callback → Promise → async/await → 异步迭代器 → AbortController → SharedArrayBuffer + Atomics → ES2025 using / await using
  • 模块系统 —— Source Text Module Record 的 4 阶段算法（Parse → Load → Link → Evaluate）、cycle handling、TLA 死锁检测、Module Namespace exotic
  • 内存模型 + 多线程 —— §24 Memory Model 形式化、SharedArrayBuffer、Atomics 11 个 API、Lock-free 数据结构（SPSC ring buffer / spinlock / 双缓冲）、Spectre 缓解与 COOP/COEP
