P2 · 设计原则与灵感

ECMAScript 的"基因图谱"——为什么 JS 是"原型的 Self"而不是"类的 Java"，为什么动态弱类型 30 年都没改，为什么 let/const + TDZ 被称为"修复型决策"，为什么 Symbol 不只是"unique key"。两章把"为什么这样设计、当时有哪些选项、TC39 怎么选的"讲透。这是给"想理解 JS 古怪行为的根因"的工程师看的。

章节数 2 / 2 ✅ 对应 ECMA-262 §4 Overview + §6 / §13 / §15 / §16 状态 全部完成

本阶段覆盖

  • 三大灵感来源 —— Self（原型继承）+ Scheme（一等函数 + 词法作用域）+ Java（C 风格语法外衣）
  • 原型对象模型 —— [[Prototype]] vs prototype、OrdinaryGet 查找、class 是糖
  • 类型系统 —— 8 个原始类型、动态绑定值不绑定变量、弱类型 = 任何转换不报错
  • 作用域与闭包 —— 词法作用域、let/const + TDZ、闭包是 [[Environment]] 副产物
  • 错误处理设计 —— ES3 才加 try/catch、Error 层级、ES2022 Error.cause
  • 迭代协议 —— Symbol.iterator 统一 6 个特性、ES2025 Iterator Helpers
  • 模块系统 —— ESM 静态结构 vs CJS 运行时、live binding、TLA 妥协
  • Symbol 与 Well-known Symbols —— 协议钩子设计范本
