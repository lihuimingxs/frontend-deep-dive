P1 · Bun 是谁

不讲技术细节，只解决两个问题：Bun 怎么走到今天（2021 Jarred Sumner 个人 demo → Oven 公司化 → Bun 1.0/1.1/1.2）、它在 JS 服务端生态里站什么位置（Bun vs Node vs Deno vs Edge runtime 各自不同的设计选择）。给后面所有「为什么这样设计」提供历史坐标。

章节数 2 主要来源 官方 blog 状态 ⏳ 待撰写

本阶段覆盖

  • 2021-2026 Bun 项目史 —— Jarred Sumner 个人项目 → Oven (YC + a16z) → 1.0 production-ready → 1.1 Windows 一级支持 → 1.2 Bun.sql + binary lockfile
  • 2021 挑战 Node 为什么合理 —— CJS/ESM 二元状态拖 5 年 / 工具链碎片化 / JSC 在 Safari 证明启动快是差异化卖点
  • Jarred 单人 → 公司化的张力 —— bus factor 优劣势 / Oven 商业化探索
  • Bun vs Node vs Deno vs Edge runtime 矩阵 —— 引擎 / 异步 / 包管理 / TS 支持 / 默认安全 / 工具链 / 启动时间 / 生态成熟度 8 维对照
  • Bun 护城河：工具链一体化 —— Node 想追也难（要打散 npm/V8/CLI 边界）
  • Deno 路线为什么受挫 —— URL imports + 默认无 Node 兼容 → 生态太薄；Deno 2.0 后转向 npm 默认兼容（与 Bun 殊途同归）
  • Bun 的弱点 —— Edge runtime 暂时填不上（V8 Isolate 多租户模型）
