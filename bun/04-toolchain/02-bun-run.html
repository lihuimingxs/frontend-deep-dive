bun run / TS 直跑 / --hot vs --watch / glob import

bun 作为脚本 runner / 开发时 runtime 的章节。bun run 不只是「跑 package.json scripts」——它是 Bun runtime 的入口。关键洞察：--hot 保留 in-memory 状态做模块替换（Bun 独有），与 Node --watch 整进程重启的差异，决定了开发体验差距。

阅读时长 ~25 分钟 前置 P3 + Node P6.1 watch mode 主要来源 bun.com/docs/runtime · bun.com/blog hot reloading
