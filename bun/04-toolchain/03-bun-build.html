bun build · 共享 parser · --compile · 与 Vite/Rolldown/Turbopack 横评

bundler 章。bun build 设计目标：在 Vite / Rolldown / esbuild / Turbopack 这个高度内卷的赛道里，靠「和 runtime / install 共享 parser」差异化。关键洞察：bun build 单看 bundler 速度差距不大，差异化在于「一体化」——和 install / run / test 共享 parser，每个工具的启动都摊薄了 parser 加载成本。

阅读时长 ~30 分钟 前置 P4.2 + Node P6.1 SEA 主要来源 bun.com/docs/bundler · Bun 1.2 bytecode caching blog
