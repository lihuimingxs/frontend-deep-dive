P4 · 一体化工具链

Bun 另一个核心卖点：install / run / build / test 是同一个 CLI，共享同一套 Zig parser。每个工具一章，讲清各自的设计 + 比独立工具快的真正原因 + 横评对手。

章节数 4 主要来源 Bun CLI 文档 状态 ⏳ 待撰写

本阶段覆盖

  • bun install —— binary lockfile (1.2+) / Catalogs / hard link 替代 pnpm symlink / 256 并发解析 / 大 monorepo 比 pnpm 快 2-4x
  • bun run / runtime —— 直跑 TS/TSX / --hot vs --watch / glob import / .env 自动加载 / JSC Inspector
  • bun build —— 共享 parser 红利 / tree shaking / source map / code splitting / --compile 替代 SEA / bytecode caching (1.2+) / CSS bundler (1.2+)
  • bun test —— Jest 兼容 expect/mock/snapshot / fake timers / 内置 coverage / fetch mock / DOM 通过 happy-dom 包
  • 共享 parser 红利 —— install 解析 package.json + run transpile TS + build parse 代码 + test 同样 parse —— 三者复用 Zig parser，启动均摊
  • 横评对手 —— bun install vs pnpm/npm/yarn / bun build vs Vite/Rolldown/Turbopack/esbuild / bun test vs Vitest/Jest/node:test
  • 何时仍 fallback —— pnpm patch / Vitest UI dashboard / Jest 高级 plugin / DOM 测试
