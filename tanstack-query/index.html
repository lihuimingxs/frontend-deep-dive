编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 主线锁定 TanStack Query v5 —— 所有"现在怎么用"的代码示例基于 v5（单一 object signature / gcTime / pending / useSuspenseQuery 三件套）；老 API（v3 / v4 多参签名 / cacheTime 命名 / loading 状态名）只在 P1.1 历史段中出现
  • 「短重述 + 链回」 —— React 19 hooks（use / useOptimistic）在 React 主题；RSC payload 与 Server Actions 在 Next.js 主题；测试栈（MSW + Testing Library）在 测试主题。本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 设计动机以 Tanner Linsley 一手出处为准 —— TanStack Query 由 Tanner 个人主导，server state ≠ client state 二分法、queryKey 设计、stale-while-revalidate 心智都有他的原文，写"为什么这么设计"必须回到一手。SWR 视角以 Vercel 官方为准
  • RSC 边界讨论必须双向引用 —— P5 讨论 TanStack Query + RSC 共存时，每一段都要交叉引用 React/Next.js 团队的官方说法 + Tanner 的回应（避免"我猜他们想这样"）

TanStack Query 是 React 客户端数据获取的事实标准库，与多个已写主题深度交叉。本主题不重复已经讲透的底层机制，但会在每个交叉点提供"短重述 + 链接"。常见对应：

  • P5.1 RSC 时代的位置 / P5.2 use(promise) 与 RSC 创建的 promise → React P6.1 Server Components + Next.js P3 RSC 与流式
  • P5.2 useMutation 调 Server Action + 双向 invalidate → Next.js P4.1 Server Actions + Next.js P5 缓存系统
