P6 · SWR + 同代人

一章把 TanStack Query 之外的客户端 fetch 库讲完：SWR 的极简哲学（Vercel + Shu Ding 视角）、Apollo / Relay 在 GraphQL 阵营的现状、RTK Query 是不是 Redux 项目的"自然延续"、Jotai-Query 等小众路线。最后给 2026 客户端 fetch 库市场份额和决策表。

章节数 1 主要来源 swr.vercel.app + Apollo / Relay / RTK Query 官方文档 + Shu Ding 关于 SWR 的设计文章 状态 ✅ 全部完成

本阶段覆盖

  • SWR 极简 API · 只有 useSWR + mutate 两个核心；没有 queryClient
  • SWR 默认更激进 · 没有 staleTime；每次 mount + focus + reconnect 都拉
  • SWR 与 Vercel 路线绑定 · 起家于 Next.js Pages Router 时代；App Router 时代被 RSC 抢戏
  • Apollo Client 现状 · GraphQL only；normalized cache 强大但运维负担重
  • Relay 现状 · Meta 内部生态外溢；非 Meta 系大公司很少用
  • RTK Query 定位 · Redux Toolkit 内置；endpoint-based；适合已 Redux 项目
  • Jotai-Query / Zustand + 自管 fetch · 状态库阵营对 server state 的回应；市场份额小
  • 客户端 GraphQL 收缩 · 2026 GraphQL 总体让位 REST + tRPC + RSC fetch
  • 2026 市场份额 · TanStack Query 70% / SWR 15% / RTK Query 10% / Apollo+Relay 5%
