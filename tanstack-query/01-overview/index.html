P1 · 是谁 / 为什么

不讲 API 细节，只解决两个问题：从 jQuery $.ajax 到 React 19 use API，客户端数据获取这条线 15 年踩过哪些坑、为什么 Tanner Linsley 把 server state 拎出来当独立概念之后整个生态愿意跟。给后面所有"为什么这样设计"提供历史坐标。

章节数 2 主要来源 Tanner Linsley 公开演讲与博文 + Shu Ding 关于 SWR 的设计文章 + React 团队 RFC（use API / Server Components） 状态 ✅ 全部完成

本阶段覆盖

  • 2006-2014 · jQuery $.ajax + 回调地狱 → ES6 Promise；React + componentDidMount 取数
  • 2015-2018 · Redux + thunk / saga 把异步当 client state 写；Apollo / Relay GraphQL 阵营起步
  • 2019 · SWR（Vercel + Shu Ding）把 stale-while-revalidate 从 HTTP RFC 5861 搬进 React
  • 2020 · React Query v1（Tanner Linsley）提出 server state ≠ client state 二分法
  • 2022 · React Query → TanStack Query 改名 + 多框架（Vue / Solid / Svelte）
  • 2023 · TanStack Query v5：单一 object signature / gcTime 改名 / Suspense mode 一等公民
  • 2024 · React 19 use(promise) + Server Components：部分 server state 跑到 server-side
  • 2026 · TanStack Query + Next.js 16 RSC 共存模式落地（hydration 桥成熟）；SWR 维护放缓
  • 同代人横评 · fetch+useEffect / Apollo / Relay / RTK Query / SWR / TanStack Query / Jotai-Query / RSC fetch 全维度对比
  • 4 个核心设计赌注 · queryKey 是身份证 / stale-while-revalidate 是默认 / 不抢 store 的活 / vendor-neutral
