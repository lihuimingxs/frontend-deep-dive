P3 · query / mutation 生命周期

这一阶段是 TanStack Query 日常写法的主战场：useQuery 怎么读（status / fetchStatus / select / placeholderData），useMutation 怎么写（onMutate cancel + 保存 prev + onError 回滚 + onSettled invalidate 四步缺一不可）。乐观更新是 server state 库的"标志性技能"，错一步就出 bug。

章节数 2 主要来源 tanstack.com/query/latest/docs/framework/react/guides + Tanner Linsley 关于 optimistic updates 的博文 状态 ✅ 全部完成

本阶段覆盖

  • status vs fetchStatus 双状态正交 · status 是数据视角，fetchStatus 是网络视角；组合产生 stale-while-revalidate UI
  • select 派生数据 + memoize · 被低估的性能武器；多组件订阅同一 query 但只关心子集时差异巨大
  • placeholderData / keepPreviousData · 分页时不闪烁的关键
  • enabled 控制 dependent query · 串行触发用 enabled 而不是 if 包 useQuery
  • queryFn 闭包陷阱 · 闭包了过期变量永远拿到旧值
  • useMutation 不会自动触发 · 调用 mutate(variables) 才执行
  • 4 步乐观更新 · cancelQueries → 保存 prev → setQueryData 写新值 → onError 回滚 / onSettled invalidate
  • useMutationState 派生乐观 UI · v5 强化的写法；不动 cache 的乐观更新
  • 与 React 19 useOptimistic 的关系 · 不冲突；前者管 cache 后者管 form transition state
