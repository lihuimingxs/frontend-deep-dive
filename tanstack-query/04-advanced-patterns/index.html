P4 · 高级查询模式

单个 useQuery 解决不了真实业务的所有形状。这一阶段把 dependent / parallel / paginated / infinite 四种结构化查询写法 + Suspense mode 三件套 + queryClient direct API + streamedQuery（LLM 流式）讲清楚。也是 P5 RSC 共存的铺垫 —— prefetch / dehydrate 是桥接 RSC 与客户端 cache 的核心 API。

章节数 2 主要来源 tanstack.com/query/latest/docs/framework/react/guides/infinite-queries + suspense + prefetching 状态 ✅ 全部完成

本阶段覆盖

  • dependent query · enabled: !!previousData 串行；不要用 if 包 useQuery
  • parallel query 静态 + 动态 · 同组件多个 useQuery / useQueries 数组
  • paginated query · placeholderData: keepPreviousData 切页不闪烁
  • infinite query · useInfiniteQuery + getNextPageParam + data.pages 数组结构 + IntersectionObserver 滚到底加载
  • useSuspenseQuery 三件套 · v5 一等公民；永远 throw promise / error；data 永远不是 undefined
  • queryClient direct API · prefetchQuery / fetchQuery / ensureQueryData；router loader 集成
  • hover prefetch · 导航点亮时数据已经在了
  • streamedQuery · v5 新；支持 SSE / chunked；与 Vercel AI SDK useChat 哲学接近
  • 与 React Router v7 / TanStack Router loader 集成 · loader 里 ensureQueryData，组件里 useSuspenseQuery
