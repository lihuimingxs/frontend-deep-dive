P5 · 缓存系统真相 ⭐

缓存是 Next.js 最被骂、用户最迷糊的部分。v13-14 默认 cache-by-default 引爆生产 stale 事故，v15 反转默认值是迟到的修正，v16 用 Cache Components + 'use cache' 把整个心智重做一遍。这一阶段把 4 年来的演变拉直，给一份能用到生产的决策树。

章节数 2 ⭐ 最难章 主要来源 nextjs.org/docs caching + Vercel "Partial Prerendering" / "Cache Components" 工程 blog + v15 / v16 release notes 状态 ✅ 全部完成

本阶段覆盖

  • v13-15 五层缓存模型 · Request Memoization / Data Cache / Full Route Cache / Router Cache / CDN
  • cache-by-default 的设计错误 · v13-14 默认 fetch 缓存导致生产 stale 数据事故的根源
  • v15 反转 · 默认 explicit-opt-in；fetch 不写选项就是 dynamic
  • cookies / headers 触发 dynamic · 整个 route 自动 dynamic 的隐式规则
  • v16 Cache Components · cacheComponents: true + 'use cache' 指令（替代 dynamicIO + unstable_cache）
  • PPR GA · 静态壳（'use cache' 包的部分）+ 动态洞（Suspense + 读 runtime APIs）
  • revalidate 三件套决策 · revalidatePath（路径粒度）/ revalidateTag（标签精准）/ updateTag（请求内立即生效）
  • 典型场景决策 · Blog（长 cache + Suspense 评论）/ Dashboard（cache 导航 + dynamic 数据）/ Todo（updateTag 立刻刷）
  • 常见陷阱 · 在 'use cache' 里读 cookies / tag 没在 fetch 声明 / Suspense fallback 设计
