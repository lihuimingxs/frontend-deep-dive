P6 · Turbopack

Turbopack 是 Vercel 用 Rust 重写的 bundler，2026 年在 Next.js 16 上同时拿下 dev + prod 双 stable，正式取代 Webpack。这一阶段讲它的增量计算心智、和 Vite/Rolldown/Bun build/Rspack 的路线分叉，以及为什么 Vercel 押 Rust 不押 Vite。

章节数 1 主要来源 turbo.build/pack docs + Tobias Koppers 演讲（前 Webpack 作者）+ vercel.com/blog Turbopack 系列 状态 ✅ 全部完成

本阶段覆盖

  • turbo-tasks 增量计算引擎 · 每个编译单元是可缓存的 task；改一个文件只重算受影响 task
  • SWC 编译 · Rust 写的 Babel 替代
  • Webpack loader 兼容 · 迁移路径与限制
  • dev + prod 同 pipeline · v16 双 stable 的工程意义
  • vs Vite + Rolldown · 双模架构 vs 单一 pipeline；esbuild → Rolldown 演化（链回 Vite 主题）
  • vs esbuild · Go vs Rust 的不同路径
  • vs Rspack · 字节跳动的 Rust Webpack-compat
  • vs Bun build · Zig + 与 Bun runtime 同源（链回 Bun 主题）
  • 为什么 Vercel 不用 Vite · 全栈控制 / RSC 流式适配 / 增量计算长期赌注
  • 2026 bundler 格局 · Rust 是赢家，路径剩两条（Turbopack vendor-locked / Vite-Rolldown 通用）
