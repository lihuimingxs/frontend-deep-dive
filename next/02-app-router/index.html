P2 · App Router 心智

App Router 用文件夹定义路由、用文件名约定层级行为，再用 'use client' / 'use server' 切边界。要写好 Next.js，先把这两件事的心智建对。

章节数 2 主要来源 nextjs.org/docs/app/building-your-application/routing + React 19 RSC RFC 状态 ✅ 全部完成

本阶段覆盖

  • page / layout / loading / error / not-found / template / default · 7 个文件约定 + 各自映射的 React 渲染语义
  • 动态段 · [slug] / [...slug] / [[...slug]]
  • 路由分组 + 私有文件夹 · (group) / _internal
  • Parallel Routes · @slot 同时渲染多个独立子路由（dashboard 多面板 / modal）
  • Intercepting Routes · (.) / (..) 不离开当前页面的"插队"渲染（图片预览 modal）
  • Multiple Root Layouts · 一个 app 多 root layout（marketing + dashboard 双套壳）
  • Server / Client 默认 · App Router 里所有组件默认是 RSC，'use client' 是模块边界 opt-in
  • 序列化规则 · Server → Client props 必须可序列化；函数 prop 仅 Server Action 可传
  • "Server 包 Client 包 Server"模式 · 用 children 让 Client 不污染整棵子树
