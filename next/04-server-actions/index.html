P4 · Server Actions

Server Actions 把"表单 + 数据变更"重新缝合 —— 一个 'use server' 函数同时是 server function 和 client 调用句柄，配合 React 19 的 useActionState / useFormStatus / useOptimistic 让表单变成全栈一等公民。这一阶段讲清它的运行时、安全边界、与 revalidate 三件套的配合。

章节数 1 主要来源 nextjs.org/docs Server Actions + React RFC #229（Actions）+ Lee Robinson "Forms in Next.js" 演讲 状态 ✅ 全部完成

本阶段覆盖

  • 3 种声明位置 · 文件级 / 函数内 / Server Component 内 inline
  • 跨边界传递 · Server Action 作为 prop 传给 Client Component / 作为 form action
  • 底层调用 · client 实际触发一次 POST 请求，路由由加密 action ID 决定
  • useActionState · 把 action 包成有 state 的 form（pending / error / data）
  • useFormStatus · 表单子组件读 pending
  • useOptimistic · 乐观更新
  • revalidatePath · 让某路径下次访问重新生成
  • revalidateTag · 让带 tag 的所有 fetch 失效
  • updateTag（v16 新） · 失效 + 当前请求内立即重算（提交后立刻看到新数据）
  • progressive enhancement · 关 JS 也能 submit form
  • 安全边界 · CSRF 防护 + 加密 ID + auth 必须重校验
  • Server Actions vs Route Handlers · 表单/变更 vs REST/webhook 决策
