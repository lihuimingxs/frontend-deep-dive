编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 主线锁定 Next.js 16 —— 所有"现在怎么用"的代码示例基于 v16；老 API（getInitialProps / Pages Router / unstable_cache 等）只在 P1.1 历史 + P5.1 缓存演进段中出现
  • 「短重述 + 链回」 —— React 19 hooks / Suspense / Concurrent / Fiber 都在 React 主题；bundler 历史在 Vite 主题；测试栈在 测试主题。本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 设计动机以 release notes / Vercel blog / RFC 为准 —— Next.js 版本演进激进，文档随版本快速变化；写"为什么这么设计"必须回到一手出处，避开二手解读
  • 缓存 / 性能数字必标条件 —— "PPR 比 SSG 快 N 倍"必须标场景（数据来源 + 流式与否 + Vercel 还是 self-host），否则不写

与既有主题的关系

Next.js 是 React 的生产承载，所以与多个已写主题大量交叉。本主题不重复已经讲透的底层机制，但会在每个交叉点提供"短重述 + 链接"。常见对应：

  • P3.1 RSC 心智模型 → React P6.1 Server Components（RSC 是 React 概念，本主题讲 Next.js 怎么实现协议）
  • P3.2 流式渲染 / Suspense / Selective Hydration → React P5.2 Suspense + P5.4 Error Boundary
  • P4.1 Server Actions 与 React 19 hooks → React P6.2 Actions（useActionState / useFormStatus / useOptimistic）
  • P6.1 Turbopack vs Vite/Rolldown/esbuild → Vite P1.1 构建工具史 + Vite P2.1 双模架构
  • P6.1 Turbopack vs Bun build → Bun P4.2 bun build
  • P7.1 V8 isolate / Edge runtime → V8 主题；Node runtime / streams → Node 主题
  • P7.2 Next.js 测试栈（Vitest + Playwright）→ 测试工具主题
  • P7.2 monorepo + Turborepo cache → pnpm P5.1 Turborepo
