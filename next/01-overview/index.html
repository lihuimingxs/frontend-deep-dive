P1 · Next.js 是谁

不讲 API 细节，只解决两个问题：Next.js 怎么从 2016 年的 SSR 工具走到 2026 年押注 RSC 的全栈元框架、它在元框架格局里站哪。给后面所有"为什么这样设计"提供历史坐标。

章节数 2 主要来源 nextjs.org/blog 各版本 release notes + Vercel 工程 blog + Lee Robinson / Tim Neutkens 公开演讲 状态 ✅ 全部完成

本阶段覆盖

  • 2016 Next.js 1.0 · Vercel/Zeit 开源；getInitialProps + Pages Router；解决 React + SSR 配置地狱
  • 2019 Next.js 9 · TypeScript / API Routes / 自动静态优化；从 SSR 工具扩展为全栈
  • 2020 Next.js 9.3 + 9.5 · getStaticProps / getServerSideProps / ISR；分化出 build-time / 运行时 / 增量三种渲染模式
  • 2022 Next.js 13 · App Router beta + RSC + Turbopack alpha + 新 image/font；最大一次断代式重写，押注 RSC
  • 2023 Next.js 14 · Server Actions stable + PPR 预览；表单全栈化
  • 2024 Next.js 15 · React 19 + async request APIs + Turbopack dev GA；默认缓存从 cache-by-default 反转为 explicit-opt-in（被骂最多的修正）
  • 2026 Next.js 16 · Turbopack prod GA + PPR GA + Cache Components（'use cache' 替代 dynamicIO）+ updateTag
  • 同代人横评 · React Router v7 / TanStack Start / Astro / Qwik City / SvelteKit / Nuxt 3 全维度对比
  • 4 个核心设计赌注 · Server-first / 约定优于配置 / 框架管缓存 / Vercel 部署优先
