P3 · RSC 与流式渲染

RSC 不是"一种 SSR"，是 React 团队定的一套组件树序列化协议。Next.js 把这个协议落到 HTTP 流上，配合 Suspense 实现"边渲染边发"。这一阶段把 RSC 神话祛魅 —— 讲清协议长啥样、stream 怎么走、hydration 在哪段发生。

章节数 2 主要来源 React RFC #188（RSC）+ react-server-dom-webpack 源码 + Next.js renderer 源码 + Sebastian Markbåge / Andrew Clark 演讲 状态 ✅ 全部完成

本阶段覆盖

  • RSC payload 形态 · 不是 HTML 也不是 JSON 的特殊序列化格式（"$","html",null,{...} 风格）
  • SSR + RSC 双流 · 首次访问 server 同时输出 HTML（首屏 + SEO）和 RSC payload（hydration / 后续导航对账）
  • 客户端导航 · Server 只返回 RSC payload，Router 局部更新树
  • flush 机制 · Suspense boundary 解析完立刻 push chunk；流式渲染本质
  • Client Component 引用 · payload 里只有引用 + 边界 prop，bundle 真正内容由 webpack/Turbopack manifest 提供
  • loading.tsx ≡ Suspense · 文件约定如何映射到 React Suspense fallback
  • 嵌套 Suspense + 分段 reveal · 多段独立流，UX 对接 Network timeline
  • Selective Hydration · React 18+ 特性，App Router 默认开
  • 避免 waterfall · 在 layout 提前并行 fetch 的反模式 vs 正确模式
  • error.tsx + reset · 错误边界与恢复
  • template.tsx vs layout.tsx · 重新挂载语义 / 何时用 template
