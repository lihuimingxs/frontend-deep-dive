编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 主线锁定 shadcn/cli v4 + Radix 统一包 —— 所有"现在怎么用"的代码示例基于 v4 + 单一 radix-ui 包；老 @radix-ui/react-* 多包仅在迁移段中出现
  • 「短重述 + 链回」 —— Tailwind utility / @theme / CVA + cn 都在 Tailwind 主题；React Hooks / RSC 在 React 主题；Server Actions 在 Next.js 主题。本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • a11y 行为以 WAI-ARIA APG 为准 —— Dialog 的 focus trap / Combobox 的 aria-activedescendant 等都是 W3C 标准；本主题点出"Radix 实现的是标准、不是发明"
  • 设计动机以 changelog + 作者访谈为准 —— 例如"为什么 copy-paste 模式"原文是 shadcn 多次访谈；写"为什么这么设计"必须回到一手出处

与既有主题的关系

shadcn/ui + Radix 是 React 项目的 UI 组件层，与几个已写主题大量交叉。本主题不重复已经讲透的底层机制，但会在每个交叉点提供"短重述 + 链接"。常见对应：

  • P1.2 同代人横评（基础层面） → Tailwind P7.1 UI 库横评（先看这个建立基本认知）
  • P2.2 asChild / cn() 工具链 → Tailwind P6.1 CVA + cn
  • P6.1 Form + Server Actions → Next.js P4.1 Server Actions
  • P6.2 Theme + 暗色 → Tailwind P5.1 响应式 + 暗色 + P3.2 CSS-first 配置
  • P7.1 高级模式（TanStack Table / 虚拟列表）→ 短重述（这些库自己有完整文档）
  • P3.1 Portal / focus trap 浏览器机制 → 浏览器渲染主题
