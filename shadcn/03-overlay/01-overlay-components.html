浮层组件全谱

浮层组件是 Radix 内部最复杂的部分 —— Dialog / Popover / DropdownMenu / Tooltip / HoverCard / ContextMenu 全谱配 6 个核心技术（Floating UI 智能定位 / Portal / Focus Trap / Scroll Lock / ESC 关闭 / 浏览器原生 popover API）。这一章把每个组件 + 每项技术讲清，让你看到"自己手写浮层为什么总有 bug"。

阅读时长 ~30 分钟 前置 P2 Radix 哲学 + WAI-ARIA a11y 主要来源 radix-ui.com docs + Floating UI docs + WAI-ARIA APG

01 速览

  1. 浮层组件家族按"语义 + a11y 行为"分 6 类：Dialog（modal、阻塞、focus trap）/ Popover（非阻塞、点外关闭、智能定位）/ DropdownMenu（菜单语义、roving tabindex）/ Tooltip（hover/focus 触发、有延迟、aria-describedby）/ HoverCard（类似 Tooltip 但内容丰富）/ ContextMenu（右键触发）。
  2. 核心技术 1 · Floating UI（前身 popper.js）：浮层定位库、自动避开视口边缘、翻转方向、collision detection、arrow 定位。Radix 内部用 @floating-ui/react。
  3. 核心技术 2 · Portal：把浮层 DOM 渲染到 document.body 末尾（不是组件树原位置）。解决 z-index 战争、不被父级 overflow / transform 影响。
  4. 核心技术 3 · Focus Trap：Dialog 打开时 Tab 在 Dialog 内循环，不跑到背景。关闭时 focus 回到触发元素。Radix 用内置 focus trap（不依赖 react-focus-lock）。
  5. 核心技术 4 · Scroll Lock：Dialog 打开时禁止 body scroll。处理 iOS Safari 的橡皮筋问题 + scrollbar gutter（防止 layout shift）。
  6. 核心技术 5 · ESC + 点外关闭：全局 keydown 监听 ESC、全局 click 监听判断点击是否在浮层外。嵌套时只关最上层（Radix 自动处理 stack）。
  7. 核心技术 6 · 浏览器原生 popover API：HTML <dialog> + popover attribute（2024+ 浏览器支持）。Radix 暂不全切（兼容性 + 行为差异），2026-2027 可能切。
  8. 动画：Radix 用 data-state="open"/"closed" 属性 + Tailwind animate-in / animate-out 实现。配合 CSS animation 或 framer-motion。
  9. Tooltip 与 disabled button：disabled button 不触发 hover，Tooltip 不会出现。要包一层 div 解决。这是常见 bug。
  10. 嵌套 Dialog / Modal 之上的 Popover：z-index + focus + ESC 处理需要谨慎。Radix 自动处理但用户要懂规则（每个浮层独立 stack）。

02 底层逻辑

浮层组件家族 · 6 类语义差异

组件 触发 阻塞 关闭方式 a11y role 典型用途
Dialogclick button是（focus trap + scroll lock）ESC / Close buttondialog表单 / 重要信息确认
AlertDialogclick button是必须明确 yes/noalertdialog"确定删除"等
Popoverclick button否（点外关闭）ESC / 点外dialog（generic）设置面板 / 表单子部分

Tooltiphover / focus否移开 / blurtooltip提示文字

选哪个不是看视觉，是看 语义：

  • 需要"用户必须看到 + 不能后台操作其他"→ Dialog（阻塞）
  • 需要"打开一段时间但不阻塞"→ Popover
  • "列出选项让选一个"→ DropdownMenu
  • "提示一句话"→ Tooltip
  • "hover 显示丰富内容"→ HoverCard
  • "右键菜单"→ ContextMenu

核心技术 1 · Floating UI 智能定位

Floating UI（前身 popper.js）解决"浮层定位"的复杂问题：

  • 基础定位：浮层位于触发元素旁边（top / bottom / left / right + start / center / end）
  • 避开视口边缘：触发元素在视口边缘，浮层应该翻转到反方向（"top"翻转为"bottom"）
  • collision detection：浮层与视口边界碰撞时调整位置
  • arrow 定位：箭头始终指向触发元素中心
  • 动态位置更新：触发元素移动 / scroll / resize 时浮层跟随

Radix 内部用 @floating-ui/react。用户用 Radix 不需要直接接触 Floating UI，但定制时要懂：

tsx 
// Radix Popover 配置定位
<Popover.Content
  side="top"          // top / bottom / left / right
  sideOffset={4}      // 距离触发元素 4px
  align="center"      // start / center / end
  alignOffset={0}
  avoidCollisions={true}      // 自动避开视口边缘（默认 true）
  collisionPadding={8}        // 视口边缘 padding
  sticky="partial"            // 触发元素 scroll 出视口时浮层是否跟随
>
  ...
</Popover.Content>

实际行为：

  • 用户在视口顶部点击触发，side="top" → 自动翻转为 side="bottom"（避开顶部）
  • 箭头自动跟随翻转
  • 视口 resize → 浮层重新计算位置

核心技术 2 · Portal 解决 z-index 战争

不用 Portal 的问题：

tsx 
// ❌ 不用 Portal：浮层在父级 DOM 内
<div className="overflow-hidden">
  <Popover>
    <PopoverContent>弹出内容</PopoverContent>
  </Popover>
</div>
// 父级 overflow-hidden 会裁切弹出内容！

用 Portal：

tsx 
// ✅ Radix Popover 默认用 Portal
<Popover.Root>
  <Popover.Trigger>Open</Popover.Trigger>
  <Popover.Portal>  {/* 渲染到 document.body 末尾 */}
    <Popover.Content>弹出内容</Popover.Content>
  </Popover.Portal>
</Popover.Root>
// 不被父级 overflow / transform 影响

Portal 解决的问题：

  • 父级 overflow: hidden 不裁切浮层
  • 父级 transform: ... 不创建新 stacking context（z-index 失效）
  • z-index 战争（浮层不需要"竞赛"高 z-index）

Portal 的代价：

  • 浮层 DOM 不在组件树原位置（DevTools 检查时跳到 body 末尾）
  • 事件冒泡仍按 React 树（Portal 内事件 bubble 到原位置父级）
  • SSR 时需要客户端 hydration 完成才能 portal（Radix 内置处理）

核心技术 3 · Focus Trap

Focus Trap 是 Dialog 关键 a11y。规则（详见 P2.1）：

  • Dialog 打开 → focus 移到 Dialog 内某个可 focus 元素
  • Tab → 在 Dialog 内循环（最后元素后跳第一）
  • Shift+Tab → 反向循环
  • 关闭 → focus 回到触发元素

Radix 内部 focus trap 实现（简化）：

tsx 
function FocusTrap({ children, isOpen }) {
  const containerRef = useRef<HTMLDivElement>(null);
  const previousFocusRef = useRef<HTMLElement | null>(null);

  useEffect(() => {
    if (!isOpen) return;

    // 1. 记住当前 focus（关闭后回去）
    previousFocusRef.current = document.activeElement as HTMLElement;

    // 2. focus 第一个可 focus 元素
    const firstFocusable = getFirstFocusable(containerRef.current);
    firstFocusable?.focus();

    // 3. 监听 Tab 实现循环
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key !== 'Tab') return;
      const focusables = getFocusables(containerRef.current);
      const first = focusables[0];
      const last = focusables[focusables.length - 1];

      if (e.shiftKey && document.activeElement === first) {
        e.preventDefault();
        last?.focus();
      } else if (!e.shiftKey && document.activeElement === last) {
        e.preventDefault();
        first?.focus();
      }
    };
    document.addEventListener('keydown', handleKeyDown);

    return () => {
      document.removeEventListener('keydown', handleKeyDown);
      // 4. 关闭时 focus 回原元素
      previousFocusRef.current?.focus();
    };
  }, [isOpen]);

  return <div ref={containerRef}>{children}</div>;
}

边角情况：

  • Dialog 内动态添加可 focus 元素（异步加载）→ 重新计算 focusables
  • Dialog 内 input 用 IME 输入时 → 不要 trap 拦 ESC
  • 嵌套 Dialog → 只 trap 最上层（Radix 自动处理 stack）

核心技术 4 · Scroll Lock

Dialog 打开时禁止 body 滚动。两种方法：

方法 1 · overflow: hidden（最常见）：

javascript 
// Dialog 打开
document.body.style.overflow = 'hidden';

// 关闭
document.body.style.overflow = '';

问题：滚动条消失会导致 layout shift（页面"跳一下"）。

方法 2 · 加 padding-right 补偿 scrollbar 宽度：

javascript 
// 测量 scrollbar 宽度
const scrollbarWidth = window.innerWidth - document.documentElement.clientWidth;

// Dialog 打开
document.body.style.overflow = 'hidden';
document.body.style.paddingRight = `${scrollbarWidth}px`;  // 补偿
// 关闭恢复

方法 3 · CSS scrollbar-gutter（现代）：

css 
html {
  scrollbar-gutter: stable;  /* 始终保留 scrollbar 空间 */
}

Radix 综合处理：用 padding-right 补偿（兼容老浏览器）+ overflow: hidden。还要处理 iOS Safari 橡皮筋（needs position: fixed + top: -scrollY）。

核心技术 5 · ESC + 点外关闭

Popover / DropdownMenu 等非阻塞浮层需要"点击浮层外关闭"。

Radix 实现：

tsx 
function useClickOutside(ref, isOpen, onClose) {
  useEffect(() => {
    if (!isOpen) return;

    const handleClick = (e: MouseEvent) => {
      if (ref.current && !ref.current.contains(e.target as Node)) {
        onClose();
      }
    };

    // 用 capture phase 确保不被冒泡阻止
    document.addEventListener('pointerdown', handleClick, true);
    return () => document.removeEventListener('pointerdown', handleClick, true);
  }, [ref, isOpen, onClose]);
}

边角情况：

  • 触发元素在浮层外 → 点击触发元素不应关闭（Radix 处理）
  • 嵌套浮层 → 只关最内层（按 stack 顺序）
  • Trigger 是 Dialog 内的按钮 → Dialog 关闭时 Popover 也关
  • 浮层内 click 不应关闭（Radix 检查 e.target 是否在 contentRef 内）

核心技术 6 · 浏览器原生 popover API

2024 年起浏览器主流支持原生 popover API：

html 
<!-- 原生 popover（2024+） -->
<button popovertarget="my-popover">Open</button>
<div id="my-popover" popover>
  This is a popover
</div>
<!-- 浏览器自动处理：定位、显示/隐藏、ESC 关闭、点外关闭 -->
html 
<!-- 原生 dialog（2022+） -->
<dialog id="my-dialog">
  <p>Modal content</p>
  <button>Close</button>
</dialog>

<script>
  document.getElementById('my-dialog').showModal();  // 打开
  // ESC 自动关闭、focus trap 自动处理
</script>

原生 API 的优势：

  • 浏览器内置 a11y / 焦点 / 关闭逻辑
  • 无 JS 开销
  • tab 自动 trap

原生 API 的限制：

  • 定位不灵活（不像 Floating UI 智能避开边缘）
  • 动画支持有限
  • 跨浏览器行为有微妙差异
  • 不能完整覆盖 React 受控/非受控模式

Radix 暂时不全切原生 API。可能在 2026-2027 年逐步采用（保留 Radix API 不变，内部改用原生）。

动画 · data-state + Tailwind animate-*

Radix 用 data-state="open"/"closed" 属性配合 CSS animation：

tsx 
<Dialog.Content className="
  fixed left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2
  bg-white p-6 rounded-lg
  data-[state=open]:animate-in
  data-[state=open]:fade-in-0
  data-[state=open]:zoom-in-95
  data-[state=closed]:animate-out
  data-[state=closed]:fade-out-0
  data-[state=closed]:zoom-out-95
  duration-200
">
  ...
</Dialog.Content>

关键点：

  • data-state 属性由 Radix 自动注入
  • shadcn 用 tailwindcss-animate plugin 提供 animate-in / fade-in 等 utility
  • Radix 在动画结束后才真正卸载 DOM（用 animation-end 监听）
  • 用 framer-motion 替代 CSS 动画也支持（用 AnimatePresence + 受控模式）

03 应用场景

场景 1 · Dialog 表单（提交后关闭）

tsx 
function ContactDialog() {
  const [open, setOpen] = useState(false);

  async function handleSubmit(data: FormData) {
    await sendContact(data);
    setOpen(false);
    toast.success('Sent!');
  }

  return (
    <Dialog.Root open={open} onOpenChange={setOpen}>
      <Dialog.Trigger asChild>
        <Button>Contact us</Button>
      </Dialog.Trigger>
      <Dialog.Portal>
        <Dialog.Overlay className="fixed inset-0 bg-black/50 data-[state=open]:animate-in data-[state=open]:fade-in-0" />
        <Dialog.Content className="fixed left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2 bg-white p-6 rounded-lg w-full max-w-md">
          <Dialog.Title className="text-lg font-semibold">Contact</Dialog.Title>
          <Dialog.Description className="text-sm text-gray-500">
            We'll respond in 24 hours.
          </Dialog.Description>
          <form action={handleSubmit} className="mt-4">
            <input name="email" type="email" required />
            <textarea name="message" required />
            <Button type="submit">Send</Button>
          </form>
        </Dialog.Content>
      </Dialog.Portal>
    </Dialog.Root>
  );
}

场景 2 · Tooltip 在 disabled button

</Tooltip.Root>

05 常见陷阱

  • Tooltip 在 disabled button 不显示 —— disabled button 不触发 hover/focus。包一层 div / span（详见场景 2）
  • Popover.Content 被父级 overflow-hidden 裁切 —— 必须用 Popover.Portal 包裹
  • Dialog 打开页面跳一下（layout shift） —— scrollbar 消失。Radix 自动加 padding-right 补偿；如果你自己写 Dialog 别忘了
  • iOS Safari Dialog 内仍可滚动背景 —— Radix 处理了 iOS 橡皮筋（用 position: fixed + top: -scrollY），自己写要复杂
  • 嵌套 Dialog ESC 关错层 —— Radix 自动处理 stack（最上层先关）。手写易错
  • 动画忘了 data-[state=closed] 部分 —— Dialog 关闭时直接消失，没 fade-out。要写两个 state 的动画
  • 用 React Portal 自己实现而不用 Radix Portal —— Radix Portal 处理 SSR / hydration / 嵌套 stack 等边角情况
  • Dialog 内 Form 提交不关闭 —— 用受控模式 setOpen(false) 在 onSubmit 后调
  • 用 z-index: 9999 解决浮层不在顶部 —— 错。改用 Portal（不需要竞赛 z-index）
  • Tooltip 内放复杂内容（按钮/链接） —— Tooltip 是"提示"，不应放可交互内容（屏幕阅读器读不到）。用 HoverCard 替代
  • DropdownMenu 项是 Link 但 click 不跳转 —— 用 asChild + Next.js Link（详见 P2.2）
  • 用 onPointerDownOutside / onEscapeKeyDown 想拦截关闭但没 preventDefault —— 要 e.preventDefault() 阻止关闭
  • 受控 Dialog 但 onOpenChange 不更新 state —— Dialog 关不上。检查 onOpenChange 是否真调 setOpen
  • 移动端 Dialog 全屏 —— Radix Dialog 不自动全屏。用 Vaul（emil_kowalski 出的 Drawer）做移动端 bottom sheet
  • 嵌套 ContextMenu / DropdownMenu submenu 闪烁 —— Radix 处理了 hover 延迟（避免误触发），自己写易闪
