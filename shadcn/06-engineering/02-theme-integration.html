Theme + 暗色 + 多品牌 + Tailwind v4 集成

shadcn 默认 token 系统 + next-themes class 策略 + shadcn theme builder 可视化选色 + 多品牌 CSS variables + Tailwind v4 @theme 集成。这一章把 globals.css 完整模板讲清，让你团队能自定义主题、支持暗色、支持多品牌、与 Tailwind v4 工作流契合。

阅读时长 ~25 分钟 前置 Tailwind P5.1 / P3.2 + P6.1 表单 主要来源 ui.shadcn.com/docs/theming + next-themes README + Tailwind v4 docs

01 速览

  1. shadcn 默认 token 系统用 HSL 颜色空间 + CSS variables。核心 token：--background / --foreground（页面底色 / 主文字）/ --primary / --primary-foreground（主色 / 主色上的文字）/ --secondary / --muted / --accent / --destructive（其他语义色）。每个 token 在 :root（亮色）和 .dark（暗色）各定义一次。
  2. shadcn token 与 Tailwind utility 自动对应：bg-background / text-foreground / bg-primary / text-primary-foreground 等。这套语义化 utility 让组件代码不需要 dark: 前缀（自动跟随主题）。
  3. 暗色模式用 next-themes（class 策略）+ shadcn 默认 .dark 选择器。next-themes 处理 SSR FOUC（首屏不闪烁）+ localStorage 持久化 + 系统切换跟随。
  4. shadcn theme builder（ui.shadcn.com/themes）：可视化选色 + 实时预览 + 生成 CSS variables。用户复制生成的 CSS 替换 globals.css 即换主题。也可以选 12 个预设 theme（zinc / slate / stone / red / orange / yellow / green / blue / violet 等）。
  5. 多品牌（white-label）：在 root 用 .theme-{brand} class 切换不同 CSS variables。同一组件代码不变，自动跟随当前 theme。详见 Tailwind P7.2。
  6. Tailwind v4 @theme 集成：v4 把 token 移到 CSS（@theme {}），shadcn v4 CLI 自动适配。@theme inline 让 shadcn token 与 v4 token 命名空间统一。
  7. shadcn 默认用 HSL（h s% l% 三个空格分隔），不是 OKLCH（与 Tailwind v4 默认 OKLCH 不同）。这是历史原因；shadcn v4 也支持 OKLCH，用户选。
  8. token 命名分两类：语义化（primary / destructive，应用级、跟随品牌）/ 原色 scale（zinc-500 / red-500，原始颜色）。shadcn 用语义化层；Tailwind 默认 utility 是原色。两者可共存。
  9. radius 集中管理：--radius 是 shadcn 圆角基准（默认 0.5rem = rounded-md）。shadcn 组件用 rounded-[var(--radius)] 跟随。改 radius 一处生效全组件。
  10. 本章是 shadcn 项目实战收束。读完能写 globals.css、配 next-themes、用 theme builder、做多品牌 white-label、与 Tailwind v4 完整集成。

02 底层逻辑

shadcn 默认 token 系统

shadcn init 时生成的 globals.css（默认 zinc base color）：

css 
@layer base {
  :root {
    /* 页面底色 + 主文字 */
    --background: 0 0% 100%;            /* white */
    --foreground: 240 10% 3.9%;         /* near black */

    /* 卡片 */
    --card: 0 0% 100%;
    --card-foreground: 240 10% 3.9%;

    /* 浮层（popover / dropdown） */
    --popover: 0 0% 100%;
    --popover-foreground: 240 10% 3.9%;

    /* 主色（按钮 / 链接） */
    --primary: 240 5.9% 10%;            /* dark gray */
    --primary-foreground: 0 0% 98%;

    /* 次要色（次要按钮 / 标签） */
    --secondary: 240 4.8% 95.9%;
    --secondary-foreground: 240 5.9% 10%;

    /* 静音 / 辅助 */
    --muted: 240 4.8% 95.9%;
    --muted-foreground: 240 3.8% 46.1%;

    /* 强调 */
    --accent: 240 4.8% 95.9%;
    --accent-foreground: 240 5.9% 10%;

    /* 危险 */
    --destructive: 0 84.2% 60.2%;       /* red */
    --destructive-foreground: 0 0% 98%;

    /* 边框 / input / ring */
    --border: 240 5.9% 90%;
    --input: 240 5.9% 90%;
    --ring: 240 5.9% 10%;

    /* radius */
    --radius: 0.5rem;
  }

  .dark {
    --background: 240 10% 3.9%;
    --foreground: 0 0% 98%;

    --card: 240 10% 3.9%;
    --card-foreground: 0 0% 98%;

    --popover: 240 10% 3.9%;
    --popover-foreground: 0 0% 98%;

    --primary: 0 0% 98%;
    --primary-foreground: 240 5.9% 10%;

    --secondary: 240 3.7% 15.9%;
    --secondary-foreground: 0 0% 98%;

    --muted: 240 3.7% 15.9%;
    --muted-foreground: 240 5% 64.9%;

    --accent: 240 3.7% 15.9%;
    --accent-foreground: 0 0% 98%;

    --destructive: 0 62.8% 30.6%;
    --destructive-foreground: 0 0% 98%;

    --border: 240 3.7% 15.9%;
    --input: 240 3.7% 15.9%;
    --ring: 240 4.9% 83.9%;
  }
}

@layer base {
  * { @apply border-border; }
  body { @apply bg-background text-foreground; }
}

关键点：

  • HSL 格式：240 10% 3.9%（不是 hsl(240, 10%, 3.9%)）—— 让 Tailwind 通过 hsl(var(--primary) / 0.5) 加 opacity
  • 不带单位：240 10% 3.9%，用时 hsl(var(--primary))
  • 每个语义色配 -foreground（在该色背景上的文字色）
  • :root 是亮色、.dark 是暗色（class 策略）

token 与 Tailwind utility 自动对应

tailwind.config（v3）或 @theme（v4）配置 utility ↔ CSS variable 映射：

typescript 
// tailwind.config.ts (v3 风格)
export default {
  theme: {
    extend: {
      colors: {
        background: 'hsl(var(--background))',
        foreground: 'hsl(var(--foreground))',
        primary: {
          DEFAULT: 'hsl(var(--primary))',
          foreground: 'hsl(var(--primary-foreground))',
        },
        secondary: {
          DEFAULT: 'hsl(var(--secondary))',
          foreground: 'hsl(var(--secondary-foreground))',
        },
        // ... 其他
      },
      borderRadius: {
        lg: 'var(--radius)',
        md: 'calc(var(--radius) - 2px)',
        sm: 'calc(var(--radius) - 4px)',
      },
    },
  },
};

v4 用 @theme：

css 
/* Tailwind v4 · globals.css */
@import "tailwindcss";

@theme inline {
  --color-background: hsl(var(--background));
  --color-foreground: hsl(var(--foreground));
  --color-primary: hsl(var(--primary));
  --color-primary-foreground: hsl(var(--primary-foreground));
  /* ... */

  --radius-lg: var(--radius);
  --radius-md: calc(var(--radius) - 2px);
  --radius-sm: calc(var(--radius) - 4px);
}

关键 @theme inline：让 token 直接映射到现有 CSS variable，不重新定义。这与 v4 默认 @theme（替代 tailwind.config）兼容。

用户写组件用语义化 utility：

tsx 
// 不用 dark: 前缀，自动跟随主题
<div className="bg-background text-foreground">
  <Button className="bg-primary text-primary-foreground">OK</Button>
  <Button variant="destructive">Delete</Button>  {/* shadcn Button 内部用 destructive token */}
</div>

对比传统 dark: 写法（重复）：

tsx 
// 老写法（不推荐）
<div className="bg-white dark:bg-gray-900 text-gray-900 dark:text-white">
  <Button className="bg-blue-500 dark:bg-blue-400 text-white">OK</Button>
</div>

shadcn 语义色让组件代码不需要懂亮色 / 暗色，由 CSS variables 切换。

next-themes 集成

已在 Tailwind P5.1 详细讲过。这里补充 shadcn 特定细节：

tsx 
// app/_components/ThemeProvider.tsx
'use client';
import { ThemeProvider as NextThemesProvider } from 'next-themes';

export function ThemeProvider({ children, ...props }) {
  return <NextThemesProvider {...props}>{children}</NextThemesProvider>;
}

// app/layout.tsx (RSC)
import { ThemeProvider } from './_components/ThemeProvider';

export default function RootLayout({ children }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <ThemeProvider
          attribute="class"           {/* 给 root 加 .dark class */}
          defaultTheme="system"        {/* 默认跟随系统 */}
          enableSystem
          disableTransitionOnChange    {/* 切换时禁用 transition 防闪烁 */}
        >
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}

// 用 useTheme
'use client';
import { useTheme } from 'next-themes';

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();
  return (
    <Button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      {theme === 'dark' ? '🌙' : '☀️'}
    </Button>
  );
}

关键 suppressHydrationWarning：next-themes 在 client 改 root class（亮 → 暗），SSR 输出与 CSR 不一致。这个 React 属性告诉 React "我知道这里 mismatch，别警告"。

shadcn theme builder · 可视化选色

访问 ui.shadcn.com/themes。

功能：

  • 预设 12 个 theme：zinc（默认）/ slate / stone / red / orange / yellow / green / blue / violet 等
  • 选 base color：调主色调（影响 primary / secondary / muted 等）
  • 选 radius：0 / 0.3rem / 0.5rem / 0.75rem / 1rem
  • 实时预览：右侧组件预览（Card / Button / Form 等）
  • 生成 CSS：复制 :root + .dark 的 CSS variables 替换你的 globals.css

这是 shadcn 设计师 / 非工程师友好工具。设计师可以在 theme builder 里调，最后 copy CSS 给工程师。

多品牌（white-label）

详见 Tailwind P7.2。shadcn 实现方式：

css 
@layer base {
  :root {
    /* 默认（zinc base） */
    --background: 0 0% 100%;
    --primary: 240 5.9% 10%;
    /* ... */
  }

  .theme-acme {
    --primary: 348 83% 47%;  /* Acme 红 */
    --primary-foreground: 0 0% 100%;
  }

  .theme-globex {
    --primary: 142 71% 45%;  /* Globex 绿 */
    --primary-foreground: 0 0% 100%;
  }

  .theme-initech {
    --primary: 263 70% 50%;  /* Initech 紫 */
    --primary-foreground: 0 0% 100%;
  }

  /* 暗色变体 */
  .theme-acme.dark {
    --primary: 348 83% 60%;
  }
}
tsx 
// app/[tenantId]/layout.tsx (Next.js)
async function TenantLayout({ children, params }) {
  const tenant = await getTenant(params.tenantId);
  return (
    <div className={`theme-${tenant.id}`}>
      {children}
    </div>
  );
}

// 用户访问 /acme → .theme-acme 应用 → primary 是红色
// 用户访问 /globex → .theme-globex 应用 → primary 是绿色

Tailwind v4 集成

v4 项目用 @theme（详见 Tailwind P3.2）。shadcn v4 CLI 自动生成 v4 兼容的 globals.css：

css 
/* Tailwind v4 + shadcn · globals.css */
@import "tailwindcss";
@plugin "tailwindcss-animate";

@custom-variant dark (&:where(.dark, .dark *));

:root {
  --background: 0 0% 100%;
  --foreground: 240 10% 3.9%;
  --primary: 240 5.9% 10%;
  --primary-foreground: 0 0% 98%;
  /* ... */
  --radius: 0.5rem;
}

.dark {
  --background: 240 10% 3.9%;
  --foreground: 0 0% 98%;
  --primary: 0 0% 98%;
  --primary-foreground: 240 5.9% 10%;
  /* ... */
}

@theme inline {
  --color-background: hsl(var(--background));
  --color-foreground: hsl(var(--foreground));
  --color-primary: hsl(var(--primary));
  --color-primary-foreground: hsl(var(--primary-foreground));
  /* ... */

  --radius-lg: var(--radius);
  --radius-md: calc(var(--radius) - 2px);
  --radius-sm: calc(var(--radius) - 4px);
}

@layer base {
  * {
    border-color: hsl(var(--border));
  }
  body {
    background: hsl(var(--background));
    color: hsl(var(--foreground));
  }
}

关键 @theme inline：

  • v4 的 @theme 会注册自定义属性（@property）让 transition 工作
  • 但 shadcn 的 token 已经是 CSS variables，不需要 @property 注册
  • @theme inline 让 token 不被注册、直接映射到现有 variable
  • 这是 shadcn v4 适配 Tailwind v4 的关键

HSL vs OKLCH

shadcn 默认用 HSL，Tailwind v4 默认 OKLCH。两者可共存：

  • shadcn token：用 HSL（历史原因 + 配 alpha 简单）
  • Tailwind 默认调色板：v4 改 OKLCH（更现代）
  • 用户写 bg-primary（shadcn token）→ HSL；用 bg-blue-500（Tailwind 默认）→ OKLCH

shadcn v4+ 也支持 OKLCH base：CLI init 时选 "OKLCH"，生成的 globals.css 用 oklch() 而非 hsl()。

选哪个：

  • 团队对 OKLCH 不熟 → HSL（默认）
  • 团队已熟悉 OKLCH 视觉语言 → OKLCH（更现代 + perceptually uniform）

radius 集中管理

shadcn 用 --radius 作为基准（默认 0.5rem），其他 radius 派生：

css 
:root { --radius: 0.5rem; }

@theme inline {
  --radius-lg: var(--radius);              /* 0.5rem */
  --radius-md: calc(var(--radius) - 2px);  /* ~0.375rem */
  --radius-sm: calc(var(--radius) - 4px);  /* ~0.25rem */
}

组件用 rounded-md / rounded-lg / rounded-sm，跟随 --radius。改 --radius 一处全 shadcn 组件视觉变化。

theme builder 提供 5 个 radius 选项（0 / 0.3 / 0.5 / 0.75 / 1rem），用户调整。

03 应用场景

场景 1 · 自定义品牌色

步骤：

  1. 开 ui.shadcn.com/themes
  2. 选 base color（如 violet）
  3. 调 radius（如 0.75rem）
  4. 预览组件
  5. 复制生成的 CSS
  6. 替换项目 globals.css 的 :root + .dark 段

或手动改 globals.css：

css 
:root {
  --primary: 263 70% 50%;  /* 你的品牌紫 */
  --primary-foreground: 0 0% 100%;
}

.dark {
  --primary: 263 70% 60%;  /* 暗色微调 */
}

场景 2 · 加自定义 token

需要 brand-secondary / brand-accent 等额外 token：

css 
/* globals.css */

  /* shadcn 默认 token */
  --background: 0 0% 100%;
  /* ... */

  /* 自定义品牌 token */

  --brand-yellow-foreground: 240 5.9% 10%;

  --gradient-to: 348 83% 60%;

.dark {
  --brand-yellow: 48 96% 60%;
  /* ... */
}
css 
/* @theme 注册 utility */
@theme inline {
  --color-brand-yellow: hsl(var(--brand-yellow));
  --color-brand-yellow-foreground: hsl(var(--brand-yellow-foreground));
  --color-gradient-from: hsl(var(--gradient-from));
  --color-gradient-to: hsl(var(--gradient-to));
}

用：

tsx 
<div className="bg-brand-yellow text-brand-yellow-foreground">

</div>

<div className="bg-gradient-to-r from-gradient-from to-gradient-to">
  ...
</div>

代码 3 · ThemeToggle 含 system 选项

tsx 
'use client';
import { Moon, Sun, Monitor } from 'lucide-react';
import { useTheme } from 'next-themes';
import { useEffect, useState } from 'react';
import {
  DropdownMenu,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuTrigger,
} from '@/components/ui/dropdown-menu';
import { Button } from '@/components/ui/button';

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();
  const [mounted, setMounted] = useState(false);
  useEffect(() => setMounted(true), []);
  if (!mounted) return null;  // 防 SSR mismatch

  return (
    <DropdownMenu>
      <DropdownMenuTrigger asChild>
        <Button variant="ghost" size="icon">
          <Sun className="h-5 w-5 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
          <Moon className="absolute h-5 w-5 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
          <span className="sr-only">Toggle theme</span>
        </Button>
      </DropdownMenuTrigger>
      <DropdownMenuContent align="end">
        <DropdownMenuItem onClick={() => setTheme('light')}>
          <Sun className="mr-2 h-4 w-4" />
          Light
        </DropdownMenuItem>
        <DropdownMenuItem onClick={() => setTheme('dark')}>
          <Moon className="mr-2 h-4 w-4" />
          Dark
        </DropdownMenuItem>
        <DropdownMenuItem onClick={() => setTheme('system')}>
          <Monitor className="mr-2 h-4 w-4" />
          System
        </DropdownMenuItem>
      </DropdownMenuContent>
    </DropdownMenu>
  );
}

import { NextRequest, NextResponse } from 'next/server';

export function middleware(req: NextRequest) {
  const host = req.headers.get('host') || '';
  const subdomain = host.split('.')[0];
  const tenant = ['acme', 'globex', 'initech'].includes(subdomain) ? subdomain : 'default';

  const res = NextResponse.next();
  res.headers.set('x-tenant', tenant);
  return res;
}

// app/layout.tsx
import { headers } from 'next/headers';

export default async function RootLayout({ children }) {
  const tenant = (await headers()).get('x-tenant') || 'default';
  const themeClass = tenant !== 'default' ? `theme-${tenant}` : '';

  return (
    <html className={themeClass} suppressHydrationWarning>
      <body>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}

05 常见陷阱

  • html 上没加 suppressHydrationWarning —— next-themes 在 client 改 root class，与 SSR 输出不一致。React 报 hydration mismatch warning
  • 用 dark: 前缀写组件而不用语义色 —— 重复 + 难维护。改用 bg-primary / text-foreground 等语义 utility
  • shadcn 用 HSL 但你写 OKLCH 值 —— hsl(...) 解析失败 → 颜色不显示。要么全 HSL、要么用 oklch() 函数
  • HSL 写法带括号 —— shadcn 期望 240 10% 3.9%（空格分隔），不是 hsl(240, 10%, 3.9%)。后者在 hsl(var(--x) / 0.5) 中不工作
  • 多品牌不用 .theme-x class 而 build 多套 CSS —— bundle 大 + 切换需要重新加载。用 CSS variables + class 切换
  • Tailwind v4 项目用老 tailwind.config.ts —— 兼容但不推荐。用 @theme inline（详见上文）
  • --radius 改了但组件圆角不变 —— 检查 @theme inline 是否定义了 --radius-lg / md / sm 派生
  • 暗色切换时页面闪一下 —— next-themes 加 disableTransitionOnChange；或 * { transition: none !important } 在 .changing-theme class 内
  • theme builder 生成的 CSS 与项目不兼容 —— v3 / v4 格式不同。手动调或重新跑 shadcn init
  • 自定义 token 不生效 —— 检查 @theme inline 是否注册 --color-{name} 映射
  • shadcn 组件用 hardcoded 颜色不用 token —— 不该。组件源码内部都用 token（bg-primary）；自己改要保持
  • --primary 与 --ring 不一致 —— shadcn 默认 --ring 跟随 --primary（focus 状态用 primary 色）。改 primary 时也改 ring
