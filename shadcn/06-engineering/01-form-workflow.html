Form 完整工作流（RHF + Zod + shadcn Form）

React 表单 2026 事实标准：react-hook-form + Zod + shadcn Form 三方协同。这一章把三方角色、典型 flow、与 Server Actions 集成、动态字段（useFieldArray）、多步表单、file upload 全讲透。读完能搭生产级表单。

阅读时长 ~30 分钟 前置 P4 表单组件 + react-hook-form 基础 主要来源 ui.shadcn.com/docs/components/form + react-hook-form docs + zod docs + 真实生产案例

01 速览

  1. 2026 React 表单事实标准是 react-hook-form (RHF) + Zod + shadcn Form 三方组合：RHF 管表单 state（uncontrolled-first 性能极好）/ Zod 定义 schema 校验（TypeScript-first）/ shadcn Form 把 RHF + Radix 控件 + a11y 包成 wrapper。
  2. shadcn Form 组件链 7 个：Form（FormProvider）/ FormField（单字段 controller + a11y id）/ FormItem（容器）/ FormLabel（label，自动绑 input）/ FormControl（input 容器，传 a11y props 给 child）/ FormDescription（辅助说明）/ FormMessage（错误消息）。
  3. 典型 flow 5 步：1) 定义 Zod schema（含校验规则）→ 2) 用 z.infer 推导 TypeScript 类型 → 3) RHF useForm + zodResolver → 4) shadcn Form 组件 render → 5) onSubmit 处理。
  4. 与 Server Actions 配合：用 useFormStatus / useActionState（详见 Next.js P4.1）+ 客户端 Zod 校验 + 服务端 Zod 再校验（双层防御）+ 错误消息流回 client。这是 2026 全栈表单的事实标准。
  5. 动态字段用 useFieldArray：数组字段（动态添加 todo / 多个邮箱）。提供 fields / append / remove / move 等操作。
  6. 多步表单：用 RHF + 受控 state 管当前步骤；不要用全局 state。每步 submit 只更新 state 不实际提交，最后一步 submit 全部数据。
  7. File upload：FormData multipart + 与 Server Actions 配合。小文件直接走 Action（<1MB），大文件用 presigned URL（详见 Next.js P4.1）。
  8. 表单 a11y：shadcn Form 自动处理 label 绑 input、aria-describedby 指向描述、aria-invalid 错误状态、错误消息自动 announce 给屏幕阅读器。
  9. 与 Combobox / Select / DatePicker 等复杂控件集成：用 RHF 的 Controller（FormField 内部已用），把 onChange / onBlur / value 转给受控组件。
  10. 本章是 shadcn 项目最常踩坑的部分。读完不再用 useState 自己写 form state、不再 typeof formData.email === 'string' 这样校验。

02 底层逻辑

三方角色

react-hook-form（RHF）：

  • 2019 起的 React 表单状态库
  • uncontrolled-first（用 ref 而非 useState，性能极好）
  • API 简洁（useForm / register / handleSubmit）
  • 与各种校验库集成（zod / yup / valibot / 自定义）

Zod：

  • TypeScript-first schema 校验库
  • z.string() / z.number() / z.object() 等定义 schema
  • z.infer<typeof schema> 自动推导 TypeScript 类型
  • 与 RHF 通过 zodResolver 集成

shadcn Form：

  • 把 RHF + shadcn 控件 + a11y 包成 wrapper
  • 不是新概念，是工程整合
  • npx shadcn add form 安装

shadcn Form 组件链

看一个最简登录表单（用 shadcn Form）：

tsx 
'use client';
import { zodResolver } from '@hookform/resolvers/zod';
import { useForm } from 'react-hook-form';
import * as z from 'zod';

import { Button } from '@/components/ui/button';
import {
  Form,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
} from '@/components/ui/form';
import { Input } from '@/components/ui/input';

// 1. 定义 Zod schema
const formSchema = z.object({
  email: z.string().email('Invalid email address'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
});

// 2. z.infer 推导类型
type FormValues = z.infer<typeof formSchema>;

export function LoginForm() {
  // 3. useForm + zodResolver
  const form = useForm<FormValues>({
    resolver: zodResolver(formSchema),
    defaultValues: {
      email: '',
      password: '',
    },
  });

  // 5. onSubmit
  function onSubmit(values: FormValues) {
    // 类型安全：values.email / values.password 都是 string
    console.log(values);
  }

  // 4. render
  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="email"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input placeholder="me@example.com" {...field} />
              </FormControl>
              <FormDescription>
                We'll never share your email.
              </FormDescription>
              <FormMessage />
            </FormItem>
          )}
        />
        <FormField
          control={form.control}
          name="password"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Password</FormLabel>
              <FormControl>
                <Input type="password" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit" disabled={form.formState.isSubmitting}>
          {form.formState.isSubmitting ? 'Logging in...' : 'Log in'}
        </Button>
      </form>
    </Form>
  );
}

关键点：

  • <Form {...form}> 用 spread 传 RHF context（FormProvider）
  • <FormField name="email"> 自动绑定到 schema 的 email 字段（TypeScript 自动检查 name 必须是 schema 字段）
  • render={({ field }) => ...} 暴露 RHF 的 field（含 onChange / onBlur / value / ref / name）
  • <Input {...field} /> 把 field props 传给 input → 受控集成
  • <FormMessage /> 自动显示当前字段错误消息（来自 Zod 校验）
  • 整个表单 a11y 完整（label 绑 input / aria-describedby / aria-invalid 等 shadcn Form 自动处理）

shadcn Form 内部 a11y 自动化

shadcn Form 自动处理的 a11y：

  1. FormLabel ↔ FormControl 自动绑定：
    FormLabel 用 for={fieldId}、FormControl 内 input 自动设 id={fieldId}。点击 label 自动 focus input。
  2. FormDescription：
    自动 id={fieldId}-description，FormControl 内 input 自动 aria-describedby={fieldId}-description。屏幕阅读器读 input 时同时读 description。
  3. FormMessage（错误消息）：
    自动 id={fieldId}-error，FormControl 内 input 自动 aria-describedby={fieldId}-error。错误消息出现时立即被屏幕阅读器读出。
  4. aria-invalid：
    错误状态时 input 自动 aria-invalid="true"。屏幕阅读器读出"invalid entry"。

如果自己写 form（不用 shadcn Form）几乎一定漏 1-2 条。

Zod 校验进阶

Zod 的常用 schema：

typescript 
import * as z from 'zod';

// 字符串
z.string()
  .min(1, 'Required')
  .max(100, 'Too long')
  .email('Invalid email')
  .url('Invalid URL')
  .regex(/^[a-z]+$/, 'Only lowercase letters');

// 数字
z.number()
  .int()
  .positive()
  .min(0)
  .max(100);

// 布尔
z.boolean();

// 枚举
z.enum(['admin', 'user', 'guest']);

// 对象
z.object({
  name: z.string(),
  age: z.number(),
});

// 数组
z.array(z.string()).min(1, 'At least one tag');

// 可选
z.string().optional();

// nullable
z.string().nullable();

// 联合
z.union([z.string(), z.number()]);

// 自定义
z.string().refine((val) => val.endsWith('.com'), {
  message: 'Must end with .com',
});

// 依赖其他字段
z.object({
  password: z.string(),
  confirmPassword: z.string(),
}).refine((data) => data.password === data.confirmPassword, {
  message: "Passwords don't match",
  path: ['confirmPassword'],  // 错误显示在哪个字段
});

// transform（解析时转换）
z.string().transform((val) => val.trim().toLowerCase());

// async refine（异步校验，比如检查 username 是否已存在）
z.string().refine(async (val) => {
  const exists = await checkUsernameExists(val);
  return !exists;
}, { message: 'Username already taken' });

与 Server Actions 集成

2026 React 表单 + Server Actions 的标准模式（详见 Next.js P4.1）：

  1. 客户端用 RHF + Zod 即时校验（提升 UX）
  2. 提交时调 Server Action
  3. Server Action 内用同一个 Zod schema再校验（双层防御）
  4. Server Action 错误流回 client（用 return 不 throw）
  5. 客户端展示错误（FormMessage 或 toast）
typescript 
// schemas/contact.ts（客户端 + 服务端共享）
import * as z from 'zod';

export const contactSchema = z.object({
  email: z.string().email(),
  message: z.string().min(10).max(1000),
});

export type ContactFormValues = z.infer<typeof contactSchema>;
typescript 
// app/_actions/contact.ts
'use server';
import { contactSchema } from '@/schemas/contact';

type State = { errors?: Record<string, string[]>; success?: boolean };

export async function submitContact(prev: State, formData: FormData): Promise<State> {
  const parsed = contactSchema.safeParse({
    email: formData.get('email'),
    message: formData.get('message'),
  });

  if (!parsed.success) {
    return { errors: parsed.error.flatten().fieldErrors };
  }

  await db.contacts.create({ data: parsed.data });
  return { success: true };
}
tsx 
// app/_components/ContactForm.tsx
'use client';
import { useActionState } from 'react';
import { zodResolver } from '@hookform/resolvers/zod';
import { useForm } from 'react-hook-form';
import { contactSchema, type ContactFormValues } from '@/schemas/contact';
import { submitContact } from '@/app/_actions/contact';

export function ContactForm() {
  const [serverState, formAction, isPending] = useActionState(submitContact, {});

  const form = useForm<ContactFormValues>({
    resolver: zodResolver(contactSchema),
    defaultValues: { email: '', message: '' },
  });

  // 客户端校验后调 server action
  function onSubmit(values: ContactFormValues) {
    const formData = new FormData();
    formData.append('email', values.email);
    formData.append('message', values.message);
    formAction(formData);  // 触发 useActionState
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)}>
        <FormField name="email" /* ... */ />
        <FormField name="message" /* ... */ />
        <Button disabled={isPending}>{isPending ? 'Sending...' : 'Send'}</Button>
        {serverState.success && <p>Sent!</p>}
      </form>
    </Form>
  );
}

动态字段 useFieldArray

数组字段（动态 todo / 多个邮箱）用 useFieldArray：

tsx 
'use client';
import { useFieldArray, useForm } from 'react-hook-form';
import * as z from 'zod';
import { zodResolver } from '@hookform/resolvers/zod';

const schema = z.object({
  emails: z.array(z.object({
    address: z.string().email(),
  })).min(1, 'At least one email'),
});

export function MultiEmailForm() {
  const form = useForm({
    resolver: zodResolver(schema),
    defaultValues: { emails: [{ address: '' }] },
  });

  const { fields, append, remove, move } = useFieldArray({
    control: form.control,
    name: 'emails',
  });

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit((data) => console.log(data))}>
        {fields.map((field, index) => (
          <div key={field.id}>  {/* field.id 稳定 key（必须） */}
            <FormField
              name={`emails.${index}.address`}  {/* 嵌套字段名 */}
              control={form.control}
              render={({ field }) => (
                <FormItem>
                  <FormLabel>Email {index + 1}</FormLabel>
                  <FormControl>
                    <Input {...field} />
                  </FormControl>
                  <FormMessage />
                </FormItem>
              )}
            />
            <Button type="button" onClick={() => remove(index)}>Remove</Button>
          </div>
        ))}
        <Button type="button" onClick={() => append({ address: '' })}>Add email</Button>
        <Button type="submit">Submit</Button>
      </form>
    </Form>
  );
}

关键点：

  • field.id 是 RHF 提供的稳定 key（不用 index）
  • 嵌套字段名 emails.${index}.address
  • append / remove / move 等方法操作数组
  • Zod schema 用 z.array 描述

多步表单（multi-step form）

3 步注册表单：

  1. 第 1 步：基本信息（姓名 / 邮箱）
  2. 第 2 步：账户安全（密码 / 安全问题）
  3. 第 3 步：偏好（语言 / 通知）
  4. 最后一步 submit 提交全部数据

策略：

  • 用 RHF 单 useForm 管所有字段（不分多个 useForm）
  • 用 useState 管当前步骤索引
  • 每步显示对应字段
  • "Next"按钮只校验当前步字段（用 trigger）；"Back"无校验
  • 最后一步 submit 才真实提交
tsx 
function MultiStepForm() {
  const [step, setStep] = useState(0);
  const form = useForm({
    resolver: zodResolver(fullSchema),
    defaultValues: { name: '', email: '', password: '', language: 'en' },
  });

  const stepFields = [
    ['name', 'email'],          // step 0
    ['password'],               // step 1
    ['language'],               // step 2
  ];

  async function handleNext() {
    const valid = await form.trigger(stepFields[step]);  // 仅校验当前步
    if (valid) setStep((s) => s + 1);
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)}>
        {step === 0 && <StepBasicInfo />}
        {step === 1 && <StepSecurity />}
        {step === 2 && <StepPreferences />}

        <div className="flex gap-2">
          {step > 0 && <Button onClick={() => setStep(s => s - 1)}>Back</Button>}
          {step < 2 && <Button onClick={handleNext}>Next</Button>}
          {step === 2 && <Button type="submit">Submit</Button>}
        </div>
      </form>
    </Form>
  );
}

File upload

小文件（<1MB）走 Server Action multipart：

tsx 
// schema 接受 File 类型
const schema = z.object({
  avatar: z.instanceof(File).refine(
    (file) => file.size < 1_000_000,
    'File must be less than 1MB'
  ),
});

function AvatarUpload() {
  const form = useForm({ resolver: zodResolver(schema) });

  function onSubmit(values) {
    const formData = new FormData();
    formData.append('avatar', values.avatar);
    formAction(formData);
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)}>
        <FormField
          name="avatar"
          render={({ field: { onChange, value, ...field } }) => (
            <FormItem>
              <FormLabel>Avatar</FormLabel>
              <FormControl>
                <Input
                  type="file"
                  accept="image/*"
                  onChange={(e) => onChange(e.target.files?.[0])}
                  {...field}
                />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit">Upload</Button>
      </form>
    </Form>
  );
}

大文件用 presigned URL（详见 Next.js P4.1）：

  1. 客户端先调 Server Action 获取 presigned URL
  2. 客户端 PUT 文件到 S3 / CF R2 等
  3. 客户端调 Server Action 通知"上传完成 + 文件 URL"
  4. 不通过 Next.js server（节省带宽）

代码 1 · 完整登录表单（含 Server Action）

typescript 
// schemas/auth.ts
import * as z from 'zod';

export const loginSchema = z.object({
  email: z.string().email('Invalid email'),
  password: z.string().min(8, 'Min 8 characters'),
});
export type LoginFormValues = z.infer<typeof loginSchema>;

// app/_actions/auth.ts
'use server';
import { loginSchema } from '@/schemas/auth';
import { redirect } from 'next/navigation';

export async function login(prev: any, formData: FormData) {
  const parsed = loginSchema.safeParse({
    email: formData.get('email'),
    password: formData.get('password'),
  });
  if (!parsed.success) return { errors: parsed.error.flatten().fieldErrors };

  const user = await authenticate(parsed.data.email, parsed.data.password);
  if (!user) return { errors: { _form: ['Invalid credentials'] } };

  await createSession(user.id);
  redirect('/dashboard');
}

05 常见陷阱

  • 用 useState 自己管 form state —— 性能差 + 没有 a11y。用 RHF
  • 不用 zodResolver 而是手写校验 —— 类型不安全 + 校验逻辑分散。用 Zod schema
  • 客户端校验后不在服务端再校验 —— 攻击者可绕过客户端。必须双层 Zod 校验
  • FormField name 拼错 —— TypeScript 不报错（FormField name 是 string）。RHF 8+ 用 generic 推导让 name 类型安全
  • useFieldArray key 用 index 不用 field.id —— React 重新渲染时 key 变，state 丢失。用 field.id
  • 多步表单 next 按钮直接进步骤不校验 —— 用 form.trigger(fieldsForCurrentStep) 校验当前步
  • file input 用 register 而不是 Controller —— file input 的 value 不能直接受控（浏览器限制）。用 Controller + onChange={(e) => field.onChange(e.target.files?.[0])}
  • refine 异步校验阻塞 UI —— async refine 慢时表单 lag。用 onBlur 触发 / debounce
  • Zod schema 客户端 + 服务端不同步 —— 维护两份。共享 schema 文件 (schemas/contact.ts)
  • 用 controlled input 但没传 onChange —— React 报 warning。FormField render 解构 field 的 onChange / value 传给 input
  • defaultValues 不写 —— useForm 没初始值，input 从 undefined 变 "" 时 React 报 controlled/uncontrolled 切换 warning
  • FormMessage 不显示错误 —— 检查 zodResolver 是否正确传入；检查 FormField name 是否匹配 schema 字段
  • Server Action redirect 在 try/catch 内 —— redirect 用 throw 实现，try 会吃掉。redirect 放 try 之外
  • 用 Server Action 但客户端没用 useActionState —— pending state / 错误 handling 自己管，繁琐。用 useActionState
