P5 · CLI + Registry ⭐

shadcn 模式真正的护城河不是组件本身，而是 CLI + Registry 工具链。这两章把 CLI v4 多框架 scaffold + Visual Builder + Registry 数据格式 + 自建私有 registry + monorepo 共享 design system 全讲清。读完能搭团队级 design system 工具链。

章节数 2 ⭐ 主要来源 ui.shadcn.com/docs CLI/Registry + 2026 changelog v4 / blocks / radix-ui 状态 ✅ 全部完成

本阶段覆盖

  • CLI v4 命令 · init / add / diff / build / create
  • 多框架 scaffold · Next.js / Vite / Laravel / React Router / Astro / TanStack Start
  • Visual Project Builder · ui.shadcn.com/create 浏览器配置 + 生成 zip / GitHub 仓库
  • components.json · style / engine / path / Tailwind 配置 / 自定义 registry URL
  • shadcn diff · 看 upstream 改动 + 决定是否 sync
  • Registry 数据格式 · registry-item.json schema (name/files/dependencies/cssVars/tailwind/docs)
  • 1300+ Blocks · dashboard / login / sidebar / form 等组合（Radix + Base UI 双变体）
  • 自建私有 registry · shadcn build → 部署到 Vercel/CF Pages/S3
  • monorepo 共享 design system · 包模式 vs 复制模式两种取舍
  • Theme 分发 · 完整 design token 作为 registry item
