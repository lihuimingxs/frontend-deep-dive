CLI v4 + Visual Builder + components.json

shadcn CLI 是 shadcn 模式的核心工具。这一章把 v4 完整命令集（init / add / diff / build / create）+ 多框架 scaffold（Next.js / Vite / Laravel / React Router / Astro / TanStack Start）+ 2026 新增 Visual Project Builder + components.json 配置 schema 讲透。读完能在团队推 shadcn 工作流。

阅读时长 ~25 分钟 前置 P1 项目史 + 用过 shadcn add 主要来源 ui.shadcn.com/docs/cli + 2026-03 changelog v4 + InfoQ Visual Builder 报道

01 速览

  1. shadcn CLI v4 的 5 个核心命令：init（起新项目 / 加 shadcn 到现有项目）/ add（加单个组件 / block）/ diff（看 upstream 改动 + 决定是否 sync）/ build（build 私有 registry）/ create（Visual Builder，开浏览器配置）。
  2. v4 多框架 scaffold（2026-03 新）：支持 Next.js / Vite / Laravel / React Router / Astro / TanStack Start 6 个框架。每个框架自己模板（router 集成 / styling / dark mode）。shadcn init 自动检测项目类型或问用户。
  3. Visual Project Builder（2026-02 推出，3 月正式 GA）：shadcn create 命令开浏览器到 ui.shadcn.com/create，可视化配置 framework / style / base color / 选组件 / theme，配完生成 zip 或 GitHub 仓库。
  4. components.json 是项目级配置文件（项目根）。核心字段：style（default / new-york）/ engine（radix / base-ui，2026 新）/ tailwind（config 路径 + cssVariables 配置）/ aliases（@/ vs ~/ 等路径别名）/ registries（自定义 registry URL）。
  5. CLI 内部工作流：1) 读 components.json；2) 从 registry（默认 ui.shadcn.com 或自定义）拉组件 source；3) 处理 import path 别名；4) 自动 npm install 缺失依赖；5) 写文件到 components/ui/。
  6. shadcn diff 是 sync upstream 改进的关键命令。show 你本地组件源码与 shadcn upstream 之间的差异。决定是否手动 merge upstream 改进（特别是 a11y 修复 / bug 修复）。
  7. 双引擎切换（2026 新）：components.json 设 "engine": "radix" 或 "engine": "base-ui"，CLI 根据这个拉对应变体。一次切换全部组件统一变化。
  8. style 选项：default（极简、默认）/ new-york（更精致、Vercel 风格）。两个 style 提供同样组件 API，但样式不同。一旦选定不要混用。
  9. 多框架 scaffold 让 shadcn 从"Next.js 专属"扩展为"React 项目通用"。Laravel + Inertia / TanStack Start / Astro 都是一等支持。
  10. 本章是"shadcn 工程化工作流"的核心。读完能起新 shadcn 项目 + 维护组件代码 + 同步 upstream 改进。

02 底层逻辑

5 个核心命令

1. shadcn init · 起新项目或加 shadcn 到现有项目

bash 
npx shadcn@latest init

# CLI 问几个问题：
# - 项目用 TypeScript 吗？(Y/n)
# - 选 style：default / new-york
# - 选 engine：radix（默认）/ base-ui
# - 选 base color：zinc / slate / stone / gray / neutral
# - CSS 变量 / Tailwind class（默认 CSS 变量）
# - tailwind.config.{ts,js} 路径
# - 全局 CSS 路径（默认 src/app/globals.css）
# - 路径别名（默认 @/components / @/lib 等）

# 跑完生成：
# - components.json
# - lib/utils.ts（cn helper）
# - 更新 globals.css（CSS variables）
# - 更新 tailwind.config（base 色集成）
# - 装 deps：tailwindcss-animate / class-variance-authority / clsx / tailwind-merge / lucide-react

2. shadcn add · 加单个组件 / block / theme

bash 
# 加单个组件
npx shadcn@latest add button

# 加多个
npx shadcn@latest add button card dialog form input

# 加 block（组合）
npx shadcn@latest add @blocks/dashboard-01

# 从自定义 registry 加
npx shadcn@latest add https://my-registry.com/registry-item.json

# CLI 做：
# 1. 读 components.json 决定 engine / aliases
# 2. 从 registry 拉对应组件源码
# 3. 检查依赖（registryDependencies + dependencies）
# 4. 自动加缺失的 sub-components（如 dialog 依赖 button）
# 5. 自动 npm install 缺失 npm 包
# 6. 写文件到 components/ui/

3. shadcn diff · 看 upstream 改动

bash 
# 检查 button 与 upstream 差异
npx shadcn@latest diff button

# 输出（git diff 风格）：
# --- a/components/ui/button.tsx (your local)
# +++ b/components/ui/button.tsx (upstream)
# @@ -10,7 +10,7 @@
# -      'inline-flex items-center justify-center rounded-md text-sm font-medium ring-offset-background transition-colors',
# +      'inline-flex items-center justify-center rounded-md text-sm font-medium transition-colors focus-visible:ring-2',

# 你决定是否 merge upstream 改进
# 偶尔跑（每 1-3 月）catch a11y / bug 修复

4. shadcn build · build 私有 registry（详见 P5.2）

bash 
# 在你的 registry 项目根
npx shadcn@latest build

# 把 components/registry/ 编译为 JSON
# 输出到 public/r/ 目录
# 部署到 Vercel / CF Pages / 任何静态文件 server

5. shadcn create · Visual Builder

bash 
npx shadcn@latest create

# 开浏览器到 ui.shadcn.com/create
# 可视化配置：
# - framework
# - style
# - base color
# - theme variables（颜色 / radius / fonts）
# - 选组件（多选）
# - 选 blocks（多选）
# - 项目名

# 配完点 "Generate"：
# - 生成 zip 下载
# - 或 push 到 GitHub 仓库
# - 含完整项目（不只是 shadcn 配置）

多框架 scaffold（2026 v4 新）

v4 之前 shadcn init 主要支持 Next.js + Vite。v4 扩展到 6 个框架：

框架scaffold 内容
Next.jsApp Router + dark mode（next-themes）+ Tailwind 集成 + 推荐 ESLint 配置
ViteVite + React + dark mode + Tailwind
LaravelLaravel + Inertia + React + Tailwind（@laravel/inertia-react）
React RouterReact Router v7 + Tailwind
AstroAstro + React Islands + Tailwind
TanStack StartTanStack Start + Tailwind + 类型安全路由

每个框架的"细节差异"：

  • Next.js：路径别名 @/、global CSS 在 app/globals.css、'use client' 标记需要
  • Vite：路径别名 @/（vite.config 配 alias）、global CSS 在 src/index.css、不需要 'use client'
  • Astro：组件用在 React Islands；style 通过 .astro 文件 import
  • Laravel：路径别名 @/（resources/js）、shadcn 与 Inertia 配合
  • TanStack Start：自动适配 TanStack Router 类型

Visual Project Builder · ui.shadcn.com/create

2026-02 推出，2026-03 正式 GA。是 shadcn 从"开发者工具"扩展为"设计师友好工具"的尝试。

工作流：

  1. 开浏览器到 ui.shadcn.com/create（或 shadcn create 命令打开）
  2. 选 framework（Next.js / Vite / TanStack Start）
  3. 选 style（default / new-york）
  4. 选 engine（Radix / Base UI）
  5. 选 base color（实时预览）
  6. 调 theme variables（radius / fonts）
  7. 选要的组件（Button / Card / Dialog 等）
  8. 选 blocks（dashboard / login form 等）
  9. "Generate" → 下载 zip 或 push GitHub

解决的问题：

  • 新项目快速 scaffold（不用记 init 一堆问题）
  • 预览主题颜色（实时看效果再确定）
  • 设计师可参与（不需要懂 CLI）

2026 仍在迭代。未来可能扩展到"组件级编辑"（拖拽改样式 → 生成代码）。

components.json 完整 schema

json 
{
  "$schema": "https://ui.shadcn.com/schema.json",

  // shadcn 风格
  "style": "default",  // "default" | "new-york"

  // 引擎选择（2026 新）
  "engine": "radix",  // "radix" | "base-ui"

  // 是否 RSC 项目（影响生成的 'use client' 标记）
  "rsc": true,

  // TypeScript / JS
  "tsx": true,

  // Tailwind 配置
  "tailwind": {
    "config": "tailwind.config.ts",  // tailwind.config 路径
    "css": "src/app/globals.css",     // 全局 CSS 路径
    "baseColor": "zinc",              // base color
    "cssVariables": true,             // 用 CSS variables（推荐）
    "prefix": ""                      // 可选 utility prefix
  },

  // 路径别名（与 tsconfig paths / vite alias 对应）
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },

  // 自定义 registry（可选，详见 P5.2）
  "registries": {
    "@my-org": {
      "url": "https://my-registry.com/r/{name}.json",
      "headers": {
        "Authorization": "Bearer ${REGISTRY_TOKEN}"
      }
    }
  }
}

关键字段说明：

style：

  • default：极简、最常用、起步项目首选
  • new-york：更精致（Vercel 风格）、特别表单组件视觉更优
  • 选定后不要换（混用导致风格不一致）

engine（2026 新）：

  • radix：默认、Radix UI 引擎、组件多 + 生态成熟
  • base-ui：Base UI 引擎（MUI 出品）、bundle 更小
  • 切换后下次 add 拉对应变体

rsc：

  • true：Next.js App Router 项目，shadcn 自动加 'use client' 在需要的组件
  • false：Vite SPA 等，不加 'use client'

cssVariables：

  • true（默认推荐）：用 CSS variables 定义 theme（详见 P6.2）
  • false：用 Tailwind class 直接（不灵活，不推荐）

aliases：影响生成代码的 import 路径。必须与 tsconfig paths 一致。

CLI 内部工作流（详细）

跑 shadcn add button 时 CLI 内部做：

  1. 读 components.json：决定 style / engine / rsc / aliases
  2. 解析 registry URL：默认 https://ui.shadcn.com/r/{style}/{name}.json；自定义 registry 用 components.json 配置
  3. fetch registry-item.json：包含组件源码 + dependencies + tailwind / cssVars 配置
  4. 解析 registryDependencies：button 依赖 Slot / cn → 自动 add 这些
  5. 解析 dependencies：button 用 lucide-react → 检查是否已装；没装则 npm install
  6. 处理 import 路径：把 registry 中的占位 @/components 替换为 components.json 配的实际别名
  7. 处理 'use client'：rsc: true 时给需要的组件加 'use client' 标记
  8. 处理 engine 替换：engine 是 base-ui 时拉对应变体的源码
  9. 写文件：组件源码到 components/ui/{name}.tsx；可能还有 hooks 到 hooks/use-{name}.ts
  10. 更新 tailwind.config / globals.css：如果组件需要新的 keyframes / CSS variables
  11. 输出：log 哪些文件被创建 / 修改

shadcn diff · 同步 upstream 改进

shadcn copy-paste 的代价是"不能 npm update"。但 upstream 仍在改进（a11y 修复 / bug fix / 新功能）。shadcn diff 帮你 catch 这些。

使用模式：

  1. 每 1-3 月跑一次 shadcn diff
  2. 看哪些组件有 upstream 改动
  3. 读 diff 决定是否 merge：
    • a11y 修复 / bug fix → 强烈建议 merge
    • 新功能 → 看是否需要
    • 样式调整 → 看是否符合你团队设计
    • API 变化 → 谨慎评估是否有 breaking change
  4. 手动 copy 改动到 components/ui/{name}.tsx
  5. git commit + 测试

这个流程比 npm update 更费力，但完全可控。换来"组件不会被远程包破坏"的好处。

选 default 还是 new-york style

两个 style 都基于 Radix + Tailwind + CVA，区别在视觉 token：

  • default：圆角偏小（rounded-md / rounded-sm）、间距紧凑、颜色克制
  • new-york：圆角更大（rounded-lg）、间距宽松、颜色对比强

视觉上 new-york 更接近 Vercel.com 的设计语言。default 更通用。

选哪个看团队偏好。一个项目内不要混（导致风格不一致）。

03 应用场景

场景 1 · 起新 React SaaS 项目

2026 推荐工作流：

bash 
# 选 1：用 Visual Builder（设计师友好）
# 浏览器开 ui.shadcn.com/create
# 配 + 下载 zip

# 选 2：用 CLI 命令（开发者熟练）
npm create vite@latest my-saas -- --template react-ts
cd my-saas
npm install tailwindcss @tailwindcss/vite

# 配 Vite plugin（详见 Tailwind 主题 P3.2）
# vite.config.ts 加 tailwindcss() plugin

# 起 shadcn
npx shadcn@latest init
# 选 default / radix / zinc / cssVariables: true

# 加常用组件
npx shadcn@latest add button card dialog form input label dropdown-menu tooltip toaster

# 开始写业务代码

场景 2 · 现有 Next.js 项目接 shadcn

bash 
cd my-existing-nextjs-app

# 装 shadcn
npx shadcn@latest init
# 自动检测 Next.js + 现有 Tailwind 配置
# 你确认 style / engine / base color

# 加组件
npx shadcn@latest add button

# 注意：
# - 检查 components.json 生成的 aliases 与 tsconfig paths 一致
# - 检查 globals.css 的 CSS variables 是否覆盖你已有 theme
# - shadcn 生成的 tailwind.config 修改可能与你已有 plugin 冲突

场景 3 · 切 default → new-york style

不推荐切（混用风格差）。如果非切不可：

  1. 更新 components.json："style": "new-york"
  2. 对每个已 add 的组件重新 add（覆盖）：npx shadcn add button --overwrite
  3. visual regression 测试
  4. 调整自定义样式

场景 4 · 维护组件库 sync upstream

每季度（或更频繁）跑：

bash 
# 检查所有组件的 upstream 差异
for component in button card dialog form input; do
  echo "=== $component ==="
  npx shadcn@latest diff $component
done

# 或检查全部
npx shadcn@latest diff

# 决定 merge 哪些改进
# 手动 copy 到 components/ui/
# git commit + 测试

bash

代码 3 · CLI 自动化（CI 集成）

yaml 
# .github/workflows/sync-shadcn.yml
# 每月自动跑 shadcn diff，PR 提示 upstream 改进

name: Sync shadcn upstream
on:
  schedule:
    - cron: '0 0 1 * *'  # 每月 1 号
  workflow_dispatch:

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install
      - run: |
          OUTPUT=$(npx shadcn@latest diff)
          if [[ -n "$OUTPUT" ]]; then
            echo "$OUTPUT" > diff-output.txt
            # 创建 issue 通知
          fi
      - uses: peter-evans/create-issue-from-file@v4
        with:
          title: "shadcn upstream changes detected"
          content-filepath: diff-output.txt

05 常见陷阱

  • aliases 与 tsconfig paths 不一致 —— 生成代码 import 失败。修一个忘了改另一个
  • shadcn add 后忘了 commit components.json —— 团队其他成员 add 时配置不一致
  • cssVariables: false 然后想用 theme —— 不灵活。改 cssVariables: true
  • style 选定后想换 —— 切换需要重新 add 所有组件 + visual regression。从一开始选定
  • engine 切换不重新 add 组件 —— 老 Radix 组件代码与 Base UI 心智混合，bug 满天飞
  • 没装 lucide-react 但 add 用 icon 的组件 —— CLI 自动装；但用其他 icon 库（heroicons）要手动改
  • shadcn diff 改动多不知道哪些重要 —— 优先 merge a11y / 安全相关；其他评估
  • 用第三方 v0.dev / Subframe 输出 shadcn 代码不跑 init —— components.json 没建，代码与项目 alias 不一致
  • Visual Builder 生成的项目缺自己的依赖 —— Visual Builder 提供基础 scaffold，团队的特殊依赖（数据库 / auth）要手动加
  • Next.js App Router 项目设 rsc: false —— 'use client' 标记不会自动加，组件用了 hooks 后报错
  • 勿动 tailwind.config 的 shadcn 自动生成段 —— 注释 "// Generated by shadcn" 段，shadcn add 时会更新
  • CLI 失败 retry 把组件源码搞乱 —— 跑前 git commit 干净状态，失败 git checkout 恢复
  • 用旧版本 CLI（v3） —— 一些 v4 命令不可用。npm i -g shadcn@latest
