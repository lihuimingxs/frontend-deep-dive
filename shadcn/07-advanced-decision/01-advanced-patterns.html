高级模式

基础 shadcn 组件不够的场景：大表（TanStack Table）/ Toast（Sonner）/ 虚拟列表（TanStack Virtual）/ 命令面板（cmdk）/ Drawer（Vaul）/ 可调面板（react-resizable-panels）/ Carousel（embla）。这一章把 7 个最常用高级模式讲清，让你在 shadcn 项目搭出复杂 UI。

阅读时长 ~25 分钟 前置 P3 浮层 + P4 表单 主要来源 各高级库官方 docs + shadcn DataTable 模板

01 速览

  1. shadcn 高级模式 7 个：TanStack Table（大表）/ Sonner（Toast）/ TanStack Virtual（虚拟列表）/ cmdk（命令面板）/ Vaul（Drawer）/ react-resizable-panels（可调面板）/ embla-carousel（Carousel）。
  2. TanStack Table v8（Tanner Linsley）：headless 表格状态库（与 Radix 哲学一致）。提供分页 / 排序 / 过滤 / 列调整 / 虚拟化等。shadcn 提供 DataTable 模板（用 shadcn Table DOM + TanStack Table 状态）。1300+ blocks 含 dashboard 模板基于 DataTable。
  3. Sonner（emil_kowalski）：替代 Radix Toast。API 更简单（toast.success("Saved!")）+ 多 toast 堆叠 + promise toast（loading → success / error）+ 自定义 toast。shadcn 推荐 Sonner over Radix Toast（2024 后默认）。
  4. TanStack Virtual：headless 虚拟化（与 Table 同生态）。大列表（10k+ 项）必备。与 Combobox / Select / Table 配合渲染百万项不卡。react-virtuoso 是组件式虚拟化替代方案。
  5. cmdk（Pkmn / Linear 团队）：命令面板（⌘K 弹窗 + 搜索 + 选项）+ 也作为 Combobox 基础（详见 P4.1）。Linear / Vercel / Raycast 等用 cmdk。
  6. Vaul（emil_kowalski）：移动端 bottom sheet。比 Dialog 更适合移动端（拉手柄上滑 / 下拉关）。shadcn 加 Drawer 组件基于 Vaul。
  7. react-resizable-panels：可调大小面板。VS Code / Linear 风格的拖拽分栏。shadcn ResizablePanelGroup 组件基于它。
  8. embla-carousel：carousel 库（基于 IntersectionObserver）。shadcn Carousel 组件基于它。轻量 + 性能好 + a11y 完整。
  9. 所有这些库都是 headless 或几乎 headless —— 与 shadcn / Radix 哲学一致。提供行为，用户加样式。
  10. 本章不是"完整 catalog"。选 7 个最常用 + 影响最大的高级模式。其他场景（数据可视化 / 富文本编辑器 / 视频播放器）shadcn 没标准方案，社区生态各自实现。

02 底层逻辑

TanStack Table v8 + shadcn DataTable

TanStack Table 是表格状态库（不是组件库）。提供：

  • 分页（client / server-side）
  • 排序（多列 / 优先级）
  • 过滤（global / per-column / fuzzy）
  • 列调整（resize / reorder / show-hide）
  • 虚拟化（与 TanStack Virtual 配合）
  • 列固定（pinning）
  • row selection
  • row expansion
  • 分组 / 聚合

headless 含义：不提供 DOM。用户自己写 <table><thead><tr><th> 等 markup，TanStack Table 提供 column / row / cell 状态。

shadcn 提供 DataTable 模板：用 shadcn Table（DOM） + TanStack Table（状态） + 内置分页 / 过滤 UI。CLI 加：

bash 
npx shadcn@latest add table
# 加基础 Table + TanStack Table 集成示例

典型 DataTable 代码（简化）：

tsx 
'use client';
import {
  ColumnDef,
  flexRender,
  getCoreRowModel,
  getPaginationRowModel,
  getSortedRowModel,
  SortingState,
  useReactTable,
} from '@tanstack/react-table';
import {
  Table, TableBody, TableCell, TableHead, TableHeader, TableRow,
} from '@/components/ui/table';
import { Button } from '@/components/ui/button';
import { useState } from 'react';

interface DataTableProps<TData, TValue> {
  columns: ColumnDef<TData, TValue>[];
  data: TData[];
}

export function DataTable<TData, TValue>({ columns, data }: DataTableProps<TData, TValue>) {
  const [sorting, setSorting] = useState<SortingState>([]);

  const table = useReactTable({
    data,
    columns,
    state: { sorting },
    onSortingChange: setSorting,
    getCoreRowModel: getCoreRowModel(),
    getPaginationRowModel: getPaginationRowModel(),
    getSortedRowModel: getSortedRowModel(),
  });

  return (
    <>
      <Table>
        <TableHeader>
          {table.getHeaderGroups().map((headerGroup) => (
            <TableRow key={headerGroup.id}>
              {headerGroup.headers.map((header) => (
                <TableHead key={header.id}>
                  {flexRender(header.column.columnDef.header, header.getContext())}
                </TableHead>
              ))}
            </TableRow>
          ))}
        </TableHeader>
        <TableBody>
          {table.getRowModel().rows.map((row) => (
            <TableRow key={row.id}>
              {row.getVisibleCells().map((cell) => (
                <TableCell key={cell.id}>
                  {flexRender(cell.column.columnDef.cell, cell.getContext())}
                </TableCell>
              ))}
            </TableRow>
          ))}
        </TableBody>
      </Table>

      {/* 分页 */}
      <div className="flex items-center gap-2 mt-4">
        <Button onClick={() => table.previousPage()} disabled={!table.getCanPreviousPage()}>
          Previous
        </Button>
        <Button onClick={() => table.nextPage()} disabled={!table.getCanNextPage()}>
          Next
        </Button>
      </div>
    </>
  );
}

用法：

tsx 
// 定义 columns
type User = { id: string; name: string; email: string };

const columns: ColumnDef<User>[] = [
  { accessorKey: 'name', header: 'Name' },
  { accessorKey: 'email', header: 'Email' },
];

// 用
<DataTable columns={columns} data={users} />

Sonner · 替代 Radix Toast

Sonner 由 emil_kowalski（Vaul / cal.com 等核心开发者）出品。2024 年 shadcn 默认推荐 Sonner over Radix Toast。

原因：

  • API 更简单：toast.success("Saved!") 一行；Radix Toast 要 useState + 控制 open
  • 多 toast 堆叠：自动管理多个 toast 显示位置
  • promise toast：toast.promise(saveData(), { loading: "Saving...", success: "Saved!", error: "Failed" })
  • 动画细腻：Vercel 风格的弹出 / 滑出动画
bash 
npx shadcn@latest add sonner
tsx 
// app/layout.tsx · 加 Toaster
import { Toaster } from '@/components/ui/sonner';

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        {children}
        <Toaster />
      </body>
    </html>
  );
}

// 任意组件用
import { toast } from 'sonner';

function SaveButton() {
  return (
    <Button onClick={() => {
      toast.success('Settings saved!');
    }}>Save</Button>
  );
}

// promise toast
function UploadButton() {
  return (
    <Button onClick={() => {
      toast.promise(uploadFile(), {
        loading: 'Uploading...',
        success: 'Uploaded!',
        error: 'Upload failed',
      });
    }}>Upload</Button>
  );
}

// 自定义 toast
toast(<CustomToast />);

// 错误 toast 含 action
toast.error('Failed to save', {
  action: { label: 'Retry', onClick: () => retry() },
});

TanStack Virtual · 虚拟列表

渲染 10k+ 项的列表会卡（DOM 太多）。虚拟化只渲染可视区域 + 一点 buffer，其他用占位元素撑高度。

TanStack Virtual 是 headless 虚拟化（与 Table 同生态）。

bash 
npm i @tanstack/react-virtual
tsx 
import { useVirtualizer } from '@tanstack/react-virtual';
import { useRef } from 'react';

function VirtualList({ items }: { items: string[] }) {
  const parentRef = useRef<HTMLDivElement>(null);

  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 40,  // 每项预估高度
    overscan: 5,             // 可视区上下各多渲染 5 项
  });

  return (
    <div ref={parentRef} className="h-[400px] overflow-auto">
      <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
        {virtualizer.getVirtualItems().map((virtualRow) => (
          <div
            key={virtualRow.key}
            style={{
              position: 'absolute',
              top: 0, left: 0, right: 0,
              height: virtualRow.size,
              transform: `translateY(${virtualRow.start}px)`,
            }}
          >
            {items[virtualRow.index]}
          </div>
        ))}
      </div>
    </div>
  );
}

替代方案：react-virtuoso（组件式 API，更易用但性能稍差）。

用 TanStack Virtual 的场景：

  • 用户列表 / 商品列表 1000+ 项
  • 聊天历史 / 消息流
  • 大 Combobox（10k 选项）
  • 大 DataTable（10k 行）

cmdk · 命令面板

已在 P4.1 详细讲过 Combobox 用法。这里讲命令面板用法（⌘K 弹窗）：

tsx 
'use client';
import * as React from 'react';
import {
  CommandDialog,
  CommandEmpty,
  CommandGroup,
  CommandInput,
  CommandItem,
  CommandList,
  CommandSeparator,
  CommandShortcut,
} from '@/components/ui/command';

export function CommandPalette() {
  const [open, setOpen] = React.useState(false);

  React.useEffect(() => {
    const down = (e: KeyboardEvent) => {
      if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
        e.preventDefault();
        setOpen((open) => !open);
      }
    };
    document.addEventListener('keydown', down);
    return () => document.removeEventListener('keydown', down);
  }, []);

  return (
    <>
      <p className="text-sm text-muted-foreground">
        Press <kbd>⌘K</kbd> to open command palette
      </p>

      <CommandDialog open={open} onOpenChange={setOpen}>
        <CommandInput placeholder="Type a command or search..." />
        <CommandList>
          <CommandEmpty>No results found.</CommandEmpty>

          <CommandGroup heading="Suggestions">
            <CommandItem onSelect={() => navigate('/dashboard')}>
              <LayoutDashboard className="mr-2 h-4 w-4" />
              Dashboard
              <CommandShortcut>⌘D</CommandShortcut>
            </CommandItem>
            <CommandItem onSelect={() => navigate('/settings')}>
              <Settings className="mr-2 h-4 w-4" />
              Settings
              <CommandShortcut>⌘,</CommandShortcut>
            </CommandItem>
          </CommandGroup>

          <CommandSeparator />

          <CommandGroup heading="Recent">
            {recentItems.map((item) => (
              <CommandItem key={item.id} onSelect={() => openItem(item.id)}>
                {item.title}
              </CommandItem>
            ))}
          </CommandGroup>
        </CommandList>
      </CommandDialog>
    </>
  );
}

命令面板的 a11y：

  • cmdk 自动处理 listbox a11y（aria-activedescendant）
  • 键盘 ↓ ↑ Enter ESC
  • 实时屏幕阅读器读出选项数

Vaul · 移动端 Drawer

Drawer 是移动端"bottom sheet" —— 从底部上滑的浮层。比 Dialog 更适合移动端（用户拇指可达）。

Vaul（也是 emil_kowalski 出品）特点：

  • 拖拽手柄上滑 / 下拉关
  • 嵌套 drawer（drawer 内 drawer）
  • snap points（半屏 / 全屏 / 部分屏）
  • Tailwind 动画兼容
bash 
npx shadcn@latest add drawer
tsx 
import {
  Drawer, DrawerContent, DrawerDescription, DrawerHeader, DrawerTitle, DrawerTrigger,
} from '@/components/ui/drawer';

<Drawer>
  <DrawerTrigger>Open</DrawerTrigger>
  <DrawerContent>
    <DrawerHeader>
      <DrawerTitle>Settings</DrawerTitle>
      <DrawerDescription>Adjust your preferences.</DrawerDescription>
    </DrawerHeader>
    <div className="p-4">...</div>
  </DrawerContent>
</Drawer>

Drawer vs Dialog 选择：

  • 桌面 → Dialog
  • 移动 → Drawer
  • 响应式 → 用 useMediaQuery 切换（桌面 Dialog / 移动 Drawer）

react-resizable-panels · 可调面板

VS Code / Linear 风格的拖拽分栏。shadcn ResizablePanelGroup 基于此：

tsx 
import {
  ResizableHandle,
  ResizablePanel,
  ResizablePanelGroup,
} from '@/components/ui/resizable';

<ResizablePanelGroup direction="horizontal">
  <ResizablePanel defaultSize={20} minSize={15}>
    Sidebar
  </ResizablePanel>
  <ResizableHandle />
  <ResizablePanel defaultSize={80}>
    <ResizablePanelGroup direction="vertical">
      <ResizablePanel defaultSize={70}>Main content</ResizablePanel>
      <ResizableHandle />
      <ResizablePanel defaultSize={30}>Bottom panel</ResizablePanel>
    </ResizablePanelGroup>
  </ResizablePanel>
</ResizablePanelGroup>

用户拖 ResizableHandle 调整 panel 大小。会持久化到 localStorage（autoSaveId 配置）。

embla-carousel · Carousel

embla 是轻量 carousel 库（基于 IntersectionObserver）。shadcn Carousel 基于它：

tsx 
import {
  Carousel,
  CarouselContent,
  CarouselItem,
  CarouselNext,
  CarouselPrevious,
} from '@/components/ui/carousel';

<Carousel>
  <CarouselContent>
    {images.map((img) => (
      <CarouselItem key={img.id}>
        <img src={img.url} />
      </CarouselItem>
    ))}
  </CarouselContent>
  <CarouselPrevious />
  <CarouselNext />
</Carousel>

// 自动播放
<Carousel
  opts={{ loop: true }}
  plugins={[Autoplay({ delay: 3000 })]}
>...</Carousel>

embla 的优势：性能好 / 轻量 / a11y 完整 / 移动端流畅。

03 应用场景

场景 1 · Admin dashboard 用户列表

10k+ 用户的 admin dashboard：

  • DataTable（TanStack Table）+ 分页（server-side）+ 排序 + 全局搜索
  • 列：头像 / 姓名 / 邮箱 / 角色 / 创建时间 / Actions
  • 每行 row.getVisibleCells() 渲染
  • Actions 列含 DropdownMenu（编辑 / 删除等）

场景 2 · 命令面板（⌘K）

SaaS 平台 / 编辑器普及的⌘K 模式：

  • cmdk + CommandDialog
  • 多个 CommandGroup（Recent / Settings / Help）
  • onSelect 跳路由 / 触发 action
  • 支持搜索（cmdk 自带 fuzzy）

场景 3 · 响应式 Drawer / Dialog

同一个组件桌面用 Dialog、移动用 Drawer：

tsx 
function ResponsiveDialog({ children, ...props }) {
  const isDesktop = useMediaQuery('(min-width: 768px)');

  if (isDesktop) {
    return (
      <Dialog {...props}>
        <DialogContent>{children}</DialogContent>
      </Dialog>
    );
  }

  return (
    <Drawer {...props}>
      <DrawerContent>{children}</DrawerContent>
    </Drawer>
  );
}

代码 3 · 虚拟化 Combobox

tsx 
function LargeCombobox({ options }) {
  const [search, setSearch] = useState('');
  const filtered = options.filter((o) => o.name.includes(search));

  const parentRef = useRef<HTMLDivElement>(null);
  const virtualizer = useVirtualizer({
    count: filtered.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 36,
    overscan: 10,
  });

  return (
    <Command shouldFilter={false}>
      <CommandInput onValueChange={setSearch} />
      <CommandList ref={parentRef} className="h-[300px] overflow-auto">
        <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
          {virtualizer.getVirtualItems().map((virtualItem) => (
            <CommandItem
              key={virtualItem.key}
              style={{
                position: 'absolute',
                top: 0, left: 0, right: 0,
                height: virtualItem.size,
                transform: `translateY(${virtualItem.start}px)`,
              }}
              value={filtered[virtualItem.index].id}
              onSelect={() => selectOption(filtered[virtualItem.index])}
            >
              {filtered[virtualItem.index].name}
            </CommandItem>
          ))}
        </div>
      </CommandList>
    </Command>
  );
}

05 常见陷阱

  • 用 TanStack Table 但自己写 table DOM —— 遗漏 a11y 标签（thead / tbody / scope 等）。用 shadcn Table 组件
  • 大列表不用虚拟化 —— 1000+ 项渲染卡。装 TanStack Virtual / react-virtuoso
  • shadcn 项目仍用 Radix Toast —— 默认推荐 Sonner（API 简单 + 多 toast 堆叠）
  • cmdk shouldFilter 没关用 server 端过滤 —— 双重过滤导致结果错。shouldFilter={false}
  • Vaul Drawer 在桌面强用 —— Drawer 是移动端 bottom sheet，桌面用 Dialog
  • useMediaQuery 在 SSR 不工作 —— 用 next-themes 类似的 mounted 检查 + 默认桌面/移动
  • react-resizable-panels 没设 autoSaveId —— 用户调整大小不持久化，每次刷新重置
  • embla autoplay 没暂停 —— hover 时不暂停，用户体验差。embla autoplay plugin 配置 stopOnInteraction
  • DataTable 用 client-side 分页处理 10k+ 数据 —— 慢。改 server-side（manualPagination + onPaginationChange）
  • 命令面板没 ⌘K 快捷键监听 —— 用户找不到入口。加 useEffect + keydown 监听
  • Toast 堆叠但用户看不到所有 —— Sonner 默认上限 3，自动 hide 老的。看 expand 选项
  • 用 v0.dev / Subframe 生成 DataTable 代码不维护 —— 视为模板起点，要 review + 改
  • embla 加多个 plugin 顺序错 —— 部分 plugin 之间有依赖。看 embla docs 推荐顺序
