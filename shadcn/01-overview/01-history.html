项目史 2018-2026

shadcn/ui + Radix 的 8 年演化主线，按"三次范式转移及其根因"讲：2018 Radix 在 Modulz 起步（headless + a11y-first）→ 2023 shadcn 公开（copy-paste 模式 + Tailwind + CVA 整合）→ 2026 双引擎 + Registry 系统（多框架 scaffold + 私有 registry + 1300+ blocks）。从一个小众 headless 库到 React UI 事实标准，这条线讲清能回答"为什么 copy-paste 模式会颠覆 npm package"。

阅读时长 ~30 分钟 前置 用过 React + 听说过 shadcn/ui 主要来源 ui.shadcn.com/docs/changelog + radix-ui.com releases + shadcn / Pedro Duarte 公开访谈

01 速览

  1. shadcn/ui + Radix 8 年由 3 次范式转移组成 —— 2018-2020 Radix 起步（Modulz 公司、headless + WAI-ARIA APG 标准化）→ 2023-03 shadcn copy-paste 革命（Hassan El Mghari 把 Radix + Tailwind + CVA 模板公开为 CLI）→ 2026-02 双引擎 + Registry 系统（Base UI 加入、统一 radix-ui 包、1300+ blocks、Visual Builder）。每次转移都解决前一版的具体痛点。
  2. 2018 年 Modulz 公司（Pedro Duarte / Vlad Moroz 等）启动 Radix 项目。设计立场：headless + a11y-first。当时主流 React UI 库（MUI / Ant Design / Chakra）都是 styled，Radix 押注 unstyled + 完整 a11y 标准实现。
  3. 2020-12 Radix Primitives v1 GA。包含 Dialog / Popover / DropdownMenu / Tooltip / Tabs 等核心 primitives。每个严格按 W3C WAI-ARIA APG 标准实现 —— 这是它最大的工程成就。
  4. 2022 Modulz 推出 Radix Themes（与 Primitives 并行的产品线）—— pre-styled 组件库，给"想要现成样式 + 不想配 Tailwind"的人。但 Themes 市场份额远不如 Primitives。
  5. 2023-03 shadcn 公开。Hassan El Mghari 把"Radix + Tailwind + CVA"模板做成 CLI 工具。最大的创新是 copy-paste 模式 —— 不用 npm install，用 CLI 把组件源码 copy 进项目。这反转了 React 组件库分发模式。
  6. 2024 shadcn 现象级。从开源到 Vercel / Linear / Resend 主流采用只用了 18 个月。GitHub stars 在 2024 年破 50k；2025 年破 70k。新 React 项目份额 60%+。
  7. 2024-09 Modulz 团队整体加入 WorkOS（B2B SaaS 公司）。这给 Radix 长期维护提供了商业支撑（之前 Modulz 是独立公司，财务压力大）。WorkOS 给 Pedro 等核心成员"专职维护 Radix"的全职岗位。
  8. 2025 第三方 shadcn 移植开花：shadcn-vue（Vue 3 + Vee Validate）/ shadcn-svelte（Svelte 5）/ NativeWind（React Native + Tailwind）。但官方 shadcn 仍 React-only。
  9. 2026-02 三件大事：shadcn 加 Base UI 双引擎支持（用户可选 Radix 或 Base UI）/ Radix 出统一包 radix-ui（替代过去几十个 @radix-ui/react-* 子包，tree-shakable）/ 1300+ blocks（dashboard / login / sidebar 等组合，Radix + Base UI 双变体）。
  10. 2026-03 shadcn CLI v4。多框架 scaffold（Next.js / Vite / Laravel / React Router / Astro / TanStack Start）+ Visual Project Builder（ui.shadcn.com/create 浏览器配置 + 生成项目）。这是 shadcn 从"组件库"向"完整项目脚手架"的扩张。

02 底层逻辑

前史 · 2010-2018 · React UI 库的 styled 主流

2013-2018 年是 React UI 库的styled 主流期：

  • Material-UI（MUI）（2014）：Material Design 在 React 实现；快速成为企业级首选
  • Ant Design（2015，蚂蚁金服）：中后台标准；中国生态主流
  • Bootstrap React / React-Bootstrap：把 Bootstrap 包成 React 组件
  • Chakra UI（2019）：基于 Emotion + 自有 props API（<Box bg="blue.500">）
  • Mantine（2021）：与 Chakra 类似但 prop-based 不那么激进

这些库的共同点：组件自带样式，用户改样式要 override theme 或写复杂 CSS-in-JS。优势是开箱即用；劣势是定制成本高 + 设计与库强耦合。

另一阵营是 headless / unstyled：

  • Reach UI（2018，Chance Strickland）：早期 headless 尝试，已停止维护
  • Headless UI（2020，Tailwind Labs）：与 Tailwind 配套；React + Vue
  • Radix Primitives（2018-2020 起步，2020-12 GA）：本主题主角

headless 阵营的赌注：用户不需要库的样式，只需要行为 + a11y + 状态管理。这与 Tailwind utility-first 哲学天然契合（Tailwind 提供样式系统，headless 库提供行为）。

2018-2020 · Radix 在 Modulz 起步

Modulz 是 Pedro Duarte 等人 2018 年创立的公司。最初目标是"设计与代码同步的工具链"——给 designers 一个工具，让他们的设计自动生成 React 代码。

Radix Primitives 最初是 Modulz 内部项目（"我们要给 designers 提供 a11y-first 组件作基础"），2019-2020 年开源。设计立场：

  • Headless：只提供行为 + a11y + 状态，不提供样式
  • WAI-ARIA APG 严格遵循：每个组件按 W3C 标准实现（详见 P2.1）
  • Compound Components：<Dialog.Root><Dialog.Trigger /><Dialog.Content /></Dialog.Root> 模式
  • Forward refs + asChild：让 Radix 与任意自定义组件无缝集成

Radix 团队的 a11y 投入超出业界平均：

  • 每个 primitive 有专门 a11y test suite
  • 用 NVDA / JAWS / VoiceOver 实测
  • 跟踪 WAI-ARIA spec 更新
  • 团队成员包括 a11y 专家（Pedro Duarte 自己也是）

这个投入让 Radix 在 a11y 领域断层式领先同代竞品。即使 Headless UI（Tailwind Labs 出品、技术名声响）也在 a11y 完整度上不如 Radix。

2020-12 · Radix Primitives v1 GA

v1 发布时已包含完整的核心 primitives：

  • Dialog（modal）+ AlertDialog（强制确认）
  • Popover + HoverCard
  • DropdownMenu + ContextMenu + Menubar
  • Tooltip
  • Tabs + Accordion + Collapsible
  • Checkbox + RadioGroup + Switch + Slider
  • Select + NavigationMenu
  • Toast + Toolbar
  • ScrollArea + Separator + AspectRatio
  • Progress + Avatar

v1 不包含的（直到今天仍不完整）：

  • Combobox（!）—— Radix 没自带 Combobox primitive，要用第三方 cmdk 或手写
  • Calendar / DatePicker —— 直到 2024 年才有官方 react-day-picker 配合
  • Toolbar（v1 时简陋）

"Radix 没 Combobox" 这个空缺一直存在 —— shadcn 用 cmdk（Linear 团队 Pkmn 开源）作 Combobox 基础。这是工程权衡，详见 P4.1。

2022 · Radix Themes · pre-styled 产品线

2022 年 Modulz 推出 Radix Themes —— 在 Primitives 基础上加预设样式。设计立场：给"不想用 Tailwind / 想要现成样式"的人。

Radix Themes 与 Radix Primitives 是两个产品：

  • Primitives：headless、用户自己写样式（与 Tailwind 配）
  • Themes：pre-styled、独立 CSS 文件（不依赖 Tailwind）

市场反响：Themes 远不如 Primitives 火。2024 年起 shadcn 接管"Radix + Tailwind + CVA"赛道，Radix Themes 进入维护期。但仍是 Modulz / WorkOS 官方支持的产品。

本主题不深入讲 Radix Themes（与 shadcn 路线不同）。

2023-03 · shadcn 公开 · copy-paste 革命

2023 年 3 月，Hassan El Mghari（Twitter @shadcn）在 GitHub 公开 shadcn/ui 项目。最初定位是"个人项目模板"——他在自己 SaaS 项目里整合了 Radix + Tailwind + CVA + cn 工具链，整理出来开源。

shadcn 的核心创新：copy-paste 模式。

  • 不发布到 npm；不是 npm package
  • 用 CLI（npx shadcn add button）把组件源码 copy 到用户项目
  • 用户对组件源码完全掌控，可任意修改
  • "不能升级"反而是优势 —— 不会被框架升级破坏

这反转了 React 组件库分发模式。传统模式：

bash 
# 传统：npm install + black box
npm install @radix-ui/react-dialog
# 用 import { Dialog } from '@radix-ui/react-dialog'
# 升级 npm 时可能 breaking change
# 想改样式要 override theme，改不动逻辑

shadcn 模式：

bash 
# shadcn：CLI copy-paste
npx shadcn add dialog
# 组件源码到 components/ui/dialog.tsx
# 完全可见、可改、可 review
# 永远不会被远程包升级破坏

shadcn 公开后头几个月数据并不爆（GitHub 几千 stars）。但 2023 下半年开始，几件事推动它快速普及：

  • Vercel 内部使用并公开（Vercel docs / cal.com 等公开使用）
  • Theo Browne / Lee Robinson 等 React KOL 推荐
  • YouTuber 大量教程
  • shadcn 自己持续在 Twitter 发新组件 + 改进

2024 · shadcn 现象级

2024 年是 shadcn 真正的爆发年。从社区项目到事实标准只用了 12 个月：

  • GitHub stars：年初 30k → 年末 60k+
  • Vercel / Linear / Resend / Cal.com / Plain.com 等明星公司公开使用
  • shadcn 个人也加入了 Vercel（2024 年某时间点）
  • 大量第三方 shadcn 模板 / blocks 库出现
  • Visual builder 第三方工具（v0.dev、Subframe 等）开始支持 shadcn 输出

shadcn 现象级的根因：

  1. copy-paste 模式真的解决了痛点：之前用 MUI / Mantine 的人都遇到过"想改某个细节但 override 痛苦"的问题，shadcn 一次性解决
  2. 整合 Radix + Tailwind + CVA：把 2023 年 React 生态最优的几块组合起来
  3. 美学过关：默认样式（Pedro Duarte 设计参考 + shadcn 调整）现代克制，符合 SaaS 审美
  4. shadcn 个人持续投入：每周更新、回应 issue、社区互动好

2024-09 · Modulz 加入 WorkOS

2024 年 9 月，Pedro Duarte 等 Modulz 核心团队整体加入 WorkOS（B2B SaaS 公司，主营 SSO / Directory Sync 等企业身份管理）。这给 Radix 长期维护提供保障：

  • Pedro 等人有了"专职做 Radix"的全职岗位
  • WorkOS 自家产品深度依赖 Radix（design system 基于 Radix）
  • 商业模式不再依赖 Modulz 公司收入（之前 Modulz 设计工具变现不顺）

2024-2025 年 Radix Primitives 持续小幅更新，但没有大版本变革。直到 2026-02 出统一包。

2026-02 · 双引擎 + 统一包 + Blocks 大扩张

2026 年 2 月是 shadcn 历史上的第二次范式转移。三件大事同时发生：

1. shadcn 双引擎：

  • shadcn 加 Base UI 支持（MUI 团队出品的 unstyled primitives）
  • 用户在 components.json 选 "engine": "radix" 或 "engine": "base-ui"
  • 每个 block 同时有 Radix 和 Base UI 两个变体
  • 这反映"shadcn 是模板而不是 Radix wrapper"的本质

为什么加 Base UI？

  • Base UI bundle 更小（适合性能敏感场景）
  • MUI 生态团队希望与 MUI 生态共存
  • 给 shadcn 用户更多选择
  • 不绑死单一 primitive 库（长期生态健康）

2. Radix 统一包 radix-ui：

  • 过去 Radix 分几十个子包（@radix-ui/react-dialog / @radix-ui/react-popover / ...）
  • 统一为单一包 radix-ui，导出所有 primitives
  • tree-shakable —— 不用的 primitive 不进 bundle
  • 新 primitives：PasswordToggleField / OneTimePasswordField

这解决了"装了 20 个 @radix-ui/react-* 让 package.json 难维护"的痛点。

3. 1300+ Blocks：

  • Block = 比组件大、比 page 小的组合（dashboard sidebar / login form / pricing table 等）
  • 含 dashboard / login / signup / sidebar / form / pricing 等完整模板
  • 同时有 Radix 和 Base UI 两个变体
  • RTL 一等公民支持

2026-03 · CLI v4 + Visual Builder

2026 年 3 月发布 shadcn/cli v4。两件大事：

1. 多框架 scaffold：

  • shadcn init 不再只支持 Next.js
  • 支持 Next.js / Vite / Laravel / React Router / Astro / TanStack Start
  • 每个框架自己模板（router 集成 / styling 配置 / dark mode 集成）
  • 自动检测项目类型或问用户

2. Visual Project Builder：

  • shadcn create 命令开浏览器到 ui.shadcn.com/create
  • 可视化配置：framework / style / base color / 选组件 / theme
  • 配完生成 zip 或 GitHub 仓库
  • 对"起新项目快速 scaffold"友好

这反映 shadcn 正在从"组件库"扩张为"完整 React 项目脚手架"。registry 不只分发组件，还分发完整 design system + 项目模板。

纵观 8 年 · 三次范式转移的根因

shadcn / Radix 三次范式转移的共同模式：每次都解决前一版的具体痛点 + 押下一代 React 生态的趋势。

  • 2018-2020 Radix 起步：解决 styled UI 库（MUI / AntD / Chakra）"定制痛苦 + a11y 不全"的痛点。押注 headless + a11y-first，赌当时刚起的 Tailwind 生态会做大
  • 2023 shadcn copy-paste：解决 npm package "黑盒 + 升级破坏 + 改不动" 的痛点。押注开发者工具链（CVA / cn / CLI）成熟到能支撑 copy-paste 模式
  • 2026 双引擎 + Registry：解决"shadcn 绑死 Radix" + "团队 design system 工具链缺失"两个痛点。押注 design system 工具链会成为团队级标准（不只是组件库）

每次转移都被市场验证。8 年后 shadcn / Radix 是 React UI 事实标准。

03 应用场景

场景 1 · 起新项目，选 Radix + shadcn 还是别的

2026 推荐栈：Tailwind v4 + shadcn/ui + Radix（默认引擎）。理由：

  • React 生态事实标准
  • shadcn CLI v4 多框架 scaffold 让起步快
  • copy-paste 模式让组件完全可控
  • Radix 提供完整 a11y
  • 团队招人友好

场景 2 · 老项目用 @radix-ui/react-* 多包，要不要切统一包

2026-02 后：建议切。理由：

  • 统一 radix-ui 包让 package.json 简洁
  • tree-shakable 不增加 bundle
  • 未来 Radix 新功能优先在统一包发布
  • 老 @radix-ui/react-* 仍维护但不再加新功能

迁移成本：低。改 import 路径（@radix-ui/react-dialog → radix-ui/Dialog）。codemod 可自动。

场景 3 · 要不要从 Radix 切 Base UI

2026 默认仍选 Radix。Base UI 何时考虑：

  • 极致性能敏感（每 KB bundle 都重要）
  • 已用 MUI 想生态共存
  • 团队偏好 MUI 风格 API

大多数项目选 Radix（生态成熟 + 社区资源多 + a11y 完整）。Base UI 是补充选项不是替代。

04 典型代码

代码 1 · 老 Radix 多包 → 统一包迁移

diff 
// 老（仍可用）
- import * as Dialog from '@radix-ui/react-dialog';
- import * as Popover from '@radix-ui/react-popover';
- import { Slot } from '@radix-ui/react-slot';

// 新（推荐）
+ import { Dialog, Popover, Slot } from 'radix-ui';

// 用法不变
<Dialog.Root>
  <Dialog.Trigger>Open</Dialog.Trigger>
  <Dialog.Content>...</Dialog.Content>
</Dialog.Root>

package.json：

diff 
{
  "dependencies": {
-   "@radix-ui/react-dialog": "^1.0.0",
-   "@radix-ui/react-popover": "^1.0.0",
-   "@radix-ui/react-slot": "^1.0.0",
+   "radix-ui": "^2.0.0"
  }
}

代码 2 · shadcn copy-paste vs npm 模式对比

bash 
# 传统 npm 模式（Mantine / MUI 等）
npm install @mantine/core @mantine/hooks
# 用：
import { Button } from '@mantine/core';
<Button>Click</Button>
# 想改 button 样式：override theme 或 sx prop（受限）
# 升级 @mantine/core 可能 breaking change

# shadcn copy-paste 模式
npx shadcn@latest add button
# 文件：components/ui/button.tsx 进项目
import { Button } from '@/components/ui/button';
<Button>Click</Button>
# 想改样式：直接改 components/ui/button.tsx
# 永远不会被远程包升级破坏

代码 3 · 双引擎切换（Radix vs Base UI）

json 
// components.json（项目根）

// 默认（Radix）
{
  "engine": "radix",
  "tailwind": { "config": "tailwind.config.ts" },
  "aliases": { "components": "@/components" }
}

// 切 Base UI
{
  "engine": "base-ui",
  "tailwind": { "config": "tailwind.config.ts" },
  "aliases": { "components": "@/components" }
}

// 切换后 shadcn add 自动拉对应变体源码

05 常见陷阱

  • 把 shadcn 当 npm 包装 —— 它不是。用 npx shadcn add CLI 添加。官方 docs 第一句就强调
  • shadcn copy 后不维护组件代码 —— 视为项目代码，要 review / test / 重构
  • 看 2023-2024 年的 shadcn 教程 —— 部分过时（CLI v4 命令变化、components.json 字段、2026 双引擎）。看 ui.shadcn.com 当前版本 docs
  • 认为 shadcn = "npm install 的另一个 UI 库" —— 完全不同心智。shadcn 是模板分发系统，不是包
  • 装了 @radix-ui/react-* 多包又装了 radix-ui 统一包 —— bundle 重复。选一个，建议统一包
  • "Radix 没 Combobox 是 bug" —— 设计选择不是 bug。Combobox 复杂度高，Radix 团队优先做更基础的；shadcn 用 cmdk 补足（详见 P4.1）
  • 2024 年公开访谈/视频里说的细节 —— 部分过期（特别是关于 monorepo / registry 的）。Visual Builder / 双引擎 / 统一包都是 2026 才有
  • 认为 Radix Themes = Radix Primitives —— 是两个产品。shadcn 用的是 Primitives 不是 Themes
  • 把 v0.dev / Subframe 当 shadcn 一部分 —— 是第三方 visual builder（输出 shadcn 代码），不是 shadcn 官方产品
  • 认为 shadcn 不能 vendor-neutral —— shadcn 是 MIT 开源，不绑特定平台。你可以部署到任何地方
  • 对"copy-paste 是技术债务"的误解 —— 这是 shadcn 设计哲学，不是债务。对比 npm 包升级破坏，copy-paste 反而更稳
  • 2026 项目仍用老 Radix 多包不切统一包 —— 不是错，但越早切越好（package.json 简洁 + 未来新功能优先在统一包）
