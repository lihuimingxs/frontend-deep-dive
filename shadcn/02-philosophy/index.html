P2 · Radix 哲学

Radix 不是发明 a11y、是把 W3C WAI-ARIA APG 标准做到位。这两章建立"看 shadcn 组件源码每行设计意图"的心智 —— 包括 headless / a11y 标准、受控非受控双模式、asChild Slot、Compound Components 共享 context。

章节数 2 主要来源 radix-ui.com docs + WAI-ARIA APG (W3C) + Pedro Duarte 演讲 状态 ✅ 全部完成

本阶段覆盖

  • headless 定义 · 提供"行为 + 状态 + a11y"，不提供样式
  • WAI-ARIA APG · W3C 维护的"组件 a11y 标准"权威文档
  • Dialog a11y 标准 · focus 移到 Dialog 内、ESC 关闭、Tab 循环、aria-modal/labelledby/describedby
  • Combobox a11y 标准 · ↓↑ 导航、Enter 选中、aria-expanded/controls/activedescendant
  • a11y 为什么难手写 · 标准复杂、浏览器/screen reader 差异、边角情况多、测试难
  • 受控/非受控 · open/onOpenChange vs defaultOpen；Radix 几乎所有 primitive 同时支持两种
  • asChild + Slot · 把组件 props/className/ref 转给传入的 children
  • Compound Components · 子组件通过 context 共享状态，不需要 prop drilling
  • API 设计 4 条原则 · Mirror 原生 HTML / Forward refs / Pass through props / Type safe
