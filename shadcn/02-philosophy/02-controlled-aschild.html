受控/asChild/Compound Components

Radix 组件 API 是 React 工程精华。这一章把受控/非受控双模式、asChild + Slot、Compound Components、API 设计 4 条原则讲透。读完能看懂 shadcn 组件源码每一行的设计意图，也能写出符合 Radix 风格的自定义组件。

阅读时长 ~25 分钟 前置 P2.1 Headless + a11y + React Hooks 基础 主要来源 radix-ui.com docs + Radix 源码 + Pedro Duarte 演讲

01 速览

  1. Radix 组件几乎都同时支持受控（open + onOpenChange）和非受控（defaultOpen）。简单场景非受控、需要外部控制 / 与表单联动用受控。这是 React 受控/非受控最佳实践的标杆实现。
  2. asChild 是 Radix 核心创新 —— 让组件不渲染自己的 DOM 元素，把行为（onClick / aria-* / data-state）转给传入的 children。配合 Radix Slot 实现。
  3. Slot 是 Radix 的工具组件 —— 把 className / props / ref 合并到 children 上。<Dialog.Trigger asChild><Button>Open</Button></Dialog.Trigger> 让 Dialog.Trigger 不渲染 button，把 onClick 等转给 Button。
  4. Compound Components（复合组件）：组件由多个子组件组成，通过 React context 共享状态。<Dialog.Root><Dialog.Trigger /><Dialog.Content /></Dialog.Root> 模式 —— 子组件不需要 prop drilling，自动通信。
  5. Compound Components 与 prop drilling / render props / 大 props 对象的对比：Compound 让 API 灵活、子组件位置任意、context 共享状态。
  6. Radix API 设计的 4 条原则：1) Mirror 原生 HTML（用 button / input 等真原生元素）；2) Forward refs（所有 primitive 用 React.forwardRef）；3) Pass through props（不消耗的 props 全部 spread 给底层 DOM）；4) Type safe（完整 TypeScript 推导）。
  7. asChild 的类型安全：通过 React.ElementType 泛型 + 与 children type 推导让 TS 自动推断"作为什么元素"的 props。
  8. 受控模式的 状态提升：当多个父级 state 需要联动 Dialog 状态（form 验证 + Dialog 打开），用受控让父级管 state。Radix 把这种灵活性内置在每个 primitive。
  9. Compound Components 的 子组件位置：可以在 Root 任何位置（甚至嵌套深层），通过 context 自动找到父级。这与"子组件必须直接相邻"的"Composition" 风格不同。
  10. 本章是 看 shadcn 源码必备。读完打开任何 shadcn 组件源码（components/ui/dialog.tsx 等）能看懂每一行 forwardRef / Slot / className spread / type 设计的用意。

02 底层逻辑

受控 vs 非受控双模式

Radix 几乎所有 primitive 同时支持两种模式：

非受控（默认） —— 组件自己管 state：

tsx 
// 非受控：用 defaultOpen
<Dialog.Root defaultOpen={false}>
  <Dialog.Trigger>Open</Dialog.Trigger>
  <Dialog.Content>...</Dialog.Content>
</Dialog.Root>

// Dialog.Root 内部 useState 管 open 状态
// 用户不需要自己 setState，Trigger / Content 自动协调

受控 —— 用户管 state：

tsx 
// 受控：用 open + onOpenChange
const [open, setOpen] = useState(false);

<Dialog.Root open={open} onOpenChange={setOpen}>
  <Dialog.Trigger>Open</Dialog.Trigger>
  <Dialog.Content>
    ...
    <Button onClick={() => setOpen(false)}>Close</Button>
  </Dialog.Content>
</Dialog.Root>

// 用户管 open state
// 可以从外部设 setOpen(true) 打开

选哪个：

  • 非受控（首选）：简单场景，组件自己管 state，少代码
  • 受控：1) 外部需要控制（form 提交后关闭）；2) 多个组件联动（form 验证 + Dialog 显示错误）；3) 测试时需要直接设状态

这套 API 是 React 受控/非受控最佳实践的标杆。同时支持两种模式让 API 灵活。

实现机制（Radix 内部）：

tsx 
// Radix 内部用 useControllableState hook
function useControllableState({ prop, defaultProp, onChange }) {
  const [uncontrolledValue, setUncontrolledValue] = useState(defaultProp);
  const isControlled = prop !== undefined;
  const value = isControlled ? prop : uncontrolledValue;

  const setValue = (next) => {
    if (!isControlled) setUncontrolledValue(next);
    onChange?.(next);
  };

  return [value, setValue];
}

// Dialog.Root 内部使用
function DialogRoot({ open, defaultOpen, onOpenChange, children }) {
  const [isOpen, setIsOpen] = useControllableState({
    prop: open,
    defaultProp: defaultOpen ?? false,
    onChange: onOpenChange,
  });
  // ... 把 isOpen / setIsOpen 通过 context 传给子组件
}

这个 hook 模式可以直接复用到自己的组件设计。Radix 提供 @radix-ui/react-use-controllable-state（统一包后是 radix-ui/internal 子路径）。

asChild + Slot · 把行为转给子元素

asChild 是 Radix 解决"如何让 Trigger 不渲染自己的 button、用我提供的 Button 组件"的方案。

没有 asChild 时：

tsx 
// ❌ 默认行为：Dialog.Trigger 渲染自己的 button
<Dialog.Trigger>Open</Dialog.Trigger>
// 渲染：<button data-state="closed" aria-haspopup="dialog" onClick={...}>Open</button>

// 但我有自己的 Button 组件 with Tailwind class + variants
// 怎么办？嵌套？
<Dialog.Trigger>
  <Button variant="primary">Open</Button>
</Dialog.Trigger>
// 渲染：<button><button>Open</button></button>
// ❌ 嵌套 button 是无效 HTML

asChild 解决：

tsx 
// ✅ 用 asChild
<Dialog.Trigger asChild>
  <Button variant="primary">Open</Button>
</Dialog.Trigger>

// 渲染：<button class="bg-blue-500 ..."  // Button 的 class
//                  data-state="closed"   // Trigger 注入的 data
//                  aria-haspopup="dialog" // Trigger 注入的 aria
//                  onClick={...}         // Trigger 的 click handler
//                  {...buttonProps}>
//             Open
//          </button>

asChild 的实现用 Radix Slot 工具组件：

tsx 
// Radix 内部 Trigger 实现（简化）
import { Slot } from 'radix-ui';

const Trigger = React.forwardRef(({ asChild, ...props }, ref) => {
  const Comp = asChild ? Slot : 'button';
  return (
    <Comp
      ref={ref}
      data-state={isOpen ? 'open' : 'closed'}
      aria-haspopup="dialog"
      aria-expanded={isOpen}
      onClick={handleClick}
      {...props}
    />
  );
});

Slot 内部做的事：把 props 合并到 children 的 ref + props 上。关键代码（简化）：

tsx 
function Slot({ children, ...slotProps }, slotRef) {
  if (!React.isValidElement(children)) return null;

  // 合并 ref
  const composedRef = composeRefs(slotRef, children.ref);

  // 合并 props
  const mergedProps = {
    ...children.props,
    ...slotProps,
    // 特殊处理 onClick / onKeyDown 等事件（链式调用）
    onClick: composeEventHandlers(slotProps.onClick, children.props.onClick),
    // 特殊处理 className（合并）
    className: cn(slotProps.className, children.props.className),
    // 特殊处理 style（合并）
    style: { ...slotProps.style, ...children.props.style },
    ref: composedRef,
  };

  return React.cloneElement(children, mergedProps);
}

asChild 的应用场景：

  • Trigger 用自定义 Button 组件
  • Link 用 Next.js Link / React Router NavLink
  • Dialog.Close 用任意自定义按钮
  • shadcn 几乎所有组件支持 asChild（继承 Radix）

Compound Components · context 共享状态

Compound Components 是 React 设计模式 —— 组件由多个子组件组成，共享 context，不需要 props 传递。

看 Dialog 的结构：

tsx 
<Dialog.Root open={isOpen} onOpenChange={setIsOpen}>
  <Dialog.Trigger>Open</Dialog.Trigger>
  <Dialog.Portal>
    <Dialog.Overlay />
    <Dialog.Content>
      <Dialog.Title>Confirm</Dialog.Title>
      <Dialog.Description>Are you sure?</Dialog.Description>
      <Dialog.Close>Cancel</Dialog.Close>
    </Dialog.Content>
  </Dialog.Portal>
</Dialog.Root>

关键点：Trigger / Content / Title / Close 之间没有显式 prop 传递。它们怎么知道彼此的状态（open 状态、aria 关联）？通过 React context。

Radix 内部（简化）：

tsx 
const DialogContext = React.createContext<{
  open: boolean;
  onOpenChange: (open: boolean) => void;
  titleId: string;
  descriptionId: string;
}>(null!);

function DialogRoot({ children, open, defaultOpen, onOpenChange }) {
  const [isOpen, setIsOpen] = useControllableState({
    prop: open,
    defaultProp: defaultOpen ?? false,
    onChange: onOpenChange,
  });

  // 自动生成稳定 id（用于 aria 关联）
  const titleId = React.useId();
  const descriptionId = React.useId();

  return (
    <DialogContext.Provider value={{ open: isOpen, onOpenChange: setIsOpen, titleId, descriptionId }}>
      {children}
    </DialogContext.Provider>
  );
}

function DialogTrigger({ asChild, ...props }) {
  const ctx = React.useContext(DialogContext);
  const Comp = asChild ? Slot : 'button';
  return (
    <Comp
      data-state={ctx.open ? 'open' : 'closed'}
      aria-haspopup="dialog"
      aria-expanded={ctx.open}
      aria-controls={ctx.contentId}
      onClick={() => ctx.onOpenChange(true)}
      {...props}
    />
  );
}

function DialogTitle({ children, ...props }) {
  const ctx = React.useContext(DialogContext);
  return <h2 id={ctx.titleId} {...props}>{children}</h2>;
}

function DialogContent({ children, ...props }) {
  const ctx = React.useContext(DialogContext);
  if (!ctx.open) return null;
  return (
    <div
      role="dialog"
      aria-modal="true"
      aria-labelledby={ctx.titleId}
      aria-describedby={ctx.descriptionId}
      {...props}
    >
      {children}
    </div>
  );
}

关键点：

  • Title 自动用 useId() 生成稳定 id（SSR/CSR 一致）
  • Content 自动 aria-labelledby 指向 Title
  • Trigger 自动 data-state / aria-expanded / aria-controls
  • 整个机制对用户透明

Compound Components vs 其他模式对比：

模式API子组件灵活度
Compound<Dialog.Root><Dialog.Trigger /><Dialog.Content /></Dialog.Root>高（任意位置）
Big props<Dialog trigger={...} content={...} />低（固定位置）
Render props<Dialog>{({ open }) => ...}</Dialog>中
Hooks-basedconst { triggerProps, contentProps } = useDialog()高但繁琐

Compound 是 Radix 选择，原因：

  • 子组件位置任意（嵌套深层也行）
  • API 直观（与 HTML 类似）
  • TypeScript 推导自然
  • 与 React 19 RSC 兼容好

API 设计的 4 条原则

Radix（和 shadcn）组件遵循的 4 条原则：

1. Mirror 原生 HTML

尽可能用真原生元素。Trigger 默认渲染 button、Content 默认渲染 div。

  • 原生元素自带 a11y（button 可 focus / 触发 keyboard）
  • asChild 让用户切到自定义元素（仍保持 a11y）
  • 不发明新元素

2. Forward refs

所有 primitive 用 React.forwardRef 包装：

tsx 
const DialogContent = React.forwardRef<
  HTMLDivElement,
  React.ComponentPropsWithoutRef<'div'>
>((props, ref) => {
  return <div ref={ref} {...props} />;
});
DialogContent.displayName = 'DialogContent';

让用户能 .focus() / .scrollIntoView() / 用 useImperativeHandle 暴露方法。

注意：React 19+ 不再需要 forwardRef（ref 自动作为 prop），但 Radix 当前版本仍兼容老 React 18，所以仍用 forwardRef。React 19 完全适配后会迁移。

3. Pass through props

不消耗的 props 全部 spread 给底层 DOM：

tsx 
const DialogContent = React.forwardRef((props, ref) => {
  const { children, ...rest } = props;  // 不解构 className / style 等
  return <div ref={ref} {...rest}>{children}</div>;
});

// 用户写 <DialogContent className="..." style="..." data-test-id="...">
// 这些都自动 spread 到 div

这让 Radix 组件与任意 className / style / data-* / 事件 handler 兼容。

4. Type safe

完整 TypeScript 推导：

tsx 
// Radix 类型
type DialogContentProps = React.ComponentPropsWithoutRef<'div'>;

// 用户写：
<Dialog.Content className="..." onClick={...} />
// className: string ✅
// onClick: (e: MouseEvent<HTMLDivElement>) => void ✅
// 拼错或类型错 → TS 报错

asChild + Slot 的类型推导更复杂（要根据 children type 推断），Radix 提供完整泛型支持。

useId · 稳定 id 生成（aria 关联必备）

Radix 组件常需要稳定 id（aria-labelledby / aria-controls）。React 18+ 提供 useId() hook：

tsx 
function DialogRoot({ children }) {
  const titleId = React.useId();   // ":r0:" / ":r1:" 等稳定 id
  const descriptionId = React.useId();
  // SSR 输出 ":r0:"，client hydration 用同样 ":r0:"
  // 不会出现 SSR / CSR mismatch
}

没有 useId 之前要自己 useState + 随机数（容易 hydration mismatch）。useId 是 React 18 给 a11y 库的核心支持。

03 应用场景

场景 1 · 自己写组件库基于 Radix

团队自建组件库，基于 Radix Primitives：

  1. 用 Compound Components 模式
  2. 用 forwardRef 包所有组件
  3. 支持 asChild（用 Slot）
  4. 支持受控/非受控双模式（用 useControllableState）
  5. 用 useId 生成 a11y id

这套模式让你的组件 API 风格与 Radix 一致 → 团队学习成本低 + 与 shadcn 兼容。

场景 2 · Form 中 Dialog 受控

表单提交后关闭 Dialog：

tsx 
function ContactForm() {
  const [open, setOpen] = useState(false);

  async function handleSubmit(data) {
    await submitContact(data);
    setOpen(false);  // 提交后关闭
    toast.success('Sent!');
  }

  return (
    <Dialog.Root open={open} onOpenChange={setOpen}>
      <Dialog.Trigger>Contact us</Dialog.Trigger>
      <Dialog.Content>
        <ContactForm onSubmit={handleSubmit} />
      </Dialog.Content>
    </Dialog.Root>
  );
}

场景 3 · asChild 与 Next.js Link

DropdownMenu 项作为路由跳转：

tsx 
import Link from 'next/link';
import { DropdownMenu } from 'radix-ui';

<DropdownMenu.Item asChild>
  <Link href="/profile">Profile</Link>
</DropdownMenu.Item>

// 渲染：<a href="/profile" class="..." data-orientation="vertical" role="menuitem">Profile</a>
// 既是 Next.js Link（client navigation）又是 Radix MenuItem（a11y）

04 典型代码

代码 1 · 自己写 Compound Component

tsx 
// MyAccordion.tsx · 自己实现 Compound Component
import * as React from 'react';

const AccordionContext = React.createContext<{
  openItems: string[];
  toggle: (id: string) => void;
}>(null!);

function AccordionRoot({ children, defaultOpen = [] }: { children: React.ReactNode; defaultOpen?: string[] }) {
  const [openItems, setOpenItems] = React.useState(defaultOpen);
  const toggle = (id: string) => {
    setOpenItems((prev) => prev.includes(id) ? prev.filter(x => x !== id) : [...prev, id]);
  };
  return (
    <AccordionContext.Provider value={{ openItems, toggle }}>
      <div>{children}</div>
    </AccordionContext.Provider>
  );
}

function AccordionItem({ id, children }: { id: string; children: React.ReactNode }) {
  return <ItemContext.Provider value={{ id }}>{children}</ItemContext.Provider>;
}

const ItemContext = React.createContext<{ id: string }>(null!);

function AccordionTrigger({ children }: { children: React.ReactNode }) {
  const root = React.useContext(AccordionContext);
  const item = React.useContext(ItemContext);
  const isOpen = root.openItems.includes(item.id);
  return (
    <button
      onClick={() => root.toggle(item.id)}
      aria-expanded={isOpen}
      aria-controls={`accordion-content-${item.id}`}
    >
      {children}
    </button>
  );
}

function AccordionContent({ children }: { children: React.ReactNode }) {
  const root = React.useContext(AccordionContext);
  const item = React.useContext(ItemContext);
  const isOpen = root.openItems.includes(item.id);
  if (!isOpen) return null;
  return <div id={`accordion-content-${item.id}`} role="region">{children}</div>;
}

// 导出 Compound API
export const Accordion = {
  Root: AccordionRoot,
  Item: AccordionItem,
  Trigger: AccordionTrigger,
  Content: AccordionContent,
};

// 用法
<Accordion.Root defaultOpen={['1']}>
  <Accordion.Item id="1">
    <Accordion.Trigger>Section 1</Accordion.Trigger>
    <Accordion.Content>Content 1</Accordion.Content>
  </Accordion.Item>
</Accordion.Root>

代码 2 · 自己写 useControllableState

tsx 
import * as React from 'react';

export function useControllableState<T>({
  prop,
  defaultProp,
  onChange,
}: {
  prop: T | undefined;
  defaultProp: T;
  onChange?: (value: T) => void;
}): [T, (value: T) => void] {
  const [uncontrolledValue, setUncontrolledValue] = React.useState(defaultProp);
  const isControlled = prop !== undefined;
  const value = isControlled ? prop! : uncontrolledValue;
  const onChangeRef = React.useRef(onChange);
  React.useEffect(() => { onChangeRef.current = onChange; });

  const setValue = React.useCallback((next: T) => {
    if (!isControlled) setUncontrolledValue(next);
    onChangeRef.current?.(next);
  }, [isControlled]);

  return [value, setValue];
}

// 使用
function MyDialog({ open, defaultOpen = false, onOpenChange }) {
  const [isOpen, setIsOpen] = useControllableState({
    prop: open,
    defaultProp: defaultOpen,
    onChange: onOpenChange,
  });
  // ... 现在组件同时支持受控/非受控
}

代码 3 · asChild 在自定义组件实现

tsx 
import * as React from 'react';
import { Slot } from 'radix-ui';

interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  asChild?: boolean;
}

const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ asChild, className, ...props }, ref) => {
    const Comp = asChild ? Slot : 'button';
    return (
      <Comp
        ref={ref}
        className={cn('px-4 py-2 bg-blue-500 text-white rounded', className)}
        {...props}
      />
    );
  }
);

// 用法 1：普通 button
<Button onClick={...}>Click</Button>

// 用法 2：作为 link
<Button asChild>
  <a href="https://example.com">Visit</a>
</Button>
// 渲染：<a href="..." class="px-4 py-2 bg-blue-500 ...">Visit</a>

05 常见陷阱

  • asChild 用了多个 children —— Slot 只接受单个 React element，多个 children 报错。包一层 Fragment 不行（要单个真元素）
  • asChild children 不是 React element —— children 是字符串 / null 时 Slot 报错。必须是 React element
  • 同时用 open 和 defaultOpen —— 受控模式忽略 defaultOpen。要么传 open + onOpenChange、要么传 defaultOpen，不要混
  • 受控但忘了 onOpenChange —— Dialog 永远不能关（用户点关闭按钮 setOpen 不被触发）
  • onClick handler 写错位置 —— asChild 时 onClick 写在 Trigger 上（被 Slot 合并到 child），写在 child Button 上（被 Slot props 覆盖，看 composeEventHandlers 链式行为）
  • 用 React.cloneElement 自己实现 asChild —— 不如用 Radix Slot（处理 ref / 事件链 / className 合并）。装 radix-ui 就有
  • Compound Components 子组件不在 Root 内 —— context 找不到，运行时崩。Trigger / Content 必须包在 Root 内
  • useId 在 SSR / CSR 不一致 —— 用了 React 17 老版本（没有 useId）。升 React 18+ 解决
  • forwardRef 忘了 displayName —— React DevTools 显示 "Anonymous"。补 Component.displayName = '...'
  • Pass through props 时解构了 className —— const { className, ...rest } = props 然后只用 rest，丢失 className。要么 cn 合并、要么不解构
  • 认为 asChild 会改变组件 a11y —— 不会。Radix 把 a11y attributes 合并到 child（asChild 后 child 仍有 aria-haspopup 等）
  • 把 Compound Components 当 namespace 用 —— Dialog.Root / Dialog.Trigger 等是 export，不是 namespace。可以单独 import
