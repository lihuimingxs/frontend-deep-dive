编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 协议机制以 RFC 一手为准 —— HPACK 哈夫曼表、QUIC 帧格式、QPACK encoder/decoder stream、congestion control 算法都引 RFC 章节号；不引 blog / 不引知乎；引用规范优先 RFC 9000-9114 / 9204 / 9218 / 8297 / 8446
  • 工程实战以头部厂商 production 数据为准 —— Server Push 实测伤性能、Early Hints 增益、BBR vs Cubic 公平性、HTTP/3 fallback 率必须有 Cloudflare / Fastly / Akamai / Google 实测；不靠"我觉得"
  • 「短重述 + 链回」 —— RSC payload 与 Server Components 在 Next.js 主题；fetch / Streams API 在 JavaScript 主题；TanStack streamedQuery 在 TanStack Query 主题；Web Vitals / CRP 在 浏览器渲染主题；本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 不重讲假设已掌握 —— HTTP 应用层基础（headers / methods / status codes）、HTTP cache 语义（Cache-Control / ETag）、TLS 1.3 record layer、TCP 内部排队论、WebRTC 信令完整链都不展开，引一手规范但本主题不复述
  • 观点强度 —— 强观点（"Server Push 是失败实验，2026 不要再配"，"HTTP/3 是新协议默认，但 HTTP/2 仍是回源主力"，"QUIC ≠ UDP，是在 UDP 上重建了一个完整传输层"）；弱观点（"WebTransport 看浏览器支持再决定"）；不观点（"WebSocket vs WebTransport 看场景"）

与既有主题的关系

HTTP/2/3/QUIC 是协议层根因；多个已写主题在应用层引用。本主题不重复那些主题的应用层论述，但会在每个交叉点提供"短重述 + 链接 + 协议层根因"。常见对应：

  • P5 资源提示家族 + LCP 优化 → 浏览器渲染主题（CRP / Web Vitals 应用层）
  • P6.1 RSC payload 协议层 + 流式渲染 → React 主题（RSC / use API） + Next.js 主题（RSC streaming / Server Actions）
  • P6.1 TanStack streamedQuery 协议层根因 → TanStack Query 主题 P4.2（双向链）
  • P6.1 Vercel AI SDK Data Stream Protocol → 待写 AI SDK 主题（双向链 · 两个主题在流式协议层强相关）
  • P6.1 fetch + ReadableStream / Streams API → JavaScript 主题（网络与流）
  • P7.1 Edge runtime 部署对协议层的影响 → Next.js 主题（Edge / Node runtime + 部署矩阵）
