P6 · 流式 + 实时通信

RSC 流式渲染、TanStack streamedQuery、Vercel AI SDK 流式响应底下都是协议层的几条特定路径。这一阶段把单向流（chunked / fetch streams / SSE）和双向（WebSocket / WebTransport / WebRTC DataChannel）两条线讲清，给前面应用层主题提供协议层根因。

章节数 2 主要来源 WHATWG Streams / EventSource / WebSocket 标准 + RFC 6455 (WebSocket) + W3C WebTransport draft + W3C WebRTC + Cloudflare/Vercel 流式工程博客 + React/Next.js 团队 RSC payload 文档 + Vercel AI SDK Data Stream Protocol 文档 状态 ✅ 全部完成

本阶段覆盖

  • chunked transfer encoding · HTTP/1.1 流式基础 · HTTP/2 DATA 帧 · HTTP/3 DATA 帧
  • fetch + ReadableStream · 浏览器侧统一抽象 · 与 chunked / DATA 帧对接
  • SSE（EventSource） · text/event-stream 格式 · 自动重连 + Last-Event-ID
  • RSC payload 协议层 · React Flight serialization · chunked 在 HTML head + body 之间流式
  • TanStack streamedQuery · 协议层根因（双向链 P5.2 子章节）
  • Vercel AI SDK Data Stream Protocol · 走 fetch + ReadableStream 不走 SSE 的工程理由
  • WebSocket 三条路 · RFC 6455 over 1.1 / RFC 8441 over 2（基本无人用）/ RFC 9220 over 3（落地缓慢）
  • WebTransport over HTTP/3 · 多 stream + datagram · 2026 Chromium-only
  • WebRTC DataChannel · DTLS over UDP · P2P + 信令 + ICE/STUN/TURN（仅横评不展开）
  • 三者横评矩阵 · 浏览器支持 / 协议层 / 多路复用 / 延迟 / 二进制 / 服务端实现 / 适用场景
