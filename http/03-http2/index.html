P3 · HTTP/2 协议层

HTTP/2 是协议层的"中间一代"：解决了 1.1 应用层 HOL blocking，但 TCP 字节流 HOL blocking 仍在；HPACK 头压缩是真创新，Server Push 是失败实验。这一阶段把"为什么 HTTP/2 必要"和"为什么 HTTP/2 不是终点"双向讲清。

章节数 2 主要来源 RFC 9113 (HTTP/2) + RFC 7541 (HPACK) + Google SPDY whitepaper + Chrome 团队关于 Server Push 实测的 2017-2020 数据 + Robin Marx 关于 TCP HOL blocking 的研究 状态 ✅ 全部完成

本阶段覆盖

  • HTTP/2 帧格式 · 9-byte 帧头 · 10 种帧类型 · stream id 奇偶分配
  • stream 状态机 · idle / open / half-closed / closed 五态
  • 多路复用 · 一个 TCP 连接承载几百到几千 stream · 解除 6 connection limit
  • flow control · stream + connection 两层 window · 与 TCP 流控为何不重复
  • prioritization tree 失败 · RFC 9113 移除 · 改用 RFC 9218 简单方案
  • HPACK 头压缩 · 静态表 61 项 + 动态表 + Huffman · 动态表必须顺序更新 → QPACK 必须重做的根
  • Server Push 兴衰 · 设计初衷 · Chrome 实测伤性能 · 2022 全面退场
  • TCP HOL blocking 限制 · 弱网下 HTTP/2 反而比 1.1 慢的根因 · QUIC 必要性
