编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 主线锁定 Tailwind v4.2 —— 所有"现在怎么用"的代码示例基于 v4.2；v3 配置（tailwind.config.js）仅在 P1.1 历史 + P3.2 迁移段中出现
  • 「短重述 + 链回」 —— 浏览器渲染 / GPU 合成在 浏览器渲染主题；Vite plugin 在 Vite 主题；Turbopack 在 Next.js 主题。本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 设计动机以 Adam Wathan 文章为准 —— "为什么 utility-first" 的原文是 Adam 2017 年的 blog；写"为什么这么设计"必须回到一手出处，避开二手解读
  • 性能数字必标条件 —— "Oxide 比 v3 快 100×" 必须标场景（首次 build / 增量 / project size），否则不写

与既有主题的关系

Tailwind 是 React / Next.js 项目的样式承载，所以与几个已写主题有交叉。本主题不重复已经讲透的底层机制，但会在每个交叉点提供"短重述 + 链接"。常见对应：

  • P3.2 与 Vite plugin 集成 → Vite P7.1 plugin 系统
  • P3.2 与 Turbopack 集成 → Next.js P6.1 Turbopack
  • P4 现代 CSS 特性（cascade layers / will-change） → 浏览器渲染主题
  • P5 响应式 + 字体优化策略 → Next.js P7.2 next/font
  • P6 CVA + className 在 React 里使用 → React 主题
  • P7 与 shadcn/ui 关系（待写 shadcn/ui 独立主题）
