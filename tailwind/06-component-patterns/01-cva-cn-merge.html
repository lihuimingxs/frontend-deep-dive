CVA + cn() + tailwind-merge

一个组件有多种变体（primary/secondary/outline/ghost）+ 多种尺寸（sm/md/lg）+ 用户传 className 想覆盖默认样式 —— 这是 Tailwind 团队规模化下能不能不崩的关键。CVA + cn() + tailwind-merge + prettier-plugin-tailwindcss 是 2024+ 事实标准工具链，由 shadcn/ui 推广。这一章给完整 Button + Card + Dialog 组件模板，让你团队代码风格一致。

阅读时长 ~25 分钟 前置 P2 utility-first + P5 变体系统 + TypeScript 基础 主要来源 CVA docs + tailwind-merge docs + shadcn/ui 源码 + Joe Bell 公开演讲

01 速览

  1. Tailwind 项目在团队规模化下的 4 个工具链组件：CVA（class-variance-authority，类型安全的变体管理）/ cn()（clsx + tailwind-merge，条件 className + 冲突解决）/ prettier-plugin-tailwindcss（自动排序 class）/ eslint-plugin-tailwindcss（静态检查）。
  2. CVA 由 Joe Bell 设计，shadcn/ui 推广为事实标准。它解决"组件多变体 + 多尺寸怎么类型安全管理"的问题。cva(base, { variants }) 接收基础 class + 变体配置，返回函数。
  3. cn() helper（来自 shadcn/ui，内部用 clsx + tailwind-merge）解决两个问题：1) 条件 className（cn('base', isActive && 'active')）/ 2) class 冲突（cn('px-4', 'px-6') 后者赢）。
  4. tailwind-merge：理解 Tailwind utility 优先级规则（px-4 与 px-6 冲突，p-4 与 px-6 也冲突）。让"用户传 className 覆盖默认"成为可能。
  5. prettier-plugin-tailwindcss：自动按推荐顺序排列 class（layout → spacing → color → state）。装上之后团队 review 不再纠结 class 顺序。
  6. eslint-plugin-tailwindcss：静态检查（catch 拼错 class、多余 utility、错误的 arbitrary value 等）。可选但推荐。
  7. CVA + cn 的标准模式让组件 API 类型安全 + 可复用 + 可被外部覆盖样式。这是 shadcn/ui 模板核心。
  8. 典型 Button 组件用 CVA 管理 5+ variants × 3+ sizes，配 VariantProps<typeof buttonVariants> 自动推导类型。TS catch 拼错的 variant 名字。
  9. "asChild" 模式（Radix 推广）：让组件渲染为传入的子元素（<Button asChild><Link>...</Link></Button>），用 Radix Slot 实现。这是 shadcn/ui 组件的标配。
  10. 这一章给的模板与 shadcn/ui 完全兼容。如果你后面要用 shadcn/ui（详见 P7），心智完全一致。

02 底层逻辑

问题：组件变体管理的痛点

没有 CVA 时，写个有 4 variants × 3 sizes 的 Button：

tsx 
// ❌ 反模式：手动拼 className
function Button({ variant = 'primary', size = 'md', children, ...props }) {
  let classes = 'inline-flex items-center justify-center rounded-md font-medium transition-colors';

  if (variant === 'primary') classes += ' bg-blue-500 text-white hover:bg-blue-600';
  else if (variant === 'secondary') classes += ' bg-gray-200 text-gray-900 hover:bg-gray-300';
  else if (variant === 'outline') classes += ' border border-gray-200 hover:bg-gray-100';
  else if (variant === 'ghost') classes += ' hover:bg-gray-100';

  if (size === 'sm') classes += ' h-8 px-3 text-xs';
  else if (size === 'md') classes += ' h-10 px-4';
  else if (size === 'lg') classes += ' h-12 px-6 text-base';

  return <button className={classes} {...props}>{children}</button>;
}

痛点：

  • 类型不安全：拼错 variant="primery" TypeScript 不报错
  • 用户传 className 不能 override（className="..." 被 base + variant + size 覆盖）
  • variant 多了代码炸（10 variants × 5 sizes 写不下去）
  • 组件库不能批量修改 base style

CVA 解决变体管理

CVA 把 variant 配置抽成结构化对象 + 自动生成函数：

tsx 
// 安装：npm i class-variance-authority

import { cva, type VariantProps } from 'class-variance-authority';

const buttonVariants = cva(
  // base styles（所有变体共享）
  'inline-flex items-center justify-center rounded-md font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2 disabled:pointer-events-none disabled:opacity-50',
  {
    variants: {
      variant: {
        primary: 'bg-blue-500 text-white hover:bg-blue-600',
        secondary: 'bg-gray-200 text-gray-900 hover:bg-gray-300',
        outline: 'border border-gray-200 bg-transparent hover:bg-gray-100',
        ghost: 'hover:bg-gray-100',
        link: 'text-blue-500 underline-offset-4 hover:underline',
        destructive: 'bg-red-500 text-white hover:bg-red-600',
      },
      size: {
        sm: 'h-8 px-3 text-xs',
        md: 'h-10 px-4',
        lg: 'h-12 px-6 text-base',
        icon: 'h-10 w-10',
      },
    },
    defaultVariants: {
      variant: 'primary',
      size: 'md',
    },
  },
);

// 使用
buttonVariants({ variant: 'primary', size: 'md' });
// 返回完整 class 字符串：'inline-flex ... bg-blue-500 ... h-10 px-4'

组件用法：

tsx 
type ButtonProps = React.ButtonHTMLAttributes<HTMLButtonElement> & VariantProps<typeof buttonVariants>;

function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <button
      className={cn(buttonVariants({ variant, size }), className)}
      {...props}
    />
  );
}

// 使用方
<Button variant="primary" size="lg">Save</Button>
<Button variant="outline">Cancel</Button>
<Button variant="ghost" className="text-red-500">Delete</Button>
{/* className 通过 cn() 与 buttonVariants 输出合并，可以覆盖默认 */}

类型安全：

tsx 
// ✅ TypeScript 自动推导
<Button variant="primary" size="md">OK</Button>

// ❌ TS 编译错误
<Button variant="primery">Typo</Button>
//              ^^^^^^^^^ Type '"primery"' is not assignable to type ...

VariantProps<typeof buttonVariants> 自动推导出：

typescript 
type VariantProps = {
  variant?: 'primary' | 'secondary' | 'outline' | 'ghost' | 'link' | 'destructive';
  size?: 'sm' | 'md' | 'lg' | 'icon';
}

cn() helper 详解

cn() 来自 shadcn/ui，是 clsx + tailwind-merge 的组合：

typescript 
// lib/utils.ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

它做两件事：

1. clsx · 条件 className：

typescript 
cn('base', isActive && 'active', { highlighted: hasError });
// 等价：[ 'base', isActive ? 'active' : '', hasError ? 'highlighted' : '' ].filter(Boolean).join(' ')

// 输入 ClassValue 类型支持：
//   字符串：'base'
//   数组：['a', 'b']
//   对象：{ active: true, highlighted: false }
//   嵌套：cn('a', ['b', { c: true }])

2. tailwind-merge · 解决 class 冲突：

typescript 
// 不用 tailwind-merge：两个 class 都进 className
cn('px-4', 'px-6');  // 'px-4 px-6' —— 浏览器按 CSS 顺序，px-6 赢（不稳定）

// 用 tailwind-merge：理解 Tailwind 优先级
twMerge('px-4 px-6');  // 'px-6' —— 智能保留后者

// 复杂冲突也能处理
twMerge('p-4 px-6');     // 'p-4 px-6' → 'py-4 px-6'（p-4 等于 py-4 + px-4，被 px-6 覆盖 px 部分）

// 不同 utility 不冲突
twMerge('px-4 mx-4');    // 'px-4 mx-4'（保留两者）

tailwind-merge 内部维护 utility 优先级表（哪些 class 是同类、谁覆盖谁）。这让"用户传 className 覆盖默认"成为可能：

tsx 
function Button({ className, ...props }) {
  return (
    <button
      className={cn(
        'px-4 py-2 bg-blue-500',  // 默认
        className,                  // 用户传入（如 'px-6 bg-red-500'）
      )}
      {...props}
    />
  );
}

<Button className="px-6 bg-red-500">Override</Button>
{/* 实际输出 className: 'py-2 px-6 bg-red-500'
    px-4 被 px-6 覆盖、bg-blue-500 被 bg-red-500 覆盖、py-2 保留 */}

没有 tailwind-merge 时：

tsx 
// ❌ 没用 tailwind-merge
<button className={`px-4 py-2 bg-blue-500 ${className}`}>
  {/* 实际：'px-4 py-2 bg-blue-500 px-6 bg-red-500'
      px-4 与 px-6 冲突，浏览器解析顺序决定（不稳定 + 优先级混乱） */}
</button>

prettier-plugin-tailwindcss · 自动排序

装这个 plugin 自动按推荐顺序排 class：

bash 
npm i -D prettier-plugin-tailwindcss
javascript 
// .prettierrc.json
{
  "plugins": ["prettier-plugin-tailwindcss"]
}

排序规则（大致）：

  1. 布局（flex / grid / block）
  2. 定位（absolute / fixed）
  3. display / visibility
  4. spacing（margin / padding / gap）
  5. size（width / height）
  6. typography（font / text）
  7. color（bg / text / border）
  8. border / radius / shadow
  9. transition / animation
  10. state variants（hover / focus / dark）

装上之后，className="text-white px-4 flex bg-blue-500 py-2" 自动变 className="flex bg-blue-500 px-4 py-2 text-white"。团队代码风格统一。

eslint-plugin-tailwindcss · 静态检查

bash 
npm i -D eslint-plugin-tailwindcss
json 
// .eslintrc.json
{
  "plugins": ["tailwindcss"],
  "rules": {
    "tailwindcss/classnames-order": "warn",
    "tailwindcss/no-custom-classname": "warn",
    "tailwindcss/no-contradicting-classname": "error",
    "tailwindcss/enforces-shorthand": "warn"
  }
}

能 catch 的问题：

  • 拼错 class 名（bg-blue-500 写成 bg-blue500）
  • 同个属性多次写（px-4 px-6 警告）
  • 能用 shorthand 时未用（w-4 h-4 警告 → 建议 size-4）

注意：与 prettier-plugin 配合时，让 prettier 管排序，eslint 管 lint，避免冲突。

asChild 模式（Radix 推广）

shadcn/ui 组件常见模式：让 Button 能渲染为 a / Link / 任何元素。

tsx 
import { Slot } from '@radix-ui/react-slot';

interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement>, VariantProps<typeof buttonVariants> {
  asChild?: boolean;
}

function Button({ className, variant, size, asChild = false, ...props }: ButtonProps) {
  const Comp = asChild ? Slot : 'button';
  return (
    <Comp
      className={cn(buttonVariants({ variant, size }), className)}
      {...props}
    />
  );
}

// 使用 1：普通 button
<Button>Click</Button>
// 渲染：<button class="...">Click</button>

// 使用 2：asChild + Next.js Link
<Button asChild variant="primary">
  <Link href="/dashboard">Dashboard</Link>
</Button>
// 渲染：<a href="/dashboard" class="...">Dashboard</a>
// （Slot 把 Button 的 className + props 转给内部 Link / a）

这种模式让 Button 既可以是 button 又可以是 link，不用维护两个组件。

03 应用场景

场景 1 · 起新组件库

步骤：

  1. 装依赖：npm i class-variance-authority clsx tailwind-merge @radix-ui/react-slot
  2. 装 dev：npm i -D prettier-plugin-tailwindcss eslint-plugin-tailwindcss
  3. 建 lib/utils.ts（cn helper）
  4. 每个组件按 CVA 模板写
  5. 团队建 component template（snippets / scaffolding）

场景 2 · 现有项目接 shadcn/ui

shadcn CLI 自动复制组件到你项目，已经按 CVA 模板写好。直接用：

bash 
npx shadcn@latest init
npx shadcn@latest add button card dialog

shadcn CLI 会：

  • 装 CVA / clsx / tailwind-merge
  • 建 lib/utils.ts（cn helper）
  • 建 components/ui/button.tsx 等组件源码
  • 组件代码完全在你项目里、可任意修改

场景 3 · 团队 code review 看到拼字符串

看到：

tsx 
<button className={`px-4 py-2 ${variant === 'primary' ? 'bg-blue-500' : 'bg-gray-200'} ${className}`}>

建议：

  • 3+ variants 的组件 → 用 CVA
  • 简单条件 → 用 cn()
  • 禁止用 + 拼字符串
  • 用户能传 className → 必须用 cn()（不然冲突）

04 典型代码

代码 1 · 完整 Button 组件（shadcn/ui 风格）

tsx 
// lib/utils.ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

// components/ui/button.tsx
import * as React from 'react';
import { Slot } from '@radix-ui/react-slot';
import { cva, type VariantProps } from 'class-variance-authority';
import { cn } from '@/lib/utils';

const buttonVariants = cva(
  'inline-flex items-center justify-center gap-2 rounded-md text-sm font-medium transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:pointer-events-none disabled:opacity-50',
  {
    variants: {
      variant: {
        default: 'bg-primary text-primary-foreground hover:bg-primary/90',
        destructive: 'bg-destructive text-destructive-foreground hover:bg-destructive/90',
        outline: 'border border-input bg-background hover:bg-accent hover:text-accent-foreground',
        secondary: 'bg-secondary text-secondary-foreground hover:bg-secondary/80',
        ghost: 'hover:bg-accent hover:text-accent-foreground',
        link: 'text-primary underline-offset-4 hover:underline',
      },
      size: {
        default: 'h-10 px-4 py-2',
        sm: 'h-9 rounded-md px-3',
        lg: 'h-11 rounded-md px-8',
        icon: 'h-10 w-10',
      },
    },
    defaultVariants: {
      variant: 'default',
      size: 'default',
    },
  },
);

export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  asChild?: boolean;
}

const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, asChild = false, ...props }, ref) => {
    const Comp = asChild ? Slot : 'button';
    return (
      <Comp
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        {...props}
      />
    );
  },
);
Button.displayName = 'Button';

export { Button, buttonVariants };

代码 3 · cn() 高级用法

tsx 
// 1. 条件 className
function Alert({ variant, className }) {
  return (
    <div className={cn(
      'p-4 rounded-md',
      variant === 'success' && 'bg-green-100 text-green-700',
      variant === 'error' && 'bg-red-100 text-red-700',
      variant === 'warning' && 'bg-yellow-100 text-yellow-700',
      className,  // 允许外部覆盖
    )}>
      ...
    </div>
  );
}

// 2. 对象语法
function Button({ disabled, loading, className }) {
  return (
    <button className={cn(
      'px-4 py-2 rounded',
      {
        'bg-blue-500 text-white': !disabled,
        'bg-gray-300 text-gray-500 cursor-not-allowed': disabled,
        'animate-pulse': loading,
      },
      className,
    )}>
      ...
    </button>
  );
}

// 3. 嵌套数组
const baseClasses = ['rounded-md', 'transition'];
function Card({ className }) {
  return <div className={cn(baseClasses, 'p-4 bg-white', className)}> ... </div>;
}

代码 4 · 复合 variants（CVA 高级）

tsx 
const buttonVariants = cva('rounded-md', {
  variants: {
    variant: {
      primary: 'bg-blue-500',
      destructive: 'bg-red-500',
    },
    size: {
      sm: 'h-8',
      lg: 'h-12',
    },
  },
  // 复合：variant=destructive + size=lg 时额外样式

    {

      size: 'lg',

    },
  ],
  defaultVariants: { variant: 'primary', size: 'sm' },
});

05 常见陷阱

  • 不用 cn() 直接拼字符串 —— 用户 className 不能覆盖、class 冲突无法解决。所有可被外部传 className 的组件都要 cn()
  • 用 clsx 不用 tailwind-merge —— 解决了条件 className 但没解决 class 冲突。要 cn() = clsx + tailwind-merge
  • CVA 后再加 className 没传给 cn() —— buttonVariants(...) 返回字符串，要再 cn(buttonVariants(...), className) 让用户能 override
  • VariantProps 类型导出错 —— VariantProps<typeof buttonVariants> 是正确写法（typeof）；VariantProps<buttonVariants> 错
  • asChild 用不带 forwardRef 的 Button —— Slot 内部需要 ref 传递。Button 组件用 React.forwardRef 包
  • prettier-plugin 与 eslint rule 冲突 —— prettier 排序后 eslint 仍报序错。让 prettier 管排序、eslint 关闭 classnames-order rule
  • tailwind-merge 不识别自定义 utility —— 自定义 utility（@utility / @theme 加的）需要在 twMerge 配置 customGroups（详见 tailwind-merge docs）
  • CVA 的 variants 名字撞 React 保留字 —— 不要用 className / children / key 作为 variant 名
  • compoundVariants 写多了维护复杂 —— 复合 3+ 维度时考虑拆组件，不要 CVA 中累积
  • 每个组件单独定义 buttonVariants —— 应共享 base style 在 lib/variants.ts，组件 import
  • "我用 styled-components 不需要 CVA" —— 与 styled-components 心智不同。Tailwind 项目不要用 styled-components 模式
  • 组件库用 cn() 但不 export —— 团队应该 export cn（通常 lib/utils.ts），让所有组件统一用
  • "shadcn/ui 太冗长，我自己实现" —— shadcn/ui 模板是工程经验沉淀，包括 ref forwarding / a11y / asChild。自己写大概率漏掉这些
