P1 · Tailwind 是谁

不讲 utility 细节，只解决两个问题：Tailwind 怎么从 2017 年 Adam Wathan 一篇争议 blog 走到 2026 年事实标准、它在 CSS 生态里站哪。给后面所有"为什么这样设计"提供历史坐标。

章节数 2 主要来源 tailwindcss.com/blog 各版本 release notes + Adam Wathan blog + "Refactoring UI" 书 状态 ✅ 全部完成

本阶段覆盖

  • 2017-08 utility-first 论战 · Adam Wathan 发表 "CSS Utility Classes and 'Separation of Concerns'"，挑战 BEM 主流
  • 2017-11 Tailwind v0.1 alpha · Adam + Jonathan Reinink + Steve Schoger 三人小组
  • 2019 v1.0 + 2020 v2.0 + 暗色模式 · 早期生态扎根
  • 2021-03 v2.2 JIT engine（实验） · 解决 v1-v2 时代"完整生成 + PurgeCSS"的慢
  • 2021-12 v3.0 JIT 默认 + arbitrary values · arbitrary values（w-[123px]）成为可能
  • 2022-2024 v3.1-3.4 · container queries / @container / :has() / aria-* / data-* 变体大量增量
  • 2025-01 v4.0 Oxide 引擎首发 · Rust 重写 + CSS-first 配置（@theme / @utility 替代 tailwind.config.js）
  • 2025-04 v4.1 · text-shadow / mask 等增强
  • 2026-02 v4.2 · Webpack plugin + 4 新色板（mauve/olive/mist/taupe）+ logical properties + 编译再快 3.8×
  • 同代人横评 · Bootstrap / CSS-in-JS / UnoCSS / vanilla-extract / CSS Modules / 纯 CSS / Sass
  • 4 个核心设计赌注 · utility-first / constraints over creativity / no semantic class / co-location with markup
