P2 · Utility-first 心智

Tailwind 是 utility-first 哲学的工具实现 —— class name 描述视觉而非语义、CSS 与 markup co-location、约束优于创造。这两章不读，后面引擎、变体、组件模式都是表面。

章节数 2 主要来源 Adam Wathan 2017 blog + "Refactoring UI" 书 + tailwindcss.com/docs Theme 状态 ✅ 全部完成

本阶段覆盖

  • semantic CSS 的悖论 · 不同组件名样式相同；class name 与样式没有 1:1 关系
  • 抽象层级错位 · semantic naming 假设组件名稳定，但产品迭代中组件名总在变
  • co-location with markup · 与 React / JSX 的"组件 = 视觉 + 行为"哲学契合
  • @apply 反模式 · 把 utility @apply 成 semantic class 等于回到 BEM 世界，失去 Tailwind 价值
  • spacing scale · 4px-based 0/0.5/1/1.5/2/2.5/3/4/5/6/8/10/12/16/20/24...
  • color palette · v4 24 个色家族 × 11 个 shade，OKLCH 色彩空间
  • typography scale · text-xs/sm/base/lg/xl/2xl... 含 font-size + line-height 配对
  • breakpoints · sm 640 / md 768 / lg 1024 / xl 1280 / 2xl 1536
  • OKLCH 色彩空间 · perceptually uniform、wide gamut 支持
  • 设计 token 与 design system · @theme 扩展、与 Figma 同步
