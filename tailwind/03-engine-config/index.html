P3 · 引擎与配置 ⭐

Tailwind 怎么把"项目里用了哪些 utility class"识别出来 + 编译成最终 CSS？v1-v3 的 JIT 演化 + v4 的 Oxide Rust 重写让这条 pipeline 跑得快。v4 的 CSS-first 配置（@theme / @utility 替代 tailwind.config.js）是 9 年来最大的断代式重写。这阶段把引擎与配置都讲透。

章节数 2 ⭐ 主要来源 tailwindcss.com/blog v3 / v4 release notes + Adam Wathan + Tailwind Labs YouTube 状态 ✅ 全部完成

本阶段覆盖

  • v1-v2.2 时代 · 完整生成所有 utility（几 MB CSS）+ PurgeCSS 删未用
  • v2.2 JIT 实验 · 扫源码、按需生成、arbitrary values 第一次成为可能
  • v3.0 JIT 默认 · 完整 build 1-3s，增量 50-200ms
  • v4 Oxide 引擎 · Rust 重写、完整 build 100-200ms（v3 的 1/5）、增量 5ms（v3 的 1/40）
  • 扫描算法 · 不解析整个 AST，用正则 + 规则识别可能是 class 的字符串
  • 动态 class 处理 · safelist / 完整 class string 写法 / 不要拼字符串
  • v3 → v4 迁移 · tailwindcss/upgrade codemod / 默认值微调 / @apply 仍可用
  • CSS-first 配置 · @import / @theme / @utility / @custom-variant / @source 五大指令
  • 与 build tool 集成 · Vite plugin / PostCSS / Webpack（v4.2 新增）/ Next.js 零配置
