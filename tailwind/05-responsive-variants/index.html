P5 · 响应式 + 变体

Tailwind 的强大很大程度来自变体系统 —— mobile-first 响应式 + 暗色模式 + hover/focus/aria/data/group/peer/has 变体让"无 JS 实现复杂交互"成为可能。这两章把响应式策略、变体全谱、arbitrary values 何时用、自定义变体讲清。

章节数 2 主要来源 tailwindcss.com/docs Responsive / Hover / Dark Mode / Variants 全套 状态 ✅ 全部完成

本阶段覆盖

  • Mobile-first · 默认是最小尺寸样式，breakpoint 前缀逐级覆盖（与传统 desktop-first 相反）
  • Breakpoints · sm 640 / md 768 / lg 1024 / xl 1280 / 2xl 1536（v4 微调）
  • Container Queries · @sm: @md: 容器前缀，组件级响应
  • 暗色 3 策略 · media（默认跟系统）/ class（手动 toggle）/ selector（v4 自定义）
  • next-themes 集成 · class 策略 + localStorage 持久化标准模式
  • 状态变体 · hover / focus / focus-within / focus-visible / active / disabled
  • 结构变体 · first / last / odd / even / nth-* / empty
  • group-* · 父元素状态控制子元素样式（无 JS 实现 hover 联动）
  • peer-* · 同级元素状态控制兄弟样式（checkbox 控制兄弟显示）
  • data-* / aria-* · 配合 Radix UI / a11y 友好
  • has-[...]: · CSS :has() 在 Tailwind 的封装
  • Arbitrary values · w-[clamp(...)] escape utility 系统、何时用
  • 自定义变体 · v4 @custom-variant
