状态变体 + arbitrary values

Tailwind 强大的根因之一在变体系统 —— hover / focus / active / aria / data / group / peer / has 让"无 JS 实现复杂交互"成为可能。这一章把变体全谱讲透 + arbitrary values 何时用 + 自定义变体。读完能写出 95% 的 UI 状态联动而不需要 JS。

阅读时长 ~30 分钟 前置 P5.1 响应式 + 暗色模式 主要来源 tailwindcss.com/docs Hover, Focus, Active / Variants 全套

01 速览

  1. Tailwind 变体系统按"什么状态触发"分 5 类：基础状态（hover/focus/active）/ 结构（first/last/odd/even）/ 群组同级（group-* / peer-*）/ 数据 / aria（data-* / aria-*）/ has-[]:（含某子元素的父元素）。
  2. 基础状态变体：hover: / focus: / focus-within: / focus-visible: / active: / disabled: / visited:。与原生 CSS 伪类一一对应。
  3. 结构变体：first: / last: / odd: / even: / nth-3:（第 3 个）/ empty:（无内容）。大幅减少 JS 计算 className 的需求。
  4. group-*：父元素加 group class，子元素用 group-hover: 响应父 hover。"hover 卡片显示按钮"经典场景。
  5. peer-*：兄弟元素加 peer class，后续兄弟用 peer-checked: 响应。"checkbox 控制兄弟显示"经典场景。
  6. data-* / aria-*：响应 data attribute / aria attribute。与 Radix UI（用 data-state）配合极佳。例如 data-[state=open]:translate-y-0。
  7. has-[...]:（v3.4+）：CSS :has() 在 Tailwind 的封装。父元素响应子元素状态。has-[input:checked]:bg-blue-500 = "含选中 input 的父元素变蓝"。
  8. arbitrary values（v3+）：w-[123px] / bg-[#ff5733] / grid-cols-[repeat(3,1fr)]。escape utility 系统的逃生口。临时一次性值用、不要滥用，否则失去 design system 约束。
  9. 自定义变体（@custom-variant）：v4 让自定义变体一行 CSS 搞定，替代 v3 plugin function。@custom-variant theme-dark (&:where([data-theme=dark] *))。
  10. 变体可任意组合（v3+ JIT 引擎能力）：group-hover:focus:dark:ring-2 完全合法。这让"无 JS 实现复杂交互"变得真实可行。

02 底层逻辑

基础状态变体

变体对应 CSS用途
hover::hover鼠标悬停（移动端不触发）
focus::focus键盘 / 鼠标 focus
focus-within::focus-within子元素 focus 时父元素响应
focus-visible::focus-visible仅键盘 focus（鼠标 click 不触发）
active::active按下时
disabled::disabled禁用状态
visited::visited已访问的链接（受浏览器隐私限制）

实战常见组合：

html 
<!-- Button 标准状态样式 -->
<button class="
  px-4 py-2 rounded-md
  bg-blue-500 text-white
  hover:bg-blue-600
  focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2
  active:bg-blue-700
  disabled:opacity-50 disabled:cursor-not-allowed
  transition-colors
">
  Click
</button>

结构变体

响应元素在 DOM 中的位置：

变体对应 CSS用途
first::first-child列表第一个
last::last-child列表最后一个
only::only-child唯一子元素
odd::nth-child(odd)奇数（1/3/5...）
even::nth-child(even)偶数
nth-N::nth-child(N)第 N 个
empty::empty无内容
html 
<!-- 表格斑马纹（无 JS） -->
<tbody>
  <tr class="even:bg-gray-50 odd:bg-white">
    <td>Row 1</td>
  </tr>
  <tr class="even:bg-gray-50 odd:bg-white">
    <td>Row 2</td>
  </tr>
</tbody>

<!-- 列表第一项无上 border -->
<ul>
  <li class="border-t first:border-t-0 py-2">Item 1</li>
  <li class="border-t first:border-t-0 py-2">Item 2</li>
</ul>

group-* · 父元素状态控制子元素

核心机制：父元素加 group class、子元素用 group-{state}: 前缀响应父状态。

html 
<!-- hover 卡片，子元素的"读更多"显现 -->
<article class="group p-4 border rounded-lg cursor-pointer hover:shadow-lg transition">
  <h2>Article Title</h2>
  <p>Excerpt...</p>
  <span class="opacity-0 group-hover:opacity-100 transition-opacity">
    Read more →
  </span>
</article>

变体支持的状态：group-hover: / group-focus: / group-active: / group-disabled: / group-data-*: / group-aria-*: 等。

命名 group（解决嵌套）：

html 
<!-- 嵌套 group：用名字区分 -->
<article class="group/card">
  <header class="group/header hover:bg-gray-100">
    <h2 class="group-hover/card:text-blue-500">
      <!-- 响应外层 card hover -->
    </h2>
    <span class="group-hover/header:underline">
      <!-- 响应内层 header hover -->
    </span>
  </header>
</article>

peer-* · 同级元素状态控制

类似 group-*，但作用于兄弟元素。peer 元素加 class，后续兄弟响应：

html 
<!-- Checkbox 控制兄弟元素显示（无 JS） -->
<label class="flex gap-2 items-center cursor-pointer">
  <input type="checkbox" class="peer" />
  <span class="peer-checked:line-through peer-checked:text-gray-400">
    Done item
  </span>
</label>

<!-- Radio 选中显示对应详情 -->
<label class="block">
  <input type="radio" name="plan" value="basic" class="peer/basic" />
  <span>Basic</span>
  <p class="hidden peer-checked/basic:block text-gray-500">
    $9/month, basic features
  </p>
</label>

注意：peer 必须是前序兄弟。CSS 没有"前面的兄弟"选择器，所以 peer 元素必须在被影响元素之前。

data-* / aria-* 变体

响应 data attribute / aria attribute。与 Radix UI / Headless UI 配合极佳：

tsx 
// Radix Dialog 内部用 data-state="open" / "closed"
import * as Dialog from '@radix-ui/react-dialog';

<Dialog.Content className="
  data-[state=open]:animate-in
  data-[state=closed]:animate-out
  data-[state=open]:fade-in-0
  data-[state=closed]:fade-out-0
">
  ...
</Dialog.Content>

aria-* 变体增加 a11y：

html 
<!-- aria-checked 控制视觉 -->
<button role="switch" aria-checked="true" class="
  bg-gray-300 aria-checked:bg-blue-500
  transition-colors
">
  ...
</button>

<!-- aria-expanded 控制旋转 -->
<button aria-expanded={isOpen} class="...">
  <ChevronIcon class="aria-expanded:rotate-180 transition-transform" />
</button>

has-[...]: · CSS :has() 封装

v3.4+ 新增。父元素响应子元素状态。详细机制见 P4。

html 
<!-- 含选中 checkbox 的 label 高亮 -->
<label class="
  block p-4 border rounded-lg cursor-pointer
  has-[input:checked]:bg-blue-50
  has-[input:checked]:border-blue-500
">
  <input type="checkbox" />
  <span>Accept terms</span>
</label>

<!-- 含错误 input 的 form 加红边 -->
<form class="
  border rounded-lg p-4
  has-[input:invalid]:border-red-500
  has-[input:invalid]:bg-red-50
">
  <input required minLength={3} />
</form>

<!-- 含 hover 任意子元素时父元素阴影 -->
<article class="has-[*:hover]:shadow-lg transition-shadow">
  ...
</article>

Arbitrary values · 何时用

Tailwind 默认 scale 不一定覆盖所有需求。arbitrary values 是 escape：

html 
<!-- 任意尺寸 -->
<div class="w-[123px] h-[456px]">...</div>

<!-- 任意颜色 -->
<div class="bg-[#ff5733] text-[hsl(200,50%,40%)]">...</div>

<!-- 任意 grid layout -->
<div class="grid grid-cols-[200px_1fr_300px]">...</div>

<!-- 任意 calc -->
<div class="w-[calc(100%-2rem)]">...</div>

<!-- 任意 selector（CSS variable 用法） -->
<div class="[--my-var:#ff5733] bg-[var(--my-var)]">...</div>

何时用 / 何时不用：

  • ✅ 临时一次性值（"这次设计稿就这个 13px"）
  • ✅ 必须精确像素（margin / padding 必须 1px）
  • ✅ 复杂 calc / grid layout
  • ❌ 项目里反复用的值 → 加到 @theme（成为 token）
  • ❌ 团队定的 design system 里没有的值 → 与设计师对齐

Adam Wathan 的建议："arbitrary 是后门、不是前门"。能用 scale 的就用 scale；scale 没有的优先扩展 @theme；临时实在没办法才 arbitrary。

变体任意组合

v3+ JIT 引擎让变体可以深度链式组合：

html 
<!-- 父 hover + focus + dark 全满足时 ring -->
<div class="group">
  <button class="
    group-hover:focus:dark:ring-2
    group-hover:focus:dark:ring-blue-400
  ">...</button>
</div>

<!-- 移动端 dark + group-hover 时不同样式 -->
<div class="group">
  <span class="md:dark:group-hover:text-blue-400">...</span>
</div>

变体顺序通常无关（CSS 选择器特异性 calc）。但响应式前缀建议在最外层（更易读）：md:dark:hover:bg-blue-500 比 dark:hover:md:bg-blue-500 更直观。

v4 自定义变体 @custom-variant

v3 时代自定义变体要写 plugin function（10+ 行 JS）。v4 一行 CSS：

css 
/* 桌面 hover（仅在桌面设备 hover 生效） */
@custom-variant desktop-hover (&:hover:where(@media (hover: hover) and (pointer: fine) *));

/* data-state */
@custom-variant data-open (&:where([data-state=open]));
@custom-variant data-active (&:where([data-state=active]));

/* RTL 文化 */
@custom-variant rtl (&:where([dir=rtl] *));

/* 多主题（不止 light/dark） */
@custom-variant theme-sepia (&:where([data-theme=sepia], [data-theme=sepia] *));
@custom-variant theme-high-contrast (&:where([data-theme=high-contrast], [data-theme=high-contrast] *));

用法（与内置变体相同）：

html 
<button class="hover:bg-blue-500 desktop-hover:bg-blue-600">
  <!-- 移动 tap 时 hover:bg-blue-500 -->
  <!-- 桌面 hover 时 desktop-hover:bg-blue-600 -->
</button>

<div data-state="open" class="data-open:translate-y-0">...</div>

<p class="text-left rtl:text-right">Hello</p>

03 应用场景

场景 1 · 无 JS 的 Tab 实现

html 
<!-- Radio + peer 实现 tab 切换（不需要 JS） -->
<div class="border-b mb-4">
  <label class="cursor-pointer">
    <input type="radio" name="tab" value="overview" class="peer/overview hidden" defaultChecked />
    <span class="px-4 py-2 inline-block peer-checked/overview:text-blue-500 peer-checked/overview:border-b-2 peer-checked/overview:border-blue-500">
      Overview
    </span>
  </label>
  <label class="cursor-pointer">
    <input type="radio" name="tab" value="details" class="peer/details hidden" />
    <span class="px-4 py-2 inline-block peer-checked/details:text-blue-500 peer-checked/details:border-b-2 peer-checked/details:border-blue-500">
      Details
    </span>
  </label>
</div>

<!-- Tab content（peer-checked 控制显示） -->
<div class="hidden peer-checked/overview:block">Overview content</div>
<div class="hidden peer-checked/details:block">Details content</div>

场景 2 · 与 Radix UI Dialog 集成

tsx 
import * as Dialog from '@radix-ui/react-dialog';

<Dialog.Root>
  <Dialog.Trigger className="px-4 py-2 bg-blue-500 text-white rounded">
    Open
  </Dialog.Trigger>

  <Dialog.Portal>
    <Dialog.Overlay className="
      fixed inset-0 bg-black/50
      data-[state=open]:animate-in data-[state=open]:fade-in-0
      data-[state=closed]:animate-out data-[state=closed]:fade-out-0
    " />
    <Dialog.Content className="
      fixed left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2
      bg-white rounded-lg p-6 shadow-lg
      data-[state=open]:animate-in
      data-[state=open]:zoom-in-95
      data-[state=closed]:zoom-out-95
    ">
      <Dialog.Title>Confirm</Dialog.Title>
      <Dialog.Close className="...">Cancel</Dialog.Close>
    </Dialog.Content>
  </Dialog.Portal>
</Dialog.Root>

场景 3 · 表单状态联动

html 
<!-- input invalid 时 label 变红、错误消息显示 -->
<div class="flex flex-col gap-1">
  <label class="text-sm peer-invalid:text-red-500">Email</label>
  <input
    type="email"
    required
    class="peer px-3 py-2 border rounded
           valid:border-green-500
           invalid:border-red-500

04 典型代码

代码 1 · 完整 Button 状态样式

tsx 
function Button({ children, ...props }) {
  return (
    <button
      className="
        inline-flex items-center justify-center gap-2
        px-4 py-2 rounded-md
        text-sm font-medium
        bg-blue-500 text-white
        hover:bg-blue-600
        focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2
        active:bg-blue-700
        disabled:opacity-50 disabled:cursor-not-allowed disabled:pointer-events-none
        dark:bg-blue-400 dark:hover:bg-blue-300 dark:active:bg-blue-200
        transition-colors
      "
      {...props}
    >
      {children}
    </button>
  );
}

代码 2 · Card 加 group hover

tsx 
function ArticleCard({ title, excerpt }) {
  return (
    <article className="
      group relative

      hover:shadow-lg hover:-translate-y-0.5
      transition-all
    ">
      <h2 className="text-lg font-semibold group-hover:text-blue-500 transition-colors">
        {title}
      </h2>
      <p className="text-gray-500 mt-2">{excerpt}</p>
      <span className="
        inline-block mt-2 text-blue-500 text-sm
        opacity-0 translate-x-0
        group-hover:opacity-100 group-hover:translate-x-1
        transition-all
      ">
        Read more →
      </span>
    </article>
  );
}

// 用法：移动设备 tap 不触发，桌面 hover 才触发

}

05 常见陷阱

  • group-hover 没在父加 group class —— 必须父元素 class="group"，子元素的 group-hover: 才生效
  • peer-* 兄弟顺序错 —— peer 必须在被影响元素之前。CSS 没有"前面兄弟"选择器
  • 嵌套 group 不命名 —— 子 group-hover 响应内层还是外层不明确。命名（group/card）解决
  • data-* 变体方括号语法错 —— data-[state=open]: 是正确的；data-state=open: 不是
  • has-[]: 在老浏览器 —— :has() 2023 年支持，2026 95%+。极老浏览器项目谨慎
  • arbitrary values 滥用 —— 项目反复出现的值应该加到 @theme 成为 token，不要每次写 arbitrary
  • arbitrary values 用变量 —— w-[${size}px] 不工作（JIT 看到的是模板字符串）。用 inline style 或固定 size enum
  • focus-visible 与 focus 混淆 —— focus 鼠标 + 键盘都触发；focus-visible 只键盘。Button 类组件用 focus-visible 更优（鼠标点击不显示难看的 ring）
  • hover 在移动端被 tap 触发然后停在 hover 状态 —— 移动设备模拟 hover 体验差。用自定义 desktop-hover 变体限定桌面
  • 变体链顺序写得难读 —— 推荐顺序：响应式前缀 → 主题 → 状态。md:dark:hover:bg-blue-500 比 dark:hover:md:bg-blue-500 直观
  • aria-* 变体写错 —— aria-[checked=true]: 是正确的；aria-checked=true: 不是
  • 在 div 上加 disabled: —— disabled 伪类只对 form 元素（input/button/select/textarea）生效。div 的 disabled 不工作
  • 用 first: 在不是直接子元素 —— first/last/odd/even 是 :nth-child，必须直接父子关系，不跨层级
  • peer 同名嵌套 —— peer/x 在嵌套 form 里冲突。用不同名字区分
  • @custom-variant 写在子 CSS 文件 —— 必须在 root（@import 入口同级）；子文件 @custom-variant 不会生效
