WebSocket 全双工

WebSocket 是唯一真正的全双工浏览器协议：服务端可以主动推送，客户端任意时刻发送。核心复杂性不在 API —— API 只有 4 个事件 —— 而在连接管理：心跳、重连、背压、鉴权。本章把生产级 WebSocket 客户端的关键点讲清楚。

阅读时长 ~22 分钟 前置 P3 异步 · P6-01 Fetch MDN WebSocket API

01 速览

javascript 
// 创建连接（ws:// 或 wss://）
const ws = new WebSocket("wss://api.example.com/socket", ["chat.v1"]);
// 第二参：subprotocol（可选数组，服务端选一个）

ws.binaryType = "arraybuffer";   // 或 "blob"（默认）

// 4 个事件
ws.addEventListener("open", () => console.log("connected"));
ws.addEventListener("message", (e) => {
  typeof e.data === "string" ? handleText(e.data) : handleBinary(e.data);
});
ws.addEventListener("close", (e) => {
  console.log("closed", e.code, e.reason, "clean?", e.wasClean);
});
ws.addEventListener("error", () => console.error("error"));

// 发送
ws.send("hello");                // 文本
ws.send(new Uint8Array([1,2,3])); // 二进制
ws.send(blob);

// 主动关闭
ws.close(1000, "bye");

// 状态
ws.readyState;                   // 0 CONNECTING / 1 OPEN / 2 CLOSING / 3 CLOSED
ws.bufferedAmount;               // 未发送的字节数（背压信号）

02 底层逻辑

帧格式简述

WebSocket 数据以帧传输（不是 HTTP body）：

  • opcode：0x1 text / 0x2 binary / 0x8 close / 0x9 ping / 0xA pong
  • fin bit：是否最后一帧（支持分片）
  • mask：客户端→服务端必须 mask（防中间代理攻击）
  • payload length：7/16/64 bit 长度
  • payload：数据

浏览器 JS 看不到这一层 —— 直接收 string 或 binary。但服务端库需要处理这些（ws、uWebSockets.js 等）。

readyState 状态机

值 常量 含义
0 WebSocket.CONNECTING 握手中
1 WebSocket.OPEN 可发收
2 WebSocket.CLOSING 已调 close，等服务端 ACK
3 WebSocket.CLOSED 完全关闭（含握手失败）

Close 代码语义

code 含义
1000 正常关闭
1001 Going Away（服务重启 / 浏览器导航离开）
1006 异常关闭（网络中断，代码不来自对方）
1008 Policy Violation（鉴权失败等）
1011 Internal Server Error
3000-3999 库保留
4000-4999 应用自定义

背压：bufferedAmount

ws.send() 是非阻塞的 —— 放入本地 buffer 后立即返回。bufferedAmount 是尚未发出的字节数。发送速度 > 网络带宽时会堆积，需要应用层做背压：

javascript 
function sendWithBackpressure(ws, data, maxBuffer = 1024 * 1024) {
  return new Promise((resolve, reject) => {
    function trySend() {
      if (ws.readyState !== WebSocket.OPEN) return reject(new Error("not open"));
      if (ws.bufferedAmount < maxBuffer) {
        ws.send(data);
        resolve();
      } else {
        setTimeout(trySend, 50);         // 等 buffer 降下来
      }
    }
    trySend();
  });
}

// 或监测 bufferedAmount 阈值暂停生产端

心跳（ping/pong）

WebSocket 规范有 ping/pong 帧（opcode 9/A），但浏览器 JS 没有 API 发送自定义 ping（只能接收自动回复 pong）。所以心跳要走应用层：

javascript 
// 应用层心跳
let heartbeatTimer;
function startHeartbeat(ws, intervalMs = 30_000) {
  heartbeatTimer = setInterval(() => {
    if (ws.readyState === WebSocket.OPEN) {
      ws.send(JSON.stringify({ type: "ping", ts: Date.now() }));
    }
  }, intervalMs);
}
function stopHeartbeat() { clearInterval(heartbeatTimer); }

// 心跳目的：
// 1. 探测连接是否仍活着（中间代理可能 kill 空闲连接）
// 2. 保活（某些代理 60s 无数据就关）
// 3. 监测延迟（计算 ping-pong RTT）

断线重连与指数退避

javascript 
class ReconnectingWS extends EventTarget {
  #url;
  #ws = null;
  #retry = 0;
  #maxRetry = 10;
  #closedByUser = false;

  constructor(url) {
    super();
    this.#url = url;
    this.#connect();
  }

  #connect() {
    this.#ws = new WebSocket(this.#url);
    this.#ws.addEventListener("open", () => {
      this.#retry = 0;
      this.dispatchEvent(new Event("open"));
    });
    this.#ws.addEventListener("message", (e) => {
      this.dispatchEvent(new MessageEvent("message", { data: e.data }));
    });
    this.#ws.addEventListener("close", (e) => {
      this.dispatchEvent(new CloseEvent("close", { code: e.code, reason: e.reason }));
      if (!this.#closedByUser && this.#retry < this.#maxRetry) {
        const delay = Math.min(30_000, 1000 * 2 ** this.#retry) + Math.random() * 1000;
        this.#retry++;
        setTimeout(() => this.#connect(), delay);
      }
    });
  }

  send(data) { this.#ws?.send(data); }
  close() { this.#closedByUser = true; this.#ws?.close(); }
}

鉴权方案

难点：WebSocket 构造器不能自定义 header（浏览器 API 限制）。几种常见解法：

  • Cookie：同源时自动带。HttpOnly cookie 最安全。跨源要 CORS 不适用 WS，用 subdomain + SameSite 方案。
  • URL 参数：wss://api/socket?token=xxx —— 会出现在日志里。
  • Subprotocol：把 token 放 Sec-WebSocket-Protocol，服务端在握手时读。
  • 连接后 auth message：OPEN 后先 send token 消息，server 验证后才推数据。
javascript 
// subprotocol 作 token 承载
const ws = new WebSocket(url, ["app.v1", `bearer.${token}`]);
// 服务端 Sec-WebSocket-Protocol header 会有这两个字符串，取 token 验证

// 或连接后鉴权
ws.onopen = () => ws.send(JSON.stringify({ type: "auth", token }));

ws.addEventListener("message", (e) => {

function sendMessage(text) {
  if (ws.readyState !== WebSocket.OPEN) {
    pendingMessages.push(text);
    return;
  }
  ws.send(JSON.stringify({ type: "send", payload: { text } }));
}

多 tab 通过 SharedWorker + WS

javascript 
// 10 个 tab 都连一个 WS 浪费资源 → 用 SharedWorker 共享一个连接
// shared-ws.worker.js
let ws = new WebSocket(...);
const ports = new Set();
onconnect = (e) => {
  const port = e.ports[0];
  ports.add(port);
  port.onmessage = (msg) => ws.send(msg.data);
  port.start();
};
ws.onmessage = (e) => {
  for (const port of ports) port.postMessage(e.data);
};

// main.js
const worker = new SharedWorker("shared-ws.worker.js");
worker.port.onmessage = (e) => handleMessage(e.data);
worker.port.start();

04 典型代码

类型安全的 WS 客户端

javascript 
type ServerToClient =
  | { type: "new-message"; payload: Message }
  | { type: "user-join"; payload: User }
  | { type: "error"; message: string };

type ClientToServer =
  | { type: "send"; payload: { text: string } }
  | { type: "typing" }
  | { type: "ping" };

function createClient(url: string) {
  const ws = new WebSocket(url);
  const bus = new EventTarget();

  ws.addEventListener("message", (e) => {
    const msg = JSON.parse(e.data) as ServerToClient;
    bus.dispatchEvent(new CustomEvent(msg.type, { detail: msg }));
  });

  return {
    send(msg: ClientToServer) { ws.send(JSON.stringify(msg)); },
    on: bus.addEventListener.bind(bus),
    close: () => ws.close(),
  };
}

带心跳 + 重连的完整客户端

javascript 
class WSClient extends EventTarget {
  constructor(url, { heartbeatMs = 30_000, heartbeatTimeout = 10_000 } = {}) {
    super();
    this.url = url;
    this.heartbeatMs = heartbeatMs;
    this.heartbeatTimeout = heartbeatTimeout;
    this.retry = 0;
    this.connect();
  }

  connect() {
    this.ws = new WebSocket(this.url);
    this.ws.addEventListener("open", () => {
      this.retry = 0;
      this.startHeartbeat();
      this.dispatchEvent(new Event("open"));
    });
    this.ws.addEventListener("message", (e) => {
      if (e.data === "pong") { this.gotPong(); return; }
      this.dispatchEvent(new MessageEvent("message", { data: e.data }));
    });
    this.ws.addEventListener("close", () => {
      this.stopHeartbeat();
      this.scheduleReconnect();
    });
  }

  startHeartbeat() {
    this.pingTimer = setInterval(() => {
      if (this.ws.readyState === WebSocket.OPEN) {
        this.ws.send("ping");
        this.pongTimer = setTimeout(() => this.ws.close(4000, "pong timeout"),
                                     this.heartbeatTimeout);
      }
    }, this.heartbeatMs);
  }
  gotPong() { clearTimeout(this.pongTimer); }
  stopHeartbeat() { clearInterval(this.pingTimer); clearTimeout(this.pongTimer); }

  scheduleReconnect() {
    const delay = Math.min(30_000, 1000 * 2 ** this.retry++) + Math.random() * 1000;
    setTimeout(() => this.connect(), delay);
  }

  send(data) { this.ws?.readyState === WebSocket.OPEN && this.ws.send(data); }
  close() { this.ws?.close(1000); }
}

发送队列 + flush

javascript 
class WSBuffered {
  constructor(ws) {
    this.ws = ws;
    this.queue = [];
    ws.addEventListener("open", () => this.flush());
  }
  send(data) {
    if (this.ws.readyState === WebSocket.OPEN) this.ws.send(data);
    else this.queue.push(data);
  }
  flush() {
    for (const data of this.queue.splice(0)) this.ws.send(data);
  }
}

// 用户在 connecting 阶段发 message 不会丢

订阅/发布模型

javascript 
class PubSub {
  constructor(ws) {
    this.ws = ws;
    this.handlers = new Map();   // topic → Set<fn>
    ws.addEventListener("message", (e) => {
      const { topic, data } = JSON.parse(e.data);
      this.handlers.get(topic)?.forEach(fn => fn(data));
    });
  }
  subscribe(topic, fn) {
    if (!this.handlers.has(topic)) {
      this.handlers.set(topic, new Set());
      this.ws.send(JSON.stringify({ cmd: "subscribe", topic }));
    }
    this.handlers.get(topic).add(fn);
    return () => this.unsubscribe(topic, fn);
  }
  unsubscribe(topic, fn) {
    const set = this.handlers.get(topic);
    set?.delete(fn);
    if (set?.size === 0) {
      this.handlers.delete(topic);
      this.ws.send(JSON.stringify({ cmd: "unsubscribe", topic }));
    }
  }
}

05 常见陷阱

浏览器 API 不能自定义 header

new WebSocket 没有第三参 headers。Authorization 只能走 cookie / URL 参数 / subprotocol / 连接后 auth 消息。Node ws 库有 headers 选项（非浏览器 API）。

close event 的 code 1006

异常关闭时（网络中断）code 是 1006（规范）不会来自对方，而是浏览器补的。wasClean: false。正常的服务端主动关是 1000-1015 或 3000+ / 4000+。

忘记做重连

网络抖动、4G 切 WiFi、服务重启都会断连。生产 WS 客户端必须有自动重连 + 指数退避 + jitter。不然用户要手动刷页面。

bufferedAmount 持续增长 → OOM

send 太快 + 网络慢 → buffer 堆积。生产级客户端要监控 bufferedAmount，超阈值暂停发送或丢弃。

ping/pong 无 JS API

浏览器 WebSocket JS 只暴露 data frame 事件。WS 协议层的 ping/pong 浏览器自动处理（不给 JS 通知）。心跳要走应用层（send 一个自定义 ping 消息）。

心跳超时判定

发 ping 要同时启动 pong 超时计时器。如果 bufferedAmount > 0 就说不定 ping 根本没发出。要检查 bufferedAmount 或直接让服务端主动 ping。

反向代理 60s 空闲断连

Nginx / CDN 默认 60s 无数据就关 WS。心跳间隔要小于该值（一般 30s）。Cloudflare Workers 对 WS 有额外规则，按文档调。

生产用 wss:// 不用 ws://

ws:// 明文传输，中间人可看/改。wss:// 是 TLS 加密的 WebSocket。混合内容（HTTPS 页面 + ws:// 连接）被浏览器阻止。
