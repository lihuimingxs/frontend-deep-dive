动态 import 与代码分割

import() 是"类函数"表达式，返回 Promise<namespace object>，可以在任何位置使用（包括非 ESM 模块）。它是代码分割的基石 —— Webpack/Rollup/Vite 看到 import() 就会把目标拆成独立 chunk。本章讲语义 + 打包器的拆分规则 + preload/prefetch。

阅读时长 ~18 分钟 前置 01 ESM · P3 Promise MDN import()

01 速览

javascript 
// 表达式形式（不是语句）—— 可以放任何位置
const mod = await import("./features/heavy.mjs");

// 解构取 named export
const { render } = await import("./ui.mjs");

// 默认导出
const Component = (await import("./Comp.mjs")).default;

// 条件加载
if (user.isAdmin) {
  const { AdminPanel } = await import("./admin.mjs");
  mount(AdminPanel);
}

// import assertions（ES2023）/ with（ES2025）
const data = await import("./config.json", { with: { type: "json" } });
// 旧语法：import("./config.json", { assert: { type: "json" } })

02 底层逻辑

import() 的规范语义

  1. 解析 specifier 到 URL（和静态 import 同规则）。
  2. 如果 module graph 里已存在 → 返回已有 module 的 namespace object 的 Promise。
  3. 否则启动 parse → link → evaluate 流程，最后 resolve 为 namespace object。
  4. 失败任一环节 → reject。
javascript 
// 同一 specifier 多次 import() 只加载一次
const a = await import("./mod.mjs");
const b = await import("./mod.mjs");
a === b;            // true（同一 namespace object）

// 不同字面量路径可能指向同一文件但被当不同 key（取决于路径规范化）
await import("./mod.mjs");
await import("./mod.mjs?v=2");   // 不同 key，重新加载（浏览器里可以这么 bust cache）

import() 不是函数

规范层 import() 是一个表达式（语法结构），不是函数调用。所以：

javascript 
const load = import;              // ❌ SyntaxError
const f = () => import;           // ❌ SyntaxError
const loader = (p) => import(p);  // ✓ 箭头函数包一层

// 但在工具链眼里它是"带特殊语义的 call expression"
// Webpack 看到 import("./x") 就生成对应的 chunk manifest

打包器的代码分割

当构建工具（Webpack / Rollup / Vite / esbuild）遇到 import("./foo")：

  1. 把 foo.mjs 及其仅被动态 import 的依赖拆成独立 chunk。
  2. 在"发起处"生成运行时代码：__webpack_require__.e("chunk-id").then(...)。
  3. 运行时通过 <script> 注入或 import() 原生动态加载该 chunk。
  4. 成功后 resolve Promise，拿到 namespace object。

Webpack Magic Comments

javascript 
// 命名 chunk
import(/* webpackChunkName: "admin" */ "./admin.mjs");
// 输出：admin.[hash].js

// Prefetch（空闲时预取）
import(/* webpackPrefetch: true */ "./profile.mjs");
// <link rel="prefetch" href="...">

// Preload（并行加载）
import(/* webpackPreload: true */ "./critical.mjs");

// 按模式合并多个 import 到同一 chunk
import(/* webpackInclude: /\.json$/ */ `./locale/${lang}.json`);

浏览器原生 preload / prefetch

html 
// 原生 <link rel="modulepreload"> 对 ESM 最准确
<link rel="modulepreload" href="/js/admin.mjs">
// 浏览器：高优先级下载 + 解析 + 建 module graph（但不执行）
// 之后 import("./admin.mjs") 立即从缓存取

// <link rel="preload" as="script"> 是通用版，但不会 parse + build graph
// modulepreload 比 preload 好 —— 除了 Safari 对它支持较晚

Import Attributes（ES2025）

javascript 
// 显式声明非 JS 模块类型
import data from "./config.json" with { type: "json" };
import styles from "./style.css"  with { type: "css" };

// 动态版
const data = await import("./config.json", { with: { type: "json" } });

// 不加 type 时加载非 JS 文件某些环境会拒绝（规范强制类型标注）
// CDN / 跨源场景：浏览器对 type 不匹配的响应会拒绝解析

03 应用场景

路由级懒加载

javascript 
// React
const routes = [
  { path: "/", component: () => import("./pages/Home") },
  { path: "/admin", component: () => import("./pages/Admin") },
  // admin chunk 只有访问 /admin 时才下载
];

// React Router v6+
import { lazy } from "react";
const Admin = lazy(() => import("./pages/Admin"));

// Next.js App Router 自动处理（每个路由段是独立 chunk）

条件重型库加载

javascript 
// 只有导出 Excel 功能被点击时才加载 exceljs (2MB+)
async function exportExcel(data) {
  const ExcelJS = (await import("exceljs")).default;
  const wb = new ExcelJS.Workbook();
  // ...
}

// 只有发生错误时加载 error reporting SDK
try { doStuff(); }
catch (err) {
  const Sentry = await import("@sentry/browser");
  Sentry.captureException(err);
}

i18n 按需加载

javascript 
// locale 文件只加载当前语言
const translations = (await import(`./locales/${navigator.language}.json`, {
  with: { type: "json" },
})).default;

// 打包器（Vite）会把 ./locales/*.json 打成单独文件并保留动态路径匹配

预取 + 交互触发

javascript 
// 在 hover / focus 时开始加载，click 时已就绪
let loadPromise = null;
button.addEventListener("mouseenter", () => {
  loadPromise ??= import("./heavy-feature.mjs");
}, { once: true });

button.addEventListener("click", async () => {
  const { run } = await (loadPromise ??= import("./heavy-feature.mjs"));
  run();
});

// 用户 hover → 下载 → click → 立即执行
// Twitter/GitHub 常见优化手段

插件系统

javascript 
// 从配置动态加载多个插件
async function loadPlugins(pluginList) {
  const plugins = await Promise.all(pluginList.map(p => import(p)));
  return plugins.map(mod => mod.default ?? mod);
}

const plugins = await loadPlugins([
  "./plugins/analytics.mjs",
  "./plugins/auth.mjs",
  process.env.ENABLE_DEV ? "./plugins/devtools.mjs" : null,
].filter(Boolean));

04 典型代码

错误处理

javascript 
async function safeImport(path, { retries = 2, fallback = null } = {}) {
  for (let i = 0; i <= retries; i++) {
    try {
      return await import(path);
    } catch (err) {
      if (i === retries) {
        console.warn("dynamic import failed", path, err);
        return fallback;
      }
      // 网络错误重试（CDN 抖动、用户网络波动）
      await new Promise(r => setTimeout(r, 200 * 2 ** i));
    }
  }
}

// chunk 加载失败常见原因：
// 1. 部署更新，旧的 HTML 指向不存在的 chunk hash → 需要整页刷新
// 2. 用户网络断开 → 重试或提示
// 3. CSP 拦截 → 检查 script-src

部署更新后的 chunk 缺失恢复

javascript 
// 生产：deploy 后 chunk 名变了，旧标签页的 import() 找不到文件
window.addEventListener("vite:preloadError", (e) => {
  // Vite 专用事件
  if (confirm("应用有更新，是否重新加载？")) location.reload();
});

// 通用方式：catch 网络错误 → 检查是 chunk load fail → reload
function isChunkLoadError(err) {
  return err.message?.includes("Failed to fetch dynamically imported module");
}

async function safeLoad(importFn) {
  try { return await importFn(); }
  catch (err) {
    if (isChunkLoadError(err)) {
      location.reload();
      throw err;
    }
    throw err;
  }
}

Idle 时预取

javascript 
// 当前页空闲时预取下一个可能访问的路由
if (typeof requestIdleCallback !== "undefined") {
  requestIdleCallback(() => {
    import(/* webpackPrefetch: true */ "./pages/Next.mjs");
  });
}

// 或用 Network Information API 判断网络质量
if (navigator.connection?.effectiveType === "4g" && !navigator.connection.saveData) {
  import("./prefetchable-feature.mjs");
}

动态 import 保持 TypeScript 类型

javascript 
// TypeScript 4.1+ 的 Awaited 可以类型推导动态 import
type HeavyModule = Awaited<typeof import("./heavy")>;
// HeavyModule = { default: ..., namedExport: ... }

// 使用
async function load(): Promise<HeavyModule> {
  return import("./heavy");
}

const m = await load();
m.namedExport;   // 有类型提示

05 常见陷阱

import() 结果要取 default

import("./foo") 返回 namespace object，不是 default 导出本身。要 (await import("./foo")).default 或 const { default: Foo } = await import("./foo")。

动态变量 specifier 让打包器放弃拆分

import(userInputPath) 打包器无法静态分析 → 可能打包所有可能文件或根本不拆分。用模板字符串 `./locales/${lang}.json` 比 pathVar 更好（很多 bundler 能匹配固定前缀的路径模式）。

import() 是昂贵操作

每次动态 import 都可能 = 一次网络请求 + parse + link + execute。高频触发的热路径不要 import 大模块。"预热"一次后缓存结果引用。

chunk 404（deploy 后旧页面 stale）

前端部署后 chunk 文件名变了，长期打开的页面上的 import() 会 404。解决：监听错误 → 提示用户刷新；或用 service worker 缓存已下载 chunk。

preload / modulepreload 的 crossorigin 属性

跨源 CDN 的模块要加 crossorigin="anonymous"，否则预加载的资源会被 CORS 规则视为新请求，重新下载。Vite 等工具会自动加。

import() 的路径相对于当前文件

和静态 import 同规则。import("./x") 相对于当前模块 URL，不是调用方。封装 lazy loader 时注意 specifier 的解析上下文。

Top-level await + dynamic import 会延迟启动

const mod = await import("./x") 放 ESM 顶层会阻塞所有依赖该模块的东西。对启动时间敏感的代码考虑 lazy 或 preload + 非阻塞初始化。

CSP 禁止 script-src 可能拦截动态 chunk

CSP 严格时需允许 'self'（静态资源同源）或明确加 chunk 的 CDN 域。strict-dynamic 可以让信任 script 加载的子 script 免检。
