P6 · 网络 & 存储

Web 应用 80% 的 bug 都和网络、缓存、认证、存储策略相关。本阶段把现代浏览器提供的全部网络和存储原语梳理一遍：fetch 的完整生命周期、WebSocket 的帧与重连、SSE 的单向流、CORS 预检、CSP directives、Cookie 的 SameSite 与 CHIPS、LocalStorage 的同步陷阱、IndexedDB 的事务模型、Service Worker 的缓存策略与跨标签通信。

章节数 8 章 · 约 18 万字 对应原站 Part 3 · Network Requests / Storing data in the browser 状态 ✅ 已完成

章节列表

  1. 01 · Fetch API 完整语义

    Request / Response 不可变对象、body 流式读取、credentials / mode / cache 选项、AbortController 取消、ReadableStream 流式响应、与 XHR 的规范差异。

  2. 02 · XHR 的残余场景

    readystate 五态机、上传进度、responseType、同步请求（仅允许的场景）、与 Fetch 的能力对比与迁移策略。

  3. 03 · WebSocket 全双工

    HTTP Upgrade 握手、帧格式、bufferedAmount 背压、心跳与重连指数退避、subprotocol 鉴权、RSV/extension 字段、二进制消息。

  4. 04 · Server-Sent Events

    EventSource、text/event-stream 格式、Last-Event-Id 断线续传、与 WebSocket 的选型、fetch + ReadableStream 的替代实现。

  5. 05 · CORS 与 CSP

    同源策略规则、简单请求 vs 预检、credentials 的额外要求、Vary: Origin、Sec-Fetch-* hints、CSP directives / strict-dynamic / nonce / Report-Only、Trusted Types 回顾。

  6. 06 · Cookie / Storage

    Cookie 的 Domain / Path / SameSite / HttpOnly / Secure / Partitioned (CHIPS)、Cookie Store API、LocalStorage / SessionStorage 同步阻塞模型、storage 事件跨标签同步、容量与序列化陷阱。

  7. 07 · IndexedDB

    对象存储、索引、事务自动提交机制、版本升级的幂等迁移、游标遍历、idb/idb-keyval/Dexie 封装库、navigator.storage.estimate/persist、OPFS 对比。

  8. 08 · Service Worker + 跨标签通信

    SW 生命周期、Cache API、四种缓存策略（cache-first / network-first / SWR / network-only）、Workbox 路由、Background Sync、BroadcastChannel / SharedWorker / postMessage 对比。

阶段要点

  • 协议层：Fetch / XHR / WebSocket / SSE 各自适合的数据方向与能力边界。
  • 安全层：CORS 的 credentials 规则、CSP 的 strict-dynamic、SameSite / Partitioned Cookie 的 2024 新默认。
  • 存储层:Cookie 负责身份、LocalStorage 负责 UI 偏好、IndexedDB 负责结构化数据 / 大容量、Cache API 负责 HTTP 响应缓存。
  • 离线与更新：SW 拦截 fetch、版本化缓存、skipWaiting 的风险、BroadcastChannel 做登出广播。
