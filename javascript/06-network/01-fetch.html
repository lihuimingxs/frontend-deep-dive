Fetch API 完整语义

fetch 是浏览器/Node 现代网络请求的标准接口。它比 XHR 更 Promise-native、支持流式响应、更精细的 CORS 控制。但也有几个反直觉的"规范设计"—— HTTP 404 不会 reject、body 只能读一次、默认不发 cookie。本章把这些细节串起来。

阅读时长 ~25 分钟 前置 P3 Promise · P3 AbortController 规范 Fetch Standard

01 速览

javascript 
// 最小示例
const res = await fetch("/api/users");
if (!res.ok) throw new Error(`HTTP ${res.status}`);
const data = await res.json();

// 完整选项
const res = await fetch(url, {
  method: "POST",
  headers: { "Content-Type": "application/json", "X-Token": "..." },
  body: JSON.stringify({ name: "Alice" }),
  credentials: "include",      // "omit" / "same-origin" / "include"
  mode: "cors",                // "cors" / "no-cors" / "same-origin"
  cache: "no-store",           // "default" / "no-store" / "reload" / "force-cache" / ...
  redirect: "follow",          // "follow" / "error" / "manual"
  referrer: "no-referrer",     // 或 URL 字符串
  referrerPolicy: "strict-origin",
  integrity: "sha256-...",     // SRI
  signal: controller.signal,   // AbortSignal
  keepalive: true,             // 页面关闭后仍发送（sendBeacon 替代）
  priority: "high",            // "high" / "low" / "auto"（Fetch Priority）
});

Response 读 body 的方式

方法 返回类型
res.json() Promise<any>
res.text() Promise<string>
res.arrayBuffer() Promise<ArrayBuffer>
res.blob() Promise<Blob>
res.formData() Promise<FormData>
res.bytes() Promise<Uint8Array>（ES2025）
res.body ReadableStream（流式）

02 底层逻辑

fetch 的错误边界

fetch 只在网络层失败时 reject：

  • DNS 解析失败、连接中断、TLS 握手失败
  • CORS 预检失败（opaque response 或 blocked）
  • abort（AbortError）
  • 请求构造错误（非法 URL / headers）

HTTP 4xx / 5xx 不 reject —— 服务器响应了就算"成功"。必须手动 if (!res.ok) throw ...：

javascript 
async function safeFetch(url, options) {
  const res = await fetch(url, options);
  if (!res.ok) {
    const text = await res.text().catch(() => "");
    throw new Error(`HTTP ${res.status} ${res.statusText}: ${text}`);
  }
  return res;
}

// 和 axios 对比：axios 默认对 4xx/5xx reject（有 validateStatus 选项）
// fetch 保持"HTTP 是成功的语义"，4xx 也是服务器正确响应

Body 只能读一次

javascript 
const res = await fetch(url);
const text = await res.text();
const data = await res.json();          // ❌ TypeError: body already read

// 想读两次 → clone
const text = await res.clone().text();
const data = await res.json();

// 或根据 content-type 决定读法
const ct = res.headers.get("Content-Type") ?? "";
if (ct.includes("application/json")) return res.json();
else return res.text();

// 这是 fetch 对"流式 body"的一致设计 —— body 本质是 stream，读过就 consumed

流式读取响应

javascript 
// 用 ReadableStream 逐块读取（适合大文件 / 流式 AI 响应）
const res = await fetch("/large-data");
const reader = res.body.getReader();
const decoder = new TextDecoder();

let received = 0;
const contentLength = +res.headers.get("Content-Length");

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  received += value.length;
  const text = decoder.decode(value, { stream: true });
  process(text);
  updateProgress(received / contentLength);
}

// 或用 async iteration（ES2024）
for await (const chunk of res.body) {
  process(chunk);   // Uint8Array
}

Headers 对象

javascript 
const headers = new Headers({ "X-Token": "abc" });
headers.append("X-Token", "xyz");       // 多值
headers.set("Content-Type", "application/json");
headers.has("X-Token");                  // true
headers.get("X-Token");                  // "abc, xyz"
headers.delete("X-Token");
headers.getSetCookie();                  // ES2023，特殊处理 Set-Cookie（不合并）

// 迭代
for (const [key, value] of headers) { console.log(key, value); }

// 注意：某些 header 是 forbidden（不能从 JS 设置）
// User-Agent, Host, Origin, Cookie, ...

credentials 策略

值 带 cookie / Authorization 浏览器默认
"omit" 不带
"same-origin" 同源带 ✓（旧浏览器）
"include" 跨源也带 ✓（现代 Chrome）

注意：credentials: "include" 对跨源请求来说要求服务器响应 Access-Control-Allow-Credentials: true 且 Access-Control-Allow-Origin 不能是 *（必须具体域名）。详见 05 CORS。

cache 选项

值 含义
"default" 按 HTTP cache header 决定（最常用）
"no-store" 不读不写 HTTP cache
"reload" 强制向服务器验证（Ctrl+F5）
"no-cache" 读 cache 但总要服务器验证
"force-cache" 优先用 cache，没有才请求
"only-if-cached" 只读 cache，没有就失败（需要 mode: "same-origin"）

03 应用场景

带超时 + 取消的 fetch

javascript 
async function apiFetch(url, { timeout = 10_000, signal, ...options } = {}) {
  // 合并多个 signal：超时 + 外部取消
  const signals = [AbortSignal.timeout(timeout)];
  if (signal) signals.push(signal);
  const combined = AbortSignal.any(signals);

  const res = await fetch(url, { ...options, signal: combined });
  if (!res.ok) throw new HttpError(res.status, await res.text());
  return res.json();
}

// 用法
const ctrl = new AbortController();
button.addEventListener("click", () => ctrl.abort());
await apiFetch("/api/data", { signal: ctrl.signal });

SSE / OpenAI 流式输出

javascript 
// 逐 token 显示 AI 回答
const res = await fetch("/api/chat", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ prompt }),
});

const reader = res.body.getReader();
const decoder = new TextDecoder();
let buffer = "";

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });
  const lines = buffer.split("\n");
  buffer = lines.pop();     // 保留最后一行（可能不完整）
  for (const line of lines) {
    if (line.startsWith("data: ")) {
      const data = JSON.parse(line.slice(6));
      appendToken(data.token);
    }
  }
}

页面关闭时上报（keepalive）

javascript

navigator.sendBeacon("/analytics", JSON.stringify(data));

// fetch + keepalive 更灵活（支持 POST、有更多 header 控制）
fetch("/analytics", {

});

// keepalive 限制：body 最大 64KB；浏览器可能有请求数限制

文件下载带进度

javascript 
async function download(url, onProgress) {
  const res = await fetch(url);
  const total = +res.headers.get("Content-Length") || 0;
  const reader = res.body.getReader();
  const chunks = [];
  let received = 0;
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    chunks.push(value);
    received += value.length;
    if (total) onProgress(received / total);
  }
  return new Blob(chunks);
}

const blob = await download("/big-file.pdf", (p) => console.log(`${(p*100).toFixed(1)}%`));
const url = URL.createObjectURL(blob);
// <a href={url} download>save</a>

04 典型代码

生产级 fetch wrapper

javascript 
class HttpError extends Error {
  constructor(status, body, url) {
    super(`HTTP ${status} at ${url}`);
    this.name = "HttpError";
    this.status = status;
    this.body = body;
    this.url = url;
  }
}

async function api(url, { timeout = 10_000, retries = 0, ...options } = {}) {
  let lastErr;
  for (let i = 0; i <= retries; i++) {
    try {
      const res = await fetch(url, {
        ...options,
        signal: options.signal ?? AbortSignal.timeout(timeout),
      });
      if (res.status >= 500) throw new HttpError(res.status, await res.text(), url);
      if (!res.ok) throw new HttpError(res.status, await res.json().catch(() => ({})), url);
      const ct = res.headers.get("Content-Type") ?? "";
      return ct.includes("application/json") ? res.json() : res.text();
    } catch (err) {
      lastErr = err;
      if (err.name === "AbortError") throw err;
      if (err instanceof HttpError && err.status < 500) throw err;
      if (i < retries) await new Promise(r => setTimeout(r, 500 * 2 ** i));
    }
  }
  throw lastErr;
}

基于 AbortSignal 的超时

javascript 
// 原生 AbortSignal.timeout（ES2024）
fetch(url, { signal: AbortSignal.timeout(5000) });

// 组合超时 + 手动取消
const userCancel = new AbortController();
const signal = AbortSignal.any([AbortSignal.timeout(5000), userCancel.signal]);
fetch(url, { signal });

// 区分错误
try { await fetch(url, { signal: AbortSignal.timeout(5000) }); }
catch (err) {
  if (err.name === "TimeoutError") showError("请求超时");
  else if (err.name === "AbortError") {/* 用户取消 */}
  else throw err;
}

fetch + Zod 校验

javascript 
import { z } from "zod";

const UserSchema = z.object({ id: z.string(), name: z.string(), age: z.number() });
type User = z.infer<typeof UserSchema>;

async function getUser(id: string): Promise<User> {
  const res = await fetch(`/api/users/${id}`);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  const data = await res.json();
  return UserSchema.parse(data);        // 运行时校验 + 类型断言
}

05 常见陷阱

HTTP 404 不 reject

最常见的 fetch 错误认知。await fetch(url) 对 404 / 500 返回 response 对象。必须 if (!res.ok)。axios 默认 reject，从 axios 迁移到 fetch 要全量改。

body 只能读一次

res.json() 已经消费 body，再 res.text() 抛错。读两次必须 res.clone()。用 interceptor / logger 时特别容易踩。

跨源请求默认不带 cookie

fetch 默认 credentials: "same-origin"。跨源要 credentials: "include" + 服务器配合 CORS 头。升级 fetch 从老 XHR 代码要注意行为差异。

FormData 不要手动设 Content-Type

浏览器根据 FormData 自动生成 Content-Type: multipart/form-data; boundary=...。手动设会丢失 boundary → 服务端无法解析。

JSON 字符串的 body 要配 header

body: JSON.stringify(data) 要手动设 Content-Type: application/json，浏览器默认 text/plain。

keepalive 限制 64KB

keepalive: true 的 body 不能超过 64KB。超了 fetch 失败。大量埋点数据用 navigator.sendBeacon（同样限制）或分片上报。

no-cors 得到 opaque response

mode: "no-cors" 允许请求跨源资源但响应 type: "opaque" —— status 总是 0、body 不可读。几乎没有实用价值（除非纯副作用请求）。

Headers 某些 key 被禁用

User-Agent / Referer / Host / Connection / Content-Length 等 forbidden header。浏览器控制台会有警告但不抛错。想模拟这些需要 Service Worker 或 fetch 拦截代理。
