Promise 组合器

4 个静态方法对应 4 种"多 Promise 聚合"策略：全部成功 / 全部完成 / 最快敲定 / 任一成功。差异在"何时决议"和"失败如何处理"。本章给出决策表 + 典型用法。

阅读时长 ~15 分钟 前置 03 Promise 对应 javascript.info Promise API

01 速览

方法 何时 fulfilled 何时 rejected 结果
Promise.all(iter) 全部 fulfilled 任一 rejected（立即） 值数组 / 第一个 reason
Promise.allSettled(iter) 全部 敲定（无论成败） 永不 reject {status, value/reason} 数组
Promise.race(iter) 最快那个 fulfilled 最快那个 rejected 第一个敲定的 value / reason
Promise.any(iter) 任一 fulfilled（最快） 全部 rejected 第一个成功的 value / AggregateError
💡 选择决策树

我需要所有结果，任一失败就放弃 → all。
我需要所有结果，不管成败 → allSettled。
我需要最快的那个 → race。
只要有一个成功就行 → any。

02 底层逻辑

Promise.all：原子性

语义："全部成功 = 整体成功；任一失败 = 整体立即失败"。第一个 rejected 出现时立刻 reject 返回 Promise，其他 Promise 仍继续执行（只是结果被忽略）。

javascript 
const results = await Promise.all([
  fetch("/a"),
  fetch("/b"),
  fetch("/c"),
]);
// results = [ResponseA, ResponseB, ResponseC]
// 顺序和输入一致（不是按完成顺序）

// 任一 reject 立刻向上抛
try {
  await Promise.all([p1, p2, p3]);
} catch (err) {
  // err 是第一个 reject 的原因
  // p2 和 p3 如果之后还会 reject，会变成 unhandled rejection ⚠
}
⚠ 其他 Promise 的 rejection 可能变 unhandled

Promise.all 在一个 reject 时立即失败，但被"忽略"的其他 Promise 如果后续也 reject，就是未被处理的 rejection。生产代码如果不想让多个失败都上报，用 allSettled + 手动筛选。

Promise.allSettled：汇总所有结果

ES2020 加入。等全部敲定，返回永远 fulfilled 的 Promise，值是结果对象数组：

javascript 
const results = await Promise.allSettled([
  fetch("/a"),
  fetch("/b"),      // 假设失败
  fetch("/c"),
]);
// results = [
//   { status: "fulfilled", value: ResponseA },
//   { status: "rejected",  reason: ErrorB },
//   { status: "fulfilled", value: ResponseC },
// ]

const succeeded = results
  .filter(r => r.status === "fulfilled")
  .map(r => r.value);
const failed = results.filter(r => r.status === "rejected");

Promise.race：最快那个敲定

第一个敲定（fulfilled 或 rejected）的 Promise 决定整体结果。常用于超时：

javascript 
// 超时模式
const result = await Promise.race([
  fetch(url),
  new Promise((_, rej) => setTimeout(() => rej(new Error("timeout")), 5000)),
]);

// ⚠ 特殊情况：空数组永远 pending
Promise.race([]);   // Promise<pending> forever

// ⚠ 被"输掉"的 Promise 仍运行
// 超时模式并没有真正取消 fetch —— 用 AbortSignal（下一章）

Promise.any：第一个成功

ES2021 加入。任一fulfilled 就 resolve；全部 rejected 才 reject 为 AggregateError：

javascript 
// 多镜像择优
try {
  const res = await Promise.any([
    fetch(primary),
    fetch(mirror1),
    fetch(mirror2),
  ]);
  // 任一先返回就用它
} catch (err) {
  console.error("all failed:", err.errors);   // AggregateError
  // err.errors 是数组，每个对应失败 Promise 的 reason
}

// 空数组立即 reject AggregateError
Promise.any([]);    // → AggregateError("All promises were rejected")

所有组合器都"并行"

它们不管理并发——输入的 Promise 在数组构造时就已开始执行。组合器只等它们结果。Promise.all([fetch(a), fetch(b)]) 的两个 fetch 是立即同时启动的。要限制并发度，见 P3-07。

输入可以是任意 iterable

javascript 
// 数组
Promise.all([p1, p2, p3]);

// 任意 iterable
Promise.all(new Set([p1, p2]));
function* gen() { yield p1; yield p2; }
Promise.all(gen());

// 非 Promise 值会被 Promise.resolve 包装
Promise.all([1, 2, 3]);          // → fulfilled([1, 2, 3])
Promise.all([1, Promise.reject("e")]);   // → rejected("e")

03 应用场景

并行加载必需资源（all）

javascript 
// 页面渲染前必须拿到 user、profile、settings
const [user, profile, settings] = await Promise.all([
  fetchUser(id),
  fetchProfile(id),
  fetchSettings(id),
]);
render(user, profile, settings);

// 比串行快 3x（假设 3 个请求时间相近）

批量处理，单个失败不影响其他（allSettled）

javascript 
// 向 N 个下游推送消息，逐个记录失败
const results = await Promise.allSettled(
  targets.map(t => notify(t, message))
);
const failed = results
  .map((r, i) => r.status === "rejected" ? { target: targets[i], error: r.reason } : null)
  .filter(Boolean);
if (failed.length) logger.warn({ failed }, "some notifications failed");

// 仍然继续运行，不会因为一个 webhook 失败就整个任务 fail

第一个成功就用（any，多镜像加速）

javascript 
// CDN / API 镜像
const endpoints = [
  "https://api.primary.com/users",
  "https://api.mirror1.com/users",
  "https://api.mirror2.com/users",
];

try {
  const res = await Promise.any(endpoints.map(u => fetch(u)));
  // ✅ 最快返回的那个就是赢家
} catch (err) {
  // 只有全部失败才来这里
  showError("所有镜像不可用");
}

超时模式（race）

javascript 
function timeout(ms, err = new Error("timeout")) {
  return new Promise((_, rej) => setTimeout(() => rej(err), ms));
}
function withTimeout(p, ms) {
  return Promise.race([p, timeout(ms)]);
}

await withTimeout(fetch(url), 5000);
// ⚠ fetch 仍在后台；要真取消用 AbortSignal.timeout(5000)（下一章）

// ES2024：AbortSignal.timeout 原生
const res = await fetch(url, { signal: AbortSignal.timeout(5000) });

聚合多个异步依赖（allSettled）

javascript 
// Dashboard 页：多个卡片独立加载，一个失败显示局部错误
const [userResult, statsResult, recentResult] = await Promise.allSettled([
  fetchUser(),
  fetchStats(),
  fetchRecent(),
]);

return {
  user:   userResult.status === "fulfilled" ? userResult.value : null,
  stats:  statsResult.status === "fulfilled" ? statsResult.value : null,
  recent: recentResult.status === "fulfilled" ? recentResult.value : null,
  errors: [userResult, statsResult, recentResult]
    .filter(r => r.status === "rejected")
    .map(r => r.reason),
};
// 模板里根据每个字段是否 null 决定显示数据还是错误卡片

04 典型代码

allSettled 结果 partition

javascript 
function partition(results) {
  const succeeded = [];
  const failed = [];
  for (const r of results) {
    if (r.status === "fulfilled") succeeded.push(r.value);
    else failed.push(r.reason);
  }
  return { succeeded, failed };
}

const { succeeded, failed } = partition(
  await Promise.allSettled(items.map(fetch))
);

带失败计数的批量

javascript 
async function batch(tasks, { allowFailures = 0 } = {}) {
  const results = await Promise.allSettled(tasks);
  const failed = results.filter(r => r.status === "rejected");
  if (failed.length > allowFailures) {
    throw new AggregateError(failed.map(f => f.reason),
      `${failed.length} tasks failed (max ${allowFailures})`);
  }
  return results.filter(r => r.status === "fulfilled").map(r => r.value);
}

// 允许 10% 失败
await batch(tasks, { allowFailures: Math.ceil(tasks.length * 0.1) });

快速竞选 + 超时

javascript 
// 多机构择一（有超时）
async function fastest(urls, timeoutMs = 5000) {
  const controller = new AbortController();
  try {
    return await Promise.any([
      ...urls.map(u => fetch(u, { signal: controller.signal })),
      new Promise((_, rej) => setTimeout(() => rej(new Error("all timed out")), timeoutMs)),
    ]);
  } finally {
    controller.abort();   // 取消剩下的 fetch
  }
}

输入扁平化

javascript 
// 混合 sync 值和 Promise
const values = await Promise.all([
  42,                        // sync → Promise.resolve(42)
  fetchData(),               // Promise
  computeSync(),             // sync function 返回值
  Promise.resolve("hi"),     // 已 resolved Promise
]);
// values = [42, data, computed, "hi"]

// 所有非 Promise 值被立即包装

05 常见陷阱

Promise.all 失败时其他 Promise 继续运行

数组里一个 Promise reject，Promise.all 立即 reject，但其他 Promise 的 rejection 可能成为 unhandled rejection。重要场景用 allSettled 或结合 AbortController 主动取消其他。

Promise.race 空数组永远 pending

Promise.race([]) 永远 pending（没有东西可敲定）。Promise.any([]) 直接 reject AggregateError。Promise.all([]) 立即 fulfill []。Promise.allSettled([]) 立即 fulfill []。知道这些边界行为。

组合器不控制并发度

Promise.all(urls.map(fetch)) 会同时发起全部请求。1000 个 URL 就是 1000 并发，浏览器限流、服务器压力、CPU 占满。限并发用 p-limit / 信号量（P3-07）。

Promise.race 的"输家"不会被取消

race 决议只是让返回的 Promise 敲定，其他 Promise 仍继续执行。超时场景要真取消（省资源）用 AbortSignal.timeout 或 AbortController。

Promise.any 结果是 AggregateError

全失败时 reject 的是 AggregateError（不是单个 reason）。err.errors 才是各个 reason 数组。err.message 是 "All promises were rejected"。工程代码里要展开看 errors。

混用 sync throw 和 Promise 组合器

Promise.all([fn1(), fn2()]) 如果 fn1 同步 throw（不返回 Promise），错误会逃出 Promise.all 直接抛到外层。一般用箭头函数包装：Promise.all([() => fn1(), () => fn2()].map(f => Promise.resolve().then(f)))。

allSettled 结果的类型判别忘记 status

TypeScript 里 result.value 只在 status === "fulfilled" 时才能访问。直接 result.value 会是 value | undefined。用 discriminated union type guard：if (result.status === "fulfilled") { result.value }。

非数组顺序相关的陷阱

Promise.all 返回的数组顺序严格对应输入顺序（不是完成顺序）。看起来 A 和 B 同时 fulfilled，但 A 的位置在 B 前面。不要根据"数组顺序"推测"完成时间"。
