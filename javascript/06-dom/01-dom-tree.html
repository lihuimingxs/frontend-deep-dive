DOM 树与 Node 接口

DOM 是浏览器把 HTML 解析成的对象树。每个节点都继承自 EventTarget → Node → (Element / CharacterData / Document / DocumentFragment)。这个继承链决定了"哪个 API 在哪个对象上可用"—— 不把层次理清，textContent 和 innerText、children 和 childNodes 的差异就永远记不住。

阅读时长 ~18 分钟 前置 P2 原型链 对应 javascript.info Walking the DOM

01 速览

接口继承层次

text 
EventTarget                    ← addEventListener / dispatchEvent 祖先
 └─ Node                       ← 通用 tree API（parentNode / childNodes / nodeType）
     ├─ Element                ← getAttribute / classList / innerHTML / closest
     │   ├─ HTMLElement        ← style / dataset / click()
     │   │   ├─ HTMLInputElement  ← value / checked
     │   │   ├─ HTMLFormElement   ← elements / submit()
     │   │   └─ ...
     │   └─ SVGElement
     ├─ Document               ← createElement / querySelector (全局)
     ├─ DocumentFragment       ← 轻量子树容器（批量 DOM 操作）
     ├─ CharacterData          ← 文本节点基类
     │   ├─ Text               ← 文本节点
     │   ├─ Comment            ← 注释节点
     │   └─ CDATASection
     ├─ DocumentType           ← <!DOCTYPE>
     └─ Attr (ES5 前)          ← 现在 Attr 不是 Node，但仍存在

nodeType 速查

值 常量 节点类型
1 Node.ELEMENT_NODE Element
3 Node.TEXT_NODE Text
8 Node.COMMENT_NODE Comment
9 Node.DOCUMENT_NODE Document
11 Node.DOCUMENT_FRAGMENT_NODE DocumentFragment

02 底层逻辑

Node 遍历 vs Element 遍历

两套并行的属性。Node 版本包含 文本节点和注释：

Node 视角 Element-only 视角
childNodes children
firstChild firstElementChild
lastChild lastElementChild
previousSibling previousElementSibling
nextSibling nextElementSibling
parentNode parentElement
childElementCount （同一个）
javascript 
// <div id="x">Hello <b>World</b>!</div>
const div = document.getElementById("x");

div.childNodes.length;              // 3（"Hello ", <b>, "!"）
div.children.length;                // 1（只有 <b>）
div.firstChild.nodeType;            // 3（TEXT_NODE）
div.firstElementChild;              // <b>World</b>

// childNodes 返回的是 NodeList
// children 返回的是 HTMLCollection
// 两者都是 live collection（下一章详细讲）

attribute vs property

一个常见误解："el.id 和 el.getAttribute('id') 是一回事" —— 不完全是：

  • Attribute：HTML 里写的字符串。
  • Property：DOM 对象上的 JS 属性，值可以是字符串、boolean、对象、函数 handler 等。
  • 初始时 property 从 attribute 同步而来；后续修改 property 不一定反映到 attribute。
javascript 
// <input id="x" type="checkbox" checked>
const cb = document.getElementById("x");

cb.checked;                          // true
cb.getAttribute("checked");          // ""（attribute 存在即字符串）

cb.checked = false;                  // 运行时修改 property
cb.checked;                          // false
cb.getAttribute("checked");          // ""  ← attribute 不跟着变！
cb.hasAttribute("checked");          // true

cb.defaultChecked;                   // true（form reset 时的值）
cb.setAttribute("checked", "");      // 改 attribute
cb.defaultChecked;                   // true

// 规则：
// - checked/value/disabled 等 boolean/值 property：运行时状态，改 property不影响 attribute
// - id/className/href 等字符串：同步（双向）
// - class/className：特殊（className 是 property，class 是 attribute，互相同步）

dataset 和 data-*

data-* 属性被映射到 element.dataset 对象上，烤串式变驼峰：

javascript 
// <div data-user-id="42" data-is-admin="true">
el.dataset.userId;            // "42"  （全字符串）
el.dataset.isAdmin;           // "true"（不是 boolean！）

// 赋值
el.dataset.theme = "dark";
// → 相当于 setAttribute("data-theme", "dark")

// 删除
delete el.dataset.theme;
// → removeAttribute("data-theme")

// ⚠ dataset 值永远是字符串。需要 number / boolean 要手动转

textContent / innerText / innerHTML

属性 读取 写入 触发 reflow
textContent 所有后代文本（含 hidden） 清空子树写文本（不解析 HTML） 否
innerText 渲染可见的文本（遵守 display:none） 同 textContent，会规范化空白 是（必须计算布局）
innerHTML 序列化子树为 HTML 解析 HTML并替换子树 是

性能敏感路径用 textContent（不读布局，不解析）。安全性用 textContent（不受 XSS 威胁）。详见 03 章。

Document / DocumentFragment 的特殊性

  • Document：唯一全局单例，document 全局变量。有 createElement、createTextNode、cookie 等独家 API。
  • DocumentFragment：轻量容器，不渲染在页面上。把子树 append 到它，然后一次性 append 到 DOM —— 只触发一次 reflow，批量插入的利器。
  • ShadowRoot：继承 DocumentFragment，是 Shadow DOM 的根（P7 讲）。

03 应用场景

DocumentFragment 批量插入

javascript 
// ❌ 每次 append 都触发 reflow（1000 个 li → 1000 次）
for (const item of items) {
  const li = document.createElement("li");
  li.textContent = item.name;
  ul.append(li);
}

// ✓ Fragment 在内存里组装，一次性插入
const frag = document.createDocumentFragment();
for (const item of items) {
  const li = document.createElement("li");
  li.textContent = item.name;
  frag.append(li);
}
ul.append(frag);   // 只一次 reflow

// ✓ 模板字符串 + 一次 innerHTML（最快，但有 XSS 风险）
ul.innerHTML = items.map(i => `<li>${escape(i.name)}</li>`).join("");

遍历过滤特定类型节点

javascript 
// 统计文本节点里某关键字出现次数
function countInText(root, keyword) {
  let count = 0;
  const walker = document.createTreeWalker(root, NodeFilter.SHOW_TEXT);
  while (walker.nextNode()) {
    const text = walker.currentNode.textContent;
    count += (text.match(new RegExp(keyword, "g")) ?? []).length;
  }
  return count;
}

// TreeWalker / NodeIterator 比手动递归 childNodes 更干净，
// 自带过滤器（SHOW_ELEMENT / SHOW_TEXT / SHOW_COMMENT 等）

区分"用户设置"和"默认"

javascript 
// <input type="email" value="x@y.com" required>
const input = form.querySelector("input");

input.defaultValue;       // "x@y.com"（HTML 里的 value）
input.value;              // 用户当前输入
input.value = "new";
input.defaultValue;       // 仍是 "x@y.com"

input.defaultChecked;     // 初始 checked
input.defaultSelected;    // option 元素的初始 selected

form.reset();             // 所有字段回到 defaultXxx

用 dataset 做组件状态标记

javascript 
// HTML
// <button data-state="closed">Toggle</button>

// CSS
// button[data-state="open"] { background: blue; }
// button[data-state="closed"] { background: gray; }

// JS
const btn = document.querySelector("button");
btn.addEventListener("click", () => {
  btn.dataset.state = btn.dataset.state === "open" ? "closed" : "open";
  // CSS 自动响应状态变化
});

// 避免操作 className 字符串，dataset 语义更清晰

SSR / 服务端 DOM 渲染

javascript 
// Node 端用 jsdom / happy-dom 模拟 Document
import { JSDOM } from "jsdom";
const { document } = new JSDOM("<div id='root'></div>").window;

const div = document.createElement("div");
div.textContent = "Hello";
document.getElementById("root").append(div);
document.body.innerHTML;
// Next.js / Remix 的 SSR 类似，但用更快的 "流式字符串化"
// 而不是真 DOM —— 性能更好但 API 兼容性有限

04 典型代码

遍历 DOM 子树

javascript 
// 递归遍历所有 Element（忽略文本/注释）
function walk(root, fn) {
  fn(root);
  for (const child of root.children) walk(child, fn);
}

// 遍历所有节点（含文本）
function walkAll(root, fn) {
  fn(root);
  for (const child of root.childNodes) walkAll(child, fn);
}

// 推荐：TreeWalker
function walkWithWalker(root, type = NodeFilter.SHOW_ELEMENT) {
  const walker = document.createTreeWalker(root, type);
  let node;
  while ((node = walker.nextNode())) yield node;
}

完整操作 attribute

javascript 
// 基础
el.getAttribute("role");              // string | null
el.setAttribute("role", "button");
el.removeAttribute("role");
el.hasAttribute("role");              // boolean

// 批量遍历
for (const attr of el.attributes) {
  console.log(attr.name, attr.value);
}

// toggle（ES2018+）
el.toggleAttribute("disabled");       // 切换存在性
el.toggleAttribute("disabled", true); // 强制添加

// 对应 property（若有）
el.role = "button";                   // ARIA role 也支持 property 访问
el.inert = true;                      // inert attribute

classList 完整 API

javascript 
el.classList.add("active", "visible");
el.classList.remove("hidden");
el.classList.toggle("open");           // 切换
el.classList.toggle("disabled", cond); // 按条件
el.classList.contains("active");       // boolean
el.classList.replace("old", "new");

// 数组化
[...el.classList];                     // ["active", "visible"]

// ⚠ classList 是 live DOMTokenList，不是数组
// className 是纯字符串 "active visible"（操作字符串风险大）

判断节点类型

javascript 
// nodeType 常量
node.nodeType === Node.ELEMENT_NODE;          // 1
node.nodeType === Node.TEXT_NODE;             // 3

// instanceof（更语义化）
node instanceof HTMLElement;
node instanceof HTMLInputElement;
node instanceof Text;

// 判断标签
el.tagName;                           // "DIV"（大写！）
el.localName;                         // "div"（小写，SVG 等用这个）
el.matches("input[type='checkbox']"); // 匹配选择器

05 常见陷阱

childNodes 包含空白文本节点

<ul>\n <li>...</li>\n</ul> 的 childNodes 不止包含 li —— 还有换行/空格文本节点。想"只要元素"用 children。

tagName 总是大写

el.tagName === "DIV"（不是 "div"）—— HTML 规范强制。SVG 里 tagName 是原样大小写。判断用 el.tagName.toLowerCase() 或用 localName。

boolean attribute 的 setAttribute 陷阱

setAttribute("disabled", false) 仍然 disable！因为 attribute 只看"是否存在"，值 "false" 字符串也是 truthy。必须 removeAttribute("disabled")。优先用 property：el.disabled = false。

innerText 触发 reflow

读 el.innerText 要强制计算布局（因为要按渲染决定可见性）。热路径用 textContent。innerHTML 也读得不便宜（要序列化）。

dataset 所有值都是字符串

el.dataset.count = 5 实际存的是 "5"。读回 Number(el.dataset.count)。boolean 也需 el.dataset.active === "true"（字符串比较）。

Attribute 选择器区分大小写

querySelector("[data-Id]") —— 现代浏览器对 HTML 不区分大小写（data-Id = data-id），但 SVG / XML 命名空间下区分。用 kebab-case 写 data- 属性名最稳。

property 和 attribute 命名不一致

HTML class → property className；HTML for → property htmlFor；HTML tabindex → property tabIndex。原因是保留字冲突。记住这几个。

parentNode 和 parentElement 差异

document.parentNode === null；document.documentElement.parentNode === document（Document 类型）；document.documentElement.parentElement === null（不是 Element）。跨 <html> 边界的话用 parentNode。
