Form 控件与验证

表单是 Web 最古老也最复杂的一块。input 的 value / checked 在 property 和 attribute 之间的差异（见 01 章）、React 的"受控 vs 非受控"、原生的 Constraint Validation API、ES2023 才标准化的 form-associated custom elements —— 这些组合起来决定了"表单 bug 为什么那么多"。本章系统梳理。

阅读时长 ~22 分钟 前置 01 DOM · 04 事件 对应 javascript.info Forms and controls

01 速览

表单控件分类

元素 value 类型 触发事件
<input type="text/email/url/number"> 字符串（number 仍是 string） input / change
<input type="checkbox"> checked（bool） change
<input type="radio"> checked + value change
<input type="file"> files（FileList） change
<input type="date/time/datetime-local"> ISO 字符串 / valueAsDate input / change
<select> value（单选）/ selectedOptions（多选） change
<textarea> 字符串 input / change

02 底层逻辑

input 和 change 事件

这是两个独立事件，触发时机差异：

input change
触发时机 值变化每一次 值变化且失焦或回车/勾选时
text input 每敲一个字符 blur 时（如果值变了）
checkbox / radio 每次勾选（新） 每次勾选（老）
select 选项变化（新） 选项变化
file input 不触发 选文件后

工程规则：想要"每次输入都响应"用 input；想要"用户确认后"用 change。React 的 onChange 实际对应原生 input（React 历史设计，名字反直觉）。

受控 vs 非受控

javascript 
// 非受控：让 DOM 保存状态
<input id="name" defaultValue="Alice" />
// 提交时取 document.getElementById("name").value
// 优点：最少代码、原生性能
// 缺点：值在 DOM 里，React 等框架想"监控"要 ref / onChange

// 受控：JS 保存状态，DOM 只是视图
const [name, setName] = useState("Alice");
<input value={name} onChange={e => setName(e.target.value)} />
// 优点：值在一个地方，易验证 / 联动 / 本地持久化
// 缺点：性能（每次输入都 setState 重渲染）

FormData：直接从表单提交

javascript 
// 从表单元素建 FormData
const form = document.querySelector("form");
const data = new FormData(form);

// 遍历
for (const [key, value] of data) console.log(key, value);

// 转对象（单值字段）
const obj = Object.fromEntries(data);

// 多值字段（checkbox group）
data.getAll("colors");             // ["red", "blue"]

// 支持文件
data.get("avatar") instanceof File;

// 提交
fetch("/api", { method: "POST", body: data });
// ↑ 自动 multipart/form-data，无需手动 Content-Type

// 用 URLSearchParams 或 JSON
fetch("/api", {
  method: "POST",
  headers: { "Content-Type": "application/x-www-form-urlencoded" },
  body: new URLSearchParams(data),
});

fetch("/api", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(Object.fromEntries(data)),
});

Constraint Validation API

HTML 原生验证：required / pattern / min / max / minlength / maxlength 等。JS 接口：

javascript 
input.checkValidity();           // boolean，不显示 UI
input.reportValidity();          // boolean，不通过时显示内置 tooltip
input.validity;                  // ValidityState 对象
// {
//   valid, valueMissing, typeMismatch, patternMismatch,
//   tooLong, tooShort, rangeUnderflow, rangeOverflow,
//   stepMismatch, badInput, customError
// }

input.validationMessage;         // 当前错误的 locale 化文本

// 设置自定义错误
input.setCustomValidity("用户名必须大于 5 字");
// 清空
input.setCustomValidity("");

// form 级
form.checkValidity();            // 所有字段
form.reportValidity();           // 显示第一个错误

// 监听
input.addEventListener("invalid", (e) => {
  e.preventDefault();            // 阻止浏览器原生 tooltip
  showMyErrorUI(input.validationMessage);
});

HTMLFormElement.elements

javascript 
const form = document.querySelector("form");

// 按名字访问（HTMLFormControlsCollection）
form.elements.username;          // 第一个 name="username" 的控件
form.elements["colors"];         // RadioNodeList（多个同名）

// 转数组
[...form.elements];              // 所有表单控件（Element）

// 提交
form.submit();                   // 跳过 validation，直接提交
form.requestSubmit();            // 触发 validation + submit 事件 + 提交
form.requestSubmit(submitterBtn);// 指定"点击"的按钮

// reset
form.reset();                    // 所有字段回到 default*

form-associated custom elements（ES2023）

自定义元素 extends HTMLElement 默认不是"表单控件"——不参与 FormData、不提交、不支持 validation。ElementInternals API 让它们可以像原生 input 一样工作：

javascript 
class MyInput extends HTMLElement {
  static formAssociated = true;             // ← 启用

  #internals;
  constructor() {
    super();
    this.#internals = this.attachInternals();
  }

  set value(v) {
    this._value = v;
    this.#internals.setFormValue(v);        // ← 提交时就包含这个
  }

  formAssociatedCallback(form) { /* 加入表单 */ }
  formDisabledCallback(disabled) { /* 表单 disable 时 */ }
  formResetCallback() { this.value = ""; }  // form.reset 时
  formStateRestoreCallback(state, mode) { /* 恢复状态 */ }

  // 参与验证
  reportValidity() { return this.#internals.reportValidity(); }
}
customElements.define("my-input", MyInput);

<form>
  <my-input name="foo"></my-input>   // ← 和 input 一样参与 FormData
</form>

03 应用场景

原生提交拦截 + fetch

javascript 
form.addEventListener("submit", async (e) => {
  e.preventDefault();
  if (!form.reportValidity()) return;        // 显示原生错误 UI

  const data = Object.fromEntries(new FormData(form));
  try {
    const res = await fetch(form.action, {
      method: form.method,
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(data),
    });
    if (res.ok) showSuccess();
    else showError(await res.text());
  } catch (err) {
    showError("network error");
  }
});

复选框组 + 自定义验证

javascript 
// <input type="checkbox" name="colors" value="red"> Red
// <input type="checkbox" name="colors" value="blue"> Blue
// ...

// 必须至少选一个
function validateColors() {
  const selected = form.querySelectorAll("input[name=colors]:checked").length;
  const firstCb = form.querySelector("input[name=colors]");
  firstCb.setCustomValidity(selected ? "" : "请至少选一个颜色");
}

form.querySelectorAll("input[name=colors]").forEach(cb =>
  cb.addEventListener("change", validateColors)
);

受控组件带 debounce

javascript 
// 受控但不每次都 rerender
import { useState, useRef } from "react";
function SearchInput({ onSearch }) {
  const [v, setV] = useState("");
  const timer = useRef();
  return <input value={v} onChange={e => {
    setV(e.target.value);
    clearTimeout(timer.current);
    timer.current = setTimeout(() => onSearch(e.target.value), 300);
  }} />;
}

// 或用 useDeferredValue / useTransition（React 18+）

自定义元素作表单控件

javascript 
// 场景：想让 <rating-input>（自定义星级评分）能被 new FormData 收到
class RatingInput extends HTMLElement {
  static formAssociated = true;
  #internals;
  #value = 0;

  constructor() {
    super();
    this.#internals = this.attachInternals();
  }

  get value() { return this.#value; }
  set value(v) {
    this.#value = Number(v);
    this.#internals.setFormValue(String(this.#value));
  }

  // 支持 name / form / disabled / validity
  formAssociatedCallback() {}
  formResetCallback() { this.value = 0; }

  // 初始挂载
  connectedCallback() {
    this.addEventListener("click", (e) => {
      const star = e.target.closest(".star");
      if (star) this.value = Number(star.dataset.rating);
    });
  }
}
customElements.define("rating-input", RatingInput);

<form>
  <rating-input name="rating"></rating-input>
  <button>Submit</button>
</form>
// FormData 能拿到 rating=5

04 典型代码

表单转 JSON

javascript 
function formToJSON(form) {
  const data = new FormData(form);
  const obj = {};
  for (const [key, value] of data.entries()) {
    // 处理多值（同名 name 多次出现）
    if (obj[key] !== undefined) {
      if (!Array.isArray(obj[key])) obj[key] = [obj[key]];
      obj[key].push(value);
    } else {
      obj[key] = value;
    }
  }
  return obj;
}

// 数字字段手动转（FormData 值都是 string 或 File）
const data = formToJSON(form);
if (data.age) data.age = Number(data.age);

完整自定义验证

javascript 
// 每个 input 的 invalid 事件显示自己的错误
form.querySelectorAll("input, select, textarea").forEach((el) => {
  el.addEventListener("invalid", (e) => {
    e.preventDefault();
    const errBox = el.nextElementSibling;
    if (errBox?.classList.contains("error")) {
      errBox.textContent = el.validationMessage;
    }
  });
  // 用户修改后立即清
  el.addEventListener("input", () => {
    el.setCustomValidity("");
    const errBox = el.nextElementSibling;
    if (errBox?.classList.contains("error")) errBox.textContent = "";
  });
});

// submit 时触发全部 invalid
form.addEventListener("submit", (e) => {
  if (!form.checkValidity()) {
    e.preventDefault();
    form.reportValidity();
  }
});

非受控 + defaultValue 模式

javascript 
// React
function Form() {
  const formRef = useRef();
  function submit(e) {
    e.preventDefault();
    const data = Object.fromEntries(new FormData(formRef.current));
    save(data);
  }
  return (
    <form ref={formRef} onSubmit={submit}>
      <input name="name" defaultValue="Alice" />
      <input name="email" defaultValue="a@b" />
      <button>Save</button>
    </form>
  );
}

// 好处：无 state、无重渲染；劣势：无法做即时联动
// 适合一次性提交的简单表单（如 Settings 页）

05 常见陷阱

input[type=number] 的 value 仍是字符串

input.value 类型永远是 string。input.valueAsNumber 返回 number。用户输入非法内容 → value 是空 string、valueAsNumber 是 NaN。

checkbox 的 value 属性不是勾选状态

cb.value 是 attribute 里的 value="..."（默认 "on"）。cb.checked 才是是否勾选。常见混淆源。

React onChange 不是原生 change

React 的 onChange 对应原生 input 事件（每次输入都触发）。原生 change 仅失焦时触发。写 React 时要意识到这个差异。

form.submit() 跳过验证

直接调 submit() 不触发 submit 事件、不验证、不显示错误。用 requestSubmit()（规范化的）模拟用户点击提交。

input[type=date] 在 Safari 历史上不支持

现在 Safari 14.1+ 已支持。老版本 / iOS 上日期选择体验差异大。复杂日期 UX 建议用第三方库。input.valueAsDate 拿 Date 对象。

FormData 不包含 disabled 字段

<input disabled> 不参与 FormData、不提交。想"只读但提交"用 readonly。

autocomplete 的多种取值

autocomplete="off" 浏览器不总是听。new-password 明确告诉浏览器"这是新密码不要填上个"。off 对密码字段常被忽略。具体值见 HTML 规范的 autofill tokens。

受控 input 的光标跳到末尾

React 受控 input 格式化值时（如金额加千分位），每次 setState 光标会跳到末尾。解决：手动 input.setSelectionRange(pos, pos) 保持位置，或用"格式化但不修改 DOM 值"的 displayValue 模式。
