DOM 查询与遍历

现代 DOM 查询有两大家族：旧 API（getElementById / ByClassName / ByTagName）和 选择器 API（querySelector / All）。返回值的live vs static特性决定了性能与陷阱。本章讲每个 API 的判定、NodeList 的迭代特性、Range/TreeWalker 的进阶用法。

阅读时长 ~18 分钟 前置 01 DOM 树 对应 javascript.info Searching: getElement*, querySelector*

01 速览

方法 返回 特性 支持
getElementById(id) Element / null 只在 document；全局唯一 id 最快
getElementsByClassName(cls) HTMLCollection live 所有 Element
getElementsByTagName(tag) HTMLCollection live 所有 Element
getElementsByName(name) NodeList live；仅 document form 控件常用
querySelector(selector) Element / null CSS 选择器 所有 Element
querySelectorAll(selector) NodeList static（快照） 所有 Element
closest(selector) Element / null 向上找祖先（含自身） 所有 Element
matches(selector) boolean 是否匹配选择器 所有 Element

02 底层逻辑

Live vs Static 集合

Live collection：反映 DOM 实时状态，访问时才查。HTMLCollection（大多旧 API 返回）是 live。
Static：调用时的快照，后续 DOM 变化不反映。querySelectorAll 返回的 NodeList 是 static。

javascript 
const liveList = document.getElementsByClassName("item");
const staticList = document.querySelectorAll(".item");

console.log(liveList.length);    // 3
console.log(staticList.length);  // 3

// 添加一个 .item
document.body.append(Object.assign(document.createElement("div"), { className: "item" }));

liveList.length;                 // 4 ← 自动更新
staticList.length;               // 3 ← 快照不变

陷阱：for 循环里修改 live 集合（getElementsByTagName("li") + appendChild(li)）会死循环。解决方法：转成数组 [...coll] 或用 querySelectorAll。

NodeList vs HTMLCollection

NodeList HTMLCollection
内容 任意 Node（含 Text） 仅 Element
live 行为 querySelectorAll 返回的是 static；childNodes 是 live 始终 live
按索引 list[i] / list.item(i) 同上 + list.namedItem("id")
forEach ✓ ✗（需要 [...coll].forEach）
for-of ✓（ES2015+） ✓

CSS 选择器性能

浏览器 从右向左匹配选择器：

javascript 
document.querySelectorAll("div.container > ul > li.active");
// 浏览器：
// 1. 找所有 .active
// 2. 对每个 .active 往上查：parent 是 ul？parent 的 parent 是 .container？
// 3. 满足所有条件的才收入

// 性能 tips：
// - 右端选择器越具体越快（.active > [id=x] 差别大）
// - 避免通用选择器 * 和深选择器 a b c d
// - ID 查找是 hash 表，最快

// 简单基准：
// getElementById         O(1)
// getElementsBy*         O(1) ~ O(n)（Live 集合缓存）
// querySelector(#x)      接近 getElementById
// querySelector(复杂)    需要遍历

closest 与 matches

javascript 
// closest：从当前元素向上（含自身）找最近匹配 selector 的祖先
el.closest("form");                      // 找所在的表单
e.target.closest("[data-action]");       // 事件委托核心（见 05 章）

// matches：当前元素是否匹配 selector
el.matches(".active");                   // boolean
// 等价旧 API: el.webkitMatchesSelector（已废弃）

// 组合：查找最近的 "某种" 祖先
function findAncestor(el, predicate) {
  while (el && el !== document) {
    if (predicate(el)) return el;
    el = el.parentElement;
  }
  return null;
}
// 现代替代：el.closest(selectorThatEncodesPredicate)

Range 与 TreeWalker

精细的文本/节点选择 API：

javascript 
// Range：选中文档的一段（选区）
const range = document.createRange();
range.setStart(el.firstChild, 2);        // 从第几个字符开始
range.setEnd(el.firstChild, 8);
range.extractContents();                  // 删除并返回 DocumentFragment
range.getBoundingClientRect();            // 选区的视口坐标

// Selection：用户选中的文本
window.getSelection().toString();
window.getSelection().getRangeAt(0);

// TreeWalker：带过滤器的遍历器
const walker = document.createTreeWalker(
  root,
  NodeFilter.SHOW_TEXT,                    // 只看文本节点
  { acceptNode(node) {
      return node.textContent.trim()
        ? NodeFilter.FILTER_ACCEPT
        : NodeFilter.FILTER_REJECT;
  }},
);
while (walker.nextNode()) { /* ... */ }

:scope 伪类

javascript 
// querySelector 默认以 document 为匹配上下文 —— 即使你调用 el.querySelector
el.querySelectorAll("> li");          // ❌ 错误：> 必须有左侧元素
el.querySelectorAll(":scope > li");   // ✓ :scope 指代 el

// 用于"我的直接子元素"等场景
const directChildren = el.querySelectorAll(":scope > [data-item]");

// 注意 :scope 在整个 document.querySelector 里相当于 :root

03 应用场景

事件委托用 closest

javascript 
list.addEventListener("click", (e) => {
  const btn = e.target.closest("[data-action]");
  if (!btn || !list.contains(btn)) return;
  // 确保点击的是我们关心的 action 按钮
  handle(btn.dataset.action);
});
// 即使点击在 button 内的 <svg> 上，closest 也能找到 action 按钮

live 集合避免轮询

javascript 
// 动态列表，只拿 live HTMLCollection 引用一次
const items = document.getElementsByClassName("item");
// 插入新 item 后不需要重新查
setTimeout(() => {
  for (let i = 0; i < items.length; i++) {  // 总是拿到最新数量
    items[i].classList.add("loaded");
  }
}, 100);

// ⚠ 如果 "加 class" 不会让元素脱离 .item，就没问题
// 但如果 for 循环内 removeChild —— 死循环危险

全局快照 + 批量操作

javascript 
// querySelectorAll 返回 static，方便批处理而不担心 live 变化
const allLinks = document.querySelectorAll("a[href^='http']");
allLinks.forEach(link => {
  link.target = "_blank";
  link.rel = "noopener noreferrer";
});

// 或数组方法
[...document.querySelectorAll(".card")].filter(c => c.offsetTop < 500);

跨 shadow DOM 查询

javascript 
// querySelector不能穿透 shadow root
const shadowHost = document.querySelector("my-widget");
shadowHost.querySelector("button");           // null（内部在 shadow 里）

// 正确姿势
shadowHost.shadowRoot.querySelector("button"); // ✓

// ⚠ closed 模式的 shadow root 外部访问不到
// Web Components 封装的强度由 mode 决定（详见 P7）

性能敏感：缓存查询结果

javascript 
// ❌ 每次 handler 都查一次
button.addEventListener("click", () => {
  document.querySelector(".modal").classList.remove("hidden");
});

// ✓ 查一次用多次
const modal = document.querySelector(".modal");
button.addEventListener("click", () => modal.classList.remove("hidden"));

// DOM 查询即使是 querySelector 也不是零成本（~10-50μs）
// 紧密循环 / 高频事件 handler 里要缓存

04 典型代码

作用域查询工具

javascript 
// 封装常见查询
const $ = (sel, root = document) => root.querySelector(sel);
const $$ = (sel, root = document) => [...root.querySelectorAll(sel)];

// 用法
const form = $("form");
$$("input", form).forEach(i => i.disabled = true);

// TS 带类型
function $<K extends keyof HTMLElementTagNameMap>(tag: K): HTMLElementTagNameMap[K] | null;
function $<E extends Element = Element>(sel: string, root?: ParentNode): E | null;

向上找祖先

javascript 
// 1. closest 直接满足
el.closest("form");
el.closest("[data-component]");

// 2. 定制条件用 parentElement 循环
function ancestorsWhile(el, predicate) {
  const out = [];
  while (el && predicate(el)) {
    out.push(el);
    el = el.parentElement;
  }
  return out;
}

// 3. 找所有祖先到 root
function ancestors(el) {
  const out = [];
  while (el && el !== document.body) { out.push(el); el = el.parentElement; }
  return out;
}

兄弟筛选

javascript 
// 当前元素的所有兄弟（除自己）
function siblings(el) {
  return [...el.parentElement.children].filter(c => c !== el);
}

// 前面的兄弟
function prevSiblings(el) {
  const out = [];
  while ((el = el.previousElementSibling)) out.push(el);
  return out.reverse();
}

// 配合 CSS 选择器
el.parentElement.querySelectorAll(":scope > :not(.active)");

Range 实现高亮

javascript 
// 用 CSS Custom Highlight API（ES2024 / CSS Highlight）
if (CSS.highlights) {
  const ranges = [];
  // 在文档里找 keyword 的所有位置（简化）
  const walker = document.createTreeWalker(document.body, NodeFilter.SHOW_TEXT);
  let node;
  while ((node = walker.nextNode())) {
    const idx = node.textContent.indexOf("重点");
    if (idx >= 0) {
      const range = new Range();
      range.setStart(node, idx);
      range.setEnd(node, idx + "重点".length);
      ranges.push(range);
    }
  }
  const highlight = new Highlight(...ranges);
  CSS.highlights.set("keyword-highlight", highlight);
  // CSS: ::highlight(keyword-highlight) { background: yellow; }
}
// 不用修改 DOM，纯 CSS 层高亮！

05 常见陷阱

live 集合在循环中修改 → 死循环

const as = document.getElementsByTagName("a"); for (let i=0; i<as.length; i++) as[i].cloneNode().appendTo(...) —— 克隆的 a 也会被 collection 包含 → 死循环。解决：[...as] 或用 querySelectorAll。

el.querySelectorAll(">div") 不工作

子元素选择器 > 必须有左操作数。用 :scope > div。

HTMLCollection 不能 forEach

document.getElementsByTagName("div").forEach → TypeError。只能 for-of 或 [...coll].forEach。要方法链用 querySelectorAll（NodeList 有 forEach）。

querySelectorAll 结果是 static snapshot

调用时生成快照，后续 DOM 变化不反映。优点是可预测、可排序；缺点是要手动重新查。vs getElementsByClassName 的 live 行为。

ID 含特殊字符

getElementById("x:y") 没问题；querySelector("#x:y") 失败（: 是伪类）。要 querySelector("#x\\:y") 转义。用 CSS.escape(id) 通用转义。

el.children vs el.childNodes

children 仅 Element，childNodes 含文本/注释。写 DOM walker 时很容易用错。见 01 章对比表。

closest 返回自身

el.closest(".active") 若 el 本身就 match，返回 el 自己（不是祖先）。想跳过自己用 el.parentElement?.closest(...)。

跨 Shadow DOM 查询失败

document.querySelector("my-widget button") 找不到 shadow 里的 button。需要通过 host.shadowRoot.querySelector("button")。用 ::part CSS 或显式暴露 API。
