事件委托与自定义事件

事件委托是 DOM 最被低估的优化：把 1000 个子元素的 listener 合并成父元素上的一个。原理靠冒泡 + 目标判定。CustomEvent 让组件之间用事件解耦通信，是 Web Components 的标准通信机制。本章把这两个"事件进阶话题"讲透。

阅读时长 ~18 分钟 前置 04 事件传播 对应 javascript.info Event delegation; Custom events

01 速览

事件委托基础

javascript 
// ❌ 给每个按钮单独挂 listener —— N 个元素 = N 次 addEventListener
for (const btn of document.querySelectorAll(".action")) {
  btn.addEventListener("click", handle);
}

// ✓ 委托：父元素一个 listener 搞定全部
list.addEventListener("click", (e) => {
  const btn = e.target.closest(".action");
  if (!btn || !list.contains(btn)) return;
  handle(btn);
});

// 好处：
// 1. 性能：listener 少了 N-1 个
// 2. 动态：新增的 .action 无需重新绑定
// 3. 内存：回收更简单

CustomEvent 基础

javascript 
// 创建带数据的事件
const event = new CustomEvent("myevent", {
  detail: { value: 42 },
  bubbles: true,
  cancelable: true,
  composed: true,     // 穿越 shadow DOM 边界
});

// 触发
el.dispatchEvent(event);

// 监听
document.addEventListener("myevent", (e) => {
  console.log(e.detail.value);     // 42
});

02 底层逻辑

事件委托的传播机制

依赖事件冒泡（见 04 章）。子元素触发 click → 冒泡到父元素 → 父元素的 listener 执行 → 用 e.target.closest(selector) 判定是否"关心的"子元素。

  • click / keydown / input / change / submit 等常见事件都冒泡。
  • 不冒泡的事件（focus / blur / load）不能用冒泡式委托 —— 要用 capture 阶段（{ capture: true }）或用 focusin/focusout（冒泡版本）。

识别委托目标的 3 种模式

javascript 
// 模式 1：closest 找到特定选择器
list.addEventListener("click", (e) => {
  const item = e.target.closest(".item");
  if (!item) return;
  handle(item);
});

// 模式 2：data-action 属性路由
toolbar.addEventListener("click", (e) => {
  const btn = e.target.closest("[data-action]");
  if (!btn) return;
  const handler = handlers[btn.dataset.action];
  handler?.(btn, e);
});

// 模式 3：matches 判断当前元素
container.addEventListener("click", (e) => {
  if (e.target.matches("button.primary")) handlePrimary(e);
  else if (e.target.matches(".card-title")) handleTitle(e);
});

// 大多数场景推荐 closest：
// 点击发生在 button 内的 <svg> / <span>，target 不是 button
// matches 只看当前元素，closest 会向上找

不冒泡事件的委托

javascript 
// focus / blur 不冒泡 → 委托要用 capture
form.addEventListener("focus", (e) => {
  // e.target 是获得焦点的 input
  highlightCurrentField(e.target);
}, true);   // ← capture 阶段

// 或用冒泡版本：focusin / focusout
form.addEventListener("focusin", (e) => {
  highlightCurrentField(e.target);
});
// focusin 冒泡，行为和 focus 类似

CustomEvent 选项详解

选项 默认 含义
detail null 任意载荷，放 event.detail
bubbles false 是否冒泡（默认不冒泡）
cancelable false 能否 preventDefault
composed false 是否跨越 Shadow DOM 边界
⚠ CustomEvent 默认不冒泡

原生事件（click、input 等）大都冒泡。CustomEvent 默认不冒泡。需要委托监听必须显式 bubbles: true。

dispatchEvent 返回值

javascript 
const event = new CustomEvent("before-close", {
  cancelable: true,
  detail: { reason: "user" },
});
const shouldClose = el.dispatchEvent(event);
// dispatchEvent 返回 !event.defaultPrevented
// 如果有 listener 调用了 preventDefault → 返回 false

if (shouldClose) doClose();
else console.log("close cancelled by listener");

// 这个"取消"协议是很多组件 API 设计的基础
// 比如 <dialog> 的 "cancel" 事件就用这个

EventTarget 作为基类

javascript 
// ES2021 起可以直接 new EventTarget（或 extends）
class MyStore extends EventTarget {
  #data = {};
  set(key, value) {
    this.#data[key] = value;
    this.dispatchEvent(new CustomEvent("change", { detail: { key, value } }));
  }
}

const store = new MyStore();
store.addEventListener("change", (e) => console.log(e.detail));
store.set("user", { name: "Alice" });

// 比自己实现 emitter 简单，API 和 DOM 一致
// Node.js 也支持 EventTarget（除了原有 EventEmitter）

React SyntheticEvent 与原生

  • React 17 前：所有事件在 document 上一个 listener 统一分发（顶层代理）。
  • React 17+：改为 root（ReactDOM.createRoot 的容器）上代理。
  • 好处：React 的"事件合成"统一跨浏览器行为、支持 Pooling、回放。
  • 坑点：原生 stopPropagation 不会阻止 React 的事件（React 先看到）；反之亦然。避免混用。

03 应用场景

大列表交互委托

javascript 
// 聊天消息列表，10000+ 条
const chatList = document.querySelector(".chat");
chatList.addEventListener("click", (e) => {
  const msg = e.target.closest(".message");
  if (!msg) return;

  if (e.target.closest(".edit-btn")) return editMessage(msg);
  if (e.target.closest(".delete-btn")) return deleteMessage(msg);
  if (e.target.closest(".react-btn")) return showReactions(msg);

  // 默认：打开详情
  showMessage(msg);
});

// 新增消息自动享有交互能力，无需 bind

Action router（data-action 模式）

javascript 
// HTML：按钮通过 data-action 声明意图
// <button data-action="save">Save</button>
// <button data-action="delete" data-id="42">Delete</button>

const actions = {
  save(btn, e) { /* ... */ },
  delete(btn, e) { confirmDelete(btn.dataset.id); },
  // 扩展只需加键
};

document.addEventListener("click", (e) => {
  const btn = e.target.closest("[data-action]");
  if (!btn) return;
  actions[btn.dataset.action]?.(btn, e);
});

// Rails / HTMX 等框架的风格，声明式且解耦

自定义组件通信

javascript 
// Web Component 触发业务事件
class PriceInput extends HTMLElement {
  #value = 0;
  set value(v) {
    this.#value = v;
    this.dispatchEvent(new CustomEvent("value-change", {
      detail: { value: v },
      bubbles: true,
      composed: true,   // 跨越 shadow DOM
    }));
  }
}

// 使用方
priceInput.addEventListener("value-change", (e) => {
  updateTotal(e.detail.value);
});

// 类比 React onChange —— 但是 DOM 标准、无框架依赖

取消协议（before-close）

javascript 
class Dialog extends HTMLElement {
  close(reason) {
    const event = new CustomEvent("before-close", {
      detail: { reason },
      cancelable: true,
    });
    if (!this.dispatchEvent(event)) return;   // 被 preventDefault
    // 真正关闭
    this.hidden = true;
    this.dispatchEvent(new CustomEvent("close", { detail: { reason } }));
  }
}

// 使用方可以拦截
dialog.addEventListener("before-close", (e) => {
  if (hasUnsavedChanges) {
    e.preventDefault();
    showSavePrompt();
  }
});

Redux-like store 用 EventTarget

javascript 
class Store extends EventTarget {
  #state;
  constructor(initial) {
    super();
    this.#state = initial;
  }
  get state() { return this.#state; }
  dispatch(action) {
    this.#state = reducer(this.#state, action);
    this.dispatchEvent(new CustomEvent("change", { detail: this.#state }));
  }
  subscribe(fn) {
    this.addEventListener("change", fn);
    return () => this.removeEventListener("change", fn);
  }
}

const store = new Store({ count: 0 });
const unsub = store.subscribe((e) => console.log(e.detail));
store.dispatch({ type: "inc" });

04 典型代码

通用委托工具

javascript 
function delegate(root, event, selector, handler, options) {
  const listener = (e) => {
    const target = e.target.closest(selector);
    if (target && root.contains(target)) handler.call(target, e, target);
  };
  root.addEventListener(event, listener, options);
  return () => root.removeEventListener(event, listener, options);
}

// 用法
const off = delegate(list, "click", ".btn", function (e, btn) {
  console.log("clicked", btn);
});
off();  // 取消

类型安全的 CustomEvent（TS）

javascript 
// TypeScript：给 CustomEvent 加上 detail 类型
interface MyEventMap {
  "user-login": { userId: string };
  "cart-update": { count: number; total: number };
}

class TypedTarget<M extends Record<string, unknown>> extends EventTarget {
  addEventListener<K extends keyof M & string>(
    type: K,
    handler: (e: CustomEvent<M[K]>) => void,
    options?: AddEventListenerOptions
  ): void {
    super.addEventListener(type, handler as EventListener, options);
  }
  dispatch<K extends keyof M & string>(type: K, detail: M[K]) {
    this.dispatchEvent(new CustomEvent(type, { detail, bubbles: true }));
  }
}

const bus = new TypedTarget<MyEventMap>();
bus.addEventListener("user-login", (e) => e.detail.userId);   // 有类型
bus.dispatch("user-login", { userId: "abc" });                 // 参数检查

Focus 变化追踪

javascript 
// 追踪表单里当前 focused 的 input
const form = document.querySelector("form");
form.addEventListener("focusin", (e) => {
  form.querySelectorAll(".hint-box").forEach(b => b.hidden = true);
  const hint = e.target.getAttribute("aria-describedby");
  if (hint) document.getElementById(hint).hidden = false;
});

// focusin/out 冒泡，focus/blur 不冒泡 —— 要用 capture 或 focusin

全局事件总线

javascript 
// 用 DOM 事件做"全局 bus"
export const bus = new EventTarget();

// 发布
bus.dispatchEvent(new CustomEvent("notify", { detail: { text: "saved" } }));

// 订阅
bus.addEventListener("notify", (e) => showToast(e.detail.text));

// 跨组件、跨模块、甚至跨 iframe（通过 postMessage 桥接）
// 比引第三方 mitt / tiny-emitter 还小

05 常见陷阱

委托时 e.target 可能是子元素

用户点击在 <button><svg></svg> Save</button> 的 svg 上 —— target 是 svg 不是 button。必须用 e.target.closest(".btn") 找到真正的 button。

Custom event 不冒泡

new CustomEvent("x") 默认 bubbles: false。委托式监听 document.addEventListener("x", ...) 收不到 —— 必须 { bubbles: true }。

composed: false 卡在 shadow 边界

自定义组件（Shadow DOM）内 dispatch 的事件默认不穿透 host。组件发业务事件必须 composed: true。但太开放（composed + bubbles）会让外部能看到组件内部事件 —— 谨慎设计公共 API。

focus / blur 不冒泡

只能用 focusin / focusout（冒泡版本）或 capture 阶段。容易忘记。

委托 handler 的性能

每次事件在 parent 触发都会跑 handler 里的 closest 查找。deeply nested 页面里高频事件（mousemove）的委托可能反而更慢。权衡：高频事件直接绑，低频交互（click）用委托。

委托 handler 在 React 里的执行顺序

直接 document.addEventListener("click", ...) 的 handler 可能比 React 的 onClick 先或后执行（取决于 React 版本和 capture 阶段）。混用会出奇怪问题。组件内部用 React onClick，全局路由用 document capture 级委托。

dispatchEvent 在同步调用者上下文

el.dispatchEvent(event) 会同步调用所有 listener，然后才返回。如果 listener 里抛错，会冒到 dispatchEvent 的 caller。处理失败链条要加 try/catch 或用 async 模式（listener 返回 Promise 自行处理）。

事件名冲突

click、change 这些原生名自定义 dispatch 会触发原生 listener。为自定义事件用命名空间前缀（myapp:save、data-change）避免冲突。
