DOM 修改与安全

DOM 修改有三条路：分步创建（createElement + append）、字符串解析（innerHTML / insertAdjacentHTML）、模板克隆（<template>.content.cloneNode）。选错了要么慢、要么 XSS。本章把每条路的使用场景、性能、安全边界讲全。

阅读时长 ~22 分钟 前置 01 DOM 树 对应 javascript.info Modifying the document

01 速览

现代 DOM 修改 API（都支持多参 + 字符串/节点混合）

方法 位置 输入
el.append(...) 子树末尾 Node / string（自动 Text 节点）
el.prepend(...) 子树开头 同上
el.before(...) el 之前 同上
el.after(...) el 之后 同上
el.replaceWith(...) 替换 el 同上
el.replaceChildren(...) 清空并重设子节点 同上
el.remove() 移除自己 无
el.insertAdjacentHTML(pos, html) 4 种位置 HTML 字符串（解析）
el.insertAdjacentElement(pos, node) 4 种位置 Node
el.insertAdjacentText(pos, str) 4 种位置 纯文本

insertAdjacentXxx 的 position："beforebegin" / "afterbegin" / "beforeend" / "afterend"。

写入内容的 3 种方式

方式 解析 HTML？ XSS 风险 性能
textContent 否 无 最快
innerHTML / insertAdjacentHTML 是 有（用户输入需转义） 解析字符串成本
createElement + append 否 无 中等

02 底层逻辑

innerHTML 和 XSS

el.innerHTML = userInput 是经典 XSS 入口。浏览器解析 HTML 后：

  • 内联 script 不执行（HTML 规范，防 SSR 遗留 XSS）。
  • 但 <img src=x onerror=...>、<svg onload=...>、javascript: URL 都会执行。
  • <iframe srcdoc>、<object>、<embed> 等 HTML 特性也是攻击面。
javascript 
// ❌ 直接 XSS
el.innerHTML = userInput;

// ❌ 仅转义 <> 不够（<img onerror> 还能靠属性注入）
function escape(s) {
  return s.replace(/</g, "&lt;").replace(/>/g, "&gt;");
}

// ✓ 用 textContent（纯文本）
el.textContent = userInput;

// ✓ 用 DOMPurify 清洗
import DOMPurify from "dompurify";
el.innerHTML = DOMPurify.sanitize(userInput);

// ✓ 原生 Sanitizer API（Chrome 119+）
el.setHTML(userInput);                       // 自带白名单，拒绝危险元素
el.setHTML(userInput, { sanitizer: new Sanitizer({ allowElements: ["b", "i", "a"] }) });

DOM 修改触发 reflow

Reflow（布局计算）和Paint（绘制）是 DOM 修改后的大开销。规则：

  • 改属性、加/删节点、resize 都可能触发 reflow。
  • 浏览器会合并相邻修改到下一帧批量处理。
  • 但读取布局属性（offsetTop、getBoundingClientRect 等）会强制立即 reflow，叫 "forced synchronous layout"。
javascript 
// ❌ 读写交替 → 每次读都触发 reflow（layout thrashing）
for (const el of items) {
  el.style.width = (el.offsetWidth + 10) + "px";
  //                ↑ 读 → 强制 reflow
  // 写入又 invalidate 布局 → 下一次读又 reflow
}

// ✓ 读写分离（所有读在前，所有写在后）
const widths = items.map(el => el.offsetWidth);
items.forEach((el, i) => el.style.width = (widths[i] + 10) + "px");

DocumentFragment：一次 reflow

javascript 
const frag = document.createDocumentFragment();
for (const item of list) {
  const li = document.createElement("li");
  li.textContent = item;
  frag.append(li);                    // 不触发 reflow（不在 document 里）
}
ul.append(frag);                      // ← 唯一一次 reflow

// fragment 被 append 后自己变空（子节点全被搬走）
// 可复用同一 fragment 做下一批插入

<template> 元素

html 
<template id="card">
  <article class="card">
    <h3></h3>
    <p></p>
  </article>
</template>

<script>
const tpl = document.getElementById("card");
for (const data of items) {
  const clone = tpl.content.cloneNode(true);   // ← deep clone
  clone.querySelector("h3").textContent = data.title;
  clone.querySelector("p").textContent = data.body;
  container.append(clone);
}
</script>

// <template> 的 content 是 DocumentFragment
// 浏览器解析时不渲染 template 内部（不执行 script / 下载 img）
// 比 innerHTML 性能好（预解析一次，克隆多次）
// 比 createElement 链好（模板在 HTML 里，结构清晰）

插入已存在的节点 vs 克隆

把一个已在 DOM 里的节点 append 到别处，它自动从原位置移除（不是复制）。想复制用 cloneNode(true)：

javascript 
const a = document.getElementById("a");
other.append(a);                       // a 从原位置移动到 other

const b = a.cloneNode(true);           // deep clone（含子树）
other.append(b);                       // a 和 b 共存

// 只克隆外壳
const shell = a.cloneNode(false);

// ⚠ cloneNode 不复制事件监听器（需要重新 bind）

DOMParser

javascript 
// 完整解析 HTML 字符串（带 head/body）
const parser = new DOMParser();
const doc = parser.parseFromString(htmlString, "text/html");
doc.body.firstElementChild;            // 拿到解析的 DOM

// vs innerHTML：DOMParser 产出独立的 document（脚本不执行、图片不加载）
// 适合解析用户提交的 HTML 做结构分析 / 清洗

03 应用场景

清空并重新填充列表

javascript 
// 老代码
ul.innerHTML = "";                     // 清空（会被 XSS 风险警告影响）
for (const item of list) ul.append(createLi(item));

// 现代
ul.replaceChildren(...list.map(createLi));
// replaceChildren 一次调用同时清空+替换
// 没有 XSS 风险（不涉及字符串解析）

交换两个节点的位置

javascript 
// 现代 API
a.replaceWith(b);
b.before(a);
// 或用 moveBefore（新 API，保留状态如 media / iframe）

// 交换时不丢内部状态（iframe 的 contentDocument 等）很重要
// React/Vue 的 keyed list 重排依赖这个特性

批量渲染用 template

html 
<template id="row">
  <tr><td class="name"></td><td class="score"></td></tr>
</template>

<script>
const tpl = document.getElementById("row");
const frag = document.createDocumentFragment();
for (const user of users) {
  const tr = tpl.content.cloneNode(true);
  tr.querySelector(".name").textContent = user.name;
  tr.querySelector(".score").textContent = user.score;
  frag.append(tr);
}
tbody.replaceChildren(frag);
</script>

<!-- 10000 行性能：
<!-- - innerHTML 拼字符串：最快但有 XSS
<!-- - template + fragment：稍慢但安全且可读
<!-- - createElement 链：最慢 -->

虚拟滚动只渲染可视区

javascript 
// 大列表（10 万+ 行）不可能全量渲染
// 只渲染可视区 + 几行 buffer，滚动时替换
function render(startIndex, count) {
  const frag = document.createDocumentFragment();
  for (let i = startIndex; i < startIndex + count; i++) {
    frag.append(createRow(data[i]));
  }
  container.replaceChildren(frag);
  container.style.paddingTop = startIndex * rowHeight + "px";
}

// IntersectionObserver 或 scroll + requestAnimationFrame 驱动
// React/Vue 生态有 react-virtual / vue-virtual-scroller

安全的富文本渲染

javascript 
// 用户发布的富文本（支持 b / i / a / ul 等）
import DOMPurify from "dompurify";
const config = {
  ALLOWED_TAGS: ["b", "i", "a", "ul", "ol", "li", "p", "br"],
  ALLOWED_ATTR: ["href", "target"],
  // 强制加 rel=noopener
  ADD_ATTR: ["target"],
};
const clean = DOMPurify.sanitize(userInput, config);
el.innerHTML = clean;

// 原生 Sanitizer API（Chrome 119+ / Safari/Firefox 尚未）

04 典型代码

h() helper 函数

javascript 
// 类似 React.createElement 的 DOM 版
function h(tag, props = {}, ...children) {
  const el = document.createElement(tag);
  for (const [key, val] of Object.entries(props)) {
    if (key === "style" && typeof val === "object") {
      Object.assign(el.style, val);
    } else if (key.startsWith("on") && typeof val === "function") {
      el.addEventListener(key.slice(2).toLowerCase(), val);
    } else if (key === "class" || key === "className") {
      el.className = val;
    } else {
      el.setAttribute(key, val);
    }
  }
  for (const child of children.flat()) {
    if (child == null || child === false) continue;
    el.append(typeof child === "string" || typeof child === "number"
      ? String(child) : child);
  }
  return el;
}

// 用法
const card = h("article", { class: "card", onClick: handleClick },
  h("h3", {}, user.name),
  h("p", {}, user.bio),
);

安全 HTML 工具函数

javascript 
// 标签模板 + 自动转义
const escape = (s) => String(s)
  .replace(/&/g, "&amp;")
  .replace(/</g, "&lt;")
  .replace(/>/g, "&gt;")
  .replace(/"/g, "&quot;")
  .replace(/'/g, "&#39;");

function html(strings, ...values) {
  return strings.reduce((s, lit, i) => s + lit + (i < values.length ? escape(values[i]) : ""), "");
}

// 用法
el.innerHTML = html`<h1>Hello ${userName}</h1>`;
// userName 自动转义；字面量部分原样保留

批量 DOM 操作

javascript 
// 一次 rAF 内合并所有 DOM 修改
let pending = false;
let ops = [];

function schedule(op) {
  ops.push(op);
  if (!pending) {
    pending = true;
    requestAnimationFrame(() => {
      const frag = document.createDocumentFragment();
      for (const op of ops) op(frag);
      target.append(frag);
      ops = [];
      pending = false;
    });
  }
}

// 使用
items.forEach(item => schedule(frag => frag.append(createLi(item))));
// 无论调多少次，都只在下一帧做一次 append

简易 diff + patch

javascript 
// 简化版 keyed reconciliation
function patchList(container, newData, keyFn, createEl) {
  const oldMap = new Map(
    [...container.children].map(el => [el.dataset.key, el])
  );
  const frag = document.createDocumentFragment();
  for (const data of newData) {
    const key = keyFn(data);
    let el = oldMap.get(key);
    if (el) {
      oldMap.delete(key);          // 存在则移到 fragment（自动移动）
    } else {
      el = createEl(data);
      el.dataset.key = key;
    }
    frag.append(el);
  }
  oldMap.forEach(el => el.remove());  // 剩下的是要删的
  container.append(frag);
}

05 常见陷阱

innerHTML 丢失事件监听器

el.innerHTML += "..." = 读出字符串、拼接、重新解析子树 → 原有子节点上的事件监听器全部丢失（因为节点被替换了）。用 insertAdjacentHTML 或 append 创建新节点方式。

append vs appendChild

append：可传多个 / 接收字符串 / 返回 undefined。appendChild：只一个 Node / 返回 appended 节点 / 不接受字符串。新代码用 append。

节点被移动而非复制

other.append(existingNode) 把 existingNode 从原位置搬到 other。想保留原位置必须 cloneNode(true) 先克隆。

cloneNode 不克隆事件监听器

addEventListener 挂的 listener 克隆时丢失。onclick= 内联属性会被克隆。工程建议用事件委托（04 章）避免这个问题。

读写交替触发 layout thrashing

循环里读 offsetHeight 再写 style 会让浏览器反复 reflow。分两阶段或用 rAF 合并。

innerHTML 不执行 script 但执行事件

<script> 标签被解析成元素但 script 不执行（规范）。但 <img src=x onerror=alert()> 的 onerror 会执行。不要认为"没 script 就安全"。

仅转义 HTML 不够防 XSS

转义 < > & " ' 能防大部分，但 javascript: URL、on* 属性、CSS url() 等都是注入面。用 DOMPurify 或 Trusted Types（P4-07）覆盖全面。

<template>.innerHTML 和 .content.innerHTML 不同

template.innerHTML 返回模板标签的 HTML 字符串（相当于里面的内容）；template.content.innerHTML 则是 DocumentFragment 的内容。克隆要用 .content.cloneNode(true)，不是 .cloneNode。
