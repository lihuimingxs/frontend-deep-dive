各种 Observer

DOM 的 4 大 Observer：Mutation / Resize / Intersection / Performance。它们是"异步观察 DOM 变化 / 视口 / 性能指标"的规范 API。相比事件监听 + 轮询，Observer 有批处理、性能优化、浏览器调度的优势。

阅读时长 ~20 分钟 前置 01 DOM · P3 Event Loop 对应 MDN MutationObserver

01 速览

Observer 观察什么 回调时机
MutationObserver DOM 树结构 / 属性 / 文本变化 microtask
ResizeObserver 元素尺寸变化 rAF 之前
IntersectionObserver 元素与视口（或 root）的交叉 空闲时批量
PerformanceObserver 性能时间线条目（LCP / FCP / long task 等） 条目生成后

共同 API 形态

javascript 
// 4 个 Observer 的 API 形态基本一致
const observer = new XxxObserver((entries) => {
  for (const entry of entries) { /* 处理 */ }
});

observer.observe(target, options);
observer.unobserve(target);      // 部分 Observer 有
observer.disconnect();            // 停止观察所有

02 底层逻辑

MutationObserver

javascript 
const mo = new MutationObserver((mutations) => {
  for (const m of mutations) {
    m.type;              // "childList" / "attributes" / "characterData"
    m.target;            // 变化的节点
    m.addedNodes;        // NodeList（新增）
    m.removedNodes;      // NodeList（移除）
    m.attributeName;     // attributes 时：属性名
    m.oldValue;          // 如果 attributeOldValue/characterDataOldValue: true
  }
});

mo.observe(el, {
  childList: true,            // 子节点增删
  attributes: true,           // 属性变化
  characterData: true,        // 文本变化
  subtree: true,              // 递归观察子树
  attributeFilter: ["class"], // 只关心某些属性
  attributeOldValue: true,    // 提供旧值
  characterDataOldValue: true,
});

// 停止
mo.disconnect();

// 特性：
// - 回调走 microtask 队列，多个 mutation 批量合并
// - 同步 DOM 修改连续发生 → 一次 callback
// - 回调里再改 DOM 会累积到下一轮

ResizeObserver

javascript 
const ro = new ResizeObserver((entries) => {
  for (const entry of entries) {
    entry.target;                     // 被观察元素
    entry.contentRect;                // {x, y, width, height}（不含 padding/border）
    entry.borderBoxSize;              // 含 border（device pixel）
    entry.contentBoxSize;             // 不含 border
    entry.devicePixelContentBoxSize;  // 设备像素精度
  }
});

ro.observe(el, { box: "content-box" });    // 或 "border-box" / "device-pixel-content-box"

// 用途：
// - 响应式组件：根据自身宽度调整布局（container queries 的 JS 实现）
// - Chart 自适应容器尺寸
// - 图片 / iframe 自动调整

// 比 window.resize 好在：
// - 观察单个元素，不用遍历
// - 浏览器在 layout 之后调 → 拿到准确尺寸
// - 不触发 layout thrashing
⚠ ResizeObserver loop 警告

如果回调里改 DOM 导致它观察的元素再次 resize → 无限循环。浏览器检测到会抛 ResizeObserver loop limit exceeded 警告。在回调里改 DOM 前检查"是否真的需要改"。

IntersectionObserver

javascript 
const io = new IntersectionObserver((entries) => {
  for (const entry of entries) {
    entry.target;
    entry.isIntersecting;       // 是否与 root 相交
    entry.intersectionRatio;    // 0-1 交叉比例
    entry.intersectionRect;     // 交叉矩形
    entry.boundingClientRect;   // target 的矩形
    entry.rootBounds;           // root 的矩形
  }
}, {
  root: null,                   // null = viewport；或任意可滚动 Element
  rootMargin: "50px",           // 扩展 root 边界（提前触发）
  threshold: [0, 0.5, 1],       // 0/50%/100% 时触发（单值或数组）
});

io.observe(el);

// 用途：
// - 懒加载图片（进入视口前 100px 开始下载）
// - 无限滚动（哨兵元素进入视口触发 load more）
// - 视频自动播放/暂停
// - 广告曝光统计
// - 滚动动画触发

// 优势：完全异步，不阻塞主线程，浏览器原生优化

PerformanceObserver

javascript 
const po = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    entry.entryType;       // "paint" / "largest-contentful-paint" / "longtask" / ...
    entry.name;
    entry.startTime;
    entry.duration;
  }
});

po.observe({
  entryTypes: ["paint", "largest-contentful-paint", "longtask", "layout-shift"],
  buffered: true,          // 包括观察前已经发生的条目
});

// 常见 entry type：
// - navigation      页面导航时长
// - resource        资源加载
// - paint           FP / FCP
// - largest-contentful-paint  LCP
// - first-input     FID
// - layout-shift    CLS
// - longtask        超过 50ms 的 task
// - measure         performance.measure 的手动标记
// - event           Event Timing API（INP）

Observer 与 rAF 的时序

HTML 规范里每帧的时序：

text 
rAF 回调
 ↓
IntersectionObserver / ResizeObserver 回调（layout 之后）
 ↓
style/layout/paint
 ↓
（下一帧）

所以 ResizeObserver / IntersectionObserver 里读布局属性是稳定的。MutationObserver 走 microtask，和 rAF 无关。

03 应用场景

懒加载图片

javascript 
// HTML：<img data-src="..." />
const io = new IntersectionObserver((entries) => {
  for (const e of entries) {
    if (e.isIntersecting) {
      const img = e.target;
      img.src = img.dataset.src;
      io.unobserve(img);
    }
  }
}, { rootMargin: "200px" });    // 提前 200px 开始下载

document.querySelectorAll("img[data-src]").forEach(img => io.observe(img));

// 现代替代：<img loading="lazy" src="...">（浏览器原生，更快）
// IntersectionObserver 适合：复杂场景（带占位图切换、动画进入）

无限滚动

javascript 
// 最后放一个哨兵元素，进入视口就 load more
const sentinel = document.querySelector(".sentinel");
const io = new IntersectionObserver(async (entries) => {
  if (!entries[0].isIntersecting) return;
  const newItems = await loadNextPage();
  appendItems(newItems);
  if (noMorePages) io.unobserve(sentinel);
});
io.observe(sentinel);

// 比监听 scroll 事件 + 距底部距离计算快得多

Container queries 的 JS 实现

javascript 
// CSS @container 出现前，只能靠 ResizeObserver 做容器响应式
const ro = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const w = entry.contentRect.width;
    entry.target.dataset.size =
      w < 400 ? "sm" : w < 800 ? "md" : "lg";
  }
});
ro.observe(card);

// CSS
// .card[data-size="sm"] { ... }
// .card[data-size="md"] { ... }
// .card[data-size="lg"] { ... }

// 现在优先用原生 CSS @container
// 但复杂 JS 逻辑还是 ResizeObserver 清晰

自动 sticky header

javascript 
// 当 header 被滚出视口顶部时加 class 做阴影等效果
const sentinel = document.createElement("div");
document.body.prepend(sentinel);
new IntersectionObserver(([entry]) => {
  header.classList.toggle("scrolled", !entry.isIntersecting);
}).observe(sentinel);

// 对比 scroll + getBoundingClientRect：零 layout thrashing

MutationObserver 跟踪 framework 插入

javascript 
// Shopify/Wix 等平台里第三方 SDK 监听 checkout 表单出现
new MutationObserver((mutations) => {
  for (const m of mutations) {
    for (const node of m.addedNodes) {
      if (node.matches?.(".checkout-form")) enhanceCheckout(node);
    }
  }
}).observe(document.body, { childList: true, subtree: true });

// ⚠ 性能陷阱：subtree + 大页面 = 频繁回调
// 尽可能缩小 root / 用 attributeFilter 减少噪音

PerformanceObserver 上报 Core Web Vitals

javascript 
// LCP / CLS / INP 上报（简化，生产用 web-vitals 库）
new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    navigator.sendBeacon("/rum", JSON.stringify({
      name: "lcp",
      value: entry.startTime,
    }));
  }
}).observe({ type: "largest-contentful-paint", buffered: true });

new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (!entry.hadRecentInput) {
      console.log("layout shift:", entry.value);
    }
  }
}).observe({ type: "layout-shift", buffered: true });

04 典型代码

宽高比监听

javascript 
function onAspectRatio(el, callback) {
  let prev = 0;
  const ro = new ResizeObserver(([entry]) => {
    const { width, height } = entry.contentRect;
    const ratio = width / height;
    if (Math.abs(ratio - prev) > 0.01) {
      prev = ratio;
      callback(ratio, width, height);
    }
  });
  ro.observe(el);
  return () => ro.disconnect();
}

// 用法：图片容器变成正方形或横向时换布局
onAspectRatio(container, (r) => {
  container.classList.toggle("portrait", r < 1);
});

元素进入视口的 Promise

javascript 
function whenVisible(el, options = {}) {
  return new Promise((resolve) => {
    const io = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        io.disconnect();
        resolve(entry);
      }
    }, options);
    io.observe(el);
  });
}

// 用法
await whenVisible(analyticsMarker);
trackView(page);

等待元素出现

javascript 
function waitFor(selector, root = document.body, timeout = 10_000) {
  const existing = root.querySelector(selector);
  if (existing) return Promise.resolve(existing);
  return new Promise((resolve, reject) => {
    const timer = setTimeout(() => {
      mo.disconnect();
      reject(new Error("timeout"));
    }, timeout);
    const mo = new MutationObserver(() => {
      const el = root.querySelector(selector);
      if (el) {
        clearTimeout(timer);
        mo.disconnect();
        resolve(el);
      }
    });
    mo.observe(root, { childList: true, subtree: true });
  });
}

// 用法：等异步加载的 widget 出现
const btn = await waitFor("#dynamic-button");
btn.click();

监控长任务

javascript 
if (typeof PerformanceObserver !== "undefined") {
  try {
    new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.duration > 100) {
          report({
            type: "long-task",
            duration: entry.duration,
            name: entry.name,
            attribution: entry.attribution?.[0]?.containerName,
          });
        }
      }
    }).observe({ type: "longtask", buffered: true });
  } catch { /* not supported */ }
}

05 常见陷阱

忘记 disconnect 导致泄漏

Observer 持有 target 的弱引用 + callback 的闭包。如果组件卸载后没 disconnect，callback 里引用的 state 就没法 GC。React 的 useEffect cleanup 里必须 disconnect。

ResizeObserver loop

回调里改尺寸导致再次回调 → 循环。浏览器会抛 ResizeObserver loop limit exceeded 错误。用 rAF 包裹 state 更新或加节流。

MutationObserver subtree 性能

{ subtree: true } + 大页面 → 每次 DOM 变化都触发。降低 root 范围 / 用 attributeFilter 限定属性名。不要在 body 上监听整个文档。

IntersectionObserver threshold 理解

threshold: 0：只要有 1px 相交就触发。1：必须 100% 进入才触发。想检测"完全离开" → threshold: 0 + 看 isIntersecting === false。[0, 0.25, 0.5, 0.75, 1] 用于阶段性触发。

PerformanceObserver buffered: true 重要性

页面早期发生的条目（如 FCP 在 0ms 时就有）如果 buffered: false，你注册得晚就看不到。总是用 buffered: true（除非明确只要"之后"的条目）。

Observer 回调里不能立即调用 .observe

MutationObserver 回调里改 DOM 会让该 observer 再次触发（下一 microtask）。循环风险。如果必须，加 disconnect 后 re-observe 或用标志防递归。

Observer 在 React Strict Mode 双触发

Dev 模式 useEffect 跑两次 → Observer 被创建两次。cleanup 要 disconnect，React 会在第二次 useEffect 前 cleanup 第一次。实际生产只跑一次。

跨 iframe 的 IntersectionObserver

root 必须和 target 在同 realm。跨 iframe 观察需要用 postMessage 通信 + 各自 iframe 里独立 Observer。
