资源加载时机

浏览器加载一个页面从 HTML 请求开始到可交互之间有多个关键时刻：DOMContentLoaded → load → 首次绘制 → LCP。script 标签的 defer/async/type=module 决定 JS 何时执行。preload/prefetch/preconnect 是 performance 工具链的核心。本章把这些时序 + Core Web Vitals 串起来。

阅读时长 ~22 分钟 前置 P3 Event Loop · P4 模块 对应 javascript.info Page lifecycle events

01 速览

页面生命周期事件

事件 触发时机 用途
DOMContentLoaded HTML 解析完成，DOM 树构建完（含 defer 脚本） 初始化 JS
load 所有资源（图片、样式、子 iframe 等）加载完 依赖图片/样式尺寸的逻辑
pageshow 页面可见（初次 + 从 bfcache 恢复） bfcache 恢复后的重新初始化
pagehide 页面即将不可见（进入 bfcache 或关闭） 代替 unload 上报
beforeunload 用户要离开（可弹确认） 未保存提示（谨慎用）
visibilitychange 标签页切换 / 最小化 暂停计时器、停止上报

<script> 标签的 4 种形态

html 
<!-- 1. 默认：阻塞解析 + 下载 + 执行 -->
<script src="a.js"></script>

<!-- 2. async：异步下载，下载完立即执行（仍可能阻塞 parser） -->
<script async src="a.js"></script>

<!-- 3. defer：异步下载，HTML 解析完按顺序执行 -->
<script defer src="a.js"></script>

<!-- 4. module：默认 defer + CORS + 严格模式 + import/export -->
<script type="module" src="a.js"></script>
<!-- 5. module + async：下载完立即执行 -->
<script type="module" async src="a.js"></script>

02 底层逻辑

HTML 解析时序

text 
<html> 开始解析
 ↓
 遇到 <script>（默认）    → 暂停解析，下载，执行，继续
 遇到 <script async>     → 下载并行，下载完暂停解析立即执行
 遇到 <script defer>     → 下载并行，到 DOMContentLoaded 前按序执行
 遇到 <link rel=stylesheet> → 下载不阻塞解析，但阻塞 script 执行和渲染
 遇到 <img>              → 下载并行，不阻塞
 ↓
 DOM 树构建完（含 defer scripts 执行完）
 → 触发 DOMContentLoaded
 ↓
 所有资源（img / iframe / css / async script 等）加载完
 → 触发 load
 ↓
 浏览器渲染：FCP（首次内容绘制）→ LCP（最大内容绘制）

defer vs async

default async defer
下载 阻塞 parser 异步 异步
执行 立即 下载完立即（可能在 DOMContentLoaded 前或后） DOMContentLoaded 之前，按声明顺序
多个 script 顺序 按声明顺序 下载顺序（不一定按声明） 按声明顺序
inline script 可用？ 可以 ✗（只对外部 src 有效） ✗（同上）

工程选择：

  • 主应用脚本 → defer 或 type=module（默认 defer）
  • 第三方埋点、无依赖的 → async
  • inline 的初始化代码 → 放 body 尾或用 DOMContentLoaded

Module script 特殊性

  • 默认 defer：不阻塞解析，等 HTML 完再执行。
  • 严格模式：自动。
  • 每个模块执行一次：同一 URL 多次 import 只加载 parse 一次。
  • 跨源要 CORS：加 crossorigin 属性并服务端返回 CORS 头。
  • 支持 top-level await（ES2022）。
  • nomodule：老浏览器 fallback 用 <script nomodule src="legacy.js">。

资源预加载指令

link rel 含义
preload 当前页必需资源，高优先级下载（但不执行）
modulepreload preload + 建 ESM module graph（执行前准备好）
prefetch 下一页可能用，低优先级预取到缓存
preconnect 对指定域名提前 DNS + TLS 握手
dns-prefetch 仅 DNS 查询（preconnect 的子集）
html 
<!-- 当前页关键 JS：高优先级下载 -->
<link rel="preload" href="/app.js" as="script" crossorigin>

<!-- 当前页关键字体：避免字体闪烁 -->
<link rel="preload" href="/fonts/main.woff2" as="font" type="font/woff2" crossorigin>

<!-- ES 模块 -->
<link rel="modulepreload" href="/js/main.mjs">
<link rel="modulepreload" href="/js/utils.mjs">

<!-- 下一页可能要访问的 API -->
<link rel="preconnect" href="https://api.example.com">
<link rel="preconnect" href="https://cdn.example.com">

<!-- 下一页资源（低优先级） -->
<link rel="prefetch" href="/next-page.html" as="document">

Core Web Vitals

指标 含义 好的阈值
LCP（Largest Contentful Paint） 最大内容绘制时间 < 2.5s
INP（Interaction to Next Paint） 交互到下次渲染的延迟（2024 起替代 FID） < 200ms
CLS（Cumulative Layout Shift） 累积布局偏移 < 0.1
FCP（First Contentful Paint） 首次内容绘制 < 1.8s
TTFB（Time to First Byte） 首字节时间 < 800ms

03 应用场景

DOMContentLoaded vs load 选择

javascript 
// DOM 就绪就跑（不等图片）—— 大部分 init 代码
document.addEventListener("DOMContentLoaded", () => {
  hydrateReactApp();
  bindGlobalShortcuts();
});

// 图片 / iframe 也完事才跑（依赖尺寸的代码）
window.addEventListener("load", () => {
  measureHeroImage();
  calculateLayout();
});

// 现代：直接把 script 放 body 尾 + defer，document.readyState 已经是 "interactive" 或 "complete"
// 不用监听事件，直接执行

脚本位置与类型的选择

html 
<!-- 头部：关键主应用（defer 最安全） -->
<head>
  <script type="module" src="/src/main.mjs"></script>
  <!-- 默认 defer，加载时机最优 -->

  <!-- 埋点 / 分析（async） -->
  <script async src="https://analytics.com/tag.js"></script>

  <!-- 关键资源预加载 -->
  <link rel="modulepreload" href="/src/main.mjs">
</head>

<!-- Body 尾：内联初始化 -->
<body>
  ...
  <script>
    // 这里可以直接操作 DOM，HTML 已全部解析
    document.body.classList.add("loaded");
  </script>
</body>

bfcache 友好

javascript 
// 浏览器 back/forward 时可能从 bfcache 恢复（完整状态）
window.addEventListener("pageshow", (e) => {
  if (e.persisted) {
    // 从 bfcache 恢复：重新启动定时器、拉新数据
    refreshData();
  }
});
window.addEventListener("pagehide", (e) => {
  if (e.persisted) {
    // 进入 bfcache：清定时器但保留 state
    clearTimers();
  }
});

// ⚠ beforeunload / unload 事件阻止 bfcache
// 能不用就不用。要上报用 pagehide + navigator.sendBeacon

离开页面前上报

javascript 
window.addEventListener("visibilitychange", () => {
  if (document.visibilityState === "hidden") {
    // 移动端切后台、PC 切标签都会触发
    navigator.sendBeacon("/api/track", JSON.stringify(pendingEvents));
    pendingEvents = [];
  }
});

// pagehide 也是（更彻底：页面即将不可见或关闭）
window.addEventListener("pagehide", () => {
  navigator.sendBeacon("/api/track", JSON.stringify(pendingEvents));
});

// sendBeacon：非阻塞 POST，浏览器保证在关闭前发出
// 不像 fetch 可能被页面卸载打断

Intersection + preload = 渐进加载

javascript 
// 用户滚动接近目标时 preload 下一个模块的 chunk
const sentinel = document.querySelector("#load-sentinel");
new IntersectionObserver(([entry]) => {
  if (entry.isIntersecting) {
    const link = document.createElement("link");
    link.rel = "modulepreload";
    link.href = "/chunks/next-feature.mjs";
    document.head.append(link);
  }
}, { rootMargin: "500px" }).observe(sentinel);

// 用户点击时 import() 已经缓存

04 典型代码

ready 工具函数

javascript 
function ready(fn) {
  if (document.readyState !== "loading") fn();
  else document.addEventListener("DOMContentLoaded", fn, { once: true });
}

// Promise 版
function ready() {
  if (document.readyState !== "loading") return Promise.resolve();
  return new Promise(r => document.addEventListener("DOMContentLoaded", r, { once: true }));
}

// 用法
await ready();
initApp();

Core Web Vitals 上报

javascript 
// 用 web-vitals 库（最简单）
import { onLCP, onINP, onCLS, onFCP, onTTFB } from "web-vitals";

function report(metric) {
  navigator.sendBeacon("/rum", JSON.stringify({
    name: metric.name,
    value: metric.value,
    delta: metric.delta,
    id: metric.id,
  }));
}

onLCP(report);
onINP(report);
onCLS(report);
onFCP(report);
onTTFB(report);

preload 的 API 创建

javascript 
function preloadScript(url) {
  const link = document.createElement("link");
  link.rel = "modulepreload";
  link.href = url;
  link.crossOrigin = "anonymous";
  document.head.append(link);
  return new Promise((resolve, reject) => {
    link.onload = resolve;
    link.onerror = reject;
  });
}

// 用户 hover 时预取
btn.addEventListener("mouseenter", () => {
  preloadScript("/heavy-feature.mjs");
}, { once: true });

visibility-aware 动画

javascript 
let rafId;
function loop() {
  draw();
  rafId = requestAnimationFrame(loop);
}

document.addEventListener("visibilitychange", () => {
  if (document.visibilityState === "hidden") {
    cancelAnimationFrame(rafId);
  } else {
    loop();
  }
});
loop();

// 游戏 / 动画 / 实时显示都应该这样做 —— 省电 + 减 CPU

05 常见陷阱

CSS 阻塞 script 执行

<link rel="stylesheet"> 后面的 script 要等样式加载完才跑（避免在 JS 里读取样式出错）。想非阻塞用 media="print" onload="this.media='all'" 技巧或关键 CSS inline。

async script 顺序不保证

两个 async script 先下载完的先执行。如果 A 依赖 B 不能都用 async。要按序用 defer 或动态 import。

module 在 Safari 早期版本有 bug

Safari 10.1 是第一个支持 ESM 的，但早期有重复执行、fetch 错误等问题。现代已稳定，但保险起见还是构建工具做 transpile。

beforeunload 阻止 bfcache

加了 beforeunload 事件 → 浏览器不缓存页面 → back/forward 慢。如果只是"提示未保存"，用 localStorage 恢复 + pagehide 保存。

unload 不可靠

移动端常常不触发（用户直接杀进程）。Safari iOS 有时也不触发。上报一律用 visibilitychange → hidden 时 sendBeacon。

preload 太多反而慢

preload 让资源抢高优先级，过多会挤占带宽 / CPU，导致关键资源延迟。只 preload 真正"当前页会用"的关键资源（通常 2-3 个字体 / 关键 JS）。Lighthouse 会提示"preload 的资源未被使用"。

字体 preload 忘记 crossorigin

字体文件需要 CORS（规范规定）。<link rel="preload" as="font"> 必须加 crossorigin，否则浏览器当作两个不同资源下载两次。

动态注入 script 不走 defer / async

document.createElement("script"); s.src = "..."; document.body.append(s) 默认是 async。想按序执行要手动等前一个 load 完。
