Proxy 与 Reflect

Proxy 允许你拦截一个对象的任意操作（读/写/删/调用/new/in/typeof 等）——是 JS 元编程的基石。Reflect 是 Proxy 的"对称 API"：把每个 trap 都映射到一个同名静态方法，让 "调用默认行为" 变成显式一行代码。本章配 Vue 3 reactivity 的最小实现，把它们的工程价值讲清楚。

阅读时长 ~25 分钟 前置 05 原型 · 06 class · 11 Symbol 对应 javascript.info Proxy and Reflect

01 速览

javascript 
const target = { name: "Alice" };
const handler = {
  get(target, prop, receiver) {
    console.log("read", prop);
    return Reflect.get(target, prop, receiver);   // 默认行为
  },
  set(target, prop, value, receiver) {
    console.log("write", prop, "=", value);
    return Reflect.set(target, prop, value, receiver);
  },
};
const proxy = new Proxy(target, handler);
proxy.name;              // read name → "Alice"
proxy.name = "Bob";      // write name = Bob

// 原对象也被改了 —— Proxy 没有自己的存储，都转发到 target
target.name;             // "Bob"

02 底层逻辑

对象的"内部方法"

规范给每个对象定义了一组内部方法：[[Get]]、[[Set]]、[[HasProperty]]、[[Delete]]、[[OwnPropertyKeys]] 等 —— 这些是所有属性访问的底层入口。

Proxy 就是把这些内部方法改成"调 handler 的 trap 方法"。如果 trap 没定义，就走默认的 Ordinary Object 行为（即 Reflect.xxx）。

Invariants（不变式）

Proxy 允许自定义行为，但不是无限制 —— 规范定义了一组 invariants，trap 的返回值违反了就抛 TypeError：

  • get：如果属性在 target 上是 writable: false, configurable: false 的数据属性，返回值必须等于 target 的值。
  • set：如果属性在 target 上是 writable: false, configurable: false，只能设为同值。
  • ownKeys：必须包含 target 所有 non-configurable 自有属性，且不能多出 target 里不存在的非新增属性（如果 target 不可扩展）。
  • getPrototypeOf：如果 target 不可扩展，返回值必须等于 target 的 prototype。

工程启示：正常 Proxy 使用不会触碰这些。它们是为了防止 Proxy 被用来伪造"规范保证"（如 Object.freeze 的对象不可变）。

Reflect 存在的三个理由

  1. trap 里调用默认行为：get(t, k) { return Reflect.get(t, k); }。如果不用 Reflect，要写 t[k]，但这会丢失 receiver 等信息。
  2. 函数式，不抛错：Object.defineProperty(o, k, desc) 失败抛错；Reflect.defineProperty 返回 boolean。写"尝试定义，失败回退"时更干净。
  3. 正确的 receiver 传递：Reflect.get(target, prop, receiver) 的 receiver 会作为 getter 的 this。Proxy 的 get trap 第三参就是为此。

receiver 是什么

Reflect.get(target, prop, receiver) 的 receiver 参数：当 target[prop] 是 getter 时，getter 的 this 会被设为 receiver（不是 target）。这是实现"继承的 getter 能看到子类字段"的关键。

javascript 
const parent = {
  get name() { return this.rawName; },
};
const child = { rawName: "Alice", __proto__: parent };

// 子类 child.name 读取时：
child.name;                           // "Alice"
// 规范上：child.[[Get]]("name", child) → 找不到 → parent.[[Get]]("name", child)
// → getter 以 child 作为 this 调用 → this.rawName === "Alice"

// 在 Proxy 里不传 receiver 会破坏继承：
const p = new Proxy(child, {
  get(t, k) { return t[k]; },          // ❌ 丢了 receiver
});
// get(t, k, receiver) { return Reflect.get(t, k, receiver); }  // ✓

可撤销 Proxy

javascript 
const { proxy, revoke } = Proxy.revocable({ x: 1 }, {});
proxy.x;                 // 1
revoke();
proxy.x;                 // TypeError: Cannot perform 'get' on a proxy that has been revoked

// 用途：把对象交给不信任的代码，用完撤销引用
// Membrane 模式（跨 realm 或跨信任边界的包装）的基础

Proxy 的性能开销

每次 proxy 访问都要调 handler trap → 调 Reflect → 最终才到 target。比直接访问慢 5-50 倍（取决于 trap 复杂度）。热点代码路径上避免裹 Proxy。Vue/Mobx/Immer 等库做了大量优化（惰性 proxy、缓存等）。

03 应用场景

Vue 3 reactivity 的最小实现

javascript 
const targetMap = new WeakMap();         // target → Map(key → Set(effects))
let activeEffect = null;

function track(target, key) {
  if (!activeEffect) return;
  let deps = targetMap.get(target);
  if (!deps) { deps = new Map(); targetMap.set(target, deps); }
  let effects = deps.get(key);
  if (!effects) { effects = new Set(); deps.set(key, effects); }
  effects.add(activeEffect);
}

function trigger(target, key) {
  const effects = targetMap.get(target)?.get(key);
  effects?.forEach(fn => fn());
}

function reactive(target) {
  return new Proxy(target, {
    get(t, key, receiver) {
      track(t, key);
      const result = Reflect.get(t, key, receiver);
      // 嵌套也要 reactive —— Vue 做懒 proxy，这里简化
      return typeof result === "object" && result !== null ? reactive(result) : result;
    },
    set(t, key, value, receiver) {
      const old = t[key];
      const result = Reflect.set(t, key, value, receiver);
      if (old !== value) trigger(t, key);
      return result;
    },
  });
}

function effect(fn) {
  activeEffect = fn;
  fn();
  activeEffect = null;
}

// 用法
const state = reactive({ count: 0 });
effect(() => console.log("count is", state.count));   // 立即打印 "count is 0"
state.count++;                                         // 自动触发 effect → "count is 1"

默认值 / 空对象安全访问

javascript 
function withDefault(obj, defaultValue) {
  return new Proxy(obj, {
    get(t, k, r) { return Reflect.has(t, k) ? Reflect.get(t, k, r) : defaultValue; },
  });
}

const counts = withDefault({ a: 1 }, 0);
counts.a;           // 1
counts.b;           // 0  （不抛错）
counts.c;           // 0

// 或者 "所有路径都可访问" 的深代理（见 dset/immer）
function nested() {
  return new Proxy({}, {
    get: (t, k) => Reflect.has(t, k) ? t[k] : (t[k] = nested()),
  });
}
const d = nested();
d.a.b.c.d = 1;      // 自动建中间层

验证 / 只读

javascript 
// 只读
function readonly(obj) {
  return new Proxy(obj, {
    set()              { throw new Error("readonly"); },
    deleteProperty()   { throw new Error("readonly"); },
    defineProperty()   { throw new Error("readonly"); },
  });
}

// 带 schema 的 setter 校验
function validated(obj, validators) {
  return new Proxy(obj, {
    set(t, k, v, r) {
      const validator = validators[k];
      if (validator && !validator(v)) throw new TypeError(`invalid ${k}`);
      return Reflect.set(t, k, v, r);
    },
  });
}

const u = validated({}, {
  age: (v) => Number.isInteger(v) && v >= 0,
  name: (v) => typeof v === "string" && v.length > 0,
});
u.age = 30;         // ✓
u.age = -5;         // TypeError

链式 API（类似 jQuery）

javascript 
function chainable(obj) {
  return new Proxy(obj, {
    get(t, k, r) {
      const val = Reflect.get(t, k, r);
      if (typeof val !== "function") return val;
      return function (...args) {
        const ret = val.apply(this === r ? t : this, args);
        return ret === undefined ? r : ret;       // 方法若无返回，返回 proxy 本身
      };
    },
  });
}

虚拟对象：按需生成属性

javascript 
// 创建一个"所有属性都是函数，且名字反映函数功能"的对象
const lib = new Proxy({}, {
  get(_, name) {
    if (name === Symbol.toPrimitive) return () => "[lib]";
    return (...args) => ({ action: name, args });
  },
});

lib.log("hello");              // { action: "log", args: ["hello"] }
lib.fetch("/api/users");       // { action: "fetch", args: ["/api/users"] }
// 类似 socket.io 风格的 RPC 代理、GraphQL client 里的 builder

04 典型代码

完整 handler（用于调试）

javascript 
function traced(obj) {
  return new Proxy(obj, {
    get(t, k, r)                   { console.log("get", k); return Reflect.get(t, k, r); },
    set(t, k, v, r)                { console.log("set", k, "=", v); return Reflect.set(t, k, v, r); },
    has(t, k)                      { console.log("has", k); return Reflect.has(t, k); },
    deleteProperty(t, k)           { console.log("delete", k); return Reflect.deleteProperty(t, k); },
    ownKeys(t)                     { console.log("ownKeys"); return Reflect.ownKeys(t); },
    getOwnPropertyDescriptor(t, k) { return Reflect.getOwnPropertyDescriptor(t, k); },
    defineProperty(t, k, d)        { return Reflect.defineProperty(t, k, d); },
    getPrototypeOf(t)              { return Reflect.getPrototypeOf(t); },
    setPrototypeOf(t, p)           { return Reflect.setPrototypeOf(t, p); },
    isExtensible(t)                { return Reflect.isExtensible(t); },
    preventExtensions(t)           { return Reflect.preventExtensions(t); },
    apply(t, thisArg, args)        { console.log("apply", args); return Reflect.apply(t, thisArg, args); },
    construct(t, args, newTarget)  { console.log("new", args); return Reflect.construct(t, args, newTarget); },
  });
}

负索引数组

javascript 
function negativeArray(arr) {
  return new Proxy(arr, {
    get(t, k, r) {
      const n = Number(k);
      if (Number.isInteger(n) && n < 0) return t[t.length + n];
      return Reflect.get(t, k, r);
    },
  });
}

const a = negativeArray([1, 2, 3, 4]);
a[-1];     // 4
a[-2];     // 3
a[0];      // 1
a.length;  // 4
a.push(5); // ✓

函数 Proxy

javascript 
// target 是函数时，apply 和 construct trap 可用
function timed(fn) {
  return new Proxy(fn, {
    apply(t, thisArg, args) {
      const start = performance.now();
      try { return Reflect.apply(t, thisArg, args); }
      finally { console.log(`${t.name} took ${performance.now() - start}ms`); }
    },
  });
}

const slowCompute = timed(function compute(n) { /* ... */ });
slowCompute(1000);           // "compute took 12ms"

简易 Membrane

javascript 
// Membrane：把一组对象"封装"在边界内，出入必须经过转换
// 这里演示最简版：所有读取的属性也被包装成 proxy
function membrane(root) {
  const cache = new WeakMap();
  function wrap(obj) {
    if (obj === null || typeof obj !== "object") return obj;
    if (cache.has(obj)) return cache.get(obj);
    const p = new Proxy(obj, {
      get: (t, k, r) => wrap(Reflect.get(t, k, r)),
    });
    cache.set(obj, p);
    return p;
  }
  return wrap(root);
}

const safe = membrane({ nested: { x: 1 } });
safe.nested.x;         // 1
// 所有嵌套对象都是独立 proxy，外部代码拿不到原引用

05 常见陷阱

忘记传 receiver，继承 getter 失效

get(t, k) { return t[k]; } 比 Reflect.get(t, k, r) 少了 receiver 参数，原型链上的 getter 里 this 会是 target 而不是 proxy。Vue 的 effect 就收不到 getter 里读的依赖。永远用 Reflect。

私有字段穿透 Proxy 的问题

new Proxy(new Foo(), {}) 然后调用 proxy.method() 里如果访问 this.#x —— ES2022 前抛 TypeError。现代引擎修复了大多数场景，但 "trap 改了 target" 的场景仍可能失败。Vue 3 的 reactive 用了 markRaw 机制跳过这种对象。

JSON.stringify 调用 proxy 的 toJSON

JSON.stringify(proxy) 会触发 get trap 去拿 toJSON 属性。如果你的 get 总是返回值（不检查属性存在），会得到意外的 toJSON 函数。用 has + Reflect.has 精确判断。

Proxy 没有内部存储，修改都转发到 target

proxy.x = 1 会改 target.x = 1。如果 target 被外部还能拿到 → 旁路访问破坏封装。想彻底隔离 → 用 WeakMap 做外部存储，或用 revocable Proxy + 不暴露 target。

循环里大量创建 Proxy

每次 reactive(obj) 都创建 Proxy 对象。Vue 的实现用 WeakMap<raw, Proxy> 缓存 —— 同一对象共享同一 proxy（也是 reactive(reactive(a)) === reactive(a) 的原因）。自己写响应式时别忘了这层缓存。

Proxy 改变了 typeof 和 instanceof？

typeof proxy 是 "object" 或 "function"（取决于 target）。proxy instanceof Klass 走原型链查找 —— 因为 Proxy 的 [[GetPrototypeOf]] 默认转发到 target，所以行为和 target 一致。但如果 handler 里覆盖了 getPrototypeOf，就能"伪装"成别的类型。

invariant 违反的 TypeError

例如：target 是 Object.freeze(obj)，你的 get trap 返回不同值 → TypeError。规范保证"冻结不可伪造"。不常见，但 Proxy + frozen target 场景要知道。

Proxy 对 Map / Set / Date 等"内部 slot"对象不透明

new Proxy(new Map(), {}).get(k) 抛 TypeError —— 因为 Map.prototype.get 需要 this 有内部 slot [[MapData]]，Proxy 没有。解决：在 handler 的 get trap 里判断方法名，把 this 换回 target：

javascript 
get(t, k) {
  const v = Reflect.get(t, k);
  return typeof v === "function" ? v.bind(t) : v;
}
