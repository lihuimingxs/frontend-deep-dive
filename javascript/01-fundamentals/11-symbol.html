Symbol 全景

Symbol 不是给业务代码直接用的 "ID 生成器"，而是 JS 给语言本身留的"协议钩子"。想让自定义类像数组一样被 for-of、像数字一样被 +、像某个 class 一样被 instanceof ——全靠 well-known symbols。本章把 Symbol 的设计哲学与 15 个内置 symbol 的实战用途讲清楚。

阅读时长 ~20 分钟 前置 01 数据类型（Symbol 基础） 对应 javascript.info Symbol type

01 速览

javascript 
// 创建：每次调用都全新、不等
const s1 = Symbol();
const s2 = Symbol("debug-tag");     // 描述字符串，仅用于 debug
s2.toString();                       // "Symbol(debug-tag)"
s2.description;                      // "debug-tag"
Symbol() === Symbol();               // false

// ⚠ 不能 new
new Symbol();                        // TypeError

// 全局注册表：同名复用同一个 Symbol
const a = Symbol.for("app.id");
const b = Symbol.for("app.id");
a === b;                             // true
Symbol.keyFor(a);                    // "app.id"  （反查）

// 作为对象 key
const PRIV = Symbol("private");
obj[PRIV] = "secret";
obj[PRIV];                           // "secret"

// 对 Object.keys / for-in / JSON.stringify 不可见
Object.keys(obj);                    // [...]  （不含 PRIV）
Object.getOwnPropertySymbols(obj);   // [PRIV]

15 个 Well-known symbols

Symbol 协议钩子 典型用途
Symbol.iterator for-of、spread、解构 自定义同步可迭代
Symbol.asyncIterator for await...of 自定义异步流
Symbol.toPrimitive ToPrimitive 强转 自定义 +/== 行为
Symbol.toStringTag Object.prototype.toString 自定义 [object X]
Symbol.hasInstance instanceof 自定义成员判断
Symbol.isConcatSpreadable Array.prototype.concat 控制 concat 是否展开对象
Symbol.species 子类方法的构造函数 让 subArray.map 返回子类
Symbol.match String.match / @@match 自定义 regex 类型
Symbol.matchAll String.matchAll 同上
Symbol.replace String.replace 同上
Symbol.search String.search 同上
Symbol.split String.split 同上
Symbol.unscopables with 作用域 Array.includes/flat 的新 API 兼容旧 with
Symbol.dispose using（ES2024） 自动资源清理（同步）
Symbol.asyncDispose await using 自动资源清理（异步）

02 底层逻辑

唯一身份的实现

每个 Symbol() 调用产生一个新的 Symbol 值，引擎内部给它分配一个唯一 seq_id（类似进程内的原子计数器）。=== 比较 Symbol 就是比较 seq_id。这是"不可伪造 ID"的技术保证 —— 除了拿到引用，没有任何办法构造出相等的 Symbol。

全局注册表：跨 realm 的 Symbol

Symbol() 的唯一性在单个 realm（同一个全局环境）内。iframe、Worker、Node vm 模块各有独立 realm。Symbol.for("key") 使用一个跨 realm 共享的字符串注册表：

javascript 
// 主文档
const main = Symbol.for("my.marker");

// iframe / Worker 里
const iframe = Symbol.for("my.marker");

// 在同一浏览器进程里
main === iframe;               // true（通过 HTML 结构化克隆传递时）

// 只有 Symbol.for 的结果可以通过 postMessage 传递
// Symbol() 创建的本地 Symbol 无法跨 realm（会变成 undefined 或抛错）

用途：跨 iframe 协议标识（"你是不是我认识的那种对象？" 检查），Node.js 的 Symbol.for("nodejs.util.inspect.custom") 就是这种用法。

Well-known symbols 作为协议钩子

JS 早期给内置行为开口只有两个名字约定：toString 和 valueOf。这有两个问题：

  1. 命名冲突 —— 业务代码里也会有 .toString 方法，意图可能完全不同。
  2. 不可扩展 —— 想给 for-of、instanceof 等新特性开口，不能用已有字符串。

ES6 引入 Symbol 后，新协议用 Symbol.xxx 作为 key：for-of 调用 obj[Symbol.iterator]、instanceof 调用 Class[Symbol.hasInstance]…… Symbol key 不会被业务字段撞上，也可以随时增加新协议。

"隐藏性"的真实含义

Symbol key 对 Object.keys、for-in、JSON.stringify、Object.assign（注意：assign 其实会拷贝 Symbol key）等默认"字符串 key 导向"的 API 不可见。但仍然可以通过：

  • Object.getOwnPropertySymbols(obj)
  • Reflect.ownKeys(obj)
  • JSON.stringify 的 replacer（如果显式指定）
  • { ...obj } 展开（会拷贝 Symbol key）

Symbol 不是"加密"。它是"默认不暴露"，拿到引用仍能访问。真私有用 class 的 #field。

Symbol 强制转换的规则

Symbol 是几乎所有强转的"坚硬异物"：

操作 结果
String(Symbol("x")) "Symbol(x)"（显式 String 可行）
`${Symbol("x")}` TypeError（模板字符串拒绝）
Symbol("x") + "" TypeError（+ 的 ToPrimitive 拒绝）
Number(Symbol()) TypeError
Boolean(Symbol()) true（只有 Boolean 强转被允许）

设计意图：强迫调用方显式意识到"这是 Symbol，你要自己决定怎么处理"，避免 Symbol 被意外吞进字符串拼接。

03 应用场景

Symbol.iterator：让自定义类型支持 for-of

javascript 
class Range {
  constructor(start, end, step = 1) {
    this.start = start; this.end = end; this.step = step;
  }
  *[Symbol.iterator]() {               // generator 是 iterable
    for (let i = this.start; i < this.end; i += this.step) yield i;
  }
}

const r = new Range(1, 10, 2);
[...r];                                // [1, 3, 5, 7, 9]
for (const n of r) { }                  // ok
const [a, b] = r;                       // 解构也走这里

Symbol.toPrimitive：接管强转

javascript 
class Money {
  constructor(cents, currency) {
    this.cents = cents;
    this.currency = currency;
  }
  [Symbol.toPrimitive](hint) {
    if (hint === "number") return this.cents / 100;          // 算术
    if (hint === "string") return `${this.cents / 100} ${this.currency}`;
    return this.toJSON();                                     // default
  }
  toJSON() { return `${this.cents / 100} ${this.currency}`; }
}

const price = new Money(12345, "CNY");
+price;                       // 123.45
`${price}`;                   // "123.45 CNY"
price + " (tax included)";    // "123.45 CNY (tax included)"
price > 100;                  // true（比较用 number hint）

Symbol.toStringTag：自定义 Object.prototype.toString

javascript 
class MyList {
  get [Symbol.toStringTag]() { return "MyList"; }
}
Object.prototype.toString.call(new MyList());   // "[object MyList]"

// 对 debug / 日志 / 类型探测很有用
function typeOf(v) {
  return Object.prototype.toString.call(v).slice(8, -1);
}
typeOf(new Map());              // "Map"（内置也这么实现）
typeOf(new MyList());           // "MyList"

Symbol.hasInstance：自定义 instanceof

javascript 
// 鸭子类型的 instanceof
class Even {
  static [Symbol.hasInstance](n) { return typeof n === "number" && n % 2 === 0; }
}
2 instanceof Even;            // true
3 instanceof Even;            // false

// 工程上：让 Rejected 这种 sentinel 在跨 realm 仍能被识别
class AbortErr {
  static [Symbol.hasInstance](e) {
    return e && e.name === "AbortError";
  }
}
caughtError instanceof AbortErr;   // 依 name 判定，不依赖引用

Symbol.dispose：显式资源管理（ES2024）

javascript 
class DBConn {
  async connect() { /* ... */ }
  [Symbol.dispose]() { this.close(); }
  close() { /* ... */ }
}

{
  using conn = new DBConn();            // ES2024 using 声明
  await conn.connect();
  // ... 使用
}  // 离开 block 自动调用 conn[Symbol.dispose]()

// 异步版：await using db = new DB();  离开时调用 [Symbol.asyncDispose]()
// 类比 C# using、Python with、Rust Drop

04 典型代码

用 Symbol 做不可伪造枚举

javascript 
// 常见做法：字符串枚举
const Status = { PENDING: "pending", DONE: "done" };
// 问题：别处写 "pending" 也能匹配，拼写错误编译期抓不到

// Symbol 版：必须拿到引用才能匹配
const Status = {
  PENDING: Symbol("pending"),
  DONE: Symbol("done"),
};
switch (task.status) {
  case Status.PENDING: ...            // ✓
  case "pending": ...                 // 永远不会匹配（Symbol !== string）
}
// 代价：不能 JSON 序列化传输，仅限内部使用

给第三方对象附 Symbol 字段

javascript 
// 给别人的对象加"内部标记"，不干扰它原有的 JSON.stringify
const CACHED = Symbol("cached");

function fetchCached(obj) {
  if (obj[CACHED]) return obj[CACHED];
  return (obj[CACHED] = computeExpensive(obj));
}

JSON.stringify(obj);             // 不包含 CACHED
Object.keys(obj);                // 不包含 CACHED
// 但引用相等仍能访问：fetchCached(obj)

iterable + async iterable 双协议

javascript 
// 既能同步也能异步迭代的数据源
class Queue {
  #items = [];
  push(x) { this.#items.push(x); }
  [Symbol.iterator]() { return this.#items[Symbol.iterator](); }
  async *[Symbol.asyncIterator]() {
    while (this.#items.length) {
      yield this.#items.shift();
      await Promise.resolve();           // 让出给微任务
    }
  }
}

for (const x of q) { ... }               // 同步
for await (const x of q) { ... }         // 异步（逐个处理，让出 CPU）

全局注册表的反查

javascript 
const s1 = Symbol.for("app.v1");
const s2 = Symbol("local");

Symbol.keyFor(s1);        // "app.v1"
Symbol.keyFor(s2);        // undefined   ← 不在注册表里

// 使用场景：跨 iframe 协议版本协商
function isCompatible(sym) {
  const key = Symbol.keyFor(sym);
  return key && key.startsWith("app.v") && parseInt(key.slice(5)) >= 2;
}

05 常见陷阱

new Symbol() 抛错

Symbol 和 BigInt 是仅有的两个"不能 new"的原始类型包装。规范显式规定了这是为了避免它们变成 wrapper 对象（那样就破坏"唯一性"保证）。用 Symbol("desc")。

模板字符串隐式转 Symbol 抛 TypeError

`${sym}`、sym + "" 都抛错。显式转用 String(sym) 或 sym.description。设计意图是防止 Symbol 被意外吞进 UI 字符串。

Symbol.for 的全局注册表泄漏到 iframe

主文档用 Symbol.for("internal")，恶意 iframe 可以 Symbol.for("internal") 拿到同一个 Symbol 并用它访问你挂在对象上的 Symbol 键。敏感数据不要用 Symbol.for 做 key。用本地 Symbol() 安全。

Object.assign 复制 Symbol 键

文档常说 "Symbol key 不可枚举"，但 Object.assign(target, source) 会复制自有 enumerable Symbol key（ES2017 起明确规范）。想保护 Symbol 属性不被意外复制，用 Object.defineProperty 设 enumerable: false。

JSON.stringify 忽略 Symbol 键和值

JSON.stringify({ [Symbol("k")]: 1, x: Symbol("v") }) 是 "{}"（忽略 Symbol 键）还是 '{"x":null}' 呢？实际是 '{}'（ES2017 标准说键忽略）。Symbol 值作为对象属性值会被忽略、作为数组元素会变 null。

Symbol.for(undefined) 返回一个"undefined" key 的 Symbol

Symbol.for(undefined) 把参数 ToString 成 "undefined" 再查注册表。这通常不是意图。对未定义值要显式校验。

Symbol.species 是个罕用坑

子类继承 Array、Promise 时，.map / .then 会调用 this.constructor[Symbol.species] 决定用哪个构造函数。默认是子类本身 —— 可能导致子类方法返回子类实例。TC39 已有提案移除 Symbol.species（太复杂、用的地方少）。
