数字与 Math

Number 背后是 IEEE 754（已在 01 章讲过）；本章专注在操作层面：如何正确取整、如何判空值、Math.* 何时会让你意外、为什么随机数不能直接用于密码学。工程师最常踩的 5 个坑都在这里。

阅读时长 ~18 分钟 前置 01 数据类型（IEEE 754 部分） 对应 javascript.info Numbers

01 速览

02 底层逻辑

isNaN vs Number.isNaN

两个名字几乎一样的函数，行为差别巨大：

javascript 
// 全局 isNaN：先 ToNumber，转不了才判断 NaN
isNaN(NaN);           // true
isNaN("abc");         // true     ← Number("abc") 是 NaN
isNaN(undefined);     // true     ← Number(undefined) 是 NaN
isNaN({});            // true

// Number.isNaN：不做强转，只判"值是不是 NaN"
Number.isNaN(NaN);           // true
Number.isNaN("abc");         // false  ← "abc" 不是 NaN，它是字符串
Number.isNaN(undefined);     // false
Number.isNaN({});            // false

永远用 Number.isNaN（或等价的 x !== x）。全局 isNaN 是 ES5 前的遗留，保留仅为兼容。同理 isFinite / Number.isFinite、parseInt / Number.parseInt。

取整的 5 种方式

javascript 
Math.round(x)    // 四舍五入；⚠ Math.round(-0.5) === 0，不是 -1（规范是"向 +∞ 舍入半值"）
Math.floor(x)    // 向下（-∞）
Math.ceil(x)     // 向上（+∞）
Math.trunc(x)    // 向零截断（去小数部分）
x | 0            // 位运算；仅 32-bit 整数安全（|Math.abs(x)| < 2³¹）
Math.round(1.5);   // 2
Math.round(2.5);   // 3
Math.round(-1.5);  // -1  （非 -2！）
Math.round(-2.5);  // -2  （非 -3！）

Math.trunc(-3.7);  // -3   向零截断
Math.floor(-3.7);  // -4   向 -∞
(3.9 | 0);         // 3    位运算截断
(-3.9 | 0);        // -3   也是向零

工程建议：向下用 Math.floor；去小数用 Math.trunc；位运算 | 0 可读性差且有 32-bit 限制 —— 不再推荐（现代 V8 两者性能接近）。Math.round 的"半值向上"规则与财务四舍五入不同，金额场景需要自己实现银行家舍入（round half to even）。

toFixed 的"浮点舍入"尴尬

javascript 
(1.005).toFixed(2);     // "1.00"  ⚠ 不是 "1.01"
// 原因：1.005 在二进制里实际是 1.00499999...，截两位仍是 1.00

(1.015).toFixed(2);     // "1.01"  ⚠ 不是 "1.02"
(1.25).toFixed(1);      // "1.3"   碰巧对

// 想要精确：先放大成整数，舍入后再缩小
function roundTo(num, decimals) {
  const factor = 10 ** decimals;
  return Math.round(num * factor) / factor;
}
roundTo(1.005, 2);      // 1.01

Math.hypot 为什么值得用

计算向量长度 √(x²+y²) 的朴素做法 Math.sqrt(x*x + y*y) 在 x/y 很大时会溢出（中间结果超 Number.MAX_VALUE 变 Infinity）。Math.hypot 用缩放算法：先除以最大分量再平方和，避免溢出，同时对极小值保留精度。

javascript 
const x = 1e200, y = 1e200;
Math.sqrt(x*x + y*y);       // Infinity  ← 中间溢出
Math.hypot(x, y);           // ≈ 1.414e200  ✓

Math.hypot(3, 4);            // 5
Math.hypot(3, 4, 12);        // 13  （3D 距离）
Math.hypot();                // 0

Math.random 不是密码学安全的

Math.random() 用的是 PRNG（伪随机，V8 使用 xorshift128+），熵来自进程启动时刻。它能预测：

  • 若攻击者能观察连续几个随机数，理论上可以推导内部状态。
  • 任何涉及安全的场景（token、session id、CSRF、密码、密钥）必须用 crypto：
javascript 
// 浏览器 / Node 19+
const buf = new Uint8Array(16);
crypto.getRandomValues(buf);          // 16 字节强随机

// UUID v4（现代 API）
crypto.randomUUID();                  // "a1b2c3d4-..."

// 0-1 的密码学安全浮点（较少直接需要）
function secureRandom() {
  const buf = new Uint32Array(1);
  crypto.getRandomValues(buf);
  return buf[0] / 0xFFFFFFFF;
}

Number() vs parseInt() vs +x

输入 Number(x) +x parseInt(x, 10) parseFloat(x)
"42" 42 42 42 42
"42px" NaN NaN 42 42
"3.14" 3.14 3.14 3 3.14
" 42 " 42 42 42 42
"" 0 0 NaN NaN
null 0 0 NaN NaN
undefined NaN NaN NaN NaN
true 1 1 NaN NaN
[42] 42 42 42 42
[1,2] NaN NaN 1 1

决策：严格数字字符串（表单、API 返回）用 Number() + Number.isFinite 校验。从 CSS "100px" 等解析前缀数字用 parseInt/parseFloat。+x 简短但在长链中可读性差。parseInt 永远显式传 radix，老 JS 对 "08" 按八进制解析会返回 0（ES5+ 修了默认是 10，但仍建议显式）。

03 应用场景

金额用整数存储

用 分 / 厘整数存储金额（DB 字段、API 字段、内存变量），展示时 (amount / 100).toFixed(2)。好处：不依赖浮点、数据库 SUM 精确、相等比较可靠。2⁵³ 分 ≈ 9 万亿元，足够电商业务。超出 → BigInt。

生成 UUID / token

javascript 
// UUID v4（浏览器、Node 19+）
crypto.randomUUID();             // "11111111-2222-4333-8444-555555555555"

// 短 token（Base64 编码 N 字节随机）
function token(bytes = 16) {
  const buf = new Uint8Array(bytes);
  crypto.getRandomValues(buf);
  return btoa(String.fromCharCode(...buf)).replaceAll("=", "").replaceAll("+", "-").replaceAll("/", "_");
}
token();                         // "a1B2c3D..." URL 安全

浮点安全比较

javascript 
// 朴素 === 不行
0.1 + 0.2 === 0.3;                              // false

// 用相对误差
function nearlyEqual(a, b, eps = Number.EPSILON * 8) {
  return Math.abs(a - b) <= eps * Math.max(1, Math.abs(a), Math.abs(b));
}
nearlyEqual(0.1 + 0.2, 0.3);                    // true

// 大数场景需要按相对误差比，不是绝对 EPSILON

本地化显示：Intl.NumberFormat

javascript 
const nf = new Intl.NumberFormat("zh-CN", {
  style: "currency",
  currency: "CNY",
});
nf.format(12345.67);              // "￥12,345.67"

// 百分比
new Intl.NumberFormat("zh", { style: "percent", maximumFractionDigits: 1 }).format(0.1234);
// "12.3%"

// 紧凑
new Intl.NumberFormat("en", { notation: "compact" }).format(12_345);   // "12K"
new Intl.NumberFormat("zh", { notation: "compact" }).format(12_345);   // "1.2万"

循环里反复 new Intl.NumberFormat 有显著开销，应该在模块顶层创建一次复用。

大整数 ID：BigInt 还是字符串？

后端 int64 ID 超 2⁵³ 时前端接收：

  • 仅作标识符（不参与算术）—— 用字符串最简单，API 层序列化为字符串。
  • 需要算术/比较（如分页游标计算）—— 用 BigInt，但要注意 JSON.stringify 不支持 BigInt，需要自定义 replacer。
  • 库支持：Mongoose、Prisma 等 ORM 都有对应选项。

04 典型代码

区间随机

javascript 
// 整数 [min, max] 均匀
function randInt(min, max) {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}
randInt(1, 6);         // 1 - 6（骰子）

// 数组随机元素
function randPick(arr) { return arr[Math.floor(Math.random() * arr.length)]; }

// 洗牌（Fisher-Yates）
function shuffle(arr) {
  const a = [...arr];
  for (let i = a.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [a[i], a[j]] = [a[j], a[i]];
  }
  return a;
}

Math.max / min 配合 spread

javascript 
const nums = [3, 1, 4, 1, 5, 9, 2, 6];
Math.max(...nums);           // 9
Math.min(...nums);           // 1

// ⚠ 不能传数组
Math.max([1, 2, 3]);         // NaN  （数组被 ToNumber → NaN）
Math.max();                  // -Infinity  ← 空参数时！
Math.min();                  // Infinity

// 数组很大（> 100k）会栈溢出，用 reduce
function maxSafe(arr) { return arr.reduce((m, v) => v > m ? v : m, -Infinity); }

钳位到区间

javascript 
const clamp = (x, lo, hi) => Math.min(Math.max(x, lo), hi);

clamp(5, 0, 10);       // 5
clamp(-5, 0, 10);      // 0
clamp(50, 0, 10);      // 10

// 进度条、音量、颜色通道、游戏边界 —— 都用得上

进制转换

javascript 
// 数字 → 字符串
(255).toString(16);           // "ff"
(255).toString(2);            // "11111111"
(255).toString(36);           // "6r"     （常用于短 ID）

// 字符串 → 数字
parseInt("ff", 16);           // 255
parseInt("11111111", 2);      // 255
Number("0xff");               // 255      （自动识别 0x 前缀）
Number("0b11111111");         // 255      （ES6+ 二进制前缀）
Number("0o777");              // 511      （八进制前缀）

格式化工具集

javascript 
// 千分位
(1234567).toLocaleString();                    // "1,234,567"（按浏览器 locale）

// 千分位 + 固定 2 位小数
(1234.5).toLocaleString("en-US", {
  minimumFractionDigits: 2, maximumFractionDigits: 2,
});                                             // "1,234.50"

// 补零
String(5).padStart(2, "0");                    // "05"
(5).toString().padStart(4, "0");               // "0005"

05 常见陷阱

Math.max() 空参返回 -Infinity

Math.max(...[]) 是 -Infinity。在筛选后数组可能为空的场景，忘记校验会把 -Infinity 写进 UI 或 DB。Math.max(0, ...arr) 或 arr.length ? Math.max(...arr) : 0。

Math.round(-0.5) === 0 不是 -1

规范是"半值向 +∞ 舍入"：-0.5 到 0 比到 -1 近（算上符号）。要对称取整用 Math.sign(x) * Math.round(Math.abs(x))，或者用 Math.trunc + 业务条件。

toFixed 的浮点舍入不"数学直觉"

(1.005).toFixed(2) 是 "1.00"，因为 1.005 的二进制不是精确值。金融计算要么整数化，要么用 decimal.js。见上面 roundTo 工具函数作权宜。

parseInt 未传 radix 的历史坑

ES5+ 默认 radix 10（不再按 "0x" 前缀当 16），但 "08"/"09" 在老引擎按八进制返回 0。强制写 parseInt(x, 10)，ESLint radix 规则能检测。

Number("") 是 0，Number(null) 也是 0

这两个是 JS 最反直觉的强转之一。表单校验 Number(input) === 0 不能区分"用户输入 0"和"用户没输"。用 Number.parseFloat 或先 .trim() 判空。

Math.random 不能用于 token / UUID

生成 session ID、防重放 token、密码盐 —— 必须 crypto.getRandomValues 或 crypto.randomUUID。Math.random 的输出在某些 V8 版本上连续调用可预测。

BigInt 和 Number 不可混算

1n + 1 抛 TypeError，必须 1n + 1n 或 Number(1n) + 1。Math.max(1n, 2n) 也不工作（Math 不支持 BigInt）。手写 a > b ? a : b。

x % n 对负数的行为

-1 % 3 === -1，不是 2。需要"真正的模运算"（总是正结果）：((x % n) + n) % n。常用于循环数组索引：arr[(i - 1 + arr.length) % arr.length]。
