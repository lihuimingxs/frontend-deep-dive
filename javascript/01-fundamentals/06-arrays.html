数组深入

JS 数组是"键为数字字符串"的特殊对象。方法多到令人发怵，但按"是否修改原数组"和"返回什么"两个维度分组后，规律清晰。本章把全部方法放进决策表，再讲三个工程师最易踩的点：稀疏数组、sort 默认比较、length 可写。

阅读时长 ~25 分钟 前置 01 - 05 对应 javascript.info Arrays; Array methods

01 速览

按"是否修改原数组"和"返回值类型"分组的完整方法表：

分类 方法 修改原数组？ 返回值
增删 push / pop ✓（尾端） 新长度 / 被弹元素
unshift / shift ✓（头端，慢 O(n)） 新长度 / 被弹元素
splice(start, delCount, ...add) ✓（任意位置） 被删除的元素数组
fill(value, start?, end?) ✓ 原数组
排序/反转 sort(cmp?) ✓ 原数组
reverse() ✓ 原数组
toSorted / toReversed / toSpliced / with ✗（ES2023 新，非破坏版） 新数组
切片/拼接 slice(start?, end?) ✗ 新数组（浅拷）
concat(...) ✗ 新数组
flat(depth?) / flatMap(fn) ✗ 新数组
查询 indexOf / lastIndexOf ✗ 索引（=== 比较）
includes ✗ boolean（SameValueZero，能找 NaN）

02 底层逻辑

数组是特殊对象

规范上数组是 exotic object：键都是 字符串化的非负整数（"0", "1", ...），但引擎对小整数键有专门的快路径存储。length 是个特殊属性 —— 设置 length = 3 会截断数组；arr[100] = 1 会把 length 自动变 101（中间 99 个空位）。

javascript 
const a = [1, 2, 3];
a.length = 1;                  // a 变成 [1]（尾部被截）
a.length = 5;                  // a 变成 [1, <4 empty>]（填充空位）
a[100] = "x";                  // length 变 101, 稀疏数组
a.length;                      // 101

// 用 Object 方法能看出是字典
Object.keys(a);                // ["0", "100"]  ← 空位不在 keys 里

V8 的 3 种 elements kind

V8 对数组元素存储分 6 种类型，简化后三档：

  • PACKED_SMI_ELEMENTS —— 纯 Small Integer，无空位。最快。
  • PACKED_DOUBLE_ELEMENTS —— 有浮点数但无空位。
  • PACKED_ELEMENTS —— 混合类型，无空位。
  • HOLEY_* —— 同上三档，但有空位。慢一个数量级，因为每次访问要查 prototype chain。

elements kind 只能"降级"（PACKED → HOLEY、SMI → DOUBLE → 任意），不能升回。一旦引入空位就永远是 HOLEY，即使你后来填上了值。

⚠ 工程启示

避免 new Array(1000)（造出 1000 个 hole）、arr[999] = x（让 arr 变稀疏）、delete arr[i]（留下空位）。热点数组循环里，用 arr.length = 0 清空（保持 PACKED），用 arr.push(null) 占位（避免空位）。

sort 的默认比较器

arr.sort() 不传比较器时，所有元素被转成字符串后按 Unicode code unit 排序。这就是为什么 [10, 2, 1].sort() === [1, 10, 2]（"10" < "2"）。

javascript 
[10, 2, 1].sort();                     // [1, 10, 2]  ⚠ 按字符串
[10, 2, 1].sort((a, b) => a - b);      // [1, 2, 10]  ✓
[10, 2, 1].sort((a, b) => b - a);      // [10, 2, 1]  降序

// 字符串排序 —— 中文/欧洲语言需要 Intl.Collator
const cn = ["张三", "李四", "王五"];
cn.sort();                                       // Unicode 顺序（不自然）
cn.sort(new Intl.Collator("zh").compare);        // 拼音顺序（自然）

// 多级排序
users.sort((a, b) =>
  a.lastName.localeCompare(b.lastName) ||
  a.firstName.localeCompare(b.firstName) ||
  a.age - b.age
);

稳定性：V8 / SpiderMonkey / JavaScriptCore 从 2019 起都保证稳定排序（ES2019 规范要求）。可以依赖：相等元素保持原顺序。

稀疏数组 vs undefined 的区别

[1, , 3] 的中间是"空位"（empty slot），不同于 [1, undefined, 3]：

操作 [1, , 3] [1, undefined, 3]
.length 3 3
arr[1] undefined（读取时） undefined
1 in arr false true
.forEach / .map / .filter 跳过空位 处理 undefined
.for-of / .spread [...] 产出 undefined 产出 undefined
Array.from(arr) 填充 undefined（去稀疏化） 原样
JSON.stringify "[1,null,3]" "[1,null,3]"

Array-like vs Iterable

Array-like：有 length + 数字键。例：arguments, NodeList, HTMLCollection, { 0: "a", length: 1 }。Iterable：实现 Symbol.iterator。例：Set, Map, generator 返回值，字符串。

Array.from 兼容两者：Array.from(nodeList)、Array.from(set)、Array.from({length: 3}, (_, i) => i * 2) → [0, 2, 4]。[...x] 只支持 Iterable。

TypedArray：二进制数据视图

Int8Array / Uint8Array / Uint8ClampedArray / Int16Array / Uint16Array / Int32Array / Uint32Array / Float32Array / Float64Array / BigInt64Array / BigUint64Array 共 11 种，都是 ArrayBuffer 上的视图：

  • 元素类型固定，赋值会做截断 / 饱和。
  • 长度不可变（创建时确定）。
  • 支持数组方法（map/filter/forEach...），但 .map 返回同类 TypedArray，不是 Array。
  • 能和 ArrayBuffer 共享底层内存（WebGL、Audio、Wasm、二进制协议）。

详细在 P7 二进制数据章节。

03 应用场景

Map / Filter / Reduce 链：数据处理的主力

javascript 
// 订单 → 按用户聚合总金额，只保留前 10 大客户
const top10 = orders
  .filter(o => o.status === "paid")
  .reduce((acc, o) => {
    acc[o.userId] = (acc[o.userId] ?? 0) + o.amount;
    return acc;
  }, {});
const sorted = Object.entries(top10)
  .sort(([, a], [, b]) => b - a)
  .slice(0, 10);

链式可读性好但有性能开销：每步产生中间数组。大规模数据（10万+）用单次 for-of / reduce 合并逻辑。

分组：Object.groupBy（ES2024）

javascript 
const byCategory = Object.groupBy(products, p => p.category);
// { electronics: [...], books: [...], ... }

// 用 Map 保留插入顺序和非字符串 key
const byMonth = Map.groupBy(orders, o => o.date.getMonth());

// Baseline：需要 Node 21+ / 现代浏览器，polyfill 简单

去重：三种工具

javascript 
// 1. 原始值去重（最快、最短）
[...new Set([1, 2, 2, 3])];                  // [1, 2, 3]

// 2. 对象按字段去重
const seen = new Set();
users.filter(u => !seen.has(u.id) && seen.add(u.id));

// 3. 保留最后出现的（reduce + Map）
const byId = new Map(items.map(i => [i.id, i]));
const deduped = [...byId.values()];

非破坏更新：ES2023 的新 API

javascript 
// 旧：展开 + 方法
const sorted = [...arr].sort();                      // 需要先拷贝
const replaced = arr.map((v, i) => i === idx ? x : v);

// 新：toXxx 方法天然非破坏
const sorted = arr.toSorted();                       // 不改 arr
const replaced = arr.with(idx, x);                   // 不改 arr
const reversed = arr.toReversed();
const spliced = arr.toSpliced(1, 2, "a", "b");

// Redux reducer 写起来更干净

TypedArray：WebGL / 二进制协议

WebGL buffer、Canvas ImageData、Web Audio 的 PCM 数据、WebSocket 二进制帧、fetch 的 arrayBuffer() —— 都是 TypedArray。Uint8Array 做图像像素的按位运算比 Array 快 20-50 倍。

04 典型代码

构造数组的 7 种方式

javascript 
[1, 2, 3]                                     // 字面量
new Array(3)                                  // [empty × 3]  ⚠ 稀疏
Array.of(1, 2, 3)                             // [1, 2, 3]    明确值列表
Array.from({ length: 3 })                     // [undefined × 3]
Array.from({ length: 3 }, (_, i) => i * 2)    // [0, 2, 4]   with mapper
Array(3).fill(0)                              // [0, 0, 0]   先 hole 再填
[...Array(3).keys()]                          // [0, 1, 2]   序列

// 推荐：Array.from 最通用、最易读
Array.from({ length: 10 }, (_, i) => i);

reduce 的典型模式

javascript 
// 求和
[1, 2, 3].reduce((s, x) => s + x, 0);               // 6

// 转成 lookup 对象
items.reduce((acc, item) => ({ ...acc, [item.id]: item }), {});
// ⚠ 每次 spread 是 O(n)，整体 O(n²)！大数据用 Map 或 mutable 更新
items.reduce((acc, item) => { acc[item.id] = item; return acc; }, {});

// 分组
orders.reduce((acc, o) => {
  (acc[o.userId] ??= []).push(o);
  return acc;
}, {});

// 并行累积多个
const { sum, max, count } = nums.reduce(
  (s, n) => ({ sum: s.sum + n, max: Math.max(s.max, n), count: s.count + 1 }),
  { sum: 0, max: -Infinity, count: 0 },
);

flat 与 flatMap 的关系

javascript 
[[1, 2], [3, 4]].flat();                     // [1, 2, 3, 4]
[[1, [2, [3]]]].flat(2);                     // [1, 2, [3]]
[[1, 2]].flat(Infinity);                     // 无限深度

// flatMap = map + flat(1)，常用于一对多映射
sentences.flatMap(s => s.split(" "));        // 把每句拆成词再合并

// 返回空数组来删除元素
users.flatMap(u => u.active ? [u.name] : []); // 过滤 + 映射一步完成

at 替代 arr[arr.length - 1]

javascript 
const arr = [10, 20, 30];
arr[arr.length - 1];        // 30  老写法
arr.at(-1);                 // 30  新写法（ES2022）
arr.at(-2);                 // 20
arr.at(0);                  // 10
arr.at(99);                 // undefined（不是抛错）
arr.at(-99);                // undefined

// 字符串、TypedArray 也有 .at
"hello".at(-1);             // "o"

Array-like → Array

javascript 
// DOM NodeList 转数组（可 map/filter）
const items = Array.from(document.querySelectorAll("li"));
const items2 = [...document.querySelectorAll("li")];    // NodeList 是 iterable，也行

// arguments（非箭头函数内）
function sum() { return Array.from(arguments).reduce((s, x) => s + x, 0); }

// 用 rest 参数代替 arguments（更干净）
function sum(...nums) { return nums.reduce((s, x) => s + x, 0); }

// Set / Map 转数组
[...new Set([1, 2, 3])];
[...new Map([["a", 1]])];                          // [["a", 1]]
[...new Map().keys()];                             // 仅 keys

05 常见陷阱

sort 默认按字符串

[10, 2, 1].sort() → [1, 10, 2]。数字排序必须传 (a, b) => a - b。中文用 Intl.Collator，否则是 Unicode 顺序。

sort / reverse / splice 是原地修改

React 里直接 state.sort() 不会触发重渲染（引用未变），且污染了状态。用 toSorted 或 [...state].sort()。

new Array(3) 造稀疏数组

new Array(3) 产生 [empty × 3]，.map 跳过空位（返回 [empty × 3]）。要填充用 new Array(3).fill(null).map(...) 或 Array.from({length: 3}, mapper)。

fill 同一引用的共享问题

new Array(3).fill([]) 得到 3 个指向同一数组的引用。arr[0].push(1) 会让 arr[1] === arr[2] === [1]。需要独立用 Array.from({length:3}, () => [])。

indexOf 找不到 NaN

[NaN].indexOf(NaN) 返回 -1（NaN === NaN false）。[NaN].includes(NaN) 返回 true（SameValueZero 算法）。判空查询用 includes。

length 可写能截断 / 扩长

arr.length = 0 是清空数组的地道写法（比 arr.splice(0) 快，比重新赋值 arr = [] 能保持引用共享）。arr.length = 100 造出 100 个空位。

for-in 数组是灾难

for-in 返回字符串索引、可能包含原型链上的属性、顺序不保证（现在是保证了但仍然不自然）。数组遍历一律用 for-of、.forEach、或经典 for (let i = 0; i < arr.length; i++)。

数组相等比较就是 ===

[1,2,3] === [1,2,3] 是 false（不同引用）。结构相等用 JSON.stringify 比较（限 POJO）、fast-deep-equal 或 Node.js 的 util.isDeepStrictEqual。

splice 的参数顺序

splice(start, deleteCount, ...insert) —— 第二参是删除数量不是结束位置。跟 slice(start, end) 的语义不一样，极容易混。记忆口诀：slice 切块不伤身，splice 操刀是动手术。
