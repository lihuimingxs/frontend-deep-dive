迭代器与 Generator

迭代协议是 JS 最"低调但关键"的扩展点：for-of、...、解构、Array.from、Map/Set 构造都走它。Generator 是一种"可暂停函数"，用最少的代码实现懒序列、协程、双向通信。理解这一章是理解 P3 异步的基础（async/await 等价于 Promise + Generator）。

阅读时长 ~25 分钟 前置 11 Symbol · 04 函数 对应 javascript.info Iterables; Generators

01 速览

三个独立但互相协作的概念：

概念 定义 例子
iterable 有 [Symbol.iterator]() 方法的对象 Array, String, Map, Set, generator 返回值, arguments, NodeList
iterator 有 next() 方法、返回 {value, done} 的对象 上面 [Symbol.iterator]() 的返回值
generator function* 返回的"可暂停函数实例"；同时是 iterable 和 iterator function* g(){ yield 1; } 的 g()
javascript 
// 协议形式
const iterable = {
  [Symbol.iterator]() {                  // 返回 iterator
    let i = 0;
    return {
      next() {                            // iterator 方法
        return i < 3
          ? { value: i++, done: false }
          : { value: undefined, done: true };
      },
    };
  },
};
for (const v of iterable) console.log(v);    // 0, 1, 2

// Generator 形式（等价）
function* gen() {
  yield 0;
  yield 1;
  yield 2;
}
for (const v of gen()) console.log(v);       // 0, 1, 2

02 底层逻辑

迭代协议的规范细节

规范定义的 iterator 接口有 3 个方法：

  • next(value?) — 必需，返回 { value, done }。
  • return(value?) — 可选，外部告诉 iterator "我提前退出了"；返回 { value, done: true }。用于 for-of break / 异常时的清理。
  • throw(error?) — 可选，外部向 iterator 注入异常；主要用于 generator。

for-of 的规范执行流程

for (const x of iter) { body } 在规范层面等价：

javascript 
const iterator = iter[Symbol.iterator]();
while (true) {
  const { value, done } = iterator.next();
  if (done) break;
  try {
    const x = value;
    /* body */
  } catch (err) {
    iterator.return?.();          // 异常时调用 return 清理
    throw err;
  }
}
// break 时也会调用 iterator.return?.()

关键启示：for-of 在 break / return / throw 退出时都会调用 iterator 的 return()，这是 generator 能做"资源清理"的依据（try/finally 里的 finally 会在 return 被调用时执行）。

Generator 的双重身份

function* 返回的对象同时是 iterable 和 iterator —— 它的 [Symbol.iterator]() 返回自己。这意味着：

  • 可以直接 for-of、spread、解构。
  • 但只能迭代一次（同一对象的 iterator，迭代完 done: true 就结束）。
  • 要重新迭代，必须重新调用生成器函数：gen() 再产生新实例。

Generator 的执行模型

javascript 
function* gen() {
  console.log("a");
  const x = yield 1;          // 暂停，把 1 抛给外部；等下次 next(arg) 时 x = arg
  console.log("b", x);
  const y = yield 2;
  console.log("c", y);
  return 3;                   // 最后一次 next 拿到 {value: 3, done: true}
}

const g = gen();
g.next();                     // "a"; { value: 1, done: false }
g.next("foo");                // "b foo"; { value: 2, done: false }  ← 参数填进 x
g.next("bar");                // "c bar"; { value: 3, done: true }
g.next();                     // { value: undefined, done: true }

"yield" 是双向的：它既是"往外抛出一个值"，也是"等外部传入一个值"的接口。第一次 next() 的参数被忽略（因为第一个 yield 前没有"停在"某个 yield 等输入）。

yield* 委托

yield* 把执行权完全转给另一个 iterable，把它产生的所有值都转发出去：

javascript 
function* inner() { yield "a"; yield "b"; }
function* outer() {
  yield 1;
  yield* inner();              // 把 inner 的所有 yield 转发出来
  yield* [2, 3];               // 任何 iterable 都行
  yield 4;
}
[...outer()];                  // [1, "a", "b", 2, 3, 4]

异步迭代器

Symbol.asyncIterator + for await of 处理异步数据流（逐项处理，不等全部完成）：

javascript 
async function* fetchPages(baseUrl) {
  let page = 1;
  while (true) {
    const res = await fetch(`${baseUrl}?page=${page}`);
    const { items, hasMore } = await res.json();
    for (const item of items) yield item;      // 逐项产出
    if (!hasMore) return;
    page++;
  }
}

for await (const item of fetchPages("/api")) {
  console.log(item);                           // 实时处理，不等全部
  if (item.isTarget) break;                    // 可随时退出，后续不再请求
}

03 应用场景

无限序列 / 懒计算

javascript 
function* naturals() {
  let n = 1;
  while (true) yield n++;         // "无限"序列，不会真无限（按需取）
}

// 取前 10 个
function* take(iter, n) {
  for (const v of iter) {
    if (n-- <= 0) return;
    yield v;
  }
}
[...take(naturals(), 10)];         // [1..10]

// 组合：斐波那契前 N 项
function* fib() {
  let [a, b] = [0, 1];
  while (true) { yield a; [a, b] = [b, a + b]; }
}
[...take(fib(), 8)];               // [0, 1, 1, 2, 3, 5, 8, 13]

树遍历（深度优先 / 广度优先）

javascript 
function* walkDFS(node) {
  yield node;
  for (const child of node.children ?? []) yield* walkDFS(child);
}

function* walkBFS(root) {
  const queue = [root];
  while (queue.length) {
    const node = queue.shift();
    yield node;
    queue.push(...(node.children ?? []));
  }
}

// 用起来像数组
for (const node of walkDFS(tree)) {
  if (node.id === target) return node;
}

数据管道：map / filter / take 组合

javascript 
function* gMap(iter, fn)    { for (const v of iter) yield fn(v); }
function* gFilter(iter, fn) { for (const v of iter) if (fn(v)) yield v; }
function* gTake(iter, n)    { for (const v of iter) { if (n-- <= 0) return; yield v; } }

// 组合管道 —— 惰性执行，不产生中间数组
const result = [...gTake(gFilter(gMap(naturals(), n => n * n), n => n % 2), 5)];
// [1, 9, 25, 49, 81]   （前 5 个奇数平方）

// ES2025 Iterator Helpers 原生支持（Chrome 122+）
// Iterator.from(naturals()).map(n => n * n).filter(n => n % 2).take(5).toArray();

异步分页流

见上面 fetchPages。对比"一次拉全部"的方式：按需获取、可随时 break、内存占用恒定。用于爬虫、日志导出、大批量数据处理。

redux-saga 的协程模式

redux-saga 用 generator 把"异步流程"写成同步的样子：yield call(api)、yield put(action)。中间件接到 yield 的 effect 对象后真正执行，结果再 next(result) 回灌。这是 generator 双向通信的经典用法。

资源清理（try / finally）

javascript 
async function* readFileLines(path) {
  const handle = await fs.open(path);
  try {
    for await (const line of handle.readLines()) yield line;
  } finally {
    await handle.close();              // for-of break 时也会执行
  }
}

for await (const line of readFileLines("log.txt")) {
  if (line.includes("ERROR")) break;   // 自动触发 finally，文件被关闭
}

04 典型代码

实现自定义 iterable

javascript 
// 方式 1：手写 iterator 对象
class Range {
  constructor(start, end, step = 1) { Object.assign(this, { start, end, step }); }
  [Symbol.iterator]() {
    let i = this.start;
    const { end, step } = this;
    return {
      next: () => i < end
        ? { value: (i += step) - step, done: false }
        : { value: undefined, done: true },
    };
  }
}

// 方式 2：用 generator（推荐，简洁）
class Range2 {
  constructor(start, end, step = 1) { Object.assign(this, { start, end, step }); }
  *[Symbol.iterator]() {
    for (let i = this.start; i < this.end; i += this.step) yield i;
  }
}

[...new Range2(1, 10, 2)];      // [1, 3, 5, 7, 9]

Generator 的双向通信

javascript 
// 日志聚合器：外部传日志，内部按条件汇总
function* aggregator() {
  let sum = 0, count = 0;
  while (true) {
    const n = yield { sum, count, avg: count ? sum / count : 0 };
    if (n === "reset") { sum = 0; count = 0; }
    else { sum += n; count++; }
  }
}

const agg = aggregator();
agg.next();                     // 启动到第一个 yield
agg.next(10);                   // { sum: 10, count: 1, avg: 10 }
agg.next(20);                   // { sum: 30, count: 2, avg: 15 }
agg.next("reset");              // { sum: 0, count: 0, avg: 0 }

ES2025 Iterator Helpers

javascript 
// Chrome 122+ / Node 22+：原生迭代器方法链
naturals()
  .filter(n => n % 2)
  .map(n => n * n)
  .take(5)
  .toArray();                   // [1, 9, 25, 49, 81]

// AsyncIterator 也有：
await fetchPages("/api")
  .filter(x => x.active)
  .take(100)
  .toArray();

// 老浏览器用 core-js polyfill，或用 itertools-ts 等库

外部提前终止

javascript 
function* gen() {
  try {
    yield 1;
    yield 2;
    yield 3;
  } finally {
    console.log("cleanup");     // 会被调用
  }
}

const g = gen();
g.next();                       // { value: 1, done: false }
g.return("done");               // "cleanup"; { value: "done", done: true }
g.next();                       // { value: undefined, done: true }   已结束

限并发的异步 iterator 消费

javascript 
// 从 async iterable 逐项取，最多 N 个并发
async function* mapConcurrent(source, fn, concurrency = 3) {
  const pool = new Set();
  for await (const item of source) {
    const p = Promise.resolve(fn(item)).then(v => {
      pool.delete(p);
      return v;
    });
    pool.add(p);
    if (pool.size >= concurrency) yield await Promise.race(pool);
  }
  yield* [...pool];             // 消费剩余
}

05 常见陷阱

Iterator 只能迭代一次

const it = arr[Symbol.iterator](); [...it]; [...it]; — 第二次是空数组。generator 同理。要"重来"必须重新获取：[...arr] 或 gen()。

第一次 next(x) 的参数被忽略

第一次 next 只负责"启动到第一个 yield"，还没有"pause in yield 等输入"的状态。想传入初始值用构造函数参数：function* gen(init) { ... }; gen(init).next();。

Generator 函数不能写成箭头

const g = () => { yield 1; } 语法错误。必须 function*。class 方法里可以 *methodName() { yield 1; }。

Symbol.iterator 和 Symbol.asyncIterator 独立

对象同时实现两者才能在 for-of 和 for await of 里都工作。单独实现其中一个，另一种循环会直接跳过或报错（取决于情况）。

for await of 可以接同步 iterable

for await (const x of [1,2,3]) 合法（每次把 value 包成 resolved Promise）。反过来 for-of 接 async iterable 是错的 —— 会拿到 Promise 对象本身，不 await。

async generator 的 return() / throw() 是 async

返回 Promise，必须 await 才知道清理完成。for await of break 时规范保证 await 返回的 Promise。

Spread 在对象字面量不走 iterable

{...arr} 不等于 [...arr]。对象 spread 走 "自有可枚举属性"，数组 spread 走 iterable 协议。后者能 spread iterator / Map / Set，前者不能。

Generator 内 this 的行为

在 class 方法里写 *methodName()，this 正常。直接 function* gen() 调用时 this 规则和普通函数一样（严格模式 undefined）。很少需要在 generator 里用 this —— 一般用闭包。

Array.from(iter) 不调用 map 参数，直到迭代完

Array.from({length: Infinity}, (_, i) => i) 会死循环。Array.from 是"立即消费全部"，不做懒计算。需要懒 → 用 generator。
