Map / Set / WeakMap / WeakSet

ES6 加入的 4 个集合类型修复了 Object/Array 作为"集合"的历史缺陷：任意 key 类型、保序迭代、O(1) 查询、NaN 查得到、对象作 key 时的 GC 自动清理。本章的关键是：什么时候该用它们而不是 Object/Array。

阅读时长 ~20 分钟 前置 05 对象 · 06 数组 对应 javascript.info Map and Set; WeakMap and WeakSet

01 速览

4 种集合类型的关键差异

key 类型 迭代顺序 大小 GC 语义 典型用途
Map 任意 插入序 .size 强引用 字典 / 缓存 / 保序记录
Set 任意（值） 插入序 .size 强引用 去重 / 成员检查
WeakMap 仅对象（含 Symbol） 不可迭代 不可查 弱引用；GC 自动清理 对象元数据 / 私有字段
WeakSet 仅对象 不可迭代 不可查 弱引用 对象的"打过标记"集合

Map / Set 的 API

javascript 
// Map
const m = new Map();
m.set(key, value);           // 可链式：m.set(...).set(...)
m.get(key);                  // 未命中返回 undefined
m.has(key);                  // boolean
m.delete(key);               // 成功返回 true
m.clear();
m.size;                      // 属性，不是 .size()
m.keys();    m.values();    m.entries();   // 都是迭代器
for (const [k, v] of m) { }   // 等价 m.entries()

// 构造：数组的数组
new Map([["a", 1], ["b", 2]]);
new Map(Object.entries(obj));   // Object → Map

// Set
const s = new Set();
s.add(v);                    // 可链式
s.has(v);
s.delete(v);
s.size;
for (const v of s) { }       // 按插入序

// 构造
new Set([1, 2, 2, 3]);       // → Set(1, 2, 3) 去重

02 底层逻辑

Map vs Object：何时该选 Map

维度 Object Map
key 类型 字符串 / Symbol 任意（含对象、NaN、-0）
迭代顺序 整数键升序 → 其他插入序 插入序（单一规则）
原型污染 有风险（__proto__、toString） 无
大小 Object.keys(o).length（O(n)） .size（O(1)）
序列化 JSON.stringify 直接支持 需要 Object.fromEntries 转换
V8 优化 有 hidden class 快路径（固定 shape） 通用哈希表

选型：

  • 用 Object：数据结构形状固定（POJO）、需要 JSON 直接序列化、代码可读性优先。
  • 用 Map：key 不是字符串、需要保序、频繁增删改查、需要查 size、key 是动态不可控（可能撞 toString）。

相等算法：SameValueZero

Map/Set/Array.prototype.includes 使用 SameValueZero 算法：

  • 基本同 ===，但 NaN 和自己相等。
  • +0 和 -0 被视为相等（和 Object.is 不同）。
javascript 
const s = new Set();
s.add(NaN);
s.has(NaN);              // true  ✓ （indexOf 找不到 NaN，Set 可以）

s.add(0);
s.has(-0);               // true  （±0 视为同值）

// 对象按引用
s.add({ a: 1 });
s.has({ a: 1 });         // false  ← 不同引用
const o = { a: 1 };
s.add(o);
s.has(o);                // true

Weak 的 GC 语义

WeakMap/WeakSet 对 key 持有弱引用：如果 key 对象在其他地方没被引用，GC 回收时这条目自动消失。因此：

  • key 必须是对象（Symbol 也可，ES2023 起）—— 原始值没有唯一身份供 GC 追踪。
  • 不可迭代 —— 迭代顺序不确定，会让 GC 行为变成可观测，破坏引擎假设。
  • 没有 .size —— 同理。
  • 只能 get/set/has/delete（外加 WeakSet 的 add/has/delete）。
javascript 
const wm = new WeakMap();
let key = { id: 1 };
wm.set(key, { computed: "value" });
wm.get(key);            // { computed: "value" }

key = null;             // 断开引用
// 某个时刻 GC 会清理 wm 中对应条目 —— 无需手动

// ⚠ 不能用原始值
wm.set("hello", 1);     // TypeError: Invalid value used as weak map key

WeakRef 与 FinalizationRegistry（高阶）

ES2021 加入的两个 API，让你能显式持有弱引用和注册 GC 回调：

javascript 
// WeakRef：弱引用一个对象
let obj = { heavy: "data" };
const ref = new WeakRef(obj);

obj = null;               // 断开强引用

// 某个时刻 GC 可能回收 obj
ref.deref();              // → obj 或 undefined（已被回收）

// FinalizationRegistry：对象被回收时触发回调
const reg = new FinalizationRegistry(heldValue => {
  console.log("cleaned up:", heldValue);
});
reg.register(someObj, "some-id");
// 当 someObj 被 GC 时，回调收到 "some-id"
⚠ WeakRef 的使用准则（来自 TC39）

"如果你觉得需要 WeakRef，可能你并不需要"。GC 的触发时机不可预测，deref() 可能几分钟才返回 undefined。真正的使用场景很窄：大对象缓存（希望内存压力时被回收）、通知订阅者（避免循环引用）。一般业务优先用 WeakMap。

Set.has vs Array.includes

Set.has 是 O(1)；Array.includes / Array.indexOf 是 O(n)。大量查找 / 去重场景差距惊人：

javascript 
// 10000 个元素做 10000 次查询
// Array.includes: ~500ms
// Set.has:        ~5ms
//               差 100 倍

const ids = new Set(allIds);                    // 建索引 O(n)
items.filter(x => ids.has(x.id));               // 查询 O(m)
// 总 O(n + m)，胜过 items.filter(x => allIds.includes(x.id)) 的 O(n*m)

03 应用场景

WeakMap 挂"私有"数据

给第三方对象附加元数据，又不想污染对象自身（污染可能破坏对方的 JSON.stringify、for-in 等）。同时当对象被 GC 时元数据自动消失 —— 不需要手动清理。

javascript 
const privateData = new WeakMap();

class User {
  constructor(name) {
    privateData.set(this, { secret: generateSecret() });
    this.name = name;
  }
  check(pwd) { return privateData.get(this).secret === pwd; }
}

// 外部看不到 secret：JSON.stringify(user) 只含 name
// 实例被 GC 时 privateData 自动清理

现代 class 有 #privateField，首选它。但给别人的对象挂私有仍需 WeakMap（例如给 DOM 节点挂状态）。

DOM 节点的事件处理器映射

javascript 
const handlers = new WeakMap();

function bind(el, fn) {
  handlers.set(el, fn);
  el.addEventListener("click", fn);
}
function unbind(el) {
  const fn = handlers.get(el);
  if (fn) el.removeEventListener("click", fn);
}

// DOM 节点从 document 移除后被 GC，WeakMap 条目自动消失
// 换成 Map 会造成"内存泄漏"（持有已移除节点的强引用）

缓存昂贵计算

javascript 
const cache = new WeakMap();

function computeExpensive(obj) {
  if (cache.has(obj)) return cache.get(obj);
  const result = /* ... 重计算 ... */;
  cache.set(obj, result);
  return result;
}

// obj 不再被引用时缓存自动清理 —— 不需要 LRU

按对象字段 groupBy

javascript 
// Map.groupBy（ES2024） —— 保序 + 支持对象 key
const byTeam = Map.groupBy(employees, e => e.team);
byTeam.get("engineering");            // 该 team 的员工数组

// 对比 Object.groupBy：key 只能字符串
const byDept = Object.groupBy(employees, e => e.department);

简易 LRU 缓存（利用 Map 保序）

javascript 
class LRU {
  constructor(max = 100) { this.max = max; this.cache = new Map(); }
  get(key) {
    if (!this.cache.has(key)) return undefined;
    const v = this.cache.get(key);
    this.cache.delete(key);         // 移到末尾
    this.cache.set(key, v);
    return v;
  }
  set(key, v) {
    if (this.cache.has(key)) this.cache.delete(key);
    else if (this.cache.size >= this.max)
      this.cache.delete(this.cache.keys().next().value);   // 删最老
    this.cache.set(key, v);
  }
}

比用 Object + 手动维护顺序数组 简洁得多。核心利用 Map 的插入序保证。

Set 做集合运算（ES2025）

javascript 
// ES2025 新增：Set 集合运算（现代浏览器已支持）
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);

a.union(b);                  // Set(1,2,3,4)
a.intersection(b);           // Set(2,3)
a.difference(b);             // Set(1)
a.symmetricDifference(b);    // Set(1,4)
a.isSubsetOf(b);             // false
a.isSupersetOf(b);           // false
a.isDisjointFrom(b);         // false

04 典型代码

频率统计

javascript 
// 统计词频
function countWords(text) {
  const counts = new Map();
  for (const word of text.toLowerCase().split(/\W+/)) {
    if (!word) continue;
    counts.set(word, (counts.get(word) ?? 0) + 1);
  }
  return counts;
}

const top10 = [...countWords(doc)]
  .sort(([, a], [, b]) => b - a)
  .slice(0, 10);

保持顺序的对象去重

javascript 
// 按对象的某个字段去重（保留首次出现）
function uniqueBy(arr, keyFn) {
  const seen = new Set();
  return arr.filter(x => {
    const k = keyFn(x);
    if (seen.has(k)) return false;
    seen.add(k);
    return true;
  });
}
uniqueBy(users, u => u.id);

// 或用 Map（保留最后一次）
const byId = new Map(users.map(u => [u.id, u]));
[...byId.values()];

Map ↔ Object / JSON

javascript 
const m = new Map([["a", 1], ["b", 2]]);

// Map → Object（仅 key 是字符串时）
Object.fromEntries(m);                     // { a: 1, b: 2 }

// Object → Map
new Map(Object.entries({ a: 1, b: 2 }));

// Map → JSON
JSON.stringify(Object.fromEntries(m));     // '{"a":1,"b":2}'
// ⚠ JSON.stringify(map) 直接调用返回 "{}"（Map 自身没有可枚举属性）

// JSON → Map
new Map(Object.entries(JSON.parse(json)));

// Map 中的对象 key 无法转 JSON —— 只能手动序列化为 [[k, v], ...]

用对象作为 Map key

javascript 
// 按"坐标对象"查找邻居 —— 在 Object 里不可能
const cells = new Map();
cells.set({ x: 0, y: 0 }, "origin");
cells.get({ x: 0, y: 0 });          // undefined  ← 不同引用！

// 要么存引用，要么用字符串 key
const posKey = (p) => `${p.x},${p.y}`;
cells.set(posKey({x:0,y:0}), "origin");
cells.get(posKey({x:0,y:0}));       // "origin"

// 对象作 key 的合理场景：挂元数据到特定对象实例
const meta = new WeakMap();
meta.set(userObj, { lastSeen: Date.now() });

迭代 Map / Set 的花样

javascript 
const m = new Map([["a", 1], ["b", 2]]);

// 默认：[key, value] 解构
for (const [k, v] of m) console.log(k, v);

// 只要 key / 只要 value
for (const k of m.keys()) ...
for (const v of m.values()) ...

// 转数组做 map/filter（Map/Set 自身没有这些方法）
[...m].filter(([k]) => k.startsWith("a")).map(([k, v]) => [k, v * 2]);

// forEach 顺序和 for-of 一致
m.forEach((value, key) => ...);     // ⚠ 回调参数是 (value, key) —— 与 Map.get(key) 方向反

05 常见陷阱

Map.forEach 回调参数顺序是 (value, key, map)

和 Array.forEach((value, index)) 对齐的设计，但 map.get(key) 是 key → value 方向。容易写反。推荐用 for-of 解构 [k, v] 更明确。

JSON.stringify(map) 返回 "{}"

JSON.stringify 序列化的是"可枚举字符串属性"，而 Map 的内容在引擎内部 slot 里不是属性。要序列化需要 [...map] 或 Object.fromEntries(map)。详见 12 章。

WeakMap key 不能是原始值

wm.set("a", 1) 抛 TypeError。原始值没有"身份"供 GC 追踪。ES2023 起 Symbol 可以（因为 Symbol 有引擎级身份），但字符串/数字/布尔仍不行。

new Set(obj) 对象不是 iterable

new Set({a: 1, b: 2}) 抛 TypeError。要转成 iterable：new Set(Object.values(obj)) 或 new Set(Object.entries(obj))。

Map/Set 没有 .map/.filter

必须转成数组：[...set].filter(...)、new Map([...map].filter(...))。社区有 tc39/proposal-collection-methods 提案，但还没落地。

Object.keys(map) 返回空数组

Map 的 entries 不是自身属性。要取 keys：[...map.keys()]。类似陷阱：for...in map 也不遍历条目。

用 Map 做大常量表可能比 Object 慢

V8 对只读 Object（固定 shape）有特殊优化，查 obj.foo 可以内联到寄存器；而 map.get("foo") 是通用哈希查询。常量、shape 固定用 Object；key 动态、频繁增删用 Map。

set.size 是属性，不是方法

set.size() → TypeError。和 Array.length 一致。但 set.keys() / set.values() / set.entries() 是方法。易混。
