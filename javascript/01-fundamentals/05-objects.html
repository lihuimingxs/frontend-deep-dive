对象深入

JS 的对象 = 一个"键（字符串 / Symbol） → 属性描述符"的字典，加上一条 [[Prototype]] 链。本章把这个结构的所有操作面梳清楚：字面量语法糖、属性描述符、浅/深拷贝、冻结策略、查询 API。原型链和 class 留到 P2。

阅读时长 ~25 分钟 前置 01 - 04 对应 javascript.info Objects

01 速览

javascript 
// 对象字面量的全部语法糖
const key = "dynamic";
const x = 1;
const obj = {
  // 1. 常规
  name: "Alice",

  // 2. 属性简写：变量名同键名
  x,                         // 等价 x: x

  // 3. 计算键：方括号里是任意表达式
  [key]: true,               // { dynamic: true }
  [`${key}-2`]: true,

  // 4. 方法简写
  greet() { return "hi"; },

  // 5. getter / setter
  get upper() { return this.name.toUpperCase(); },
  set upper(v) { this.name = v.toLowerCase(); },

  // 6. 展开：深浅复制/合并
  ...other,
};

02 底层逻辑

属性描述符：每个属性的 4 + 2 个开关

数据属性的 4 个描述符：

  • value — 属性的值
  • writable — 能否被赋值修改
  • enumerable — 是否出现在 for-in/Object.keys/JSON.stringify
  • configurable — 能否被 delete、能否改描述符

访问器属性的 4 个描述符（前两个与数据属性互斥）：

  • get / set — 读取/赋值时触发的函数
  • enumerable、configurable — 同上

obj.foo = 1 这种赋值默认创建的属性：writable: true, enumerable: true, configurable: true。Object.defineProperty 默认所有开关都是 false —— 这不对称，容易踩。建议显式写齐。

javascript 
const o = {};
Object.defineProperty(o, "hidden", {
  value: 42,
  writable: true,
  enumerable: false,       // 不出现在遍历里
  configurable: true,
});
Object.keys(o);            // []
o.hidden;                  // 42
o.hidden = 100;            // 可写

// class 的方法、Array.prototype.push 等都是 enumerable: false
// 这是为什么 for-in 数组不会拿到方法名

Getter / Setter：声明式计算属性

javascript 
const user = {
  firstName: "Alice",
  lastName: "Wong",
  get fullName() { return `${this.firstName} ${this.lastName}`; },
  set fullName(v) { [this.firstName, this.lastName] = v.split(" "); },
};
user.fullName;                // "Alice Wong"   —— 像属性一样访问
user.fullName = "Bob Li";
user.firstName;               // "Bob"

// 性能注意：getter 在 V8 里比数据属性慢约 2-5 倍
// 热点路径上频繁访问同一 getter 结果要手动缓存

浅拷贝与深拷贝：5 种工具的边界

方式 深度 支持的类型 性能
{ ...obj } 浅 只拷自有可枚举 + 字符串键（不含 Symbol 取决于 TC39 规则，现在已拷 Symbol） 快
Object.assign({}, obj) 浅 同上 快
JSON.parse(JSON.stringify(obj)) 深 仅 JSON 兼容（丢失 Date/Map/Set/BigInt/函数/undefined/Symbol/循环引用） 慢
structuredClone(obj) 深 原生；支持 Date/Map/Set/TypedArray/循环引用；不支持函数/Symbol/DOM 中等
_.cloneDeep(obj) (Lodash) 深 最全；含函数引用浅拷、原型链保留 中等偏慢

选择建议：现代浏览器/Node 18+ 用 structuredClone（原生、无依赖、正确）。需要深拷函数或有特殊 class 实例用 Lodash。简单 POJO 用 JSON.parse(JSON.stringify)（性能不敏感时也能用）。

不可变性：三个层级的冻结

API 新增属性 删除属性 修改值 改描述符
preventExtensions ✗ ✓ ✓ ✓
seal ✗ ✗ ✓ ✗
freeze ✗ ✗ ✗ ✗

三者都是浅层的。Object.freeze({ nested: { x: 1 } }) 能改 obj.nested.x。真正的深冻结需要手写递归（Object.values(o).forEach(v => typeof v === "object" && deepFreeze(v)); return Object.freeze(o);）。

属性顺序的规范规则

ES2020 起标准化了枚举顺序（Object.keys、for-in、JSON.stringify 等都遵循）：

  1. 整数字符串键（"0", "1", "42"）按数值升序
  2. 其他字符串键按插入顺序
  3. Symbol 键按插入顺序（仅 Reflect.ownKeys 可见）

副作用：{ "2": "a", "1": "b", "x": "c" } 的 Object.keys 是 ["1", "2", "x"]（数字被重排）。需要保序插入 → 用 Map。

V8 的 Hidden Class（引擎视角）

V8 把每个对象关联到一个 "hidden class"（类似 C++ 的 vtable），描述属性的内存布局。相同形状的对象共享同一 hidden class —— 属性访问通过 offset 直接读内存，速度接近静态语言。

工程意义：让对象的"形状"保持一致。

  • 在构造函数里统一初始化所有字段（哪怕设为 null），避免后续动态加属性。
  • 避免 delete obj.key（触发 hidden class 去优化）；改成 obj.key = undefined。
  • 同一数组别混存 Number 和 Object —— V8 对纯数字数组有特殊优化（Packed SMI 数组）。

03 应用场景

Record / Dictionary：用 null 原型避免污染

javascript 
// ❌ 普通对象作 map，"toString" / "hasOwnProperty" 会撞上原型
const counts = {};
counts["toString"] = 1;      // 覆盖了 Object.prototype.toString
"hasOwnProperty" in counts;  // true 但不是你插入的

// ✅ 无原型的"纯字典"
const counts2 = Object.create(null);
counts2["toString"] = 1;     // 安全
"hasOwnProperty" in counts2; // false
// 代价：丢失 .toString / .hasOwnProperty 等方法

不可变状态管理

Redux / Zustand / Jotai 等库要求每次更新返回新对象（便于浅比较触发重渲染）。典型更新模式：

javascript 
// 顶层字段
{ ...state, count: state.count + 1 }

// 嵌套字段（要逐层展开）
{ ...state, user: { ...state.user, name: "Alice" } }

// 深层嵌套用 Immer 的 produce（原地写法 + 底下不可变更新）
import { produce } from "immer";
const next = produce(state, draft => {
  draft.user.profile.avatar.url = "...";
});

Getter 做延迟/计算属性

javascript 
// 配置的"最终"值从多个源派生
const config = {
  baseUrl: "https://api.example.com",
  version: "v2",
  get apiUrl() { return `${this.baseUrl}/${this.version}`; },
};
config.apiUrl;            // 每次重新计算
config.version = "v3";
config.apiUrl;            // "https://api.example.com/v3"  ← 自动反映

Object.freeze 做常量表

TypeScript 里常写 const COLORS = { RED: "#f00", BLUE: "#00f" } as const; 得到类型层面不可变。运行时防止被改就加 Object.freeze(COLORS)。React 19 的 cache、路由配置等全局数据都这么做。

Property descriptor 做 console.log 秘境

在自己的 class 上定义 enumerable: false 的内部字段，能让 JSON.stringify(instance) 只序列化业务字段、不泄漏内部状态。ORM、validator 等库常用。

04 典型代码

查对象结构的 6 个 API

javascript 
const o = { a: 1 };
Object.defineProperty(o, "hidden", { value: 2, enumerable: false });
const s = Symbol("key");
o[s] = 3;

Object.keys(o);                        // ["a"]                仅自有 enumerable 字符串
Object.getOwnPropertyNames(o);         // ["a", "hidden"]      所有自有字符串（含不可枚举）
Object.getOwnPropertySymbols(o);       // [Symbol(key)]        所有自有 Symbol
Reflect.ownKeys(o);                    // ["a", "hidden", Symbol(key)]   所有自有
Object.getOwnPropertyDescriptor(o, "hidden"); // { value:2, writable:false,... }
Object.getPrototypeOf(o);              // Object.prototype

浅拷贝与深拷贝对比

javascript 
const src = { a: 1, nested: { x: 10 }, d: new Date(), arr: [1,2,3] };

// 浅拷贝
const a = { ...src };
a.nested.x = 99;              // ⚠ src.nested.x 也变了
a.d === src.d;                // true （同一 Date 引用）

// 深拷贝（推荐）
const b = structuredClone(src);
b.nested.x = 99;              // ✓ src.nested.x 仍是 10
b.d instanceof Date;          // true （正确保留类型）
b.d === src.d;                // false

// JSON 法（丢类型）
const c = JSON.parse(JSON.stringify(src));
c.d instanceof Date;          // false ⚠ 变成字符串了

// 循环引用：structuredClone 支持，JSON 不支持
const circular = { self: null }; circular.self = circular;
structuredClone(circular);     // ✓
JSON.stringify(circular);      // TypeError

in vs hasOwn vs ?.

javascript 
const o = { a: undefined };

"a" in o;                       // true  （有这个键，即使值是 undefined）
o.a === undefined;              // true  （无法区分"未定义"和"没这个键"）
Object.hasOwn(o, "a");          // true  （ES2022 推荐；替代 hasOwnProperty.call）

"toString" in o;                // true   ← 继承自 Object.prototype
Object.hasOwn(o, "toString");   // false  ← 只查自有属性

// 老写法（在某些 null-prototype 对象上会崩）
o.hasOwnProperty("a");          // 可能 throw 如果 o 是 Object.create(null)
// 总是更安全的写法
Object.hasOwn(o, "a");

用 defineProperty 实现只读常量

javascript 
const api = {};
Object.defineProperty(api, "VERSION", {
  value: "1.0.0",
  writable: false,
  enumerable: true,
  configurable: false,
});
api.VERSION = "2.0";          // 严格模式抛错；非严格静默失败
api.VERSION;                  // 仍为 "1.0.0"

// 批量定义
Object.defineProperties(api, {
  NAME: { value: "Mira", enumerable: true },
  SECRET: { value: "xxx", enumerable: false },   // 不出现在 JSON.stringify
});

深冻结工具函数

javascript 
function deepFreeze(obj) {
  if (obj === null || typeof obj !== "object") return obj;
  // 先递归冻结属性，再冻结 obj 本身
  for (const key of Reflect.ownKeys(obj)) {
    deepFreeze(obj[key]);
  }
  return Object.freeze(obj);
}

const cfg = deepFreeze({
  endpoints: { api: "/api", ws: "/ws" },
  features: { darkMode: true },
});
cfg.endpoints.api = "/v2";     // 严格模式抛错

05 常见陷阱

Object.freeze 是浅冻结

第一层不可变，嵌套对象不受影响。TypeScript 的 Readonly<T> 也只是一层。Record 类型不可变用 Readonly<T> + runtime deepFreeze。

展开语法不拷贝 getter/setter，只拷值

{ ...obj } 会触发 getter 取值后复制，新对象里是数据属性。如果需要保留描述符用 Object.defineProperties(target, Object.getOwnPropertyDescriptors(source))。

对象键都是字符串（或 Symbol）

obj[1] === obj["1"]，因为 key 会被 ToString。obj[{a:1}] 的 key 是 "[object Object]"，不同对象作 key 会冲突。需要对象作 key 用 Map。

Object.keys 不包含 Symbol 键

JSON.stringify 也不会序列化 Symbol 键。需要完整遍历用 Reflect.ownKeys。

整数键的"神秘重排"

{ "2": "x", "1": "y", "a": "z" } 的 Object.keys 顺序是 ["1", "2", "a"]。ES2020 前非标准，现在是规范行为。需要保序用 Map。

delete 对 non-configurable 属性静默失败（非严格）

严格模式下抛 TypeError；非严格模式下返回 false。ESM 默认严格 —— 对 const 变量、Object.freeze 过的属性执行 delete 会报错。

对象展开不继承原型

{ ...instance } 丢掉 class 的原型链，得到的是纯对象。克隆 class 实例应用 structuredClone（保留类型）或 Object.assign(Object.create(Proto), instance)。

JSON.stringify 按 getter 值序列化

{ get x() { return Math.random(); } } 每次 JSON.stringify 得到的 x 值都不同。大多数场景是好事，但在 hash 值、签名等场景需要注意"同一对象序列化两次不等"的可能性。
