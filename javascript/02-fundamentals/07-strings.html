字符串深入

字符串是 JS 里被"看起来简单实则复杂"的典型。UTF-16 存储、Unicode 归一化、本地化比较、字素簇切分、标签模板的 AST 级能力 —— 工程代码里但凡涉及"用户输入 / i18n / 搜索匹配 / SQL 构造"就会碰到。本章集中整理所有高频操作。

阅读时长 ~25 分钟 前置 01 数据类型（UTF-16） 对应 javascript.info Strings

01 速览

常用方法按目的分组：

目的 方法 备注
访问 length UTF-16 code unit 数，不是字符数
s[i] / s.at(i) 按索引；at 支持负索引
charCodeAt / codePointAt 前者 UTF-16，后者 Unicode code point
for...of s 按 code point 迭代（emoji 完整）
切取 slice(start?, end?) 支持负索引；推荐
substring(start, end) 负数当 0；start > end 会交换参数
substr(start, len) 已弃用，用 slice
查找 indexOf / lastIndexOf 按位置；=== 比较
includes / startsWith / endsWith boolean；语义清晰
match / matchAll 正则匹配；matchAll 返回迭代器
search 正则首次位置（少用）
替换 replace(re/str, new) 无 /g 时只换首个
replaceAll(re/str, new) ES2021；替换所有匹配
变形 toLowerCase / toUpperCase ASCII 地区安全；土耳其等语言需 toLocaleLowerCase
trim / trimStart / trimEnd 仅处理 Unicode 空白；不处理全角空格 U+3000？是会处理
padStart / padEnd 补齐到指定长度
repeat(n) 重复 n 次

拼接 concat / + / 模板字符串 现代代码用模板字符串

localeCompare(other, locale, opts) 本地化比较；返回 -1/0/1

02 底层逻辑

模板字符串的规范定义

反引号字符串被解析器拆成两个数组：字面量片段和表达式。

javascript 
const name = "Mira";
const greeting = `Hello, ${name}!`;

// 等价于
const greeting = "Hello, " + name + "!";

// 支持任意表达式、多行、保留缩进
const html = `
  <div class="card">
    <h2>${user.name.toUpperCase()}</h2>
    <p>${user.bio || "无简介"}</p>
  </div>
`;

标签模板：最强大的语法糖

在模板字符串前加函数名，变成"调用这个函数"，但参数是拆分后的字面量数组 + 表达式值：

javascript 
function tag(strings, ...values) {
  // strings 是字面量片段数组（有 .raw 属性）
  // values 是插值结果数组
  return strings.map((s, i) => s + (values[i] ?? "")).join("");
}

const name = "Mira", age = 2;
tag`name=${name}, age=${age}`;
// 调用时参数：
// strings = ["name=", ", age=", ""]
// values  = ["Mira", 2]

这让标签函数能按需处理插值 —— 比如 SQL 转义、HTML 转义、CSS 语法生成、GraphQL 查询、i18n 等。styled-components、lit-html、graphql-tag 都是这个机制。

String.raw

内置标签函数，返回原始字符串（不处理 \n 等转义）：

javascript 
`\n`.length;               // 1  （换行符）
String.raw`\n`.length;     // 2  （反斜杠 + n）

// 用途：Windows 路径、正则、LaTeX
const path = String.raw`C:\Users\Alice\Documents`;
const re = new RegExp(String.raw`\d+\.\d+`);

Unicode 归一化：同一个字符的多种字节表示

"é" 能有两种 Unicode 表示：

  • NFC：U+00E9（单个预组合字符）
  • NFD：U+0065 U+0301（e + 组合重音符）
javascript 
const a = "é";            // "é" NFC
const b = "é";      // "é" NFD
a === b;                       // false!  视觉相同，字节不同
a.length;                      // 1
b.length;                      // 2
a.normalize() === b.normalize();  // true  默认 NFC
a.normalize("NFD") === b;         // true

工程意义：来自不同源的字符串（macOS 文件名是 NFD、大部分文本编辑器是 NFC、用户粘贴带组合字符的 emoji）做比较前必须 .normalize()。四种形式：

  • NFC（默认）—— 规范组合：尽可能合并。大多数场景用这个。
  • NFD —— 规范分解。
  • NFKC —— 兼容组合：把"兼容等价"字符（㈣ → (四)、㎏ → kg）也合并。搜索/索引用这个。
  • NFKD —— 兼容分解。

Intl.Collator：本地化比较

a < b 按 Unicode code unit 排序（纯字节比较）。对用户可见的排序应该用 Intl.Collator：

javascript 
// 快速用法
"张三".localeCompare("李四", "zh");              // 1（"李" 拼音靠前）
"a".localeCompare("B", "en", { sensitivity: "base" });  // -1（忽略大小写）

// 批量排序创建复用 Collator（性能提升 10-100 倍）
const cn = new Intl.Collator("zh", { numeric: true });
files.sort(cn.compare);    // "file1", "file2", "file10", "file20"  数字顺序而非字典序

// sensitivity 控制
// "base"     → a === A, a === á
// "accent"   → a === A, a !== á
// "case"     → a !== A, a === á
// "variant"  → 默认；严格按 Unicode

Intl.Segmenter：字素簇 / 词 / 句切分

按用户视觉"字符"切割字符串 —— 处理 emoji、ZWJ 序列、组合字符、肤色修饰符。ES2024 标准化，Chrome 87+ / Safari 14.1+ / Node 16+。

javascript 
const s = "👨‍👩‍👧 Hello 你好";

[...s].length;                                 // 11（按 code point）
s.length;                                      // 更多（UTF-16 units）

const seg = new Intl.Segmenter("zh", { granularity: "grapheme" });
[...seg.segment(s)].length;                    // 9（按字素簇，emoji 家族算 1）

// 按词切分（中文需要语言）
const word = new Intl.Segmenter("zh", { granularity: "word" });
for (const { segment, isWordLike } of word.segment("学习 JavaScript 编程")) {
  if (isWordLike) console.log(segment);         // "学习", "JavaScript", "编程"
}

字符串的不可变性与引擎优化

所有字符串方法返回新字符串，原字符串不变。引擎内部有三种常见存储优化：

  • Sequential string —— 常规连续字符数组。
  • Cons string / Rope —— a + b 结果是个"两个子串的树"，懒展开（避免立即分配大内存）。
  • Sliced string —— s.slice(i, j) 只存指针 + 偏移，不复制。

这就是为什么频繁拼接 s += chunk 在现代引擎上不算灾难性。但循环里用 arr.push(chunk) + arr.join("") 仍然更一致地快。

03 应用场景

SQL 转义：标签模板防注入

javascript 
// 危险 —— 用户输入直接拼进 SQL
const bad = `SELECT * FROM users WHERE name = '${input}'`;
// input = "'; DROP TABLE users; --" 则裂开

// 安全 —— 标签函数区分字面量和插值，自动参数化
function sql(strings, ...values) {
  return {
    text: strings.reduce((q, s, i) => q + s + (i < values.length ? `$${i + 1}` : ""), ""),
    values,
  };
}
const q = sql`SELECT * FROM users WHERE name = ${input}`;
// { text: "SELECT * FROM users WHERE name = $1", values: [input] }
// 交给 pg.query(q) 做参数化查询

HTML 安全转义：lit-html 模式

javascript 
const esc = v => String(v)
  .replace(/&/g, "&amp;").replace(/</g, "&lt;").replace(/>/g, "&gt;");

function html(strings, ...values) {
  return strings.reduce((s, lit, i) => s + lit + (i < values.length ? esc(values[i]) : ""), "");
}
const name = "<script>";
html`<h1>${name}</h1>`;
// "<h1>&lt;script&gt;</h1>"   ← 自动转义

模糊搜索：normalize + 小写 + 去重音

javascript 
// 让 "Café" 能被 "cafe" 搜到
function fuzzy(s) {
  return s.normalize("NFD")
    .replace(/\p{Diacritic}/gu, "")    // 去组合重音符
    .toLowerCase();
}
fuzzy("Café") === fuzzy("cafe");      // true

文件名排序：numeric 比较

javascript 
const files = ["file1.txt", "file10.txt", "file2.txt"];

files.sort();
// ["file1.txt", "file10.txt", "file2.txt"]  ← 字典序（错）

files.sort(new Intl.Collator(undefined, { numeric: true }).compare);
// ["file1.txt", "file2.txt", "file10.txt"]  ← 数字序（对）

截取用户输入前 N 个"可见字符"

javascript 
function truncate(s, n) {
  const seg = new Intl.Segmenter();
  const chars = [];
  for (const { segment } of seg.segment(s)) {
    chars.push(segment);
    if (chars.length >= n) break;
  }
  return chars.join("");
}
truncate("👨‍👩‍👧 Hello 你好", 5);        // "👨‍👩‍👧 Hel"
// 简单方案 (适合 BMP 字符)：[...s].slice(0, n).join("")

04 典型代码

replace 的所有姿势

javascript 
// 1. 字符串替换一次
"a-b-c".replace("-", "/");                // "a/b-c"

// 2. 全部替换
"a-b-c".replaceAll("-", "/");             // "a/b/c"
"a-b-c".replace(/-/g, "/");               // "a/b/c"  旧写法
// ⚠ replaceAll 要求正则必须有 /g，否则 TypeError

// 3. 用捕获组
"John Doe".replace(/(\w+) (\w+)/, "$2, $1");   // "Doe, John"

// 4. 用函数（按每次匹配计算替换）
"item-123".replace(/\d+/, n => `[${+n * 2}]`);  // "item-[246]"

// 5. 正则 + 命名捕获组（ES2018）
"2024-03-15".replace(/(?<y>\d+)-(?<m>\d+)-(?<d>\d+)/, "$<d>/$<m>/$<y>");
// "15/03/2024"

padStart / padEnd

javascript 
// 零填充
String(5).padStart(3, "0");         // "005"
String(42).padStart(3, "0");        // "042"
String(300).padStart(3, "0");       // "300"（不截断）

// 对齐输出
const data = [["Alice", 90], ["Bob", 100], ["Eve", 5]];
data.forEach(([n, s]) => console.log(n.padEnd(10) + String(s).padStart(5)));
// Alice         90
// Bob          100
// Eve            5

// 时间戳
new Date().getHours().toString().padStart(2, "0");  // "09"

slice / substring 对比

javascript 
const s = "Hello";

s.slice(1, 3);           // "el"
s.slice(-3);             // "llo"      ← 支持负数
s.slice(1, -1);          // "ell"

s.substring(1, 3);       // "el"
s.substring(-3);         // "Hello"    ← 负数当 0
s.substring(3, 1);       // "el"       ← 会交换参数

// 一律用 slice。substring 行为怪异，substr 已弃用

split 的进阶

javascript 
"a,b,c".split(",");                      // ["a", "b", "c"]
"a,b,c,d".split(",", 2);                 // ["a", "b"]  ← 截断
"".split(",");                           // [""]  ← 不是 []
"abc".split("");                         // ["a", "b", "c"]  ⚠ 按 UTF-16 切，emoji 会裂
[..."🔥abc"];                             // ["🔥", "a", "b", "c"]  ✓ 按 code point

// 正则分隔 + 捕获组：组内容也在结果里
"a1b2c".split(/(\d)/);                   // ["a", "1", "b", "2", "c"]

数字格式化（字符串）

javascript 
// ❌ 手动 + 容易出错
(12345.6).toFixed(2);                    // "12345.60"

// ✓ Intl.NumberFormat 处理千分位、货币、本地化
new Intl.NumberFormat("zh-CN").format(12345.6);                    // "12,345.6"
new Intl.NumberFormat("zh-CN", { style: "currency", currency: "CNY" }).format(12345.6);
// "¥12,345.60"
new Intl.NumberFormat("en-US", { notation: "compact" }).format(12345);
// "12K"
new Intl.NumberFormat("de-DE").format(12345.6);                    // "12.345,6"

// 性能：循环前预创建 formatter，调用 .format 复用

05 常见陷阱

"abc".length 数的是 UTF-16 code unit

"👨‍👩‍👧".length === 8。"字符数"用 [...str].length（code points）或 Intl.Segmenter（字素簇，最正确）。

substring 的负数被当成 0

"abc".substring(-1) 是 "abc"，不是 "c"。想要"从尾部取"用 slice(-1)。

replaceAll 传正则必须有 /g

"a-b-c".replaceAll(/-/, "/") 抛 TypeError。设计如此，为了让语义明确。要么传字符串，要么传带 /g 的正则。

toLowerCase 在土耳其语不对

土耳其语的 "I" 小写是 "ı"（无点），"İ" 小写才是 "i"。"I".toLowerCase() 在浏览器默认 locale 下可能不如预期。涉及 locale 敏感的大小写用 toLocaleLowerCase("tr")。

比较前忘了 normalize

macOS 传上来的文件名是 NFD，数据库里存的是 NFC。fileName === record.name 可能假性不等。统一 .normalize() 后再比。

new String("abc") 不是字符串

typeof new String("abc") === "object"，new String("abc") === "abc" 是 false。永远不要 new String，详见 01 章装箱与拆箱。

split("") 切碎 emoji

"🔥".split("") → ["\uD83D", "\uDD25"]（孤立代理对）。按字符切用 [...str] 或 Intl.Segmenter。

+ 拼接时对象会调 toString

"hello " + { x: 1 } → "hello [object Object]"。自定义 class 想要友好字符串表示，override toString 或 Symbol.toPrimitive。

标签模板 strings.raw vs strings

strings 是经过转义处理的；strings.raw 保留原始转义序列（\n 是两个字符）。写自定义 DSL 若需要原样文本用 strings.raw。
