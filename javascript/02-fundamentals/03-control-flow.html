控制流

控制流是把表达式组织成程序的骨架。JS 的分支/循环表面上像 C，但有几个关键差异：switch 用 === 匹配、for-of 依赖迭代协议、for-in 会爬原型链、forEach 不能 break、for (let i ...) 每次迭代都生成新绑定。这些细节决定了你的循环是正确的还是看起来正确的。

阅读时长 ~20 分钟 前置 01 · 类型 ; 02 · 操作符 对应 javascript.info Conditional branching; Loops

01 速览

语句 关键行为 典型用途
if / else / else if 条件走 ToBoolean 分支
? : 三元 唯一表达式形式的条件 赋值、JSX
switch === 匹配，需要 break 防 fall-through 枚举、状态机
for (init; cond; step) C 风三段式，let 每次迭代新绑定 索引可控的计数循环
for...of 走 Symbol.iterator 协议，可 break 数组/Map/Set/字符串/自定义 iterable
for...in 遍历可枚举字符串键（含原型链） 仅用于纯对象且你真需要原型链键
while / do...while 条件循环；do-while 保证至少执行一次 未知次数的条件循环
break / continue 支持 label 跨层跳转 嵌套循环早退
.forEach / .map ... 函数式；不能 break 数据映射/聚合

02 底层逻辑

if 条件的 ToBoolean 表

条件语句对非 boolean 值触发 ToBoolean。8 个 falsy 值（其余全 truthy）：

javascript 
false · 0 · -0 · 0n · "" · null · undefined · NaN
// 一切对象都 truthy，包括 [] / {} / new Boolean(false) / new Number(0)

switch 的严格相等匹配

switch (x) 的每个 case v: 分支实际上等价于 if (x === v)。不是 ==。因此：

javascript 
const x = "5";
switch (x) {
  case 5: console.log("number"); break;   // 不会匹配（"5" !== 5）
  case "5": console.log("string"); break; // ✓ 命中
}

JS 没有模式匹配（TC39 stage 1 提案 Pattern Matching 尚未落地），但可以用 switch (true) 技巧做范围匹配：

javascript 
switch (true) {
  case score >= 90: grade = "A"; break;
  case score >= 80: grade = "B"; break;
  case score >= 60: grade = "C"; break;
  default: grade = "F";
}

Fall-through 不是 bug，是特性

case 之后没有 break / return 会继续执行下一个 case。用于合并多个分支：

javascript 
switch (day) {
  case "Sat":
  case "Sun":
    type = "weekend"; break;
  default:
    type = "weekday";
}

但意外遗漏 break 是经典 bug 源。ESLint no-fallthrough 规则会要求显式 // falls through 注释 —— 强烈建议开启。

迭代协议：for-of 的规范依据

for-of 不是"遍历数组"，而是"遍历可迭代对象"。可迭代对象必须实现 Symbol.iterator，返回一个迭代器（有 next() 方法，返回 { value, done }）。

内置可迭代：Array、String、Map、Set、NodeList、arguments、TypedArray、generator 函数返回值。普通 Object 不可迭代 —— for (const x of {a: 1}) 抛 TypeError。遍历对象用 Object.entries(obj)、Object.keys(obj)、Object.values(obj)。

for-in vs for-of 的根本区别

for-in for-of
遍历什么 可枚举字符串属性名 迭代器产生的值
是否走原型链 是（包含继承的可枚举属性） 否
顺序 规范保证：整数键按升序 → 其他字符串键按插入序 迭代器决定
Symbol 键 不包括 —
稀疏数组 跳过空位 空位产出 undefined

for (let i ...) 的每次迭代新绑定

这是 ES6 特意修复 var 时代闭包 bug 的机制。规范规定：for 循环初始化里用 let/const 时，每次迭代都创建一个全新的词法环境，i 的副本被克隆。

javascript 
// var 时代的经典面试题
const fs = [];
for (var i = 0; i < 3; i++) fs.push(() => i);
fs.map(f => f());            // [3, 3, 3]  ← 共享同一个 i

// let 自动修复
const gs = [];
for (let i = 0; i < 3; i++) gs.push(() => i);
gs.map(g => g());            // [0, 1, 2]  ← 每次迭代新 i

break / continue 的 label 跨层

JS 没有 goto，但支持给循环/块打 label，break label 能跨越多层循环退出 —— 比用 flag 变量清晰：

javascript 
outer:
for (let y = 0; y < rows; y++) {
  for (let x = 0; x < cols; x++) {
    if (grid[y][x] === target) {
      found = [y, x];
      break outer;          // 一次跳出两层
    }
  }
}

03 应用场景

用 lookup 表替代长 if/switch

3+ 分支开始考虑把 if/switch 重构成 Record<string, fn>：

javascript 
// 难维护
function route(action) {
  if (action === "CREATE") return handleCreate();
  if (action === "UPDATE") return handleUpdate();
  if (action === "DELETE") return handleDelete();
  throw new Error(`unknown: ${action}`);
}

// 数据驱动
const handlers = {
  CREATE: handleCreate,
  UPDATE: handleUpdate,
  DELETE: handleDelete,
};
function route(action) {
  const fn = handlers[action];
  if (!fn) throw new Error(`unknown: ${action}`);
  return fn();
}

优势：新增分支不改控制流、可以动态注入、方便做单测 mock。

Generator + for-of 实现 lazy 计算

Generator 函数实现迭代协议 —— 用 for-of 消费时才按需生成值。适合"逻辑上无穷但实际只取前 N 个"的场景（分页、树遍历、ID 生成）。

javascript 
function* range(start, end, step = 1) {
  for (let i = start; i < end; i += step) yield i;
}
for (const n of range(0, 10, 2)) console.log(n);  // 0 2 4 6 8

// 惰性、可 break、可组合
function* take(iter, n) {
  for (const v of iter) { if (n-- <= 0) return; yield v; }
}

状态机：switch + 独立 reducer

Redux-style reducer 本质是状态机；switch 的分支匹配 action type 是天然适配。React 的 useReducer 也沿用这个模式。详细见 P2 装饰器与 class 一章。

循环中的异步：for-of + await

for-of 能正确配合 await 顺序执行；forEach + async 回调只会并发发射所有请求但不等它们完成。并发用 Promise.all(items.map(asyncFn))。详细语义见 P3 异步章。

04 典型代码

遍历各种集合

javascript 
// 数组 —— for-of 拿值，entries 同时拿索引
for (const v of arr) ...
for (const [i, v] of arr.entries()) ...

// 对象 —— 用 Object.* 转 iterable
for (const [k, v] of Object.entries(obj)) ...
for (const k of Object.keys(obj)) ...

// Map / Set 原生可迭代
for (const [k, v] of map) ...
for (const v of set) ...

// 字符串按 code point 遍历（处理 emoji 正确）
for (const ch of "Hi👋") ...

// NodeList / HTMLCollection（浏览器）
for (const el of document.querySelectorAll("li")) ...
// getElementsByTagName 返回的 HTMLCollection 在 for-of 中也能用
// 但 for-in 在 DOM 集合上会返回意外的数字索引之外的键

break / return 的选择

javascript 
// 找到就退出：两种写法
function find1(arr, pred) {
  for (const item of arr) if (pred(item)) return item;
  return undefined;
}
// 内置等价写法
function find2(arr, pred) { return arr.find(pred); }

// 早退有副作用时，return 更清晰（不需要 label）
// break 仅用于"不能改成函数提前 return"的场景

forEach 不能 break

javascript 
// ❌ 无效：throw/return 不会终止 forEach
[1,2,3,4].forEach(n => {
  if (n > 2) return;         // 只跳过当前迭代，循环继续
  console.log(n);
});

// ✅ 需要提前退出用 for-of / some / every
[1,2,3,4].some(n => {
  if (n > 2) return true;    // 返回 true 停止遍历
  console.log(n);
});

// ✅ 或者用 find 取代（如果目的是"找到即停"）
const found = arr.find(pred);

循环边界的典型写法

javascript 
// 处理队列直到空
while (queue.length) {
  const task = queue.shift();
  process(task);
}

// 轮询直到条件满足（带最大次数防死循环）
let attempts = 0;
while (!ready() && attempts++ < 10) {
  await sleep(1000);
}

// 至少执行一次 —— do-while 的少数正当场景
let input;
do {
  input = prompt("输入一个数");
} while (Number.isNaN(+input));

三元 vs if/else 赋值

javascript 
// ✅ 简单二选一：三元
const label = isAdmin ? "管理员" : "用户";

// ❌ 嵌套三元（可读性差）
const grade = score >= 90 ? "A" : score >= 80 ? "B" : score >= 60 ? "C" : "F";

// ✅ 改成 lookup 或函数
function getGrade(s) {
  if (s >= 90) return "A";
  if (s >= 80) return "B";
  if (s >= 60) return "C";
  return "F";
}

05 常见陷阱

for-in 遍历数组的坑

for (const i in arr) 返回字符串索引（"0", "1", ...），还可能包含原型链上的可枚举属性（老代码如果给 Array.prototype 加方法就会漏出来）。永远不要用 for-in 遍历数组，用 for-of、.forEach 或 for (let i = 0; ...)。

switch 忘写 break 导致串 case

TypeScript 的 noFallthroughCasesInSwitch 和 ESLint 的 no-fallthrough 都能拦截。刻意 fall-through 时加显式注释 // falls through。

switch 匹配用 ===，字符串/数字必须一致

从 URL params 拿的永远是字符串；switch (Number(pageParam)) 或 switch (pageParam) 要提前决定。混用会导致无分支命中，悄悄走到 default。

case 里声明变量的作用域问题

case "A": let x = 1; break; case "B": let x = 2; ... 会报 already declared，因为 case 块没有自己的作用域。用 { } 包一下：case "A": { let x = 1; break; }。

continue 会跳过 for 的 step 吗？

不会。for (init; cond; step) 的 continue 会先执行 step 再回到 cond。但在 while 中 continue 不执行任何递增 —— 如果你用 while 做计数循环，要在 continue 之前手动 i++，否则死循环。

for...in 顺序 = 整数键升序 → 其他插入序

ES2020 标准化了这个顺序。副作用：const obj = { 2: "a", 1: "b", x: "c" } 的 Object.keys 是 ["1", "2", "x"]（数字键提前了）。需要保序用 Map。

forEach 的 this 与箭头函数

arr.forEach(cb, thisArg) 第二参数设 this。但用箭头函数时 thisArg 被忽略（箭头函数的 this 是词法的）。这是常见面试题。详细见 P2 this 章节。

do-while 后的分号

do { ... } while (cond) 后面必须加分号，否则 ASI 可能把下一行识别成调用。这是 JS 语法规则，不是风格偏好。
