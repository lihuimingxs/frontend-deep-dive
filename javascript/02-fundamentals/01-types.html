数据类型系统

JavaScript 是动态类型、弱类型语言，但"弱"不代表"没有规则"。本章把类型系统拆成"值的存储方式"和"类型行为"两个层面来看 —— 理解了前者，后续关于比较、赋值、内存、装箱的一切行为都不再令人意外。

阅读时长 ~25 分钟 前置 任意一门编程语言 对应 javascript.info Data types

01 速览

ECMAScript 规范定义了 8 种类型：7 种原始类型（primitive）+ 1 种对象类型（Object）。

类型 typeof 字面量 存储 不可变？
Undefined "undefined" undefined 按值 ✓
Null "object" ⚠ 历史遗留 null 按值 ✓
Boolean "boolean" true / false 按值 ✓
Number "number" 42, 3.14, NaN, Infinity 按值（64-bit IEEE 754） ✓
BigInt "bigint" 10n, BigInt("1e20") 按值（任意精度） ✓
String "string" "abc", 'abc', `abc` 按值（UTF-16 编码单元序列） ✓
Symbol "symbol" Symbol("tag") 按值（不可伪造 ID） ✓
Object "object" / "function" {}, [], function(){} 按引用 ✗
💡 记住这条分界线

原始类型按值传递且不可变；对象按引用传递且可变。这是整个 JS 行为大厦的地基 —— 参数传递、=== 语义、Object.freeze 的边界、structuredClone 的必要性，都从这里派生。

02 底层逻辑

引擎层的值表示：NaN-boxing 与 SMI

JS 变量是"有类型的值"而不是"有类型的槽位"。V8、JavaScriptCore、SpiderMonkey 都用类似的技术把所有值压进一个 64-bit 机器字：

  • SMI（Small Integer） —— V8 把 -2³¹ ~ 2³¹-1 的整数直接塞进指针低位（用末位 tag bit 区分）。算术运算不需要分配堆对象，极快。
  • NaN-boxing —— IEEE 754 双精度有 2⁵² 种 NaN 表示，引擎用其中"未使用"的 NaN 编码 undefined / null / bool / 小字符串 / 对象指针。
  • HeapObject —— 所有对象、函数、长字符串、大数字都是堆上对象，值里存指针。

工程意义：1 + 1 不会触发分配，但 1.1 + 1.1 可能（"double 逃逸"）；+ 拼接长字符串会分配新字符串对象；BigInt 总是堆对象。在热点循环里避开这些才能拿到 C 级性能。

引用语义的真实含义

"引用传递"在 JS 里有个精确的含义：变量持有的是对象头指针的副本。因此：

  1. 把变量赋给另一变量 = 拷贝指针（两个变量指向同一对象）
  2. 把变量作为参数传给函数 = 拷贝指针（函数内改属性影响外面）
  3. 在函数内把参数重新赋值为新对象 = 只改局部指针，外面不变
javascript 
function mutate(obj) {
  obj.n = 99;           // 改对象内容 → 外部可见
}
function reassign(obj) {
  obj = { n: 99 };      // 重新赋值 → 只改局部指针
}
const a = { n: 1 };
mutate(a);    console.log(a.n); // 99
reassign(a);  console.log(a.n); // 仍为 99

Number 的本质：IEEE 754 双精度

JS 的 Number 是 64-bit 双精度浮点数：1 位符号 + 11 位指数 + 52 位尾数。

  • 安全整数范围：Number.MIN_SAFE_INTEGER = -(2⁵³ - 1)，Number.MAX_SAFE_INTEGER = 2⁵³ - 1。超出此范围整数运算不再精确，因为尾数只有 52 位。
  • 浮点运算不精确 并非 JS 独有，Python、Java、C 里 0.1 + 0.2 同样等于 0.30000000000000004。根因：0.1 和 0.2 在二进制下是无限循环小数，截断必然丢精度。
  • 特殊值：NaN、Infinity、-Infinity、-0（负零）。NaN !== NaN 是 IEEE 754 规定的，判断要用 Number.isNaN()。
javascript 
0.1 + 0.2 === 0.3                // false
0.1 + 0.2                         // 0.30000000000000004

Number.MAX_SAFE_INTEGER           // 9007199254740991
9007199254740992 === 9007199254740993  // true (都无法精确表示)

Object.is(-0, 0)                  // false  ← 唯一能区分 ±0 的方法
1 / -0                            // -Infinity

Number.isNaN(NaN)                 // true
NaN === NaN                       // false  ← IEEE 754 规定

String 的本质：UTF-16 编码单元序列

规范规定 String 是 16-bit 编码单元（code unit）的不可变序列。"abc".length 数的不是字符数，也不是 Unicode code point 数，而是 UTF-16 code unit 数。

Unicode 基本多文种平面（BMP, U+0000 ~ U+FFFF）内的字符用一个 code unit 表示；辅助平面（U+10000 ~ U+10FFFF，如 emoji、某些 CJK 扩展汉字）用两个 code unit（代理对 / surrogate pair）表示。

javascript 
"🔥".length              // 2  ← 一个字符占 2 个 code unit
"🔥".charCodeAt(0)       // 55357 (高位代理)
"🔥".charCodeAt(1)       // 56613 (低位代理)
"🔥".codePointAt(0)      // 128293  ← 真正的 code point

[..."🔥"].length                        // 1   ← 迭代器按 code point
"🔥".split("")                          // ["\uD83D", "\uDD25"]  ← 裂了
Array.from("🔥")                        // ["🔥"]
new Intl.Segmenter().segment("🔥").length // 需借助 Intl.Segmenter 做字素簇切分
⚠ 实战警告

生产代码里"截取前 N 个字符"用 str.slice(0, N) 可能切在代理对中间，把 emoji 砍成乱码。正确做法：用 Array.from(str)、[...str] 或 Intl.Segmenter。见 常见陷阱。

BigInt：任意精度整数

BigInt 解决 Number 超过 2⁵³ 后精度丢失的问题。底层是按需分配的整数数组（类似 Python 的 int）。区别于 Number：

  • 字面量后缀 n：10n。
  • 不能与 Number 直接混算（1n + 1 抛 TypeError），必须显式转换。
  • JSON.stringify 不支持 BigInt（抛 TypeError），需要自定义 replacer 或转为字符串。
  • Math.* 方法不支持 BigInt。

Symbol：值空间唯一的 key

每次调用 Symbol() 都返回全新、不相等、不可伪造的值。描述字符串（传参）只用于 debug。

  • 内部 slot：Symbol 值在引擎内部有唯一 seq_id，=== 比较本质是比较这个 id。
  • 全局 Symbol 注册表：Symbol.for("k") 总是返回同一个 Symbol，跨 realm（iframe/worker）共享。
  • Well-known symbols：Symbol.iterator、Symbol.asyncIterator、Symbol.hasInstance、Symbol.toPrimitive 等是语言约定的协议钩子。
  • 不可枚举性：Symbol 作为对象键时，Object.keys / for...in / JSON.stringify 都看不到；需要 Object.getOwnPropertySymbols。

装箱与拆箱

原始值没有方法，那 "abc".toUpperCase() 怎么可能工作？引擎在点号求值时临时包裹一个 String 对象（wrapper），调用方法后立即丢弃。这叫自动装箱（auto-boxing）。

javascript 
const s = "abc";
s.toUpperCase();          // 临时 new String(s).toUpperCase() 后丢弃
s.foo = 1;                // 赋值的也是临时 wrapper → 立刻被丢
console.log(s.foo);       // undefined

const S = new String("abc");  // 显式装箱（↓ 几乎永远不要这样写）
typeof s;                 // "string"
typeof S;                 // "object"
s === S;                  // false  ← 值 vs 对象
⛔ 永远不要 new 原始类型

new String / new Number / new Boolean 生成的是 wrapper 对象，与原始值 === 不等，会把你的代码变成 debug 噩梦。仅Symbol 和 BigInt 不能 new（会直接抛错）—— 这是引擎层面的保护。

03 应用场景

区分 null 与 undefined 的语义

undefined：变量声明未赋值、函数未返回、对象缺失属性、形参未传。由引擎产生，表达"此处没有值"。

null：程序员主动赋的"空值"。表达"此处我知道应该有值，但现在没有"。

好的团队约定：对外 API（DB/JSON/HTTP）用 null 表示业务空值；代码内部用 undefined 表示缺失。TypeScript 的 strictNullChecks + 联合类型可以把这个约定编译期强制。

BigInt 在后端 ID 场景

Java / Go 服务的 int64 ID 传到前端，JSON 数字超过 2⁵³ 会丢精度（如 Twitter 早期 snowflake ID）。解决方案：后端把 ID 序列化为字符串，前端用 BigInt 计算或仅作标识符用途。

Symbol 实现真正的"私有"属性

在 class 私有字段（#field）出现之前，Symbol 是唯一能实现"外部无法访问"属性的方式，因为 Symbol 作为 key 时被 Object.keys、for...in、JSON.stringify 忽略。现在 class 私有字段是首选，但实现通用协议（Symbol.iterator）仍然必须用 Symbol。

浮点精度在金额计算中的处理

金融场景严禁直接用 Number 做金额加减。三条标准路径：

  1. 整数化：把金额统一转成最小单位（分、厘）用整数存储计算，展示时再除。简单、快速，2⁵³ 分 ≈ 9 万亿元，对大多数业务够用。
  2. 字符串 + 库：decimal.js / big.js，任意精度但有运算开销。
  3. BigInt + 定点数：自己维护精度 scale，适合不愿引依赖的项目。

React 的 key、Vue 的 reactivity 追踪

React 用 key 的 === 判断 reconciliation，本质依赖原始值按值比较。Vue 3 的 reactive 用 WeakMap<rawObject, Proxy> 做缓存 —— WeakMap 的 key 必须是对象（引用语义）正是为了能在对象被 GC 后自动清理条目。

04 典型代码

精准判断类型

typeof 速查但不完整（null/array/date 都是 "object"）。生产代码里的 typeOf 函数：

javascript 
function typeOf(v) {
  if (v === null) return "null";
  const t = typeof v;
  if (t !== "object" && t !== "function") return t;
  // [object Array] / [object Date] / [object Map] / ...
  return Object.prototype.toString.call(v).slice(8, -1).toLowerCase();
}

typeOf(undefined);    // "undefined"
typeOf(null);         // "null"
typeOf(NaN);          // "number"
typeOf(10n);          // "bigint"
typeOf(Symbol());     // "symbol"
typeOf([]);           // "array"
typeOf(new Date());   // "date"
typeOf(/x/);          // "regexp"
typeOf(new Map());    // "map"
typeOf(() => {});     // "function"

安全的值比较

javascript 
// 1. SameValue 算法（等价于 Object.is）
Object.is(NaN, NaN);        // true
Object.is(-0, 0);           // false
Object.is({}, {});          // false（引用不同）

// 2. 对象结构相等：JSON 序列化仅适用于简单数据
JSON.stringify({ a: 1 }) === JSON.stringify({ a: 1 });  // true
// 但无法处理 undefined / 函数 / Symbol / 循环引用 / Date / Map / Set

// 3. 生产推荐
import { isEqual } from "lodash-es";  // 或 fast-deep-equal、Node.js 的 util.isDeepStrictEqual

数值边界

javascript 
// 安全整数检查
Number.isSafeInteger(2 ** 53);      // false
Number.isSafeInteger(2 ** 53 - 1);  // true

// 浮点相等用 epsilon
function approxEq(a, b, eps = Number.EPSILON * 10) {
  return Math.abs(a - b) < eps;
}
approxEq(0.1 + 0.2, 0.3);           // true

// 保留小数位（注意四舍五入规则）
(0.1 + 0.2).toFixed(2);             // "0.30"  ← 返回字符串
Number((0.1 + 0.2).toFixed(2));     // 0.3

// 整数除法
Math.trunc(7 / 2);                  // 3 (向零截断)
Math.floor(-7 / 2);                 // -4 (向负无穷)
(7 / 2) | 0;                        // 3  (位运算，仅 32-bit 整数安全)

Unicode 安全的字符串操作

javascript 
const s = "Hi👋🌍";

s.length;                           // 7 (UTF-16 code units)
[...s].length;                      // 5 (code points)

// 按字符截取
[...s].slice(0, 3).join("");        // "Hi👋"

// 正则匹配 emoji（需要 /u flag）
/./.test("👋");                     // true（按 code unit，可能对半）
/\p{Emoji}/u.test("👋");            // true（Unicode 属性转义，推荐）

// 遍历
for (const ch of s) console.log(ch);  // H, i, 👋, 🌍

// 字素簇（考虑组合字符、肤色修饰符等，ES2024+）
const seg = new Intl.Segmenter("en", { granularity: "grapheme" });
[...seg.segment("👨‍👩‍👧")].length;    // 1（一个完整 emoji 家族）

Symbol 实现迭代协议

javascript 
// 让任意对象支持 for...of / 解构 / spread
class Range {
  constructor(start, end) { this.start = start; this.end = end; }
  [Symbol.iterator]() {
    let i = this.start;
    const end = this.end;
    return {
      next() {
        return i < end
          ? { value: i++, done: false }
          : { value: undefined, done: true };
      },
    };
  }
}
for (const n of new Range(1, 4)) console.log(n);  // 1 2 3
[...new Range(1, 4)];                              // [1, 2, 3]

05 常见陷阱

typeof null === "object"

历史遗留 bug（第一版 JS 用低 3 位 tag 区分类型，对象是 000，null 恰好是全 0 指针）。判空必须 === null 或 == null（同时判 undefined）。

typeof 数组 / Date / Map 都是 "object"

用 Array.isArray()、instanceof Date、Object.prototype.toString.call(v) 等替代。注意 instanceof 跨 iframe/realm 不可靠（构造函数不同）。

NaN !== NaN

NaN 自身不相等，[NaN].includes(NaN) 反而是 true（用 SameValueZero）。判断 NaN 用 Number.isNaN()，不要用全局 isNaN()（它会先做 Number 强转，isNaN("abc") 也是 true）。

"0" / "" / [] / {} 的布尔性

布尔上下文中为 falsy: false, 0, -0, 0n, "", null, undefined, NaN。一切对象都是 truthy，包括 [], {}, new Boolean(false)。一个经典 bug：if (form.checkbox) 不能用 new Boolean(false) 的值做判断。

字符串截取切坏 emoji

"🔥🔥".slice(0, 1) 得到半个 emoji（孤立代理）。安全方法：[...str].slice(0, n).join("")。更严谨（组合字符、肤色）用 Intl.Segmenter。

+0 与 -0

0 === -0 是 true，但 1 / 0 是 Infinity，1 / -0 是 -Infinity。真需要区分用 Object.is(x, -0)。

parseInt("08") 的遗留坑

ES5 前 parseInt 对 "08"、"09" 会按八进制解析返回 0。ES5+ 必须显式传 radix：parseInt("08", 10)。更安全：Number("08") 或 +"08"。

JSON.stringify 丢失数据

undefined、函数、Symbol 值作为属性会被忽略；作为数组元素变成 null；BigInt 抛错；Date 变字符串；Map/Set 变 {}。复杂数据用 structuredClone（浏览器/Node 原生）或 devalue 等库。
