Intl 国际化 API

浏览器内置了整套基于 ICU/CLDR 数据的国际化实现（ECMA-402）。覆盖：语言敏感排序 (Collator)、日期/时间格式化 (DateTimeFormat)、数字/货币/单位 (NumberFormat)、相对时间 (RelativeTimeFormat)、列表 (ListFormat)、复数规则 (PluralRules)、字素簇/词/句切分 (Segmenter)。比第三方库（moment/dayjs/numeral）更准确、免打包、支持数百种 locale。本章是 R&D 工程师出海/本地化时必看。

阅读时长 ~25 分钟 前置 P1 字符串 · Date 规范 ECMA-402

01 速览

  • Locale 字符串：BCP-47 格式，zh-CN、en-US、zh-Hant-HK、en-GB-u-hc-h12（带扩展）。
  • 构造器通用签名：new Intl.X(locales, options)；locales 为字符串或数组（fallback 顺序）。
  • Intl.DateTimeFormat 是语言感知的 Date 格式化，比 Date.toLocaleString() 更灵活（formatToParts 可细粒度控制）。
  • Intl.NumberFormat 支持 compact（1.5K）、currency（¥1,234.56）、unit（10 km/h）、scientific、engineering。
  • Intl.PluralRules 告诉你某个数字在某语言属于哪个复数类（zero/one/two/few/many/other）—— 不要自己写 n === 1 ? ... : ...。
  • Intl.Segmenter（2023 全支持）正确切分字素簇（👨‍👩‍👧‍👦 是 1 个）、词（中文无空格也能分词）、句。
  • 性能：构造 Intl.X 实例有开销（加载 locale 数据）；常用的要缓存。

02 底层逻辑

Intl.DateTimeFormat

const d = new Date("2026-04-24T10:30:00Z");

// 预设 dateStyle / timeStyle（推荐，简洁）
new Intl.DateTimeFormat("zh-CN", { dateStyle: "long", timeStyle: "short" }).format(d);
// "2026年4月24日 18:30"（假设用户时区 UTC+8）

new Intl.DateTimeFormat("en-US", { dateStyle: "long", timeStyle: "short" }).format(d);
// "April 24, 2026 at 6:30 PM"

// 细粒度控制
new Intl.DateTimeFormat("zh-CN", {
  year: "numeric", month: "2-digit", day: "2-digit",
  hour: "2-digit", minute: "2-digit",
  timeZone: "Asia/Tokyo",
  timeZoneName: "short",
  hour12: false,
  calendar: "gregory",          // 或 "chinese"、"japanese"、"hebrew"
}).format(d);
// "2026/04/24 19:30 GMT+9"

// formatToParts：拿到每一段（for 自定义样式）
const parts = new Intl.DateTimeFormat("en").formatToParts(d);
// [{type:"month",value:"4"}, {type:"literal",value:"/"}, ...]

// formatRange（ES2021）：日期区间
const start = new Date("2026-04-20");
const end = new Date("2026-04-24");
new Intl.DateTimeFormat("en-US", { month: "short", day: "numeric" })
  .formatRange(start, end);  // "Apr 20 – 24"
new Intl.DateTimeFormat("zh-CN", { month: "long", day: "numeric" })
  .formatRange(start, end);  // "4月20日至24日"

Intl.NumberFormat

const n = 1234567.89;

// 默认：千分位
new Intl.NumberFormat("zh-CN").format(n);  // "1,234,567.89"
new Intl.NumberFormat("de-DE").format(n);  // "1.234.567,89"（德语用 . 分千，, 分小数）
new Intl.NumberFormat("hi-IN").format(n);  // "12,34,567.89"（印度分位）

// Currency
new Intl.NumberFormat("en-US", { style: "currency", currency: "USD" }).format(n);
// "$1,234,567.89"
new Intl.NumberFormat("zh-CN", { style: "currency", currency: "CNY" }).format(n);
// "¥1,234,567.89"
new Intl.NumberFormat("ja-JP", { style: "currency", currency: "JPY" }).format(n);
// "￥1,234,568"（日元无小数）

// Compact 紧凑
new Intl.NumberFormat("en", { notation: "compact" }).format(1.5e3);  // "1.5K"
new Intl.NumberFormat("zh-CN", { notation: "compact" }).format(1.5e8); // "1.5亿"

// Unit
new Intl.NumberFormat("en-US", {
  style: "unit", unit: "kilometer-per-hour", unitDisplay: "long",
}).format(60); // "60 kilometers per hour"

new Intl.NumberFormat("zh-CN", {
  style: "unit", unit: "megabyte",
}).format(50); // "50 MB"

// 科学记数
new Intl.NumberFormat("en", { notation: "scientific" }).format(1234);
// "1.234E3"

// 精度控制
new Intl.NumberFormat("en", {
  minimumFractionDigits: 2,
  maximumFractionDigits: 2,
}).format(1.5); // "1.50"

// 有效数字
new Intl.NumberFormat("en", {
  maximumSignificantDigits: 3,
}).format(1234); // "1,230"

// signDisplay：控制符号
new Intl.NumberFormat("en", { signDisplay: "always" }).format(5);  // "+5"
new Intl.NumberFormat("en", { signDisplay: "exceptZero" }).format(0); // "0"（不加 +）

Intl.RelativeTimeFormat

const rtf = new Intl.RelativeTimeFormat("zh-CN", { numeric: "auto" });
rtf.format(-1, "day");    // "昨天"（numeric:auto 触发特殊词）
rtf.format(-3, "day");    // "3天前"
rtf.format(1, "year");    // "明年"

const rtf2 = new Intl.RelativeTimeFormat("en", { numeric: "always" });
rtf2.format(-3, "day");   // "3 days ago"（不会变 yesterday）
rtf2.format(0, "day");    // "in 0 days"

// 自己封装 "距离现在"
function timeAgo(date, locale = "zh-CN") {
  const rtf = new Intl.RelativeTimeFormat(locale, { numeric: "auto" });
  const diff = (date.getTime() - Date.now()) / 1000;
  const units = [
    ["year", 31536000],
    ["month", 2592000],
    ["week", 604800],
    ["day", 86400],
    ["hour", 3600],
    ["minute", 60],
    ["second", 1],
  ];
  for (const [unit, sec] of units) {
    if (Math.abs(diff) >= sec) return rtf.format(Math.round(diff / sec), unit);
  }
  return rtf.format(0, "second");
}

Intl.Collator（语言敏感排序）

// 普通字符串比较 ≠ 字典序
["B", "a", "Z", "å"].sort(); // ["B", "Z", "a", "å"] ASCII 序

const coll = new Intl.Collator("sv", { sensitivity: "base" });
["B", "a", "Z", "å"].sort(coll.compare); // 瑞典语里 å 在 z 之后

// 选项
new Intl.Collator("zh", {
  sensitivity: "base",   // 忽略大小写、重音
  caseFirst: "upper",     // 大写在前
  numeric: true,          // "file10" 在 "file2" 之后（自然排序）
  ignorePunctuation: true,
});

// 中文拼音排序
new Intl.Collator("zh-Hans-CN").compare("中", "文"); // 负数（中 < 文 拼音 zh < wen？实际 z > w）
// 更确切：返回 +/-

Intl.PluralRules

const pr = new Intl.PluralRules("en");
pr.select(0);  // "other"  (0 apples)
pr.select(1);  // "one"
pr.select(2);  // "other"

const pl = new Intl.PluralRules("pl"); // 波兰语三种
pl.select(1);  // "one"
pl.select(2);  // "few"
pl.select(5);  // "many"
pl.select(1.5); // "other"

const ar = new Intl.PluralRules("ar"); // 阿拉伯语六种
ar.select(0);  // "zero"
ar.select(1);  // "one"
ar.select(2);  // "two"
ar.select(6);  // "few"
ar.select(100); // "many"

// 正确本地化 "X items"
function items(n, locale) {
  const pr = new Intl.PluralRules(locale);
  const table = {
    "en": { one: "1 item", other: `${n} items` },
    "pl": { one: "1 element", few: `${n} elementy`, many: `${n} elementów`, other: `${n} elementu` },
  };
  return table[locale][pr.select(n)] ?? table[locale].other;
}

new Intl.ListFormat("en", { type: "conjunction" }).format(["A", "B", "C"]);
// "A, B, and C"
new Intl.ListFormat("en", { type: "disjunction" }).format(["A", "B", "C"]);

Intl.Segmenter（2023 稳定）

// 字素簇：视觉上的"一个字"
const gs = new Intl.Segmenter("en", { granularity: "grapheme" });
[...gs.segment("a👨‍👩‍👧‍👦b")].map(s => s.segment);
// ["a", "👨‍👩‍👧‍👦", "b"]   注意 family emoji 是一个 grapheme
// 而 "a👨‍👩‍👧‍👦b".length === 13（code unit），split("") 全打乱

// 词（中文无空格也能分词）
const ws = new Intl.Segmenter("zh", { granularity: "word" });
[...ws.segment("自然语言处理很复杂")].filter(s => s.isWordLike).map(s => s.segment);
// ["自然", "语言", "处理", "很", "复杂"]

// 句
const ss = new Intl.Segmenter("en", { granularity: "sentence" });
[...ss.segment("Hello! How are you? I'm fine.")].map(s => s.segment);
// ["Hello! ", "How are you? ", "I'm fine."]

Intl.Locale（操纵 BCP-47）

const loc = new Intl.Locale("zh-Hant", { region: "HK", numberingSystem: "hanidec" });
loc.toString();      // "zh-Hant-HK-u-nu-hanidec"
loc.language;         // "zh"
loc.script;           // "Hant"
loc.region;           // "HK"
loc.maximize();       // "zh-Hant-HK"（填充缺省）
loc.minimize();       // "zh-Hant-HK"

// 从 navigator.language 构造
new Intl.Locale(navigator.language).language;

货币展示（多币种应用）

function formatCurrency(
  amount: number,
  currency: string,
  locale = navigator.language,
) {
  return new Intl.NumberFormat(locale, {
    style: "currency",
    currency,
    // 大金额用紧凑表示，小金额用精确
    notation: Math.abs(amount) >= 100000 ? "compact" : "standard",
    compactDisplay: "short",
  }).format(amount);
}

formatCurrency(1234567, "USD", "en-US"); // "$1.2M"
formatCurrency(1234, "USD", "en-US");     // "$1,234.00"
formatCurrency(1234, "USD", "ja-JP");     // "1,234.00 米ドル"

正确地切中文字符串（字素簇）

// ❌ [...str] 按 code point 切，家庭 emoji 会被切散
[..."👨‍👩‍👧"].length;   // 5（爸、ZWJ、妈、ZWJ、娃）

// ❌ str.substring(0, 10) 按 code unit，可能切断 surrogate pair
"🎉hello".substring(0, 1); // "\uD83C"（半个 emoji）

// ✅ 用 Segmenter
function truncateByGrapheme(str: string, maxGraphemes: number, locale = "en") {
  const seg = new Intl.Segmenter(locale, { granularity: "grapheme" });
  const graphemes = [...seg.segment(str)].slice(0, maxGraphemes);
  return graphemes.map(g => g.segment).join("");
}
truncateByGrapheme("👨‍👩‍👧hello", 2); // "👨‍👩‍👧h"

React 场景：缓存 formatters

// 每次渲染 new Intl.DateTimeFormat 很慢
// 按 locale 缓存

const dtfCache = new Map<string, Intl.DateTimeFormat>();
function getDtf(locale: string, opts: Intl.DateTimeFormatOptions) {
  const key = locale + JSON.stringify(opts);
  let dtf = dtfCache.get(key);
  if (!dtf) {
    dtf = new Intl.DateTimeFormat(locale, opts);
    dtfCache.set(key, dtf);
  }
  return dtf;
}

// 或用 useFormatter()（next-intl 已内置缓存）
const formatter = useFormatter();
formatter.dateTime(date, { dateStyle: "long" });

function sortByName<T>(items: T[], getName: (x: T) => string, locale = navigator.language) {
  const coll = new Intl.Collator(locale, { sensitivity: "base", numeric: true });

04 典型代码

通用时间差显示

const timeUnits: [Intl.RelativeTimeFormatUnit, number][] = [
  ["year", 31_536_000_000],
  ["month", 2_592_000_000],
  ["week", 604_800_000],
  ["day", 86_400_000],
  ["hour", 3_600_000],
  ["minute", 60_000],
  ["second", 1_000],
];

export function timeAgo(date: Date | number, locale = navigator.language): string {
  const rtf = new Intl.RelativeTimeFormat(locale, { numeric: "auto", style: "long" });
  const diff = (new Date(date).getTime()) - Date.now();
  for (const [unit, ms] of timeUnits) {
    if (Math.abs(diff) >= ms || unit === "second") {
      return rtf.format(Math.round(diff / ms), unit);
    }
  }
  return rtf.format(0, "second");
}

ICU MessageFormat 替代（复数 + 性别）

// 简化的 ICU-like 模板渲染
function formatMessage(
  template: string, // "{count, plural, one {1 message} other {# messages}}"
  vars: Record<string, number | string>,
  locale = "en",
): string {
  // 使用 intl-messageformat 库或 formatjs.io
  // Intl 原生没有完整 ICU MessageFormat，但 PluralRules 足够应对 90% 场景
  const pr = new Intl.PluralRules(locale);
  const nf = new Intl.NumberFormat(locale);

  return template.replace(/\{(\w+), plural,\s*([^}]+)\}/g, (_, name, rules) => {
    const n = Number(vars[name]);
    const form = pr.select(n);
    const map: Record<string, string> = {};
    // rules 解析从略；实际用 intl-messageformat
    return map[form] ?? map.other ?? String(n);
  });
}

05 常见陷阱

陷阱 1：硬编码 "zh-CN" 忽略用户 locale

Date.toLocaleString("zh-CN") 永远用中文格式，不尊重用户偏好。应用层从 i18n 框架（next-intl / vue-i18n）拿 locale，通过参数传入。

陷阱 2：navigator.language ≠ 应用 locale

用户浏览器可能是英文但打开中文版应用。不要读 navigator.language 做业务，应用层用自己的 locale state（cookie/URL/上下文）。

陷阱 3：每次渲染都 new Intl.X

构造 Intl 实例要加载 locale 数据，毫秒级开销。列表 1000 项每行 new → 几十 ms。缓存 formatter 实例（见上面 dtfCache 例子）。

陷阱 4：timeZone 未指定 → 用户本地时区

服务端 Next.js SSR 用 new Intl.DateTimeFormat(locale) 渲染，用户端 hydrate 可能时区不同 → hydration 不匹配 warning。始终显式 timeZone: "UTC" 或用户时区。

陷阱 5：PluralRules 不处理复数文本拼接

它只告诉你 "one" / "few" / "many"，不生成字符串。文本仍要按语言在翻译文件里写。自己写 n === 1 ? "item" : "items" 在俄语/阿拉伯语等会错得离谱。

陷阱 6：str.split("") / [...str] 切不了 emoji

[..."👨‍👩‍👧"] 有 5 个 code point（不是 1 个字）。按"字"切一定用 Intl.Segmenter。

陷阱 7：Collator.compare 的性能

用 Collator.compare 每次比较都穿过 ICU，比 a < b 慢很多倍。大量排序前用 const cmp = coll.compare 缓存函数引用。

陷阱 8：货币符号占位不一致

style: "currency" 默认前后位置、空格按 locale 决定。如果 UI 强制 "¥ 123"，用 formatToParts 重新拼接，不要硬写。

陷阱 9：JPY / KRW 无小数

日元韩元 minimumFractionDigits 默认 0。如需强制 2 位小数显式设置，但不符合当地习惯。

陷阱 10：Segmenter 在老 Safari 不可用

Safari 14.1 才支持，iOS 14 以下无。移动端要检测 "Segmenter" in Intl，fallback 到按字符粗切。
