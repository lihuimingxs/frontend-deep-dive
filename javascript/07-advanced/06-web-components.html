Web Components

Web Components 是浏览器原生的组件化模型：customElements.define() 注册自定义标签、attachShadow 隔离样式与 DOM、<slot> 组合内容。它是框架无关的：同一个 <my-dialog> 能在 React / Vue / Svelte / 纯 HTML 下工作。本章讲清生命周期、Shadow DOM 的样式隔离规则、以及与 React 的互操作陷阱。

阅读时长 ~25 分钟 前置 P5 DOM 规范 HTML · Custom Elements · Shadow DOM

01 速览

  • Custom Elements 名字必须包含连字符（my-button），与原生标签区分。
  • 两类：autonomous（继承 HTMLElement）和 customized built-in（继承 HTMLButtonElement 等，Safari 不支持）。
  • 生命周期：constructor → connectedCallback → attributeChangedCallback → disconnectedCallback → adoptedCallback。
  • Shadow DOM 隔离样式：外部 CSS 不穿透进来（除了 inherited 属性和 custom properties），内部样式不泄漏出去。
  • <slot> 让父组件塞入内容（composition），默认匿名 slot 或 name="xxx"。
  • Declarative Shadow DOM（<template shadowrootmode="open">）支持 SSR，2023 起 Chrome / Safari / Firefox 全支持。
  • form-associated custom elements（formAssociated = true）可参与 form 提交、validity。

02 底层逻辑

最小 Custom Element

class MyCounter extends HTMLElement {
  #count = 0;
  #btn;

  static observedAttributes = ["start"];

  constructor() {
    super();
    // 不能访问 attributes / children（可能还未 parse）
    const shadow = this.attachShadow({ mode: "open" });
    shadow.innerHTML = `
      <style>
        button { font: inherit; padding: 8px 12px; }
        :host([disabled]) button { opacity: 0.5; pointer-events: none; }
      </style>
      <button type="button">0</button>
    `;
    this.#btn = shadow.querySelector("button");
    this.#btn.addEventListener("click", () => this.#increment());
  }

  connectedCallback() {
    // 插入到 DOM 时调用；可读 attributes
    this.#render();
  }

  disconnectedCallback() {
    // 从 DOM 移除时清理（取消订阅、移除外部监听）
  }

  attributeChangedCallback(name, oldValue, newValue) {
    if (name === "start") {
      this.#count = Number(newValue) || 0;
      this.#render();
    }
  }

  adoptedCallback() {
    // 元素被移到新 document（iframe 间移动）
  }

  #increment() {
    this.#count++;
    this.#render();
    this.dispatchEvent(new CustomEvent("change", { detail: this.#count, bubbles: true }));
  }

  #render() {
    this.#btn.textContent = this.#count;
  }
}

customElements.define("my-counter", MyCounter);
<my-counter start="10"></my-counter>
<script>
  document.querySelector("my-counter").addEventListener("change", (e) => {
    console.log(e.detail); // 11, 12, ...
  });
</script>

生命周期顺序

new MyCounter()                ← constructor（不操作 attributes/children）
  ↓
Parser/JS 插入到文档
  ↓
attributeChangedCallback(name)  ← 每个 observed attribute 初始值 = null → 现值
  ↓
connectedCallback()            ← DOM 完整、可操作
  ↓
[... 运行中 ...]
  ↓
disconnectedCallback()
  ↓
[也许 adoptedCallback → 另一 connectedCallback]

Shadow DOM 隔离规则

项 穿透 shadow？
querySelector（light） ❌ 外部查不到内部节点
外部 CSS p { color: red } ❌ 不影响 shadow 内 p
inherited CSS（color、font） ✅ 透传
CSS custom properties（--x） ✅ 透传
事件（默认 bubbles） ✅ 穿透到外部（composed: true 时）
focus ✅ 穿透（元素可接受焦点）
form 提交 / validation ❌ 需 form-associated（下述）

Shadow 内的样式选择器

/* :host = shadow 宿主元素 */
:host { display: block; }
:host([disabled]) { opacity: 0.5; }
:host(.active) { border-color: blue; }

/* :host-context(selector) = 宿主 matches 某父元素 */
:host-context(.dark-theme) { color: white; }

/* ::slotted(selector) = 被 slot 投影的元素（仅一层深度） */
::slotted(h2) { font-size: 1.5em; }
::slotted([slot="icon"]) { color: red; }

/* ::part(name) = 暴露给外部的"样式锚点" */
::part(button) { ... } /* 外部写 */
<button part="button">Click</button> /* 内部标记 */

Slot 组合

<!-- 定义 -->
<template id="dialog-template">
  <style>
    header { background: #eee; }
    footer { background: #fafafa; }
  </style>
  <header><slot name="title"></slot></header>
  <main><slot><!-- 默认内容 --></slot></main>
  <footer><slot name="actions"></slot></footer>
</template>

<!-- 使用 -->
<my-dialog>
  <h2 slot="title">Confirm</h2>
  <p>Are you sure?</p>
  <button slot="actions" type="button">OK</button>
  <button slot="actions" type="button">Cancel</button>
</my-dialog>

Declarative Shadow DOM（SSR）

<!-- 服务端直接返回，无需 JS 就能渲染 shadow -->
<my-card>
  <template shadowrootmode="open">
    <style>h2 { color: blue }</style>
    <h2><slot name="title"></slot></h2>
    <slot></slot>
  </template>
  <span slot="title">Hello</span>
  <p>content</p>
</my-card>

浏览器解析 <template shadowrootmode> 时自动 attachShadow + 填入内容，JS 端 customElements.define 后升级到完整元素。配合 <link rel="stylesheet"> 可让样式提前应用（避免 FOUC）。

Form-Associated Custom Element

class MyInput extends HTMLElement {
  static formAssociated = true;
  static observedAttributes = ["value"];

  #internals;

  constructor() {
    super();
    this.attachShadow({ mode: "open" }).innerHTML = `<input type="text">`;
    this.#internals = this.attachInternals();
    const input = this.shadowRoot.querySelector("input");
    input.addEventListener("input", () => {
      this.#internals.setFormValue(input.value);
      this.#internals.setValidity(
        input.value ? {} : { valueMissing: true },
        "请输入值",
      );
    });
  }

  // 被 form.reset() 调用
  formResetCallback() { this.shadowRoot.querySelector("input").value = ""; }
  formDisabledCallback(disabled) { /* ... */ }
  formStateRestoreCallback(state, mode) { /* 历史恢复 */ }
}
customElements.define("my-input", MyInput);

03 应用场景

设计系统的跨框架组件

// 同一个 <design-button> 能在 React/Vue/纯 HTML 用
class DesignButton extends HTMLElement {
  static observedAttributes = ["variant", "disabled"];
  // ... 基于 attribute 渲染
}
customElements.define("design-button", DesignButton);

// React 使用（React 19+ 支持传 props 映射到 attribute）
<design-button variant="primary">Save</design-button>

// Vue 使用（vue.config: isCustomElement）
<design-button :variant="variant">Save</design-button>

Lit、FAST、Stencil 是流行的 WC 开发库，都生成标准的 custom element，可在任何框架使用。Shoelace / Spectrum / Material Web 都是基于 WC 的设计系统。

第三方小部件嵌入

Stripe Elements、Google Maps、Intercom chat widget 等越来越多用 Web Components + Shadow DOM：样式不被宿主站点污染，也不污染宿主。iframe 曾经是隔离方案，但 WC 无 iframe 的跨源通信问题。

渐进增强

<!-- 老浏览器或 JS 挂之前：显示基本 html -->
<video-player src="a.mp4">
  <video controls src="a.mp4"></video> <!-- fallback -->
</video-player>

<script>
  class VideoPlayer extends HTMLElement {
    connectedCallback() {
      // 替换 fallback 为自定义 UI
      this.attachShadow({ mode: "open" });
      // ...
    }
  }
  customElements.define("video-player", VideoPlayer);
</script>

富文本编辑器组件

class RichEditor extends HTMLElement {
  static formAssociated = true;

  constructor() {
    super();
    const shadow = this.attachShadow({ mode: "open" });
    shadow.innerHTML = `
      <div part="toolbar" role="toolbar">
        <button type="button" data-cmd="bold">B</button>
        <button type="button" data-cmd="italic">I</button>
      </div>
      <div part="editor" contenteditable="true"></div>
    `;
    this.internals = this.attachInternals();
    this.editor = shadow.querySelector('[part="editor"]');
    this.editor.addEventListener("input", () => {
      this.internals.setFormValue(this.editor.innerHTML);
    });
    shadow.querySelectorAll("button").forEach(btn => {
      btn.addEventListener("click", () => document.execCommand(btn.dataset.cmd));
    });
  }
}
customElements.define("rich-editor", RichEditor);

04 典型代码

响应式 Custom Element（属性 ↔ 属性同步）

class MyToggle extends HTMLElement {
  static observedAttributes = ["checked"];

  get checked() { return this.hasAttribute("checked"); }
  set checked(v) {
    if (v) this.setAttribute("checked", "");
    else this.removeAttribute("checked");
  }

  attributeChangedCallback(name, _old, val) {
    if (name === "checked") this.#render();
  }

  connectedCallback() { this.#render(); this.#bind(); }

  #render() {
    this.shadowRoot.innerHTML = `
      <style>
        :host { display: inline-block; cursor: pointer; }
        .track { width: 32px; height: 18px; border-radius: 9px; background: #ccc; }
        :host([checked]) .track { background: #3b82f6; }
      </style>
      <div class="track"></div>`;
  }

  #bind() {
    this.addEventListener("click", () => {
      this.checked = !this.checked;
      this.dispatchEvent(new Event("change", { bubbles: true }));
    });
  }

  constructor() { super(); this.attachShadow({ mode: "open" }); }
}
customElements.define("my-toggle", MyToggle);

Lit 最小示例（推荐生产用）

import { LitElement, html, css } from "lit";
import { property, customElement } from "lit/decorators.js";

@customElement("my-greeting")
export class MyGreeting extends LitElement {
  static styles = css`h1 { color: var(--primary, blue); }`;

  @property() name = "World";

  render() {
    return html`<h1>Hello, ${this.name}!</h1>`;
  }
}

Lit（Google 出品）提供响应式 property、模板字符串 HTML、样式隔离，产物是标准 Web Component。对长期项目和跨框架设计系统非常合适。

与 React 互操作（React 19+）

// React 18 及以前只能 setAttribute 字符串；19 支持 property 赋值
// 传递对象/数组必须用 ref 手动设置

function App() {
  const ref = useRef<HTMLElement>(null);

  useEffect(() => {
    if (ref.current) {
      ref.current.data = complexObject; // 作为 property 赋值
    }
  }, [complexObject]);

  return (
    <my-chart
      ref={ref}
      // 字符串 attribute 可直接写
      theme="dark"
      // React 19+ 会尝试用 property
      labels={labels}
    />
  );
}

05 常见陷阱

陷阱 1：constructor 里访问 attributes / children

HTML parser 可能还未把 attributes / children 解析完。移到 connectedCallback。observedAttributes 的初始值会通过 attributeChangedCallback 传达。

陷阱 2：name 不带连字符

customElements.define("mybutton", ...) 抛 SyntaxError。名字必须含连字符（"-"），且不能是已保留的（如 annotation-xml）。

陷阱 3：事件 composed: false 无法穿透 shadow

默认 new Event("change") 的 composed 是 false，shadow 外监听不到。要广播事件加 { bubbles: true, composed: true }。click 等原生事件默认 composed: true。

陷阱 4：External CSS 不影响 shadow 内

用 <style> 内联到 shadow 里，或用 :host 让外部通过 attribute 控制，或用 CSS custom properties（--brand-color）。

陷阱 5：React 传 boolean / object 作为 attribute

React 18 会 toString 后 setAttribute → attr="true" 或 attr="[object Object]"。React 19 / Preact 支持 property 赋值，或用 ref 手动设。

陷阱 6：form-associated 在 Safari 16 以下不可用

formAssociated 2022 才全浏览器支持。旧 Safari 必须 fallback：用 hidden input 同步值到 form。

陷阱 7：Shadow DOM 性能

每个 shadow root 是独立的"树"，选择器全部重新匹配。一个页面几千个 Web Component 可能比普通 DOM 慢。量大用 closedShadow（外部不可查）或避免 shadow。

陷阱 8：Adopted Stylesheets 跨 shadow 复用

多个 shadow root 各自内联 <style> 会重复解析。用 const sheet = new CSSStyleSheet(); sheet.replaceSync(css); shadow.adoptedStyleSheets = [sheet]; 共享。

陷阱 9：slotchange 事件没触发

slot.addEventListener("slotchange", ...) 只在 slot 被赋值/取消赋值时触发，不在 slotted 元素内部改变时。需要后者用 MutationObserver。

陷阱 10：upgrade 顺序

HTML parser 遇到 <my-el> 时 customElements.define 还没执行，元素暂是 "undefined"；等 define 被调用后升级（调用 constructor + connectedCallback）。要等 customElements.whenDefined("my-el") 再操作属性。
