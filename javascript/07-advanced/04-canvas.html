Canvas / OffscreenCanvas

Canvas 2D 是浏览器里最成熟的像素绘制 API：图形编辑、数据可视化、图片处理、PDF 渲染都靠它。OffscreenCanvas 把渲染搬到 Worker，避免复杂绘制阻塞 UI。本章讲清状态栈机制、HiDPI 缩放、ImageData 的像素操作，以及何时该升级到 WebGL/WebGPU。

阅读时长 ~22 分钟 前置 01 Binary · P4 Workers 规范 HTML · Canvas

01 速览

  • Canvas 绘制 API 立即生效：每次 fill() / stroke() 都直接改像素，没有场景图（与 SVG 相反）。
  • Canvas 的状态栈（save/restore）管理变换、样式、裁剪区、alpha、compositeOperation；容易遗漏导致状态泄漏。
  • HiDPI 缩放：canvas 的 width/height（位图尺寸）要 × devicePixelRatio，CSS size 保持逻辑像素，ctx.scale(dpr, dpr) 放大绘制。
  • OffscreenCanvas 可在 Worker 中运行；主线程用 transferControlToOffscreen() 转交渲染权，UI 不会被卡。
  • ImageData：Uint8ClampedArray 按 RGBA 顺序排列，[R,G,B,A, R,G,B,A...]；willReadFrequently: true 让 getImageData 快 10 倍。
  • Canvas 2D 不能做复杂的 3D / 海量粒子 → 升级 WebGL；WebGPU（2024 GA）是未来，更像 Vulkan/Metal。

02 底层逻辑

获取 Context 与基本绘制

const canvas = document.querySelector("canvas");
const ctx = canvas.getContext("2d", {
  alpha: false,              // 不需要透明 → 背景白，合成少一步
  desynchronized: true,      // 允许 GPU 异步呈现（低延迟场景）
  willReadFrequently: false, // 若会频繁 getImageData，设 true（CPU-backed）
  colorSpace: "display-p3",  // 宽色域（iOS/macOS 可更鲜艳）
});

// 矩形快捷方法
ctx.fillStyle = "#f00";
ctx.fillRect(10, 10, 100, 50);
ctx.strokeStyle = "blue";
ctx.lineWidth = 2;
ctx.strokeRect(10, 10, 100, 50);
ctx.clearRect(0, 0, canvas.width, canvas.height); // 清空

// 路径（Path）API
ctx.beginPath();
ctx.moveTo(50, 50);
ctx.lineTo(200, 50);
ctx.quadraticCurveTo(250, 100, 200, 150);
ctx.closePath();
ctx.fill();
ctx.stroke();

// 独立 Path2D 对象（可缓存复用，比每次 beginPath 快）
const p = new Path2D("M10 10 h 80 v 80 h -80 Z"); // 支持 SVG path 字符串
ctx.fill(p);

状态栈 save / restore

ctx.save();
ctx.translate(100, 100);
ctx.rotate(Math.PI / 4);
ctx.scale(2, 2);
ctx.fillStyle = "red";
ctx.clip(path);
// ...绘制旋转+缩放+裁剪的内容
ctx.restore(); // 恢复所有状态（变换、样式、裁剪、alpha...）

// 栈深度建议 ≤ 数十层；过深影响性能
// 每次 save 必须对应一次 restore，否则状态泄漏到后续绘制

受 save/restore 影响的状态：fillStyle / strokeStyle / globalAlpha / lineWidth / font / textAlign / 变换矩阵 / 裁剪区 / globalCompositeOperation。不受影响：当前路径、像素内容、canvas.width/height。

HiDPI（Retina）正确缩放

function setupCanvas(canvas, cssWidth, cssHeight) {
  const dpr = window.devicePixelRatio || 1;
  canvas.style.width = cssWidth + "px";
  canvas.style.height = cssHeight + "px";
  canvas.width = Math.round(cssWidth * dpr);
  canvas.height = Math.round(cssHeight * dpr);

  const ctx = canvas.getContext("2d");
  // 之后所有坐标都用 CSS 像素（dpr 自动放大）
  ctx.scale(dpr, dpr);
  return ctx;
}

// Retina 屏 dpr=2，200×100 CSS 像素 → 400×200 位图，字体和线条都清晰

不做这步：Retina 屏上文字/线条模糊；做了 canvas.width = cssWidth * dpr 但忘 ctx.scale(dpr, dpr)：内容绘制成原尺寸，显示只占 1/4。

ImageData 像素操作

// 读像素
const img = ctx.getImageData(0, 0, canvas.width, canvas.height);
// img.data: Uint8ClampedArray, length = w * h * 4
// [R₀, G₀, B₀, A₀, R₁, G₁, B₁, A₁, ...]

// 灰度
for (let i = 0; i < img.data.length; i += 4) {
  const gray = 0.299 * img.data[i] + 0.587 * img.data[i + 1] + 0.114 * img.data[i + 2];
  img.data[i] = img.data[i + 1] = img.data[i + 2] = gray;
  // img.data[i+3] 是 alpha
}

// 写回
ctx.putImageData(img, 0, 0);

// 新建 ImageData
const blank = new ImageData(100, 100); // 全 0
const colored = new ImageData(new Uint8ClampedArray([255,0,0,255]), 1, 1);

性能：getImageData 触发 GPU→CPU 同步，默认慢；willReadFrequently: true 让 canvas 用 CPU-backed 存储，getImageData 零开销但绘制稍慢。只读模式下建议开。

OffscreenCanvas

// 方式 1：直接在 Worker 新建
// worker.js
const canvas = new OffscreenCanvas(400, 300);
const ctx = canvas.getContext("2d");
ctx.fillRect(0, 0, 100, 100);
const bitmap = canvas.transferToImageBitmap(); // 产生帧
postMessage({ bitmap }, [bitmap]);

// 方式 2：主线程把 HTMLCanvasElement 的渲染权转给 Worker
// main.js
const main = document.querySelector("canvas");
const offscreen = main.transferControlToOffscreen();
// main 这时不能再 getContext，所有绘制在 Worker 里
worker.postMessage({ canvas: offscreen }, [offscreen]);

// worker.js
self.onmessage = (e) => {
  const ctx = e.data.canvas.getContext("2d");
  // Worker 里绘制不阻塞主线程；绘制结果直接出现在页面
};

ImageBitmap：已解码图像

// 从 Blob / File / <img> / <canvas> 创建
const bitmap = await createImageBitmap(blob, {
  resizeWidth: 800,
  resizeHeight: 600,
  resizeQuality: "high",
});

// ImageBitmap 是 Transferable → 可零拷贝发到 Worker
worker.postMessage({ bitmap }, [bitmap]);
// Worker 里直接 ctx.drawImage(bitmap, 0, 0)；不需重新解码

bitmap.close(); // 释放 GPU 内存，不 close 会泄漏

导出 Canvas 为图片

// 同步（base64）：大图慢
const dataUrl = canvas.toDataURL("image/png");
const webp = canvas.toDataURL("image/webp", 0.85);

// 异步（Blob）：推荐
canvas.toBlob((blob) => {
  downloadBlob(blob, "export.png");
}, "image/png");

// OffscreenCanvas 版本
const blob = await offscreen.convertToBlob({ type: "image/png" });

CompositeOperation（混合模式）

ctx.globalCompositeOperation = "source-over";   // 默认
// destination-out: 现有内容减去新绘制形状（擦除）
// multiply / screen / overlay / lighten: Photoshop 混合模式
// difference / hue / saturation / color / luminosity (Canvas 2D Level 2)

// 擦除效果
ctx.globalCompositeOperation = "destination-out";
ctx.beginPath();
ctx.arc(x, y, r, 0, 2 * Math.PI);
ctx.fill();

03 应用场景

图片压缩 + 格式转换

async function compress(file: File, maxW: number, quality = 0.8, format: "image/webp" | "image/jpeg" = "image/webp") {
  const bitmap = await createImageBitmap(file);
  const ratio = Math.min(1, maxW / bitmap.width);
  const w = Math.round(bitmap.width * ratio);
  const h = Math.round(bitmap.height * ratio);

  const canvas = new OffscreenCanvas(w, h);
  const ctx = canvas.getContext("2d")!;
  ctx.drawImage(bitmap, 0, 0, w, h);
  bitmap.close();

  return canvas.convertToBlob({ type: format, quality });
}

简易画板

const ctx = setupCanvas(canvas, 800, 600);
ctx.lineCap = "round";
ctx.lineJoin = "round";
ctx.lineWidth = 3;

let drawing = false;
canvas.addEventListener("pointerdown", (e) => {
  drawing = true;
  canvas.setPointerCapture(e.pointerId);
  ctx.beginPath();
  ctx.moveTo(e.offsetX, e.offsetY);
});
canvas.addEventListener("pointermove", (e) => {
  if (!drawing) return;
  // PointerEvent 支持压感
  ctx.lineWidth = 3 + e.pressure * 8;
  ctx.lineTo(e.offsetX, e.offsetY);
  ctx.stroke();
});
canvas.addEventListener("pointerup", () => { drawing = false; });

Worker 里做复杂 rendering（不阻塞 UI）

// main.ts
const offscreen = canvas.transferControlToOffscreen();
const worker = new Worker(new URL("./render-worker.ts", import.meta.url), { type: "module" });
worker.postMessage({ type: "init", canvas: offscreen }, [offscreen]);
worker.postMessage({ type: "draw", data: largeDataset });

// render-worker.ts
let ctx: OffscreenCanvasRenderingContext2D;
self.onmessage = (e) => {
  if (e.data.type === "init") {
    ctx = e.data.canvas.getContext("2d")!;
  } else if (e.data.type === "draw") {
    for (const point of e.data.data) {
      ctx.fillRect(point.x, point.y, 1, 1); // 主线程 0 开销
    }
  }
};

屏幕截图（html2canvas 替代）

Canvas 不能直接截图 DOM 元素（安全隔离）。SVG <foreignObject> + drawImage(svgBlob) 是一个技巧，但跨源资源会污染 canvas。生产库：html2canvas、modern-screenshot。截图包含视频 → 用 navigator.mediaDevices.getDisplayMedia()。

可视化大数据（分块绘制）

// 百万数据点：一次性循环会卡住主线程
function drawLargeDataset(data: Point[]) {
  const CHUNK = 5_000;
  let i = 0;
  function tick() {
    const end = Math.min(i + CHUNK, data.length);
    for (; i < end; i++) ctx.fillRect(data[i].x, data[i].y, 1, 1);
    if (i < data.length) requestAnimationFrame(tick); // 下一帧继续
  }
  requestAnimationFrame(tick);
}

04 典型代码

useCanvas Hook

import { useEffect, useRef } from "react";

export function useCanvas(
  draw: (ctx: CanvasRenderingContext2D, size: { w: number; h: number }) => void,
) {
  const ref = useRef<HTMLCanvasElement | null>(null);

  useEffect(() => {
    const canvas = ref.current;
    if (!canvas) return;

    const dpr = window.devicePixelRatio || 1;
    const resize = () => {
      const rect = canvas.getBoundingClientRect();
      canvas.width = Math.round(rect.width * dpr);
      canvas.height = Math.round(rect.height * dpr);
      const ctx = canvas.getContext("2d")!;
      ctx.setTransform(dpr, 0, 0, dpr, 0, 0);
      draw(ctx, { w: rect.width, h: rect.height });
    };

    const ro = new ResizeObserver(resize);
    ro.observe(canvas);
    return () => ro.disconnect();
  }, [draw]);

  return ref;
}

EMA 平滑笔触

// 直接 lineTo 每个鼠标点会产生锯齿
// 二次曲线用"当前点和上一点的中点"做控制点，线条更平滑

let prev: { x: number; y: number } | null = null;
function drawSmooth(ctx: CanvasRenderingContext2D, pt: { x: number; y: number }) {
  if (!prev) { ctx.beginPath(); ctx.moveTo(pt.x, pt.y); prev = pt; return; }
  const mid = { x: (prev.x + pt.x) / 2, y: (prev.y + pt.y) / 2 };
  ctx.quadraticCurveTo(prev.x, prev.y, mid.x, mid.y);
  ctx.stroke();
  prev = pt;
}

像素采样（取颜色）

function getPixel(ctx: CanvasRenderingContext2D, x: number, y: number) {
  // 只取 1×1，避免全画布扫描
  const { data } = ctx.getImageData(x, y, 1, 1);
  return { r: data[0], g: data[1], b: data[2], a: data[3] };
}

// 颜色选择器场景：每次移动光标都 getImageData，注意 willReadFrequently

05 常见陷阱

陷阱 1：CSS 拉伸 vs 位图尺寸

忘了设 canvas.width / height，只用 CSS 拉大 → 绘制内容被拉伸模糊。width/height 是位图尺寸，style.width/height 是显示尺寸，两套不相关。

陷阱 2：save 没对应 restore

循环里 save() → translate → 绘制，但某条件分支忘 restore，栈越涨越深，状态也跑飞。严格 save/restore 配对，用 try/finally 包裹。

陷阱 3：修改 canvas.width 清空全部内容

canvas.width = newW 即使新值等于旧值也会清空画布 + 重置所有状态到默认。resize 要先读出当前状态再恢复，或用离屏 canvas 缓存。

陷阱 4：跨源图片污染 canvas

drawImage 跨源图片后，canvas 被标记 "tainted"：getImageData、toBlob、toDataURL 全抛 SecurityError。需要 img.crossOrigin = "anonymous" + 服务端返回 Access-Control-Allow-Origin。

陷阱 5：fillText 的基线对不齐

默认 textBaseline = "alphabetic"（字母底线，不含下沿）；想 Y 坐标就是字符框顶部设 textBaseline = "top"；居中设 "middle"。不同字体不一致，需实测。

陷阱 6：globalAlpha 叠加 ≠ CSS opacity

多个半透明形状叠加会变深（累加不透明度）。想要 CSS 的 opacity 效果，先把内容画到离屏 canvas，再以 globalAlpha 一次性 drawImage 过来。

陷阱 7：OffscreenCanvas 的浏览器支持

Safari 16.4+ 才支持 OffscreenCanvas。老 Safari 需检测降级到主线程 canvas。

陷阱 8：ImageBitmap 不 close 泄漏 GPU 内存

每次 createImageBitmap 占用 GPU 纹理；频繁创建不 close，Chrome 会 OOM 重载页面。用完 bitmap.close()。

陷阱 9：toDataURL 卡主线程

几 MB canvas 的 toDataURL 同步执行，主线程几十到几百 ms 卡死。改用 toBlob（内部 async）或 OffscreenCanvas + convertToBlob。

陷阱 10：Canvas 的最大尺寸限制

不同浏览器有不同限制：Safari iOS 约 4096×4096 或 16M 像素；Chrome 约 32k×32k。超过会静默变黑/失败。超大图像要切片渲染。
