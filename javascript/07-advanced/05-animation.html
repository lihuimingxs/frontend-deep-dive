Animation & requestAnimationFrame

浏览器把每一帧分为：JS 运行 → 样式计算 → 布局 → 绘制 → 合成。要做流畅动画必须知道如何在帧预算（60fps 下 16.67ms，120Hz 下 8.33ms）内完成 JS 工作，以及哪些 CSS 属性可以"只合成不布局"。本章讲清 RAF 的时序语义、Web Animations API 的编程式动画、FLIP 技术、以及 2024 新的 scroll-timeline。

阅读时长 ~22 分钟 前置 P3 Event Loop · P5 Observer 规范 W3C Web Animations

01 速览

  • requestAnimationFrame(cb)：cb 在浏览器下一次重绘前执行；后台标签自动暂停；时间戳是 DOMHighResTimeStamp（ms，带小数）。
  • 帧预算：60Hz → 16.67ms，120Hz → 8.33ms。JS + 样式计算 + 布局必须在此内完成，否则丢帧（jank）。
  • 只触发合成（compositing）的属性最便宜：transform、opacity、filter、clip-path。改 left/top/width/height 触发布局（layout），重则 reflow 整个页面。
  • Web Animations API：element.animate(keyframes, options) 返回 Animation 对象，可 play/pause/reverse/finished Promise。
  • FLIP（First/Last/Invert/Play）：用 transform 反演布局差异，把 layout 动画变成合成动画。
  • CSS @scroll-timeline / view-timeline（2024）：滚动驱动动画，无需 JS。

02 底层逻辑

一帧的生命周期（浏览器渲染管线）

vsync 信号（60Hz 每 16.67ms）
  ↓
1. Input（处理 pointer/keyboard 事件）
2. JS（定时器回调、Promise microtask、requestAnimationFrame）
3. Style（计算元素样式）
4. Layout（计算位置与尺寸；若上一步改了几何属性）
5. Paint（光栅化绘制到 layer）
6. Composite（把 layer 合成到屏幕；GPU）
  ↓
  IntersectionObserver / ResizeObserver callback（本帧末）
  requestIdleCallback（如还有空闲时间）
  ↓
vsync：送画面到显示

rAF 的时序语义

requestAnimationFrame((ts) => {
  // ts: DOMHighResTimeStamp，跟 performance.now() 同起点
  // 所有同一帧触发的 rAF 回调共享同一个 ts（重要！）
});

// 错误示范：两次 rAF 拿不同时间戳
rAF((ts1) => {
  rAF((ts2) => {
    console.log(ts2 - ts1); // ≈ 16.67，不是 0
  });
});

// 后台标签切换时 rAF 自动暂停，不消耗 CPU
// setTimeout 不会暂停 → 用 setTimeout 做动画会在后台跑飞

rAF vs setTimeout(0)

rAF setTimeout(0)
频率 跟屏幕刷新率（60/90/120Hz） 浏览器节流至 ~4ms（深嵌或后台 1s）
后台标签 暂停 照常执行（耗电）
时序 Paint 前 下一个宏任务
时间戳 高精度、同帧共享 自己 Date.now() 拿
视觉同步 天然跟 vsync 对齐 可能在中间触发，引起撕裂

属性成本分层

层级 成本 典型属性
Composite only GPU 合成，最便宜 transform、opacity、filter（部分）、will-change 声明层
Paint CPU 重绘当前层 color、background、box-shadow
Layout 重排几何 + 可能级联 width、height、top/left、font-size、display

动画永远用 transform + opacity。需要移动位置用 transform: translate(x, y) 而非 left/top。

Web Animations API

const anim = element.animate(
  [
    { transform: "translateX(0) rotate(0)", opacity: 1 },
    { transform: "translateX(100px) rotate(90deg)", opacity: 0.5, offset: 0.7 },
    { transform: "translateX(200px) rotate(180deg)", opacity: 0 },
  ],
  {
    duration: 1000,
    iterations: 2,
    direction: "alternate",
    easing: "cubic-bezier(0.4, 0, 0.2, 1)",
    fill: "forwards",
  },
);

anim.pause();
anim.play();
anim.reverse();
anim.playbackRate = 2; // 2 倍速

// 监听完成
await anim.finished;
console.log("done");

// 取消
anim.cancel();

// 监听所有动画
element.getAnimations().forEach(a => a.cancel());

CSS transition / animation 协作

/* CSS 负责样式状态过渡，JS 切换 class */
.box { transition: transform 300ms cubic-bezier(0.4, 0, 0.2, 1); }
.box.moved { transform: translateX(100px); }

// JS 触发 → 交给 CSS 渲染
box.classList.add("moved");
// 等动画结束
await new Promise(r => box.addEventListener("transitionend", r, { once: true }));

优势：CSS transition 在合成线程运行，JS 主线程卡时动画仍流畅（除非属性触发 layout）。

FLIP 技术

// 场景：列表重排后元素位置变了，要做平滑过渡
// 1. First：记录旧位置
const first = element.getBoundingClientRect();

// 2. Last：改 DOM 到新位置（瞬间）
parent.appendChild(element);
const last = element.getBoundingClientRect();

// 3. Invert：用 transform 把元素反向拉回到旧位置（视觉不变）
const dx = first.left - last.left;
const dy = first.top - last.top;
element.animate(
  [{ transform: `translate(${dx}px, ${dy}px)` }, { transform: "translate(0)" }],
  { duration: 400, easing: "ease-in-out" },
);
// 4. Play：上一步的动画就是"从旧位置平滑回到新位置"

关键：只用 transform 做动画，不碰布局属性。整个"重排"对合成线程是瞬时的 + 一个 transform 动画。

View Transitions API（跨页面动画）

// Chrome 111+：自动产生两个快照（before/after）并在 CSS 中过渡
if (document.startViewTransition) {
  document.startViewTransition(() => {
    updateDOM();
  });
}

// CSS 自定义
::view-transition-old(root) { animation: fade-out 300ms; }
::view-transition-new(root) { animation: fade-in 300ms; }

// 跨文档（MPA）view transitions 2024 也已可用
@view-transition { navigation: auto; }

scroll-timeline / view-timeline（2024）

/* 滚动位置驱动动画，不需 JS */
@keyframes fade-in {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.item {
  animation: fade-in linear;
  animation-timeline: view();      /* 元素进入视口时的进度 */
  animation-range: entry 0% cover 30%;
}

03 应用场景

rAF 驱动的游戏循环

let last = performance.now();
let running = true;

function loop(now: number) {
  if (!running) return;
  const dt = (now - last) / 1000; // 秒
  last = now;

  update(dt);    // 基于 dt 移动物体（帧率无关）
  render();

  requestAnimationFrame(loop);
}

requestAnimationFrame(loop);

// 暂停 / 恢复
document.addEventListener("visibilitychange", () => {
  if (document.hidden) running = false;
  else { running = true; last = performance.now(); requestAnimationFrame(loop); }
});

滚动节流（结合 rAF）

let ticking = false;
window.addEventListener("scroll", () => {
  if (ticking) return;
  ticking = true;
  requestAnimationFrame(() => {
    updatePosition(); // 同帧内只跑一次，避免每个 scroll 事件都算
    ticking = false;
  });
}, { passive: true }); // passive: 不阻塞滚动

列表项 FLIP 动画（React）

// 记录 + 反演 + 播放
const refs = useRef(new Map<string, DOMRect>());

// 每次 render 前记录
useLayoutEffect(() => {
  for (const el of container.current.children) {
    refs.current.set(el.dataset.id, el.getBoundingClientRect());
  }
});

// 列表变化后触发 FLIP
useEffect(() => {
  for (const el of container.current.children) {
    const id = el.dataset.id;
    const oldRect = refs.current.get(id);
    const newRect = el.getBoundingClientRect();
    if (!oldRect) continue;
    const dx = oldRect.left - newRect.left;
    const dy = oldRect.top - newRect.top;
    if (dx || dy) {
      el.animate(
        [{ transform: `translate(${dx}px, ${dy}px)` }, { transform: "none" }],
        { duration: 300, easing: "cubic-bezier(0.4, 0, 0.2, 1)" },
      );
    }
  }
}, [items]);

数字滚动 / 计数动画

function animateNumber(el: HTMLElement, from: number, to: number, duration = 800) {
  const start = performance.now();
  function tick(now: number) {
    const t = Math.min(1, (now - start) / duration);
    const eased = 1 - Math.pow(1 - t, 3); // easeOutCubic
    el.textContent = String(Math.round(from + (to - from) * eased));
    if (t < 1) requestAnimationFrame(tick);
  }
  requestAnimationFrame(tick);
}

骨架屏 / 加载动画

.skeleton {
  background: linear-gradient(90deg, #eee 0%, #f5f5f5 50%, #eee 100%);
  background-size: 200% 100%;
  animation: shimmer 1.5s linear infinite;
}
@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}

用 background-position 属于 paint 层，比 left 好但不是合成。纯合成方案是带 gradient 的 pseudo-element + transform。

04 典型代码

缓动函数库（无依赖）

const easing = {
  linear: (t: number) => t,
  easeInQuad: (t: number) => t * t,
  easeOutQuad: (t: number) => t * (2 - t),
  easeInOutQuad: (t: number) => t < 0.5 ? 2 * t * t : -1 + (4 - 2 * t) * t,
  easeInCubic: (t: number) => t ** 3,
  easeOutCubic: (t: number) => 1 - (1 - t) ** 3,
  easeInOutCubic: (t: number) => t < 0.5 ? 4 * t ** 3 : 1 - (-2 * t + 2) ** 3 / 2,
  easeOutBack: (t: number) => 1 + 2.70158 * (t - 1) ** 3 + 1.70158 * (t - 1) ** 2,
  easeOutElastic: (t: number) => {
    if (t === 0 || t === 1) return t;
    return Math.pow(2, -10 * t) * Math.sin((t - 0.0375) * (2 * Math.PI) / 0.15) + 1;
  },
};

prefers-reduced-motion 兼容

const motionSafe = () => !window.matchMedia("(prefers-reduced-motion: reduce)").matches;

function animate(el: Element, keyframes: Keyframe[], opts: KeyframeAnimationOptions) {
  if (!motionSafe()) {
    // 直接设到终态
    Object.assign((el as HTMLElement).style, keyframes.at(-1));
    return { finished: Promise.resolve() };
  }
  return el.animate(keyframes, opts);
}

rAF Promise 化 loop

const nextFrame = (): Promise<number> => new Promise(requestAnimationFrame);

async function animation(from: number, to: number, duration: number) {
  const start = await nextFrame();
  while (true) {
    const now = await nextFrame();
    const t = Math.min(1, (now - start) / duration);
    element.style.transform = `translateX(${from + (to - from) * t}px)`;
    if (t === 1) return;
  }
}

05 常见陷阱

陷阱 1：用 left/top 做动画

每一帧触发 layout + paint + composite，慢设备上卡。统一用 transform: translate() 只触发 composite。

陷阱 2：读写混用导致 forced reflow

for (el of list) { el.style.transform = ...; const h = el.offsetHeight; } —— 每次读 offsetHeight 会强制同步布局。先全部读再全部写：const heights = list.map(el => el.offsetHeight); for (el of list) el.style.x = ...。

陷阱 3：后台标签 setInterval 跑飞

用 setInterval 做游戏循环，切后台后 CPU/电池被吃光。rAF 会自动暂停；setInterval 要手动 visibilitychange 监听后 pause。

陷阱 4：未用 will-change 但改 transform

单次动画没事；但频繁改 transform 没提前声明 will-change: transform，浏览器每次临时创建合成层，有短暂掉帧。动画开始前加，结束后移除（长期 will-change 占 GPU 内存）。

陷阱 5：WAAPI 的 fill 默认是 "none"

动画结束后元素回到初始态。想保留终态设 fill: "forwards"，但过多 forwards 动画会阻止后续样式覆盖。替代做法：动画 finish 后 JS 里设最终 style。

陷阱 6：Safari 对 rAF timestamp 的精度差异

Safari 的 ts 精度比 Chrome 低（1ms vs 5μs）。基于高精度差值计算速度时跨浏览器会有微小差异；游戏/物理模拟要用固定步长 + 累加器。

陷阱 7：动画合并 reflow

同帧内改 100 个元素的 style，浏览器只 reflow 一次（batch）。但中间读取 getBoundingClientRect 会打破 batch。用 requestAnimationFrame 集中写入。

陷阱 8：transition 立即触发不生效

同步设置 class 再改属性：el.classList.add("t"); el.style.opacity = 0 —— transition 可能已在初始值为 0 的 DOM 上立即生效。需 requestAnimationFrame 或强制 reflow（读 offsetHeight）在两步之间插入。

陷阱 9：height: auto 不能 transition

CSS transition 不支持 auto 值。高度展开动画用 max-height（估大值）或 JS 测量后显式设 px；2023 后 Chrome 支持 interpolate-size: allow-keywords 让 auto 可动画。

陷阱 10：长时间 JS 卡住 rAF

rAF 回调里跑 500ms 的循环，主线程一直卡；下一帧的 rAF 被顺延。重活切分成多帧（时间切片）或用 Worker。
