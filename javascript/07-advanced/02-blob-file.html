Blob / File / FormData

Blob 是浏览器里表示任意二进制数据（文件、图片、音视频、网络响应）的统一容器，File 是带文件名/修改时间的 Blob 子类，FormData 是 multipart 编码的辅助类。处理上传/下载/图片/媒体必然要用。本章讲清它们的语义、URL.createObjectURL 的生命周期、以及大文件分片上传的实战代码。

阅读时长 ~22 分钟 前置 01 Binary · P6 Fetch 规范 W3C File API · FormData

01 速览

  • Blob 不可变：创建后不能改内容，只能 slice 出新 Blob；Blob.prototype.size / type / arrayBuffer() / text() / stream() / slice()。
  • File extends Blob：额外有 name 和 lastModified；从 <input type="file">、DataTransfer、new File([...], "a.txt") 得到。
  • Blob 和 ArrayBuffer：Blob 是"懒"的（可能存磁盘），ArrayBuffer 一定在内存。大文件优先 Blob。
  • URL.createObjectURL(blob) 生成 blob:https://site/uuid，必须 revokeObjectURL 释放否则常驻内存。
  • FormData 自动设置 Content-Type: multipart/form-data; boundary=...；手动 set Content-Type 会覆盖 boundary 导致后端解析失败。
  • FileReader 是 2010 年老 API（回调/事件），2026 年 Blob.text() / arrayBuffer()（Promise）已全面可用，新代码不再用 FileReader。

02 底层逻辑

Blob 的构造与读取

// 从各种源构造
const blob1 = new Blob(["hello"], { type: "text/plain" });
const blob2 = new Blob([new Uint8Array([1, 2, 3])], { type: "application/octet-stream" });
const blob3 = new Blob([blob1, blob2]);       // 串联多个 Blob
const blob4 = blob1.slice(1, 4, "text/plain"); // "ell"，不复制数据（懒视图）

blob1.size;    // 5
blob1.type;    // "text/plain"

// 读取为各种格式（全部 Promise）
await blob1.text();          // "hello"
await blob1.arrayBuffer();    // ArrayBuffer(5)
await blob1.bytes();          // Uint8Array(5)（ES2024，新）
const stream = blob1.stream(); // ReadableStream<Uint8Array>（流式读大文件）

// 老 API（不要在新代码用）
const fr = new FileReader();
fr.onload = () => console.log(fr.result);
fr.readAsText(blob1);

URL.createObjectURL 生命周期

const url = URL.createObjectURL(blob); // "blob:https://site/abc-def"
// 把 Blob 引用在当前 document 的内部表中
img.src = url;

// ⚠️ Blob 会在 document 销毁前一直驻留内存
// 必须手动释放
URL.revokeObjectURL(url);

// 但别过早释放 —— img.src 还在用就不能 revoke
img.onload = () => URL.revokeObjectURL(url); // 加载完再释放

为什么默认不释放：浏览器无法知道 url 字符串被拷贝到哪些地方（DOM 属性、CSS、发给服务端）。Blob URL 的生命周期必须手动管理，就像 C 的 malloc/free。

data: URL vs blob: URL

data:URL blob:URL
编码 base64（+33% 体积） 指向 Blob 对象（零开销）
长度 可达几 MB（URL 越长越卡） 恒定短 URL
生命周期 字符串存在即有效 需 revoke
适合 小图标、CSS 内嵌 大图/视频预览、下载
CSP 需 data: source 需 blob: source

FormData 的三种构造

// 1. 空 FormData + 手动 append
const fd = new FormData();
fd.append("name", "Alice");
fd.append("avatar", fileInput.files[0]);
fd.append("avatar", anotherFile, "alt.png"); // 同一 key 多值（数组）
fd.set("name", "Bob"); // 替换现有 key

// 2. 从 <form> 元素构造（自动读取所有字段）
const form = document.querySelector("form");
const fd2 = new FormData(form);

// 3. 只包含提交按钮的字段（ES2022）
const fd3 = new FormData(form, submitButton);

// 遍历
for (const [key, value] of fd.entries()) { /* ... */ }

// 发送
await fetch("/upload", { method: "POST", body: fd });
// → Content-Type 自动变 multipart/form-data; boundary=----WebKit...
// ⚠️ 不要手动 set Content-Type header，浏览器需要生成 boundary

Blob 的底层存储

浏览器为 Blob 维护内部表（Blob registry），每个 Blob 有 UUID 和可能的磁盘文件。小 Blob（< ~40KB）常驻内存，大 Blob 转为磁盘文件。blob.slice() 只创建视图不复制底层数据。

Chrome 对单个 origin 的 Blob 总大小也有限额（与 IndexedDB 共享配额）；生产环境读大文件用 blob.stream() 流式处理，避免 arrayBuffer() 一次性加载到内存。

03 应用场景

图片预览与压缩

async function previewAndCompress(file: File, maxDim = 1600, quality = 0.85) {
  // 1. 生成 blob URL 给 <img>
  const previewUrl = URL.createObjectURL(file);
  imgEl.src = previewUrl;
  imgEl.onload = () => URL.revokeObjectURL(previewUrl);

  // 2. 用 createImageBitmap 解码 + 缩放
  const bitmap = await createImageBitmap(file, {
    resizeWidth: maxDim,
    resizeQuality: "high",
  });

  // 3. 画到 canvas 并导出
  const canvas = new OffscreenCanvas(bitmap.width, bitmap.height);
  const ctx = canvas.getContext("2d")!;
  ctx.drawImage(bitmap, 0, 0);
  bitmap.close(); // 释放 GPU 纹理

  const compressedBlob = await canvas.convertToBlob({ type: "image/webp", quality });
  return compressedBlob;
}

客户端下载

function downloadBlob(blob: Blob, filename: string) {
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  document.body.appendChild(a);
  a.click();
  a.remove();
  // 延迟 revoke，防止某些浏览器下载队列还没读
  setTimeout(() => URL.revokeObjectURL(url), 1000);
}

// CSV 导出
function exportCsv(rows: string[][], filename = "data.csv") {
  const csv = rows.map(r => r.map(c => `"${c.replace(/"/g, '""')}"`).join(",")).join("\n");
  const blob = new Blob(["﻿" + csv], { type: "text/csv;charset=utf-8;" });
  //                    ↑ UTF-8 BOM，让 Excel 识别中文
  downloadBlob(blob, filename);
}

大文件分片上传（带断点续传）

const CHUNK_SIZE = 5 * 1024 * 1024; // 5MB

async function uploadInChunks(file: File, uploadUrl: string) {
  // 1. 计算文件指纹（用于秒传 / 断点）
  const hash = await sha256OfFile(file);

  // 2. 询问服务端已传到第几片
  const { uploaded } = await fetch(`${uploadUrl}/status?hash=${hash}`).then(r => r.json());

  const total = Math.ceil(file.size / CHUNK_SIZE);
  for (let i = uploaded; i < total; i++) {
    const start = i * CHUNK_SIZE;
    const end = Math.min(start + CHUNK_SIZE, file.size);
    const chunk = file.slice(start, end);

    const fd = new FormData();
    fd.append("hash", hash);
    fd.append("index", String(i));
    fd.append("chunk", chunk, `${hash}-${i}`);
    await fetch(`${uploadUrl}/chunk`, { method: "POST", body: fd });
  }

  // 3. 通知合并
  await fetch(`${uploadUrl}/merge`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ hash, total, filename: file.name }),
  });
}

async function sha256OfFile(file: File): Promise<string> {
  // 大文件用流式哈希（避免 arrayBuffer 爆内存）
  const reader = file.stream().getReader();
  // 浏览器 crypto.subtle 不支持增量 hash；大文件用 SubtleCrypto 需整块传
  // 实战用 wasm 版 blake3 / spark-md5 做流式
  const buf = await file.arrayBuffer(); // ≤ 100MB 可接受
  const h = await crypto.subtle.digest("SHA-256", buf);
  return Array.from(new Uint8Array(h)).map(b => b.toString(16).padStart(2, "0")).join("");
}

04 典型代码

安全的 Blob URL 管理 Hook

import { useEffect, useState } from "react";

export function useObjectUrl(blob: Blob | File | null) {
  const [url, setUrl] = useState<string | null>(null);

  useEffect(() => {
    if (!blob) { setUrl(null); return; }
    const u = URL.createObjectURL(blob);
    setUrl(u);
    return () => URL.revokeObjectURL(u); // 组件卸载或 blob 变化时释放
  }, [blob]);

  return url;
}

// 使用
function Preview({ file }: { file: File | null }) {
  const url = useObjectUrl(file);
  return url ? <img src={url} alt="" /> : null;
}

拖放上传区

function DropZone({ onFiles }: { onFiles: (files: File[]) => void }) {
  const [dragging, setDragging] = useState(false);

  return (
    <div
      onDragEnter={() => setDragging(true)}
      onDragLeave={() => setDragging(false)}
      onDragOver={(e) => e.preventDefault()} // 必须阻止默认，否则 drop 不触发
      onDrop={async (e) => {
        e.preventDefault();
        setDragging(false);
        const files: File[] = [];
        for (const item of e.dataTransfer.items) {
          if (item.kind === "file") {
            const f = item.getAsFile();
            if (f) files.push(f);
          }
        }
        onFiles(files);
      }}
      className={dragging ? "ring" : ""}
    />
  );
}

流式读取大文本文件

async function* iterateLines(file: File): AsyncGenerator<string> {
  const reader = file.stream().pipeThrough(new TextDecoderStream()).getReader();
  let buffer = "";
  while (true) {
    const { value, done } = await reader.read();
    if (done) { if (buffer) yield buffer; return; }
    buffer += value;
    const lines = buffer.split("\n");
    buffer = lines.pop() ?? "";
    for (const line of lines) yield line;
  }
}

// 处理几 GB 的日志文件不爆内存
for await (const line of iterateLines(logFile)) {
  if (line.includes("ERROR")) console.log(line);
}

05 常见陷阱

陷阱 1：FormData 时手动设 Content-Type

fetch(url, { headers: { "Content-Type": "multipart/form-data" }, body: fd }) → 没 boundary，后端解析失败。让浏览器自动设置，你只 append FormData。

陷阱 2：忘了 revokeObjectURL 内存泄漏

单页应用每次显示 100 张图生成 100 个 blob URL，切页不 revoke，原 blob 常驻内存。DevTools Memory → 看 "Blob" count 确认。

陷阱 3：download 属性的安全限制

跨源 URL 的 <a download> 属性被浏览器忽略（防止下载钓鱼）。跨源下载必须先 fetch → Blob → 本站 blob URL。

陷阱 4：大文件 arrayBuffer() 爆内存

await file.arrayBuffer() 对 2GB 文件一次性载入 2GB 内存 → 移动端直接 OOM。大文件用 file.stream() 或 file.slice(start, end).arrayBuffer() 分片。

陷阱 5：Safari 对 File 的 lastModified 精度

Safari 只保留到秒，Chrome 保留毫秒。用 lastModified 做去重 key 时两浏览器行为不一致，建议用 name + size + hash(first 64KB)。

陷阱 6：input.files 没有 Array 方法

input.files 是 FileList（类数组），不能 .map。用 Array.from(input.files) 或 [...input.files]。

陷阱 7：Blob 的 type 不做验证

new Blob([...], { type: "image/png" }) 里 type 完全是字符串，浏览器不校验内容是否真是 PNG。上传后端必须用 magic number（文件头 4 字节）校验，防止 .jpg 改后缀传 .php。

陷阱 8：同 key 多次 append 是追加，不是覆盖

fd.append("a", 1); fd.append("a", 2) → 两个值都在（后端通常解析为数组）。想替换用 fd.set("a", 2)。

陷阱 9：FileReader 的事件顺序

老代码常用 FileReader：事件是 loadstart → progress* → load / error / abort → loadend。loadend 无论成功失败都触发，常被用错。新代码直接用 await blob.text()。

陷阱 10：blob URL 在 iframe / Worker 中的作用域

blob URL 绑定在创建它的 document 上。主线程创建的 blob URL 在 Worker 中 fetch 会失败（不同 scope）。Worker 内需要用就在 Worker 内 URL.createObjectURL。
