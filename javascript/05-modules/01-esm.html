ESM 规范与模块图

ES Modules 是 JS 的原生模块系统。和 CJS 的"每次 require 就当场执行"不同，ESM 用三阶段加载模型：解析 → 链接 → 执行。理解这三个阶段能解释所有 ESM"看起来反常"的行为 —— 循环依赖为什么能工作、export 为什么是 live binding、为什么 import 语句必须在顶层。

阅读时长 ~22 分钟 前置 P1 对象 · P2 作用域 对应 javascript.info Modules, introduction

01 速览

javascript 
// ===== 导出 =====
export const PI = 3.14;               // 命名导出
export function add(a, b) { return a + b; }
export default class App { }           // 默认导出（每模块至多一个）

// 合并导出
const x = 1, y = 2;
export { x, y };
export { x as X };                     // 重命名

// 从其他模块再导出
export { foo } from "./other.mjs";
export * from "./other.mjs";           // 批量（不含 default）
export { default as Foo } from "./other.mjs";

// ===== 导入 =====
import App, { PI, add as sum } from "./app.mjs";
import * as ns from "./app.mjs";        // namespace import
import "./side-effect.mjs";             // 只执行不导入
const mod = await import("./lazy.mjs"); // 动态 import

// ===== 元信息 =====
import.meta.url;                        // 当前模块 URL
import.meta.resolve("./x.mjs");         // 解析路径（新版支持）

如何启用 ESM

环境 启用方式
浏览器 <script type="module" src="...">
Node.js 文件后缀 .mjs，或 package.json 设 "type": "module"
TS/构建器 默认 ESM（Vite、Next.js、Bun、Deno 全部如此）

02 底层逻辑

三阶段加载模型

规范把 ESM 加载严格拆成三步，不能跨模块交织：

  1. Parse（解析构造） —— 递归抓所有 import 依赖，建立 module graph。静态分析 import/export 语句，但不运行任何代码。
  2. Link（链接） —— 为每个模块创建 module environment record；把 import 的"名字"和 export 的"槽"绑定到一起（live binding）。
  3. Evaluate（执行） —— 按依赖拓扑顺序执行模块顶层代码。每个模块只执行一次（有 module 缓存）。
javascript 
// main.mjs
import { counter, inc } from "./counter.mjs";
console.log(counter);   // 0
inc();
console.log(counter);   // 1  ← 值跟着原始模块变

// counter.mjs
export let counter = 0;
export function inc() { counter++; }

"counter 导出后还能变"这件事 CJS 做不到 —— CJS 的 exports.x 是值拷贝。ESM 是live binding：import 的是一个"引用槽"，不是"当前值"。

import / export 被 hoist 到模块顶部

规范规定：import 和 export 声明只能在模块顶层（不能放 if/function 里）。因为 parse 阶段就要全部收集完毕。

javascript 
// ❌ 语法错误：import 不能在函数里
function load() {
  import { x } from "./mod.mjs";   // SyntaxError
}

// ✓ 如果要条件/延迟加载，用动态 import（下一章详讲）
async function load() {
  const { x } = await import("./mod.mjs");
}

Live binding 的实现

import { x } from "./m" 里的 x 不是变量，是对模块 m 的 x binding 的引用。规范层面它是 ResolveBinding 到远端 Environment Record 的一条 link：

  • 读：始终拿到 m 当前的 x 值。
  • 写：不允许。x = 1 在 strict（ESM 默认严格）下抛 TypeError。import 变量是 const-like。
  • 结构变化：无。名字集在 link 阶段就冻结了，运行时不能新增 export。

Module namespace object

javascript 
import * as m from "./mod.mjs";

m.x;                    // 访问 export
m.default;              // 访问默认导出
Object.keys(m);         // 全部 export 名
typeof m;               // "object"
m[Symbol.toStringTag];  // "Module"

// ⚠ 限制
m.foo = 1;              // TypeError（frozen）
delete m.x;             // TypeError
Object.isFrozen(m);     // true
Object.getPrototypeOf(m);  // null  ← 特殊对象

// 动态 import 返回的也是这种对象（见 P4-03）
const m2 = await import("./mod.mjs");
m2.x;                   // 同 m.x

模块图与循环依赖

同一 URL 只解析/执行一次。如果 A import B，B import A —— Link 阶段就能绑定成功（虽然值 still undefined）：

javascript 
// a.mjs
import { b } from "./b.mjs";
export const a = "A";
console.log("a.mjs seeing b =", b);   // undefined（B 还没跑到 export b）

// b.mjs
import { a } from "./a.mjs";
export const b = "B";
console.log("b.mjs seeing a =", a);   // undefined（但 binding 已建立）

// index.mjs
import "./a.mjs";

// 输出：
// b.mjs seeing a = undefined         ← b 先跑（依赖深度优先）
// a.mjs seeing b = "B"               ← 回到 a 时 b 已执行完

和 CJS 循环 require 的区别：ESM total live binding —— 先放 undefined 槽，后续访问时总能拿到最新值。CJS 是"你拿到的是当下的 exports 对象副本"，循环 require 常拿到半成品。

import.meta

javascript 
// 每个 ESM 里可用的元信息对象
import.meta.url;            // "file:///abs/path/to/module.mjs"
import.meta.resolve("./x.mjs");  // 解析得到的绝对 URL（现代浏览器 / Node 20+）

// Node 特有
import.meta.dirname;        // 所在目录绝对路径（Node 20.11+）
import.meta.filename;       // 模块文件绝对路径

// 读取同目录资源
const configUrl = new URL("./config.json", import.meta.url);
const config = await fetch(configUrl).then(r => r.json());

// 检测是否主入口（替代 require.main === module）
if (import.meta.main) { /* 作为 entry 运行 */ }  // Deno / 部分 runtime

03 应用场景

模块天然 singleton

ESM 的 "同 URL 只执行一次 + 顶层 const" 让模块天然是 singleton，不用写 Singleton pattern：

javascript 
// db.mjs
import { Pool } from "pg";
export const pool = new Pool({ connectionString: process.env.DB_URL });

// 其他任何模块 import 得到的都是同一个 pool
import { pool } from "./db.mjs";

Live binding 实现"模块级状态"

javascript 
// config.mjs
export let locale = "zh-CN";
export function setLocale(l) { locale = l; }

// 任何 consumer
import { locale, setLocale } from "./config.mjs";
console.log(locale);     // "zh-CN"
setLocale("en-US");
console.log(locale);     // "en-US"  ← 自动更新
// 注意：import 的 locale 是 const-like，但 setLocale 修改的是原模块的 let

顶层 await 做异步初始化

javascript 
// config.mjs
const response = await fetch("/api/config");
export const config = await response.json();

// 使用方 —— import 会等 config.mjs 完成
import { config } from "./config.mjs";
// config 已就绪

规范术语 Top-Level Await（ES2022）。模块图会等 async 模块 fulfill 才继续。见 P3-05。

Browser 原生模块加载

javascript 
// 现代浏览器直接支持，开发环境可零构建
// <script type="module" src="./main.mjs"></script>

// main.mjs
import { render } from "./ui.mjs";
import { api } from "./api.mjs";
render(await api.load());

// 浏览器会：
// 1. 下载 main.mjs → 解析 → 发现 import ./ui.mjs 和 ./api.mjs
// 2. 并发下载这两个
// 3. 递归解析它们的依赖
// 4. 全部 link 后按拓扑执行
// 5. ⚠ 每个 import 都是一个独立 HTTP 请求（无 bundler），依赖多会很慢

import.meta.url 读取同目录资源

javascript 
// shader.mjs 需要读取同目录的 shader.glsl
const shaderUrl = new URL("./shader.glsl", import.meta.url);
const response = await fetch(shaderUrl);
export const shaderSource = await response.text();

// 这比 CJS 时代的 __dirname + fs.readFileSync 更通用
// 浏览器和 Node 都能跑（前者 fetch URL, 后者 fs.readFile(fileURL)）

04 典型代码

默认 + 命名导出混合

javascript 
// lib.mjs
export default function main() { /* ... */ }    // 默认导出
export const version = "1.0";                    // 命名导出
export const utils = { /* ... */ };

// 使用方
import main, { version, utils } from "./lib.mjs";
//       ↑ 默认 import       ↑ 命名 import（大括号内）

// namespace import
import * as lib from "./lib.mjs";
lib.default;    // 默认导出
lib.version;
lib.utils;

Re-export（桶文件）

javascript 
// index.mjs（把多模块聚合成一个入口）
export { Button } from "./Button.mjs";
export { Input } from "./Input.mjs";
export * from "./hooks.mjs";              // 批量（不含默认）
export { default as Modal } from "./Modal.mjs";   // 别名重出

// 使用
import { Button, Input, useStorage, Modal } from "./ui";

// ⚠ 注意桶文件对 tree-shaking 的影响（见 P4-04）

条件加载（静态 + 动态混合）

javascript 
// ❌ 不能条件 static import
// if (process.env.NODE_ENV === "dev") import { devTools } from "./dev.mjs";

// ✓ 动态 import
let devTools = null;
if (process.env.NODE_ENV === "development") {
  ({ devTools } = await import("./dev.mjs"));
}

// 或 lazy 一次：
let loadDev;
async function getDev() {
  loadDev ??= import("./dev.mjs");
  return await loadDev;
}

安全处理循环依赖

javascript 
// a.mjs
import { helper } from "./b.mjs";
export function doA() {
  return helper();            // ⚠ 顶层调用会 undefined，但函数内调用没事
}

// b.mjs
import { doA } from "./a.mjs";
export function helper() { return "h"; }
// export function dangerous() { return doA(); }  // 也 OK（运行时）
// doA();    // ❌ 顶层立即调用：a 还没执行完，doA 是 TDZ

05 常见陷阱

import 不能在顶层之外

静态 import 只在模块顶层。函数里、if 里、try 里都不行。要动态就用 await import()。

import 绑定不可赋值

import { x } from "./m"; x = 1; 在 strict（ESM 默认）抛 TypeError。想"修改 m 的 x"必须导出 setter 或改 x 为普通 let + 提供 setter。

ESM 在浏览器的 MIME 类型

浏览器要求 type="module" 脚本的 Content-Type 是 text/javascript / application/javascript。服务器返回 text/plain 的 .js 文件会被拒绝加载。Vite/Next dev 服务器都处理好了；自建静态服务要检查。

ESM 的 script 默认 defer

<script type="module"> 行为等同于 defer：HTML 解析完才执行。不会阻塞 parsing。不用手写 defer。

Node 里 __dirname / __filename 在 ESM 不可用

ESM 下用 import.meta.dirname / import.meta.filename（Node 20.11+）或 new URL(".", import.meta.url).pathname。Node 22+ 也允许 require(esm)（有限制）但最佳实践仍是 import.meta。

循环依赖顶层调用是 TDZ

上面 a.mjs 例子：函数内调用循环导入的函数没事（运行时已链接好）；顶层立刻调用会触发 TDZ（依赖方还未执行完）。把初始化搬进 function 或用懒求值。

Re-export 破坏 tree-shaking

桶文件 export * from "./big.mjs" 让打包器认为"整个 big 被用" —— 即使你只用其中一个。加 sideEffects: false 让打包器大胆裁剪。详细在 P4-04。

import * as m 的 m 是 frozen 的

试图 m.foo = 1 在严格模式抛 TypeError。这是规范对命名空间对象的保护 —— 模块导出集不能运行时扩展。

相对路径必须带 ./

浏览器 ESM 里 import x from "utils.mjs"（无 ./）是 bare specifier，需要 import map 解析。写 ./utils.mjs 才是相对路径。Node 同理（ESM 里）。
