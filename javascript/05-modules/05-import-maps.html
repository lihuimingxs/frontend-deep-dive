import maps 与模块解析

import "lodash" 浏览器不知道去哪找 —— 必须告诉它。工具链里是打包器的 resolver；原生浏览器里是 import maps；Node 里是 exports/imports 字段。本章把三种环境下的"specifier → URL"规则讲清楚。

阅读时长 ~20 分钟 前置 01 ESM · 02 CJS 互操作 规范 WICG Import Maps

01 速览

Specifier 的三种类型

类型 例子 解析方式
Relative ./utils.mjs / ../shared.mjs 相对当前模块 URL
Absolute URL https://cdn.example.com/lib.mjs 直接用
Bare specifier lodash / @org/pkg / #internal 需要 resolver（import map / node modules / exports）

三种环境的 resolver

  • 浏览器：<script type="importmap"> JSON 配置
  • Node.js：node_modules 目录查找 + package.json exports/imports
  • 打包器：resolve.alias/paths/tsconfig + 上述规则

02 底层逻辑

浏览器 Import Maps

html 
<script type="importmap">
{
  "imports": {
    "lodash": "https://cdn.skypack.dev/lodash-es",
    "lodash/": "https://cdn.skypack.dev/lodash-es/",   // 斜杠结尾：子路径映射
    "react":  "https://esm.sh/react@18.3.1",
    "react-dom/client": "https://esm.sh/react-dom@18.3.1/client",
    "@/": "/src/"
  },
  "scopes": {
    "/admin/": {
      "react": "https://esm.sh/react@19.0.0"   // admin 目录用 React 19
    }
  }
}
</script>

<script type="module">
  import _ from "lodash";               // 走 map → skypack
  import { debounce } from "lodash/debounce";  // 子路径
  import Foo from "@/components/Foo.mjs";  // /src/components/Foo.mjs
</script>

Chrome 89+ / Firefox 108+ / Safari 16.4+ 支持。一个页面只能有一个 importmap，必须在任何 type="module" script 之前。

Scopes：作用域级别的 override

scopes 允许在特定路径下覆盖全局映射。用于"页面不同子路由用不同版本库"或"某个第三方包自己要求用特定依赖版本"：

json 
{
  "imports": {
    "react": "https://esm.sh/react@18"
  },
  "scopes": {
    "/legacy/": {
      "react": "https://esm.sh/react@17"   // legacy 代码用 React 17
    }
  }
}

Node.js exports 字段

包作者用 exports 字段声明"哪些子路径可以被外部 import"+ "针对不同环境给不同文件"：

json 
{
  "name": "mylib",
  "type": "module",
  "exports": {
    ".": {                                // 主入口
      "types": "./dist/index.d.ts",
      "browser": "./dist/browser.mjs",    // bundler 看到 target=web
      "import": "./dist/index.mjs",       // ESM 消费者
      "require": "./dist/index.cjs",      // CJS 消费者
      "default": "./dist/index.mjs"       // fallback
    },
    "./utils": "./dist/utils.mjs",        // 子路径导出
    "./locale/*.json": "./dist/locales/*.json",  // 模式匹配
    "./package.json": "./package.json"    // 允许读 package.json
  }
}

// 消费者
import mylib from "mylib";                 // → dist/index.mjs
import { helper } from "mylib/utils";      // → dist/utils.mjs
import zh from "mylib/locale/zh.json";     // → dist/locales/zh.json

exports 字段强封装：没在 exports 里列的子路径，消费者无法 import（Node 报 ERR_PACKAGE_PATH_NOT_EXPORTED）。这防止了库内部文件被外部误用。

条件导出的优先级

条件键 何时匹配
types TypeScript 解析类型时（必须放最前！）
browser 构建器 target 是浏览器
deno / bun / workerd 对应 runtime
node Node.js
import ESM 消费者
require CJS 消费者
default 兜底（必须放最后）
development / production NODE_ENV 值（Node 22+）

顺序敏感！从上到下匹配第一个 true 的键。错顺序会让 TypeScript 拿到 .js 而不是 .d.ts。

Subpath imports（# 前缀）

imports 字段（和 exports 平级）用于包内部的路径别名：

json 
// package.json
{
  "imports": {
    "#utils": "./src/utils/index.mjs",
    "#config": {
      "development": "./config/dev.mjs",
      "production": "./config/prod.mjs",
      "default": "./config/default.mjs"
    }
  }
}

// 包内部代码
import { helper } from "#utils";        // ← 不用 ../../utils
import config from "#config";           // 按环境选文件

替代相对路径（../../../utils）和 tsconfig paths。规范标准，Node 原生支持，不依赖构建工具。

打包器的路径别名

javascript 
// Webpack
{
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "src"),
      "@shared": path.resolve(__dirname, "../shared"),
    }
  }
}

// Vite / Rollup
{
  resolve: {
    alias: [{ find: "@", replacement: "/src" }]
  }
}

// tsconfig.json（TypeScript 识别）
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"],
      "@shared/*": ["../shared/*"]
    }
  }
}

// ⚠ tsconfig paths 只是编译时别名
// 运行时需要打包器也识别（Webpack/Vite 自动读 tsconfig.paths，Bun 也是）

03 应用场景

无构建开发（本地 ESM + import map）

html 
<!-- index.html -->
<script type="importmap">
{
  "imports": {
    "lit": "https://esm.sh/lit@3",
    "@/": "./src/"
  }
}
</script>
<script type="module" src="./main.mjs"></script>

<!-- main.mjs -->
import { html, render } from "lit";
import { App } from "@/app.mjs";

// 适合：demo、playground、小项目、教学
// 不适合：大项目（HTTP 请求数爆炸，不支持 TS 等）

monorepo 内部路径别名

javascript 
// package.json (内部私有包)
{
  "imports": {
    "#domain": "./src/domain/index.mjs",
    "#services": "./src/services/index.mjs",
    "#db/*": "./src/db/*"
  }
}

// 业务代码
import { User } from "#domain";
import { sendEmail } from "#services";
import { userRepo } from "#db/user";

// 好处：代码移动时不用改 import 路径
// Node 原生支持 imports 字段，无需打包器

dev / prod 条件导出

json 
// 库提供 dev 版（带警告 / 额外检查）和 prod 版（轻量）
{
  "exports": {
    ".": {
      "development": "./dist/index.dev.mjs",
      "production": "./dist/index.prod.mjs",
      "default": "./dist/index.prod.mjs"
    }
  }
}

// React、Redux 等库的标准做法
// Node 22+ 自动按 NODE_ENV 选择

浏览器直连 CDN 的 React 项目

html 
<script type="importmap">
{
  "imports": {
    "react": "https://esm.sh/react@18.3.1?dev",
    "react-dom": "https://esm.sh/react-dom@18.3.1?dev",
    "react-dom/client": "https://esm.sh/react-dom@18.3.1/client?dev"
  }
}
</script>
<script type="module" src="./app.mjs"></script>

<!-- app.mjs -->
import { createRoot } from "react-dom/client";
import { App } from "./App.mjs";
createRoot(document.getElementById("root")).render(<App />);

<!-- 无构建工具、无 node_modules、可直接访问
⚠ 生产仍建议打包（HTTP 请求数 / 代码分割 / 压缩） -->

TypeScript paths + 实际运行时配对

javascript 
// tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}

// 配套 vite.config.ts
import { fileURLToPath, URL } from "node:url";
export default {
  resolve: {
    alias: {
      "@": fileURLToPath(new URL("./src", import.meta.url))
    }
  }
}

// 或用 vite-tsconfig-paths 插件自动同步

04 典型代码

完整 exports 示例（类型 + 双模）

json 
{
  "name": "@myorg/ui",
  "type": "module",
  "version": "2.0.0",
  "main": "./dist/index.cjs",            // 老式 main（fallback）
  "module": "./dist/index.mjs",          // module 字段（bundler 识别）
  "types": "./dist/index.d.ts",
  "sideEffects": false,
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.mjs",
      "require": "./dist/index.cjs"
    },
    "./Button": {
      "types": "./dist/Button.d.ts",
      "import": "./dist/Button.mjs",
      "require": "./dist/Button.cjs"
    },
    "./styles.css": "./dist/styles.css",
    "./package.json": "./package.json"
  },
  "typesVersions": {
    "*": {
      "*": ["dist/*"]
    }
  }
}

跨 runtime 包（Node + Browser + Deno）

json 
{
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "deno": "./dist/deno.mjs",
      "bun": "./dist/bun.mjs",
      "browser": "./dist/browser.mjs",
      "node": {
        "import": "./dist/node.mjs",
        "require": "./dist/node.cjs"
      },
      "default": "./dist/index.mjs"
    }
  }
}

// 每个 runtime 可以用对应的 API（如 Node fs、Deno 权限 API）

动态生成 import map

javascript 
// 从 lockfile 生成 import map（常见 tool：@jspm/generator）
import { Generator } from "@jspm/generator";

const generator = new Generator({
  mapUrl: import.meta.url,
  defaultProvider: "esm.sh",
  env: ["production", "browser"],
});

await generator.install(["react@18", "react-dom@18"]);
console.log(JSON.stringify(generator.getMap(), null, 2));

// 输出可以直接写进 <script type="importmap">

Node 的 resolution 调试

bash 
// 看 Node 怎么解析一个包
node --experimental-specifier-resolution=explicit --trace-module-resolution my-script.mjs

// 或在代码里
import { resolve } from "node:path";
console.log(import.meta.resolve("lodash"));    // 返回解析后的 URL

// node_modules 查找顺序：
// 1. 当前目录的 node_modules
// 2. 上级目录的 node_modules
// 3. ... 直到文件系统根

05 常见陷阱

exports 字段存在就是"只能 import 列出的路径"

一旦 package.json 有 exports，没列的路径消费者无法 import（即使文件存在）。升级库时加 exports 可能破坏消费者的深层 import。要保留老路径就在 exports 里显式列出。

条件导出顺序错，TypeScript 拿不到类型

types 必须排在 import/require之前。否则 TS 先匹配到 import 拿到 .mjs 而不是 .d.ts → 类型失效。

import map 只能有一个

每个页面最多一个 <script type="importmap">，且必须在所有 type="module" 之前。多个会被忽略。

import map 不支持继承

子 iframe / Worker 不能继承父文档的 import map，需要自己声明。importmap-external 提案还没普及。

tsconfig paths 运行时不生效

paths 只是 TS 编译器的解析别名。运行时必须有对应的 bundler 配置（Webpack alias、Vite resolve.alias）或 Node 的 imports 字段。Bun / Deno 也支持 tsconfig paths。

Subpath pattern 必须包含 *

"./locale/*": "./locale/*.mjs" 中 * 在 key 和 value 必须配对。不能 "./locale/*": "./locale/all.mjs"（固定 value，不支持）。

browser 字段 vs exports.browser

老式 browser 字段：根级别，仅 bundler 识别；新式用 exports.browser 条件更精细。混用可能冲突。建议只用 exports（支持全部场景）。

imports 字段的 # 不能用于 exports

#xxx 是 subpath imports 专用，只在包内部可用。不能 import "pkg/#xxx" 从外部 import。外部用 exports。
