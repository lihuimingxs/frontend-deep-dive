class 与继承

class 是原型链的语法糖，但加了几条行为约束："body 默认严格"、"必须 new 调用"、"方法默认 non-enumerable"、"子类必须 super()"。本章把语义约束、继承机制、static 成员的查找、new.target、mixin 模式讲完整。

阅读时长 ~22 分钟 前置 05 原型链 对应 javascript.info Classes

01 速览

javascript 
class User {
  // 1. 实例字段（ES2022）
  name;
  #secret;                      // 私有字段
  static defaultAge = 18;        // 静态字段

  // 2. 构造器
  constructor(name, secret) {
    this.name = name;
    this.#secret = secret;
  }

  // 3. 原型方法（实例间共享）
  greet() { return `Hi, ${this.name}`; }

  // 4. 静态方法
  static create(name) { return new User(name, ""); }

  // 5. getter / setter
  get upper() { return this.name.toUpperCase(); }
  set upper(v) { this.name = v.toLowerCase(); }

  // 6. 静态块（ES2022）
  static {
    User.totalRegistered = 0;
  }
}

class Admin extends User {
  constructor(name, secret, perms) {
    super(name, secret);       // 必须先 super() 才能访问 this
    this.perms = perms;
  }
  canDo(action) {
    return this.perms.includes(action) || super.canDoBase?.(action);
  }
}

6 种 class 成员 + 它们的存储位置：

形式 存在哪 通过什么访问
实例字段 x; 实例自身 instance.x
私有字段 #x; 实例（私有 slot） this.#x（只在 class 内）
方法简写 m() {} Class.prototype instance.m()
getter/setter Class.prototype 像属性
静态方法 static m() {} Class 自身 Class.m()
静态字段 static x; Class 自身 Class.x

02 底层逻辑

class 是特殊的 function

javascript 
class X {}
typeof X;                     // "function"
X.prototype.constructor === X;  // true

// 但和普通函数的差异：
X();                          // TypeError: Class constructor X cannot be invoked without 'new'
// class body 默认严格模式 —— 即使外层非严格
// 方法默认 enumerable: false —— 不出现在 for-in / Object.keys
Object.keys(X.prototype);      // []（class）
Object.getOwnPropertyNames(X.prototype); // ["constructor", ...methods]

class 不会提升

new Foo(); class Foo {} 抛 ReferenceError（TDZ）。与 let/const 一致。

字段与构造器的执行顺序

对于 class A 的实例化 new A()：

  1. 分配新对象，[[Prototype]] 设为 A.prototype。
  2. 按源码顺序依次初始化实例字段（x = expr;）—— 在这个阶段 this 就是新对象。
  3. 执行构造器体。

extends A 的子类 new B()：

  1. 进入 B 的 constructor；this 此时未绑定（TDZ）—— 访问 this 抛 ReferenceError。
  2. 遇到 super(args) → 调用 A 的 constructor。A 的字段初始化 + A 构造器体全部跑完后，this 绑定为新对象。
  3. 回到 B：依次执行 B 的实例字段初始化、B 构造器体剩余部分。
javascript 
class A {
  a = console.log("A field");
  constructor() { console.log("A ctor"); }
}
class B extends A {
  b = console.log("B field");
  constructor() {
    // console.log(this);     // ReferenceError: Must call super before using 'this'
    super();
    // 到这里 A 的 field 和 ctor 跑完 → this 可用
    console.log("B ctor");
  }
}
new B();
// A field
// A ctor
// B field
// B ctor

super 的两种形式

  • super(args) —— 只能在构造器里；调用父类 constructor。
  • super.method(args) / super.prop —— 在任何方法里；访问父类 prototype 上的成员。

规范细节：super.method 的查找用的是"当前方法所在的 home object"的 [[Prototype]]，不是 this 的原型链。这避免了钻石继承时的方法无限递归。

static 成员也走原型链

class B extends A {} 时：

  • B.prototype.__proto__ === A.prototype（方法继承）
  • B.__proto__ === A（static 继承 —— class 本身作为对象，也有原型链）
javascript 
class A { static hi() { return "hi"; } }
class B extends A {}

B.hi();                        // "hi"   ← static 方法被继承！

Object.getPrototypeOf(B);      // A  （不是 Function.prototype）
B.__proto__ === A;             // true

new.target 元属性

在构造器里 new.target 是"实际被 new 的 class"。不通过 new 调用时是 undefined。

javascript 
class Shape {
  constructor() {
    if (new.target === Shape) throw new Error("Shape is abstract");
  }
}
class Circle extends Shape { }

new Circle();                  // ✓
new Shape();                   // Error: abstract

class 表达式

javascript 
// 可赋给变量
const MyClass = class { };

// 命名表达式（名字仅内部可见）
const C = class InnerName {
  static who() { return InnerName.name; }
};
C.who();                       // "InnerName"

// 立即 new
const instance = new class { greet() { return "hi"; } };
instance.greet();              // "hi"

// class factory（mixin 的基础）
function makeClass(base) {
  return class extends base { };
}

03 应用场景

领域模型

javascript 
class User {
  #password;
  constructor(name, password) {
    this.name = name;
    this.#password = password;
    this.createdAt = new Date();
  }

  check(pwd) { return this.#password === pwd; }
  changePassword(old, next) {
    if (!this.check(old)) throw new Error("wrong password");
    this.#password = next;
  }
  toJSON() {
    return { name: this.name, createdAt: this.createdAt };  // 不泄漏私有
  }
}

自定义 Error 类（P1-14 已讲过）

javascript 
class AppError extends Error {
  constructor(message, options) {
    super(message, options);
    this.name = this.constructor.name;
  }
}
class NotFoundError extends AppError { }
class PermissionError extends AppError { }

try { /* ... */ }
catch (err) {
  if (err instanceof NotFoundError) { /* ... */ }
}

Mixin pattern

JS 单继承。用 class factory 组合多个"能力"：

javascript 
const Timestamped = (Base) => class extends Base {
  constructor(...args) {
    super(...args);
    this.createdAt = new Date();
  }
};

const Serializable = (Base) => class extends Base {
  toJSON() { return { ...this }; }
};

class User {
  constructor(name) { this.name = name; }
}

class SpecialUser extends Timestamped(Serializable(User)) { }
const u = new SpecialUser("Alice");
u.name;           // "Alice"
u.createdAt;      // Date
u.toJSON();       // { name, createdAt }
u instanceof User;  // true

Singleton（单例）

javascript 
class DB {
  static #instance;
  constructor() {
    if (DB.#instance) return DB.#instance;   // return 对象覆盖 this
    DB.#instance = this;
    // ... 初始化
  }
}

new DB() === new DB();        // true

但 ESM 里通常不需要手写单例 —— 直接 export const db = new DB()，模块本身已经是 singleton。

抽象基类 + 模板方法

javascript 
class BaseQueue {
  constructor() {
    if (new.target === BaseQueue) throw new Error("abstract");
  }
  push(item) { throw new Error("not implemented"); }   // 抽象方法
  pop()      { throw new Error("not implemented"); }
  size()     { throw new Error("not implemented"); }

  // 模板方法：依赖抽象方法
  clear() { while (this.size() > 0) this.pop(); }
}

class ArrayQueue extends BaseQueue {
  #data = [];
  push(item) { this.#data.push(item); }
  pop()      { return this.#data.shift(); }
  size()     { return this.#data.length; }
}

04 典型代码

super 在钻石继承里的正确性

javascript 
class A { hi() { return "A"; } }
class B extends A { hi() { return super.hi() + "B"; } }
class C extends B { hi() { return super.hi() + "C"; } }

new C().hi();                  // "ABC"
// super.hi() 查找的是"定义方法的 class 的父类 prototype"
// 在 B 中查 A.prototype.hi，在 C 中查 B.prototype.hi
// ——不是 this 的原型链，避免无限递归

getter/setter 在继承中的行为

javascript 
class Box {
  #value = 0;
  get value() { return this.#value; }
  set value(v) { this.#value = v; }
}
class Logged extends Box {
  set value(v) {
    console.log("set to", v);
    super.value = v;          // 通过 super 调用父类 setter
  }
  get value() {
    return super.value;       // 通过 super 调用父类 getter
  }
}
const b = new Logged();
b.value = 42;                 // "set to 42"
b.value;                      // 42

静态工厂 + 子类多态

javascript 
class Shape {
  static fromPoints(points) {
    // this 在 static 方法里是调用该方法的 class
    return new this(points);
  }
}
class Circle extends Shape {
  constructor(points) { super(); this.center = points[0]; }
}

Shape.fromPoints([p1, p2]);    // Shape 实例
Circle.fromPoints([p1, p2]);   // Circle 实例  ← this 指向 Circle

Reflect.construct 动态 new

javascript 
// 等价 new Class(...args)
const instance = Reflect.construct(Class, args);

// 进阶：用另一个 class 的 prototype 初始化（polyfill Error 的 trick）
class MyError extends Error {
  constructor(msg) {
    super(msg);
    // V8 会让 this 的原型链是 MyError.prototype，正确
  }
}

// Reflect.construct 还能指定 new.target
const x = Reflect.construct(Base, args, Derived);
// 等价"以 Derived 为 new.target 调 Base"

05 常见陷阱

子类构造器里 super() 前用 this

ReferenceError: Must call super constructor... 。规则记死：extends 的子类 constructor 第一句应该是 super(...)（除非子类没有显式 constructor，引擎自动生成默认 constructor(...args) { super(...args); }）。

子类字段初始化覆盖父类字段

class A { x = 1 } class B extends A { x = 2 } —— B 实例的 x 是 2。顺序：父字段→父构造器→子字段→子构造器。子字段即使和父字段同名也会覆盖。

忘 new 调用 class

TypeError。ES6 class必须用 new。保护：在 ESLint 开 new-cap；或用构造函数（非 class）+ new.target 检测。

class 方法不可枚举，但 Object.assign 拷贝不过去

Object.assign(target, instance) 只拷自有可枚举 → class 方法（住在 prototype 上，不可枚举）全丢。要保留方法用 Object.create(Object.getPrototypeOf(instance)) 或 structuredClone（也不完美，class 实例克隆是 JS 老大难）。

extends null 的限制

class A extends null {} 合法但必须手写 constructor 且不能 super()（因为没父构造器），且必须自己用 Reflect.construct 初始化 this。极少用，了解即可。

class field 初始化值的 this

class A { x = this.computeInitial(); computeInitial() { ... } } —— 字段初始化时 this 是新实例，但原型方法已可调用（prototype 在 class 评估时就定好了）。顺序问题：如果 computeInitial 依赖后面字段，后面字段还未初始化。

static 字段初始化里的 this

static count = 0; static inc() { this.count++; } —— this 是 class 本身。但如果 inc 被提取 const f = Klass.inc; f()，this 丢失，this.count 抛错（严格模式）。static 方法同样受 this 规则约束。

class body 默认严格模式

即使整个文件非严格，class body 里的代码按严格模式执行。未声明变量赋值抛错、delete non-configurable 属性抛错等。这对"从老代码迁到 class"的项目偶尔造成意外。
