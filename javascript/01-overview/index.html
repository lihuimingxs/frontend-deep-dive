P1 · JavaScript 是谁

这一阶段不讲技术细节，只解决两个问题：JavaScript 怎么走到今天、它在前端 / 服务端 / 边缘 / 移动生态里站什么位置。给后面 P2-P8 所有"为什么 JS 长这样"提供历史与生态坐标。本阶段是参照 V8 / ECMA / Node / React 主题的"P1 是谁"模式补的——它们都用 P1 开篇讲项目史与同代人，JS 主题原本只直接进语言基础，少了这层。

章节数 2 / 2 ✅ 主要来源 Brendan Eich 演讲 · ECMA-262 历史段 · TC39 流程 状态 ✅ 全部完成

本阶段覆盖

  • 项目史 30 年 —— Brendan Eich 1995 年 10 天写出 Mocha → JavaScript 命名争议（Java 营销）→ 1996 浏览器战争 + JScript 分裂 → ES1-3 + ES4 失败 → 2008 V8 + Node 爆发 → 2015 ES2015 现代化重启 → 2020+ Edge Runtime / Deno / Bun 多元运行时
  • 关键设计决策 —— 为什么 JavaScript 名字蹭 Java、为什么 ES4 放弃、为什么 ES5 是救场之作、为什么 ES2015 是分水岭
  • JS 在生态中的位置 —— 设计哲学（动态弱类型 + 单线程 + 异步 + 原型链）/ 编译到 JS 的语言分支（CoffeeScript / Dart / TS / ReScript）/ 平台分支（浏览器 / Node / Deno / Bun / Workers / RN / Electron）/ 与其他动态语言对比（Python / Ruby / PHP）
  • "JS 三大设计错误" —— Eich 自己反思的设计遗憾（type coercion / global scope / null vs undefined 等）

💡 这一阶段与 ECMA 主题的关系

P1.1 项目史语言层视角（JavaScript 这门语言怎么演化）；ECMA P1.1 是规范层视角（ECMA-262 标准怎么编辑）。两者互为对照——同一段历史，从语言用户视角看是"我能用什么"，从标准编辑视角看是"哪些进了规范"。详见 ECMA P1.1 · 标准发展史。
