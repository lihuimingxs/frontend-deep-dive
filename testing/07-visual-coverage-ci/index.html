P7 · 视觉回归 + Coverage + CI 决策树

全主题的"实战收束"。把 P1-P6 的工具知识落地到 CI 流水线：视觉回归怎么集成、覆盖率到底测什么、flaky 怎么治理、最终给一份完整 GitHub Actions 配置 + 决策树。中立——不当任何家广告。

章节数 1 主要来源 Chromatic / Percy / Playwright snapshot 文档 + 真实开源项目 CI 配置 状态 ✅ 全部完成

本阶段覆盖

  • 视觉回归技术· 截图对比 + 像素差分阈值 + ignore region
  • 三家视觉工具· Chromatic（基于 Storybook）/ Percy（跨平台）/ Playwright snapshot（内置 + 免费）
  • Coverage 真相· Istanbul（成熟慢）vs V8（Vitest 默认快）；100% 覆盖 ≠ 0 bug
  • 覆盖率三种粒度· branch / function / statement
  • CI 配置· Vitest + GitHub Actions cache；Playwright + Docker；trace artifact 上传
  • 与 Turborepo cache 配合· 链回 pnpm P5.1
  • flaky 治理· retries 区分 flaky vs 真失败 / 仪表盘 / 每周 review / 根因排查
  • 决策树· 单元 = Vitest + Testing Library + happy-dom；集成 = Vitest + MSW；e2e = Playwright；视觉 = Chromatic
  • 5 年展望· Vitest 取代 Jest / Playwright 取代 Cypress / Component Testing 模糊边界 / AI 辅助测试生成
