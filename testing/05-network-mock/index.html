P5 · 网络 mock

集成测试做得对不对，看网络 mock 这层。MSW 把"在网络层拦截"做成事实标准——同一份 handlers 在 dev / test / Storybook 三处复用。本阶段讲透 MSW 与 nock / fetch-mock / 手写 vi.mock 的取舍。

章节数 1 主要来源 mswjs.io / Aleksandra Sikora 演讲 / undici 拦截器源码 状态 ✅ 全部完成

本阶段覆盖

  • MSW 设计原则· 网络层拦截 vs 应用层 mock；测试代码与生产代码完全一致
  • 三模实现· 浏览器 Service Worker / Node undici interceptor / WebSocket（MSW 2.0+）
  • handlers 写法· http.get / path params / query / dynamic response
  • server / worker 启动· server.listen + resetHandlers + close 生命周期
  • vs nock· nock 仅 Node + 老接口；MSW 接管市场份额
  • vs fetch-mock· 替换 global fetch 让应用代码感知；不推荐
  • vs vi.mock('axios')· 仅简单场景；多 endpoint 时累
  • 常见陷阱· Service Worker cache 干扰 / handlers 顺序 / Node 18+ 不需 node-fetch
  • 工程化· 一份 handlers 服务 dev / test / Storybook 三场景
