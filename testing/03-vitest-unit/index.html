P3 · Vitest + 单元测试

单元测试主战场。Vitest 在 2021-2026 年从"Vite 生态的 Jest 替代"变成新项目默认。本阶段讲它的内部架构（Vite 复用 + ESM + workspace + browser mode），以及与 Jest / Bun test / node:test 的横评。

章节数 2 主要来源 vitest.dev / Anthony Fu blog / Tinypool / Astro / Vue 迁移案例 状态 ⏳ 待写作

本阶段覆盖

  • Vitest 与 Vite 的关系· 复用 Vite dev server 编译能力（链回 Vite P2.1）
  • 三层架构· 主进程（Vite）+ Worker 进程（隔离测试）+ Tinypool 进程池
  • API 兼容 Jest· describe / test / expect / vi.mock / vi.fn 几乎可直接迁移
  • workspace 模式· vitest.workspace.ts 多 project 配置（链回 pnpm P4.2）
  • browser mode（Vitest 2.0+ GA）· 真浏览器 via Playwright；适合 layout / Canvas / 组件库
  • 类型测试· expectTypeOf + assertType 在测试中混用
  • 覆盖率 provider· v8（默认）vs istanbul（详见 P7）
  • 4 家 runner 横评· Vitest 3 / Jest 30 / Bun test / node:test 全维度对比
  • Jest 30 现状· OpenJS 治理 / ESM 仍痛苦 / Babel 依赖
  • Bun test 链回· 详见 Bun P4.4
  • node:test 链回· 详见 Node P6.1
  • 真实迁移案例· Astro / Vue / Vite 自家用 Vitest；Next.js 仍推荐 Jest
