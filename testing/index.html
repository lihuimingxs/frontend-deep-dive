编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 「短重述 + 链回」 —— Bun test 内部 / node:test / Vite dev server 复用 / workspace test 运行都在 Bun / Node / Vite / pnpm 主题；本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • API 定义优先官方文档 —— vitest.dev / playwright.dev / testing-library.com / mswjs.io 是表面规则的源头
  • 设计动机以作者文章为准 —— Kent C. Dodds blog（Testing Library + Testing Trophy 概念原作者）/ Anthony Fu blog（Vitest 主作者）/ Andrey Lushnikov 演讲（Playwright）；写"为什么这么设计"必须回到一手出处
  • 性能与 flaky 数字必标条件 —— "Vitest 比 Jest 快 N 倍"需要标项目规模 + ESM/CJS + workspace 维度，否则不写
