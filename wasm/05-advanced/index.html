P5 · 高级特性提案

Threads / SIMD / WasmGC / Tail call / Exception handling / JSPI 是 wasm 2.0+ 的关键演化。这一阶段两章覆盖。每个特性都标注 phase（4 浏览器默认开 / 5 进入正式 spec）+ 主流引擎实现状态。

章节数 2 主要来源 WebAssembly proposals · v8.dev/blog · spec issues 状态 ✅ 全部完成

本阶段覆盖

  • Threads 提案 (phase 4)—— Web Worker + SharedArrayBuffer；必须 cross-origin isolated（链回浏览器渲染主题）
  • Atomic ops—— atomic.wait / notify / rmw；与 ECMA §24 内存模型对接
  • SIMD 提案 (phase 5, 2.0 正式)—— v128 + 200+ 指令；x86 SSE / ARM NEON 映射到统一抽象
  • autovectorization vs explicit SIMD intrinsics—— 编译器自动 vs 手写
  • WasmGC (phase 5, 2023 正式)—— struct / array / i31ref + subtyping；引擎 GC 接管
  • Tail call (phase 5, 2024 正式)—— return_call / return_call_indirect；让函数式语言可生产部署
  • Exception handling (phase 4)—— try / catch / throw；让 C++ / Java / JS 异常可表达
  • JSPI (phase 4)—— 同步 wasm 调异步 JS；底层用 stack switching
