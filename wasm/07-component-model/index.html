P7 · Component Model 与典型应用

wasm 2026 的"未来"在 Component Model + WIT —— 让 wasm 真正成为"跨语言的二进制库格式"。本阶段一章把它和典型应用一起讲。

章节数 1 主要来源 github.com/WebAssembly/component-model · bytecodealliance · 各应用官方博客 状态 ✅ 完成

本阶段覆盖

  • Component Model—— wasm core module 之上的封装层，加上"接口、类型、链接"概念
  • Component vs Module 区别—— core 只有数值类型；component 有 string / list / record / variant / option / result
  • WIT (WebAssembly Interface Types)—— IDL；定义函数签名 + 类型；类似 protobuf 但 wasm 原生
  • WIT 的 world 概念—— 一组 imports + exports = wasm 想要的环境契约
  • 跨语言互操作黄金路径—— Rust 写组件 → 编译 wasm component → 通过 WIT 接口被 Go 写的组件调用，不需要 C ABI / FFI
  • 工具链—— wasm-tools / wit-bindgen / cargo-component
  • typical apps：Figma · Photoshop Web · Google Earth Web · FFmpeg.wasm · SQLite WASM · DuckDB-WASM · Pyodide · Roblox Studio · Shopify Functions · Envoy Wasm Plugins · Polkadot / Internet Computer / NEAR 智能合约
