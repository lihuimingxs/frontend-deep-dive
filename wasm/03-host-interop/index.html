P3 · 与宿主互操作

wasm 是嵌入式语言，单独跑没意义 —— 必须和宿主（浏览器 / Node / wasmtime）交换数据。这一阶段两章讲清内存 + 接口。

章节数 2 主要来源 WebAssembly JS API · wasm-bindgen 文档 · v8.dev/blog 状态 ✅ 全部完成

本阶段覆盖

  • Linear Memory—— 连续 byte 数组 + 64KB 页 + memory.grow + Memory64 提案
  • JS 端访问 memory—— WebAssembly.Memory.buffer 是 ArrayBuffer，可被 TypedArray view
  • Tables—— 函数引用 / externref 的"另一种内存"，间接调用 + 函数指针替身
  • call_indirect 指令—— 通过 table 索引 + 类型签名调用；C/C++ vtable / 函数指针的编译目标
  • Bulk memory ops—— memory.copy / fill / init / drop（2.0 正式）
  • Imports / Exports 的契约—— wasm 模块声明依赖与暴露，instantiate 时由宿主提供
  • JS-WASM 跨边界调用性能—— 两边不对称（JS-to-WASM 几乎免费，WASM-to-JS 有开销）
  • 数据交换的"序列化税"—— 复杂数据必须穿越 linear memory ↔ JS 堆；wasm-bindgen 模式标准化
  • externref 直传—— 2.0 后 JS 对象作 opaque 引用直接传给 wasm，不必序列化
