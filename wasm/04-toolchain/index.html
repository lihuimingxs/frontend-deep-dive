P4 · 跨语言编译路径

wasm 的多语言是它最大的差异化。这一阶段两章覆盖系统语言（Rust/C++/Zig）+ 托管语言 / 其他（Go/AssemblyScript/Java/Kotlin/Dart/.NET/Python）。

章节数 2 主要来源 rustwasm.github.io · emscripten.org · tinygo.org · kotlinlang.org · pyodide.org 状态 ✅ 全部完成

本阶段覆盖

  • Rust + wasm-pack + wasm-bindgen 黄金组合—— 70%+ 生产 wasm 用 Rust；LLVM 后端 + 无 GC + Cargo 生态
  • C/C++ via Emscripten—— 模拟 POSIX runtime 让现有代码不改源就能跑；产物大、启动慢
  • Zig 原生支持—— 编译器内置 wasm32 / wasm64 target；显式分配哲学契合 wasm
  • Go 两条路—— 官方 GOOS=js 带完整 GC（~2MB）；TinyGo 子集（~50KB）
  • AssemblyScript—— TypeScript 子集 → wasm；前端工程师无障碍但有 GC + 性能不及 Rust
  • WasmGC 让托管语言翻身—— Java/Kotlin/Dart/.NET/OCaml 不必带自己的 GC runtime
  • Pyodide—— CPython 解释器编译 wasm + 数值库；不是 Python 编译，是"在 wasm 里跑 Python 解释器"
  • 代码体积管理—— wasm-opt / strip / panic abort / lto fat / codegen-units 1
