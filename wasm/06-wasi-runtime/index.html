P6 · WASI 与服务端运行时

wasm 走出浏览器是它最大的"超出预期" —— 服务端运行时让 wasm 与 Docker 直接竞争。这一阶段两章把它讲透。

章节数 2 主要来源 wasi.dev · bytecodealliance.org · wasmtime / wasmer / wazero / WasmEdge 文档 状态 ✅ 全部完成

本阶段覆盖

  • WASI Preview 1 (2020-2022)—— POSIX 风格 imports；fd_read / fd_write / clock_time_get；广泛实现但有缺陷
  • Preview 1 设计缺陷—— 复制 POSIX 但 wasm 没进程概念；网络模型不完整；不支持 async；没有接口定义标准
  • Preview 2 (2024 正式)—— 转向 Component Model + WIT；不再模仿 POSIX，用接口契约
  • Capability-based 安全—— 宿主显式授权（"只能读这目录"）；比 Linux capabilities 细粒度
  • wasi:io / wasi:filesystem / wasi:http / wasi:clocks / wasi:random—— Preview 2 的 worlds
  • wasmtime—— Bytecode Alliance；Rust 实现 + Cranelift 后端；事实标准
  • wasmer / wazero / WasmEdge—— 多后端 / 纯 Go / AI 优化各自定位
  • wasm vs Docker—— 冷启动快两个数量级 + 内存基线小 50x + capability 细粒度
  • Cloudflare Workers / Fastly Compute@Edge / Vercel Edge / Shopify Functions—— 主要部署平台
