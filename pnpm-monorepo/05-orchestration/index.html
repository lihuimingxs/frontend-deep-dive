P5 · 任务编排

光有 workspace 不够：50 个包跑 build 要 30 分钟，每次 PR CI 都重跑就是浪费。本阶段讲两家把"任务图 + 缓存"做成主业的工具——Turborepo 与 Nx。

章节数 2 主要来源 turbo.build/repo/docs / nx.dev / Jared Palmer 演讲 / Nx 团队 blog 状态 ⏳ 待写作

本阶段覆盖

  • Turborepo 设计哲学 · 不管包管理；建在 npm/Yarn/pnpm/Bun 之上的 task runner + cache layer
  • turbo.json 核心字段 · tasks.dependsOn / inputs / outputs / cache / globalEnv
  • input/output hash · 文件 + 环境变量 + 工具版本 hash → cache key
  • 远程 cache · Vercel Remote Cache + 自建 S3 / Cloudflare Workers 兼容层
  • Turborepo 2 改进 · filter UI 可视化 cache 命中
  • 环境变量边界陷阱 · 漏配 globalEnv 导致跨环境 cache 命中误判
  • Nx 设计哲学 · 框架式 monorepo：项目图 + plugin + generator + nx migrate
  • Nx 项目图 · 自动从 package.json/tsconfig/源码 import 推断；可视化交互图
  • nx affected · 项目图差分实现"只跑被改影响的项目"
  • nx migrate · 升级版本时自动改 plugin 配置 + 跑 codemods（与 Turborepo 最大差异）
  • Nx Cloud + DTE · 远程 cache + Distributed Task Execution 多 CI 节点并行
  • 横评决策矩阵 · 团队规模 / 项目数 / 框架需求 / cache 复杂度 4 维度推荐
  • 超大规模选项 · Bazel / Rush / Pants 适用边界（短点出，详见 P7）
