P4 · Workspace 协议

monorepo 的核心问题不是"放在一个 repo"，而是"内部多个包如何彼此引用 + 如何统一升级 + 如何选择性构建"。本阶段讲清楚 workspace 协议这一层。

章节数 2 主要来源 pnpm.io / yarnpkg.com / docs.npmjs.com / bun.com/docs 4 家 workspace 文档 状态 ✅ 全部完成

本阶段覆盖

  • 4 家 workspace 配置入口 · npm/Yarn classic/Yarn Berry/pnpm/Bun 各自怎么声明
  • workspace:* 协议 · workspace:^ / workspace:~ / workspace:./path 写法 + publish 时如何被改写
  • catalog 协议（2025 共识） · pnpm 主导 → Bun / Yarn 跟进；命名 catalog 的用途
  • link: 协议 · 与 workspace: 的区别（link 不在 publish 时改写）
  • --filter 选择器 · 按包名 / 路径 / git 变更 / 依赖图前后向选择
  • topological run · pnpm -r run 的依赖排序 + 并行度 + 失败行为
  • 内部包联调 hot reload · 链回 Vite 主题；Vite 监听 node_modules 陷阱
  • TypeScript path mapping vs workspace symlink · 编辑期 vs 运行期的角色边界
  • .npmrc / .pnpmrc · workspace 内的配置作用域优先级
