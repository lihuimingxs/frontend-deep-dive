P1 · 包管理器是谁

不讲技术细节，只解决两个问题：包管理 15 年怎么走到今天、同代人现在的格局。给后面所有"为什么这样设计"提供历史坐标。

章节数 2 主要来源 Isaac Z. Schlueter / Zoltan Kochan / Maël Nison / Jared Palmer 公开文章 + 各家官方公告 状态 ✅ 全部完成

本阶段覆盖

  • 2010 npm 1 · Isaac Z. Schlueter 早期设计：嵌套 node_modules
  • 2015 npm 3 扁平化 · hoisting 解决磁盘 + 路径长度，引入幻影依赖
  • 2016 Yarn classic · Facebook 出品：lockfile + 并行下载 + 离线 cache
  • 2017 pnpm · Zoltan Kochan：CAS + symlink，主张"扁平化是错的"
  • 2019 Yarn Berry / PnP · Maël Nison：取消 node_modules 的激进路线
  • 2022 Lerna 退场 + Turborepo · Vercel 收购，monorepo 工具链重塑
  • 2023 Bun install · Jarred Sumner：Zig + 二进制 lockfile，把"快"做到极致
  • 2025 Catalog 标准化 · pnpm 主导 → Bun / Yarn 跟进的 workspace 协议
  • 2025 JSR · Deno 主导的下一代 registry：原生 ESM + TS
  • 同代人横评 · 6 家全维度对比（速度/磁盘/严格性/workspace/兼容性）
