P6 · 版本与发布

monorepo 写出来要发到 npm。但 12 个包同时 publish、版本怎么联动、内部 workspace:* 怎么改写、CI 怎么对抗供应链攻击——这些是 P6 的主题。

章节数 1 主要来源 changesets/changesets repo / npm provenance 文档 / typescript-eslint / Astro / Vite 项目实践 状态 ✅ 全部完成

本阶段覆盖

  • Changesets 是什么 · Atlassian 出品；markdown 文件 → changelog → 自动发版 PR 工作流
  • 核心工作流 4 步 · pnpm changeset → CI version → Version Packages PR → 合并 publish
  • 独立 vs 固定版本 · 独立（默认）vs fixed: ['@app/*']（紧耦合 SDK）；何时选哪个
  • prerelease 模式 · changeset pre enter alpha；自动加版本后缀
  • snapshot 模式 · 每个 PR 发临时版本（0.0.0-pr-123-abc）让外部消费者验证
  • npm provenance · GitHub Actions OIDC 自动签名包来源；对抗供应链攻击
  • workspace:* 在 publish 时的处理 · pnpm/Bun/Changesets 自动改写为真实版本号
  • 私有 registry 选项 · Verdaccio (自建) / npm Enterprise / GitHub Packages / JFrog Artifactory
  • scope mapping 配置 · .npmrc 中按 scope 路由不同 registry
  • 真实开源案例 · typescript-eslint / Astro / Vite / Remix 都用 Changesets
