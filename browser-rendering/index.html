编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比 V8 / Node 有以下额外约定：

  • 「短重述 + 链回」 —— 当一个机制在 V8 / ECMA / JavaScript 主题已详述时（如 V8 编译流水线 / GC / Microtask Queue / Promise），本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 版本号与浏览器都要标 —— 浏览器特性 cross-browser 兼容性差异大，每个特性必标"Chrome X / Safari Y / Firefox Z"
  • web.dev / Chromium 一手优先 —— 每个机制的"权威定义"必须来自浏览器引擎团队官方文档；caniuse.com 验证兼容性
  • 关注 Spec → 实现差距 —— W3C / WHATWG 规范定义行为，但浏览器实现往往有 quirks；涉及 corner case 时同时引用规范 + 引擎源码 + caniuse

与既有主题的关系

本主题不重复 V8 / ECMA / JS 主题已经讲透的底层机制，但会在每个交叉点提供"短重述 + 链接"。常见对应：

  • P1.2 引擎对照 → V8 P1.2 V8 分身史 + Bun P2.1 JavaScriptCore
  • P2.1 Renderer 进程内 V8 → V8 P5.1 Embedder API（每 Renderer 一个 V8 Isolate）
  • P2.2 Site Isolation + COOP/COEP → ECMA P7.5 §24 Memory Model（SharedArrayBuffer + Atomics）
  • P4.2 Long Tasks + scheduler → ECMA P7.3 Event Loop 规范
  • P4.3 GC pause 作 jank 来源 → V8 P4.2 Orinoco GC
  • P4.1 fetch / preload → 待写的 HTTP/2/3 主题
