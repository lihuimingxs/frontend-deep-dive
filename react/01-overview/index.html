P1 · React 是谁

这一阶段不讲技术细节，只解决两个问题：React 怎么走到今天、它的设计哲学是什么。给后面所有"为什么这样设计"提供历史与基因坐标。

章节数 2 / 2 ✅ 主要来源 React RFCs · Andrew Clark / Dan Abramov 演讲与 blog 状态 ✅ 全部完成

本阶段覆盖

  • 项目史 12 年 —— Facebook 内部 chat 通知 bug → 2013 JSConf 公开 + JSX 争议 → 2017 Fiber 架构重写 → 2018 Hooks 革命 → 2022 Concurrent → 2024 Server Components 稳定
  • 关键设计决策 —— JSX 不是模板是 JS 表达式 / Fiber 重写是 React 的"创世"事件 / Hooks 是函数组件能拥有状态的运行时机制 / 特性发布节奏慢的根因（Facebook 数十亿用户规模约束）
  • 4 大设计哲学 —— UI = f(state) 声明式 / 单向数据流 / Composition over inheritance / 不可变 + 重新渲染
  • 设计代价 —— JSX 不是 HTML / 必须 key 才能高效 list reconciliation / 不可变更新繁琐 / 函数组件每次渲染重新创建函数
