P7 · 生态与工程化

React 19 时代主流生态系统化梳理 + "选什么 / 何时选"决策框架。一章覆盖元框架、状态管理、数据 fetching、路由、表单、测试、UI 库。不是教你怎么用每个库，而是给读者一个"在新项目启动 / 老项目升级时如何选型"的判断锚。

章节数 1 / 1 ✅ 主要来源 各库官方文档 + 2026 React 生态共识 状态 ✅ 全部完成

本阶段覆盖

  • 元框架 —— Next.js App Router（事实标准）/ React Router v7（Remix 合并后）/ TanStack Start（新兴）/ Vite + React（SPA 模式）
  • 状态管理 —— Zustand（最简）/ Jotai（原子化）/ Redux Toolkit（传统）/ XState（状态机）/ MobX（响应式）
  • 数据 fetching —— TanStack Query（事实标准）/ SWR（轻量）/ Apollo Client（GraphQL）/ tRPC（端到端 TS）
  • 路由 —— React Router v7（事实标准）/ TanStack Router（类型安全）/ 元框架内置
  • 表单 —— React Hook Form（最少重渲染）/ Formik（传统）/ TanStack Form（新兴）/ 19 Actions 替代
  • 测试 —— Vitest + React Testing Library（事实标准）/ Playwright / Cypress（E2E）
  • UI 库 —— shadcn/ui（复制源码模式，事实标准）/ Radix UI（headless）/ MUI / Mantine / Tailwind CSS
  • 决策框架 —— 何时 Next.js App Router vs Vite SPA / 何时 Zustand vs Redux Toolkit / 何时 TanStack Query vs RSC
