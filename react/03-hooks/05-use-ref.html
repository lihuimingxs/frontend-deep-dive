P3.5 · useRef / useImperativeHandle

useRef 是 React 的"逃生口"—— 让你脱离 React 的渲染体系，操作 DOM 或保留跨渲染的可变值。useImperativeHandle 是配套的"自定义 ref 接口" 工具。React 19 引入ref as prop—— 不再需要 forwardRef，直接 function Comp({ ref }) 接收 ref。本章讲清这两个 Hook 的3 种典型用法、callback ref 模式、19 时代 ref API 演进，以及"什么时候不该用 ref" 的关键判断。

📐 useRef 是逃生口，不是常规工具

React 哲学是"UI = f(state)"（P1.2）—— 任何能用 state 表达的，都该用 state，不该用 ref。useRef 的合法场景是"非 React 体系的可变值"（DOM 节点 / 命令式 SDK / 跨渲染保留的临时变量）。**滥用 useRef 是新人转 React 最常见的反模式之一**。

阅读时长 ~25 分钟 前置 P2.4 State 基础 + P3.2 useEffect 对应 react.dev useRef · useImperativeHandle

01 速览

三句话定义本章：

  1. useRef(initialValue) 返回 { current: T } 容器。修改 ref.current 不触发重渲染—— 这是 ref 与 state 的核心区别。ref 对象引用跨渲染稳定（同一引用），可以放心传给任何依赖数组（同 setState dispatch，P3.1 讲过）。
  2. 3 个合法用法：① DOM ref（操作 DOM 节点）② 跨渲染保留可变值（如 timer ID / 第三方 SDK 实例 / 滚动位置）③ 缓存上次的 props / state（实现 usePrevious）。不该用：派生 state（用 useMemo）/ 触发渲染的状态（用 useState）。
  3. React 19 革命：ref as prop 取代 forwardRef。19 之前自定义组件接收 ref 必须 forwardRef(function...)，19 之后直接 function Comp({ ref })。useImperativeHandle 仍存在但不需要 forwardRef 包装。callback ref 在 19 也支持 cleanup 函数。

看一段最小代码识别 useRef 的核心特性：

javascript · useRef 完整骨架 
function Component() {
  // ① DOM ref —— 引用 DOM 节点
  const inputRef = useRef(null);

  // ② 跨渲染可变值 —— interval ID
  const timerRef = useRef(null);

  useEffect(() => {
    timerRef.current = setInterval(() => {
      // 修改 ref 不触发渲染
      console.log('tick');
    }, 1000);
    return () => clearInterval(timerRef.current);
  }, []);

  // ③ 缓存上次 props
  const prevQueryRef = useRef('');
  // 用 useEffect 记录每次 query 变化
  // ...

  return (
    <input
      ref={inputRef}                              {/* React 自动设 inputRef.current */}
      onChange={() => inputRef.current?.focus()}  {/* 命令式调用 */}
    />
  );
}

// 关键：
// - inputRef = { current: null } 一直保持同一引用
// - 修改 inputRef.current 不触发任何渲染
// - 直到组件 mount 后 React 自动赋值（ref attach 在 commit 阶段）

02 底层逻辑

useRef 完整签名与心智

typescript · useRef 类型定义 
// 重载 1：传初始值
function useRef<T>(initialValue: T): MutableRefObject<T>;

// 重载 2：T 包含 null 时（DOM ref 的常见用法）
function useRef<T>(initialValue: T | null): RefObject<T>;

// 重载 3：完全 readonly（极少用）
function useRef<T = undefined>(): MutableRefObject<T | undefined>;

interface MutableRefObject<T> { current: T; }
interface RefObject<T> { readonly current: T | null; }

// 用法对比：
const inputRef = useRef<HTMLInputElement>(null);   // RefObject（DOM ref）
//    inputRef: { readonly current: HTMLInputElement | null }
//    React 内部赋值，开发者只读

const counterRef = useRef(0);                       // MutableRefObject（手动管理）
//    counterRef: { current: number }
//    可写：counterRef.current = 5

// 关键事实：
// 1. ref 对象引用永远稳定 —— 同一组件实例的 useRef 永远返回同一对象
// 2. 修改 ref.current 不触发渲染（React 不订阅 ref 变化）
// 3. 初始值仅在首次 render 设置（同 useState 行为）
// 4. ref 在 commit 阶段被 React 赋值（DOM ref）—— 早期 render 阶段是 null

用法 1：DOM ref（操作 DOM 节点）

javascript · DOM ref 经典模式 
// 1. 自动 focus
function AutoFocusInput() {
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    inputRef.current?.focus();  // mount 后聚焦
  }, []);

  return <input ref={inputRef} />;
}

// 2. 滚动控制
function ScrollableList() {
  const listRef = useRef<HTMLUListElement>(null);

  const scrollToTop = () => {
    listRef.current?.scrollTo({ top: 0, behavior: 'smooth' });
  };

  return (
    <>
      <button onClick={scrollToTop}>Top</button>
      <ul ref={listRef}>...</ul>
    </>
  );
}

// 3. 测量 DOM 尺寸
function MeasuredDiv() {
  const divRef = useRef<HTMLDivElement>(null);
  const [size, setSize] = useState({ width: 0, height: 0 });

  useLayoutEffect(() => {  // 用 useLayoutEffect 避免闪烁（P3.2）
    if (divRef.current) {
      const { width, height } = divRef.current.getBoundingClientRect();
      setSize({ width, height });
    }
  }, []);

  return <div ref={divRef}>{size.width} × {size.height}</div>;
}

// 4. 集成第三方 DOM 库（详见 P3.2 应用场景）
function ChartContainer() {
  const containerRef = useRef<HTMLDivElement>(null);
  useEffect(() => {
    const chart = new ChartJS(containerRef.current, ...);
    return () => chart.destroy();
  }, []);
  return <div ref={containerRef} />;
}

// 关键时序：
// - render 阶段：inputRef.current 仍是初始值（null）
// - commit 阶段（P4.1 详述）：React attach DOM 节点到 ref.current
// - useEffect / useLayoutEffect 跑时：可访问 ref.current
// - useState 触发的 render 中：ref.current 仍指向上次 commit 的 DOM

用法 2：跨渲染保留可变值

javascript · 跨渲染可变值经典模式 
// 1. 保存 timer ID（避免泄漏）
function Countdown() {
  const [seconds, setSeconds] = useState(60);
  const timerRef = useRef<number | null>(null);

  const start = () => {
    if (timerRef.current) return;  // 已开始，跳过
    timerRef.current = window.setInterval(() => {
      setSeconds(s => s - 1);
    }, 1000);
  };

  const stop = () => {
    if (timerRef.current) {
      clearInterval(timerRef.current);
      timerRef.current = null;
    }
  };

  useEffect(() => stop, []);  // unmount cleanup

  return ...;
}

// 2. 保存第三方 SDK 实例
function MapView({ center }) {
  const containerRef = useRef(null);
  const mapRef = useRef<MapSDK | null>(null);

  useEffect(() => {
    mapRef.current = new MapSDK(containerRef.current);
    return () => mapRef.current?.destroy();
  }, []);

  // 后续 effect 访问 mapRef.current 操作 map
  useEffect(() => {
    mapRef.current?.setCenter(center);
  }, [center]);

  return <div ref={containerRef} />;
}

// 3. 缓存最新值供 effect 访问（解 useEffect 闭包陷阱）
function Component({ onChange }) {
  // onChange 每次 render 可能新引用 —— 不想列入 effect 依赖
  const onChangeRef = useRef(onChange);

  // 每次 render 同步最新 onChange 到 ref
  useEffect(() => {
    onChangeRef.current = onChange;
  });   // 不传依赖 —— 每次 render 后跑一次

  // effect 不依赖 onChange，但能访问最新版本
  useEffect(() => {
    const handler = (e) => onChangeRef.current(e);  // 永远最新
    window.addEventListener('keydown', handler);
    return () => window.removeEventListener('keydown', handler);
  }, []);   // 仅 mount 时建立
  // 注意：React 19 useEffectEvent 提案（P3.2）是这个模式的官方解
}

// 4. 跟踪 mount 状态（避免组件卸载后 setState）
function Component() {
  const isMountedRef = useRef(true);
  useEffect(() => () => { isMountedRef.current = false; }, []);

  const fetchData = async () => {
    const data = await fetch('/api');
    if (!isMountedRef.current) return;  // 已卸载，跳过 setState
    setData(data);
  };
  // 注意：现代代码用 AbortController 更优（P3.2 详述）
}

用法 3：缓存上次值（usePrevious 模式）

javascript · usePrevious 经典实现 
function usePrevious<T>(value: T): T | undefined {
  const ref = useRef<T>();

  useEffect(() => {
    ref.current = value;  // commit 后存当前值
  });

  return ref.current;  // render 中返回的是"上次 commit 的值"
}

// 用法
function Component({ count }) {
  const prevCount = usePrevious(count);

  return (
    <div>
      Now: {count}, was: {prevCount}
    </div>
  );
}

// 时序：
// 第一次 render（count = 0）：
//   - ref.current 还是 undefined（effect 还没跑）
//   - 返回 undefined
//   - commit 后 effect 跑：ref.current = 0
//
// 第二次 render（count = 1）：
//   - ref.current 是 0（上次 commit 的值）
//   - 返回 0
//   - commit 后 effect 跑：ref.current = 1
//
// 这是 React 中"读取上次状态" 的经典模式
// 注意：ref 在 render 中读 / effect 中写 —— 渲染纯函数性不破坏（render 不写 ref）

useImperativeHandle 自定义 ref 接口

默认 ref 转发拿到的是 DOM 节点。useImperativeHandle 让组件自定义 ref 暴露的方法—— 类似类的 imperative API：

javascript · useImperativeHandle 完整模式 
// React 19+ 写法（无 forwardRef）
type ModalHandle = {
  open: () => void;
  close: () => void;
};

function Modal({ ref }: { ref?: React.Ref<ModalHandle> }) {
  const [open, setOpen] = useState(false);

  // 自定义 ref 暴露的方法
  useImperativeHandle(ref, () => ({
    open: () => setOpen(true),
    close: () => setOpen(false),
  }), []);  // 依赖数组：仅 mount 时创建一次

  if (!open) return null;
  return <div className="modal">...</div>;
}

// 用法
function App() {
  const modalRef = useRef<ModalHandle>(null);
  return (
    <>
      <button onClick={() => modalRef.current?.open()}>Open Modal</button>
      <Modal ref={modalRef} />
    </>
  );
}

// React 18 及以下写法（forwardRef）—— 19 退役但仍兼容
const Modal = forwardRef<ModalHandle>(function Modal(props, ref) {
  const [open, setOpen] = useState(false);

  useImperativeHandle(ref, () => ({
    open: () => setOpen(true),
    close: () => setOpen(false),
  }), []);

  if (!open) return null;
  return <div className="modal">...</div>;
});

// 何时该用 useImperativeHandle：
// 1. 组件需要暴露命令式 API（open / close / focus / scrollTo / play）
// 2. 不想直接暴露内部 DOM 节点（封装边界）
// 3. 整合命令式第三方库时

// 何时不该用：
// - 大多数 React 模式 —— 用 props + state 表达，更声明式
// - 仅暴露一个 DOM 操作 —— 直接用 ref={内部节点 ref}

React 19 ref as prop（取代 forwardRef）⭐

javascript · forwardRef → ref as prop 
// React 18 及以下：必须 forwardRef
const Input = forwardRef<HTMLInputElement, InputProps>(function Input(props, ref) {
  return <input {...props} ref={ref} />;
});

// 用 forwardRef 的痛点：
// 1. 函数签名复杂（props 在第一个参数，ref 在第二个）
// 2. TypeScript 类型啰嗦
// 3. 多包一层导致 displayName 容易丢
// 4. 与高阶组件配合困难

// React 19+：ref 作为普通 prop
function Input({ ref, ...props }: InputProps & { ref?: React.Ref<HTMLInputElement> }) {
  return <input {...props} ref={ref} />;
}

// 优势：
// 1. 函数签名扁平（ref 就是 prop）
// 2. TypeScript 类型自然
// 3. 与高阶组件配合简单（ref 只是 prop）
// 4. 不需要包装层

// 19 兼容性：
// - forwardRef 仍可用（向后兼容）
// - 推荐新代码用 ref as prop
// - 老库逐步迁移
// - codemod 工具：npx react-codemod replace-act-import

// 完整 19 模式：自定义 imperative handle
type DialogHandle = {
  open: () => void;
  close: () => void;
};

function Dialog({ ref, title, children }: {
  ref?: React.Ref<DialogHandle>;
  title: string;
  children: React.ReactNode;
}) {
  const [open, setOpen] = useState(false);
  useImperativeHandle(ref, () => ({
    open: () => setOpen(true),
    close: () => setOpen(false),
  }), []);
  return open ? (
    <div className="dialog">
      <h2>{title}</h2>
      {children}
    </div>
  ) : null;
}

callback ref + React 19 cleanup ⭐

除了对象 ref（useRef 返回的），React 还支持 callback ref—— 把函数当 ref 传：

javascript · callback ref 模式 
// 基础 callback ref
<input ref={(node) => {
  console.log('attached:', node);
  // node 是 DOM 节点（mount 时）或 null（unmount 时）
}} />

// 经典用例：测量 DOM
function MeasuredDiv() {
  const [size, setSize] = useState(null);

  return (
    <div ref={(node) => {
      if (node) {
        const rect = node.getBoundingClientRect();
        setSize({ width: rect.width, height: rect.height });
      }
    }}>
      {size ? `${size.width} × ${size.height}` : '...'}
    </div>
  );
}

// React 19 增强：callback ref 支持 cleanup 函数
<input ref={(node) => {
  console.log('attached');
  // 注册某些操作
  return () => {
    console.log('detached');
    // cleanup 操作（19 之前必须用 useEffect）
  };
}} />

// 实际用例（19）：
<div ref={(node) => {
  if (!node) return;
  const observer = new ResizeObserver(handleResize);
  observer.observe(node);
  return () => observer.disconnect();   // 19 自动调用
}} />

// 19 之前同款逻辑要这样写：
const divRef = useRef(null);
useEffect(() => {
  if (!divRef.current) return;
  const observer = new ResizeObserver(handleResize);
  observer.observe(divRef.current);
  return () => observer.disconnect();
}, []);
<div ref={divRef} />
// 19 callback ref 模式更简洁

// callback ref vs 对象 ref 取舍：
// - 对象 ref：组件多处用、传给子组件、配合 useImperativeHandle
// - callback ref：仅 attach/detach 时点要做事 / 多 ref 合并 / 需要 cleanup（19）

useRef 不该当 useState 用

这是新人最常踩的反模式—— 用 ref "避免重渲染" 但导致 UI 不更新：

javascript · ref 不该当 state 用 
// ❌ 用 ref 存"会显示在 UI 上的值"
function BrokenCounter() {
  const countRef = useRef(0);

  return (
    <button onClick={() => {
      countRef.current++;        // 修改 ref
      console.log(countRef.current);  // log 是更新的
    }}>
      Count: {countRef.current}    // ⚠️ UI 永远显示 0！
                                    // 因为修改 ref 不触发渲染
    </button>
  );
}

// ✅ UI 显示的值用 useState
function CorrectCounter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

// 决策：
// - 修改后要在 UI 上反映 → useState
// - 修改后不需要在 UI 上反映 → useRef
// - 派生值（其他 state 计算出来的）→ 直接计算或 useMemo
// - 跨渲染的副作用结果（timer ID / SDK 实例）→ useRef

// 测试：在脑里问 "这个值变化时 UI 需要重新渲染吗"
// - 是 → useState
// - 否 → useRef

03 应用场景

场景 A · 实现 useDebounce 自定义 Hook

搜索框防抖：用户停止输入 500ms 后才触发查询。用 useRef 保存 timer ID：

javascript · useDebounce 
function useDebounce<T>(value: T, delay = 500): T {
  const [debouncedValue, setDebouncedValue] = useState(value);
  const timerRef = useRef<number | null>(null);

  useEffect(() => {
    if (timerRef.current) clearTimeout(timerRef.current);

    timerRef.current = window.setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      if (timerRef.current) clearTimeout(timerRef.current);
    };
  }, [value, delay]);

  return debouncedValue;
}

// 用法
function Search() {
  const [query, setQuery] = useState('');
  const debouncedQuery = useDebounce(query, 500);

  // 仅 debouncedQuery 变化时才触发
  useEffect(() => {
    if (debouncedQuery) fetchResults(debouncedQuery);
  }, [debouncedQuery]);

  return <input value={query} onChange={e => setQuery(e.target.value)} />;
}

场景 B · 集成命令式动画库（GSAP / Framer Motion）

用 GSAP 做复杂动画。useRef 引用 DOM + useImperativeHandle 暴露 play / pause API：

javascript · GSAP 动画封装 
type AnimationHandle = {
  play: () => void;
  pause: () => void;
  reset: () => void;
};

function AnimatedBox({ ref, color }: {
  ref?: React.Ref<AnimationHandle>;
  color: string;
}) {
  const boxRef = useRef<HTMLDivElement>(null);
  const tweenRef = useRef<gsap.core.Tween | null>(null);

  useEffect(() => {
    if (!boxRef.current) return;
    tweenRef.current = gsap.to(boxRef.current, {
      x: 200,
      duration: 2,
      paused: true,  // 不立即播放
    });
    return () => tweenRef.current?.kill();
  }, []);

  useImperativeHandle(ref, () => ({
    play: () => tweenRef.current?.play(),
    pause: () => tweenRef.current?.pause(),
    reset: () => tweenRef.current?.progress(0).pause(),
  }), []);

  return <div ref={boxRef} style={{ background: color, width: 50, height: 50 }} />;
}

// 用法
function App() {
  const animRef = useRef<AnimationHandle>(null);
  return (
    <>
      <AnimatedBox ref={animRef} color="red" />
      <button onClick={() => animRef.current?.play()}>Play</button>
      <button onClick={() => animRef.current?.pause()}>Pause</button>
    </>
  );
}

场景 C · 19 callback ref 替代 useEffect 测量 DOM

React 19 callback ref + cleanup 让"测量 DOM 尺寸"模式更简洁：

javascript · 18 vs 19 测量 DOM 
// === React 18 写法 ===
function MeasuredBox() {
  const ref = useRef<HTMLDivElement>(null);
  const [size, setSize] = useState({ width: 0, height: 0 });

  useEffect(() => {
    if (!ref.current) return;
    const observer = new ResizeObserver(([entry]) => {
      const { width, height } = entry.contentRect;
      setSize({ width, height });
    });
    observer.observe(ref.current);
    return () => observer.disconnect();
  }, []);

  return <div ref={ref}>{size.width} × {size.height}</div>;
}

// === React 19 callback ref 写法（更简洁）===
function MeasuredBox() {
  const [size, setSize] = useState({ width: 0, height: 0 });

  return (
    <div ref={(node) => {
      if (!node) return;
      const observer = new ResizeObserver(([entry]) => {
        const { width, height } = entry.contentRect;
        setSize({ width, height });
      });
      observer.observe(node);
      return () => observer.disconnect();   // 19 自动调用 cleanup
    }}>
      {size.width} × {size.height}
    </div>
  );
}
// 减少 useRef + useEffect 两层嵌套

04 典型代码

多 ref 合并（forkRef 模式）

javascript · 同一节点多个 ref 
// 痛点：同一 DOM 节点要给多个用户（自家 ref + 第三方库 ref）
// 用 callback ref 合并

function mergeRefs<T>(...refs: Array<React.Ref<T> | undefined>): React.RefCallback<T> {
  return (node) => {
    refs.forEach((ref) => {
      if (!ref) return;
      if (typeof ref === 'function') ref(node);
      else (ref as React.MutableRefObject<T | null>).current = node;
    });
  };
}

// 用法
function Input({ ref: externalRef }: { ref?: React.Ref<HTMLInputElement> }) {
  const internalRef = useRef<HTMLInputElement>(null);

  return <input ref={mergeRefs(internalRef, externalRef)} />;
}

// 现在 internalRef 和 externalRef 都指向同一个 input 节点
// 库类用例：react-aria、react-spring 等都用这个模式

用 useRef 跟踪首次渲染

javascript · useFirstRender 自定义 Hook 
function useIsFirstRender() {
  const isFirstRef = useRef(true);
  useEffect(() => {
    isFirstRef.current = false;
  }, []);
  return isFirstRef.current;
}

// 用法
function Component({ value }) {
  const isFirstRender = useIsFirstRender();

  useEffect(() => {
    if (isFirstRender) return;   // 首次 render 跳过
    // 仅后续 value 变化时执行
    fetchData(value);
  }, [value]);

  return ...;
}

// 注意 StrictMode 双调：开发环境 isFirstRender 行为可能不直观
// 更可靠：用 useEffect 不跑 vs 跑（自然区分 mount vs update）

05 常见陷阱

⚠️ 陷阱 1：用 ref 当 state（UI 不更新）

const countRef = useRef(0) + 在 JSX 用 {countRef.current}—— 修改 ref 不触发渲染，UI 永远显示初值。

规避：UI 显示的值必须用 useState。判断方法：这个值变化时 UI 需要更新吗—— 是用 state，否用 ref。

⚠️ 陷阱 2：在 render 阶段读 / 写 ref

function Component() { ref.current = 5; return ...; }—— render 阶段写 ref破坏纯函数性，Concurrent rendering 时不安全（多次调用会写多次）。

规避：render 阶段不读不写 ref（除非确实需要 lazy initial 类的特殊场景）。读写都放 useEffect / event handler / useImperativeHandle。

⚠️ 陷阱 3：DOM ref 在 render 阶段访问报 null

const inputRef = useRef(null); console.log(inputRef.current);（在 render 函数体）—— 永远 null，因为 ref 在 commit 阶段才 attach。

规避：DOM ref 仅在 useEffect / useLayoutEffect / event handler 内访问（mount 之后才有 DOM）。

⚠️ 陷阱 4：useImperativeHandle 不传 deps 导致每次 render 重建

useImperativeHandle(ref, () => ({ ... }))（不传 deps）—— 每次 render 重建 ref handle 对象，传给 parent 的 ref 永远变化，破坏 memo / useEffect 稳定性。

规避：always 传 deps 数组。useImperativeHandle(ref, () => ({ ... }), []) 或者列出影响 handle 内函数行为的依赖。

⚠️ 陷阱 5：把 setState 放进 ref 看起来 OK 但 stale

const setStateRef = useRef(setState)—— setState 引用永远稳定（P3.1），不需要 ref 包装。这种"过度防御"反而让代码难读 + 容易 stale（如果之后改成 set state 通过 props 传，ref 不会更新）。

规避：理解哪些值引用稳定（setState / dispatch / Context Provider value 等），不要给它们加 ref 包装。
