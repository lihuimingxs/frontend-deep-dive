Track A · 历史脉络与设计

5 章按 4 个问题展开 React 13 年演化：为什么存在（A1）→ 它是什么（A2）→ 机器怎么跑（A3）→ 单线程怎么并发（A4），最后一章讲现代生态与工程化决策树（A5）。这 4 问骨架是从 ECMAScript 主题继承的"叙事母版"——每个 React 设计决策都能在这条 timeline 上找到位置。

章节数 5 / 5（A1 + A2 + A3 + A4 已完成） 来源 react.dev · React RFCs · Andrew Clark / Sebastian Markbåge / Dan Abramov 的 blog 前置 JS 基础 + ES2015 class 概念

本 Track 解决什么问题

  • React 为什么这么古怪 —— A1 + A2 共同解释：JSX 设计争议、单向数据流的来历、UI = f(state) 哲学
  • Fiber 到底是什么 —— A3 拆开 Fiber 节点 / 双缓冲 / Work loop / Lane 模型 / Hooks linked list 的全部内部数据结构
  • Concurrent 怎么做到"单线程并发" —— A4 串起时间切片 → Suspense → Transition → Server Components 的完整异步演进
  • 2025 年该用什么 React 生态 —— A5 给出元框架 / 状态管理 / 数据 fetching / 路由 / UI 库的决策树

章节清单

序号 章节 核心收获
A1 起源与演化 ——「为什么存在？」 2013 Facebook 起源、JSX 早期争议、Fiber 重写、Hooks 革命、Concurrent / Server Components / Actions
A2 设计哲学 ——「它是什么？」 UI = f(state)、单向数据流、Composition over inheritance、不可变 + 重新渲染 4 个核心决策
A3 Fiber 架构 ——「机器怎么跑？」⭐ Fiber 节点、双缓冲、Work loop、Lane 模型、Hooks linked list、Reconciliation 算法
A4 Concurrent 与异步 ——「单线程怎么并发？」 时间切片、Suspense、useTransition、Server Components 异步边界、Actions 与表单
A5 生态与工程化 🚧 元框架（Next.js / Remix / Vite）、状态管理（Zustand / Redux Toolkit / Jotai / XState）、数据 fetching（TanStack Query / SWR / RSC）决策树

阅读建议

  • 建议连读 A1 → A2 → A3 → A4：4 问是一条叙事弧，分开读会泄气
  • 可独立跳读 A5：作为新项目选型 onboarding，看完 A5 就能定 React 19 时代的工程方案
  • 写业务 React 工程师：A1 + A2（不必读 A3/A4 的引擎细节）
  • 架构师 / 想理解引擎：A1-A4 全读 + 配合 Track B 官网知识 补 API 细节

与既有主题的对应

A 轨与 JavaScript / TypeScript / ECMAScript 主题有大量交叉。本轨严格遵守「短重述 + 链回」：

  • A2 设计哲学 → JS 闭包 / 原型链（不可变性 vs 引用相等）
  • A3 Fiber → ECMA A3 执行模型（Agent / Realm 概念） + JS 宏任务/微任务
  • A4 Concurrent → ECMA A4 异步演进（5 代异步演化）
