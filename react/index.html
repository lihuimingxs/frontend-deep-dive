编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。但 React 主题相比 JS / TS / ECMA 有以下额外约定：

  • Track B 每章先讲心智模型 —— "这个 API 为什么存在" 永远在 "怎么用" 之前。这是 React 主题相对其他主题更强调的部分
  • 「短重述 + 链回」 —— JS 闭包 / Promise / Event Loop、TS 类型系统、ECMA microtask 调度等已讲透的机制，本主题用 1-3 句话点本质 + 链回
  • RFC 引用 —— 关键设计决策必引 RFC 原文（Server Components / Actions / Hooks 等）
  • 仅 A3 Fiber 章读源码 —— 其他章节以官网 + RFC 为限，避免实现细节稀释心智模型
  • ⚠️ 标注 —— Class component / forwardRef / defaultProps / String refs 等已退役 API 始终标注（19 仍支持但新代码避免）

与既有主题的关系

本主题不重复 JS / TS / ECMA 已讲透的内容，但在每个交叉点提供"短重述 + 链接"。常见对应：

  • Hooks 闭包陷阱 → JS 闭包内存模型
  • useEffect 内 async / Promise → JS P3 异步 & 错误（全 8 章）
  • React Props 类型 → TS Functions / Generics
  • Fiber 微任务调度 → ECMA A4 异步演进 / JS 宏任务/微任务
