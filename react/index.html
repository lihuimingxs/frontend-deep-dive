P1 · React 是谁

✅ 2 章完成

React 项目史 2013-2025（Facebook chat 通知 bug → 2013 JSConf 公开 → 2017 Fiber 重写 → 2018 Hooks 革命 → 2022 Concurrent → 2024 RSC 稳定）+ 设计哲学（UI = f(state) / 单向数据流 / Composition over inheritance / 不可变 + 重新渲染）。给后面所有"为什么这样设计"提供历史与基因坐标。

项目史 12 年 JSX 设计 UI = f(state) 单向数据流 Composition 不可变更新

P2 · 渲染心智模型

✅ 5 章完成

写 React 组件需要建立的核心心智模型：React 元素是对象不是 DOM、JSX 是 JS 表达式不是模板、组件是函数（class 已老）、Props 设计与 ref as prop（19）、State 触发 render 的批处理与函数式更新、合成事件系统、受控 vs 非受控表单。

React 元素 JSX runtime Props / children State 批处理 条件 / 列表 / key 合成事件

P3 · Hooks 全谱

✅ 6 章完成

函数组件怎么拥有状态——Hooks 是 React 19 时代的核心。useState（hook linked list 节点 + dispatch 引用稳定 + Object.is bailout）/ useEffect（commit 后异步 + cleanup + 依赖数组）/ useReducer + useContext / useMemo + useCallback（缓存语义 + 滥用反模式）/ useRef（DOM ref + mutable instance var）/ 自定义 Hooks。

useState 引擎层 useEffect 时机 useReducer useContext useMemo / useCallback useRef 自定义 Hooks

P4 · Fiber 内部机制

✅ 2 章完成 · 含最难章

React 内部最神秘也最重要的部分。Fiber 节点 = 工作单元（链表而非树、可中断遍历）+ 双缓冲（current / workInProgress）+ work loop（performUnitOfWork / shouldYield）+ Lane 模型（31-bit bitmask 优先级）+ Render / Commit 阶段拆分 + Hooks linked list（解释为什么 Hooks 必须按相同顺序调用）+ Reconciliation diff 算法。

Fiber 节点 双缓冲 Work loop Lane 模型 Render / Commit Hooks linked list Reconciliation

P5 · Concurrent 并发

✅ 4 章完成

把 P4 Fiber 能力翻译成用户可见特性——时间切片（5ms work + yield to browser → 60fps）+ 优先级调度 + Suspense（throw promise 模式 + RSC 流式渲染）+ useTransition / useDeferredValue（低优先级更新）+ Error Boundary（错误恢复策略 + 与 Suspense 协作）。

时间切片 Suspense throw promise useTransition useDeferredValue Error Boundary

P6 · Server Components 与 Actions

✅ 4 章完成

React 19 时代的核心新特性。Server Components（服务器状态 + 客户端状态两层模型 / 'use client' 边界 / 流式渲染）+ Actions（useActionState / useFormStatus / useOptimistic / 表单 progressive enhancement）+ use API（在 render 中读 Promise + Context）+ 现代杂项（ref as prop / Context 简化 / document metadata）。

Server Components 'use client' Actions useActionState useOptimistic use API ref as prop

编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。React 主题相比其他主题有以下额外约定：

  • 「短重述 + 链回」 —— 当一个机制在 JavaScript / TypeScript / V8 / ECMA 主题已详述时（如闭包陷阱、Promise、Event Loop、V8 Hidden Class、Module Records），本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 设计动机 vs API 说明 —— Reference 章节不只讲 API 怎么用，更讲 RFC 设计动机（为什么会有这个 API）+ 心智模型 + 边界情况 + 与 React 内部机制的关系
  • RFC 优先 —— 关键设计决策必引 RFC（Server Components #188 / Actions #229 / Hooks #68）
  • 源码访问受限 —— 仅 P4.1 Fiber 章涉及 reconciler / scheduler 源码，其他章节不读源码避免实现细节稀释心智模型
  • ⚠️ 标注 —— Class component 永远标"已老但仍在维护"；deprecated API（forwardRef / defaultProps / componentWillMount）永远标"19 退役"

与既有主题的关系

React 主题不重复 JS / TS / V8 / ECMA 已经讲透的底层机制，但会在每个交叉点提供"短重述 + 链接"，重点讲 React 在应用层做的设计。常见对应：

  • P6.1 RSC → 独立的 Next.js 主题（待写，讲 App Router 实现 RSC 的工程层）
