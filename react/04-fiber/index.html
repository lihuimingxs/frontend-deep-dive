P4 · Fiber 内部机制

React 内部最神秘也最重要的部分。讲清楚 2016 年 Fiber 重写的具体机制——理解后所有"为什么 useEffect 在 commit 后跑"、"为什么 Concurrent 能时间切片"、"为什么 Suspense 能 throw promise"都有了根因。

章节数 2 / 2 ✅ 主要来源 React 源码 reconciler / scheduler 包 · Andrew Clark / Lin Clark 演讲 状态 ✅ 全部完成

本阶段覆盖

  • Fiber 节点 = 工作单元 —— 链表而非树（可中断遍历）+ 关键字段（type / stateNode / child / sibling / return / memoizedState / flags / lanes）
  • 双缓冲 —— current tree（屏幕上的）+ workInProgress tree（构建中的）；commit 时切换
  • Work loop —— workLoopSync vs workLoopConcurrent；performUnitOfWork → beginWork → completeUnitOfWork；shouldYield 让出主线程
  • Lane 模型（React 18+）—— 31-bit bitmask 优先级；SyncLane / TransitionLane / IdleLane 等；多个更新合并到同一 lane
  • Render / Commit 阶段 —— Render 可中断、Commit 不可中断（Before mutation / Mutation / Layout 三子阶段 + 异步 Passive effects）
  • Hooks linked list —— 解释为什么 Hooks 必须按相同顺序调用
  • Reconciliation diff 算法 —— 同 type 复用 / 不同 type 销毁 / 列表 key 配对
