Track B · React 官网知识精读

按 react.dev Reference 系统化的 19 章 / 4 Part。每章不是"API 字典"，而是"心智模型 + 完整签名 + 典型代码 + 边界情况 + 与 React 内部机制的关系"。读完后你能解释为什么 React 的每个 API 是这个样子，而不只是"知道怎么用"。

章节数 1 / 19（B1 已完成） 来源 react.dev Reference + React RFCs 前置 JS 基础 + 推荐先看 A1 起源 + A2 设计哲学

本 Track 解决什么问题

  • React API 为什么是这个样子 —— 每章先讲心智模型（"这个 API 为什么存在"）再讲签名
  • useEffect 到底怎么用 —— B7 单独成章，给出最难 Hook 的完整心智模型 + 陷阱大全
  • React 19 怎么写 —— Part 4 详述 Server Components / Actions / `use` API / ref as prop
  • 什么时候该用什么 Hook —— Part 2 系统化讲 6 类 Hook 的取舍
