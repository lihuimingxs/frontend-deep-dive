测试 + 部署 + 监控 + caching + cost 控制 + AI Gateway

把 AI SDK 上生产之前要解决：怎么测试 LLM 流式响应、Edge / Node / Bun runtime 在长 agent 任务上的限制、监控（OTel / DevTools）、caching（不每次都打 LLM）、cost 控制（token 烧钱）。这一章一章覆盖这 6 个工程话题，给生产部署提供完整 checklist。

阅读时长 ~22 分钟 前置 读过 P2-P4（架构 + Stream Protocol + Agent + Tool） 主要来源 ai-sdk.dev/docs/troubleshooting + Vercel Edge runtime 文档 + Anthropic Prompt Caching docs + Vercel AI Gateway 文档 + OpenTelemetry SDK 文档 + 测试主题 P6 MSW

01 速览

  1. 测试三件套：(a) MockLanguageModelV2（v5+ 内置）让 streamText 在测试里返回固定响应；(b) MSW 网络层 mock（链回 测试主题 P6）；(c) tool execute 函数直接 unit test（不需要跑 LLM）。不要 mock useChat / streamText 的内部，mock 边界（network 层）。
  2. Edge runtime 边界：Vercel Edge Function / Cloudflare Workers 默认 25-30s timeout（可调到 5 分钟）。某些 Node API 不支持（fs / child_process / native modules）。长 agent 任务（5+ 分钟）不要 Edge，用 Node runtime 或后台 worker。
  3. Bun runtime（链回 Bun 主题）：完整 Web Streams 支持；Bun.serve 流式响应；ai SDK v5+ 完全兼容。生产越来越多用 Bun（性能 + DX）。
  4. Cloudflare Workers：Streams 完整；CPU time limits（典型 30s，付费 5 分钟）；适合 edge 流式响应 + 接 MCP HTTP servers。
  5. OTel observability（v6 内置）：默认导出 spans for streamText / generateText / tool execution / agent step。开箱接 Datadog / New Relic / Honeycomb / Langfuse。
  6. DevTools（v6 浏览器扩展）：可视化看 useChat 内部 state / message parts / tool calls / agent steps。类似 React DevTools 但专门给 AI SDK。
  7. Caching 三层：(a) Anthropic Prompt Caching（长 system prompt cache，重复请求只付 cache 价 25-90% 折扣）；(b) Vercel AI Gateway caching（v6 引入，统一 cache 层）；(c) 应用层 cache（Redis / KV，相同输入完全 cache 化整个响应）。
  8. Cost 控制：监控 usage（token 数）+ AI Gateway budget controls + 模型分级（Haiku 跑 routing / Sonnet 跑核心 / o1 仅复杂推理）+ prompt caching。不监控 token 消耗 = 月底惊喜账单。
  9. AI Gateway（v6 起）：单一 API key + zero markup pricing + 统一 observability + budget + caching + fallback + prompt caching auto-enable。多 provider 切换需求大的项目几乎没理由不用。

02 底层逻辑

测试：边界 mock，不内部 mock

AI SDK 测试的核心原则：mock 边界（network 层 / model 层），不 mock SDK 内部。

typescript 
// ❌ 错误：mock useChat 的内部
vi.mock('ai/react', () => ({
  useChat: () => ({ messages: [...], sendMessage: vi.fn() }),
}));
// 问题：mock 的内部和真实 SDK 行为可能 drift
// 升级 SDK 后测试假绿

// ✅ 正确：mock model 层 (MockLanguageModelV2)
import { MockLanguageModelV2 } from 'ai/test';
import { streamText } from 'ai';

const mockModel = new MockLanguageModelV2({
  doStream: async () => ({
    stream: simulateReadableStream({
      chunks: [
        { type: 'text-delta', textDelta: 'Hello' },
        { type: 'text-delta', textDelta: ' world' },
        { type: 'finish', finishReason: 'stop', usage: { ... } },
      ],
    }),
  }),
});

const result = streamText({ model: mockModel, prompt: 'test' });
// 走完整 SDK 路径，但不打实际 LLM

tool execute 单元测试

Tool execute 函数本质是普通函数，直接 unit test，不需要跑 LLM：

typescript 
// tools/lookupOrder.ts
import { tool } from 'ai';
import { z } from 'zod';

export const lookupOrder = tool({
  description: 'Look up an order',
  parameters: z.object({ orderId: z.string() }),
  execute: async ({ orderId }) => await db.order.findUnique({ where: { id: orderId } }),
});

// __tests__/lookupOrder.test.ts
import { lookupOrder } from '@/tools/lookupOrder';

test('lookupOrder 找到订单', async () => {
  const result = await lookupOrder.execute({ orderId: 'ORD-1' });
  expect(result.id).toBe('ORD-1');
});

test('订单不存在返回 null', async () => {
  const result = await lookupOrder.execute({ orderId: 'NONEXISTENT' });
  expect(result).toBeNull();
});

Agent 集成测试

typescript 
import { supportAgent } from '@/agents/support';
import { MockLanguageModelV2 } from 'ai/test';

test('客户问订单状态会调 lookupOrder', async () => {
  // 替换真实 model 为 mock
  const mockModel = new MockLanguageModelV2({
    doStream: async () => ({
      stream: simulateReadableStream({
        chunks: [
          {
            type: 'tool-call',
            toolCallId: 'call_1',
            toolName: 'lookupOrder',
            input: { orderId: 'ORD-42' },
          },
          // 模拟 tool 调用后 LLM 总结
          { type: 'text-delta', textDelta: 'Your order is shipped.' },
          { type: 'finish', finishReason: 'stop' },
        ],
      }),
    }),
  });

  // Agent 类支持 model override (在 generate 调用时)
  const result = await supportAgent.generate({
    model: mockModel,
    prompt: 'Where is order ORD-42?',
  });

  expect(result.toolCalls).toContainEqual(
    expect.objectContaining({ toolName: 'lookupOrder' })
  );
});

Runtime 边界对照

Runtime Stream 支持 Timeout Subprocess（MCP stdio） 2026 适用
Vercel Edge ✅ 25-300s ❌ 简单 chat / generative UI
Vercel Node Function ✅ 15min（Pro） ✅ 复杂 agent + MCP subprocess
Cloudflare Workers ✅ 30s-5min ❌ 边缘流式 + HTTP MCP
Bun.serve ✅ 无限 ✅ 自建 server / 高性能 chat
Node Express / Hono ✅ 无限 ✅ 传统 backend
Cron / Worker queue — 无限 ✅ 批处理 / 长 agent task

OTel Observability

v6 起 AI SDK 内置 OpenTelemetry instrumentation：

typescript 
// 启用 OTel
import { streamText } from 'ai';
import { openai } from '@ai-sdk/openai';

await streamText({
  model: openai('gpt-4o'),
  prompt: 'Hello',
  experimental_telemetry: {
    isEnabled: true,
    functionId: 'chat-handler',          // 标识这次调用
    metadata: { userId: 'user-123' },     // 附加 metadata
  },
});

// SDK 自动导出 spans:
// - ai.streamText (主 span)
// - ai.streamText.doStream (调 model)
// - ai.toolCall (每个 tool 调用)
// - ai.streamText.doStream.text-delta (流式事件)

// 接 OTel collector → Datadog / Honeycomb / Langfuse / 自建

Prompt Caching（Anthropic）

系统 prompt 长（含大量 context / few-shot 例子 / 工具描述）时，每次请求重发整个 prompt 浪费 token。Anthropic Prompt Caching 让长 prompt 在 server 端 cache，重复请求只付 25% 价格（cache write 25% 高，cache read 90% 低）：

typescript 
import { anthropic } from '@ai-sdk/anthropic';

const result = await streamText({
  model: anthropic('claude-sonnet-4-6'),
  system: longSystemPrompt,  // 几千 token 的 system prompt
  messages: [...],

  // 启用 prompt caching
  providerOptions: {
    anthropic: {
      cacheControl: { type: 'ephemeral' },  // cache 5 分钟
    },
  },
});

// 重复请求时只付 cache read 价（10% 价格）

2026 状态：

  • Anthropic 全 model 支持 prompt caching
  • OpenAI 也有类似功能（automatic prompt caching for context > 1k tokens）
  • Google Gemini context caching API
  • AI Gateway 自动启用 prompt caching（不用应用层显式配置）

AI Gateway 工程价值

AI Gateway（v6 起）是 Vercel 提供的统一 LLM 访问层：

  • 单一 API key：不用为每个 provider 单独申请
  • Zero markup pricing：付 provider token 原价（不加价）
  • $5/月免费额度：Vercel 团队赠送
  • 统一 observability：所有 LLM 调用走 Gateway，一站监控
  • Budget controls：设月度上限避免账单意外
  • Caching：Gateway 层 cache（相同输入 cache 化）
  • Fallback：一个 provider 挂了自动切另一个
  • Auto prompt caching：长 prompt 自动启用 provider 各自的 prompt caching

不用 Gateway 的情况：

  • 合规要求 LLM 调用必须直连 provider（不经第三方）
  • 需要 provider-specific feature 而 Gateway 不暴露
  • 已有完整 observability / billing 基础设施，Gateway 只是冗余

03 应用场景

场景 A · 短 chat → Edge runtime

用户提问 → LLM 流式回答（< 30s 完成）。Vercel Edge Function 默认 25s timeout 通常够用。低延迟（edge POP）+ 自动 scale。

场景 B · 复杂 research agent → Node Function

研究 agent 跑几分钟（多步 search / 总结）→ Edge 25s timeout 不够。Vercel Node Function（Pro 15min）或自建 Express / Hono。配 OTel 监控 step 进度。

场景 C · 批处理 agent → Cron / Worker queue

每天处理 1000 个 ticket → 不是用户实时等的请求 → 进 queue（Inngest / Trigger.dev / 自建 BullMQ）→ worker 慢慢处理 + 失败重试 + 进度持久化。不要在 HTTP request 跑长任务。

场景 D · 多 provider 项目 → AI Gateway

用户可选 GPT / Claude / Gemini → 不用各自 SDK + 各自 API key。Gateway 一份代码 + 一个 dashboard + budget 控制 + auto fallback。2026 多 provider 项目几乎默认 Gateway。

场景 E · 长 system prompt → Prompt caching

客服 agent 的 system prompt 含完整产品 context + tool 描述 + 历史 few-shot 例子（5-20k token）。每次请求重发整个 prompt → token 账单贵 5x。开 prompt caching → cache read 价格 10%，账单减少 80%+。

场景 F · 调试用 DevTools 浏览器扩展

useChat 行为奇怪（tool 不触发 / state 不对 / 流式卡）→ 装 AI SDK DevTools 看 internal state + message parts + tool calls timeline。比 console.log 高效十倍。

04 典型代码：生产 checklist 实战

测试：MockLanguageModelV2 完整例子

typescript 
import { describe, test, expect } from 'vitest';
import { MockLanguageModelV2, simulateReadableStream } from 'ai/test';
import { streamText } from 'ai';

describe('chat handler', () => {
  test('简单文本响应', async () => {
    const mockModel = new MockLanguageModelV2({
      doStream: async () => ({
        stream: simulateReadableStream({
          chunks: [
            { type: 'text-delta', textDelta: 'Hello, ' },
            { type: 'text-delta', textDelta: 'world!' },
            { type: 'finish', finishReason: 'stop', usage: { promptTokens: 10, completionTokens: 5 } },
          ],
        }),
      }),
    });

    const result = streamText({ model: mockModel, prompt: 'test' });
    let text = '';
    for await (const chunk of result.textStream) {
      text += chunk;
    }
    expect(text).toBe('Hello, world!');
  });

  test('tool 调用流程', async () => {
    const mockModel = new MockLanguageModelV2({
      doStream: async ({ inputFormat }) => {
        // 第一次调返回 tool call
        // (生产代码可基于 messages 长度判断是第几次调用)
        return {
          stream: simulateReadableStream({
            chunks: [
              { type: 'tool-call', toolCallId: 'call_1', toolName: 'getWeather', input: { city: 'NYC' } },
              { type: 'finish', finishReason: 'tool-calls' },
            ],
          }),
        };
      },
    });

    const result = streamText({
      model: mockModel,
      tools: {
        getWeather: tool({
          description: '...',
          parameters: z.object({ city: z.string() }),
          execute: async ({ city }) => ({ city, temp: 72 }),
        }),
      },
      prompt: 'NYC weather',
      maxSteps: 1,
    });

    const toolCalls = await result.toolCalls;
    expect(toolCalls[0].toolName).toBe('getWeather');
  });
});

Edge runtime + 优雅 timeout

typescript 
// app/api/chat/route.ts
export const runtime = 'edge';
export const maxDuration = 60;  // 1 分钟（pro 计划可设更长）

import { streamText, convertToModelMessages } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function POST(req: Request) {
  const { messages } = await req.json();

  // 创建 abort controller 处理 timeout
  const abortController = new AbortController();
  setTimeout(() => abortController.abort(), 55_000);  // 比 maxDuration 早 5s

  const result = streamText({
    model: openai('gpt-4o'),
    messages: convertToModelMessages(messages),
    abortSignal: abortController.signal,
    onError: ({ error }) => {
      if (error.name === 'AbortError') {
        console.log('Stream aborted due to timeout');
      }
    },
  });

  return result.toUIMessageStreamResponse();
}

Anthropic Prompt Caching

typescript 
import { streamText } from 'ai';
import { anthropic } from '@ai-sdk/anthropic';

// 长 system prompt（5k+ token）
const longSystemPrompt = `
You are a customer support agent for ACME Corp.

Company info: ${COMPANY_INFO_5000_TOKENS}
Product catalog: ${PRODUCT_CATALOG_3000_TOKENS}
Common issues and resolutions: ${COMMON_ISSUES_4000_TOKENS}

Tone: professional, helpful, concise.
`;

const result = streamText({
  model: anthropic('claude-sonnet-4-6'),
  system: longSystemPrompt,
  messages: convertToModelMessages(userMessages),

  providerOptions: {
    anthropic: {
      cacheControl: { type: 'ephemeral' },  // 5 分钟 cache TTL
    },
  },
});

// 第一次请求: 全量 token 价 + cache write fee（25% 加价）
// 后续 5 分钟内重复请求: cache read 价（10% 价格）
// 如果 system prompt 是 12k token，长期下来 token 账单减少 80%+

cost 控制：模型分级

typescript 
// 模型分级 cost 控制
async function smartChat(prompt: string) {
  // 1. 先用便宜 model 分类（路由）
  const { object: routing } = await generateObject({
    model: gateway('anthropic/claude-haiku-4-5'),  // 便宜
    schema: z.object({
      complexity: z.enum(['simple', 'medium', 'complex']),
      requiresReasoning: z.boolean(),
    }),
    prompt: `Classify this query: ${prompt}`,
  });

  // 2. 根据分类选 model
  let model;
  if (routing.complexity === 'simple') {
    model = gateway('anthropic/claude-haiku-4-5');  // $0.80/M tokens
  } else if (routing.requiresReasoning) {
    model = gateway('openai/o1');  // $15/M tokens 但是推理强
  } else {
    model = gateway('anthropic/claude-sonnet-4-6');  // 中档 $3/M
  }

  return streamText({ model, prompt });
}

// cost 实测优化:
// - 60% 用户查询是 simple → Haiku ($0.80/M)
// - 30% 是 medium → Sonnet ($3/M)
// - 10% 复杂推理 → o1 ($15/M)
// vs 全部 GPT-4o ($5/M):
// 平均 cost 降低 ~50%

05 常见陷阱

陷阱 1 · 在 Edge runtime 跑长 agent

Edge 25s 默认 timeout，长 agent 任务（research agent / 多步骤推理）跑不完。修复：(a) 用 Node Function（Pro 15min）；(b) 拆任务进 queue；(c) 调 maxDuration 但仍有上限。

陷阱 2 · mock useChat 内部

测试 drift + SDK 升级测试假绿。修复：mock 边界（MockLanguageModelV2 替 model 层）。tool execute 直接 unit test。

陷阱 3 · 没用 prompt caching 长 system prompt

每次请求重发 5-20k token 的 system prompt → token 账单膨胀 5-10x。修复：开 prompt caching（Anthropic / OpenAI / Google 都支持）；或用 AI Gateway 自动启用。

陷阱 4 · 没监控 token usage

"我用 Vercel AI SDK 一切顺利" 直到月底 $50000 账单。修复：(a) 每次响应记录 usage.totalTokens；(b) 用 AI Gateway dashboard 看实时 spend；(c) 设 budget alert（超 X $/天 通知）。

陷阱 5 · 都用最贵 model 跑所有任务

客服路由 / 内容分类 / 简单总结 都用 GPT-4 / Claude Opus → 浪费。修复：模型分级（Haiku 跑 routing / Sonnet 跑核心 / Opus / o1 仅复杂推理）。可以省 50-80% cost。

陷阱 6 · OTel 配置但不接 collector

v6 内置 OTel 但你没接 collector → spans 写到本地 stdout 没人看。修复：接 Datadog / Honeycomb / Langfuse / Vercel Observability 任一个。OTel 就是为了"接了能用"。

陷阱 7 · 在 Cloudflare Workers 跑 stdio MCP

Workers 不允许 subprocess。修复：HTTP MCP server（远程跑）；或换 Node runtime。

陷阱 8 · AI Gateway $5 免费额度后没 budget

月底惊喜账单。修复：dashboard 设月度 budget 上限；超就 hard stop（拒绝新请求）+ 通知 ops。
