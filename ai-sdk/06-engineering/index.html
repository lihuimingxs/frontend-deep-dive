P6 · 工程化与生产

把 AI SDK 上生产之前要解决：怎么测试 LLM 流式响应、Edge / Node / Bun runtime 在长 agent 任务上的限制、监控（OTel / DevTools）、caching（不每次都打 LLM）、cost 控制、AI Gateway 是不是必须。这一阶段一章覆盖 6 个工程话题。

章节数 1 主要来源 ai-sdk.dev/docs/troubleshooting + Vercel Edge runtime 文档 + Anthropic Prompt Caching docs + Vercel AI Gateway 文档 + OpenTelemetry SDK 文档 + 测试主题 P6 MSW 状态 ✅ 全部完成

本阶段覆盖

  • 测试 · MockLanguageModelV2（v5+ 内置）+ MSW 网络层 mock + tool unit test

  • Edge runtime 边界 · 25s 默认 timeout（可调）· 某些 Node API 不支持
  • Node runtime · 完整支持 · 长 agent 任务首选
  • Bun（链回 Bun 主题）· 完整 Web Streams · Bun.serve 流式
  • Cloudflare Workers · Streams 完整 · CPU time limits
  • OTel observability（v6 内置）· 默认导出 spans · Datadog / Honeycomb / Langfuse 接入
  • DevTools（v6 浏览器扩展）· 看 useChat state · agent step / tool execution timeline
  • Anthropic Prompt Caching · 长 system prompt cache · 重复请求只付 cache 价
  • AI Gateway · 统一 cache + observability + budget + fallback
  • cost 控制 · usage 监控 · 模型分级（Haiku 跑 routing / Sonnet 跑核心）
