Generative UI: streamUI + RSC + 与 React 19 use API 协作

Generative UI 是 Vercel 提的概念：LLM 不只生成文本，生成 React 组件树。背后机制：LLM 输出"该渲染什么 UI"决策 → SDK 在 server-side 直接渲染 React 组件 → 客户端通过 RSC payload 边收边渲染。这一章讲 streamUI / createStreamableUI / createStreamableValue + 与 useChat 共存策略 + 与 React 19 use API 协作。

阅读时长 ~22 分钟 前置 读过 P3.1（useChat） + 看过 React Server Components 至少一次 主要来源 ai-sdk.dev/docs/ai-sdk-rsc + Vercel Generative UI 设计文章 + React 19 use API RFC + Next.js RSC 文档（链回 Next.js / React 主题）

01 速览

  1. Generative UI 核心思路：LLM 输出"该渲染什么 UI" 决策（不是 HTML，是 component invocation 决策）→ SDK 根据 LLM 决策在 server-side 直接渲染 React 组件（RSC）→ 客户端通过 RSC payload 边收边渲染。
  2. streamUI（ai/rsc）：核心 API。在 Server Action / Route Handler 调用，返回 { value: ReactNode }。返回值可在 RSC 中直接渲染。
  3. createStreamableUI / createStreamableValue：底层 primitives。streamUI 是基于它们的高层封装。复杂场景可以直接用 primitives。
  4. 关键约束：streamUI 必须在 Server Action 或 Route Handler 调用（不能在 Client Component），它依赖 RSC server-side 渲染能力。
  5. tools 在 Generative UI 里的角色：每个 tool 不只 execute 数据，还可以 generate（async generator 生成 React 元素）。tool 决定渲染什么组件。
  6. 与 useChat 共存：useChat 管 chat history（client state）；streamUI 渲染 inline tool result UI（server-rendered 组件嵌进 message）。两者不冲突，2026 推荐组合。
  7. 与 React 19 use API 协作：streamUI 返回的 React Node 在 RSC 中直接渲染；同时也可以用 createStreamableValue 创建 promise，让 client component 用 use API 消费（流式）。
  8. 2026 状态：ai/rsc 仍是较新且演进中的 API。Vercel 官方推荐："简单 chat 用 useChat；需要丰富 UI 渲染（卡片 / 表单 / 嵌入式可视化）用 streamUI"。
  9. 典型场景：(a) 用户问"我下周怎么穿"→ LLM 调 getWeather + 渲染 OutfitCard；(b) "帮我建订餐表单"→ LLM 生成 form schema + 渲染 Form 组件；(c) "比较这两个产品"→ LLM 渲染 ComparisonTable 组件。

02 底层逻辑

Generative UI 设计意图

传统 chat UI：LLM 输出文本 → 前端用 markdown 渲染。问题：

  • 无法渲染交互组件（按钮 / 表单 / 滑块）
  • 无法嵌入数据可视化（图表 / 地图 / 卡片）
  • LLM 必须用文字描述"建议你点这里" —— 用户体验差

Generative UI 解法：让 LLM 直接决定渲染什么 React 组件。LLM 不输出 HTML（不安全），输出"调用 ComponentX 组件 with props Y"决策。Server 端按决策渲染组件，client 流式收。

streamUI 核心 API

tsx 
// app/actions.tsx (Server Action)
'use server';
import { streamUI } from 'ai/rsc';
import { openai } from '@ai-sdk/openai';
import { z } from 'zod';

export async function chat(message: string) {
  const result = await streamUI({
    model: openai('gpt-4o'),
    prompt: message,

    // text: 当 LLM 输出文本时怎么渲染
    text: ({ content }) => <div>{content}</div>,

    // tools: 每个 tool 决定渲染什么组件
    tools: {
      showWeather: {
        description: 'Show current weather for a city',
        parameters: z.object({ city: z.string() }),
        generate: async function* ({ city }) {
          // 1. 立即 yield Loading（用户看到加载状态）
          yield <Loading message={`Fetching weather for ${city}...`} />;

          // 2. 实际调外部 API
          const weather = await getWeather(city);

          // 3. return 最终组件
          return <WeatherCard data={weather} />;
        },
      },

      showProductCard: {
        description: 'Show a product card',
        parameters: z.object({ productId: z.string() }),
        generate: async function* ({ productId }) {
          yield <ProductSkeleton />;
          const product = await db.product.findUnique({ where: { id: productId } });
          return <ProductCard product={product} />;
        },
      },
    },
  });

  return result.value;  // 返回 ReactNode 给客户端
}

generate 是 async generator

tool 的 generate 是 async generator（不是普通函数）。这让 tool 可以：

  • 多次 yield 渲染中间状态（loading → progress → 最终）
  • return 最终组件
  • 异步等待（fetchData / 调外部 API）

每次 yield 客户端立即看到新组件。这是 Generative UI 的"流式"含义。

Client Component 调 Server Action 拿 ReactNode

tsx 
// app/page.tsx
'use client';
import { useState } from 'react';
import { chat } from './actions';

export default function ChatPage() {
  const [conversation, setConversation] = useState<React.ReactNode[]>([]);
  const [input, setInput] = useState('');

  async function sendMessage() {
    // 调 Server Action 拿 ReactNode
    const ui = await chat(input);

    setConversation((prev) => [
      ...prev,
      <UserMessage key={prev.length} text={input} />,
      ui,  // 直接塞 ReactNode 到 state
    ]);
    setInput('');
  }

  return (
    <div>
      {conversation.map((node, i) => <div key={i}>{node}</div>)}
      <input value={input} onChange={(e) => setInput(e.target.value)} />
      <button onClick={sendMessage}>Send</button>
    </div>
  );
}

关键：Client Component 把 Server Action 返回的 ReactNode 直接塞进 state。React 知道怎么渲染（包括等待流式 yield 的中间状态）。

底层：createStreamableUI / createStreamableValue

streamUI 是高层封装。底层 primitives 是：

tsx 
'use server';
import { createStreamableUI, createStreamableValue } from 'ai/rsc';

export async function customServerAction(input: string) {
  // streamable UI - 创建可流式更新的 ReactNode
  const ui = createStreamableUI(<Loading />);

  // streamable value - 创建可流式更新的纯数据
  const value = createStreamableValue('');

  (async () => {
    // 异步更新 UI
    await someWork();
    ui.update(<Step1 />);

    await moreWork();
    ui.update(<Step2 />);

    // 同时流式更新 value
    for await (const chunk of streamFromLLM(input)) {
      value.append(chunk);
    }

    ui.done(<FinalUI />);
    value.done();
  })();

  // 立即返回（不阻塞）
  return { ui: ui.value, value: value.value };
}

// 客户端
'use client';
import { useState } from 'react';
import { readStreamableValue } from 'ai/rsc';

function Component() {
  const [text, setText] = useState('');
  const [ui, setUI] = useState<React.ReactNode>(null);

  async function handle() {
    const { ui: streamableUI, value: streamableValue } = await customServerAction('input');

    setUI(streamableUI);

    // 用 readStreamableValue 边读边更新
    for await (const chunk of readStreamableValue(streamableValue)) {
      setText(chunk);
    }
  }

  return <div>{ui} {text}</div>;
}

这套 primitives 的优势：

  • 更细粒度控制：streamUI 是"按 LLM tool calls 决定渲染"模式；primitives 让你完全自定义流式逻辑
  • 多个独立流：可以同时跑 ui / value / progress 三个独立 streamable
  • 非 LLM 场景也能用：流式数据库查询 / 流式文件处理 / 流式计算

与 React 19 use API 协作

React 19 引入 use(promise) API：消费一个 promise，pending 时 throw promise（被 Suspense 接住）。streamUI / createStreamableValue 都和 use API 配合好：

tsx 
// Server Component 创建 promise
async function ServerPage() {
  const dataPromise = streamFromLLM(...);  // 不 await
  return <ClientChild promise={dataPromise} />;
}

// Client Component 用 use 消费
'use client';
import { use } from 'react';

function ClientChild({ promise }: { promise: Promise<Data> }) {
  const data = use(promise);  // pending 时 throw, 被外层 Suspense 接住
  return <div>{data.text}</div>;
}

// 父组件
<Suspense fallback={<Loading />}>
  <ServerPage />
</Suspense>

这套是RSC 时代流式数据消费的现代姿势。详见 React 主题 P6.1。streamUI 是高层抽象（专为 LLM Generative UI），use API + createStreamableValue 是底层 primitives（任意流式数据）。

useChat vs streamUI 共存

2026 推荐组合：useChat 管 chat history + streamUI 渲染 inline tool result UI。

text 
useChat:
  - 维护 client-side chat history (messages: UIMessage[])
  - 管 input / submit / streaming state
  - 持久化 (localStorage / DB)
  - 跨组件共享 messages

streamUI:
  - LLM 决定渲染什么组件
  - 在 Server Action 内部跑
  - 返回 ReactNode 给 useChat 的 message part 渲染

组合姿势:
  1. 用户输入 → useChat handleSubmit
  2. POST 到 Route Handler
  3. Route Handler 内部调 streamUI
  4. 返回的 UI 通过 v5 SSE 协议传到前端
  5. useChat 把 UI 塞到 message.parts (作为 'data-ui' part 类型或类似)
  6. 前端渲染 message.parts 时把 UI part 渲染出来

注意：2026 年 useChat 和 streamUI 的官方"无缝集成"仍在演进。多数项目仍是选其一：

  • 纯 chat（文字 + 简单 tool result）→ useChat
  • 丰富 UI 渲染（多种卡片 / form / 可视化）→ streamUI
  • 极复杂场景 → 两者一起用（自己处理桥接）

03 应用场景

场景 A · 旅行助理（混合渲染）

用户："我下周去东京"→ LLM 调 getWeather 渲染 WeatherCard + 调 getFlights 渲染 FlightList + 调 suggestActivities 渲染 ActivityCards。streamUI 让 LLM 决定哪些卡片渲染，每个卡片是 server-rendered React 组件。

场景 B · 数据分析助手

用户："本月销售情况" → LLM 查 DB + 生成 Chart 组件 + 解释文字。Chart 是真实 React 组件（recharts / chart.js 等），server-rendered + 流式。

场景 C · 表单生成器

用户："建一个产品反馈 form" → LLM 决定字段 → 生成 Form 组件含各种 input。比 streamObject 生成 form schema + 前端渲染更直接。

场景 D · 比较 / 推荐

用户："iPhone 15 vs 16 Pro" → LLM 渲染 ComparisonTable + RecommendationCard。表格是结构化 React 组件，比 markdown table 美观且支持交互。

场景 E · 简单 chat → 仍用 useChat 不需要 streamUI

纯文字 chat（客服 / 翻译 / 总结）→ useChat 完全够用。不要为了"现代"硬上 streamUI，它有额外复杂度（必须 RSC + Server Action）。

// app/components/Loading.tsx
export function Loading({ message }: { message: string }) {
  return (
    <div className="loading">
      <span className="spinner">⏳</span> {message}
    </div>
  );
}

// app/components/WeatherCard.tsx
export function WeatherCard({ data }: { data: WeatherData }) {
  return (
    <div className="weather-card">
      <h3>{data.city}</h3>
      <p>{data.temperature}°C, {data.condition}</p>
      <p>Wind: {data.wind} km/h</p>
    </div>
  );
}

// app/components/OutfitCard.tsx
export function OutfitCard({ items, reasoning }: { items: string[]; reasoning: string }) {
  return (
    <div className="outfit-card">
      <h3>Outfit Suggestion</h3>
      <ul>
        {items.map((item, i) => <li key={i}>{item}</li>)}
      </ul>
      <p className="reasoning">{reasoning}</p>
    </div>
  );
}

// app/actions.tsx (Server Action)
'use server';
import { streamUI } from 'ai/rsc';
import { openai } from '@ai-sdk/openai';
import { z } from 'zod';
import { Loading } from './components/Loading';
import { WeatherCard } from './components/WeatherCard';
import { OutfitCard } from './components/OutfitCard';

export async function askAssistant(input: string) {
  const result = await streamUI({
    model: openai('gpt-4o'),
    prompt: input,
    text: ({ content }) => <p className="message">{content}</p>,
    tools: {
      getWeather: {
        description: 'Get current weather for a city',
        parameters: z.object({ city: z.string() }),
        generate: async function* ({ city }) {
          yield <Loading message={`Fetching weather for ${city}...`} />;
          const data = await fetchWeather(city);
          return <WeatherCard data={data} />;
        },
      },
      suggestOutfit: {
        description: 'Suggest outfit based on weather',
        parameters: z.object({
          weather: z.string(),
          temperature: z.number(),
        }),
        generate: async function* ({ weather, temperature }) {
          yield <Loading message="Thinking about outfit..." />;
          const items = await getOutfitItems(weather, temperature);
          const reasoning = await explainChoice(weather, items);
          return <OutfitCard items={items} reasoning={reasoning} />;
        },
      },
    },
  });
  return result.value;
}

// app/page.tsx
'use client';
import { useState } from 'react';
import { askAssistant } from './actions';

export default function HomePage() {
  const [conversation, setConversation] = useState<React.ReactNode[]>([]);
  const [input, setInput] = useState('');

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    const userInput = input;
    setInput('');

    setConversation((prev) => [
      ...prev,
      <p key={`u-${prev.length}`} className="user-msg">You: {userInput}</p>,
    ]);

    const ui = await askAssistant(userInput);
    setConversation((prev) => [...prev, ui]);
  }

  return (
    <div className="chat-container">
      {conversation.map((node, i) => (
        <div key={i} className="message-row">
          {node}
        </div>
      ))}
      <form onSubmit={handleSubmit}>
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Ask about weather and outfits..."
        />
        <button type="submit">Send</button>
      </form>
    </div>
  );
}

用 createStreamableValue 实现自定义流式

tsx 
'use server';
import { createStreamableValue } from 'ai/rsc';
import { streamText } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function generateText(prompt: string) {
  const stream = createStreamableValue('');

  (async () => {
    const { textStream } = streamText({
      model: openai('gpt-4o'),
      prompt,
    });
    for await (const delta of textStream) {
      stream.append(delta);
    }
    stream.done();
  })();

  return stream.value;
}

// 客户端用 readStreamableValue 消费
'use client';
import { readStreamableValue } from 'ai/rsc';

const streamValue = await generateText('Write a poem');
let text = '';
for await (const chunk of readStreamableValue(streamValue)) {
  text += chunk;
  setText(text);
}

错误处理：Suspense + Error Boundary

tsx 
'use client';
import { Suspense } from 'react';
import { ErrorBoundary } from 'react-error-boundary';

<ErrorBoundary fallback={<ErrorMessage />}>
  <Suspense fallback={<Loading />}>
    {streamUIResult /* ReactNode from server */}
  </Suspense>
</ErrorBoundary>

05 常见陷阱

陷阱 1 · 在 Client Component 调 streamUI

streamUI 必须在 Server Action / Route Handler 调用 —— 它依赖 RSC server-side 渲染能力。在 client 调会报错。修复：把 streamUI 调用移到 server 文件（'use server' 标记），client 通过 server action 触发。

陷阱 2 · tool generate 不是 async generator

generate 必须是 async generator（用 function* + async）。如果你写普通 async function 只能 return 一次 → 失去"流式 yield 中间状态"能力。修复：generate: async function* () { yield <Loading />; ... return <Final />; }。

陷阱 3 · streamUI 返回 React Node 但 Client Component 不知怎么用

需要把 ReactNode 塞进 state（setState(node) 或 push 到数组）。React 自动处理渲染（包括等待流式 yield）。修复：理解 ReactNode 是"一等公民"，Server Action 返回它就像返回字符串一样自然。

陷阱 4 · streamUI 当 useChat 用

streamUI 不维护 message history（每次调都是独立的 prompt）。如果你想要多轮对话，要么自己管 history（每次拼 prompt），要么用 useChat。修复：根据需求选 —— 多轮 chat 用 useChat；单次"输入 → 渲染卡片"用 streamUI。

陷阱 5 · 在 streamUI 的 tool generate 里拿不到外部 closure

generate 是 async generator，但仍能访问 Server Action 的 closure（user / req / db）。如果你拿不到，检查是否在正确的作用域。修复：generate 直接定义在 streamUI 调用里就能访问外部变量。

陷阱 6 · 期待 streamUI 自动处理 chat persistence

streamUI 不知道"对话历史"概念。如果用户刷新页面，streamUI 之前生成的 UI 都丢了。修复：(a) 用 useChat 管 messages 历史 + tool 渲染走 useChat 的 part 系统；(b) 自己存 ReactNode metadata 到 DB，刷新时重新调 server action 还原。

陷阱 7 · 嵌套 client component 在 streamUI 返回值里没标记

streamUI 返回的 ReactNode 是 server-rendered。如果里面嵌入 Client Component，必须 import 时已是 'use client' 标记的组件。修复：交互组件（onClick 等）必须 'use client'；纯展示组件（仅 props 渲染）可以是 server component。

陷阱 8 · Suspense / Error Boundary 包裹位置错

Suspense / Error Boundary 必须在 Client Component 包 streamUI 返回的 UI（不是在 Server Action 内）。修复：在 client 渲染 streamUI 结果的位置用 Suspense + ErrorBoundary。
