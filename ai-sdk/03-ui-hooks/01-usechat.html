useChat / useCompletion / useObject 全谱（v5+ UIMessage 心智）

useChat 是 Vercel AI SDK 最重要的 hook，2026 年 React/Next.js 项目集成 LLM 几乎都从它开始。这一章把 useChat / useCompletion / useObject 三个 UI hook 在 v5+ UIMessage 心智下的完整使用讲清：API 全谱、parts 类型、自定义 data part、与 Server Action 配合、多框架（Vue / Svelte / Angular）等价物。

阅读时长 ~22 分钟 前置 读过 P2（架构 + Stream Protocol） 主要来源 ai-sdk.dev/docs/ai-sdk-ui + ai-sdk.dev/docs/ai-sdk-ui/use-chat + Vercel useChat 实战博文 + Next.js Server Action 文档

01 速览

  1. 三个核心 UI hook：(a) useChat 多轮对话；(b) useCompletion 单次补全（不是 chat）；(c) useObject 流式生成结构化对象（按 Zod schema）。
  2. useChat({ api: '/api/chat' }) 返回 { messages, input, setInput, status, sendMessage, regenerate, stop, error }。messages 是 UIMessage[]（v5+ 心智）。
  3. UIMessage 结构：{ id, role, parts: Part[], metadata? }。parts 是各种 type-specific part（text / tool-{name} / data-{custom} / source / step-start / 等）。
  4. parts 类型：text（文字）/ tool-{name}（v5+ type-specific tool 调用）/ data-{custom}（应用自定义数据）/ source（RAG citation）/ step-start | step-end（multi-step agent）/ file（attachment）等。
  5. useCompletion：单次输入 → 单次流式输出，不是多轮对话。{ completion, complete, input, setInput, isLoading, stop }。适合"提示词改写"/"翻译"等单次操作。
  6. useObject：流式生成结构化对象。前后端都用同一个 Zod schema。{ object, submit, isLoading, error, stop }。边收边给逐渐完整的 object，UI 边收边渲染。适合"生成 form / 表格 / 卡片"。
  7. convertToModelMessages：v5+ 关键边界转换函数。在 server side 把 UIMessage[] 转成 ModelMessage[]（发给 LLM 的格式）。必须用，否则 type 不兼容。
  8. 多框架等价物：v5 起 React / Vue / Svelte / Angular feature parity。ai/react / ai/vue / ai/svelte / ai/angular。
  9. 2026 实际项目分布：useChat（~85% 项目）/ useCompletion（~15%）/ useObject（~30%）/ useAssistant（OpenAI Assistants API 专用，~5% 收缩中）。

02 底层逻辑

useChat 完整 API

typescript 
import { useChat } from 'ai/react';

const {
  messages,         // UIMessage[]
  input,            // string (用户输入)
  setInput,         // (value: string) => void
  handleInputChange,// React event handler 简写
  handleSubmit,     // form submit handler 简写
  sendMessage,      // (text: string) => void  显式发消息
  status,           // 'submitted' | 'streaming' | 'ready' | 'error'
  error,            // Error | undefined
  regenerate,       // () => void  重新生成上一条 assistant 消息
  stop,             // () => void  中止当前流
  setMessages,      // (messages: UIMessage[]) => void
} = useChat({
  api: '/api/chat',          // 后端 endpoint
  initialMessages: [...],    // 初始 messages
  headers: { ... },          // 自定义 headers
  body: { customField: ... },// 额外 request body
  onFinish: (message) => { ... },
  onError: (error) => { ... },
});

UIMessage 完整结构

typescript 
type UIMessage = {
  id: string;
  role: 'user' | 'assistant' | 'system';
  parts: Array<
    // 文字
    | { type: 'text'; text: string }

    // tool 调用 (v5+ type-specific)
    | {
        type: `tool-${string}`;       // 如 'tool-getWeather'
        toolCallId: string;
        state: 'partial-call' | 'call' | 'result' | 'error';
        input?: any;
        output?: any;
        errorText?: string;
      }

    // 应用自定义 data part
    | {
        type: `data-${string}`;        // 如 'data-progress'
        id?: string;
        data: any;
      }

    // RAG citation
    | { type: 'source'; source: { id: string; url?: string; title?: string } }

    // multi-step agent (v6+)
    | { type: 'step-start' }
    | { type: 'step-end' }

    // attachment
    | {
        type: 'file';
        url: string;
        mediaType: string;
      }

    // reasoning trace (o1 / Claude extended thinking)
    | { type: 'reasoning'; reasoning: string }
  >;
  metadata?: any;  // 不发给 LLM
};

handleSubmit 流式响应过程

text 
用户输入 + 点 submit
    ↓
useChat.handleSubmit (form 默认行为)
    ↓
SDK 把 input 加进 messages (新增 UIMessage role: 'user')
    ↓
SDK 序列化整个 messages 数组发到 /api/chat (POST)
    ↓
─────────────── 后端 ───────────────
Route Handler 收到 messages
    ↓
convertToModelMessages(messages) → ModelMessage[]
    ↓
streamText({ model, messages: modelMessages, tools? })
    ↓
result.toUIMessageStreamResponse() 返回 v5 SSE 协议响应
─────────────── 后端 ───────────────
    ↓
SDK 边收 SSE events 边累加进 messages state
    ↓
触发 useChat 订阅组件 re-render
    ↓
status: 'submitted' → 'streaming' → 'ready'

useCompletion API

useCompletion 是单次输入 → 单次流式输出，不维护对话历史：

typescript 
import { useCompletion } from 'ai/react';

const {
  completion,      // string (累加的输出)
  complete,        // (prompt: string) => void
  input,
  setInput,
  handleInputChange,
  handleSubmit,
  isLoading,
  stop,
  error,
} = useCompletion({
  api: '/api/completion',
});

// 用法:
<form onSubmit={handleSubmit}>
  <input value={input} onChange={handleInputChange} />
  <button type="submit">翻译</button>
</form>
<div>{completion}</div>

useCompletion 与 useChat 的区别：

  • useCompletion 不维护 messages 历史
  • useCompletion 不支持 tool calling（仅文本）
  • useCompletion 的 server endpoint 用 streamText + toTextStreamResponse()（注意是 toTextStream 不是 toUIMessageStream）

useObject API

useObject 流式生成结构化对象：

typescript 
import { useObject } from 'ai/react';
import { z } from 'zod';

// 同一个 schema 前后端共用
const orderSchema = z.object({
  customer: z.string(),
  items: z.array(z.object({
    name: z.string(),
    quantity: z.number(),
    price: z.number(),
  })),
  total: z.number(),
});

const {
  object,        // Partial<OrderType> 边收边变化（中间状态可能不完整）
  submit,        // (input: any) => void
  isLoading,
  error,
  stop,
} = useObject({
  api: '/api/generate-order',
  schema: orderSchema,
});

// 用法
<button onClick={() => submit({ description: 'Apple x2, Banana x3' })}>
  Generate Order
</button>
{object?.customer && <p>Customer: {object.customer}</p>}
{object?.items?.map(...)} {/* 边收边渲染 */}
{object?.total && <p>Total: ${object.total}</p>}

关键：partial object 不一定符合 schema（中间字符串可能截断），但最终 object 一定符合。前端渲染时要 optional chaining。

自定义 data part

v5+ 重要新能力：在流式响应里推自定义数据，前端按 part type 接收：

typescript 
// 后端 (Route Handler)
import { streamText, createUIMessageStreamResponse } from 'ai';

export async function POST(req: Request) {
  const { messages } = await req.json();

  return createUIMessageStreamResponse({
    execute: async (writer) => {
      // 推自定义 data part (不进 LLM context)
      writer.write({
        type: 'data-progress',
        data: { step: 1, total: 5, message: 'Loading data...' },
      });

      const data = await fetchExpensiveData();

      writer.write({
        type: 'data-progress',
        data: { step: 2, total: 5, message: 'Calling LLM...' },
      });

      // 调 LLM 流式
      const result = streamText({
        model: openai('gpt-4o'),
        messages: convertToModelMessages(messages),
      });

      // 把 streamText 的 stream 也写进 writer
      writer.merge(result.toUIMessageStream());

      writer.write({
        type: 'data-progress',
        data: { step: 5, total: 5, message: 'Done' },
      });
    },
  });
}

// 前端
const { messages } = useChat({ api: '/api/chat' });
messages.map((m) => m.parts.map((part) => {
  if (part.type === 'data-progress') {
    return <ProgressBar step={part.data.step} total={part.data.total} />;
  }
  // 其他 part type
}));

多框架等价物

v5 起 Vue / Svelte / Angular feature parity。同一份后端代码，不同前端 hook：

typescript 
// React
import { useChat } from 'ai/react';
const { messages } = useChat({ api: '/api/chat' });

// Vue 3 Composition API
import { useChat } from 'ai/vue';
const { messages } = useChat({ api: '/api/chat' });
// messages 是 Vue ref

// Svelte 5
import { Chat } from 'ai/svelte';
let chat = new Chat({ api: '/api/chat' });
// chat.messages 是 $state rune

// Angular
import { useChat } from 'ai/angular';
const chat = useChat({ api: '/api/chat' });
// chat.messages 是 signal

useAssistant（已收缩）

useAssistant 是给 OpenAI Assistants API 用的专门 hook（threads / runs / file 上传）。2026 年使用率收缩，原因：

  • OpenAI Assistants API 自身演进缓慢
  • 大部分 chat 应用用 useChat + 自己存 thread state 即可
  • 需要"OpenAI Assistants 特有功能"（比如 file_search built-in tool）的项目仍可用

新项目优先选 useChat。

03 应用场景

场景 A · 客服 chat（多轮对话）→ useChat

用户与 LLM 多轮对话；LLM 调 tool 查订单 / 发通知。useChat 维护完整 messages history + 流式渲染 + tool call UI。这是 useChat 的本垒。

场景 B · 提示词改写 / 翻译（单次）→ useCompletion

"把这段中文翻译成英文" / "把这段提示词改得更精炼"。单次输入 → 单次流式输出，不需要 multi-turn history。useCompletion 比 useChat 更轻。

场景 C · 流式生成 form / 表格 → useObject

用户描述需求 → LLM 流式生成 order 确认 form / 比较表格 / 推荐卡片。useObject + Zod schema 让前端边收边渲染逐渐完整的 object。比 useChat + 自己 parse JSON 更可靠。

场景 D · 调 Server Action（不是 Route Handler）→ useChat with prepareRequestBody

Next.js 项目想用 Server Action 而不是 Route Handler。useChat 仍能用：通过 experimental_prepareRequestBody 自定义请求 + Server Action 内部 streamText。详见 04 节代码。

场景 E · 进度条 + LLM 流式 → 自定义 data part

chat 应用里 backend 处理慢（先查 DB → 调 LLM → 后处理）。用 data part 推 progress：data-progress step 1/5 → 5/5。前端按 part type 渲染进度条 + LLM 文字流式同时显示。用户感知 backend 在工作。

场景 F · RAG citation → source part

RAG 应用：LLM 回答时引用 N 篇文档。后端在响应里发 source parts（含 url / title），前端渲染成 citation 卡片。比塞进 text 让 LLM 自己打 footnote 更可靠。

04 典型代码：完整端到端

useChat 端到端最小例子

typescript 
// app/api/chat/route.ts
import { streamText, convertToModelMessages, type UIMessage } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();
  const result = streamText({
    model: openai('gpt-4o'),
    messages: convertToModelMessages(messages),
  });
  return result.toUIMessageStreamResponse();
}

// app/chat/page.tsx
'use client';
import { useChat } from 'ai/react';

export default function ChatPage() {
  const { messages, input, setInput, handleSubmit, status } = useChat({
    api: '/api/chat',
  });

  return (
    <div>
      {messages.map((m) => (
        <div key={m.id}>
          <strong>{m.role}</strong>:
          {m.parts.map((part, i) =>
            part.type === 'text' ? <span key={i}>{part.text}</span> : null
          )}
        </div>
      ))}
      <form onSubmit={handleSubmit}>
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          disabled={status !== 'ready'}
        />
      </form>
    </div>
  );
}

useChat 调 Server Action（不是 Route Handler）

typescript 
// app/actions.ts
'use server';
import { streamText, convertToModelMessages } from 'ai';
import { openai } from '@ai-sdk/openai';
import { createStreamableValue } from 'ai/rsc';

export async function chatAction(messages: UIMessage[]) {
  const stream = createStreamableValue('');
  (async () => {
    const result = streamText({
      model: openai('gpt-4o'),
      messages: convertToModelMessages(messages),
    });
    for await (const delta of result.textStream) {
      stream.append(delta);
    }
    stream.done();
  })();
  return stream.value;
}

// 但实际上更常见用法是：useChat 仍走 /api/chat (Route Handler)

    required: z.boolean(),
    options: z.array(z.string()).optional(),

// app/api/generate-form/route.ts
import { streamObject } from 'ai';
import { openai } from '@ai-sdk/openai';
import { formSchema } from '@/schema';

export async function POST(req: Request) {
  const { description } = await req.json();
  const result = streamObject({
    model: openai('gpt-4o'),
    schema: formSchema,
    prompt: `Generate a form with fields for: ${description}`,
  });
  return result.toTextStreamResponse();
}

// app/form-gen/page.tsx
'use client';

import { formSchema } from '@/schema';

    api: '/api/generate-form',
    schema: formSchema,
  });

    <div>
      <button onClick={() => submit({ description: 'A signup form for SaaS' })}>
        Generate
      </button>
      {object?.title && <h2>{object.title}</h2>}
      {object?.description && <p>{object.description}</p>}
      {object?.fields?.map((field, i) => (
        field?.name && (
          <div key={i}>
            <label>{field.label}</label>
            <input type={field.type} name={field.name} required={field.required} />
          </div>
        )
      ))}
    </div>
  );
}

自定义 data part 实现 progress 进度

typescript 
// 后端
import { createUIMessageStreamResponse, streamText, convertToModelMessages } from 'ai';

export async function POST(req: Request) {
  const { messages } = await req.json();
  return createUIMessageStreamResponse({
    execute: async (writer) => {
      writer.write({ type: 'data-progress', data: { step: 1, total: 3, label: 'Searching...' } });
      const docs = await searchKnowledgeBase(messages);

      writer.write({ type: 'data-progress', data: { step: 2, total: 3, label: 'Generating...' } });

      const result = streamText({
        model: openai('gpt-4o'),
        system: `Use these docs as context: ${JSON.stringify(docs)}`,
        messages: convertToModelMessages(messages),
      });
      writer.merge(result.toUIMessageStream());

      writer.write({ type: 'data-progress', data: { step: 3, total: 3, label: 'Done' } });
    },
  });
}

// 前端
{messages.map((m) =>
  m.parts.map((part, i) => {
    if (part.type === 'text') return <span key={i}>{part.text}</span>;
    if (part.type === 'data-progress')
      return <ProgressIndicator key={i} step={part.data.step} total={part.data.total} label={part.data.label} />;
    return null;
  })
)}

带 metadata 的 useChat（前端 state 不污染 LLM）

typescript 
'use client';
import { useChat } from 'ai/react';

const { messages, sendMessage, setMessages } = useChat({ api: '/api/chat' });

// 用户点赞某条消息（更新 metadata，不重发给 LLM）
function likeMessage(id: string) {
  setMessages((prev) =>
    prev.map((m) =>
      m.id === id
        ? { ...m, metadata: { ...m.metadata, liked: true, likedAt: Date.now() } }
        : m
    )
  );
}

// 渲染时显示 metadata
{messages.map((m) => (
  <div key={m.id}>
    {m.parts.map(...)}
    {m.metadata?.liked && <span>👍 已赞</span>}
    <button onClick={() => likeMessage(m.id)}>赞</button>
  </div>
))}

05 常见陷阱

陷阱 1 · 把 metadata 当 part 用

metadata 不会发给 LLM 也不会进 part 数组。如果你想让 LLM 看到某条信息（用户偏好 / 时间戳），应该塞进新 part type 或 message text。修复：明确"前端展示用 metadata；发给 LLM 用 parts"。

陷阱 2 · 渲染 partial object 时直接用 schema.parse

useObject 的 partial 中间状态不一定符合 schema → schema.parse 抛错。修复：渲染时用 optional chaining（object?.fields?.map(...)）；validate 仅在最终 object。

陷阱 3 · useChat 初始 messages 没设

每次 mount 都从空开始。如果用户刷新页面会丢失对话历史。修复：通过 initialMessages 注入（从 server / localStorage / 数据库读）。

陷阱 4 · 流式中改 messages 数组

流式响应时（status === 'streaming'）用 setMessages 改 messages 可能与 SDK 内部累加冲突。修复：流式中只用 SDK 提供的 API（sendMessage / regenerate / stop）；改 metadata / 编辑历史等操作要在 status === 'ready' 时做。

陷阱 5 · useCompletion 端 toUIMessageStreamResponse

useCompletion 端要 toTextStreamResponse()（纯文本 stream），不是 toUIMessageStreamResponse()（含 parts 结构）。混了 → useCompletion 收不到。修复：useChat 端用 toUIMessageStreamResponse；useCompletion 端用 toTextStreamResponse。

陷阱 6 · v4 → v5 升级后 input value 丢失

v4 用 handleInputChange + input；v5 仍可用但推荐 setInput。混用可能让 input 控制不对。修复：统一用 setInput 风格（onChange={(e) => setInput(e.target.value)}）。

陷阱 7 · 多个 useChat 实例共享 state 期待

同一组件树里两个 useChat 是独立实例，不共享 messages。如果你想跨组件共享，要么 lift up state 到 context，要么用 setMessages 同步。修复：用 React Context / Zustand / Jotai 把 messages 提升到全局。

陷阱 8 · 用 useAssistant 启新项目

useAssistant 是 OpenAI Assistants API 专用，2026 使用率收缩。修复：新项目优先 useChat + 自己管 thread state；除非确定要用 OpenAI Assistants 特有 feature（file_search 等）。
