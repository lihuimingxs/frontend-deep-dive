P1 · 是谁 / 为什么

不讲 API 细节，只解决两个问题：把 LLM 接进 web 应用 5 年里发生了什么、为什么 2026 主线是 Vercel AI SDK v6 的 Agent abstraction 而不是 LangChain JS 的 chains/agents。给后面所有"为什么这样设计"提供历史与思想坐标。

章节数 2 主要来源 Vercel AI SDK GitHub release notes + vercel.com/blog/ai-sdk-{5,6} + Anthropic eng blog + LangChain release history + OpenAI Function Calling 公开记录 状态 ✅ 全部完成

本阶段覆盖

  • 2020-11 · OpenAI GPT-3 公开 API（raw HTTP，无 SDK）
  • 2022-11 · ChatGPT 发布 → LLM 应用爆炸
  • 2023-01 · LangChain JS 第一波 LLM orchestration（chains / prompts / agents）
  • 2023-06 · OpenAI Function Calling（第一个标准化 tool calling 协议）
  • 2024-03 · Vercel AI SDK v3：useChat 进入主线 + Generative UI 实验
  • 2024-04 · Anthropic Tool Use（Claude API tool calling）
  • 2024-11 · MCP 公开（Anthropic 主导）
  • 2024-12 · Vercel AI SDK v4：多 provider 抽象成熟
  • 2025-08 · Vercel AI SDK v5：UIMessage / ModelMessage 二分；流式协议切到 SSE 标准；多框架 feature parity
  • 2025 · Anthropic Claude Agent SDK 公开（同一个 loop 驱动 Claude Code）
  • 2026-02 · Vercel AI SDK v6：Agent abstraction + AI Gateway + Human-in-the-loop
  • 2026 内 · Vercel AI SDK v7 canary：WorkflowAgent + durable execution + loop control
  • 三阶段范式演化 · 包 LLM API（2023-2024）→ agent loop 一等公民（2025-2026）→ durable execution（2026 起）
