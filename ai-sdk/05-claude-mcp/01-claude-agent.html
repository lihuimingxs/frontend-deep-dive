Claude Agent SDK 设计哲学 + 与 Vercel 两种范式对比

Claude Agent SDK 是 Anthropic 给"构建生产 AI agents"的官方库，同一个 agent loop 驱动 Claude Code 自身（dogfooding）。这一章把它的核心组件（agent loop / subagents / hooks / sessions / built-in tools / MCP）+ 与 Vercel AI SDK 的 8 维对比讲清，给"哪个 SDK 该用在哪"提供决策基础。

阅读时长 ~22 分钟 前置 读过 P1.2（设计哲学）+ P4（Vercel Agent abstraction） 主要来源 code.claude.com/docs/en/agent-sdk + github.com/anthropics/claude-agent-sdk-python + github.com/anthropics/claude-agent-sdk-typescript + Anthropic engineering blog（"How we built Claude Code"）

01 速览

  1. Claude Agent SDK 是 Anthropic 官方的"agent 构建库"，Python + TypeScript 双语言。同一个 agent loop 驱动 Claude Code 自身（CLI 工具）。
  2. 核心组件：(a) Agent Loop（与 Claude Code 共用）；(b) Subagents（独立 context window + 独立 prompt + 独立 tool 权限）；(c) Hooks（PreToolUse / PostToolUse / UserPromptSubmit / Stop 等完整生命周期）；(d) Sessions（SDK 内部管 conversation state）；(e) Built-in tools（Read / Edit / Bash / Glob / WebSearch / Task 等 Claude Code 同款）；(f) MCP integration（中心地位）。
  3. Subagents 关键：每个 subagent 有自己的 context window，解决主 agent context 膨胀问题。父 agent 通过特殊 tool 调子 agent。messages 含 parent_tool_use_id 字段追踪谁调谁。
  4. Hooks 完整生命周期：PreToolUse（拦截 tool 调用前）/ PostToolUse（修改 tool 输出，2026-05 起可通过 hookSpecificOutput.updatedToolOutput 替换）/ UserPromptSubmit / Stop。比 Vercel 的 onStep / onFinish 更细粒度。
  5. In-process MCP servers（2026-05 改进）：自定义工具不再需要单独进程跑 MCP server。直接在 Python / TS 进程内实现，性能 + 部署都更简单。
  6. 跨范式 8 维对比：主战场 / UI 层 / Agent 抽象 / 多 provider / MCP / Hooks / 状态管理 / Dogfooding 八个维度，两家差异明显。
  7. 2026 选型决策：(a) 前端 web 应用 → Vercel；(b) CLI 工具 / 终端 agent → Claude SDK；(c) 后端复杂 agent orchestration（多 subagent / 长任务）→ Claude SDK；(d) 简单后端 agent → 都行，看团队栈；(e) 前后端混合 → 各管一边。
  8. 不是"一个替代另一个"：两家解决不同问题，部分重叠。2026 起 MCP / streaming / multi-provider 等维度收敛，但核心定位（前端 SDK vs agent SDK）仍不同。

Subagents：独立 context 解决膨胀问题

主 agent 跑长任务时 context window 会膨胀（messages + tool results 累积）。Subagents 解决这个问题：

typescript 
import { ClaudeAgent } from '@anthropic-ai/claude-agent-sdk';

const agent = new ClaudeAgent({
  systemPrompt: 'You are a code review coordinator.',
  subagents: {
    codeReviewer: {
      systemPrompt: 'You are a strict code reviewer focused on security and performance.',
      tools: ['Read', 'Glob', 'Grep'],  // 只给读权限
      // 独立 context window —— 主 agent 不看 review 的中间过程
    },
    testRunner: {
      systemPrompt: 'You are a test runner.',
      tools: ['Bash', 'Read'],
    },
    securityScanner: {
      systemPrompt: 'You scan code for security issues.',
      tools: ['Read', 'Grep', 'WebSearch'],
    },
  },
  // 主 agent 用 Task tool 调 subagent
  tools: ['Task'],
});

await agent.run({
  prompt: 'Review PR #42: run tests, check security, summarize findings',
});

// 主 agent 流程:
// 1. 主 agent 用 Task tool 调 codeReviewer
// 2. codeReviewer 在自己的 context 里完成 review
// 3. 返回 review summary 给主 agent (主 agent 只看 summary 不看 review 过程)
// 4. 主 agent 用 Task tool 调 testRunner / securityScanner
// 5. 综合三个 subagent 结果给最终回复

关键好处：

  • 主 agent context 不膨胀：subagent 内部细节不进主 agent context
  • 专项化：每个 subagent 用 specialized prompt + 受限 tool 权限
  • 可并行：理论上多个 subagent 可并发跑（CLAUDE_CODE_FORK_SUBAGENT=1）

Hooks 完整生命周期

typescript 
const agent = new ClaudeAgent({
  systemPrompt: '...',
  tools: [...],

  hooks: {
    // 用户提交 prompt 前
    UserPromptSubmit: async ({ prompt }) => {
      // 可以修改 prompt（添加 context）
      return { prompt: `[User session: ${userId}]\n${prompt}` };
      // 或返回 { decision: 'block', reason: '...' } 阻止
    },

    // tool 调用前（可拦截 / 修改 input / 阻止）
    PreToolUse: async ({ tool, input }) => {
      console.log(`Calling ${tool.name} with`, input);
      if (tool.name === 'Bash' && input.command.includes('rm -rf')) {
        return { decision: 'block', reason: 'Dangerous command blocked' };
      }
      // log 到 observability
      observability.log({ event: 'tool_call', tool: tool.name, input });
    },

    // tool 调用后（可修改 output, 2026-05 新能力）
    PostToolUse: async ({ tool, input, output }) => {
      // 脱敏输出再喂回 LLM
      if (tool.name === 'Read' && output.includes('SECRET=')) {
        const sanitized = output.replace(/SECRET=\S+/g, 'SECRET=***');
        return {
          hookSpecificOutput: {
            updatedToolOutput: sanitized,
          },
        };
      }
    },

    // agent 结束前
    Stop: async ({ messages, finalAnswer }) => {
      // 持久化对话历史
      await db.session.create({ data: { messages, finalAnswer } });
    },
  },
});

对比 Vercel 的 onStep / onFinish：Vercel 仅在 step 之间和最终调；Claude SDK 在每个 tool 前后 + prompt 提交前 + agent 停止前都有钩子。更细粒度。

Claude Agent SDK 内置一组 tool（与 Claude Code CLI 同款）：

这些 tools 是"用 LLM 操作文件系统 / shell / web"这一类需求的标准抽象。Vercel AI SDK 没有等价物（你要自己实现）。

Sessions：SDK 管对话状态

typescript 
// Claude SDK session
const agent = new ClaudeAgent({ ... });

const session = await agent.startSession({ sessionId: 'user-123-thread-1' });

await session.send({ prompt: 'What is the weather?' });
await session.send({ prompt: 'Forecast for tomorrow?' });  // 自动接续上下文

// 对比 Vercel: 你自己管 messages: UIMessage[]，每次请求传完整数组

选择：

  • Vercel 让开发者管 state（前端 useChat / 后端自己存）
  • Claude SDK 让SDK 管 state（默认）

哪个更好？看场景。前端 chat（需要客户端展示）→ Vercel 让 state 在前端更顺；CLI / 后端 agent（不需要前端展示历史）→ Claude SDK session 更省事。

维度 Vercel AI SDK v6 Claude Agent SDK 2026-05
主战场 React/Next.js web 应用 CLI / 后端 / 任何 agent 场景
UI 层 useChat / useObject / streamUI 一等公民 没有内置 UI 层
Agent 抽象层级 Agent / ToolLoopAgent Agent loop + Subagents（更细粒度）

Dogfooding Vercel.com / v0 内部使用 Claude Code 自己驱动（生产级）

共同点（2026 收敛）

  • 都支持 MCP（外部工具接口）—— Claude SDK 中心地位 / Vercel experimental
  • 都支持 streaming（Vercel SSE / Claude SDK 内部 stream）
  • 都支持 multi-provider（程度不同）—— Vercel 完全 neutral / Claude 经 Bedrock / Vertex
  • 都有 hooks / interception—— Vercel onStep / Claude 完整 4 个 hooks
  • 都有 Agent abstraction—— Vercel v6 起 / Claude 一直有

03 应用场景

场景 A · 代码 review agent (CLI / GitHub Action)

跑在 CI 里 → review PR + 写 comment + 跑测试。Claude Agent SDK 的本垒。subagents（codeReviewer / testRunner / securityScanner）协作；hooks 实现自定义日志 / 阻止危险命令；built-in Read / Edit / Bash 直接用。

场景 B · 客服 chat (web 应用)

客服 chat 需要前端流式 UI + tool 调用 + 用户审批。Vercel AI SDK 的本垒。useChat hook + 后端 supportAgent + Human-in-the-loop。Claude SDK 没 UI 层，做 web chat 不合适。

场景 C · 数据 ETL agent (后端批处理)

每天跑批处理：从多个数据源拉数据 → 清洗 → 转换 → 入库。需要 agent loop + 错误重试 + observability hooks。Claude Agent SDK 更顺（Python + 完整 hooks）。

场景 D · 旅行助理 (web + 复杂 generative UI)

"我下周去东京"→ 展示天气卡 / 航班 / 活动推荐 / 订餐建议。Vercel AI SDK + streamUI 的本垒。Claude SDK 没 generative UI 概念。

场景 E · 前后端混合：双 SDK 协作

前端 chat（Vercel useChat）→ 用户问"帮我重构这个项目" → 后端调 Claude SDK agent（含 codeReviewer / refactorer / testRunner subagents）→ 后端流式返回进度 → 前端展示。2026 主流架构之一。详见 P5 场景 F。

场景 F · DevOps automation agent

监控告警触发 → agent 跑 diagnose（看日志 / 查指标 / 跑测试）→ 决定 mitigation（restart / scale up / rollback）→ 执行（Bash / kubectl）。Claude Agent SDK + built-in Bash + hooks 阻止危险命令。

04 典型代码：完整 Claude Agent SDK 实战

代码 review agent（TypeScript）

typescript 
import { ClaudeAgent, query } from '@anthropic-ai/claude-agent-sdk';

const agent = new ClaudeAgent({
  systemPrompt: `You are a code review coordinator.
  Use subagents to delegate specialized tasks.
  Synthesize their findings into a final review.`,

  // 主 agent 能用 Task tool 调 subagent
  allowedTools: ['Task', 'Read'],

  subagents: {
    codeReviewer: {
      systemPrompt: 'You are a strict code reviewer. Focus on logic, edge cases, error handling.',
      allowedTools: ['Read', 'Grep'],
    },
    securityScanner: {
      systemPrompt: 'Scan for security vulnerabilities (SQL injection, XSS, secrets).',
      allowedTools: ['Read', 'Grep', 'WebSearch'],
    },
    testRunner: {
      systemPrompt: 'Run tests and report failures with actionable feedback.',
      allowedTools: ['Bash', 'Read'],
    },
  },

  hooks: {
    PreToolUse: async ({ tool, input }) => {
      // 阻止危险命令
      if (tool.name === 'Bash' && /rm -rf|sudo|chmod 777/.test(input.command)) {
        return { decision: 'block', reason: 'Dangerous command blocked' };
      }
      // log
      await db.auditLog.create({ data: { event: 'tool_use', tool: tool.name, input } });
    },

    PostToolUse: async ({ tool, output }) => {
      // 脱敏 secret
      if (typeof output === 'string' && /SECRET|API_KEY/.test(output)) {
        return {
          hookSpecificOutput: {
            updatedToolOutput: output.replace(/(SECRET|API_KEY)\s*=\s*\S+/g, '$1=***'),
          },
        };
      }
    },
  },
});

// 调用
for await (const message of query({
  agent,
  prompt: 'Review PR #42 in this repo',
})) {
  console.log(message);
  // message 含: text / tool_use / tool_result / subagent results / final summary
}

Python 等价代码

python 
from anthropic_agent_sdk import ClaudeAgent, query

agent = ClaudeAgent(
    system_prompt="You are a code review coordinator...",
    allowed_tools=["Task", "Read"],
    subagents={
        "code_reviewer": {
            "system_prompt": "Strict code reviewer...",
            "allowed_tools": ["Read", "Grep"],
        },
        "security_scanner": {
            "system_prompt": "Scan for vulnerabilities...",
            "allowed_tools": ["Read", "Grep", "WebSearch"],
        },
    },
    hooks={
        "PreToolUse": pre_tool_use_hook,
        "PostToolUse": post_tool_use_hook,
    },
)

async def pre_tool_use_hook(tool, input):
    if tool.name == "Bash" and "rm -rf" in input.get("command", ""):
        return {"decision": "block", "reason": "Dangerous"}

async for message in query(agent=agent, prompt="Review PR #42"):
    print(message)

Vercel + Claude 混合架构

typescript 
// 前端 (Vercel useChat)
'use client';
import { useChat } from 'ai/react';

function Chat() {
  const { messages, sendMessage } = useChat({ api: '/api/refactor' });
  return (...);
}

// 后端 Route Handler (Vercel) → 调 Claude SDK
import { ClaudeAgent, query } from '@anthropic-ai/claude-agent-sdk';

const refactorAgent = new ClaudeAgent({
  systemPrompt: 'You refactor code...',
  subagents: { codeAnalyzer: ..., refactorer: ..., testRunner: ... },
  hooks: { ... },
});

export async function POST(req: Request) {
  const { messages } = await req.json();
  const lastMessage = messages[messages.length - 1].parts.map(p => p.text).join('');

  // 把 Claude SDK 的输出流转成 v5 SSE 协议给 Vercel useChat
  return new Response(
    new ReadableStream({
      async start(controller) {
        for await (const msg of query({ agent: refactorAgent, prompt: lastMessage })) {
          if (msg.type === 'text') {
            const event = JSON.stringify({ type: 'text-delta', textDelta: msg.text });
            controller.enqueue(`data: ${event}\n\n`);
          }
        }
        controller.close();
      },
    }),
    { headers: { 'Content-Type': 'text/event-stream' } }
  );
}

05 常见陷阱

陷阱 1 · 期待 Claude Agent SDK 提供 useChat-like UI hooks

没有。这是设计选择不是缺失。修复：前端 web chat 用 Vercel；CLI / 后端 agent 用 Claude SDK；混合架构两者各管一边。

陷阱 2 · 在 web 后端硬上 Claude Agent SDK

简单 web 后端"前端 useChat → 后端 streamText"完全够用。Claude SDK 的 subagents / hooks 在简单场景是 overkill。修复：复杂 agent orchestration 才用 Claude SDK；简单后端用 Vercel streamText 即可。

陷阱 3 · subagent context 跨 subagent 共享期待

subagents 是独立 context。codeReviewer 看到的内容 securityScanner 看不到。修复：(a) 主 agent 把 subagent A 的 summary 作为 prompt 给 subagent B；(b) 用 in-process MCP / DB 让 subagents 共享 state。

陷阱 4 · 期待 Claude SDK 能用 OpenAI / Google model

Claude SDK 默认 Anthropic。可以经 Bedrock / Vertex AI 切，但不像 Vercel 完全 vendor-neutral。修复：如果需要切多 provider → Vercel；如果接受锁定 Claude（含经 Bedrock / Vertex）→ Claude SDK。

陷阱 5 · hooks 里做重操作阻塞 agent loop

PreToolUse / PostToolUse 同步等待 → agent 卡住。修复：hooks 只做 lightweight observability + 必要的拦截判断；重操作（写 DB / 发外部请求）fire-and-forget 或 enqueue 到 queue。

陷阱 6 · 没用 in-process MCP 而用外部进程

2026-05 起 Claude SDK 支持 in-process MCP server，性能 + 部署都更简单。仍用外部进程 = 浪费。修复：自定义工具用 createInProcessMCPServer；外部 MCP server 仅用于"已有的第三方 server"（GitHub MCP / Slack MCP 等）。

陷阱 7 · 用 Claude SDK 做"前端 SDK"

Claude SDK 没 UI hooks / 没 React 集成 / 不暴露 streaming UI。修复：理解它定位是"agent 库"不是"前端 SDK"。前端要用 useChat → 走 Vercel。

陷阱 8 · 期待 Subagent 是 LangChain agent

Claude SDK 的 Subagent 是"独立 context 的 agent"，不是 LangChain 那种"高层 orchestration"。Subagent 共用同一个 agent loop，仅是 context 隔离。修复：理解 Subagent 不是 chain / pipeline / DAG，是简单的"独立 context + 受限权限"封装。
