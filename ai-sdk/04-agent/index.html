P4 · Agent 抽象 ⭐

Vercel AI SDK v6 主线最重要的两章。Agent / ToolLoopAgent 把"调 LLM → 看 tool calls → 调 tool → 喂回 → 直到 stop"这个循环抽象成可复用类；Tool use 在 v5+ 加了流式 input + Human-in-the-loop 审批 + structured output with tool calling。这是 v6 与 v5 最大的差异点。

章节数 2 主要来源 ai-sdk.dev/docs/agents + vercel.com/blog/ai-sdk-6 + Anthropic Tool Use docs + OpenAI Function Calling docs + Zod 文档 状态 ✅ 全部完成

本阶段覆盖

  • 裸 streamText + tools 的局限 · 自己写 tool loop / 多步骤推理 / 复用难
  • v6 Agent abstraction · 一次定义 model + system + tools 多处复用
  • ToolLoopAgent · 内置 tool loop · maxSteps · stopWhen · onStep
  • structured output with tool calling · v6 stable · "调 tool 后返回结构化 object"
  • Tool 定义 · description + Zod parameters + execute
  • 流式 input（v5+ 默认）· partial-call → call → result 状态机
  • Human-in-the-loop（v6 stable）· requiresApproval · 用户批准后继续
  • 多 provider 协议差异 · OpenAI Function Calling vs Anthropic Tool Use vs Google · SDK 抽象
  • 错误处理 · execute 抛错作为 result 喂回 LLM · onError 回调 · retry 策略
