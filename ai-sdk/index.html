编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。本主题相比其他主题有以下额外约定：

  • 主线锁定 Vercel AI SDK v6（current latest，2026-02）—— 所有"现在怎么用"的代码示例基于 v6（Agent abstraction / SSE 协议 / UIMessage 心智）；v5 仅在 P1 历史段 + P2.2 协议演化段出现；v7 canary 仅在 P7 展望段提
  • 「短重述 + 链回」 —— RSC / use API 在 React 主题；Server Actions / Edge runtime 在 Next.js 主题；HTTP 流式协议层（SSE / fetch streams）在 HTTP 主题；测试栈（MSW）在 测试主题；本主题用 1-3 句话点到本质，紧跟一句"详细的 X 请参考 Y"
  • 设计动机以 Vercel / Anthropic 一手出处为准 —— Vercel AI SDK 演化以 vercel.com/blog 设计文章为准；Claude Agent SDK 概念以 code.claude.com + Anthropic eng blog 为准；MCP 以 modelcontextprotocol.io 为准；不引第三方 blog 复述
  • 跨范式对比要双向引用 —— P5 讨论"Vercel Agent vs Claude Agent SDK"时，每一段都要交叉引用两边官方说法（避免"我猜他们想这样"）
  • 观点强度 —— 强观点（"前端 chat 选 Vercel AI SDK 没争议"，"v4 → v5 是协议层 + state 模型双重重做"，"MCP 是 2026 工具接口事实标准"）；弱观点（"Vercel vs Claude Agent SDK 看场景"）；不观点（"Mastra / CopilotKit 等 niche 框架看团队偏好"）
