P2 · Vercel AI SDK 心智

Vercel AI SDK 不是单一的"LLM 客户端"，是分三层（Core / UI / RSC）的 SDK 家族。这一阶段把三层架构 + Core 四件套（generateText / streamText / generateObject / streamObject）+ Stream Protocol 从 v4 自家 NDJSON 切到 v5 SSE 标准的演化讲清。

章节数 2 主要来源 ai-sdk.dev/docs（v6 主线）+ vercel.com/blog/ai-sdk-5（v5 协议层切换设计文章）+ vercel.com/blog/ai-sdk-6（v6 spec 改进）+ HTTP 主题 P6.1（流式协议层根因） 状态 ✅ 全部完成

本阶段覆盖

  • 三层架构 · Core（ai）/ UI（ai/react 等）/ RSC（ai/rsc）
  • Core 四件套 · generateText / streamText / generateObject / streamObject
  • provider 抽象 · OpenAI / Anthropic / Google / Mistral / Bedrock 一份代码切换
  • AI Gateway（v6 起）· zero markup pricing · $5/月免费额度 · 统一 observability
  • Stream Protocol 演化 · v4 自家 Data Stream Protocol（NDJSON）→ v5 SSE 标准 → v6 维持
  • UIMessage vs ModelMessage 二分（v5 起）· 前端 state vs 发给 LLM 的精简表示
  • tool-invocations type-specific parts（v5 起）· tool-TOOLNAME 替代通用 tool-invocation
  • v5 升级最大迁移点 · 自定义 server 必须适配新协议（FastAPI / Python backend 是大坑）
