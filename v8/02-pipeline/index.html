P2 · 编译流水线

沿一段 JS 代码的旅程铺开：源码 → AST → bytecode → 优化机器码。先讲全景图与 tier-up 机制，再各自展开 Ignition 解释器和优化编译器三件套（Sparkplug / Maglev / TurboFan）。这是 V8 主题最长的一阶段。

章节数 3 主要来源 v8.dev/docs · launching-ignition-and-turbofan / sparkplug / maglev 博客 状态 ⏳ 待撰写

本阶段覆盖

  • 4 层 Pipeline 全景 —— Parser（lazy parsing）→ Ignition → Sparkplug → Maglev → TurboFan / Turboshaft，配合 OSR + Concurrent Compilation
  • Ignition 解释器 —— register-based VM、bytecode handlers（CSA）、Feedback Vector、Code Cache & Script Streaming、Generator / Async 状态机化、Strict Mode / TDZ 在 bytecode 层
  • Builtins 不走这条路 —— Array.prototype.map 等内置函数用 Torque DSL 写、预编译嵌进 Snapshot
  • 优化编译器三件套 —— Sparkplug（无 IR 1:1 JIT）/ Maglev（SSA + CFG 中层）/ TurboFan（Sea of Nodes + 2024 Turboshaft 后端重写）
  • 反优化机制 —— Eager / Lazy deopt、OSR 栈帧切换、deopt loop 排查
