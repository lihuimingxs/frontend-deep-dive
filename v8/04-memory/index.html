P4 · 内存与 GC

Hidden Class 让属性访问快，但对象本身的内存表示和何时回收是另一组独立问题。这一阶段讲 V8 的堆布局、对象编码、Pointer Compression、Orinoco GC，以及与 Cppgc / Oilpan 的双堆协作。

章节数 2 主要来源 v8.dev/blog · pointer-compression / trash-talk / concurrent-marking 状态 ⏳ 待撰写

本阶段覆盖

  • Heap 分代结构 —— Young (NewSpace) / Old / Code / Map / Large Object / Trusted Space
  • 对象编码 —— Tagged Pointer + SMI（Small Integer）+ HeapNumber + Mutable Heap Numbers + 多种 String 表示
  • Pointer Compression（V8 8.0+）—— 4GB cage、32-bit offset、堆 -43%、为 Sandbox 铺路
  • Conservative Stack Scanning（V8 11.2+）—— 让编译器丢掉 stack maps，为 Cppgc 互操作铺路
  • Orinoco GC 三件套 —— Parallel Scavenger（young gen）+ Concurrent Marking（old gen）+ Mark-Compact + Write Barrier + Remembered Set
  • Cppgc / Oilpan 与 V8 双堆协作 —— Unified Heap 跨堆 marking、TracedReference
