P6 · V8 与 WASM

这一阶段是与独立的 WebAssembly 主题之间的"接口章" —— 只讲 V8 怎么实现 Wasm（Liftoff + TurboFan + WasmGC），不讲 wasm32 ISA / WASI / Component Model 等通用 Wasm 知识（那些在 WASM 主题）。

章节数 1 主要来源 v8.dev/docs/wasm-compilation-pipeline · holiday-season-2023 博客 状态 ⏳ 待撰写

本阶段覆盖

  • Wasm 双层管道 —— Liftoff（baseline，1:1 无 IR，类似 Sparkplug）+ TurboFan（共享 JS 路径）+ Tier-up
  • WasmGC（2023 正式）—— 让 Java / Kotlin / Dart / Scala 这些"托管语言"高效编译到 wasm，性能 2x
  • JS Promise Integration (JSPI) —— 让同步 wasm 调用异步 JS API
  • Memory64 提案（V8 11+）—— 64-bit 索引，让 SQLite-WASM / duckdb-wasm 这类大数据应用可用
  • 跨边界调用开销 —— wasm-to-JS vs JS-to-wasm（V8 优化方向不对称）
  • Wasm 内存与 V8 堆 —— 线性内存（cage 外 ArrayBuffer）vs WasmGC 对象（cage 内）
