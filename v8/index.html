V8 Engine 深度学习 · 编译、内存与嵌入

基于 v8.dev 官方文档 与 V8 团队博客 整理。回答三个问题：V8 怎么走到今天（17 年项目史 + 同代人对照）、它怎么把 JS 真正跑起来（4 层 JIT 流水线 + Hidden Class + Inline Cache）、它怎么不爆内存又能被嵌入到任何地方（Orinoco GC + Snapshot + Embedder API + V8 Sandbox）。补全 JavaScript 与 ECMAScript 之后的"机器怎么真的跑"最后一环。

受众 已学完 JS / ECMA 的工程师 来源 v8.dev / V8 GitHub / 团队博客 章节 12 章 / 6 阶段 状态 ✅ 12/12 章完成

分阶段目录

共 6 个阶段、12 章，沿一段 JS 代码的旅程铺开：V8 是谁 → 编译流水线 → 让动态变快 → 内存与 GC → 嵌入与诊断 → V8 与 WASM。建议顺序阅读；性能调优工程师可直接跳到 P2.1 / P3 / P5.2，Node / Deno / Workers 嵌入开发者可直接跳到 P5.1。

✅ 2 章完成

项目史 17 年 同代人 JSC / SM / Hermes V8 嵌入生态 Workers Isolate-per-tenant

P2 · 编译流水线

4 层 Pipeline Ignition + Bytecode Sparkplug / Maglev / TurboFan Builtins / Torque Code Cache Deopt / OSR

P3 · 让动态语言变快

✅ 2 章完成

V8 的核心魔法：用 Hidden Class + Inline Cache 把 JavaScript 的"动态"伪造成"静态"。讲清 Map / DescriptorArray / TransitionArray 三件套、in-object / fast / dictionary 三档存储、Slack Tracking、Elements Kind；以及 IC 在 Mono → Poly → Mega 的状态机演化、Feedback Vector 与编译器的协作。

Hidden Class / Map Transition Tree Slack Tracking Elements Kind IC Mono → Mega Feedback Vector

P4 · 内存与 GC

✅ 2 章完成

V8 在堆上怎么放对象、怎么压缩指针、怎么不卡主线程地回收。Heap Layout（Young / Old / Code / Map / Trusted Space）+ Tagged Pointer + SMI + Pointer Compression（4GB cage） + Orinoco GC（Parallel Scavenger + Concurrent Marking + Mark-Compact + Write Barrier）+ Cppgc / Oilpan 与 V8 的双堆协作。

Heap Layout SMI / HeapNumber Pointer Compression Orinoco GC Concurrent Marking Cppgc / Oilpan

P5 · 嵌入与诊断

Isolate / Context HandleScope Snapshot --trace-deopt d8 / Turbolizer V8 Sandbox

P6 · V8 与 WASM

Liftoff JIT WasmGC JSPI Memory64 跨边界调用开销

编写约定

每章遵循统一的 6 段结构：速览 → 底层逻辑 → 应用场景 → 典型代码 → 常见陷阱 → 延伸阅读。但 V8 主题相比 ECMA 有以下额外约定：

  • 「短重述 + 链回」 —— 当一个机制在 ECMA 主题已详述时（如 Internal Methods / Realm / Memory Model），本主题用 1-3 句话点到本质，紧跟一句"详细规范请参考 X"，重点讲 V8 怎么实现
  • 版本号必标 —— V8 改动激烈，没有版本号的论述不可信。每个特性必标"V8 X.Y 引入 / V8 X.Y 移除"
  • v8.dev 一手优先 —— 每个机制的"权威原文"必须是 v8.dev 博客或 v8.dev/docs。社区博客只用来佐证或补充实测数据
  • 机器码示例只用 d8 —— 所有 --print-bytecode --trace-deopt --trace-ic 输出都要用 V8 自带 d8 命令复现，不依赖 Chrome / Node 内部
  • ⚠️ 标注 —— Crankshaft / Full-codegen 永远标"2017 已删除"；2018+ 出现这两个名字的资料是过时博客
