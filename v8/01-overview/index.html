P1 · V8 是谁

这一阶段不讲技术细节，只解决两个问题：V8 怎么走到今天、它在 JS 引擎家族里站什么位置。给后面所有"为什么这样设计"提供历史坐标。

章节数 2 主要来源 v8.dev/blog · Wikipedia 状态 ⏳ 待撰写

本阶段覆盖

  • 项目史 17 年 —— 三阶段叙事：单层 JIT 时代（2008-2014）→ 重构为现代管道（2015-2017，Ignition+TurboFan 大重构 / Crankshaft 退役）→ 多层管道精细化（2018-2025，Sparkplug / Maglev / Pointer Compression / V8 Sandbox）
  • 架构突变的根因 —— 每次大改背后的具体压力源（不是"过时"，是"跟不上 ES2015+ 新语法"或"启动期 TF 来不及"）
  • 同代人对照 —— JSC（4 层管道更早 / iOS 内存约束）、SpiderMonkey（研究驱动）、Hermes（反向证明 JIT 不必需）
  • V8 的"分身史" —— Chrome / Node / Deno / Cloudflare Workers / Electron 各自如何裁剪、加约束
