P1 · Node 是谁

这一阶段不讲技术细节，只解决两个问题：Node 怎么走到今天、它在服务端 / Edge JS 生态里站什么位置。给后面所有"为什么这样设计"提供历史与对照坐标。

章节数 2 主要来源 Ryan Dahl 演讲 · OpenJS 治理文档 状态 ⏳ 待撰写

本阶段覆盖

  • 项目史 17 年 —— Ryan Dahl 2009 起源 / 2014 io.js 分裂治理失败 / 2015 Foundation 重组 / 2018 Ryan 自我批评启发 Deno / 2022 Bun 挑战 / 2024+ Node 22 内置工具回应
  • 关键治理事件 —— io.js 分裂的根因、Foundation 模式如何成为大型 OSS 标配
  • Ryan Dahl 自我批评 —— 模块系统 / package.json / 安全性等问题如何反过来驱动 Node 后续改进
  • 同代人对照 —— Node vs Deno（V8 + Tokio + 权限）vs Bun（JSC + Zig + 工具一体化）vs Edge Runtime（受限子集）
  • Node 的护城河 —— npm 2.5M+ 包生态决定任何替代品要么兼容、要么放弃 80% 现成生态
