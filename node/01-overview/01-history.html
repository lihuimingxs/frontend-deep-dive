Node 项目史 2009-2025

Node 17 年的故事讲成一条主线 —— 不是按版本号，而是按"每次治理 / 架构突变及其根因"。从 2009 年 Ryan Dahl 在 JSConf EU 演讲发布 Node 0.1.0，到 2014 io.js 分裂治理失败、2015 Foundation 重组、2018 Ryan 自己批评 Node 启发 Deno、2022 Bun 挑战、2024 Node 22 内置工具回应——理解这条线之后，许多 Node 当下的设计取舍就有了根因。

阅读时长 ~30 分钟 前置 任意 Node 经验 主要来源 Ryan Dahl 演讲 · OpenJS 治理文档

01 速览

  1. Node 是 Ryan Dahl 2009 年发布的服务端 JS 运行时——动机是"Apache 处理高并发的方式（线程 per 请求）有问题，需要事件驱动"。
  2. Node 史上最关键的治理转折是 2014-2015 io.js 分裂 + 重新合并——倒逼 Joyent 放弃单一公司控制，催生 Node Foundation。
  3. 今天 Node 在面对 Deno 和 Bun 的双重挑战——Node 22+ 内置 fetch / test / watch / Permission 等工具是对这种压力的回应。

05 常见陷阱

io.js (2014-2015) 已 abandoned

看老资料提到 io.js X.X 版本就是 Node 4 之前的"分支版本"——今天没有任何意义，所有特性都进了 Node。

Node 奇数版本不是 LTS

Node 21 / 23 / 25 是"current"版本，6 个月后就 EOL；生产用偶数 LTS。

Ryan Dahl 的"10 regrets"不等于"Node 是失败的"

他批评的是设计选择，不是项目本身。Node 仍是服务端 JS 主导生态。
