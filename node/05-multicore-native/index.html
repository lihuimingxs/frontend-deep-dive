P5 · 多核与原生扩展

Node "单线程"是 JS 层面的——通过 Worker Threads / Cluster / Child Process 三件套和 N-API 原生扩展可以充分利用多核 + 调用 native 库。这一阶段把这两条扩展路径讲清。

章节数 2 主要来源 nodejs.org/api/worker_threads · nodejs.org/api/n-api 状态 ⏳ 待撰写

本阶段覆盖

  • Worker Threads（Node 10.5+） —— 进程内多 V8 Isolate；postMessage 走 structured clone；CPU 密集 JS 适用
  • Cluster 模块 —— fork 多进程跑同一段代码；共享 listening port；I/O 密集 web server 多核扩展
  • Child Process —— spawn / fork / exec 子进程；可以跑非 JS 程序；IPC 通信
  • 三者选型判据 —— CPU 密集 / Web 多实例 / 调外部程序
  • SharedArrayBuffer + Atomics —— Workers 之间真正共享内存的唯一方式
  • MessageChannel / MessagePort —— 双向通信原语；可以跨 worker 转移
  • 三种 native module 写法 —— raw V8 / N-API / node-addon-api（现代默认）
  • N-API ABI 稳定性 —— 用 N-API v8 写的二进制能在所有 Node 18+ 跑
  • node-gyp + prebuild-install —— 工程实践：避免用户安装时编译
  • WASM 替代方案 —— 越来越多 native 需求改用 wasm（如 esbuild）
