P4 · Streams 与网络栈

Node 标准库最强的部分。Streams 是 Node 数据处理的统一抽象——http req/res 是 Stream、fs.createReadStream 是 Stream、Worker 通信也是 Stream；http / undici / fetch 是 Node 服务端编程的核心；fs / Buffer / process 是其他三个高频 API 群。这是 Node 主题最长的一阶段。

章节数 3 主要来源 nodejs.org/api/stream · nodejs.org/api/http · undici 文档 状态 ⏳ 待撰写

本阶段覆盖

  • Streams 4 类 —— Readable / Writable / Duplex / Transform
  • flowing vs paused 两种模式 —— stream 心智模型的关键分歧
  • highWaterMark + Backpressure —— 让 Node 不爆内存的核心机制
  • pipeline / async iterator / WHATWG Streams —— 现代 stream 消费方式与跨标准融合
  • HTTP server / client —— http / https / HTTP/2 / TLS / undici / 内置 fetch（Node 18+）
  • undici 高性能 client —— Node 团队写的连接池 + Keep-Alive 自动管理
  • fs Promise API —— fs.promises 现代写法 + readFile vs createReadStream 选择
  • Buffer 不在 V8 堆 —— ArrayBuffer Allocator 分配，自 Node 4 起本质是 Uint8Array
  • process 运行时基础 —— env / argv / exit / uncaughtException
