HTTP / undici / fetch

Node 服务端的核心场景——HTTP server / client。讲清传统 http 模块（基于 Streams）、新一代 undici（Node 团队自己写的高性能 client，用作内置 fetch 后端）、HTTP/2 支持、TLS。读完能选对 client，知道为什么 Node 18+ 的 fetch 是 undici。

阅读时长 ~30 分钟 前置 P4.1 Streams 主要来源 nodejs.org/api/http · undici 文档
📝 章节状态

骨架版本。对照大纲 node/outline.md § P4.2。

01 速览

  1. Node http server 整个 API 是 Streams 的应用——req 是 Readable、res 是 Writable。
  2. undici 是 Node 18+ 内置 fetch 的后端——比旧 http.request 快 2-3×。
  3. HTTP/2 是单独的 http2 模块——不是 http 模块的扩展，使用方式略有不同。

02 底层逻辑

http 模块（2009-）

正文待写。

  • createServer((req, res) => ...)：req 是 IncomingMessage（Readable）、res 是 ServerResponse（Writable）
  • 整个 API 是 Streams 应用——读 body 用 stream API（async iterator / pipeline）
  • Keep-Alive 自动管理
  • HTTP/1.1 only（HTTP/2 在 http2 模块）

https 模块

正文待写。

  • 同 http 但带 TLS 终止
  • 证书管理（key / cert / ca 选项）
  • ALPN 协议协商：客户端告知支持哪些协议（http/1.1, h2）

http2 模块（Node 8.4+）

正文待写。

  • HTTP/2 server / client
  • 流多路复用（一个 TCP 连接上跑多个并发 request）
  • Server Push（已废弃，Chrome 已不支持）
  • API 与 http 不兼容（新概念：Http2Session / Http2Stream）

undici（Node 18+ 内置）

正文待写。

  • Node 团队 Matteo Collina 主导写的高性能 HTTP/1.1 client
  • 显著快于旧 http.request（2-3× 吞吐）
  • 连接池 + Keep-Alive 自动管理
  • Node 18+ 内置，作为 fetch 的实现后端
  • 独立 npm 包发布，旧 Node 也能用

fetch 内置（Node 18+）

正文待写。

  • API 与浏览器 fetch 一致（Web Standards）
  • 实现基于 undici
  • 不再需要 node-fetch 之类的 polyfill
  • 仍然是实验性（Node 21 起稳定）

TLS / SSL

  • tls 模块作底层抽象

04 典型代码

javascript 
// 传统 http.createServer + Streams 风格
import { createServer } from 'node:http';
import { pipeline } from 'node:stream/promises';

const server = createServer(async (req, res) => {
  if (req.method === 'POST' && req.url === '/upload') {
    // req 是 Readable，写到磁盘
    await pipeline(req, createWriteStream('upload.bin'));
    res.statusCode = 200;
    res.end('OK');
  }
});
server.listen(3000);

// 现代客户端用 fetch（基于 undici）
const res = await fetch('https://api.example.com/data', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ x: 1 }),
});
const data = await res.json();

05 常见陷阱

fetch 在 Node 仍是"实验性"标记

Node 21+ 起 stable，但 21 之前 console 会有 warning。

http2 API 与 http 不互通

不能直接换；通常用反向代理（nginx）做 HTTP/2 终止给 Node 看 HTTP/1.1。

Keep-Alive 默认行为陷阱

http.Agent 默认 Keep-Alive 关闭（Node 19+ 改为开启）—— 老代码可能因这变更性能曲线突变。
