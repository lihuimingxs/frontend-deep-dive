P6 · 现代 Node 工程

Node 22+ 引入的"内置工具"——回应 Bun / Deno 的工具一体化趋势。一章覆盖：内置 node:test runner、Permission Model（仿 Deno）、Single Executable Application (SEA)、async_hooks / AsyncLocalStorage（OpenTelemetry 基础）、Watch mode、Inspector Protocol。

章节数 1 主要来源 nodejs.org/api/test · nodejs.org/api/permissions · async_hooks 状态 ⏳ 待撰写

本阶段覆盖

  • node:test（Node 18+ 实验，20 稳定） —— 内置测试 runner；TAP 输出；mock / subtests / 并行执行；竞争 vitest / Jest
  • node:assert —— 内置断言库；strict mode 与 web 标准对齐
  • Permission Model（Node 20+ 实验，22+ 稳定化） —— --permission --allow-fs-read=./data --allow-net=api.example.com
  • Single Executable Application (SEA) —— Node + 用户脚本打包成单文件可执行
  • async_hooks + AsyncLocalStorage —— 跨 async boundary 传递 context；OpenTelemetry / 分布式追踪基础
  • Inspector Protocol —— --inspect 启动；DevTools / VSCode 调试；详见 V8 P5.2
  • Watch mode（Node 18+） —— --watch app.js 自动重启，减少 nodemon 依赖
