CJS 与 ESM 双系统：解析算法与配置

把"为什么我的 import 跑不起来"这种典型问题的根因讲清楚。重点是解析算法 + package.json 字段。CJS 用 Node 自家算法，ESM 用 ECMA 规范算法——同一个路径可能解析成不同文件，这是双系统并存最大的工程现实。

阅读时长 ~30 分钟 前置 ECMA P7.4 Module Records 主要来源 nodejs.org/api/esm · nodejs.org/api/packages
📝 章节状态

骨架版本。对照大纲 node/outline.md § P3.1。

01 速览

  1. CJS 是"运行时模块"：被 require 时执行；同步；可以条件 require。
  2. ESM 是"静态模块"：解析时 V8 已知所有依赖；异步；不能条件 import（除非用 import()）。
  3. package.json 的 exports 字段是双系统兼容的核心——通过 conditional exports 给不同 import 方式不同入口。

02 底层逻辑

CommonJS 解析算法（Node 2009-）

正文待写。

  • 同步：require('foo') 阻塞等加载完成
  • Node 自家算法（不是 ES 规范）：先看 core module → 再看 file → 再看 node_modules 向上查找
  • 支持省略扩展名（.js / .json / .node 自动加）
  • 支持条件 require：if (x) require('./a'); else require('./b')

ESM 解析（Node 12+）

正文待写。

  • 异步：import 是规范级 async
  • 按 ECMA Module Records 规范解析（详见 ECMA P7.4）
  • HTTP / file URL 都支持
  • 必须带扩展名（.js / .mjs）—— 这是 ESM 与 CJS 最大的解析差异
  • 不能条件 import（除非用动态 import()）

package.json 字段全谱

正文待写。

  • type: "module"：把 .js 文件按 ESM 处理（默认是 CJS）
  • main：CJS 入口（旧）
  • module：ESM 入口（bundler 约定，非 Node 标准）
  • exports：现代标准，覆盖以上字段
  • imports：包内导入路径别名（如 #config）
  • types：TypeScript 类型入口

Conditional Exports：key 顺序敏感

正文待写。

json 
{
  "name": "my-lib",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    },
    "./utils": {
      "import": "./dist/utils.js",
      "require": "./dist/utils.cjs"
    }
  }
}

key 顺序敏感：earlier entries 优先级更高。一般规则"specific → general"——types 在前，import 在后。

导入时的扩展名问题

正文待写。

  • ESM：必须 import x from './foo.js'（带 .js）
  • CJS：可以 require('./foo')（不带）
  • TypeScript 与 Node 的奇怪交互：写 import x from './foo.js'（带 .js 而非 .ts）才能让 tsc 输出后 Node ESM 能跑
  • 这是 TS + ESM 项目最常见的踩坑点

--experimental-vm-modules / --loader hooks

正文待写。自定义模块加载器（用于 ts-node、Jest ESM、Vitest 等）；Node 20+ 改名为 --import + register hook API。

04 典型代码

json 
// 现代库 package.json 双格式发布
{
  "name": "my-lib",
  "type": "module",
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.cjs",
      "default": "./dist/index.js"
    }
  }
}

05 常见陷阱

ESM 里没有 __dirname / __filename

用 import.meta.url + fileURLToPath 替代。

conditional exports 顺序不对会解析错文件

types 必须放最前；import / require 顺序也敏感。

TS 里 import 必须带 .js 不是 .ts

这是 TS5 NodeNext 模式的"路径写运行时实际路径"约定。
