P3 · 模块系统

Node 是 JS 历史上第一个有"真模块系统"的运行时（2009 CommonJS）。15 年后这套系统与 ES Modules 标准并存，造就了 Node 工程最复杂的工程现实——双系统并存 + Dual Package Hazard。

章节数 2 主要来源 nodejs.org/api/esm · nodejs.org/api/packages 状态 ⏳ 待撰写

本阶段覆盖

  • CommonJS 解析算法（Node 2009-） —— require 同步 / Node 自家算法 / node_modules 向上查找
  • ESM 解析（Node 12+） —— import 异步 / 按 ECMA Module Records 规范 / 必须带扩展名
  • package.json 字段全谱 —— type / main / module / exports / imports / types
  • Conditional Exports —— import / require / types / browser / node 等条件 + key 顺序敏感
  • Dual Package Hazard —— require 和 import 加载的是两个独立实例 → 单例失效 / instanceof 失败
  • 3 种规避策略 —— 库纯无状态 / CJS 设为 ESM wrapper / 只发 ESM
  • 现代库设计模板 —— tshy / tsup / unbuild 构建工具 + 双格式发布
  • Top-Level Await —— 仅 ESM 支持，让模块"等待"才完成 evaluation
