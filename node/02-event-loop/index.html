P2 · libuv 与 Event Loop

Node 的核心创新——单线程 + 事件驱动 + 异步 I/O 跑高并发服务端——这套机制由 libuv 提供。两章：先讲 libuv 自己（独立 C 库的设计），再讲 Node 如何在 libuv 上叠 V8 microtask、process.nextTick、unhandled rejection 等运行时语义。

章节数 2 主要来源 docs.libuv.org · nodejs.org/learn/event-loop 状态 ⏳ 待撰写

本阶段覆盖

  • libuv 起源与定位 —— 2011 从 Node ev/eio 抽离 / 2013 独立项目 / 跨平台异步 I/O 库
  • libuv 核心抽象 —— Handle（长寿对象）vs Request（短期一次性）
  • 跨平台 I/O 多路复用 —— Linux epoll / macOS kqueue / Windows IOCP，libuv 给 Node 统一抽象
  • Worker Pool（默认 4 线程） —— 文件 I/O / DNS / 用户 work 走这里；UV_THREADPOOL_SIZE 调整
  • 6 个 Event Loop phase —— timers / pending callbacks / idle, prepare / poll / check / close
  • Node 视角的运行时语义 —— V8 microtask 在 phase 之间 drain、process.nextTick / setImmediate 精确时序
  • 常见坑 —— nextTick 递归饿死后续 phase、setImmediate vs setTimeout(0) 的微妙差异
