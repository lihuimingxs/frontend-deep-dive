Part 2 · Handbook 基础

TypeScript 不是新语言 —— 它是 JavaScript 的「开发期约束系统」。本 Part 的 6 章构成一个完整的心智模型： 类型系统如何工作 → 日常类型工具 → 类型如何随代码流动 → 函数 / 对象 / 类的所有 TS 增强。读完 Part 1，绝大部分日常 TS 代码你都能精确理解为什么这样写。

章节数 6 / 6 ✅ 对应官方 Handbook v2 · 第 1-6 章 前置 JavaScript 基础（推荐先学 JS · P2 高阶语言）

本 Part 解决什么问题

  • 为什么编译通过却线上还会爆炸 —— 第 01 章解释类型擦除、信任边界、strict 档位
  • 什么时候用 type 什么时候用 interface —— 第 02 章给出能力差异决策树
  • 为什么 if (x) 之后 TS 就知道 x 不是 null 了 —— 第 03 章拆解 control flow analysis
  • 函数重载和泛型函数应该怎么写 —— 第 04 章覆盖参数 / 返回值 / this / overload 全谱
  • readonly / index signature / tuple 各自的边界在哪 —— 第 05 章
  • TS class 比 JS class 多了什么 / 哪些是软隐私 / 什么时候用 abstract —— 第 06 章

章节清单

序号 章节 核心收获
01 The Basics 与类型系统本质 类型擦除、值/类型双命名空间、tsc 三阶段、strict 档位、target vs lib 关系
02 Everyday Types 结构化类型、type vs interface、unknown 安全替代 any、literal type、as const
03 Narrowing 与控制流分析 七大 narrowing 手段、discriminated union、never 兜底、TS 5.5 推断式 type predicate
04 Functions 函数类型、overload、泛型函数、this 参数、void / never / unknown 返回类型
05 Object Types readonly 意图标注、index signature、excess property check、tuple、interface vs intersection
06 Classes 软隐私 vs #field、parameter properties、abstract、this 类型、Liskov 在结构化类型下

Part 1 内部依赖关系

建议按顺序阅读 01 → 02 → 03，这三章构成基础心智模型。04 / 05 / 06 各自完备，可独立跳读：

  • 01 → 02 → 03 必须连读：02 引入的 union 类型在 03 才学如何安全使用
  • 04 Functions 依赖 02 的 type alias / interface 知识
  • 05 Object Types 依赖 02 的对象字面量类型
  • 06 Classes 依赖 04 的 method 类型和 05 的 readonly

与 JavaScript 章节的对应

本 Part 大量引用 JS 运行时机制作为底层基础。如果某章读起来有疑惑，先看对应 JS 章节：

  • 01 The Basics → JS 作用域与 TDZ / 闭包内存模型
  • 02 Everyday Types → JS JS 数据类型系统 / 原型链
  • 03 Narrowing → JS async/await 中的闭包陷阱
  • 04 Functions → JS this 四规则
  • 05 Object Types → JS 原型链
  • 06 Classes → JS class 与继承 / 私有字段与装饰器
