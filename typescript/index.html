编写约定

每章遵循与 JavaScript 章节相同的 6 段结构，但 TS 的"底层逻辑"段会重点讨论：

  • 类型系统如何工作 —— 结构化类型（structural typing）vs 名义类型（nominal typing）
  • 类型擦除 —— 编译产物中类型完全消失，运行时类型守卫的工作机制
  • 协变 / 逆变 / 双变 / 不变 —— 函数参数、返回值、数组、Promise 各自的方差规则
  • 类型推导算法 —— 何时窄化、何时扩宽（widening）、字面量类型何时被保留
  • 分布式条件类型 —— T extends U ? X : Y 在 union 上的分发行为

与 JavaScript 章节的关系

TypeScript 是 JavaScript 的 superset，本节会大量交叉引用 JavaScript 深度学习 中的运行时机制章节：

  • TS Classes → JS 原型链 / class 与继承
  • TS Decorators → JS 私有字段与装饰器
  • TS Modules → JS P4 · 模块 & 元编程
  • TS Iterators / JSX → JS 迭代器与 Generator
