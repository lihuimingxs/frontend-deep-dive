16 · Iterators / JSX

两个独立小专题。Iterators 部分讲 ES 迭代协议（Iterable<T> / Iterator<T>）、Generator、async iterator 在 TS 中的类型化，以及 downlevelIteration 配置坑。JSX 部分讲 5 种 jsx 模式（preserve / react / react-jsx / react-native / react-jsxdev）、jsxImportSource 自定义 runtime、JSX.IntrinsicElements augmentation、以及 .tsx 中尖括号断言与 JSX 标签的冲突。

阅读时长 ~22 分钟 前置 06 Classes

01 速览

Iterators 部分核心：

typescript 
// Iterable<T> 是有 [Symbol.iterator]() 方法的对象
// Iterator<T> 是有 next() / return?() / throw?() 方法的对象

// 自定义可迭代对象
class Range implements Iterable<number> {
  constructor(public start: number, public end: number) {}
  *[Symbol.iterator]() {
    for (let i = this.start; i <= this.end; i++) yield i;
  }
}

for (const n of new Range(1, 3)) console.log(n);   // 1, 2, 3

JSX 部分核心：

tsx 
// .tsx 文件支持 JSX 语法
function Hello({ name }: { name: string }) {
  return <div>Hello {name}</div>;
}

// JSX 编译为 React.createElement 或 _jsx 调用
// 由 tsconfig 的 jsx 选项决定

02 底层逻辑

1. ES 迭代协议：Iterable + Iterator 双协议

ES 把"可迭代"拆成两个协议：

  • Iterable<T>：有 [Symbol.iterator](): Iterator<T> 方法
  • Iterator<T>：有 next(): { value: T; done: boolean } 方法
typescript · lib.es2015.iterable.d.ts 
interface IteratorYieldResult<TYield> { done?: false; value: TYield }
interface IteratorReturnResult<TReturn> { done: true; value: TReturn }

interface Iterator<T, TReturn = any, TNext = undefined> {
  next(...args: [] | [TNext]): IteratorYieldResult<T> | IteratorReturnResult<TReturn>;
  return?(value?: TReturn): IteratorYieldResult<T> | IteratorReturnResult<TReturn>;
  throw?(e?: any): IteratorYieldResult<T> | IteratorReturnResult<TReturn>;
}

interface Iterable<T, TReturn = any, TNext = undefined> {
  [Symbol.iterator](): Iterator<T, TReturn, TNext>;
}

// IterableIterator：自身既是 Iterable 又是 Iterator
interface IterableIterator<T> extends Iterator<T> {
  [Symbol.iterator](): IterableIterator<T>;
}

内置可迭代类型：Array / Map / Set / String / TypedArray / NodeList / generator。for...of、扩展运算 ...arr、解构 const [a, b] = arr、Array.from()、Promise.all() 等都依赖此协议。

2. Generator：自动满足两个协议

function* 生成器函数返回 Generator<T, TReturn, TNext>，自动同时满足 Iterable 和 Iterator：

typescript 
function* range(start: number, end: number): Generator<number, void, unknown> {
  for (let i = start; i <= end; i++) yield i;
}

const gen = range(1, 3);

// 用作 Iterator
gen.next();   // { value: 1, done: false }
gen.next();   // { value: 2, done: false }
gen.next();   // { value: 3, done: false }
gen.next();   // { value: undefined, done: true }

// 用作 Iterable
for (const n of range(1, 3)) console.log(n);

// Generator 类型三参数：
// Generator<TYield, TReturn, TNext>
//   TYield  — yield 表达式产生的类型
//   TReturn — return 语句的值类型
//   TNext   — next(x) 传入的参数类型
function* g(): Generator<number, string, boolean> {
  const flag: boolean = yield 1;   // 调用 .next(true) 时 flag 为 true
  yield 2;
  return "done";
}

3. Async iterator / async generator：处理流

异步版本：AsyncIterable<T> / AsyncIterator<T>，next() 返回 Promise<{value, done}>。async function* 自动满足：

typescript 
// 处理 SSE 流
async function* readSSE(url: string): AsyncGenerator<string, void, unknown> {
  const response = await fetch(url);
  const reader = response.body!.getReader();
  const decoder = new TextDecoder();
  while (true) {
    const { done, value } = await reader.read();
    if (done) return;
    yield decoder.decode(value);
  }
}

// 消费用 for await...of
for await (const chunk of readSSE("/events")) {
  console.log(chunk);
}

4. downlevelIteration：target ES5 时的兼容性开关

target 降到 ES5 时，TS 默认假设运行环境没有 Symbol.iterator，所以 for...of 在数组上工作但在 Map / Set 上不行：

typescript 
// tsconfig: { target: "ES5", downlevelIteration: false }
const m = new Map();
m.set("a", 1);

for (const [k, v] of m) { /* ... */ }
// ❌ Type 'Map' is not assignable to 'IterableIterator<[any, any]>'.
// ⚠️ TS 不会替你 emit 对应的 polyfill 代码

// 方案 A：开 downlevelIteration
// tsconfig: { downlevelIteration: true }
// → TS emit 完整的 Symbol.iterator 兼容代码

// 方案 B：把 target 拉高到 ES2015+（推荐）
// → 所有 ES2015 迭代特性原生支持，产物更小

实践：现代项目永远 target: "ES2017" 或更高（覆盖 95%+ 浏览器），不需要这个开关。仅旧 IE 项目才需要 downlevelIteration。

5. JSX：把 XML 字面量编译为函数调用

JSX 不是 TS 特有，是 React 团队提的语法扩展。它把 <div /> 这样的 XML 字面量编译为函数调用。TS 通过 .tsx 扩展名 + jsx tsconfig 选项支持它。

jsx 选项的 5 种模式：

模式编译结果用途
"preserve" 保持 JSX 不编译 babel / swc / Vite 后续处理
"react" React.createElement(...) 旧 React（< 17）
"react-jsx" _jsx(...) 自动 import React 17+ 推荐
"react-jsxdev" _jsxDEV(...) 含位置信息 React 17+ dev 模式
"react-native" 类似 preserve（RN 自己处理） React Native
tsx · 源 
function Hello({ name }: { name: string }) {
  return <div className="greeting">Hello {name}</div>;
}
javascript · jsx: react（旧） 
import React from "react";   // 必须显式 import React
function Hello({ name }) {
  return React.createElement("div", { className: "greeting" }, "Hello ", name);
}
javascript · jsx: react-jsx（新） 
import { jsx as _jsx } from "react/jsx-runtime";   // ← 编译器自动 import
function Hello({ name }) {
  return _jsx("div", { className: "greeting", children: ["Hello ", name] });
}
// 不再要求 import React！

6. jsxImportSource：自定义 JSX runtime

配合 jsx: "react-jsx"，可以指定从哪个包 import jsx-runtime。这让 emotion / theme-ui / preact 等库能"接管" JSX：

json · tsconfig 
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "@emotion/react"
    // 编译产物：import { jsx as _jsx } from "@emotion/react/jsx-runtime"
  }
}

在文件级别覆盖：

tsx 
/** @jsxImportSource @emotion/react */

function Hello() {
  return <div css={{ color: "red" }}>Hi</div>;   // emotion 的 css prop
}

7. JSX.IntrinsicElements 与 augmentation

TS 通过 global namespace JSX 来定义 JSX 类型。JSX.IntrinsicElements 列出所有可用的"小写元素"（HTML 标签）。

typescript · 简化版（实际由 @types/react 提供） 
declare global {
  namespace JSX {
    interface IntrinsicElements {
      div: { className?: string; children?: ReactNode; ... };
      a: { href?: string; ... };
      // ... 100+ HTML 元素
    }
    interface Element { ... }       // JSX 表达式的返回类型
    interface ElementAttributesProperty { props: {} }   // 决定从 prop 取
    interface ElementChildrenAttribute { children: {} } // 决定 children prop 名
  }
}

要给 JSX 加自定义元素（如 web components），用 module augmentation：

typescript 
declare global {
  namespace JSX {
    interface IntrinsicElements {
      "my-custom-element": {
        attr1?: string;
        attr2?: number;
        children?: any;
      };
    }
  }
}

export {};

8. .tsx 中尖括号断言与 JSX 冲突

02 章讲过 <T>value 是旧式断言语法。在 .tsx 文件里它与 JSX 标签冲突：

tsx 
// ❌ 在 .tsx 中
const x = <string>value;   // TS 看这是 JSX 标签 <string>...
//                          ❌ JSX element 'string' has no corresponding closing tag

// ✅ .tsx 中只能用 as
const x = value as string;

// 旧式 .ts 中两种都能用
const y = <string>value;   // ✓ 仅 .ts
const z = value as string;  // ✓ 推荐

规则：永远用 as T。<T>value 是已淘汰语法。

03 应用场景

场景 A · 自定义可迭代对象（分页 / 流式 API）

typescript 
// 自动分页迭代器
async function* paginate<T>(
  fetchPage: (cursor?: string) => Promise<{ items: T[]; nextCursor?: string }>,
): AsyncGenerator<T> {
  let cursor: string | undefined;
  do {
    const { items, nextCursor } = await fetchPage(cursor);
    for (const item of items) yield item;
    cursor = nextCursor;
  } while (cursor);
}

// 调用方一行消费
for await (const user of paginate(c => api.listUsers({ cursor: c }))) {
  console.log(user);
}

场景 B · async generator 处理 LLM SSE 流

typescript 
async function* streamLLM(prompt: string): AsyncGenerator<string> {
  const res = await fetch("/api/llm", {
    method: "POST",
    body: JSON.stringify({ prompt, stream: true }),
  });
  const reader = res.body!.getReader();
  const decoder = new TextDecoder();
  let buffer = "";
  while (true) {
    const { done, value } = await reader.read();
    if (done) return;
    buffer += decoder.decode(value);
    const lines = buffer.split("\n");
    buffer = lines.pop() ?? "";
    for (const line of lines) {
      if (line.startsWith("data: ")) {
        const data = JSON.parse(line.slice(6));
        if (data.text) yield data.text;
      }
    }
  }
}

// 在 React 组件里消费
async function chat() {
  let answer = "";
  for await (const chunk of streamLLM("hello")) {
    answer += chunk;
    setMessage(answer);   // 增量更新 UI
  }
}

04 典型代码

三种 yield 形态

typescript 
function* g(): Generator<number, string, boolean> {
  // 简单 yield
  yield 1;

  // 接收 next() 传入的值
  const flag: boolean = yield 2;
  console.log(flag);   // 调用 .next(true) 时为 true

  // yield* 委托其他 generator
  yield* otherGen();

  // return 设置 done: true 时的 value
  return "done";
}

function* otherGen() {
  yield 100;
  yield 200;
}

const it = g();
it.next();        // { value: 1, done: false }
it.next();        // { value: 2, done: false }
it.next(true);    // 内部 flag = true，{ value: 100, done: false }
it.next();        // { value: 200, done: false }
it.next();        // { value: "done", done: true }

双向通信 generator（自动机模式）

typescript 
function* stateMachine(): Generator<
  string,    // yield 类型 — 当前 state 名
  void,      // return 类型
  string     // next 传入 — 事件名
> {
  let state = "idle";
  while (true) {
    const event = yield state;
    if (state === "idle" && event === "start") state = "running";
    else if (state === "running" && event === "stop") state = "idle";
  }
}

const sm = stateMachine();
sm.next();          // { value: "idle" }
sm.next("start");   // { value: "running" }
sm.next("stop");    // { value: "idle" }

JSX 全套类型导入

tsx 
import {
  ReactNode,             // 任意可渲染节点（含 string / number / null / Element / array）
  ReactElement,           // JSX 表达式返回的具体 element
  ComponentProps,         // 取某组件 props
  ComponentPropsWithRef,
  ComponentPropsWithoutRef,
  ChangeEvent,            // 事件类型
  MouseEvent,
  FormEvent,
  KeyboardEvent,
  CSSProperties,          // style prop 类型
  Ref,                    // useRef 返回的 ref
  HTMLAttributes,         // 通用 HTML attrs
  DetailedHTMLProps,
} from "react";

interface ButtonProps extends ComponentPropsWithoutRef<"button"> {
  variant: "primary" | "secondary";
}

function Button({ variant, ...rest }: ButtonProps) {
  return <button type="button" {...rest} className={`btn-${variant}`} />;
}

const handler = (e: ChangeEvent<HTMLInputElement>) => {
  e.target.value;   // ✓
};

JSX.Element vs ReactNode 的区别

tsx 
// JSX.Element / ReactElement — 必须是单个 JSX 表达式
function Foo(): JSX.Element {
  return <div />;
}

// ReactNode — 任何可渲染（string / null / number / array / fragment）
function Bar({ children }: { children: ReactNode }) {
  return <div>{children}</div>;
}

// 一般 children 用 ReactNode（更宽容）；
// 函数返回值用 ReactNode 或 JSX.Element 都行
function App() {
  return (
    <Bar>
      hello                   {/* string */}
      {42}                     {/* number */}
      {null}                   {/* null */}
      <div>nested</div>        {/* JSX.Element */}
      {[1, 2, 3].map(n => <span key={n}>{n}</span>)}   {/* array */}
    </Bar>
  );
}

Iterable<T> 作为函数参数

typescript 
// 接受任何可迭代对象
function sum(nums: Iterable<number>): number {
  let total = 0;
  for (const n of nums) total += n;
  return total;
}

sum([1, 2, 3]);                     // ✓ Array
sum(new Set([1, 2, 3]));            // ✓ Set
sum(new Map().keys());              // ❌ keys 是 string[]
sum((function* () { yield 1; })()); // ✓ Generator

// 生成器作为参数（更广义）
function consumeAsync<T>(it: AsyncIterable<T>) {
  for await (const item of it) console.log(item);
}

05 常见陷阱

陷阱 1 · for...in 在 array 上枚举字符串 key
typescript 
const arr = ["a", "b", "c"];

// ❌ for...in 枚举的是 key
for (const k in arr) {
  console.log(k);   // "0", "1", "2"  字符串！
  console.log(arr[k]);   // 通过但语义诡异
}

// ✅ for...of 才是值
for (const v of arr) console.log(v);   // "a", "b", "c"

// ✅ 要 index：entries
for (const [i, v] of arr.entries()) console.log(i, v);

规则：数组永远用 for...of，对象自有 key 用 Object.keys / entries。for...in 几乎不该出现在新代码里。

陷阱 2 · target ES5 不开 downlevelIteration 时 spread / for...of 失败
typescript 
// target: ES5
const m = new Map();
const arr = [...m];   // ❌ Type 'Map' is not array type

// 数组虽然能 for...of 但 Map / Set / Generator 不行
// 解：开 downlevelIteration（产物变大）或拉高 target
陷阱 3 · .tsx 中尖括号被理解为 JSX
tsx 
const x = <string>value;          // ❌ JSX 标签

const a = arr.map<string>(...);    // ✓ 这里是泛型实参，OK
const fn = <T,>(x: T) => x;        // ✓ 加逗号让 TS 区分泛型 vs JSX
const fn2 = <T extends {}>(x: T) => x;   // ✓ 加约束也行

规避：.tsx 中

  • 断言永远用 as T
  • 箭头函数泛型加逗号 <T,> 或加约束 <T extends {}>
陷阱 4 · jsx mode 选错
项目推荐 jsx
Vite / Next.js（让 bundler 处理）preserve 或 react-jsx
纯 tsc 编译，React 17+react-jsx
纯 tsc 编译，React 16 或更早react
React Nativereact-native
开发模式（要 source 行号信息）react-jsxdev

规避：模板项目（create-vite / create-next-app）已经配好，新项目从模板起步即可。

陷阱 5 · async generator 的 throw 处理
typescript 
async function* readStream(): AsyncGenerator<string> {
  try {
    yield "first";
    yield "second";
  } finally {
    console.log("cleanup");   // 仅在 done 或 throw / return 时跑
  }
}

const it = readStream();
await it.next();   // first
await it.return(); // 提前结束 — finally 会跑

// for await...of 中 break 会自动调 .return()，触发 finally
for await (const x of readStream()) {
  if (x === "first") break;   // → 自动 cleanup
}

生成器的 finally 是清理资源的好地方（关闭文件 / 取消订阅 / 释放锁），但要注意它仅在被消费完 / 提前 return / throw 时才跑 — 没人消费它就一直挂着。

陷阱 6 · React.FC 已被官方"温和劝退"
tsx 
// 老写法（不推荐，但仍可用）
const Hello: React.FC<{ name: string }> = ({ name }) => <div>{name}</div>;

// 新写法 — 直接函数 + props 类型
function Hello({ name }: { name: string }) {
  return <div>{name}</div>;
}

// 或匿名箭头
const Hello = ({ name }: { name: string }) => <div>{name}</div>;

React 团队在 18+ 文档中倾向直接函数 + props 类型注解。React.FC 的隐式 children 在 React 18 已移除。
