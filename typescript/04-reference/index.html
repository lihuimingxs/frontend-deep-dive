Part 4 · Reference

这一 Part 是 TS 文档中"每个开发者都会用到，但深度讲解少"的话题。包括 Modules（CJS/ESM 互通）、Decorators（Stage 3 vs legacy）、Declaration Merging（给第三方库打类型补丁的杀手锏）、Iterators 与 JSX 配置、以及 Type Compatibility（协变/逆变元层）。最后一章 17 是 Part 3 最难，理解后回头看 Part 2 也会更深。

章节数 5 章 对应官方 Handbook · Reference 前置 Part 1 + Part 2

本 Part 解决什么问题

  • 怎样让 TS 项目正确识别 ESM / CJS 边界 —— 13 Modules
  • NestJS 的 @Decorator 和 TS 5.0 标准 decorator 是同一回事吗 —— 14 Decorators
  • 怎么给 Express Request 加 user 字段而不改源代码 —— 15 Declaration Merging
  • JSX runtime 配置和 jsxImportSource 是干嘛的 —— 16 Iterators / JSX
  • 为什么 (x: Animal) => void 不能赋给 (x: Dog) => void —— 17 Type Compatibility（最深）

章节清单

序号章节核心收获
13 Modules ES module 静态结构、import type、esModuleInterop、verbatimModuleSyntax、namespace 反模式
14 Decorators Stage 3 标准 vs legacy、context 对象、求值顺序、reflect-metadata、为什么 NestJS 仍用 legacy
15 Declaration Merging / Mixins / Symbols module augmentation 实战、mixin pattern、unique symbol、well-known symbol
16 Iterators / JSX 迭代协议、generator、JSX 5 种模式、jsxImportSource、JSX.IntrinsicElements augmentation
17 Type Compatibility & Inference 结构化兼容、协变/逆变/双变/不变、strictFunctionTypes、contextual typing、best common type

难度提示

  • 13 Modules —— 中等。CJS/ESM 互通是工程话题，需要配合 19 章 tsconfig 一起读
  • 14 Decorators ⚠️ —— TS 改动最剧烈的领域。同一份知识在不同年代答案不同，要分清 Stage 2 / Stage 3
  • 15 Declaration Merging —— 中等。一旦学会，给第三方库加 Patch 这件事永远不再是问题
  • 16 Iterators / JSX —— 偏轻。两个独立小专题
  • 17 Type Compatibility ⚠️ —— 本 Part 最难。协变/逆变是函数式 / 库作者必须掌握的元能力
