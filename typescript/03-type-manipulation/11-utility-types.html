11 · Utility Types

TS 标准库提供了 22+ 个常用 utility types，分四个家族：属性变换 / union 操作 / 函数操作 / 字符串。每一个都能用 09/10 章的 conditional + mapped + template literal 工具复现 —— 读它们的源码（lib.es5.d.ts 几百行）是学 TS 类型操作最有效的方式。本章按家族过一遍每个 utility 的实现剖析、应用场景、限制，重点拆 Awaited 递归、NoInfer 推断阻断、ThisType 上下文施加这三个最不直觉的工具。

阅读时长 ~30 分钟 前置 09 Conditional + 10 Mapped 对应 typescriptlang.org Utility Types

01 速览

02 底层逻辑

属性变换族 —— 6 个一行搞定

typescript · lib.es5.d.ts 
type Partial<T> = { [P in keyof T]?: T[P] };
type Required<T> = { [P in keyof T]-?: T[P] };
type Readonly<T> = { readonly [P in keyof T]: T[P] };
type Pick<T, K extends keyof T> = { [P in K]: T[P] };
type Record<K extends keyof any, T> = { [P in K]: T };

// Omit 借助 Exclude 实现（5.0 之前更复杂）
type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;

注意几个细节：

  • Pick 的 K 受约束 K extends keyof T —— 防止指定不存在的字段
  • Omit 的 K 是 keyof any（即 string | number | symbol），不受 keyof T 约束 —— 这是 TS 故意为之，让 Omit 兼容更多场景（详见陷阱 1）
  • Record 用 K extends keyof any，所以 Record<"a" | "b", number>、Record<number, string> 都合法
  • Required 用 -? 显式去掉 optional；Readonly 用默认 readonly（即 +readonly）

union 操作族 —— 依赖分发性

typescript 
type Exclude<T, U> = T extends U ? never : T;
type Extract<T, U> = T extends U ? T : never;
type NonNullable<T> = T & {};   // 5.0+ 简化版（之前用 Exclude<T, null | undefined>）

Exclude / Extract 都依赖 09 章讲过的 distributive。T 是裸类型参数，union 自动展开。

NonNullable 在 TS 5.0 改成 T & {}（即与"任意非 null/undefined 值类型"求交）。这等价于 Exclude<T, null | undefined>，但更简洁、性能更好。

函数操作族 —— 全靠 infer

typescript 
type Parameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;

type ConstructorParameters<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: infer P) => any ? P : never;

type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : any;

type InstanceType<T extends abstract new (...args: any) => any> =
  T extends abstract new (...args: any) => infer R ? R : any;

type ThisParameterType<T> =
  T extends (this: infer U, ...args: never) => any ? U : unknown;

type OmitThisParameter<T> =
  unknown extends ThisParameterType<T> ? T :
  T extends (...args: infer A) => infer R ? (...args: A) => R : T;

注意 abstract new (...) 的写法 —— TS 4.2+ 引入，让 ConstructorParameters / InstanceType 也能用于 abstract class（06 章讲过）。

Awaited<T> —— 递归解 Promise / thenable

这是用 conditional + recursion 实现 deep unwrap 的最佳示例：

typescript · lib.es5.d.ts 
type Awaited<T> =
  T extends null | undefined ? T :                         // 1. null/undefined 不变
  T extends object & { then(onfulfilled: infer F, ...args: infer _): any } ?
    F extends ((value: infer V, ...args: infer _) => any) ?
      Awaited<V> :                                          // 2. thenable —> 递归 unwrap
      never :
    T;                                                       // 3. 其他原样返回

为什么这么复杂？因为 Promise 不是唯一的 thenable —— TS 用 { then(...): any } 这种 duck typing 匹配所有"长得像 Promise"的对象。

typescript 
type R1 = Awaited<Promise<string>>;             // string
type R2 = Awaited<Promise<Promise<number>>>;    // number   ← 递归
type R3 = Awaited<number>;                       // number   ← 不是 thenable
type R4 = Awaited<{ then: (cb: (v: boolean) => void) => void }>;   // boolean   ← duck typed thenable

// async function 推 ReturnType 自动 unwrap
async function fetchUser(): Promise<User> { /* ... */ return {} as User }
type R = Awaited<ReturnType<typeof fetchUser>>;   // User

NoInfer<T>（TS 5.4）—— 阻止位置参与推断

07 章已经讲过 NoInfer 的应用。这里看实现：

typescript · lib.es5.d.ts 
// 5.4+ 标准实现（intrinsic — 编译器特殊处理）
type NoInfer<T> = intrinsic;

这又是一个 intrinsic（编译器内置）—— 它不能用普通 conditional + mapped 实现。语义：在类型推断算法中跳过这个位置，让另一个位置主导。

typescript 
// 经典场景：函数有"主参 + 副参"，希望主参推断 T
function configure<T extends string>(
  initial: T,                            // T 在此推断
  validators: NoInfer<T>[],              // 此处不参与，仅检查
): T {
  return initial;
}

// initial 主导 T = "ready"
const x = configure("ready", ["ready", "loading", "done"]);
//                                        ^^^^^^^^^^^^^^^^^^^^^
//                                        Argument of type '"loading"' not assignable to '"ready"'
// 因为 T 已锁为 "ready"，validators 数组只接受 "ready"

ThisType<T> —— 给对象字面量"标注"上下文 this

这是 utility 中最不直觉的一个 —— 它本身没有任何"形状要求"，但出现在对象字面量类型的 intersection 中时，TS 会用它推断对象内方法的 this 类型。

typescript 
interface VueLikeOptions<D, M> {
  data(): D;
  methods: M & ThisType<D & M>;   // 关键：ThisType 让 methods 的 this 推断为 D & M
}

declare function makeVue<D, M>(opts: VueLikeOptions<D, M>): D & M;

const app = makeVue({
  data() {
    return { count: 0 };
  },
  methods: {
    increment() {
      this.count++;        // ✓ this: { count: number } & { increment(): void }
    },
    log() {
      console.log(this.count);   // ✓ count 自动可见
    },
  },
});

要"--noImplicitThis 配合"才生效。Vue 2 的 Options API 类型支持就靠它。

字符串 utility —— 全是 intrinsic

typescript 
type Uppercase<S extends string> = intrinsic;
type Lowercase<S extends string> = intrinsic;
type Capitalize<S extends string> = intrinsic;
type Uncapitalize<S extends string> = intrinsic;

它们也无法用 mapped + template literal 写出来 —— 字符的"大小写转换"在 TS 类型层不是普通操作。运行时没有对应函数（"abc".toUpperCase() 需要自己写）。

03 应用场景

场景 A · DTO 转换（数据库实体 ↔ API 形状）

typescript 
interface User {
  id: string;
  email: string;
  name: string;
  passwordHash: string;
  createdAt: Date;
  updatedAt: Date;
}

// 创建：不需要数据库自动生成的字段
type CreateUserInput = Omit<User, "id" | "createdAt" | "updatedAt">;

// 更新：除 id 外可选
type UpdateUserInput = Partial<Omit<User, "id" | "createdAt" | "updatedAt">>;

// 公共：不能暴露 passwordHash
type PublicUser = Omit<User, "passwordHash">;

// API 响应 list：只返回 id 和 name
type UserListItem = Pick<User, "id" | "name">;

// 全 readonly 版本（防止层间被改）
type ReadonlyUser = Readonly<User>;

场景 B · React Props 派生

typescript 
// 把原生 button 的所有 props 拿来 + 加自定义 variant
type ButtonProps = ComponentPropsWithoutRef<"button"> & {
  variant: "primary" | "secondary";
  size?: "sm" | "md" | "lg";
};

// 也可以反向：从一个 React 组件取 props
import { Button } from "./Button";
type Props = ComponentProps<typeof Button>;

// 取 onClick 的事件类型
type ClickHandler = NonNullable<ButtonProps["onClick"]>;
//   ^? (e: MouseEvent<HTMLButtonElement>) => void
type ClickEvent = Parameters<ClickHandler>[0];
//   ^? MouseEvent<HTMLButtonElement>

场景 C · Reducer / Action 类型派生

typescript 
// 从已有 reducer 函数反推 state / action 类型
function reducer(state: { count: number }, action: { type: "inc" } | { type: "set"; value: number }) {
  switch (action.type) {
    case "inc": return { count: state.count + 1 };
    case "set": return { count: action.value };
  }
}

type State = Parameters<typeof reducer>[0];     // { count: number }
type Action = Parameters<typeof reducer>[1];    // { type: "inc" } | { type: "set"; value: number }
type Result = ReturnType<typeof reducer>;       // { count: number }

// 进一步：从 Action union 取出特定 action 的 payload
type SetAction = Extract<Action, { type: "set" }>;   // { type: "set"; value: number }

场景 D · 第三方库返回类型反推

typescript 
// drizzle-orm 的 query builder
import { db, users } from "./schema";

const findUsers = () => db.select().from(users).where(...).execute();
type UserList = Awaited<ReturnType<typeof findUsers>>;
//   ^? User[]   自动跟随 schema 变化

// 第三方 SDK 没导出某个返回类型
declare const stripe: { charges: { create(opts: any): Promise<{ id: string; amount: number }> } };

type ChargeResult = Awaited<ReturnType<typeof stripe.charges.create>>;
//   ^? { id: string; amount: number }

04 典型代码

常用组合模式

typescript 
// Partial + Pick
type DraftUser = Partial<Pick<User, "name" | "email">>;

// Required + Pick
type RequiredFields = Required<Pick<User, "id" | "email">>;

// Readonly + Omit
type ImmutableNoTimestamps = Readonly<Omit<User, "createdAt" | "updatedAt">>;

// Pick + Record（构造从 keys 派生的对象）
type Errors<K extends string> = Record<K, string | undefined>;
type FormErrors = Errors<"name" | "email" | "password">;
//   ^? { name: string | undefined; email: string | undefined; password: string | undefined }

// 反向 NonNullable
type StrictUser = { [K in keyof User]: NonNullable<User[K]> };

构造类型安全的 form helper

typescript 
function createForm<T extends Record<string, unknown>>(initialValues: T) {
  return {
    initialValues,
    // 提交需要全部字段
    onSubmit: (cb: (values: Required<T>) => void) => cb,
    // 重置只接受 Partial
    setValues: (values: Partial<T>) => values,
    // 错误对象的 key 集合就是表单字段
    errors: {} as Partial<Record<keyof T, string>>,
  };
}

const form = createForm({ name: "", age: 0 });
form.setValues({ name: "Alice" });   // ✓ Partial OK
form.errors;                          // { name?: string; age?: string }

函数操作族实战：包装 fetch 自动派生类型

typescript 
// 给现有 API client 加 cache 装饰
function withCache<F extends (...args: any[]) => Promise<any>>(fn: F): F {
  const cache = new Map<string, Awaited<ReturnType<F>>>();
  //                                  ^^^^^^^^^^^^^^^^^^^^^   反推内部值类型
  return ((...args: Parameters<F>) => {
    const key = JSON.stringify(args);
    if (cache.has(key)) return Promise.resolve(cache.get(key));
    return fn(...args).then(v => {
      cache.set(key, v);
      return v;
    });
  }) as F;
}

declare function fetchUser(id: string): Promise<User>;
const cached = withCache(fetchUser);
const u = await cached("u_1");   // u: User

InstanceType 用于工厂函数

typescript 
function factory<C extends new (...args: any[]) => any>(
  Ctor: C,
  ...args: ConstructorParameters<C>
): InstanceType<C> {
  return new Ctor(...args);
}

class User { constructor(public name: string) {} }
class Post { constructor(public title: string, public body: string) {} }

const u = factory(User, "Alice");                  // u: User
const p = factory(Post, "Hello", "World");         // p: Post
factory(User, "Alice", "extra");                   // ❌ 多了参数

可选链 +Required 处理可选树

typescript 
interface Config {
  api?: {
    url?: string;
    timeout?: number;
  };
}

// "处理"过程：把可能 undefined 的字段填默认值，输出严格类型
function normalize(c: Config): Required<{ api: Required<NonNullable<Config["api"]>> }> {
  return {
    api: {
      url: c.api?.url ?? "https://default",
      timeout: c.api?.timeout ?? 5000,
    },
  };
}

const cfg = normalize({});
cfg.api.url;        // ✓ string（不再 undefined）
cfg.api.timeout;    // ✓ number

05 常见陷阱

陷阱 1 · Omit 不要求 K 是 keyof T
typescript 
interface User { id: string; name: string }

// ❌ 拼错字段名也不报
type Bad = Omit<User, "naem">;
//   ^? { id: string; name: string }   "naem" 不存在但 TS 不报

// 这是设计如此 — 让 Omit 兼容更广
// 但你失去了一道防错网

规避：写一个严格的 StrictOmit：

typescript 
type StrictOmit<T, K extends keyof T> = Omit<T, K>;

type Bad = StrictOmit<User, "naem">;
//   ^^^^^^^^^^^^^^^^^^^^^^^^^^^^   ❌ '"naem"' is not assignable to 'keyof User'
陷阱 2 · Partial / Required / Readonly 不递归
typescript 
interface Nested {
  user: { name: string };
}

type R = Required<Nested>;
//   ^? { user: { name: string } }   外层 user 必填，内层 name 仍按原样

type P = Partial<Nested>;
//   ^? { user?: { name: string } }   外层 user 可选，内层 name 仍必填

type O = Readonly<Nested>;
//   ^? { readonly user: { name: string } }   外层 readonly，内层 name 仍可改

规避：自己写递归版（10 章），或用 type-fest 的 PartialDeep / ReadonlyDeep。

陷阱 3 · ReturnType 在重载函数上只取最后一个签名
typescript 
function fn(x: string): string;
function fn(x: number): number;
function fn(x: any): any { return x }

type R = ReturnType<typeof fn>;
//   ^? any   ← 只取了"实现签名"的返回值（更宽）

根因：04 章讲过 overload 的"实现签名宽松，对外签名精确"。ReturnType 只看到最后一个签名（实现的）。

规避：尽量用 union 参数代替 overload；或者用 type-fest 的 OverloadReturnType。

陷阱 4 · Awaited 仍受递归深度限制
typescript 
// 几层 Promise 嵌套都 OK
type R1 = Awaited<Promise<Promise<Promise<number>>>>;   // number

// 但循环引用会爆栈
interface Circular {
  then(cb: (v: Circular) => void): void;
}
type R2 = Awaited<Circular>;
//   ^? Circular 自身 — TS 检测到循环，停止递归
陷阱 5 · ThisType 必须配合 noImplicitThis
typescript 
// 没开 noImplicitThis 时，对象字面量内 this 推断为 any
interface Opts {
  data: number;
  methods: { increment(): void } & ThisType<Opts>;
}

const o: Opts = {
  data: 1,
  methods: {
    increment() { this.data++ }   // ❌ 'this' implicitly has type 'any'（如果没开 noImplicitThis）
  },
};

规避：tsconfig 里启用 "noImplicitThis": true（包含在 strict）。

陷阱 6 · Pick 把 method 也当字段
typescript 
class User {
  name = "";
  greet() { return `Hi ${this.name}` }
}

// Pick 不区分字段和方法
type R = Pick<User, "greet">;
//   ^? { greet: () => string }

// 想"只 Pick 数据字段"需要先过滤
type DataOnly<T> = Pick<T, { [K in keyof T]: T[K] extends Function ? never : K }[keyof T]>;
type R2 = DataOnly<User>;
//   ^? { name: string }
