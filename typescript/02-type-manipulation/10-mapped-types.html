10 · Mapped Types & Template Literal Types

Mapped types 是类型层的"循环"——遍历一个对象的每个 key 做映射；template literal types 是类型层的"字符串拼接"。两者配合 09 章的 conditional types 构成 TS 最强大的类型代码生成器：从一个简单类型自动派生出 getter / setter 全套、从配置 schema 派生出表单字段、从路由模板派生参数对象。本章拆解这两个工具的修饰符语义、key remapping、union 笛卡尔积、内置 intrinsic 字符串类型，并演示它们组合的"代码生成"能力。

阅读时长 ~35 分钟 前置 08 keyof/typeof + 09 Conditional 对应 typescriptlang.org Mapped Types + Template Literal

01 速览

Mapped type 最小例子：

typescript 
// 把对象 T 的每个字段都变 readonly
type Readonly<T> = { readonly [K in keyof T]: T[K] };

// 把对象 T 的每个字段都变 optional
type Partial<T> = { [K in keyof T]?: T[K] };

// Pick：只保留指定 key
type Pick<T, K extends keyof T> = { [P in K]: T[P] };

Template literal type 最小例子：

typescript 
type Greeting = `Hello, ${string}`;
const a: Greeting = "Hello, world";   // ✓
const b: Greeting = "Hi";              // ❌

// Union 在插值位置 → 笛卡尔积
type Color = "red" | "blue";
type Shade = 100 | 500;
type Class = `bg-${Color}-${Shade}`;
//   ^? "bg-red-100" | "bg-red-500" | "bg-blue-100" | "bg-blue-500"

组合（key remapping，TS 4.1+）：

typescript 
// 把 { name, age } 变成 { getName, getAge }
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
};

interface User { name: string; age: number }
type UserGetters = Getters<User>;
//   ^? { getName: () => string; getAge: () => number }

02 底层逻辑

1. Mapped type 是类型层的 Object.fromEntries(...)

想象 mapped type 是一个类型层的循环：对 keyof T 的每个 key 跑一次，输出 key + value。运行时等价物：

typescript · 类比 
// 类型层
type Mapped<T> = { [K in keyof T]: T[K] };

// 运行时等价
function mapped<T extends object>(o: T): T {
  return Object.fromEntries(
    Object.keys(o).map(k => [k, (o as any)[k]])
  ) as T;
}

Mapped type 的语法分三部分：

  1. 修饰符：readonly / ? + 加减号 +/-
  2. [K in Keys]：枚举 key 来源（通常 keyof T）
  3. value 类型：可以引用 K，通常 T[K]

2. Modifier 的 + / - 语义

mapped type 不只能"加"修饰符，还能"减"。这点常被忽略：

typescript 
// 加 readonly（+ 是默认，可省）
type Readonly<T> = { +readonly [K in keyof T]: T[K] };
type Readonly<T> = { readonly [K in keyof T]: T[K] };  // 等价

// 加 optional
type Partial<T> = { [K in keyof T]+?: T[K] };
type Partial<T> = { [K in keyof T]?: T[K] };  // 等价

// 减 readonly — 这就是 Mutable
type Mutable<T> = { -readonly [K in keyof T]: T[K] };

// 减 optional — 这就是 Required
type Required<T> = { [K in keyof T]-?: T[K] };

interface User {
  readonly id: string;
  name?: string;
}

type M = Mutable<User>;
//   ^? { id: string; name?: string }   id 不再 readonly

type R = Required<User>;
//   ^? { readonly id: string; name: string }   name 不再 optional

关键认知：mapped type 默认会"带过"原类型的 readonly / ? 修饰符。所以 Partial<Readonly<T>> 仍然是 readonly 的；要去掉必须显式 -readonly。

3. Key remapping as（TS 4.1+）：循环输出可改 key

没有 key remapping 时，mapped type 的输出 key 必须等于输入 key（只能改 value 类型）。as 让输出 key 也可计算：

typescript 
// 老语法（无 remapping）
type Same<T> = { [K in keyof T]: T[K] };
//                              ^   K 直接当输出 key

// 新语法（带 as）
type Renamed<T> = { [K in keyof T as `prefix_${string & K}`]: T[K] };
//                                ^^^^^^^^^^^^^^^^^^^^^^^^^^^^   重命名

interface User { name: string; age: number }
type R = Renamed<User>;
//   ^? { prefix_name: string; prefix_age: number }

关键能力：as 后面可以是任意类型表达式。配合 conditional 可以过滤 key（让某些 key 输出 never 即被丢弃）：

typescript 
// 只保留 string 字段
type StringFields<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K]
};

interface Mixed { name: string; age: number; email: string }
type R = StringFields<Mixed>;
//   ^? { name: string; email: string }   age 被 never 过滤掉

这是写"选出符合某条件的字段"的标准 idiom。lib.es5.d.ts 里 Pick / Omit 的现代实现都依赖这种过滤。

4. 为什么 key remapping 需要 string & K

keyof T 的类型是 string | number | symbol（即所有可能 key）。但 Capitalize 等内置只接受 string。所以需要"把 K 收窄到 string 的部分"：

typescript 
// ❌ 直接 Capitalize<K>
type Bad<T> = { [K in keyof T as `get${Capitalize<K>}`]: () => T[K] };
//                                              ^   Type 'K' does not satisfy the constraint 'string'

// ✅ string & K：取 K 中是 string 的部分
type Good<T> = { [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K] };

// 或更精确：约束 T 的 key 只能是 string
type Good2<T extends Record<string, unknown>> = {
  [K in keyof T as `get${Capitalize<K extends string ? K : never>}`]: () => T[K]
};

string & K 是 intersection —— 把 K（可能是 string|number|symbol）与 string 取交集，结果是 K 中的 string 部分。number / symbol 部分变 never，被自动过滤。

5. Template literal type：类型层的字符串模板

Template literal types 让你在类型层做字符串拼接。语法和 JS 模板字符串相同：

typescript 
type World = "world";
type Greet = `Hello, ${World}!`;        // "Hello, world!"

// 占位符可以是 string / number / boolean / bigint 字面量
type N1 = `value: ${42}`;                // "value: 42"
type N2 = `enabled: ${true}`;            // "enabled: true"

// 占位符可以是 string / number / bigint / boolean（接受任意字符串/数字/布尔）
type Fmt<S extends string> = `[${S}]`;
type R = Fmt<"abc">;                     // "[abc]"

6. Union 在插值位置 → 笛卡尔积

这是 template literal 最强大也最危险的特性：union 在插值位置自动展开成所有组合。

typescript 
type Color = "red" | "blue" | "green";
type Shade = 100 | 500 | 900;

type ClassName = `bg-${Color}-${Shade}`;
//   ^? "bg-red-100" | "bg-red-500" | "bg-red-900"
//      | "bg-blue-100" | ... | "bg-green-900"   3×3 = 9 种组合

组合数 = 各位置 union size 的乘积。3 × 3 还好，但：

  • 10 个 color × 10 个 shade × 5 个 prefix = 500 个
  • 20 × 20 × 20 = 8000 个 — 编译器开始喘
  • 30 × 30 × 30 = 27000 个 — TS 直接拒绝展开（"Expression produces a union type that is too complex to represent"）

规避：组合大集合时改用 string 做约束（不展开），用 ${string} 占位符匹配任意字符串。例如 Tailwind 的实际类型用 string 而不是穷举所有 class。

7. 内置字符串 utility 是 intrinsic

Uppercase / Lowercase / Capitalize / Uncapitalize 这四个内置类型是编译器特殊实现（intrinsic），不是普通 mapped type 能写出来的。

typescript 
type A = Uppercase<"hello">;        // "HELLO"
type B = Lowercase<"WORLD">;        // "world"
type C = Capitalize<"alice">;       // "Alice"
type D = Uncapitalize<"Hello">;     // "hello"

// 它们的"实现"看起来很奇怪
// type Uppercase<S extends string> = intrinsic;

// 配合 template literal 做转换
type GetterName<K extends string> = `get${Capitalize<K>}`;
type R = GetterName<"name">;         // "getName"

这四个 intrinsic 在运行时没有对应函数—— 它们是纯类型工具。要在运行时做大小写转换仍要写 "hello".toUpperCase() 或自己写 capitalize。

8. Conditional + Mapped + Template 三剑客组合

三者的真实威力在组合。一个真实例子：从对象类型自动派生出"事件总线 emitter"：

typescript 
// 输入：事件 → payload 映射表
type EventMap = {
  click: { x: number; y: number };
  submit: { values: Record<string, string> };
  reset: void;
};

// 输出：emitter 接口
//   { onClick: (h: (p: { x; y }) => void) => void
//     onSubmit: ...
//     onReset: ... }
type Emitter<T> = {
  [K in keyof T as `on${Capitalize<string & K>}`]: (handler: (payload: T[K]) => void) => void;
};

type AppEmitter = Emitter<EventMap>;
//   ^? { onClick: (handler: (payload: { x: number; y: number }) => void) => void;
//        onSubmit: ...
//        onReset: ... }

03 应用场景

场景 A · 表单字段类型自动派生

typescript 
interface User {
  name: string;
  email: string;
  age: number;
}

// 给每个字段加一个 "Error" 后缀
type FormErrors<T> = {
  [K in keyof T as `${string & K}Error`]?: string;
};

type UserFormErrors = FormErrors<User>;
//   ^? { nameError?: string; emailError?: string; ageError?: string }

// 或：value 和 error 共存的 form state
type FormState<T> = {
  [K in keyof T]: { value: T[K]; error?: string; touched: boolean };
};

type UserForm = FormState<User>;
//   ^? { name: { value: string; error?: string; touched: boolean };
//        email: ...; age: ... }

场景 B · 嵌套对象的扁平 key 字符串（i18n）

typescript 
// 把嵌套对象的 key 路径展平为 "a.b.c" 字符串 union
type FlatKeys<T, Prefix extends string = ""> = {
  [K in keyof T & string]: T[K] extends object
    ? FlatKeys<T[K], `${Prefix}${K}.`>
    : `${Prefix}${K}`;
}[keyof T & string];

const messages = {
  hello: "Hello",
  user: {
    profile: "Profile",
    settings: {
      theme: "Theme",
      language: "Language",
    },
  },
} as const;

type Key = FlatKeys<typeof messages>;
//   ^? "hello" | "user.profile" | "user.settings.theme" | "user.settings.language"

function t(key: Key): string { /* ... */ return "" }
t("user.settings.theme");   // ✓
t("user.typo");              // ❌

这是 react-i18next / next-intl 等库提供的"类型化 key"基础。配合 09 章 conditional types 递归。

场景 C · 路由参数对象类型

typescript 
// 从路由模板抽出参数 union
type ExtractParams<T extends string> =
  T extends `${string}/:${infer P}/${infer Rest}` ? P | ExtractParams<`/${Rest}`> :
  T extends `${string}/:${infer P}` ? P :
  never;

// 用 mapped type 把 union 变成 { param1: string, param2: string } 对象
type RouteParams<T extends string> = {
  [K in ExtractParams<T>]: string;
};

type P1 = RouteParams<"/users/:userId/posts/:postId">;
//   ^? { userId: string; postId: string }

function navigate<T extends string>(path: T, params: RouteParams<T>) {}
navigate("/users/:userId", { userId: "u_1" });   // ✓
navigate("/users/:userId", {});                    // ❌

场景 D · Tailwind 风格的样式生成（约束规模）

typescript 
// ⚠️ 注意规模 — 这种穷举只对小集合可行
type Size = "sm" | "md" | "lg";
type Color = "primary" | "secondary";

type SizeClass = `text-${Size}`;
//   ^? "text-sm" | "text-md" | "text-lg"

type ColorClass = `bg-${Color}` | `text-${Color}`;
//   ^? "bg-primary" | "bg-secondary" | "text-primary" | "text-secondary"

// 大集合时：用 string 占位符 + 运行时校验
type AnyButtonClass = `btn-${string}`;   // 接受任意 btn- 开头字符串
function button(cls: AnyButtonClass) { /* ... */ }
button("btn-foo");   // ✓
button("primary");   // ❌

04 典型代码

所有内置 utility 都是 mapped type

typescript · lib.es5.d.ts 
// 全部就是这么简单
type Partial<T> = { [P in keyof T]?: T[P] };
type Required<T> = { [P in keyof T]-?: T[P] };
type Readonly<T> = { readonly [P in keyof T]: T[P] };
type Pick<T, K extends keyof T> = { [P in K]: T[P] };
type Record<K extends keyof any, T> = { [P in K]: T };

DeepReadonly / DeepPartial（递归 mapped type）

typescript 
type DeepReadonly<T> = T extends (infer U)[]
  ? ReadonlyArray<DeepReadonly<U>>
  : T extends object
    ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
    : T;

interface Config {
  api: { url: string; timeout: number };
  features: string[];
}

type Frozen = DeepReadonly<Config>;
const c: Frozen = { api: { url: "x", timeout: 100 }, features: ["a"] };
// c.api.url = "y";    // ❌
// c.features.push();  // ❌

// DeepPartial 同理
type DeepPartial<T> = T extends object
  ? { [K in keyof T]?: DeepPartial<T[K]> }
  : T;

过滤字段：只保留 / 排除某种类型的字段

typescript 
// 过滤掉 method 字段
type DataFields<T> = {
  [K in keyof T as T[K] extends Function ? never : K]: T[K];
};

class User {
  name = "";
  age = 0;
  greet() { return `Hi ${this.name}` }
}
type UserData = DataFields<User>;
//   ^? { name: string; age: number }   greet 被过滤

// 反过来：只保留 method
type Methods<T> = {
  [K in keyof T as T[K] extends Function ? K : never]: T[K];
};
type UserMethods = Methods<User>;
//   ^? { greet: () => string }

// 提取所有 readonly 字段（用 09 章的 Equal 高级技巧）
type WritableKeys<T> = {
  [K in keyof T]-?: (<U>() => U extends { [P in K]: T[K] } ? 1 : 2) extends
    (<U>() => U extends { -readonly [P in K]: T[K] } ? 1 : 2)
    ? K
    : never
}[keyof T];

把 union 转 intersection（高级 trick）

typescript 
type UnionToIntersection<U> =
  (U extends any ? (x: U) => void : never) extends ((x: infer I) => void) ? I : never;

type R = UnionToIntersection<{ a: 1 } | { b: 2 }>;
//   ^? { a: 1 } & { b: 2 }

// 原理：
// 1. distributive：每个成员变成 (x: T) => void
// 2. 用 infer 在"参数位置"推断 — 函数参数是逆变（17 章），union 在逆变位置变 intersection

Template literal 配 infer 解析字符串

typescript 
// 把 "user_name" 转成 "userName"
type SnakeToCamel<S extends string> =
  S extends `${infer Head}_${infer Tail}`
    ? `${Head}${Capitalize<SnakeToCamel<Tail>>}`
    : S;

type R1 = SnakeToCamel<"hello_world_foo">;   // "helloWorldFoo"

// 把对象的所有 snake_case key 转成 camelCase
type CamelCaseKeys<T> = {
  [K in keyof T as SnakeToCamel<string & K>]: T[K];
};

interface ApiResponse {
  user_id: string;
  user_name: string;
  created_at: number;
}
type R2 = CamelCaseKeys<ApiResponse>;
//   ^? { userId: string; userName: string; createdAt: number }

所有 modifier 组合一览

typescript 
interface Original {
  readonly id: string;
  name?: string;
  age: number;
}

// 全部 readonly + optional
type Full<T> = { readonly [K in keyof T]?: T[K] };
type R1 = Full<Original>;
//   ^? { readonly id?: string; readonly name?: string; readonly age?: number }

// 去掉所有 readonly + optional
type Bare<T> = { -readonly [K in keyof T]-?: T[K] };
type R2 = Bare<Original>;
//   ^? { id: string; name: string; age: number }

// 仅去 readonly
type Mut<T> = { -readonly [K in keyof T]: T[K] };

// 仅去 optional
type Req<T> = { [K in keyof T]-?: T[K] };

// 添加新 optional 同时保留 readonly
type Opt<T> = { [K in keyof T]+?: T[K] };

05 常见陷阱

陷阱 1 · 忘了减号 readonly / optional 默认带过
typescript 
interface User {
  readonly id: string;
  name?: string;
}

// ❌ 想"重新映射"，结果 readonly 和 ? 都被保留
type Mapped<T> = { [K in keyof T]: T[K] };
type R = Mapped<User>;
//   ^? { readonly id: string; name?: string }

// ✅ 显式去掉
type Bare<T> = { -readonly [K in keyof T]-?: T[K] };
type R2 = Bare<User>;
//   ^? { id: string; name: string }

规避：写 mapped type 时明确想保留还是去掉修饰符。"重新映射"的语义是模糊的，要么 + 要么 -。

陷阱 2 · key remapping 时 K 的类型范围
typescript 
// keyof T 实际是 string | number | symbol
type Bad<T> = { [K in keyof T as `prefix_${K}`]: T[K] };
//                                          ^   Type 'K' is not assignable to 'string'...

// 解法 1：string & K 收窄
type Good1<T> = { [K in keyof T as `prefix_${string & K}`]: T[K] };

// 解法 2：约束 T 的 key 必须是 string
type Good2<T extends Record<string, unknown>> = { [K in keyof T as `prefix_${K}`]: T[K] };

// 解法 3：在 keyof 后过滤
type Good3<T> = { [K in keyof T as K extends string ? `prefix_${K}` : never]: T[K] };
陷阱 3 · 大 union 笛卡尔积导致编译爆炸
typescript 
// ❌ 这个会让编译器吃力或直接拒绝
type C = "red" | "blue" | "green" | "yellow" | "purple" | "pink" | "orange" | "brown";
type S = 50 | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900;
type Class = `bg-${C}-${S}`;
//   ^? 80 个组合 — 还能展开

type V = `${"hover" | "focus" | "active" | "disabled"}:${Class}`;
//   ^? 320 个组合 — 开始慢

type R = `${"sm" | "md" | "lg"}:${V}`;
//   ^? 960 个组合 — 编译器抗议

规避：组合数 > 几百时，把外层换成 ${string} 占位（不展开），用运行时 + IDE 完成提示弥补。Tailwind 实际类型用 string 不穷举。

陷阱 4 · 过滤字段时忘了 [keyof T]
typescript 
// 想：取出所有 string 字段的 key union
// ❌ 错的（少了 [keyof T]）
type StringKeys1<T> = {
  [K in keyof T]: T[K] extends string ? K : never;
};
// 类型实际是：{ name: "name"; age: never; email: "email" }
// 不是 union！

// ✅ 加 [keyof T] 取所有 value 的 union
type StringKeys2<T> = {
  [K in keyof T]: T[K] extends string ? K : never;
}[keyof T];

interface U { name: string; age: number; email: string }
type R = StringKeys2<U>;   // "name" | "email"

记忆口诀："写一个 mapped 选 key，最后加 [keyof T] 取出 union"。

陷阱 5 · 递归 mapped type 没有 base case 导致无限
typescript 
// ❌ 没有 base case — 编译器一直递归到深度上限
type DeepReadonly<T> = { readonly [K in keyof T]: DeepReadonly<T[K]> };

type R = DeepReadonly<{ a: number }>;
//   ^? { readonly a: { readonly toString: { readonly toString: ... } } }
//      把所有 string 的 prototype method 也递归了

// ✅ 加 base case：原始类型不递归
type DeepReadonly<T> = T extends object
  ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
  : T;

规避：递归 mapped type 必须有 conditional 兜底，区分"object 才递归" vs "原始类型直接返回"。

陷阱 6 · Capitalize / Uppercase 没运行时对应

这四个内置类型只活在编译期：

typescript 
type R = Capitalize<"hello">;   // "Hello"

// 运行时仍要自己写
function capitalize<S extends string>(s: S): Capitalize<S> {
  return (s.charAt(0).toUpperCase() + s.slice(1)) as Capitalize<S>;
}

const r = capitalize("hello");
//    ^? "Hello"
