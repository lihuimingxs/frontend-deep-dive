Part 5 · 工程化

前 17 章是 TS 语言本身。本 Part 是把它落地到真实工程的最后一公里：怎么写 .d.ts 给老 JS 库加类型、怎么配 tsconfig（30+ 选项怎么挑）、怎么把现有 JS 项目渐进迁移、怎么处理 CJS / ESM 互通的痛点。这是和"产品"距离最近的一 Part —— 错配一个选项可能让整个项目编译爆炸。

章节数 3 章 对应官方 Handbook · Project Configuration / tsconfig 参考 前置 13 Modules

本 Part 解决什么问题

  • 怎么给一个无类型的 JS 库加类型 / 怎么给 svg 文件 import 加类型 —— 18 Declaration Files
  • tsconfig 30+ 选项怎么挑、strict 包含什么、Project References 是什么 —— 19 项目配置
  • 怎么把老 JS 项目分阶段迁移、怎么处理 CJS/ESM 双世界 —— 20 Migration & Interop

章节清单

序号章节核心收获
18 Declaration Files .d.ts 不输出值、ambient declaration、module augmentation 实战、通配 module 给资源加类型、npm 包发布带类型
19 项目配置 strict 13 个子 flag、module vs moduleResolution、bundler 模式、Project References、incremental build
20 Migration & Interop JS → TS 渐进、allowJs + checkJs、@ts-expect-error、CJS/ESM 互通深坑、信任边界与 zod、TS 5.x 新工程特性

难度提示

  • 18 Declaration Files — 中等。一旦掌握，给任何无类型库加类型这件事永久解锁
  • 19 项目配置 — 中等偏高。选项多，但每条规则都直接对应工程问题
  • 20 Migration & Interop — 综合实战。整合前 19 章 + 大量工具链知识

本 Part 是 TS 学习的终点

完成本 Part 后，你已经掌握 TS 的全部知识：

  • 语言本身：类型系统、type narrow、函数 / 对象 / class
  • 类型操作：泛型、conditional、mapped、template literal、utility
  • 引用：modules、decorators、JSX、type compatibility 元层
  • 工程化：声明文件、tsconfig、迁移与互通

后续推荐：精读 node_modules/typescript/lib/lib.es5.d.ts 巩固 utility types；接触 type-challenges 巩固类型操作；精读 zod / Drizzle / TanStack Router 等库源码看高级用法。
