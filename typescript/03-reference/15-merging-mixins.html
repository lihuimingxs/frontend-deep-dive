15 · Declaration Merging / Mixins / Symbols

三个独立但都"小而精"的话题，合并成一章。Declaration merging 是 TS 给"全局类型扩展"和"第三方库 patch"留的逃生口；Mixin 是"组合优于继承"的 TS 实现；Symbol 把"运行时唯一性"提升到类型层（unique symbol）。三者表面无关，但都有一个共同主题：给已有结构"打补丁"或"扩展"，而不修改原代码。

阅读时长 ~32 分钟 前置 13 Modules 对应 typescriptlang.org Declaration Merging + Mixins + Symbols

01 速览

三个主题的最小例子：

typescript · declaration merging 
// 同名 interface 自动合并
interface User { name: string }
interface User { age: number }
// 实际：interface User { name: string; age: number }

// 给第三方 module 加字段（module augmentation）
declare module "express-serve-static-core" {
  interface Request {
    user?: { id: string };
  }
}
typescript · mixin 
// 函数式 class 组合
type Constructor<T = {}> = new (...args: any[]) => T;

function Timestamped<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    timestamp = Date.now();
  };
}

class User { name = "" }
const TimestampedUser = Timestamped(User);
const u = new TimestampedUser();
u.name; u.timestamp;   // ✓ 同时拥有
typescript · Symbol 
// 普通 Symbol — 运行时唯一
const s1 = Symbol("foo");
const s2 = Symbol("foo");
s1 === s2;   // false

// unique symbol — 类型层也唯一
const k1: unique symbol = Symbol("k1");
const k2: unique symbol = Symbol("k2");
type T1 = typeof k1;   // typeof k1（独立类型）
type T2 = typeof k2;
type Eq = T1 extends T2 ? true : false;   // false

02 底层逻辑

1. Declaration merging 的 4 种形态

TS 中有 4 种"同名声明合并"规则：

声明类型合并规则
interface + interface 所有字段合并；同名不同类型字段会冲突报错
namespace + namespace 所有 export 成员合并到同一命名空间
namespace + class / function / enum 给 class / function / enum 附加额外成员
module augmentation 跨文件给已有 module 加字段（最常用）
typescript · 全部形态 
// 1. interface 合并
interface Box { width: number }
interface Box { height: number }
// 等价 interface Box { width: number; height: number }

// 2. namespace 合并
namespace MyApp { export const x = 1 }
namespace MyApp { export const y = 2 }
MyApp.x; MyApp.y;   // ✓ 同一 namespace

// 3. namespace + function（给函数加 static 字段）
function f() { return f.count++ }
namespace f { export let count = 0 }
f.count;             // ✓
f();                  // ✓
// 这是 Chai / lodash 等老库的常见模式

// 4. namespace + class（给 class 加内嵌类型）
class Cat { name = "" }
namespace Cat {
  export type Breed = "siamese" | "persian";
}
const c = new Cat();
const b: Cat.Breed = "siamese";   // ✓ Cat.Breed 是类型

2. Module augmentation —— 工程项目最常用形态

"给一个已存在的 module 加字段，但不修改它的源代码" —— 这是 declaration merging 在工程中最高频的使用方式：

typescript · types/express.d.ts 
// 让 TS 信任 Express Request 上有 req.user 字段
// （由 auth middleware 在运行时注入）
declare module "express-serve-static-core" {
  interface Request {
    user?: { id: string; email: string };
    requestId: string;
  }
}

// 这个文件里只能有 declare module {...} 或者其他 declarations
// 不能有 import / export（否则不会全局生效）
// — 但这里有 declare module，TS 自动识别为全局扩展声明文件

关键约束：augmentation 文件必须是 module（含 import 或 export）但 augmentation 内部的 module name 必须匹配原 module。

常见的 module augmentation 场景：

  • 给 Express / Fastify 加 req.user
  • 给 next-auth Session 加业务字段
  • 给 styled-components / emotion Theme 提供精确类型
  • 给 React Router useLocation state 加类型
  • 给 vue 的 Options API computed 加方法

3. Global augmentation：扩展全局类型

扩展 Window / NodeJS.ProcessEnv / globalThis 等全局对象需要 declare global：

typescript 
// 给 Window 加字段
declare global {
  interface Window {
    __APP_VERSION__: string;
    gtag?: (...args: any[]) => void;
  }
}

// 给 process.env 加字段（让 process.env.X 类型化）
declare global {
  namespace NodeJS {
    interface ProcessEnv {
      DATABASE_URL: string;
      NODE_ENV: "development" | "production" | "test";
    }
  }
}

// 必须有 export 让此文件成为模块
export {};

注意 declare global 的两个细节：

  • 必须在模块文件内（即文件含 import 或 export）
  • 内部用 interface 不用 type（type 不能 declaration merge）

4. Mixin pattern：函数式 class 组合

OOP 中"组合优于继承"的实现。Mixin 是接受 class 返回 class 的工厂函数。TS 提供专门的语法支持：

typescript 
// 1. 定义 Constructor 类型 helper
type Constructor<T = {}> = new (...args: any[]) => T;

// 2. Mixin = 接受 Base 类，返回扩展类的工厂
function Serializable<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    serialize(): string {
      return JSON.stringify(this);
    }
  };
}

function Loggable<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    log(): void {
      console.log(this.constructor.name, this);
    }
  };
}

// 3. 应用多个 mixin
class User {
  constructor(public name: string) {}
}

const FullUser = Loggable(Serializable(User));
const u = new FullUser("Alice");

u.name;        // ✓ 来自 User
u.serialize(); // ✓ 来自 Serializable
u.log();       // ✓ 来自 Loggable

5. Constrained mixin：要求 Base 满足某形状

mixin 通常需要访问 Base 的某些字段。约束 Base 类型让 mixin 内可安全使用：

typescript 
// 要求 Base 必须有 name 字段
type Named = Constructor<{ name: string }>;

function Greetable<TBase extends Named>(Base: TBase) {
  return class extends Base {
    greet(): string {
      return `Hello, ${this.name}!`;   // ✓ this.name 可见
    }
  };
}

class User { constructor(public name: string) {} }
const Greeter = Greetable(User);
new Greeter("Alice").greet();   // "Hello, Alice!"

class Anonymous {}
// const Bad = Greetable(Anonymous);   // ❌ Anonymous 没 name 字段

6. Symbol：运行时唯一性

Symbol() 创建运行时唯一值。每次调用都创建新的 symbol，即使描述相同：

typescript 
const a = Symbol("foo");
const b = Symbol("foo");
a === b;   // false — 每次调用都是新 symbol

// 全局共享：Symbol.for(key) 在全局 registry 查
const c = Symbol.for("shared");
const d = Symbol.for("shared");
c === d;   // true

// Symbol 作为对象 key（避免与 string key 冲突）
const TYPE = Symbol("type");
const obj = { [TYPE]: "user", name: "Alice" };

// JSON.stringify 不会序列化 symbol key
JSON.stringify(obj);   // {"name":"Alice"}

7. unique symbol：把唯一性提到类型层

unique symbol 是类型，不是值。它表示"这个具体的 symbol 值"，即使另一个 symbol 在运行时不同，类型层也不可赋值：

typescript 
// unique symbol 只能给 const 或 readonly static
const FOO: unique symbol = Symbol("FOO");
const BAR: unique symbol = Symbol("BAR");

// FOO 和 BAR 都是 symbol，但类型层是不同类型
type FooType = typeof FOO;   // typeof FOO（不是 symbol）
type BarType = typeof BAR;

const f: FooType = FOO;       // ✓
const f2: FooType = BAR;      // ❌ Type 'typeof BAR' not assignable to 'typeof FOO'

// 配合 indexed access 创建结构层 nominal 区分
interface ApiKey { [FOO]: string }
interface SecretKey { [BAR]: string }

function useApi(k: ApiKey) {}
useApi({ [FOO]: "abc" });    // ✓
useApi({ [BAR]: "abc" });    // ❌ — BAR symbol 不匹配 FOO key

应用：redux-toolkit 用 unique symbol 区分不同 action type（运行时是不同 symbol，类型层不可混用）。

8. Well-known symbols：JS 协议钩子

ES 标准定义了一组 Symbol.xxx，每个都是某个协议的钩子：

typescript 
// Symbol.iterator — 让对象可被 for...of
class Range {
  constructor(public start: number, public end: number) {}
  *[Symbol.iterator]() {
    for (let i = this.start; i <= this.end; i++) yield i;
  }
}
for (const n of new Range(1, 3)) console.log(n);   // 1 2 3

// Symbol.asyncIterator — for await...of
class Stream {
  async *[Symbol.asyncIterator]() {
    yield 1; yield 2; yield 3;
  }
}
for await (const n of new Stream()) console.log(n);

// Symbol.toPrimitive — 类型转换控制
class Money {
  constructor(public amount: number) {}
  [Symbol.toPrimitive](hint: "number" | "string" | "default") {
    if (hint === "number") return this.amount;
    return `$${this.amount}`;
  }
}
+new Money(10);            // 10
`${new Money(10)}`;        // "$10"

// 还有 Symbol.toStringTag / Symbol.hasInstance / Symbol.isConcatSpreadable / ...
// 共 15+ 个 well-known symbols（详见 JS 章节 11-symbol）

03 应用场景

场景 A · Express + auth middleware 类型扩展

typescript · types/express.d.ts 
declare module "express-serve-static-core" {
  interface Request {
    user?: { id: string; email: string; role: "admin" | "user" };
    requestId: string;
    startedAt: number;
  }
}
typescript · 在 handler 中使用 
import { Request, Response } from "express";

function authMiddleware(req: Request, res: Response, next: NextFunction) {
  const token = req.headers.authorization;
  req.user = decodeToken(token);    // ✓ TS 已知 user 字段
  next();
}

app.get("/me", (req, res) => {
  if (!req.user) return res.status(401).end();
  res.json({ id: req.user.id, email: req.user.email });   // ✓ 类型化
});

场景 B · 给 Window / process.env 加全局类型

typescript · src/global.d.ts 
declare global {
  // 浏览器：Window 上的全局变量（webpack DefinePlugin 注入的）
  interface Window {
    __APP_VERSION__: string;
    __FEATURE_FLAGS__: Record<string, boolean>;
  }

  // Node：process.env 类型化
  namespace NodeJS {
    interface ProcessEnv {
      DATABASE_URL: string;
      JWT_SECRET: string;
      NODE_ENV: "development" | "production" | "test";
      PORT?: string;
    }
  }
}

export {};   // 必须 — 让此文件是模块

⚠️ 这是 hint 不是运行时担保。process.env.DATABASE_URL 仍可能是 undefined（如果环境变量没设）。生产代码用 zod 校验 process.env：

typescript 
const env = z.object({
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  NODE_ENV: z.enum(["development", "production", "test"]),
}).parse(process.env);

// env.DATABASE_URL 类型 + 运行时都担保是 string

场景 C · Mixin 应用：游戏开发的能力组合

typescript 
type Constructor<T = {}> = new (...args: any[]) => T;

function Movable<TBase extends Constructor<{ x: number; y: number }>>(Base: TBase) {
  return class extends Base {
    move(dx: number, dy: number) { this.x += dx; this.y += dy; }
  };
}

function Drawable<TBase extends Constructor<{ x: number; y: number }>>(Base: TBase) {
  return class extends Base {
    draw(ctx: CanvasRenderingContext2D) {
      ctx.fillRect(this.x, this.y, 10, 10);
    }
  };
}

function HpBar<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    hp = 100;
    takeDamage(d: number) { this.hp = Math.max(0, this.hp - d); }
  };
}

class Entity {
  constructor(public x: number, public y: number) {}
}

const Player = HpBar(Drawable(Movable(Entity)));
const p = new Player(0, 0);
p.move(10, 20);
p.takeDamage(5);
p.draw(ctx);
p.hp;            // 95

场景 D · Symbol 做 action type / 内部 key

typescript 
// Redux 风格的 unique action type
const INCREMENT: unique symbol = Symbol("INCREMENT");
const DECREMENT: unique symbol = Symbol("DECREMENT");

type Action =
  | { type: typeof INCREMENT; amount: number }
  | { type: typeof DECREMENT; amount: number };

function reducer(state: number, action: Action): number {
  switch (action.type) {
    case INCREMENT: return state + action.amount;   // ✓ narrow
    case DECREMENT: return state - action.amount;
  }
}

reducer(0, { type: INCREMENT, amount: 5 });   // ✓
// 不同库的 symbol 永远不会冲突 — 即使两个库都用了 "INCREMENT" 描述

// Symbol 做对象的"内部" key，不影响 JSON
const META = Symbol("metadata");
const obj: { name: string; [META]?: { version: number } } = { name: "x" };
obj[META] = { version: 1 };

JSON.stringify(obj);   // {"name":"x"}  symbol key 不出现

04 典型代码

给 styled-components 提供精确 Theme 类型

typescript · src/types/styled.d.ts 
import "styled-components";

// 自定义 theme
const theme = {
  colors: {
    primary: "#3178c6",
    secondary: "#f5f5f5",
  },
  spacing: {
    sm: 8,
    md: 16,
    lg: 32,
  },
} as const;

// 让 styled-components 知道 theme 形状
declare module "styled-components" {
  export interface DefaultTheme {
    colors: typeof theme.colors;
    spacing: typeof theme.spacing;
  }
}

// 现在所有 styled.div 的 props.theme 都精确：
const Box = styled.div`
  color: ${p => p.theme.colors.primary};   // ✓ 自动补全
`;

函数 + namespace 合并：lodash 风格

typescript 
// 主函数 + 静态方法
function fetch(url: string): Promise<Response> { /* ... */ return undefined as any }

namespace fetch {
  export const VERSION = "1.0";
  export function withTimeout(url: string, ms: number) { /* ... */ }
  export type Options = { headers?: Record<string, string> };
}

fetch("/api");                    // ✓
fetch.VERSION;                     // ✓
fetch.withTimeout("/api", 5000);   // ✓
const opts: fetch.Options = {};    // ✓ 类型用法

Mixin + interface declaration merging（类型补丁）

typescript 
// 一个特殊技巧：mixin 后用 interface 合并补类型
class Animal {
  constructor(public name: string) {}
}

// Mixin 后类型推断有局限，可手动 interface 合并补
function Disposable<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    isDisposed = false;
    dispose() { this.isDisposed = true; }
  };
}

const DisposableAnimal = Disposable(Animal);
interface DisposableAnimal {
  // 这里可以加额外类型补充（如果工厂函数的类型推断不够精确）
  customField?: string;
}

const a: DisposableAnimal = new DisposableAnimal("Cat");
a.customField = "extra";   // ✓

Symbol 做"运行时类型标记"（替代 instanceof）

typescript 
// 跨 realm（iframe / Worker）时 instanceof 失效
// 用 Symbol 做"鸭子类型"标记

const ResultSymbol = Symbol.for("Result");

class Ok<T> {
  readonly [ResultSymbol] = "ok";
  constructor(public value: T) {}
}

class Err<E> {
  readonly [ResultSymbol] = "err";
  constructor(public error: E) {}
}

function isResult(x: unknown): x is Ok<unknown> | Err<unknown> {
  return typeof x === "object" && x !== null && ResultSymbol in x;
}

// 即使从 iframe / Worker 传来的 Result 实例
// instanceof Ok 会失败，但 isResult 仍能识别

05 常见陷阱

陷阱 1 · interface 合并冲突字段静默 / 报错
typescript 
interface Box { x: number }
interface Box { x: string }   // ❌ Subsequent property declarations must have the same type

// 但 method overload 可以共存（合并为重载）
interface Box { fn(x: string): void }
interface Box { fn(x: number): void }
// 实际：Box 有重载 fn(x: string): void; fn(x: number): void

规避：合并扩展只加新字段；不要试图"覆盖"原有字段类型。要改类型只能 fork 库或 type + Pick/Omit 重新定义。

陷阱 2 · type 不能 declaration merge
typescript 
type User = { name: string };
type User = { age: number };   // ❌ Duplicate identifier 'User'

// 必须用 interface
interface User { name: string }
interface User { age: number }   // ✓

规则：库公共类型 / 全局对象类型用 interface（用户可扩展）；内部类型组合 / union 用 type alias。02 章已讲过这个决策树。

陷阱 3 · module augmentation 在错误位置不生效
typescript 
// ❌ 在 .ts 文件内但被当成"全局脚本"（无 import/export）
declare module "express" {
  interface Request { user?: User }
}
// 不生效 — 因为此文件不是模块

// ✅ 加 export 让它是模块
declare module "express" {
  interface Request { user?: User }
}
export {};   // 关键

规避：augmentation 文件结尾加 export {};，或者文件内有任何其他 import / export。

陷阱 4 · Mixin 静态成员丢失类型
typescript 
function Cached<TBase extends Constructor>(Base: TBase) {
  return class extends Base {
    static instances = new Map<string, any>();   // 静态成员
    constructor(...args: any[]) { super(...args) }
  };
}

class User { static count = 0 }

const CachedUser = Cached(User);
CachedUser.instances;   // ✓ 来自 mixin
CachedUser.count;        // ✓ 来自 User
// 但...IDE 静态成员补全可能不全（取决于 TS 版本和具体推断）

规避：mixin 不设计 static 成员；或者用 declaration merging 补充类型：

typescript 
interface CachedUser {
  // 实例字段类型补充
}
namespace CachedUser {
  // 静态字段类型补充
}
陷阱 5 · Symbol 描述相同但运行时不同
typescript 
const a = Symbol("foo");
const b = Symbol("foo");
a === b;          // false ⚠️ 多人误以为是 true

// 要全局共享：Symbol.for
const c = Symbol.for("foo");
const d = Symbol.for("foo");
c === d;          // ✓ true

// Symbol 与 Symbol.for 的差异
//  - Symbol("x")     总是新 symbol，不可全局查找
//  - Symbol.for("x") 全局 registry，同 key 返回同一个
陷阱 6 · unique symbol 必须 const 或 readonly static
typescript 
// ❌ let / var 不行
let s: unique symbol = Symbol();   // A variable whose type is a 'unique symbol' must be 'const'

// ✅ const
const s: unique symbol = Symbol();

// ✅ readonly static
class C {
  static readonly KEY: unique symbol = Symbol("KEY");
}
type CKey = typeof C.KEY;   // 独立类型

原因：unique symbol 的类型是"这个具体 symbol 值"。如果变量可重新赋值，类型就不再唯一。
